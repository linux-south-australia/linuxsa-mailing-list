Re: 486 8MB + Linux

8M of memory will be crippling and old laptop hd are slow for swapspace.
Try and find some second hand RAM and do an upgrade. I think 16M would 
be an absolute minimum but more would be good.

Slackware would be a good choice for a dist. You would be struggling to 
run much more than a few shells and vi under 8M. Apache+PHP+Mysql are 
not going to run, more like boa or thttpd.

You could try an embedded/router dist. They usually put a lot of effort 
into stripping down the kernal, compiling against a smaller libc etc. 
The trouble is they also tend to replace /bin with busybox, vi with ae 
and bash with ash. You end up giving up so much to run in low memory it 
takes all the fun away.
