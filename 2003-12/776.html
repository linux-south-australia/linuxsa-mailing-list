Ultimately, a code management system is a personal choice. However, there 
are a number of different systems out there. Many people choose to use 
CVS, however, quite a lot of companies seem to use Perforce. I know that 
we run an enormous number of branches, and we sling code around from one 
branch to another with very few ill affects.

However, I am not suggesting you use perforce or indeed anything other 
than CVS. The only reason I mention perforce is to demonstrate that there 
are other needs out there that are not handled well by CVS.

However, the cost of not using CVS for a small project is that you cannot 
leverage off of the help and experience that is out there.
