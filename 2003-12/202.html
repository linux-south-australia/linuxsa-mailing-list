Re: ADSL -- class C and Routing

Quoting Luke Vanderfluit <luke@chipcity.com.au>:

> Is it possible to route a Class C address range to the consumer end of
> an ADSL connection? What would that cost? 

It is possible, but with all of the background noise on the Internet these days,
it's not very practical for a DSL connection. For every IP you have routed to
your connection, expect about 12k per hour of scans, pings, and worm nuisance.
Over a class C with 256 addresses, that's over 3 megabytes, every hour. 

Since a worthwhile number of end users have broadband these days, any web sites
you host will have to share that 512K. This is fine for a few sites, but if a
whole class C is really required it's going to be slow.

16 IP's (a /28) is a more reasonable request for a DSL connection. By the time
you have that many servers (or SSL sites) it will probably be cost-effective to
colocate a server at an ISP instead.
