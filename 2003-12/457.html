Re: installing linux on a M$ partition?

On Sat, 13 Dec 2003 9:03 pm, davidn@rebel.net.au wrote:
> Damien Uern <carrigan_2606@optusnet.com.au> wrote:
> > It may be possible because the linux NTFS driver supports writing safely
> > to files on an NTFS partition as long as the size of the file isn't
> > changed.
>
> Which, if I may translate, means, "No, you can't run Linux on NTFS."

How so? Let's say you make a 2GB datafile in Windows on your NTFS partition. 
You then boot into a distro's installer that supports such a setup. You tell 
it the name and location of the file that you want to install Linux into. It 
formats it as ext2 or ext3 or whatever, and installs Linux onto it. All of 
this can be done without modifying the size of the file. Setups like this 
have existed for FAT partitions for a long time, now it (should be / is) 
available for NTFS partitions (except for actually creating the file).

To make installation easy, Linux could be installed under Windows using some 
sort of InstallShield or WISE installer. And it could stick an entry in 
C:\boot.ini to boot Linux.

This may be of help:

http://www.linux.org/docs/ldp/howto/Loopback-Root-FS.html
