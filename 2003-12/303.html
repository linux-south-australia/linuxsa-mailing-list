Re: We got mentioned in parliament

James Leone <linuxcpa@netscape.net> wrote:
>davidn@rebel.net.au wrote:
>> Normally it would be servers that need configuring, such as Postgres,
>> Sendmail and Lpd. 
> These are useful and a PITA to learn to configure.

That's odd, because sendmail, postgres and lpd are all so well documented
that they are easy to configure.  That's assuming you read documentation,
but I suspect that when you say, "easy", you mean, "can be used without
documentation."  I've already stated my opposition to that principle.


> If the installer created /etc/*.conf files, what's the beef, you can 
> still edit them.
> You will still have the source code to automate the installation. What's 
> there to loose
> but making Unix easier for the novice. I don't see a point here.

I don't object to whizzy installers, I object to sub-standard, or even
missing, documentation.  I have no objection to a programming coming
with no installer, but I do object to it coming with no documentation.


>> Those black boxes are based on, 'let us do the hard work,
>> you don't need to know,' which is ultimately untrue.  And when you do
>> need to know you discover that the black box is really a wall which you
>> cannot see beyond.
> My guess is that the wall comes as a result of the software being closed 
> source.

When you first, last and only view of configuration comes from a configure
program, and allowing that they generally only provide access to the
'commonly used options,' then you conclude that if you can't set it using
the wizard it can't be done.  There's a wealth of power under the hood,
but you don't get to use it if you don't lift the bonnet.
