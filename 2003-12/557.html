Re: Passwords

At 2003/12/16 10:24+1030  David Lloyd writes:
> 
> Hmmm...
> 
> > So wherever possible, turn off password logins and only only allow ssh
> > key based authorisation.
> 
> What makes you think that people won't use the same ssh key for:
> 
>  * sourceforge
>  * blahforge
>  * fooforge
> 
> ...and they may not even have passwords setup for them...

If sourceforge gets broken into and all the passwords logged then they
also get access to blahforge and fooforge. If only ssh-key logins are
allowed then they can't get logins into blahforge and fooforge.

Assuming users don't use ssh-agent forwarding - which you should only
do to highly trusted machines anyway, and never by default.

Of course if the machine that the ssh key is sitting on is compromised
then they get access to all the machines you use, but they would have
anyway. it should be a bit easier to keep the workstation/laptop more
secure as they can be kept behind firewalls and need to run less
services, and these days you can easily be a bit more paranoid and
keep your ssh keys on a usb key.
