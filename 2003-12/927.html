No we aren't.  Or if we are, it isn't what you're getting at.

Hint:  SCO complained about ABIs.  Your post mentions this, then goes
off on a tangent about APIs.  ABI != API.

APIs are the documented programming interfaces used to write applications;
The stuff that shows up in sections 2 and 3 of the manpages.

ABIs are different;  They're the combination of executable file formats,
system call invocation methodology, stack frame layout, VM map layout,
register conventions, manifest constant values, and other things which
enable an OS to run a binary executable.

The POSIX spec describes an API.  ABIs are things like COFF, IBCS2, etc.
