Re: A variation of Copy Command

Magpie,

> I need to copy files form a dodgy hard drive. My last effort resulted in
> partial success. A lot of files were copied before it hung. But a new
> copy command will start all over again. Is their some way to copy so
> that this does not occur ? i.e. like wget. It would start form where it
> left off.

*hmmm*

I'm not sure how you could do this with the copy command, but "tar" has
incremental semantics. That is to say, tar can copy stuff but only stuff
that hasn't changed since the last run of tar.

How would I do this:

 1. Run the tar over the network
 2. Put the tar directory to store the info about the incremental backups
    on a disk not on the faulty drive

man tar

(look for the -g command)

You could also use rsync however I'm not quite sure whether tar or rsync
would be make more intensive disk accesses.
