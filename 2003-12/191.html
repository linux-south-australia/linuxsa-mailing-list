Re: Error compiling 2.4.18-26.7.x kernel

On Fri, 2003-12-05 at 16:41, markthurston@ihug.com.au wrote:
> I am trying to compile a kernel. The source I am using is Redhat 7.3.
> The kernel version is 2.4.18-26.7.x
> I am using the source from /usr/src/linux-2.4.18-26.7.x. (copied to 
> a /home/user directory)
> The configuration file I'm using is kernel-2.4.18-i686.config 
> from /usr/src/linux-2.4.18-26.7.x/configs.
> I copied the file to my /home/user directory and renamed it to .config
> The processor on the machine is an Intel Xeon (single cpu).
> 
> I then ran:
> 
> make oldconfig
> make dep
> make bzImage

I had a similar issue using about that vintage of Redhat kernel some
months back and I finally discovered that for some reason, I needed to
perform a "make mrproper" prior to the 3 make commands you have
specified above.  I've never had to do a make mrproper on freshly
installed Redhat source before, but on this occasion I did (I can't
remember the exact version).

Just note of course that 'make mrproper' will destroy your .config file,
so you'll need to re-copy the config after the 'make mrproper' command.

Hope that helps.
