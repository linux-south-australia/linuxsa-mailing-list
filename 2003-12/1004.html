> used or why this method should not be used, or maybe it is used alot but I
> just haven't seen it that much?

Yeah, it's a useful thing to use but:

- it means you HAVE to be root to run things in /etc/inittab (ie. to edit 
the file, kill -1 1 to reread inittab etc
- you have to edit a relatively critical file to change how it starts
- it's harder to integrate into a package system (eg. look how easy it is to 
drop in files into the rcX.d hierarchy and then remove them
- inittab tends to be more useful for very stable tasks which don't need a 
lot of "interaction" from inittab.  If something tries to restart too often 
etc you can lose control of the machine more easily.
- you have to be root or have root access ...
