I'd have to agree.

Documentation is a mostly boring an menial task but it is something that 
*has* to be done.  I've lost count of the number of times recently that 
I've gone to "man" trying to decipher the arcane options on a seldom 
used command only to find that said command either doesn't have a man 
page, or it's so poorly written that extracting any useful information 
from it would take longer than working out how to use the utility by 
trial and error. Not to mention the insistance of some developers to 
forsake "man" in favour of "info".

Since I don't tend to hang out in the KDE/Gnome world much (I *do* so 
like the OSX interface), I have little comment on the "help" systems 
used in those desktop environments. However, on a related note, both 
environments (and maybe KDE more than Gnome) have a serious conistancy 
problem when it comes to where settings are kept. Some are set in the 
Control Panel, some in config files, some in Properties panes under 
right click menus, all very mish-mash and not easily found in a hurry.
