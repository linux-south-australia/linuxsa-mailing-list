I think that this is part of the power of the idea. However, an audit of 
the test env used will be needed. Perhaps users would be better off doing 
initial testing to look for bugs with final test runs being done by 
someone like OSDL with the gear. 

However, it might be possible to do something like SETI at home, with a 
dedicated bunch or 1,000, or 10,000, or 100,000 etc people loading the 
same software and partitioning the test set and each running a small 
portion of the test suite. If it is accompanied by some un-modifiable 
certificate of the suite of software that had on the machine under test, 
this could be a powerful way of testing.
