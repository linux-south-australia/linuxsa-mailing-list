Re: GNU Grep 2.5.1 speedup patches follow...

At 2003/12/16 22:03+1030  behoffski writes:
> My hope is that by posting these patches to this list, the
> wider programming community can help make the patches less
> risky for the GNU Grep maintainers to adopt.  The main hope
> is to establish a record of good performance and good
> reliability, partially by having the code operate in a wide
> range of systems, and partially by having the changes
> inspected by a wider audience.  I also hope that other
> components of code quality, such as coding style and
> portability, can also be improved by this exposure.

The VSC-lite test suite is freely available from
http://www.opengroup.org/testing/downloads.html.  Its a subset of the
full VSC test suite used for POSIX conformance testing and includes
some tests for grep.

I'd recommend you also check that your patches handle multi-byte
charaters properly too, as this is becoming more important (some
patches are moving into grep for this).

The other thing you can try is to approach some of the maintainers of
the package for the distributions.  Sometimes they will accept patches
not yet in upstream if there is a good enough reason for them to do
so.
