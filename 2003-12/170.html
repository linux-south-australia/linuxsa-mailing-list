Re: A variation of Copy Command

In the immortal words of magpie <cabernet@e-access.com.au>...
> I need to copy files form a dodgy hard drive. My last effort resulted
> in partial success. A lot of files were copied before it hung. But a
> new copy command will start all over again. Is their some way to copy
> so that this does not occur ? i.e. like wget. It would start form
> where it left off.

If you are using Linux/BSD/Some flavour of unix then try using Rsync, it
works just as well for local files as it does for remote ones
eg:
rsync -rvP /source/dir /destination/dir

You will find it here if you don't have it already

http://rsync.samba.org/
