Re: /var/log/lastlog way too big!

Hang on a moment - lastlog is *not* a log file as such - it's a sparse
file that (from memory) is indexed by the UID of the user and contains
the date and terminal that the user last logged in on.

davidn@rebel.net.au wrote:

> Yes you can, however an even better idea is to truncate it using the
> Bash command "> /var/log/lastlog".

There is no need to truncate it, unless it has become corrupted.

Paul Shirren wrote:

> Or logrotate. Really this is something you should be automating.
> Remember laziness is an important quality for a good sys admin.

There is no need to rotate it, ever.  It only grows by the number
of users you have on your system - if you only ever have four users,
and they've already logged in at least once, the file will never grow
beyond the size it is.

If a person hasn't logged in the machine for four weeks (or years),
the behaviour shouldn't be "last logged in: never" because someone
has taken it upon themselves to rotate what is essentially a database
file.
