Re: Slow SMB Speeds

On Sun, 7 Dec 2003, Brendan Grossman wrote:

> On Sunday 07 December 2003 19:01, Steve wrote:
> >
> > Test the file under FTP that will help exclude network in Linux and close
> > in on SMB issues.
> > let us know the results
> 
> Ok, seems to be a networking issue, not just SMB. It only seems to be 
> uploading that's slow though, not downloading - as i use the same interface 
> for the Internet through another box. It doesn't seem to get above 2kb/s and 
> keeps stalling.

There are a couple of possible problems: packet loss, poor window size 
on the server, poor buffer size on the client. However, these last two 
will really only be a problem, I think, if the delay bandwidth is large 
enough. Also, the fact that download works OK tends to suggest that buffer 
sizes are OK on the client, as that affects the window size on the 
download direction.

Perhaps you should grab a packet capture and then look to see if there is 
any strange behaviour, like excessive re-transmits etc. The latest 
versions of Ethereal can mark retransmits for you.
