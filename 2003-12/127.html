Actually if you search the archives here you will find Simon Hackett
gives an excellent explanation of why BADSL services actually occupy
*four* static IP addresses for 1 static IP delivered to a BADSL
customer, over Bridged Ethernet.  Basically it's because the bridged
ethernet IP is delivered as a /30 subnet consisting of (anyone feel free
to correct me if I'm wrong):

1 IP for the Network
1 IP for remote gateway address
1 actual IP (delivered to customer)
1 IP for broadcast

So there is no comparison between providing a dynamic IP over PPPoX and
a bridged ethernet static IP service.  It is considerably more "IP
Address costly" for Internode to provide the BADSL static IP.
