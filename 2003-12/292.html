Re: Frustrating samba locking

Kirsty Rae wrote:
> If anyone has any suggestions on where to look to diagnose this problem I 
> would really appreciate it.  If you need more information, I'll do my best 
> to get it to you.  If you have enough information to recommend possible 
> fixes, well even better.

this may be barking up the wrong tree, but have a look at the
unix permissions that files get created in the smb.conf file...

eg;

[infrastructure]
    comment = IT Infrastructure project
    path = /usr/home/ITInfrastructure
    valid users = kim cam
    public = no
    writeable = yes
    printable = no
    directory mask = 777
    force directory mode = 777
    create mask = 775

because there are only a couple of users that use this share
and need to login i have set the mask to 775. not secure
and not ideal, but this seemed to get around the "file locking"
problem we had.

this may not fix your problem, but hopefully its somewhere to
start looking.

and theres also;

    kernel oplocks = no

but i don't know that thats relavent ... =)
but also worth a look.
