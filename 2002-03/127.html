Re: [OT] xDSL IP address change

Ben,
What Simon says is correct if the ppp/pppoe client is actively visible to
the 'USER'
In instances where the client is actually on a router/gateway using
ipmasq/ipfwd then the 'users' on lan behind the router will never know
other than for a small delay until connection is re-established.

Also the 'users' ip address will never change as such because of the length
of dhcp leases or fixed ip addresses on the lan (10.* or 198.* ranges).

So in short they will never know in this case if their ISP assigned IP
changes on them.

Evan

>Dynamic IP ADSL connections use PPPoE or PPPoA.
>
>Both of them are PPP -the same protocol that drives dialup sessions.
>
>So the same underlying situation applies - you -can- determine if you 
>are online or not, because there is an active session layer (inside 
>PPP) maintained when the connection is alive.
>
>Connection drop is both (a) explained and (b) obvious in reasonable 
>PPPoE clients, rendered in a window which is broadly similar to a 
>dialup access dialog box.
