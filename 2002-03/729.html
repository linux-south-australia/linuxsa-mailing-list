if the proxy is configured properly then i really can't see a problem with
having a transparent proxy. the only issue raised here is whether the proxy
should redo the dns lookup on the requested url or whether it should use the
IP you were trying to connect to.

in the situation in the slashdot article the guy was trying to use an
alternate DNS root and because the proxy was redoing the dns lookup, the
proxy was returning errors as it was only configured with the ICANN root. if
the proxy had been configured to utilise the destination IP on the outbound
packets this wouldn't have been a problem.
