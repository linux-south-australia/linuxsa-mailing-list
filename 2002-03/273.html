Re: FW: zlib security advisory

Poul Henning-Kamp has emailed the FreeBSD lists to say that he doesn't
think phkmalloc (the version of malloc() in FreeBSD's libc) has this 
problem.

That isn't to say that there won't be some kind of advisory, but it'll
probably only affect individual applications which are linked against
their own malloc, instead of using the one which comes with FreeBSD.

It'll still be a good idea to upgrade zlib, but probably won't be as
critical as it'll be for the rest of the world.
