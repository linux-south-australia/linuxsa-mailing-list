Re: XWindows Woes on Industrial PC

Daryl Pilkington wrote:
> I've done a default install of RH 7.2, but XWindows doesn't start.
> 
> The Chipset is an embedded Intel 815 connected to a Toshiba TFT
> flat-panel.
> I suspect I need to tweak for a flat-panel.

Doubtful.

> The error message is:
> 
> INIT: Id "x" respawning too fast: disabled for 5 minutes
> 
> Any ideas?

This likely means that your system is trying to spawn an X server from 
init (the sort of thing that happens if you want X to start when you 
boot the machine).  Make the system boot in a non-X runlevel (usually 
runlevel 3 on Red Hat) before you go any farther.  This is done in 
/etc/inittab.

Have you tried running an X configuration tool to configure X?  Run 
either Xconfigurator or X -configure (assuming you're running XFree86 
4.x).  I think if you specify a resolution other than 640x480 during the 
installation process, bad things happen.  (I always let the installer 
detect the video card, then accept all defaults, then run Xconfigurator 
once the system's up and running.)
