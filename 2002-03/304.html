> As I recall when you pass init= to the kernel it needs to point 
> to a valid root partition.
> 
> You're pointing it to /bin/bash, which is a shell, not a root 
> partition.
> 
> try "init=/dev/hda5" or something like that, might need some trial
> and error before you pick the right partition tho ;)

Sorry, this is wrong.

init= points to how you want to init, root= points to the root filesystem.

So in the case where you know root, but /etc/fstab is carp, and lilo is 
confused, try:

linux root=/dev/hdaX init=/bin/bash

or indeed just boot via CDROM or floppy into rescue mode.  It's really 
wonderful nowadays.  Trust me.
