Just discovered something buggy with Linux.  I suspend my laptop
instead of shutting it down normally and currently I'm upto 4 days
worth of uptime which took 16 days to get (since Linux doesn't count
uptime while suspended).  This is all fine except it screws with the
start time of processes.  

For example, I last booted my laptop on the 4th of March so I'd expect
that the init process would be listed as starting on the 4th of March
but instead it's listed as starting the 16th of March (ie today's date
minus my uptime).

vodka@mikey/ % ps xaug |head -2
USER       PID %CPU %MEM   VSZ  RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  1208  464 ?        S    Mar16   0:03 init
vodka@mikey/ % date
Wed Mar 20 18:44:40 CST 2002
vodka@mikey/ % uptime
  6:44pm  up 4 days, 16:35,  5 users,  load average: 0.05, 0.05, 0.00
vodka@mikey/ % ls -l /var/log/dmesg 
-rw-r--r--    1 root     root         5429 Mar  4 11:19 /var/log/dmesg

(/var/log/dmesg is written when the machine boots)
