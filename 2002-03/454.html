The correct way to configure XFree4.x.x is to issue XFree86 -configure
in a root prompt or xf86cfg for the gui. Although the older commands
xf86config(text mode) and/or Xconfigurator will work as well, they both
produce the old xfree3.x.x syntax which while still backward compatible
in the newer XFree4 server is not technically correct.

 The redhat install cd can also be used as a rescue disk by typing
rescue as the install type when booting with it. This will mount your
system at /mnt/sysimage and allow you to edit files and correct any
problems.
