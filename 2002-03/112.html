Probably not. fstab contains lines such as:

---
/dev/hda3       /               ext2    defaults,errors=remount-ro
0       1
---
...

What you have pasted appears to be output from df (I don't know what the
/etc/fstab executable you are running is supposed to do -- that's just
the config file on my system... but it looks like df...)

What you need to do is investigate the relevant line from your
/etc/fstab file (view the file, don't run it -- I'm guessing your config
file masqerades as a script of sorts), and see if it would still be
correct... you were using hdc1 -- is the new drive going to be the
secondary master IDE device (hdc)? Is there a filesystem, as specified
in the fstab file (ext2 for mine, above), on the first partition (hdc1)?
Is it going to be mounted in the same place, in the same way as the
original one (mine is mounted on / with some specific options...)? If
all is the same, go for it.

> 
> Or I just power off the computer and replace this old
> removabel hard disk with a new one without mofifying
> the /etc/fstab file, then reboot??

I would suggest powering down, installing the new drive, then powering
up... You may also need to ensure that no data in /u1 is important
anywhere else -- although, if you discover it is, you can just put the
drive back in there no worries...
