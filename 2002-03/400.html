Michael

> This likely means that your system is trying to spawn an X server from
> init (the sort of thing that happens if you want X to start when you
> boot the machine).  Make the system boot in a non-X runlevel (usually
> runlevel 3 on Red Hat) before you go any farther.  This is done in
> /etc/inittab.

He can't get it to boot ;-P

The best way to do that would be to:

1) type "linux single" at the lilo: font
 (or work out how to get Grub to go to single user mode)

2) edit /etc/inittab with vi, emacs, pico or some editor

Change:

id:5:initdefault:   

To:

id:3:initdefault: 

Then type "telinit 3"...or reboot...
