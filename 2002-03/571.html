Re: Virtual Mail Domains w/ sendmail

Hi,

I can only speak from a Qmail point of view, but the following should be
applicable to any MTA that runs maildirs;

Damien Wilson wrote:

> What I would like is for it to be a similar setup to the NTMail config where we could have info@domain1 and info@domain2 logging
> into their respective mail servers (mail.domain1 and mail.domain2) both using the username info (domain1 and domain2 are on the same
> box but different IP address).

This is possible, but you're going to run out of address space quickly
if you use a distinct IP for each virtual domain.

What you can do instead is have the user login using their email address
as a username, ie: user@domain.com.au. For email clients that don't
support this, (like old Eudora) you can use user%domain.com.au as a
fallback.

> We will be shifting approximately 35,000 email accounts for our domain plus about 400 hosted domains with that number steadily
> rising.

Have a look at single-uid mail solutions. 35000 unix user accounts could
get messy and slow unless you're using something like PAM or another
backend to manage it.
 
> Another question, can sendmail handle this on one box or are we going to have to look at several boxes for this?

If all 35000 users are regularly using this, then it's going to be a big
load for one box. Qmail copes pretty well with volume in my experience,
where the same amount of mail cripples sendmail on the same hardware.
