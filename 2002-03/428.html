Re: Urgent web server problems.

MWP wrote:

> Once this had been done, the server was put back up, but the loadavg shot
> up to >100 with mysql using all the cpu time. This was with only a slight
> web serving load.

Have you had a look at `mysqladmin processlist` to see if you can
identify a particular query that's causing this CPU usage?

If you didn't recreate all of the mysql database, it might also be worth
shutting it down and running isamchk on the db files to confirm that all
of the indexes are intact. If you have some particularly large indexed
tables that see a lot of activity, you can increase the key buffer size,
and change some INSERT statements to INSERT DELAYED to make the database
behave better.
