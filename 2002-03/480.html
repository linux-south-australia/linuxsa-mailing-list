Re: mount as non-root

On Fri, Mar 15, 2002 at 01:27:34PM +1030, Ben Williams wrote:
> i had trouble before with mount reporting "smbmnt must be installed suid 
> root for direct user mounts (500,500) smbmnt failed: 1" - i chmod'ed 
> /usr/bin/smbmnt to suid root, which solved that problem. i can't mount 
> still tho.

check you (as non-root) have write permission on the mount point
(eg. /mnt/music is owned by fred:users and is rwxr-xr-x
or fred is in the 'users' group and perms are rwxrwxr-x or
variations thereof).

Also check out the uid and gid options to mount (or smbmount)
if you want the mounted stuff to be non-root as well - useful
if you want to be able to write files in there as non-root.
