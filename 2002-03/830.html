RE: Bastille Firewall on Manduck 8.1 Blocking my Samba...

Thursday, March 28, 2002, 9:29:04 AM, James wrote:

> Can anyone help me get my samba shares and my ssl connections running? I
> have Bastile firewall running on Manduck 8.1. The kernel is new enough for
> IPCHAINS.

> I tried adding a script to /etc/rc.d/rc.local:

> ipchains -A input -p tcp -j ALLOW --destination-port 135:139 -i eth0

Hmmm, methinks you're going to have to allow all high ports too.
(Unless Samba is a lot nicer about networking than windows, and since
it's designed to be compatible with windows, I doubt it).

Ports you almost always need for Windows shares are:

139 - nbsession
137 - nbname
135 - msrpc-loc
>1023 - for RPC calls
sometimes 427 - has something to do with domains, IIRC.
sometimes 138

I also can't remember which of these are UDP and which TCP off the top
of my head.

I can probably look it up if you get really stuck, although a search
of MS TechNet will provide the ports you're after...
