Re: mount as non-root

On Fri, 2002-03-15 at 15:32, Michael Wardle wrote:
> Ben Williams wrote:
> > i've got a RH7.2 desktop. i'm trying to mount remote smb shares on our 
> > windows network as a normal (non-root) user, and i can't get it to work. 
> > the error i get is:
> > 
> > [benw@orion mnt]$ mount docs
> > cannot mount on /mnt/docs: Operation not permitted
> > smbmnt failed: 1
> 
> The easiest way is to set the setuid bit on smbmount smbmnt and 
> smbumount and make sure they're owned by root.
> 
> > i had trouble before with mount reporting "smbmnt must be installed suid 
> > root for direct user mounts (500,500) smbmnt failed: 1" - i chmod'ed 
> > /usr/bin/smbmnt to suid root, which solved that problem. i can't mount 
> > still tho.
> 
> Did you also set smbmount and smbumount?

[benw@orion benw]$ l `which smbmnt`
-rwsr-sr-x    1 root     root         400k Aug 14  2001 /usr/bin/smbmnt*
[benw@orion benw]$ l `which smbmount`
-rwsr-sr-x    1 root     root         409k Aug 14  2001
/usr/bin/smbmount*
[benw@orion benw]$ l `which smbumount`
-rwsr-sr-x    1 root     root         398k Aug 14  2001
/usr/bin/smbumount*
[benw@orion benw]$ l `which mount`
-rwsr-sr-x    1 root     root          56k Jul 25  2001 /bin/mount*
[benw@orion benw]$

-> i.e. all of them (mount, smbmount, smbmnt, smbumount) are all suid to
root.
