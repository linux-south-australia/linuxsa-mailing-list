Re: linux as a desktop (was Re: Transparent Proxying)

Jordan Noblet wrote:
> - browser address bars should automatically select the address when you
> single left click or right click on them (for copying to clipboard or
> overwriting).  If I want to paste an address in I have highlight the
> current address, delete it, and hope that the address I want is still in
> the clipboard because (heaven forbid) I clicked the left mouse button.

You can do this in Mozilla.
<URL:http://www.geocities.com/pratiksolanki/>;

Put this line in prefs.js in your Mozilla profile directory
user_pref("browser.urlbar.clickSelectsAll", true);

Personally, I greatly dislike this behavior, and am glad it can be
disabled.

Of course, different browsers handle things different ways.
- Galeon
- Konqueror
- Mozilla Browser
- Netscape Navigator
- Opera
- ...

>>From what I gather, KDE and Gnome have different copy/paste systems and
> copying/pasting between application types is a nightmare.

In most X applications, use left-mouse-down to start selecting the text,
left-mouse-up to finish selecting the text, and middle-mouse to paste at
the current cursor point.

I think Qt is working on better cut-and-paste support, and Windows-like
control-c, control-v, control-x should work much better in the upcoming
KDE 3.0
