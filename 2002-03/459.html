> Alternatively, is it possible to build a kernel for architecture
> X but on another computer and then put it all together using

> 
> I've transferred my main drive to a K6-iii 450 and it doesn't like
> my kernel to the extent the compuer just reboots (by itself)....

That will probably be the main reason for the rebooting. The cpu type in
the kernel is incorrect for the machine. You can build a kernel on any
machine so long as you select all the options appropriate for the
destination machine, including the correct cpu type in General Setup.
To avoid any conflict with the host machine's kernel and modules edit
the kernel Makefile and set the EXTRAVERSION variable to suit. Like
EXTRAVERSION=K6-iii
Then you can make the kernel as normal, and lastly copy over the files
you need.
You will need /usr/src/linux/arch/i386/boot/bzImage of course, and dont
forget the /lib/modules/2.x.x-K6-iii/ directory as well.
