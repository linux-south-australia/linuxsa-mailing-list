RE: Finding Strings in ascii files

I messed about a little, and came up with this... Anyone got a more
elegant way of making this work?

----
richard@gecko:~$ cat asciifile
#!/bin/bash
file $1 | grep -q ASCII
richard@gecko:~$ find . -type f -exec ./asciifile {} \; -exec grep
mystring {}
----

asciifile is mode 755...

note nothing's fully tested -- it works correctly on my system in one
directory... although may get things that aren't ASCII text (check the
magic file --  'grep ASCII /usr/share/misc/magic' on my system) ...
you'd want to put full pathnames, test, etc... but that's the general
idea...

I couldn't, from the top of my head work out how to put the | into the
command line, hence the need for the script...

Oh, and BTW, I just looked at the magic no's file again, and you will
also miss out on a lot of text files that way... (eg "Bourne shell
script text" may be something you wanted... this could be a messy thing
to try to do...)
