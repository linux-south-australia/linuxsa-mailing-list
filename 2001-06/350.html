Re: Learning how to programme in Linux

Alan Kennington writes:
> If your header files are correctly ported to C++, as most
> linux header files are, then you should be able to
> compile C programs with a C++ compiler without change.
> I'd call that an "extension"!

Actually with the release of the C99 standard this is no longer true
and C++ is not simply a superset of C.

And in terms of design of a program C++ is very different. I'd go as
far as to say that I think many of the problems that large C++
projects suffer from are due to experienced C programmers programming
in C++ and treating the language as "C with extensions".

If you plan on doing C++ programming then its probably better to _not_
learn C first.
