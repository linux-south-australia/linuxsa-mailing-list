As your message is short, it is hard to tell if it is NIS that is problem.
NIS is a way of distributing setup information throughout a network.
Do you have a NIS server?
Can you ping it?
Does the command "ypwhich" report your NIS server?
Does the user account kishna exist on the NIS master?
If so, have you pushed this to the NIS clients?
Does the command "ypcat -k passwd" contain kishna?
If so, does /etc/nsswitch.conf have NIS listed before files on the password
line?
