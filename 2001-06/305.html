All jokes aside, you are only going to learn to program if you actually
start programming.

The environments that are readily available under Linux are:

  C, C++, Perl, and Python

My approach would be to buy a book on any of those environments, and work
through them, doing the exercises. Next, come up with some non-trivial
project and start doing it.

However, when it comes to big projects, then you need to learn more. Most
big projects are done in C. There is not a lot of C++ out there in big
projects, but there are notable exceptions: Jade and HylaFAX being two that
I am aware of.
