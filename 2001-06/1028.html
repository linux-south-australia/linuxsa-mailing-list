apache/php/netpbm and PATH

hi all.

i'm trying to execute a command under PHP/Apache. my script does the
following:

<?
...
$cmd = "anytopnm $src_file | pnmscale $params | ppmtojpeg > $dest_file";
exec($cmd);
...
?>

(basically, it uses the netpbm library to make a thumbnail of a file. the
steps are 'convert $src_file to pnm format | scale that output | write a
jpeg file to $dest_file'.)

the problem is, anytopnm relies on other programs (jpegtopnm, giftopnm) to
do the actual conversion. these programs are under /usr/local/netpbm/bin,
not normally a part of the PATH. i've added that directory to the PATH
directive in /etc/profile, and when i'm shelled in using bash that command
works fine. but it won't execute under apache/php! the command runs, but the
output file is 0 bytes long.

i tried this without /usr/local/netpbm/bin in the PATH - it returned errors
(obviously) saying "command not found" for those 3 commands, and wrote a
0-byte output file. (i.e. it had the same symptom - 0 byte output - as when
it's executed under apache.) this is what makes me think it is a PATH issue.
i've tried su'ing to nobody, and that folder is in the path and the command
executes properly.

what am i doing wrong? help!

the system is running apache 1.3.19, php 4.0.5 loaded as an apache module,
and netpbm 9.14, on redhat 7.0.
