try `file core`... should tell you what program it was that dumped.

You may have heard the expression "to dump core" before... It happens when a
program dies an unnatural death, and the memory image at the time is saved
to disk so if you want to get really enthusiastic, you could debug it and
find out what went wrong... You may see this happen occasionally when
netscape (as a common eg) dies, and says somethign like "Segmentation fault:
core dumped" -- there will now be a netscape core file lying around
somewhere (not garanteed to be in an obvious place...) ... "core", by the
way, is an old term for memory...

If, like 99% of the known world, you aren't into debugging core, you can set
a variable (something like CORE_SIZE -- someone help here?... or maybe man
core would help) that will restrict the maximum size of a core file. Set
this to 0.
