Emergency Recovery ... How TO?

Hi,

I did something extremely stupid a little while ago. I ignored some error
messages when I made some software. I copied the .so that resulted into
/lib and created a file called /etc/ld.so.preload which had one entry: /lib.

As soon as I did, nothing would run; the run-time loader complained of
shared library problems.

Even a reboot into single user mode failed, as init failed with the same
message and the system panic'd. I suspected that even booting and bypassing
init would have similar problems, because almost everything is dynamically
linked.

At that point I had to think seriously about how to recover. Here are the
steps I used:

1. Boot the RH7.1 installation CD into text mode.

2. When it came up, ALT-F2 to get to the shell there.

3. cd /tmp
   mknod hda5 b 3 5 # Where my root file system is
   fsck /tmp/hda5
   mkdir /mnt/xxx
   mount /tmp/hda5 /mnt/xxx
   rm /mnt/xxx/etc/ld.so.preload
   rm /tmp/xxx/lib/fakecd.so
   umount /mnt/xxx

4. Reboot, and all is now OK.

Question is, what other ways are there.

Perhaps there is a need for an advanced recovery HOWTO.
