Re: RAID Array Problems

I'm no expert by far so please double check what I say.

having said that I've just finished setting up a raid 5 array using linux 
software raid. correct me if I'm wrong but raid 1 is mirroring right? the 
reason for the super block problem is as anthony said, they weren't synced 
correctly when the array went down. now assuming you are using the software 
raid that is in the 2.4 kernel (or a patched 2.2 with the same raid 
versions) you could try a mkraid /dev/md0 to resync the superblocks but this 
can be VERY DANGEROUS unless the raidtab exactly represents the array you 
are using. Even still, it's not guaranteed to work A possibly safer but more 
expensive and time consuming method would be to completely replace one of 
the drives with a new one. this would possibly cause the machine to start a 
resync of the two drives.

These are just some ideas but would need to be confirmed by someone with 
more experience.
