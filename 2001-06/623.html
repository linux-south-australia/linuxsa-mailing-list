Re: spaces in filenames

> I have recently come across a problem with bash scripts where bash fails
> to recognise escapes or quotes in filenames.

More likely you've come across a problem with bash scripts where
the author didn't really know what he or she was doing.  Few people
seem to understand how or when the shell splits commands into words.
This is wrong:

  #! /bin/sh
  command $1

This is how it should be done:

  #! /bin/sh
  command "$1"

This is wrong:

  #! /bin/sh
  for f in $*; do command $f; done

This is how it should be done:

  #! /bin/sh
  for f in "$@"; do command "$f"; done

Take particular note of "$@" instead of "$*".  This, quite simply,
is an arcane incantation that separates the gurus from the wannabe's.
It also happens to be in the man page, but if you run linux maybe you
didn't get that optional component; apparently Microsoft often leave
it out.
