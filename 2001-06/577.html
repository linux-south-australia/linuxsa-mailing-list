Re: perl cgi and apache

Hmmm...

> If I do a httpd -l, it shows that mod_perl isn't compiled in, but I do have
> mod_perl installed physicall (from the red hat rpm), however I was under the
> impression that with a scriptalias, Apache will attempt to execute all files
> in the ScriptAlias directory rather than display them.
> 
> Any suggestions?

Does Apache know what "type" your scripts are....

AddType text/html .epl
AddType text/html .cgi

For example. If it doesn't have these types it will probably consider
your scripts to be text scripts and display them.

Also, when you shunted the scripts over did you:

a) check to make sure that the user that apache runs as (ps ax, or ps
eax piped to more or less will find that out) can actually run the
scripts?

b) did you check to see if the scripts installed as a sensible user?
(use a ls -lR)
