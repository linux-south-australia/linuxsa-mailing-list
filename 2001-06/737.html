Maybe.  Or perhaps he wants to make break-ins more difficult.  It's not
a bad idea to have your binaries on a read-only file system - it
certainly makes installing a root-kit harder.

If you want to make a file system read-only, you can either do that in
fstab, or you could use LIDS (http://www.lids.org).

If you specify that the file system is read-only in /etc/fstab, it can
usually be remounted (by root) as writable.

Lids lets you specify files, directories, and filesystems that are
read-only (not even root can modify them) as well as providing heaps
of other cool security features.  These security features can be
enabled and disabled on the fly by entering a password.

Of course you'll want stuff like /tmp and /var to be writable - you
could either put these on unprotected partitions, or configure LIDS to
allow files in the appropriate places to be modified.
