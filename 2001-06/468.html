Re: Samba with OS/2

Daryl Pilkington wrote:

> Based on this information, do people think an OS/2 requester or server
> will talk to a Linux Samba server?

Yes. Have a look at the OS2_client.txt file in the textdocs directory of
your samba distributions documentation.

> Can Samba replace a PDC yet?

Yes and no. It depends on the workstations you have in the domain, what
feature you see as being important for a PDC to possess, etc, etc.

Bad news:
* There are/were problems with Win2K SP2
* The Active Directory stuff doesn't exist

Good news:
* A great deal more RPC stuff works in Samba 2.2, which means you can
modify certain things using the Windows GUI utilities.
* ACLs on Linux (and a few other systems) are supported (and can be
modified from Windows)
* Windows NT domain members are officially supported now
* Windows 2K can join the domain in compatibility mode
* It's still not windows...

> What testing have PDC replacements had with other than Windoze clients?

Unsure. I believe people are using OS/2, but it's not a core concern of
the Samba team AFAIK.
