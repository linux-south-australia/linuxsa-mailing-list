Re: for loop (bash) with spaces in filenames

On Mon, 25 Jun 2001, Peter Freeman wrote:

> for f in `cat /pathtolist/mp3.txt` do cp "$f" /newdirectory done
>
> Can anyone see the glaring holes in the syntax?  Or is there a better
> way to do it?

Even with the syntax errors fixed up, I can't figure out a way to make
it handle the spaces properly.

Try this instead:

  xargs -l cp {} /newdirectory < /pathtolist/mp3.txt

You may want to do a "mv" instead of a copy if you want to speed it up
and you're pretty sure what you're doing. A suggestion is to try this:

  xargs -l echo cp {} /newdirectory < /pathtolist/mp3.txt

to check the commands being issued, then remove the "echo" to actually
perform the command.
