Re: qmail-pop3 authentication through NT4 PDC

Charlie M MAO wrote:
> 
> I installed qmail on Linux RH 7.0 system. I don't have problem to access
> mail message from Outlook if the client has a password on Linux machine.
> However we use NT4 as PDC, all staff accounts are currently on that machine
> and most of staff don't have password on Linux machine. How can I setup
> qmail to authenticate qmail-pop3 client through NT4 machine?

The pop3d that comes with qmail is actually a handful of different
programs that run in this order:

qmail-pop3d
checkpasswd
qmail-popup

qmail-popup requires that checkpasswd sets the environment variables
USER, SHELL, HOME, UID and exits 0, which gets its input (username,
password, apop_ts) as arguements from qmail-pop3d.

You could write an alternative checkpassword in perl, and use the
Authen::SMB module to check the validity of the username/password
against the NT server.
