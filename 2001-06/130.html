Re: software RAID versus hardware RAID

Mark wrote:

> If you are interested in a RAID controller, look at an ABIT KT7A-RAID
> motherboard.  They take IDE drives and are very quick.  The configuration
> is simple and allows mirrors, striping and striping with mirrors.  If you
> do go down this route, make sure you put the latest bios on them.

Does this work with linux?

Most of the Motherboard-based raid controllers seem to be the 'Promise'
item, which doesn't work with linux AFAIK, and isn't a real hardware
raid controller. It appears to be a hardware interface to facilitate a
software raid system that runs on windows.
