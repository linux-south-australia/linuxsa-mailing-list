> I did do this, in fact I tried a lot of variations to see what I felt most
> comfortable with.
> Lilo was giving me errors going via rescue mode saying it cant find lilo.conf
> I cant recall IF I ran "lilo -r /mnt/sysimage" exactly 

Lilo saying that it can't find lilo.conf is a good indicator that you
didn't use '-r /mnt/sysimage' (either that, or lilo.conf didn't
actually exist there).

> Could you explain what this does for me.

It tells lilo to run itself in a chrooted environment so that it
thinks that /mnt/sysimage is the root directory.  All paths (eg.
/etc/lilo.conf, and the files referenced from within there such as the
ones under /dev and /boot) are now relative to the new root directory,
so lilo will be able to find the file /etc/lilo.conf even though it
really is /mnt/sysimage/etc/lilo.conf.

An equivalent process to running lilo -r /mnt/sysimage is:

chroot /mnt/sysimage
lilo
exit
