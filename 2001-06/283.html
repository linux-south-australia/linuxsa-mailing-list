Re: Net-SNMP (aka UCD-SNMP)

On Thu, Jun 07, 2001 at 09:05:10PM +0930, Andrew Reid wrote:

 > Hmmm... Obviously UCD-SNMP provides some way of monitoring your network 
 > traffic as Mark said, but apparently the FAQ is a little unclear on this.
 > Can anyone offer a clarification here?

The standard SNMP interface MIB provides per-interface bytecounters.
Applications programs, or cron jobs, or mrtg, or Cricket, or any number
of other tools can accumulate those bytecounters to generate hourly, 
daily, monthly, or average byte utilization figures.

You can also build your own MIBs to add more extensive monitoring, or
use one of the MIBs which come bundled with the SNMP server to get
things like CPU utilization, disk space monitoring, memory utilization,
etc.

There's also an SNMP module in CPAN for perl programs which provides
an object-oriented interface to SNMP servers, permitting you to send
requests or traps to then in whatever way you want.
