> Perl is not really a programming language. It's an
> interpreter language. I.e. it's just a script language.

I don't distinguish programming languages by how they are executed.
Java, for example, is widely considered a programming language yet it's
executed in much the same fashion as PERL. That is to say, the Java is
compiled at run time and then run on the host machine.

Java, of course, adds the virtual machine in between the real host
machine and the programming language but that is not really part of this
discussion.

[Side Note:

PERL is compiled and optimised by the Perl Interpreter into an internal,
tokenised representation. The PERL interpreter interprets that
representation. 

]

PERL 5.6.0 displays:

1) Control structures (for, while, if)
2) A method to work with complex structures (through references)
3) Functions/Procedures (called "subs" in PERL)
4) Support for Objects
5) Support for Modularised code

Furthermore it can do all processing without calling a "helper program"
such as "ls", "cat" or "ls". It does (eventually) speak system calls to
the system. I think this qualifies PERL as a programming language; that
it is also an excellent scripting language is true but not, in my
opinion, mutually exclusive.

Also, speed of execution is only one variable amongst many. Try writing:

#! /usr/bin/perl

while(<>) {
	print "Found Him!\n" if m/lloy0076/;
	print "$_\n";
}

With just the C standard libraries. Much more code, more code to go
wrong and you could easily fall into portability dramas.
