Re: Multiple dumps to one tape

Andrew Reid wrote:

You clock seems to be several days out ...

> I'm currently playing around with dump to backup various filesystems on my 
> mail server.
> 
> I've got a question regarding the ability of dump to write more than one 
> filesystem to the tape in a mannar that allows each one to be restored 
> seperately.
> 
> For example, I currently run dump like so to backup my mail:
> 
> dump -0 /var
> 
> If I was to run the following commands, will the two seperate filesystems 
> be able to be restored by restore at a later date?
> 
> dump -0 /var ; dump -0 /usr

No. AFAIK, the second will overwrite the first. Try:

dump -0f /dev/nst0 /var; dump -0 /usr

This will cause the tape to stay where it was after the first and put 
the second after it.
