Re: IP Masquade network and Voice Chat

Rhett!

> In their FAQ, a voice chat needs an outgoing tcp connection on port 6901.  (I
> think 10.1.1.4 also needs to be able to receive a connection on port 6901)
> UDP packets are also sent/received on port 6901.

# This masquerades any tcp connection to or from port 6901
ipchains -A forward -i ppp+ -s any/0 -d 10.0.0.0/24 6901 -p tcp -j
ACCEPT

# This does the same for udp
ipchains -A forward -i ppp+ -s any/0 -d 10.0.0.0/24 6901 -p udp -j
ACCEPT

A few caveats:

1) If you have a default DENY rule you'll have to allow TCP/UDP 6901
through the firewall
 - before those two rules

2) If you're using iptables (i.e. not ipfilter as describe by A.
Kennington last meeting)
 - there's probably a better way to do that
 - check the documentation

3) If MSN needs to speak _directly_ with the machine behind the
firewall...
 - you may need to check up port forwarding rather than masquerading
