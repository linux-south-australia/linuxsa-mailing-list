re: strange hanging of server

Andrew,

Hardware hiccups and general network malaise may cause a networked Linux to
hang. Some things to try:

 - Check out the network card in your Linux box (if necessary, boot to DOS and
run the card's diagnostic utility).

 - Check all the network cables connectors and sockets. Wiggle things to check
for intermittent connections. Don't forget to check the network hub too.

 - Also on each of the Windows boxes, check that their cards are NOT set to
use real-mode drivers or a different duplex mode or something similar. They may
still work but most of their traffic will be mangled packets or re-sends; which
will upset a network...

 - Most cheap hubs don't support full-duplex networking. Some cards use
full-duplexing by default.

 - Can the Windows boxes network to each other in TCP/IP? Do this test with all
boxes disconnected from the network except two. Remove netbeui if it's there,
then try copying a few hundred megs of files from one box to the other.  If the
copying dies after a couple of minutes then one of the boxes has a problem.
Repeat with another pairing to see which box is the culprit. But if this malaise
affects all pairs of Windows boxes then your network has a problem.

 - Try replacement network cards. Beware of some cards claiming "NE2000
compatible" which _don't_ work with the Linux NE2000 modules.


I'm assuming you have already checked out user permissions, firewalling, the
config of Apache/PPP/mail/etc...
