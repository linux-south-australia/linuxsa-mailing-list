Re: software RAID versus hardware RAID

On Monday,  4 June 2001 at 11:21:58 +0930, Glen Turner wrote:
> Greg Lehey wrote:
>>
>>>> Can anyone tell me if I would be better off with
>>>> hardware or software RAID?
>>>
>>> Hardware RAID is faster than software RAID (less CPU overhead) but
>>> more expensive.
>>
>> As a general statement, only the second part is correct.  Hardware
>> RAID is not necessarily faster than software RAID.
>
> I've found the operational issues to be:

I can't speak for others, but here's how Vinum handles these issues:

>  - can you reconfigure it using the serial console

Yes.

>  - can you easily have a RAIDed root disk

For some definition of "easily", yes.  Hardware RAID usually wins on
this one because the software it runs is not stored on the root
partition.

>  - can you stack (eg: mirror two RAID5s)

Yes.

>  - what's the performance of a RAID5 parity rebuild

I/O bound.

>  - when the mirror breaks, can you hot swap in a drive
>    and get it online without needing to stop the
>    other disk (or worse, reboot).

Yes, assuming you have hot-pluggable hardware.

>  - does it crash and burn with non-identical drives
>    (as inevitably happens when a drive fails in
>     three years time and you look for a replacement
>     for the one you had on the shelf).

No.
