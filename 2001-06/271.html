Re: MRTG graphing

Daryl Tester wrote:
> 
> "James Mclean" <james@adam.com.au> wrote:
> 
> > According to the MRTG website, no extra SNMP software needs
> > to be installed.
> 
> Maybe not, but don't the interfaces you want to monitor need
> to be SNMP'able?

Certainly not. You can have MRTG run any command you like and it will
work as long as returns two integers seperated by '\n' each time it's
called. I had a daemon listening on a UDP port for commands. When it
received such a command it would return results from data in /proc. MRTG
would run the client program which would send the relevant commands to
the server process. I would run MRTG on one machine, have it show
various meters for all of my Linux servers without the use of SNMP at
all.

Hope it helps,
