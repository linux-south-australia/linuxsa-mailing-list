Re: PHP as a Scripting Language

michaeld@senet.com.au wrote:
> 
> David Lloyd enlightened us with:
> > Has anyone had any experience using PHP as a scripting language (much
> > like you might use Perl or a bash script)? Is it reasonably efficient or
> > as developed as say, Perl or bash?

> Sure, php could be used like that.  But with terabytes of perl in existance,
> the simplicity of python, and the variety of awk/sed/TCL/[t[c]|ba]sh scripting
> languages available as well, the question begs, why?[2]

PHP can run like a normal scripting language, but it is missing things
like parameters and stdin handling.

PHP has been designed as an embedded scripting language, which makes it
suitable for web-like applications. Writing a telnetable BBS for linux
would be an ideal job for PHP, any system with a lot of content and some
requirement to change data on the fly would be suited to it.

One other interesting project is php-GTK which looks like an easy way to
build cross-platform applications with a minimum of new skills to learn
if you already know PHP.
