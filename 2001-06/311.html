Re: Learning how to programme in Linux

Adam!

There is the original K&R C Book in an Adelaide book store for $8...but

1) Theory

Go to a library and borrow a book on generic C or whatever programming
language you want to use. I'd suggest, for Linux, that you go C, C++,
PERL or PYTHON. A LISP or PASCAL book might be useful but the ones I've
mentioned are more widely used from what I've seen.

Get the theory from them (what loops are, oo, methodology).

2) Practice

You certainly have PERL, Python, C and C++ on a Linux system. PERL, C
and C++ exist on my BSD system. It's best to make "silly" programs that
do nothing in particular at the start just to get the feel of
programming.

It's like a house - learn how to make the bricks and foundations before
building the house.

3) Source Code

At the same time, delve into some source code. You don't need to
understand it perfectly, just look for its structure at first. And then
use your books (if you have any), the man pages, the documentation to
understand the code.

By understanding and looking at others' source code, you'll find hints
and tips for your own.

Hope this helps.
