Re: software RAID versus hardware RAID

Hi.

Well, maybe. Not misassumption - just analysis of the arguments proposed.
If there is a third then roll it in. The proposition that:

CPU + Raid Software + system Load

Results in a better solution than:

Cpu + system Load with off-cpu Raid software running on dedicated CPU

Still needs proving. Like I have said before, I'm happy to take it on
board, and I don't wish to downplay the marvel of being able to do this in
software on the system CPU or anything like that. I just want to know 'Is
it really better' My questioning is not based on can I do it cheaper (or
dearer), but is this a better solution?
