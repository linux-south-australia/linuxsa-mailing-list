You can probably get it now provided you didn't totally wipe Win ME off.
Try these things:

1) man lilo.conf
2) man lilo
3) man fdisk

Now, you need to configure your lilo.conf similar to the configuration
that comes up under the lilo.conf man page. If you've lost the linux man
pages, you can goto http://www.linuxdoc.org/ where there are online
copies (or http://linux.senet.com.au/ but I've actually found that to be
slower than the real site, go figure :-P).

Use fdisk /dev/hda (or /dev/hdb or /dev/sd0 [I think for SCSI]) and do a
"p". You'll see something like:

    Device Boot    Start       End    Blocks   Id  System
/dev/hda1   *         1        32    257008+  83  Linux
/dev/hda2            33        45    104422+  82  Linux swap
/dev/hda3            46       143    787185   83  Linux

One of them will be a FAT32, VFAT or MS-DOS. That will be the partition
WinME is on. When the lilo.conf page tells you to insert the name of the
drive, or partition, try that one.

Provided you leave the stuff about Linux alone, you shouldn't be able to
lose control of the computer.

I hope this helps you enough to get going.

Incidentally, I'm reasonably certain that if you have a Windows
installation and are careful when installing RedHat 7.1 not to overwrite
it, that RH 7.1 can get dual boot going automagically.
