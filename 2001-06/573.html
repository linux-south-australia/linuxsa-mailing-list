perl cgi and apache

I installed a webmail program yesterday on a red hat 7 box, it uses cgi
scripts in perl to do its work. Apache however just displays the perl source
code and doesn't execute it. I've set up Aliases and scriptaliases in
httpd.conf as below:

ScriptAlias /nswm/cgi-bin/ "/opt/nswm/cgi-bin/"
<Directory "/opt/nswm/cgi-bin">
    AllowOverride None
    Options ExecCGI
    Order allow,deny
    Allow from all
</Directory>

Alias /nswm "/opt/nswm/"

If I do a httpd -l, it shows that mod_perl isn't compiled in, but I do have
mod_perl installed physicall (from the red hat rpm), however I was under the
impression that with a scriptalias, Apache will attempt to execute all files
in the ScriptAlias directory rather than display them.

Any suggestions?
