RE: One User, Many Passwords

> > Is there any easy way to convince Linux to accept multiple
> passwords for
> > one user?
> >
> > (eg, user foo's will gain access if password is "p1" or
> "p2" or
> > "whatever_other_ones_there_could_be")

I think you can get this effect as follows (ignoring /etc/shadow for now):

/etc/passwd:
...
realuser:<p0>:400:400:The Real User:/bin/bash
fakeuser1:<p1>:400:400:Fake User 1:/bin/bash
fakeuser2:<p2>:400:400:Fake User 2:/bin/bash
fakeuser3:<p3>:400:400:Fake User 3:/bin/bash
...

I *think*...

I'm pretty sure I have seen this used to gain root access to a machine where
we had administrative control over the NIS database -- a user called admin
with uid 0 was added, so you knew that passwd, and once you got in, Unix saw
uid 0 and thought "hey wow -- this guy's root!"
