VNC server help

1. Does anyone know whether work exists that integrates Unix vncserver
with the configuration for an existing X configuration?

2. What about VNC server based on XFree86 4?

To explain.

I'm starting to do some of my work using a shared VNC server session. I
tried this some time ago but gave up because of the CPU hit. With a
reasonably modern machine this isn't a problem any more.

So on a main system I run:

    vncserver -alwaysshared -depth 24 -geometry 1280x1024 :1

When I am sitting at the console of this system I do:

    startx   (which has a nice lightweight "twm&" in the .xinitrc)
    left-click, start an xterm, then
    vncviewer -fullscreen -truecolour localhost:1

and up comes my nicely-configured desktop running one of my heavyweight
window managers (unless I'm having a fvwm2 day :) Pressing F8 lets me get
back to the master session via a popup menu. All standard stuff. Then I
can start sessions involving long runs, large datasets, big virtual discs
etc and provided the entire machine doesn't crash (never yet!) I can do
what I like with the console (control-alt-backspace, install new NVidia
drivers...) without disturbing anything. Then I go somewhere else on a
quick network (such as home, where careful readers may recall I have
non-Telstra ADSL :) and do a similar vncviewer command over an ssh
connection.

Trouble is, my main X server is nicely configured and I want to have that
on my VNC'd desktop. I use XFree86 version 4 so I don't have a font server
(but I'm doubtful if a VNC port of ver 4 exists, I've never heard of one.)
I'm hoping there's a set of scripts somewhere that will do the
configuration for me, or maybe I'm just missing some obvious way of using
an existing XF86Config file.
