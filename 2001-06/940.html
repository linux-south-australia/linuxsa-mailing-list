Re: tcsh does not work as root shell in SuSE 7.1??

The usual reason why you get "No such file or directory" when the file is
clearly there, is because some other file, needed to startup the image,
is missing.  For example:

  $ cat demo
  #! /bin/nothere
  echo "hello world"
  $ ls -l demo
  -rwxrwxr-x 1 davidn davidn  34 Jun 26 21:29 demo
  $ ./demo
  bash: ./demo: No such file or directory

If you're trying to start a binary, particularly one that uses shared
libraries (which your /bin/tcsh does), then it might be that shared
library which doesn't exist.  Yet you say you can run it as an ordinary
user.  Are you sure?  Is it possible you have /bin/tcsh and /usr/bin/tcsh
and only one of them works?

Try:

  joe-user$ /bin/tcsh
  tcsh%

Then try:

  root# /bin/tcsh

Does it really work for one and not the other?  If it really works for
both, make sure the shell really is set correctly in /etc/passwd:

  $ grep root /etc/passwd
  root:Khhl129V.298:0:0:Super User:/root:/bin/tcsh

Have fun.
