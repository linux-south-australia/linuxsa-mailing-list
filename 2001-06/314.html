Re: LPD encryption

On Fri, 8 Jun 2001, Glen Turner wrote:

> Dan Shearer wrote:
> > 
> > > Does anyone know if it is possible to send an encryted print job via LPD to
> > > a Win32 box.
> > 
> > This is in a general class of problem that says: "how do I get protocol X
> > encrypted if I'm using a Windows box at one or even both ends of a
> > connection and the Windows software has, of course, no clue about
> > encyption?"
> > 
> > The answer is much the same as it is on Unix, where you wrap the protocol
> > with ssh.
> 
> If the printer is PostScript, you can also encrypt the print
> job at source and let the printer decrypt them.  Spoolers then
> need no crypto capacility.

Is there any way of doing this other than having an IPP
(http://www.pwg.org/ipp) compliant printer and printing system (eg
http://www.cups.org) that doesn't involve carefully matching encrypted
printer models with binary-only US-export-grade Windows drivers? I'm not
aware of IPP-compliant printers with crypto enabled.

It is thankfully a long time since I had to do this, but last I knew the
only practical way to do a print spooling filter on NT (eg for encrypting
postscript as generated by the Microsoft Postscript Driver, which is
pretty good quality) was to write a DLL in C.
