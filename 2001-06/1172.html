Re: ppp dialin client cant resolve names.

You're dialin client can't find a nameserver or resolve names, but squid
can probably because it knows a name server. Try setting the dialin
client such that it knows where a name server is (by adding a name
server address to /etc/resolv.conf).

Is 192.168.22.1 really a name server? I think that PPP setting says "if
the client is using a ppp daemon capable of understanding the MS
nameserver extensions, then I'll set that as my name server addresses".
