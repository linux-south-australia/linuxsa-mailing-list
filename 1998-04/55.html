Re: long file names on cdroms with redhat

On Sun, 19 Apr 1998, Michael Pearson - Alcaron wrote:
> I recently had a cd burnt from a vfat file system, which contained long
> file names. my friend's slackware box recognises these lfn's, but my

If your friends kernel is (I think), in the 2.1 series, it will recognise
the "Joliet" long file naming scheme.  Kernels prior to that though, i.e
Redhat 5.0, will not recognise the lfns.

Instead of Joliet, you can use "Rock Ridge" - a more standard and common
scheme; lfns will then be recognised on all unixes.

Why did Microsoft devise Joliet you ask?  Why did they not just comply
with the already well-established Rock Ridge you ask?

A: Because they're Microsoft!
