Re: F3 dies under X

To both of the X related messages,

Chances are either your Xdefaults are getting in the way or your
window manager is getting in the way.

If you have upgraded from RH 4.2 there is an errata with the
.XClients file, but there may still be compatibility problems.

Try creating a new user, clearing out all X related files.

Login as the user at a console and start X, using

	X &

all by itself.  And putting it in the background.

When X starts up switch back to the virtual console, and
run the program that is causing the problem using

	xterm -display :0.0  

I think that the <ALT-F3> problem should work.  

If it still exhibits the problem you might have something deeper.

If not then try things incrementelly, add a window manager, 
copy some of your problematic files over and see how it goes.

Tell us how it goes.

Cheers,
