The first one is how a shell normally responds to a nonexistant command.

Its the second alternative that needs investigation.  Here are some
suggestions:
- you may have an alias called wine that attempts to start a file called
wine through an absolute reference to the current directory,  
- the $PATH variable may be incorrectly set in the second case, 

It would be helpfull to see what the command "which wine" turns up.

>& How do i make a 3MB file 2 1.5MB files & reassemble them?
>

Sounds like you need to carry a large file from one machine to another via
floppy disks.  Here's how I do it:

dd if=LARGEFILE of=file_1 bs=1k count=1440
dd if=LARGEFILE of=file_2 bs=1k skip=1440

Then, at the destination:

cat file_1 file_2 > LARGEFILE

Have a look at the man page dd(1), the key points in the above example are:
if = input file
of = output file
bs=block size
1440 : use this number to fill up a 3.5" floppy.

If you wanted the first file exactly 1.5 million bytes, then the following
combination would be applicable:
....  bs=1 count=1500000
