> Lo and behold it had no support for the DPT SCSI controller that the
> CD-ROM and 4GB were hooked up to. The model is PM2044W. I got it on by

Hmmm... the Red Hat Linux Hardware Compatibility Listing lists the
PM2044 as a supported SCSI controller.  The driver for it (eata_dma)
is in the standard Linux kernel.

> borrowing a IDE CD-ROM from another computer. I now want to setup the
> SCSI devices so we can use them. I was wondering if anyone knows if any
> Linux drivers exist for the specified SCSI card? And if so where can i
> obtain them (obviously) and how do I add them to the system?

Does the output of 'cat /proc/pci' mention the card?

If you're building your own kernel, just add the eata_dma support.  If
you want to do it with the kernel you get from Red Hat, start off by
adding the following line to /etc/conf.modules:

alias scsi_hostadapter eata_dma

then run 'mkinitrd /boot/initrd-2.0.0 2.0.0' (where 2.0.0 is your
kernel version)

and add a line like this to your /etc/lilo.conf under the appropriate
'image=' line:

initrd=/boot/initrd-2.0.0

(and of course, don't forget to rerun lilo)

Hope this helps,
