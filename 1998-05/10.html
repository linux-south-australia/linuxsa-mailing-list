Damn the carriage return

Hello Everyone,
                         I am using Red Hat Linux ( version 4.2 ) on a
DX2 486. I use Linux mainly for C programming. I have encountered a
major difference between Linux and Unix in regards to the carriage
return character. For example if I write a simple while loop that keeps
scanning a string until it encounters a return char, Linux forces the
user to press return twice in order for the user to continue using the
program. Unix however has no problem at all with a loop structured this
way. I have no idea why this is, I have been told that it may have
something to do with the keyboard buffer. If anyone has any ideas or
suggestions I would be happy to hear from them.
