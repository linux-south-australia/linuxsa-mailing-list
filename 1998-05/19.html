> any suggestions how I can get the grep to read "Apr 3" instead of
> "Apr" "3"  and return all lines in the file with the string "Apr  3" ?

Try the following...

----------------------------------------
#!/bin/bash

MY_DATE="`date +%b%e`"           # Get the returned value as a single string

tail any_file | grep "$MY_DATE"  # Force grep to include spaces in the search
----------------------------------------

I find that 'cat' is usually a better choice than 'tail' in this situation as
it's more obvious what you are doing (unless of course you only want a certain
numer of lines from the file).

Also, if you don't need MY_DATE later on you can of course compress all of this
down to     grep "`date +%b%e`" any_file      to get the same result.

Getting the hang of quoting rules in shell scripts can be tricky at first, but
just remember that anything in quotes is treated as a single argument, including
setting variables (as with MY_DATE above).

Have fun...
