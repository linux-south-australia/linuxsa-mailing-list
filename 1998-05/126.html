Re: DMA xfer vs mem-mapped IO

> Hiyas
> Could anyone tell me what the relative performance of DMA transfers are
> compared to (PCI) memory-mapped IO are?

for copying a block of data from ram to a pci device mapped to memory?, i
would say that they would be very close. what with pipe bursting and
caching, maybe the cpu copying would be a little infront?, but dma
copying is done independant of the cpu, and can go on processing something
else (cached instructions, fetched intructions etc).

i'm going to shut up here because i haven't done any dma programming in a
few years and have forgotten most of the dma mechanics. dma (the bus
negotiation of dma) probably works a little bit differently with these
modern motherboards.

for non-systematic memory copying- use the cpu, for systematic, timed
memory copying, you're far better off using dma as the cpu can do other
things.

hope that helps.
