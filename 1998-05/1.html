Re: Language compilers in Linux

Hi

I'm using dlxasm and sim on linux. dlxasm is tricky to compile but when you
get it working it generates code that is identical to the dlxasm on the
computer science
machines at the uni. As for sim (I couldn't get xdlx to compile) well I
actually prefer it to
xdlx, when you get good with it its much quicker (IMO) than xdlx, it does
however have
a problem, when sim loads the .out file (ie search.out) it reverses all the
data ie 32 bit
data isn't loaded properly, this is fine for the .code area but the .ascii &
.asciiz data
gets messed up, ie "Adelaide" looks like "ledAaide".

I have found this to be simply a small hassel, you can get around it by
doing what I have done,
I generated new data that is much more simple than the provided data and
used this for
program development then when you think your program works take it across
onto
one of the CS machines and put the real data back in.
hope this helps.

Alan Boldock
Adelaide Uni Dept of Psychology.
