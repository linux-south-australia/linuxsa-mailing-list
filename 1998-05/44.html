There are a few names for it: "backtick", "grave accent" (I think
that's the one), "open single quote".  There's at least one other
name, but 3 is all my memory extends to.

In case someone hasn't explained this before, two backticks around a
command tell the shell to run that command and place its output back
on to the command line, then run the overall command that results.
Eg.:

kill -HUP `cat /var/run/httpd.pid`

translates to:

run the command: cat /var/run/httpd.pid
the output is eg.: 123
so the shell then runs: kill -HUP 123
