Re: Linux & Dial-Up Internet / Netscape

On Wed, 20 May 1998, Adam Hawes wrote:

> 1.  I want to use my Linux box for Internet access, but at present my
> ISP has a byte limit of 25mb/week. I would like to know if it is
> possible to set up a byte (download and upload) counter in Linux, or
> whether I will have to stick with my windoze box (which does have a byte
> counter for modem activity) until my account runs out and I get
> something else.

IP Accounting should do the job:
ipfwadm -A in -i -W ppp0

Use "ipfwadm -A -l" to list the rules and counters. There are lots of
other options you can use with ipfwadm.

This alone won't do it however. If you reboot during the week the counters
will be lost so you need to record them in a file everytime you shutdown.
You will either have a write a couple a of scripts to manage the counters,
or find a package which does it for you. Does anyone know of a package
which can do this sort of thing easily? I have some scripts which work for
me but they are very hackish and I doubt they would work in any other
situation :).

"pppstats" and "ifconfig ppp0" will also return link usage. However they
will only work while the ppp link is running, so IP accounting is probably
better.

> 2.  I have noticed that Netscape 4 (shipped with Red Hat 5) drops out
> seemingly at random. I have seen the same thing on the Unix system at
> Uni, where netscape simply disappears from the screen. Is there a
> solution, or a patch to stop this from happening?

Redhat 5 has Netscape 4.04 doesn't it? Netscape 4.05 was released a while
ago. I run Netscape with Java & Javascript turned off. It has never
crashed on me like this. With Java & Javascript turned on it tends to be a
little unstable :(.

You could try telling Netscape to use different libraries instead of the
libc and libxpm that come with Redhat. There are a few pages on the net
which describe how to wrap Netscape in this way. They also provide the
libraries which have been specifically compiled to try to minimize the
bugs in Netscape. It may work for you, it may not..
