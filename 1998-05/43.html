Re: Stopping Apache

> Q2 - Almost the same as Q1. How do I force Apache to re-read the
> configuration files? The HUP command doesn't seem to work either.

Like everyone else said, but in addition, you can use 'USR1' instead
of 'HUP'.  USR1 is a graceful restart, which means that any requests
that apache is currently serving will be allowed to complete instead
of being terminated.
