Stopping Apache

Two questions

Q1 - How do I stop Apache? The Apache manual says use the command:
kill -TERM httpd.pid, but when I send the command as root, the computer
returns: kill: no such pid httpd.pid. Yet I issue the command from the
directory were httpd.pid exists or with the full path. When I issue the
ps -x -a command, httpd.pid is clearly running and so is the Web server.

Q2 - Almost the same as Q1. How do I force Apache to re-read the
configuration files? The HUP command doesn't seem to work either.
