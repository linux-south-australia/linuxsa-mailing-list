My guess is that you built a kernel of a version for which you already
had some modules installed.  The modules you built should work fine;
the errors should be mentioning modules that you didn't compile.

To avoid the warnings: before running "make modules_install", go to
your /lib/modules directory and move the "2.0.27" (or whatever)
directory out of the way (eg. "mv 2.0.27 2.0.27.backup").

> I can't really tell if the modules work or not (Like i said - i plan
> to use them l8ter,) but they appear to install OK.

A reasonably easy way to tell if your modules work is to build things
like the floppy and printer port drivers as modules, and check to see
if those devices still work afterwards :).
