Actually, that says it should NOT be portmapper, but portmap.

The portmap man page confirms this:

     This portmap version is protected by the tcp wrapper library. You
     have to give the clients access to portmap if they should be
     allowed to use it. To allow connects from clients of the .bar.com
     domain you could use the following line in /etc/hosts.allow:

     portmap: .bar.com

     You have to use the daemon name portmap for the daemon name (even
     if the binary has a different name)
