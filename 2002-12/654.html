I am running FreeBSD 4.6 STABLE on a d-link equipped wireless AP.
It is cranking along, and everything is running peachey apart from one
thing i cant work out regarding executing certain commands on bootup to
keep it an AP instead of a node. THis isnt a wireless question, it is an
OS question, so please bear with me :)

I have put my sysctl commands in /etc/sysctl.conf which is good.

I need to know where i can put the following:

1.	kldload bridge - my guess would be in /boot/loader.conf as it is a
	loadable kernel module

2.	various wicontrol commands - i was thinking in rc.local

3.	various ifconfig commands - quite sure these go in rc.conf with the
	others.

The sequence is important - it must go:

1. kldload bridge
2. sysctl commands
3. wicontrol commands
4. ifconfig commands
