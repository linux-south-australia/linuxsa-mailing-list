[OT]: Cisco PIX IDS

I've got this nice, shiny, $10K Cisco PIX 515E sitting on the floor
next to me at the moment, and I'm currently doing some thinking about
how I'm going to configure it.

My question is about the IDS that this little beastie has. I'd just
like to know what the scope of it is.

For example, there's protocol "fixups" configured; http, ftp, smtp and
a few others all seem to be buffered on the PIX and checked for bad
stuff before they're sent through to the server.

Things like Code Red, which can be easily identified by looking at the
HTTP request *should* be stopped by Internode, before they even get to
us, but there's bound to be some other thing pop up in similar fashion
one of these days, and when that day comes, I want to be able to
update the PIX and have it deal with things, rather than my webserver.

Is that kind of thing part of the PIX IDS job description, or should I
be looking somewhere else?

I've seen products out on the web that plug into the URL filter
mechanism within the PIX, mentioning something about being able to
block Code Red, but I'm under the impression that the URL filter only
applies to *outgoing* traffic, rather than incomming or both.

If someone could clear up the confusion, I'd be most grateful.
