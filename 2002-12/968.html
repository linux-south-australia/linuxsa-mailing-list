> GUI interfaces (using something such as glade, kdevelop or some other
> gui maker) and do general purpose programming.
> 
> I lean towards Perl myself, but I wonder what people's thoughts about

I'm a big fan of perl for most things, but probably not native GUI
stuff. Python is perhaps a little better suited, but IIRC there's a
little bit of work installing everything you need to have Glade generate
code for you (as is probably the case with perl).

Python has the advantage of forcing people into doing many things The
Right Way(tm). Things like indentation and whitespace, which are
optional in perl are forced in python. This and other features make
Python a reasonable choice for someone who is learning to code.

Hope it helps
Matt
