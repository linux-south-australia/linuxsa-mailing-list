This is exactly the point people need to take into account when they 
point at x86 installers for debian based systems and say "why can't 
debian do it like that?". Debian supports many more architectures than 
just x86, so to keep things consistent the installer needs to be 
portable across all those architectures.

IMO The main problems with the debian installer are:

1. It implies that you should use tasksel and/or dselect to install your 
optional packages.

2. The X setup normally leads to X getting set up to use /dev/mouse 
which doesn't exist by default and so X never usually starts first time.

3. There is essentially no hardware detection, which is ok if you know 
what hardware you have and what drivers are required, but otherwise can 
be more annoying.

The hardware detection can be done manually, at least on x86 using 
apt-gettable tools (or what is becomming my preferred method - stick 
knoppix in and see what drivers it comes up with) but is not portable 
obviously. Package selection through apt/aptitude is much nicer than 
dselect, and doesn't pull in a plethora of packages you didn't really 
want, as happens when installing stuff using tasksel.
