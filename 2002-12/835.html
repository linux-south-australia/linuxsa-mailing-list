Windows is probably driving it harder.  Unix kernels go into a halt wait
state when they have nothing to do (e.g. tasks waiting on disk I/O), and
are woken up by interrupts.  Windows doesn't appear to be built on this
model (or is always so damn busy it never gets the opportunity to halt).
