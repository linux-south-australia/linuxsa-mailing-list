Brian et al,

> Trouble is, lilo looks for lilo.conf before it will install.  Of
> course it can't find it where it should be in /etc, because in rescue
> mode it is necessary to mount the partitions in separate directories,
> so lilo.conf is in /mytemp5/etc.  I might have copied lilo.conf to the

You want to do something like:

lilo -r /mytemp5

This tells lilo to pretend that "/mytemp5" is the root directory "/"
BEFORE it attempts to look for configuration files, boot sectors and so
forth.

A slightly more complex-but possibly more sane way especially if you've
got different versions of lilo and other issues-would be to use "chroot"
itself:

chroot /mytemp5
lilo
^D

[i.e. control d]

man lilo
man chroot
