RE: Report tool

Andrew Galdes <AGaldes@elders.com.au> wrote:
> This is the result and it works fine.
>cat /my/log/file.log | sort | cut -d'.' -f1 | uniq -c

Don't say "cut something | sort" because that's just making the system
work harder than it needs to.  Say "sort something"; it's more efficient.
Then, don't say "sort | cut", say "cut | sort" because that's even
more efficient.  The efficiency comes from sending less data down each
stage of the pipeline.  In your case, you're sending the complete file
to sort when sort could read the file directly (thus cat doesn't run).
Then saying "sort | cat" makes sort sort the complete file and send
it to cut, whereas if you say "cut | sort" you only send the field you
want to sort, and better still, it only has to sort the field you want
(instead of the whole file.)

If you're not interested in field counts you can use "sort -u" instead of
"uniq".
