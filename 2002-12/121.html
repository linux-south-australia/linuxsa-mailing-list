RE: Database migration

One obvious way of migrating would be to use a dump/restore approach. Most
SQL based databases have a facility to write  tables as SQL. In MySQL it's
called dumping. In SQL Server it's called scripting. It may be that
Pervasive SQL 2000 has this sort of facility. You should then be able to
execute the SQL on the target to load the data. The only problem you might
have would be variations in SQL syntax but as the SQL used for this sort of
thing is rather "vanilla" this is unlikely to be a problem.
