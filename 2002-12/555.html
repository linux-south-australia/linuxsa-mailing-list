Since you are referencing redhat.com below, I'am assuming you're using Red

> According to some documentation I've found:
> http://www.redhat.com/mirrors/LDP/HOWTO/Kernel-HOWTO-2.html
> 
> I need to run 'make config' or 'make menuconfig' (as I'm working from the 
> command line only, no X), however I can't get either of them to run, 
> this is the answer I get after I do 'make config'
> make: *** No rule to make target `config'.  Stop.

Have you cd'd into /usr/src/linux?

To reiterate, if you're doing the Red Hat kernel rebuild thing, you probably 
want to:

-1) cd /usr/src/linux
0) Update "EXTRAVERSION" in /usr/src/linux/Makefile appropriately
1) make mrproper
2) cp Configs/whatever-is-an-appropriate-starting-conf-file .config
3) make oldconfig
4) make config (or xconfig or menuconfig or whatever)
5) make dep
6) make bzImage
7) make modules
8) make modules_install
9) copy bzImage /boot/somemeaningful-name
10) Update /etc/lilo.conf or /etc/grub.conf appropriately and run lilo if 
appropriate.

Of course be careful!  Have a boot floppy ready before trying your new kernel 
in case you stuff up.  Or know how to do a CD rescue.  Or employ Netcraft 
Australia to build you a new kernel[2] :)
