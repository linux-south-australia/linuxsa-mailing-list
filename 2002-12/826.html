Hi Damian..

Have you looked at "lmsensors":

http://secure.netroedge.com/~lm78/

Most distro's have a package available.  It requires kernel support for the
I2C bus which you might need to compile in and of course you need an SM-bus
on your mainboard, but most do these days.  The package itself doesn't do
anything special other than report to stdout the various parameters on the
SM-Bus.  Here is an example of the output of mine:

adm1025-i2c-0-2d
Adapter: SMBus I801 adapter at efa0
Algorithm: Non-I2C SMBus adapter
+2.5V:     +2.51 V  (min =  +2.22 V, max =  +2.72 V)
+3.3V:     +3.30 V  (min =  +2.93 V, max =  +3.59 V)
+5V:       +4.97 V  (min =  +4.45 V, max =  +5.44 V)
+12V:     +11.87 V  (min = +10.68 V, max = +13.06 V)
VCC:       +3.40 V  (min =  +2.93 V, max =  +3.59 V)
SYS Temp: +46.0°C   (min =  +0°C, max = +60°C)
CPU Temp: +51.0°C   (min =  +0°C, max = +60°C)

Darn setiathome keeps the CPU hot! :)  Anyway I used a small awk program
called by a cron job to produce a log file that looks like this (updated on
the minute).

Dec 17 17:02:59 alpha sensors: CPU Temp: +51.0°C, System Temp: +46.0°C
Dec 17 17:03:59 alpha sensors: CPU Temp: +52.0°C, System Temp: +46.0°C
Dec 17 17:05:00 alpha sensors: CPU Temp: +51.0°C, System Temp: +46.0°C
Dec 17 17:06:01 alpha sensors: CPU Temp: +51.0°C, System Temp: +46.0°C
Dec 17 17:06:59 alpha sensors: CPU Temp: +51.0°C, System Temp: +46.0°C
Dec 17 17:08:00 alpha sensors: CPU Temp: +51.0°C, System Temp: +46.0°C
...etc, etc.

It is handy to see what's happening inside the server box, especially when
I'm interstate for a few weeks and it gets a bit warm back home :)

Hope that is of some help?

Cheers,
Graham
