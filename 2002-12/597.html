RE: SAMBA in RH7.2

>But I probably need something more specific to the RH7.2 install.

Not a whole lot to it really.  We've several boxes still running
Samba on RH7.2, and all of them have been installed using the makerpms
script that is packaged with the source, though I usually modify the 
samba2.spec file before compiling to add in ACL support (after recompiling
a new kernel with ACL support and installing the ACL utils obviously...)

Both the makerpms script and the spec file is located in 
./samba-srcdir/packaging/RedHat
and its as simple as checking the spec file for the configure options you
require, then running "sh makerpms.sh".  If it does it's business then 
you should end up with a samba rpm in /usr/src/redhat/RPMS/i386.

(for a full list of the possible configure switches, run "./configure
--help"
from the ./samba-srcdir/source directory)


>Where does Samba install itself to?
>How do I find & read the man pages?
>Where do I find/put the smb.conf file?

If you install using the above procedure, you can do an "rpm -qpl
samba.x.x.x.rpm",
which will list all the files contained in the said rpm and the full path to
each.

Ditto for the man pages, they should be listed from rpm -qpl.

smb.conf is usually installed to /etc/samba/ these days, some distros still
put it
in /etc, or create a symlink in /etc to point to the file in /etc/samba.
The above
rpm puts it in /etc/samba, along with smbpasswd and so forth.

HTH
