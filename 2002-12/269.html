Re: grr @ grep syntax

Adam Smith <adam.smith@sageautomation.com> wrote:
> Just briefly, I'm attempting get a list of all files with '.cdr' in the
> filename.  The following syntaxes from $directory give me the identical
> results:
> 
> find * | grep .cdr
> 
> find * | grep \.cdr

Aside from the improper and inefficient use of find, which others have
already discussed, the reason the two different commands give identical
results is that the backslash in the second command is a shell-escape.
It supresses any special meaning that the shell might give the following
character; thus the argument passed to grep is in fact .cdr.  You do want
to escape the dot to grep, so you need to pass \. , which can be done
as "\.cdr" (ie using quotes, either double or single), or as \\.cdr.
The latter is a shell-escape: The first backslash removes the special
meaning from the second, in other words it becomes a literal backslash.
