crontab/crondaily questions

Every morning when I get in to work, one of my two debian (woody) boxes is
running cron.daily (this is at 7.30ish), which would be be fine if it
weren't for the fact that cron.daily includes the sceurity check scripts
which are hideously disk intensive and really prevent me from using the
machine for the first 10 minutes I'm here.

This iriated me so much I changed the relevant crontab line so:
25 3	* * *	root	test -e /usr/sbin/anacron || run-parts --report /etc/cron.daily

However, it still runs around 7:30 in the morning

Questions:
1. Why? I thought crontab file entries in debian needed no other action
than to edit the file, indeed:
# /etc/crontab: system-wide crontab
# Unlike any other crontab you don't have to run the `crontab'
# command to install the new version when you edit this file.

2. The other machine also changed (6:30 start since it's less often used
interactively and so is less irritating when it doesn't respond quickly),
but doesn't seem to run at 7:30.

3. Should I just come into work 10 minutes later?


thanks
Dan
