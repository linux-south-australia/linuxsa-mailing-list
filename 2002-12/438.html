> curmbox=F000000001&a=b7e80126f407abc5055ce88bfdfcc0b7&mailto=1&to=srebi 
> @console.com&msg=MSG1039447518.136&start=1560329&len=2478&src=&type=x>>

> Relaying denied
>

One way to fix this would be to add the following to your sendmail.cf  
file:

# Access list database (for spam stomping)
Kaccess hash -o /etc/mail/access.db

Then create a file called: /etc/mail/access containing:

127.0.0.1 RELAY
a.b.c RELAY

(Where a.b.c is your local lan's subnet eg 192.168.0)
You should then be able to send mail.

> Secondly,
>
> When i send mails from the same mail server ..it gives
> an error saying " no route to host" there by not
> delivering the mail.
>

This could be a couple of things.
Do you have a default route ?
Do you have a valid DNS server (that you can get to) in your  
resolv.conf ?
