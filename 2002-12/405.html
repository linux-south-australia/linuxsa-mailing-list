ssh public key auth

I have a couple boxen at work (one is slackware and the other I think is
redhat) both have the same version of sshd and no differences between
/etc/ssh/sshd_config. Both machines have a user called backup and on
each machine I used that user to generate an ssh keypair. Used scp to
copy ~/.ssh/id_dsa.pub to ~/.ssh/authorized_keys2 on the remote machine.

The problem is this: logging in from one machine to the other doesn't
prompt for a password, yet the other way around it still asks for a
password[1].

Seems very weird to me, as (as far as I know) they are both set up the
same. I used the exact same commands to create the keypairs, to copy the
public key over, and I double checked the permissions.

If anyone has any suggestions for when I next get to have a go at
getting the thing working, it would be much appreciated.
