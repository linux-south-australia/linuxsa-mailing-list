Re: binding < port 1024 as non-root (was: RE: simple squid setup)

Richard Russell wrote:

> (I'm actually
> thinking of some sort of ACL for inet sockets, like you can do with files...

There is a capability called CAP_NET_BIND_SERVICE for allowing a non-root
process to bind to ports below 1024; unfortunately, it allows the process
to bind to any port, not just a specific one.  I understand that NT allows
you to specify ACLs on any system object, which may include TCP/IP ports
(I know that it does include file and processes, but you know how
orthogonal Microsoft think).

> perhaps one could set up some sort of port forwarding to a higher port and
> restrict that, rather than relaxing restrictions on a <1024 port.

That'd work too - ipchains does has a redirect function.  The down side
would be that anyone could run a listener on that port if the intended
one wasn't running.  It's always about trust.
