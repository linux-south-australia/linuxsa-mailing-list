RE: binding < port 1024 as non-root (was: RE: simple squid setup)

> Actually, now that you mention it (again) -- is there any way that inetd can
> be used to pass an open socket to a user process that is already running?

I'm hazy on the details, but BSD lets you pass file descriptors through
a socket.  That's right: One process can open a file, and then pass
its file descriptor, through a socket, to another process.  I believe
they need to be unix domain sockets, which limits both processes to
the same machine (nor surprising.)  Linux might have the same facility.
(NB an open socket is accessed via a file descriptor.)

Inetd doesn't do this, but yo8 could modified it.  Of course, then it
wouldn't be inetd.
