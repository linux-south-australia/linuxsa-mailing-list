> I want to use mutt for a mail client, on my slow(er) home machine. Mutt
> works fine, but it has one drawback - it seems (as far as I can tell) to
> send mail by connecting to the SMTP port on the local machine, and I
> can't get Sendmail to talk coherently to the outside world.
> 
> It breaks (as far as I can tell) because it doesn't present a registered

You might look for the stuff below in /etc/sendmail.cf....

##################
#   local info   #
##################

Cwlocalhost

# my official domain name
# ... define this only if sendmail cannot automatically determine your domain
#Dj$w.Foo.COM
Djmail.bigpond.com  <-------- change this to your mail server but leave the
Dj as it's kinda important!

Let me know if it works......
