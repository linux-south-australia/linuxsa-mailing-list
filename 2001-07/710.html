RE: Shared email across multiple OS on one system

> I'd like to maintain a single mail store and have it accessible from
> Linux/Win98/Win2k and to be able to download new messages (under whichever
> OS is running at the time) from multiple POP servers while leaving the
> messages on those servers.

you sound like you don't really want to leave them on the servers... if you
do, try running IMAP servers instead of POP -- then you don't need a local
mail store...

> While I've had a degree of success with sharing a common message store
> (using various MUAs available for those Operating Systems), the main
> stumbling block is that the tracking of the most recently downloaded
> messages isn't shared, so whenever I switch OS I'll be
> downloading the same
> messages a second or third time. This is a problem due to the
> combination of
> a large volume of messages and the large size of some of them.

personally, I'd either set your MUAs to remove messages, or, if you want,
have one remove messages, while the others leave them there -- that way you
always have all your messages in one place, yet you can see new messages
when you are on a different OS...

> I've just tried running a multi-platform mail server (CommuniGate
> Pro) which
> has an RPOP facility and connecting to that using IMAP. Unfortunately it
> suffers the same issue (multiple downloads from the POP servers) when
> switching between OSes.
>
> Anyone have any ideas on how this might be achieved or am I simply asking
> too much? I'm not having much luck finding references to this
> scenario in my
> web searches. Is running a web server on a separate machine the
> only answer?

I think this depends on what you are really trying to acheive here... why do
you want to do this? (not that there aren't any good reasons, in fact there
are a few, but what are _your_ reasons?)
