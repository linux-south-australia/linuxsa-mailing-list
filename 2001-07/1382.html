Alan Kennington wrote:
> This sort of argues in favour of just not using the mailto:
> construction in HTML, because anyone who really
> wants to make contact can do a copy/paste anyway.
> 
> But even if SirCam isn't currently able to parse e-mail addresses
> without the mailto: anchor prefix, a later variant _may_ be able
> to do so.

We already HAVE this problem with spam harvesters! Ways to confound these 
include

 - Mangle the address in the link eg:
   "mailto:someREMOVE-THISone@example.com">someone@example.com<

 - Build the address from character codes eg:
   "mailto:GHd...

 - Build the address with Javascript eg:
   document.write('mailto:someone')
   document.write('@')
   document.write('example.com')

Unfortunately there are shortcomings with all of these methods. And 
cut-&-paste immediately imposes a convenience problem. People have got 
used to the "click on an address and your emailer automatically appears" 
paradigm...
