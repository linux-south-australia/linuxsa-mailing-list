Re: SCSI low level format

David Newall wrote:

>> It is unlikely (extremely) that the raid card would be able to store
>> information via SCSI that a normal partitioning utility or just plain
>> "dd if=/dev/zero of=/dev/sda" won't be able to access.  Most
>> drives don't support the concept of a "low level format" these
>> days.

> Wrong.  Completely wrong.  And by the way, SCSI disks DO support
> low level formatting.

"The Format Unit command (operation code x'04') writes zeros to
 all sectors of the Logical Unit."  - One of IBM's SCSI storage
PDF books, and not dissimilar to the dd command I gave above.

In addition, "If FmtDat is one, then the target will receive a
Defect List via Data Out." has a one to one correspondence to
the Reassign Blocks command, so Format Unit still doesn't do
anything particularly special.  Which SCSI command were you
thinking of?

And I'm still curious as to how RAID data could be stored that
you can't get at.
