Funny DHCP/NFS Problems

Hello

I am trying to do a network install of red hat 7.1 from a 7.1 server. I am
on a closed network that does not have a gateway or nameserver.

I have set up a simple dhcpd.conf file in /etc that only allocates ip
addresses based on a fixed MAC address.

I also have nfs set up to serve out a directory containing a red hat 7.1
image.

I am booting via a boot disk created by using dd to copy across the
bootnet.img file.

The process works fine until the network parameters and image mount point
need to be set. If I select DHCP the install does not get past this point.
An error window comes up stating that it was unable to mount the directory
and the console displays another message regarding an inability to contact
the portmapper on the server node. However when I check the logs I can see
that an IP address has been served OK.

If I use a fixed based address and specify NO gateway the process works.
With a gateway it falls over with the same error messages as above.

I should state the the red hat OS on the server is vanilla, with only
dhcpd.conf, exports, telnet, wu-ftp and the hosts files altered.

Any suggestions as to whats going on would be great!

Thanks
