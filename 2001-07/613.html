Re: PHP Require and Include

On Thu, Jul 12, 2001 at 12:45:11PM +0930, Andrew Halliday wrote:

> Include will do the C/C++ equivalent of #include *every time you run that
> statement*.  Require will only ever include the file once, PHP keeps tabs of
> what files have been included and doesnt include one file more than once.
> This protects against recursive inclusion.

Incorrect. The require_once() and include_once() functions do what
you've outlined above.

The difference between require() and include() is clearly stated in the
require() page in the manual:

"Unlike include(), require() will always read in the target file, even if
the line it's on never executes. If you want to conditionally include a
file, use include(). The conditional statement won't affect the
require(). However, if the line on which the require() occurs is not
executed, neither will any of the code in the target file be executed."

In real terms, require() and include() aren't really that much
different. However, it was rumoured that require() is less resource
intensive, but I'm not sure.
