Reza Roboubi wrote:
> 
> ps AlH > out
> 
> and it always truncates something.

If you are using "ps" for further processing by a program and
don't care for inter-Unix compatibility, it is much easier to
use the files under /proc.  Each process has a directory
with its Process ID and under that directory is a lot of
useful stuff about the process.

eg:
  cat /proc/1/status
