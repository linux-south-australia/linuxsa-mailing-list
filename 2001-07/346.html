Re: making changes to eth0 stick

Evan Bourlotos writes:
> 
> Where it has (I assume) it will not put the MTU variable in the ifconfig
> command line because there is no reference to it other than the one place.
> It makes ense as typically you will change the MTU for ppp but I have to
> change it for dsl to something <= 1470 I think. I set it to 1002.

Well if DSL is the only reason you're doing it, and you're using pppoe
(the roaring penguin version) have a look at the -m option implemented
on the more recent versions:

       -m MSS Causes  pppoe to clamp the TCP maximum segment size
              at the specified value.  Because of PPPoE overhead,
              the  maximum segment size for PPPoE is smaller than
              for  normal  Ethernet  encapsulation.   This  could
              cause problems for machines on a LAN behind a gate­
              way using PPPoE.  If you have a LAN behind a  gate­
              way, and the gateway connects to the Internet using
              PPPoE, you are strongly recommended  to  use  a  -m
              1412  option.  This avoids having to set the MTU on
              all the hosts on the LAN.
