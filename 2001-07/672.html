Re: PHP Require and Include

> On Thu, Jul 12, 2001 at 12:45:11PM +0930, Andrew Halliday wrote:
>
> > Include will do the C/C++ equivalent of #include *every time you run
that
> > statement*.  Require will only ever include the file once, PHP keeps
tabs of
> > what files have been included and doesnt include one file more than
once.
> > This protects against recursive inclusion.
>
> I didn't comment specifically on your what you've said about recursive
> inclusion, which is worth noting.
>
> require() does NOT protect aginst recursive inclusion. It will loop
> until it times out.
>
> For example, if you were to have two files, index.php and test.php
> containing the following:
>
> ---index.php---
>
> print ("I'm index.php<br>") ;
> require ("./test.php") ;
>
> ---test.php---
>
> print ("I'm test.php<br>") ;
> require ("index.php");
>
> Loading index.php in a browser will cause PHP to execute the code
> line-by-line:
>
> Print out the text "I'm index.php<br>"
> Load and evaluate the file "./test.php"
> Print out the text "I'm test.php<br>"
> Load and evaluate the file "./index.php"
> ...
>
> Substituting require() for require_once() would have a different effect:
>
> Print out the text "I'm index.php<br>"
> Load and evaluate the file "./test.php"
> Print out the text "I'm test.php"
> Load and evaluate the file "./index.php"
> Print out the text "I'm index.php<br>"
> END (PHP has already included "./test.php" from index.php

Yep yep, thats all cool.  I just cant believe i made that initial mistake -
i think someone once told me that to begin with ... so when i saw the
