making changes to eth0 stick

Hi list

A while ago I posted a query re changes using ifconfig eth0
As suggested by Richard Sharpe..

I looked in my ifup script and the only reference to using the MTU value
dfined in /etc/sysconfig/network-scripts/ifcfg-eth0 was for ppp not for eth0.
This was strange so I modified the script
/etc/sysconfig/network-scripts/ifup to use the defined MTU value. This
solved my problem 

Now I am curious is this likely an omission by RH in their setup or is it
common?
Any comments appreciated.

Evan


At 05:21 PM 5/30/01 +0930, Matthew Geddes wrote:
>Evan Bourlotos wrote:
>> 
>> Hi all,
>> 
>> I was wondering how I can through ifconfig change a value and make it stick
>> forever (well until I change it to something else).
>> 
>> I keep changing mtu to a lower value than 1500 but it keeps going back on
>> reboot.
>> 
>
>If you're using a RedHat-based distribution, you'll want to modify the
>ifconfig command that lives in /etc/rc.d/init.d/network.

Definitely not!

If you check /etc/sysconfig/network-scripts/ifup, you will find that it
uses the variable $MTU.

I suggest you add 'MTU=value' to /etc/sysconfig/network-scripts/ifcfg-eth0
or whatever device you want to modify, and, voila, on the next reboot, you
should find that your MTU is correct.

What is surprising is that so few check these scripts ...
