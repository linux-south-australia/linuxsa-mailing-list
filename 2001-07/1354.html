Re: IJF, NAT & NetMeeting/CuSeeMe

35 seconds' search on www.microsoft.com yields:


http://www.microsoft.com/Windows/NetMeeting/Corp/ResKit/appendixb/default.asp

>NetMeeting and other H.323 compliant audio/video programs are not 
>designed to work with network address translation (NAT).
>                                     NAT translates all IP addresses 
>on a local area network (LAN) to a single routable IP address.
>
>                                     There are some T.120 and H.323 
>compliant NAT implementations that work with NetMeeting. To 
>determine if your NAT connection
>                                     is compliant, consult the 
>documentation for your NAT product.

Why don't you stop using NAT, or get a better NAT firewall?

If your NAT thingy is configurable, try configuring it to bend all 
non-specified incoming port access attempts to the machine running 
Netmeeting. The problem will be that it's trying to pull udp frames 
in from the remote site on a port number that you aren't properly 
handling in your NAT configuration.

A number of NAT devices allow this sort of thing - often it's called 
'exposed IP address' mode or similar - a mode where all incoming 
frames that don't have a home to go to get sent to a specific machine 
behind the NAT gateway.
