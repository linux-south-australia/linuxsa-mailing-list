Re: DNS Caching

Mick Collins wrote:
> 
> Is it possable to cache incomming dns requests?

Yes.  It's called a "caching-only DNS forwarder".
Well worth configuring at the end of a slow link.
I wrote a AUSCERT advisory a few years ago on
how to configure one using BIND version 8; have a
look on http://www.auscert.org.au/

Red Hat Linux has a RPM that configured a caching-
only BIND DNS forwarder, but the configuration is
broken.

If you have dynamic addresses, then you need to
rewrite part of the configuration file each
time DHCP assigns a new DNS server.  The DHCP
clients have hooks you can use to do that.

> and my dns to log the ip and if it changes (only
> if the ip changes) it will get it from cache and
> be quicker?

That's not quite the mechanism.  Each DNS name has
a Time To Live that is supplied when the name is
returned.  The forwarder will retain any queried name
until the Time To Live expires.  The next query
against that name will cause the forwarder to
resolve the name externally.  It then again
caches the result and TTL.
