Re: linux X windows screen lock useless?

Virtual terminals are a real pain on public user machines,
and not just from a screensaver point of view.  It's
all too easy to use a VTY, switch and then forget to
log it off.

The usual sort of "terminal lab" configuration to get
around the "I don't want no VTY" problem is:

 - alter /etc/inittab to only run the console
     Delete all lines with gettys from tty1, tty2, ...

 - force X to use the console, change /etc/X11/xdm/Xservers
     :0 local /usr/X11R6/bin/X vt0

 - default to X on startup, alter /etc/inittab so that initdefault
   is run level 5
     id:5:initdefault:

 - modify /etc/syslog.conf to pump messages to a central
   server rather than to the console and run xconsole -iconic
   to grab the few remaining messages

   [the above is from memory and may have minor errors]

If you are running X from the command line to get ssh-agent,
start ssh-agent from ~/.xsession instead.

Adjust accordingly for Gnome Display Manager.

Note that there's lots else in the boot sequence to
secure on terminal-room machines.  I've posted previously
about that.
