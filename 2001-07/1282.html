Re: invoking WvDial on boot

Michael!

> I would like to know what is the best way to invoke WvDial on boot (redhat
> 7.1) and do I have to do it with a "&" to make it detach from the
> terminal???

If you're usual command to do what you want is:

[root@kratz.com]# ifup ppp0

Then use the "onboot=yes" option in the appropriate
/etc/sysconfig/network-scripts/ifcfg-ppp# file.

If you don't use that command (eg, you call wvdial from the command line
yourself) then take a look at /etc/rc.d/init.d/syslog or one of the
files and:

* man bash
* man chkconfig
* man wvdial

That is to say, clone the way the script is written (man chkconfig) -
for bash help (man bash or info bash) - and then you can use the
chkconfig command or ntsysv etc just as you would for httpd or named.

A brief skeleton:

#! /bin/bash
# chkconfig: 35 05 05
# description: my script

case $0

start)

# do something
;;

stop)

# do something
;;

Now, you'll notice that the other scripts will source
/etc/rc.d/init.d/functions and you'll see functions like "daemon".
That's how those OK's etc get printed. I'd suggest you might want to
investigate using them :-) Who knows? If you do it REALLY well you might
be able to convince RedHat et al to patch it into a release...
