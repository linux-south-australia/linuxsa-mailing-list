SOLUTION: NTP QUERY

Hello all

Managed to solve the NTP problems I was having. Was to do with the driver I
was using.

My server line was;

server 127.127.8.0 mode 5

Which indicates a generic driver (with SUN conotations). I changed this to

server 127.127.1.0

Which is a local undisciplined clock (don't worry guys it is not connected
to the net!)

It also took about 10 - 20 mins to synchronize properly.

Other than that all is working well.

NOTE: We will eventually be hooking it to a private hardware time source.
