Re: TTYS Data Loggin

>We have an Alcatel telephone system here at work, and we have a serial
>cable which sends straight data logging every call made and received on
>the phone system. What we wish to do, is capture all the data from the
>PABX to a file, so we can then sift through the log files later to trace
>any misuse of the phone system.

Phone system SMDR data works very well with SQL, if you brew up a few
scripts to import it you'll be able to produce all sorts of colourful graphs
and stats to impress people later on. Not to mention that it's a lot easier
to do all sorts of queries on what phone users are doing, but you might want
to check with the latest privacy/telco legislation before openly doing so..
