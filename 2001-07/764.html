Re: Shared email across multiple OS on one system

"W. Sierke" wrote:
> 
> I'd like to maintain a single mail store and have it accessible from
> Linux/Win98/Win2k and to be able to download new messages (under whichever
> OS is running at the time) from multiple POP servers while leaving the
> messages on those servers.

This is what I do.

Use fetchmail to POP download and delete the mails from the
POP accounts and SMTP deliver them to a local IMAP server.
Filter the incoming mail with procmail before dropping
it into the IMAP server.  Point your various mail readers
to that IMAP server 

Configure procmail to filter each mailing list into its own
IMAP folder.  Also configure procmail to delete duplicated
e-mails (such as when people that send the same message to
you and the list).

Configure sendmail to send outgoing mail to the ISP you
prefer, you might want to set this to never bring
up the interface itself.  Point your mail readers at the
local sendmail for outgoing mail.

You're not asking too much.  Installing the Red Hat fetchmail,
procmail, sendmail, imap pretty much installs the infrastructure
out-of-the-box which you then configure for your circumstances.
e-smith does it even more simply.
