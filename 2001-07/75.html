Network Security

Just a general question regarding firewalling theory that someone may
care to share an opinion on.

If I'm putting in a Firewall for a network connected to two LANs and a
DMZ (such as ours), would it be advisable to run DNS on the border
firewall machine?

I'd have thought that it'd provide a way of getting past the firewall
and into your network if there were remote exploits in the name server
running in the DMZ.

A diagram may provide a slightly more understandable view of the
goings-on:

                                    DMZ
                              /
+------------+ TCP/UDP 53     \     +-------------+
|   Border   |----------------------|  DNS Server |
|  Firewall  |                \     +-------------+
+------------+                /
                              \

I'd have thought that doing this could cause problems if the name server
in the DMZ had some remote exploits (such as some 8.2.x versions of
bind).

Can anyone confirm or reject that suspicion of mine?
