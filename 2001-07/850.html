Re: read from stderr

Hi There!

> can anyone explain me how i can read from a channel such as standard
> error in a bash script?

man bash

Then type "/stdin" and you'll come up with a section that starts:

<huge snip>
              /dev/stdin
                     File descriptor 0 is duplicated.
              /dev/stdout
                     File descriptor 1 is duplicated.
              /dev/stderr
                     File descriptor 2 is duplicated.
              /dev/tcp/host/port
                     If  host  is  a  valid hostname or Internet
address, and port is an integer port number, bash attempts to
                     open a TCP connection to the corresponding socket.
              /dev/udp/host/port
                     If host is a valid hostname or Internet address,
and port is an integer port  number,  bash  attempts  to
                     open a UDP connection to the corresponding socke
</huge snip>


Further on it says:

    The general format for redirecting input is:

              [n]<word

(where n can be a file descriptor, see the numbers above).

So, off the top of my head, one could do 0</dev/stderr, which is to say
hijack the output of stderr and redirect it to /dev/stdin.

Note: I'm only reading man pages here and haven't actually tested what I
just said. There are also probably better ways to do this.
