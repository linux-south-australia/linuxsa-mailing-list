database problems since Samba 2.2.1a installation

Hi List

I upgraded our Samba box from 2.0.7 to 2.2.1a over the weekend.
(replaced really, the 2.0.7 install was blown away before installing
2.2.1a so as to ensure no conflicts)

Since then we've had problems with a FoxPro database, which runs from the
"public"
share (see config below)
It only allows one user at a time if we're lucky (its not consistent),
mostly
the errors are access denied to some of the files, but the users group in
question has full rwx access to that directory (and all subdirs & files)

I've played with the oplock settings and turned strict locking on &
off, but this doesnt seem to make any difference.

The smb.conf file wasn't changed in any way & is the same for both 2.0.7
& 2.2.1a install (except for the veto oplock files, I didnt have that in
2.0.7)

Are there any default parameters which have changed from 2.0.7 to 2.2.1a
which could be causing the problems?



(i haven't included the globals section, but can if need be)


[Public]
	comment = Public Folder
	path = /storage/public
	valid users = @everyone, @users
	admin users = @itstaff
	force group = everyone
	read only = No
	create mask = 0770
	force create mode = 0770
	directory mask = 0770
	force directory mode = 0770
	guest ok = No
	veto oplock files =
/*.dbf/*db/*.cdx/*.DBF/*DB/*.CDX/*CME*/*.fpt/*.FPT/
