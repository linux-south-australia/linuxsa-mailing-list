I have this problem with nmap all the time. I occurs when my ISP
has WWW traffic (port 80) redirected automagically through their
proxy/cache engines (can you say WCCP on Cisco ? :-P ). In this
particular case you probably have an ISP that has been a little
clever and made sure that they have covered all of the bases for
possible WWW traffic (port 80, 3128 & 8080).

Has your box got a Web server running and available (ie not hidden
by ipchains ?) If not try the same test "telnet <host ip> 80".
If you end up with the same result you can be reasonably sure
that it's just a cache redirection that is causing false positives
for nmap.
