Re: postfix

On Sat, Jul 14, 2001 at 12:17:03PM +0930, David Lloyd wrote:

> I'm almost certain an improperly configured Postfix MTA is less secure
> than a correctly configured Sendmail.

Well, duh. If Postfix is configured to relay mail or do other equally
evil things, of course a Sendmail configured to *not* do this would be
more "secure". I'd have thought that would go without saying.

There is a reason why one could consider Postfix more secure than
Sendmail. Firstly, the default configuration for Postfix is pretty
secure. It doesn't need blood, sweat and tears to get it going, which is
more than what can be said for Sendmail.

Secondly, configuring Postfix is considerably easier for the newcommer.
This results in less config botches which results in a decreased
likelyhood of security problems caused by misconfiguration.

I'm not going to get into a "Which is the best MTA" debate. I think my
opinion (and reasons to support that opinion) have been quite clearly
canvased here.
