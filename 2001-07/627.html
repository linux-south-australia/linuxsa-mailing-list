It really depends if you want to be accessing this from other devices, or if
logging on to the machine in question is possible...

I'm sure there would be ways to make this program accessible from outside --
I'm thinking cgi scripting, but there would also be other ways... if you are
always coming from unix, try using `ssh <host> lastlog`, and if you wanted
to do it without passwords, set up ssh's authorised_keys (?) file
correctly... in fact, if you wanted to be really funky, I'm sure you could
set up sudo to mean that only certain users on certain machines could run:
`sudo -l lastlog ssh <host> lastlog [<user>]` or something without a
password... I _think_ that could be made quite secure... :)
