Last time I saw this it was a reverse DNS problem - the POP3
connection was timing out while the server was trying to do a
reverse lookup on the client's IP number.  You should make sure that
your DNS is properly configured and either:
- have valid reverse DNS entries for each client, or
- have valid entries in /etc/hosts for each client.

But I don't know that the above is a complete solution to the problem.
Is the timeout just a symptom of another problem?
