Michael!

> As root, I did:
> /usr/sbin/mysqld --basedir=/ --datadir=/var/lib/mysql --user=mysql
> +--pid-file=/var/lib/mysql/myhost.mydomain.pid --skip-locking
> +--skip-grant-tables

You must start the server itself up as root. MySQL works directly with
the file system and the root user is the only user guaranteed to be able
to actually change the tables. Replace that "--user=mysql" with
"--user=root" and run it as root.
