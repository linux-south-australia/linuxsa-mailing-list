Re: Why do we hate HTML mail?

On Wed, 11 Jul 2001, David Newall wrote:
> > It's counterproductive because we (most of us) don't want to receive
> > HTML mail.
> 
> Okay.  This rather boils down to, "we don't want to receive HTML mail
> but can't give a cogent reason why not."  Personal taste, I suppose,
> is a reason nobody can argue with, but it hardly advances a reasoned
> debate on the pros and cons of HTML.
<snip>

The reasons we do not like HTML are actually quite sound:
1.) It increases bandwidth without commensurate increase in message
clarity. 
ie HTML is inefficient.
2.) On this and similar *nix-centred lists, many people are using
mailers which render HTML as unreadable garbage, so that the message
simply isn't read.
ie HTML is ineffective. 
3.) Though I haven't seen this myself, I am told that some HTML messages
can crash some non-HTML-compliant mailers.
ie HTML can be a considerable nuisance.

Our dislike of HTML isn't a matter of prejudice.
As I see it, this thread has been about how best to help newcomers
avoid posting in this unwanted HTML format - or any other than plain,
wrapped text, come to that.
ie we want the (for want of a better description) Unix Standard to
apply)
