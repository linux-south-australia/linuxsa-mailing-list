Re: Puzzle in C

Stephen White wrote:

> It just seems rather clumsy to do it this way, simply because var_args
> is lame enough to need an initial parameter

There's no type information stored on the stack, so varargs has no
way of determining what's stored there.  What's more, you still have
to abide by the casting rules (eg, char casts to int, so you must
retrieve an int off the stack, not char).

> and C provides no way of accessing stack frames to handle returns.

Well, it ain't really designed for that.  Would creating your
own argument stack help?
