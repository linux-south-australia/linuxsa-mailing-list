Re: Use of bc

David,

Being accustomed to dc, I will limit my answer to that.

Dc supports input and output in bases ranging from 2 to 16 (it might
even be higher on some platforms.)  The i command sets the input base
to top of stack.  The o command sets the output base to top of stack.
For purposes of input in bases larger than 10, digits after 9 are A
through F, not a through f.  The p command prints top of stack and so
converting from one base to another is as simple as setting the input
and output bases (both default to 10) and then entering and printing
each number.

If you set the input base before the output base remember that the
new output base must be entered in terms of the input base.  For example,
to convert from binary to octal could be done as "2i 100o .... p", or as
"2 8 oi ...p", or as "8o 2i ...p".

Surely there's no more to say on this.
