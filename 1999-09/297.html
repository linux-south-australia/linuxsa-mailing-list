Phil Pittard wrote:

 > IF a linux box is unattended,  even if there are no open consoles, THEN
 > anyone who knows the method I use, can get in & change ANY/ALL passwords
 > in the time it takes to reboot and then type 4 short commands!!  IMHO,
 > there is NO security on an unattended linux box which is open to "public
 > access"!  IF the linux box is mission-critical etc, then leaving it
 > "insecure/unattended" is very DUMB!

This is not unique to Linux.  Despite the opinions aired on this
mailing list over the last 24 hours, it's a basic tenet of information
security that there is no security whatsoever if an attacker has
physical access to the machine, no matter what OS it's running on.

BIOS passwords, single user passwords, root passwords, user passwords,
and everything else is totally useless in the face of an attacker who
can steal your hard disk (or remove it, alter it in another machine,
and replace it).  If you have any information worth protecting, pay a
lot of attention to network security, use crypto, and keep the machine
in a physically secure area.

 [ incidentally, has anyone thought about the fact that SCSI host adaptor
   target IDs are usually BIOS-settable, which means that someone can
   break into your SCSI-based systems by walking up to your machine with
   a laptop SCSI host adaptor set to an unused ID, plugging it into
   your computer's external SCSI connector, and mounting your disks?
   No password required, you don't need to open the case, and boot-time
   passwords are useless... ]
