Re: Linux Console Security (was prev Q:Lost Password etc)

Another thing while we are on this topic,
I have been involved with a Linux Proxy server at my school, and the way we handled
physical security was to have no keyboard, mouse or monitor (we disabled the
keyboard in the BIOS so that the server can reboot without intervention in the case
of a power outage) and to put the box under a desk in an office that has someone in
it for most of the day. The other thing is the room is locked, but is well lit and
has lots of windows, so anyone trying to tamper with the system will be clearly
visable. It is strange how often it is better to have the server out in plain view
rather than tucked away where people can stuff around with it unseen!
We then set up ssh to allow login over the network, so if I need to change
something, I don't have to use the computer. That way, there is no reason anyone
should ever have to touch the computer itself, short of hardware failure.

Can anyone provide some other examples of physical security?
