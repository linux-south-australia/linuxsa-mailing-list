Re: Linux Console Security (was prev Q:Lost Password etc)

The point that you seem to have missed is that login security is
pointless if people have completely free access to the machine. If
someone can do whatever they like (eg:
open case to wipe BIOS so as to bypass boot password so as to boot off a
floppy disk so as to mount the root partition and edit the /etc/passwd
file)
then there's nothing you can do to stop them, short of encrypting your
entire filesystem and decrypting it to RAM every time you boot up after
requiring a manually typed in 1024 byte password.
Security is made on the assumption that while the machine is maybe not
locked into a safe, it is at least vaguely watched to stop people
tampering with it physically. 

No matter how good your software security, it's still not going to stop
someone picking the machine up and walking off with it down the street!
Security through obscurity is not a valid method of securing something,
because while the users might not know about stuff, I'll bet the hackers
find out first. I'd rather know what I have to look out for, rather than
blindly living in ignorance.

Toby
