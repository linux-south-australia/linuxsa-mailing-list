Confiuring XDMCP Broadcast

Hi group:

I have installed XWin32 for Windows 95, and am trying to connect to a Redhat
6 box. I would like to be able to:

1. Have the box show up on the XDMCP Selector list
2. Run a full desktop session on XWin32 (RSH, single window)

Here are my observations so far (thanks to the archive for reference):

1. I can get applications up by manually initiating a remote login:
	- Double-Click XWin32
	- telnet to the Redhat box (in my case 1.2.3.1)
	- log in as root
	- issue "export DISPLAY=1.2.3.2:0.0" where 1.2.3.2 is the 95 box
	- issue "xcalc" (or xterm, netscape, or other x app)

2. I modified the /etc/X11/xdm/Xservers file as follows:
	changed:	:0 local /usr/X11R6/bin/X
	to:		:0 local /usr/X11R6/bin/X -broadcast
   and restarted the box hoping that would cause the broadcast.

The Redhat box is configured with defaults, and works with manual login, so
I suspect that I have to alter the server parms to allow XDMCP broadcasts.

I am having trouble finding documentation to RTFM. Perhaps if someone could
point me in the right direction. This would make an excellent HOWTO.

Any hints would be appreciated by this newbie.
