This is simply a bug in netscape. XmTextField is a Motif name for a single
line of text. From the output above netscape is screwing up the font
information it's handing to the text field. Let me guess you either get
this is messanger or on a web page with a form you can fill out.

A possible way around it is to change the fonts your using under netscape
(the fixed font). I can't guarentee it will work but it might.
