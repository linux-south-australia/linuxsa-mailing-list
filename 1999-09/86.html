Fw: Warm boot from Linux to Windows

 If I recall right, there was a BIOS set up interrupt (INT19 form memory)
 which could be called to start a warm boot.

 It worked sometimes in DOS, and was dodgy in windows (i think windows
 overwrote the int vector).

 Anyway, maybe this is a possiblity as a quick fix.
 I havnt looked into it, but im sure if the INT19 vector is preserved from
 the start of the linux boot, there shouldnt be a problem changing the
 shutdown code (or whatever code normaly reboots the machine) to simply call
 INT19 last.
