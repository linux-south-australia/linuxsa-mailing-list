Re: Printing

Alex Wilkinson wrote:

> Does anyone know of an application that can print documents that can
> only be accessed in a browser.
> I know I can print on page at a time but this will take a long time.
> 
> I would like to print /usr/doc/LDP/sag and nag
> 
> but it is all in HTML .
> 
> So I can't go lpr -d *.*

0) Netscape 

 netscape &
 netscape -remote 'openURL(file:///home/abc/example.html)' -remote 'print'

 Start Netscape, then use the remote X commands to tell it what to do.
 Of course, it won't work without X running, so this isn't a general
 solution.


1) Lynx

  lynx -dump file:///home/abc/example.html | lpr

  Rough and ready results.


2) Emacs

  Because Emacs is big enough to do anything.

  M-x w3-find-file ~/example.html
  M-x ps-print-buffer-with-faces

  A bit rough, but much more readable than Lynx.

  You can also start TeX from W3, and the results
  are very, very nice.

  You can run Emacs in a batch mode, so you don't
  need an attached terminal (ie: you could set
  it up in a printer filter).
