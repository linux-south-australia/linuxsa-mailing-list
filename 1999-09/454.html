Re: general modem problems (was pppd problems)

On Tue, 21 Sep 1999, Jake Hawkes wrote:
> A PPP connection keeps dying - not dropping out, but dying.

Oh, another easy one.  If the line stays up it's obviously isn't a
line/modem problem, so it must be a ppp problem.  BTW I believe you can
shut down a ppp daemon and restart it successfully.  I've never done that,
in fact I was horrified to hear it was possible.  If the PPP keeps shutting
down, does restarting it help?  Also, is it really pppd that shuts down, or
is it something crazy like routes disappearing? If you are running a
routing daemon (e.g. routed or gated), *DON'T*.  If you've only got one
external link your routing table only needs a single static, default route.
