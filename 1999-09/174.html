Rapidly reconfiguring (frame-relay) networks

Hi,

my favourite customer has an Australia-wide frame relay network using Telstra.

Offices in Sydney, Brisbane, Melbourne and Adelaide.

Adelaide is the center of the world :-) and contains the Internet connection.

We now use rsync to copy the application data to Sydney over the network
during the night, and we now want to look at what happens if Adelaide takes
a direct hit with a tactical nuke, or some less catastrophic event that
none-the-less takes much of the Adelaide infrastructure off the air.

If we are to bring up another state office and allow all other states to
continue to work, we need to consider a number of things:

1. The name servers must be relocatable and resolv.conf should perhaps
resolve locally first and then remotely secondly
2. DNS should run in each state office
3. The Frame-relay network should be fully mesh connected to avoid Adelaide
being the problem area.
4. It should be possible to bring up the internet gateway (squid etc) in
each state quickly.

Can anyone else suggest anything else I should be thinking of?
