Hi again... re my previous post.... part of the problem is solved...
ipfwadm -F -p deny
ipfwadm -m -F -a accept -S192.168.1.0/24 -D0.0.0.0
is now
ipchains -P forward DENY
ipchains -A forward -j MASQ -s 192.168.1.0/24 -d 0.0.0.0

It works... found it buried in the ipchains how-to  (section 8.2
[Examples of translated ipfwadm commands] to be exact)
Only reason I looked there was because of the "ipchains errors" in
kfirewall, so thought I would read up on ipchains! RTFM hmmmmm!!
