Phil and others,

There is a lot of publicly available information on how to get root access to a box already, (eg. Linux Pocketbook even has a section) all that one has to do is to look for it. The amount of times I have booted linux, from boot disks, to do something with some piece of hardware that can't be done with some other operating systems....like the slackware boot disks no password required but full access to the machine.  The point being that information for getting into and changing the root password is definately publicly available and as already metioned a few times, a box may be only as secure as its physical security.  A point that all sys admins should consider.

Good healthy discussion on an issue that we all should be a very interested in.  :^)

> Step 1:
> Reboot PC
> 
> Step 2:
> At the LILO:  prompt hit TAb to get a list of the various boot
> options.... select any linux boot (ie/ LINUX    MOUNT     BACKUP  etc)
> For the purposes of the exercise we will assume that there is a lilo
> boot  called LINUX   :)
> 
> Step3:
> type this
> 
> linux init=/bin/sh
> 
> Step4:
> Within seconds (ish) you will see the linux prompt (# or whatever). You
> can use most (if not all)  Linux commands although you may need to use
> the full path for some.
> type this
> 
> mount -n -o remount,rw /
> 
> (NB: even if you did a 'mount' command  after getting the  prompt & it
> said that / was mounted rw , still do the above command.... 99% chance
> that you are not really mounted rw!!!)
> 
> Step 5:
> type this
> passwd
> 
> now enter the new root passwd!!
> OR
> 
> passwd (username)
> 
> to change any other users password!!
> 
> Step 6:
> type this
> 
> remount -n -o remount,ro /
> 
> Step 7:
> type this:
> 
> /sbin/reboot
> OR
> /sbin/shutdown -r now
> 
> Step8:
> IF the box doesnt reboot, wait til you cant hear any hard drive activty
> & then press reset
> 
> That's it.... THIS obviously only works at the keyboard BUT it renders
> any insecure linux box in an office/school/ISP's etc extremely
> vulnerable!!
