> If all else fails here's another solution:
> 
> 1) Reinstall but DO NOT format ANYTHING except the swap space
> 2) Reenter a new password

	Good God! This is a heinous indeed course of action ! :)

	Before trying the above, you could:

	Boot using a bootdisk -- don't worry about any options at the 
	lilo prompt, just boot a root shell.
	Type the following:

		cd /tmp
		mkdir root
		mknod /tmp/hda1 b 3 1
		mount -t ext2 /tmp/hda1 /tmp/root 
		cd /tmp/root/etc/
		vi passwd

	and then remove the 'x' or '*' from the second colon-separated field
	for the root entry (ie the entry that starts with "root").
	Then just type:
		cd /
		umount /tmp/root
		reboot
	If the mount command fails, or there is no etc subdirectory, then
	you'll need to try a different disk partition. Try random
	mknod commands, substituting the minor device number (the '1') for
	various other numbers.
