On Wed, 30 Jul 1997, Gordon Rowell wrote:

> On Tue, 29 Jul 1997, Peter McCarthy wrote:
> 
> > Now how do I get named to first try the hosts file then resolve from the
> > name server ? 
> 
> {Being picky here} It's not actually named, it's the resolver library
> which determines whether to use named or the hosts file. The resolver
> library can choose which nameservice to use - hosts, named, NIS (not that
> you would want to choose that option). 

Sadly, it's actually a bit more complicated than that. Some applications
do not use the standard resolver library, but do their own name
resolution - sendmail, qmail and squid come to mind. If you really want
these to use /etc/hosts instead of DNS, you'll have to work out how to
configure them (if you can at all) on a case by case basis.
