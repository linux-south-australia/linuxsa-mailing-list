Re: Linux Partitions

Benjamin Close wrote:
> 
> Hi All,
>         Is there any possible way to move a linux partition without losing
> any data? I've recently deceided to get rid of DOS (shudder) and will use
> the space for linux. I know I could just simply create a new ext2
> parition and have it automount but I was curious to know if you can move a
> linux partion like you can a DOS partition.

There are a few ways of transferring a partition.

1) 'dd' it, this will create an EXACT copy of the partition.
	Not all that useful.
2) tar -cvpf a partition, this will create a copy of the files on the
	partition.
3) use dump (a newer option).  This works in a similar
	way to the Solaris/SunOS dump (or ufsdump).  I haven't used it
	and so I can't comment on it's use.
4) use cp.  This is a bad choice since you lose permissions, and
	will have problems with /dev.

What exactly are you trying to achieve.
