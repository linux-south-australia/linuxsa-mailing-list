Re: Help with pipes...

On Mon, 20 Oct 1997 pschulz@adelaide.on.net wrote:

> I want to do something like:   'cat ftp.in | ftp > ftp.log'
> so that I can ftp from/to different directories in one hit
> (where commands are stord in ftp.in).  Unfortunately this 
> doesn't work..  as the ftp seem to wait for something in the pipe.

The following arrangement works as per your desire.  It requires the file
~/.netrc (which in turn requires that your version of ftp uses this
automatic scheme).

My ~/.netrc looks like this:

machine ftp.netcraft.com.au
login anonymous
password tom@tyco.net.au

Refer to the man page for more info on the above.

The ftp.in file (for this example), looks like this:

open ftp.netcraft.com.au
ls
bye

You would add some file transfer commands to this list...

The following command gets it all rolling:

ftp -v <ftp.in >ftp.out 2>&1

The command assumes the bash shell.  Note that piping is not involved.  If
output is being sent to a program, then piping must be used; if it is
going to a "file", then redirection must be used.

And, to observe what is going into the file ftp.out, use the following
command:

tail -f ftp.out

Good luck!
