Re: SMB printing problem

> I have a linux box (slackware) set up, to give samba file and 
> printing access to two win95b machines.
> File acces is fine, but when i print from one of the window$ machines, 
> it prints until about 1/3 pf the way down page three or four ... 
> where the printer stops, still showing it's data light and the print 
> job disappears from the queue.
> 
> Anyone have any ideas of things i should try, or similar experiences? 

I would try printing to a file, and testing things like 'copy file
lpt1' on the Windows box, then 'lpr file', and 'cat file >
/dev/lp[012]' on the Linux box.  Testing Samba, lpr, etc. separately
should give you a better idea of what exactly is failing.

Do you have "mx:0" set in your printcap?

> I am considering installing redhat from the cd i have (powertools, 
> 4.2) but the problem is space - i only have about 300 megs total, 
> at the moment slackware is only 60 megs or so, so before i delete 
> it and try redhat, can anyone tell me if it's possible to install a 
> redhat system in around 80 megs? all i need is samba file&print, 
> wpe, gcc, telnet/ftp, and umsdos (the second and third drives are in 
> umsdos at the moment)

A minimal install of Red Hat will fit in about 50-60 MB.  I don't know
how much extra space samba, wpe, gcc, etc will take up.  Since both
Slackware and Red Hat are Linux, if you have the same programs
installed, the space taken up by either should be quite similar.

Also, before deleting Slackware, make sure you have some way to
recover it if things don't work out.
