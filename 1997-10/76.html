Re: Help with pipes...

  
> I want to do something like:   'cat ftp.in | ftp > ftp.log'
> so that I can ftp from/to different directories in one hit
> (where commands are stord in ftp.in).  Unfortunately this 
> doesn't work..  as the ftp seem to wait for something in the pipe.
> 
The command is basically fine. The only catch is that what comes 
out of the pipe appears on stdin for the following command and
the ftp password is not read from stdin! So the ftp process will sit
and wait for you to enter it from the terminal.You should also notice that
the password prompt is not written to stdout ie it wont show up on
ftp.log, it should appear on the screen if you enter the command
interactively. 

So the key is make sure that the commands you give ftp don't cause it to
prompt you for a password (use 'ftp -n' and the 'user' command).
