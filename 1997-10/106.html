Re: A couple of problems

> There are some issues about efficiency however.  This includes
> some shells will go and search the path for a program.  If you
> have a convoluted path (Note: in this case that is unlikely) you
> can get definite performance increases through a carefully 
> crafted path statement.

Efficiency shouldn't be a problem since most (all?) shells maintain a
hash table of previously-used commands so they don't need to search
the PATH more than once for any particular command.

At least the following shells use a hash table: ash, bash, csh, ksh,
tcsh, and zsh.  In fact, csh and tcsh search the PATH once at startup
(and whenever you change the PATH), so the ordering does not affect
efficiency at all.
