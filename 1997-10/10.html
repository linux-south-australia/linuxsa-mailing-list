Re: Networking problems with RH4.2

> A friend has a recent installation of RH4.2 that loads up alright,
> mounts network file systems, runs Netscape, etc, but can't ping
> network hosts, telnet, rlogin, etc.  Any pointers?

Almost certainly a problem with /etc/services.  Make sure it exists,
has reasonable data in it, and has the correct permissions (644 or
rw-r--r--).  /etc/services is contained in the setup rpm if you need
to get it back.

> Oct  1 08:50:36 trex ypbind[158]: broadcast: RPC: Timed out. 
> Oct  1 08:51:40 trex ypbind[158]: broadcast: RPC: Timed out. 
> Oct  1 08:52:44 trex ypbind[158]: broadcast: RPC: Timed out. 
> Oct  1 08:53:48 trex ypbind[158]: broadcast: RPC: Timed out. 

You can probably stop these messages by removing all the yp packages
(presuming of course you don't need it :).
