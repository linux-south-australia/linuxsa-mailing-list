> As a keen but new student of Linux, I have a couple of problems:

> 1.	I can’t execute C programs.  I write a C program, say test.c, then
> compile it with:
> 
> 	gcc test.c -o foo
> 	
> to produce the executable file ‘foo’.  When I type ‘foo’ to execute the
> program I get:
> 
> 	bash : foo : command not found
> 
> Anybody know why it might be doing this?
> 
We really should have a AusLUG FAQ, this would be top of the list :).

Answer: When you type a command the shell looks through your execution
path (type `echo $PATH` to see it) to find the executable.  If your
current directory isn't in the path the shell won't find the program and
will report "command not found".  Every directory has a sub-directory "."
which is a pointer to itself.  So to execute a program in the current
directory type `./foobar` (where foobar is the name of the program).  It
is NOT good practice to have "./" in the path there are various reasons
for this but I won't go into it here.
