Re: elm filter

Hi all,

> I have been using the elm mail filter to filter incomming messages before 
> they are put in my inbox. My filter rules look something like this:
> 
> # rule 1
> if (from contains "linuxsa") then save ~/Mail/linuxsa
> 
> this doesn't work because mail from linuxsa has a recent-from field. What 
> I would like to be able to do is:
> 
> # rule 1
> if (resent-from contains "linuxsa") then save ~/Mail/linuxsa
> 
> this also doesn't work as the elm filter doesn't support the resent-from 
> field. Any solutions or work arounds?

Unfortunatly I don't know enough about elm to be able to offer much help
with elm directly but this may still be of some help to you or others.

I have setup procmail to sort my mail, which I will say was quite easy.
If you are using RedHat and installed procmail rpm then it will be
ready to go and all you have to do is create the configuration file
~/.procmailrc

This is how I sort linuxsa mail...

# Linuxsa
:0 w
* ^X-loop: linuxsa@linuxsa.org.au
| /usr/lib/mh/rcvstore +linuxsa -create

As you can see, all it does is searches for the 'X-loop' line.
I am using exmh to read my mail so procmail just passes mail to rcvstore,
but I am sure you all get the idea.

And incase anyone wants some more examples...

# Mail for my alias LoveMachine
:0 w
* ^To:.*lovemachine
| /usr/lib/mh/rcvstore +myfollowers -create

# And finally, drop it in the inbox if nothing matched.
:0 w
| /usr/lib/mh/rcvstore +inbox -create

Of course never forget the trusty...
man procmail
man procmailrc
