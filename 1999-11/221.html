Re: Internet Routing Security

Matthew Salkeld wrote:

[Please consider using carriage returns more often.]

> One question regarding squid's ACL 'thing', if the 'policy' has to be
> written in the squid.conf, could it be changed (ie rooms given access then
> being denyed access a short time later) without having to restart squid?

You can have squid reload its configuration without restarting it.  Squid
also has an authentication scheme which uses an external process, but
I'm not sure how applicable it would be to your scenario.

> Also, would it be possible to have each room on a different 'subnet',
> while all being physically on the same network, and then just add and
> remove routes from the 'subnet' to the server?

Personally, an "Access Denied" from the squid server would be "friendlier"
(?) and easier to diagnose if you have problems than to have your route to
the server go missing, especially if you were that same server for other
services (eg, email, samba).
