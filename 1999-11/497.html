RE: Serial Port Problems

> I have a APC UPS attached to a Redhat 6.0 machine. I am using the cable
> for unix as supplied by APC and powerchute plus 4.whatever.
>
> My problem is that when I run the config script for powerchute, it tells
> me that the serial port is configured for modem control (which is
> understandable as I did have a modem on that port previously).
>
> What I need to know is what file (and where do I look) has the port
> settings which nominate whether it is under modem control or unknown
> etc.

stty -a < /dev/ttyS[x]

will tell you the current settings for /dev/ttyS[x].

Do a 'man stty' for more details.

stty will also allow you to /set/ the device options.

The config script may possibly be looking for the link /dev/modem - check
this out.  You may have a /dev/modem linked to the UPS's serial port.
