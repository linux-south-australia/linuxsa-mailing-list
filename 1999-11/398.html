Jo\\ao Campos wrote:

> I couldn't find the file (because it was probably deleted), but I will
> append some of my squid/log/cache.log that I think might be relevant:
> 
> (several like the first two messages)
> 1999/11/17 19:07:31| storeSwapOutFileOpened: Unable to open swapfile:
> /usr/local/squid/cache/00/02/00000252
>         (19) No such device

OK, that's squid's interpretation of a swap file, not Linux/Unix's.

Firstly, try:

    ls -l /usr/local/squid/cache/00/02/00000252

then if the file exists:

    cat /usr/local/squid/cache/00/02/00000252 > /dev/null

then:

    rm /usr/local/squid/cache/00/02/00000252

If the file has been "deleted" by fsck, then it most likely
is currently either residing in /lost+found (modify the path
appropriately for the filesystem it's supposed to reside on),
or it's living in Spain with breathing difficulties.  If it's
in lost+found, try the above again.

BTW, Does anyone know of a low level file system editor (apart
from beav :-)?
