It looks like apache _is_ running, as he's getting messages from the
httpd process ('Internal Server Error' & 'Unauthorised Access', etc.)

I've seen 'Internal Server Error' crop up with malformed scripts (i.e.
not giving the correct ourput)- I'm not too sure what other problems
would generate this error. If you're getting 'Unauthorised Access'
messages, check the permissions of your files/directories. Beyond that,
I haven't got much in the way of suggestions :/

One thing you could try is having personal cg-bin subdirectories off
your main cgi-bin directory - not sure if that would help or not...
