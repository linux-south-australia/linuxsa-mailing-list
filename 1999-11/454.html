RE: UPS for Linux

On Mon, 22 Nov 1999, Nick Morrison wrote:

> A handy idea - if you're in need of a working monitor for your servers
> during a power failure - invest in a monitor/keyboard/mouse switch.  That
> way you can plug it and a single monitor into the UPS for all x computers
> and have only one monitor sucking on the power.
> 
> Long live monitor/keyboard/mouse switches.

You might find the `serial console' kernel option and running
RS-232 cables to an ancient PC is cheaper.  This also allows
logging of all console messages to a file.

You don't need to run X on the PC, just use Ctrl-Alt-Fn to change
screens.  You can cut and paste using text mode mouse.  You can
also direct all the messages from all the ports to a single
screen.  This sounds bizaare, but is actually useful when you
are interested more that a new console message hsa been generated
on any of the machines, and can then Ctrl-Alt-Fn to see the detail.

Not that I want to knock keyboard/monitor switches.  They are
independent of OS and you can really stack hardware into a
24in rack.  Do any of them change to the screen that is
currently outputing messages?
