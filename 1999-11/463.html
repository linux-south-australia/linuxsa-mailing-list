Internal and External Mail - ACL?

Hello all!

I have a network here with 80-odd clients.  I have a RH6.0 Linux box being
an internet gateway.  It is running sendmail and currently allows relaying
from any PC on the local network OUT to the internet.

I'd be interested to hear people's suggestions as to how I could set up the
system to allow only certain clients access to the mail /relay/ function.

ie:-

. I need *everyone* to be able to send *internal* mail.
. I need *some people* to be able to send *external* mail.

Similar to an ACL in squid - we have a select few with access to the
internet proxy, by way of username/password authentication.

Has anyone set up anything similar in the past?  Does sendmail have an ACL
type function?  I can't find anything about it on sendmail.net or
sendmail.org...  maybe I'm blind?
