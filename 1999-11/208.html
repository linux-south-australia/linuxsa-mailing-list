[off topic] - problems migrating gif -> png in web page

Hi all,

sorry this is off topic i.e. not Linux specific, please delete now if
you don't want to read about how I am having trouble converting my html
pages from using gif picture files to png picture files.

I decided to heed advice on one of the Linux discussion forums I read
regularly and convert all gif images to png. The conversion went
smoothly on the local box (manduck linux 6.1, netscrape 4.61) using the
gif2png file converter. Viewing the html pages locally works well, all
images still viewable as expected. I used vi to edit the html, and moved
the gif images to a temp directory so as not to confuse myself.

After loading the pages to the ISP (uses NT) the pages show a 'broken
image link' where the png images are located on the page. I tried
hitting 'reload' to make sure it was not a caching problem. I saved the
html page as seen from the server using 'save as' and checked that the
page points to the right image file name. I re-loaded the png files to
the server making sure I used binary mode ftp transfer. Still no luck.

The question to the group is, has anyone come across a similar problem,
or seen any discussion that might shed light? I would be most grateful
for enlightenment.

If you want to see a live example, point at:
http://homepages.picknowl.com.au/alphart/raffle99.html
