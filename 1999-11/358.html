Setup for Client Firewall Support Using the DirectPlay Dynamic Range=20

To join a game through a firewall, the client must be using DirectX 6.0 =
or=20
later and communicating with a host using DirectX 6.0 or later. To =
support=20
applications using the DirectPlay dynamic range, a firewall =
administrator=20
needs to allow an initial outbound TCP connection on port 47624, with=20
subsequent connections of inbound TCP ports 2300-2400, outbound TCP =
ports=20
2300-2400, inbound UDP ports 2300-2400, and outbound UDP ports =
2300-2400.=20
For a chart highlighting these requirements, see the Dynamic Range Port=20
Reference Chart.=20

Example of a Microsoft Proxy Server 2.0 Configuration:=20

Internet Service Manager, Windows Sockets Proxy service: Service =
Properties=20
Add protocol "DirectPlay (client)" under Protocols tab.=20
Initial connection=20
47624 TCP Outbound=20
Subsequent connections=20
2300-2400 TCP Inbound=20
2300-2400 TCP Outbound=20
2300-2400 UDP Inbound=20
2300-2400 UDP Outbound=20
Give appropriate permissions to "DirectPlay (client)" under Permissions =
tab.=20
Add to file "mspclnt.ini", which is in the C:\Msp\Clients folder by =
default.=20
[Common Configuration]=20
RemoteBindUdpPorts=3D2300-2400=20
ServerBindTcpPorts=3D2300-2400=20
KillOldSession=3D1=20

If anyone could help me and where to start it would be great=20
