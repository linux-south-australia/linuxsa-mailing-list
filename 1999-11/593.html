Re: X Refreshing

lloy0076 wrote:

> Is there any way to force X to redraw the various windows on screen more
> often than the default?

I believe it's the application's responsibility to redraw itself, not X.
You'll find that it generally afflicts programs that aren't written in
an "event friendly" manner (eg, app reads large file from disk from start
to finish but doesn't pause to process events from windowing system).
