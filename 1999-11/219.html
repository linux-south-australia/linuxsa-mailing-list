Matthew Tippett wrote:

> I personally find that typically FTP is more reliable, (since you can drop

Plus, if you connect to one of the "better" FTP servers, you can restart
a file download (no more downloading the file again from scratch - just
download that remaining 1% :-).  See your favourite client's reget (or
equivalent) command.  Note that this does need to be supported at the
server, not just at the client end.
