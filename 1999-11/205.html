Linux Vs NT

I recently set up and was paid to set up a linux dial in server.
Originally the dial-ins were dealt with by NT RAS on an Intel PII with
128mB RAM running NT. I set up this dial in with an Intel 486SX-6mHZ
with 8mB of RAM with a stock RedHat 6.0 installation.

I can proudly report that

1) Linux goes no slower than NT
2) Linux *often* goes faster than the original server [even if it were
to go the same speed it would be quite amazing]
3) Linux has only ever not been able to deal with an incoming connection
because of THEIR fault not ours

Furthermore there have been some dramas which others have attempted to
blame on the linux system which are attributed eventually to the
permanent connection or the caller's error. I would stake my bets with
linux. It's quite astounding to se a pissy i486 with 8mB of RAM walk all
over an iPenitum II with 128mB if RAM....
