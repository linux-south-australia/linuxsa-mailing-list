If the portmapper is running, you can find out what the mapping from
RPC server to port is can be done wiht the command

	'rpcinfo -p <somehost>'

But then again, if you do know for sure what you are trying to find out
using lsof is a better option.

> ===============================================
> rat:root#> lsof -i udp:752
> lsof: WARNING: compiled for Linux release 2.0.34; this is 2.0.36.
> lsof: kernel symbol address mismatch: get_options
>       get_kernel_syms() value is 0x108ce0; /System.map value is 0x108a90.
>       There were 354 additional mismatches.
>       /System.map and the booted kernel may not be a matched set.
> rat:root#> 
> ===============================================
>
> Wierd, n'est-ce pas?

Not too wierd, the thing that lsof does is trawl all through system
references to filesystems, processes, networking and such.  So if it is
run against a kernel that it hasn't been built against, then there may be
quite a few required system calls that may not be friendly.
