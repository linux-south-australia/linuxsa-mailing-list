samba login troubles

I have sort of an odd situation with my linux box.  I am running Red Hat
6.1 on a PI 266, 96 meg ram, with Samba v2.06.  Samba is running, and I am
able to connect to the linux box from within Win98 (Second Edition).  I
have not tried any other versions of Win32, because only the lab machines
are able to access the linux box, and all the lab machines are running
Win98 SE.  Each windows machine is configured to login to an NT domain,
with an identical user/password, to provide the machines access to a
shared drive.  This presents a problem when a privledged user (one with
access to the linux box) wishes to access their home directory on the
linux box.  Since the lab is public, only lab employees have accounts on
the linux box.  I have tried using \\ServerName\homes%cgilmore to access
my home directory, but I get an Not Found error.  

My logs show "... connect to service cgilmore as user dmdluser ...", which
means that Samba isn't interpreting the sharename%username option right,
or I am presenting it wrong ;) 

Samba is currently configured to run as SHARE, and I have also tried
adding a UserList (even though it is a security risk) to run the password
through.

Any ideas?
