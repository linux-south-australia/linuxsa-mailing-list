RE: moving /usr/local & /home

At 15:17 18/11/99 +1030, Justin wrote:
>
>So if I shrink the NTFS partition with Partition Magic and then run fdisk
>from the command prompt in linux and resize it there, will everything be
>ok?

I'm not so sure that fdisk will resize anything for you (pretty sure
it won't actually, but I'm not an fdisk guru so pls correct me if wrong)

What fdisk will definately do is let you CREATE a new partition with the free
space left over from resizing the NTFS partition. Then mount that new
partition
to "/usr/local/" or wherever you need more space (after mounting it quickly to
"/mnt", copying all the contents over with "cp -a /usr/local/* /mnt/", 
umount "/mnt/" and then finally wiping the contents of your current
"/usr/local").
If you paranoid about wiping the current "/usr/local/" contents just rename
the
"local" dir (with mv) and create a fresh "local" dir for mouting the new
partition.

There was something I heard about recent versions of PartMagic supporting
"ext2"
fs resizing but I have an older version so I cannot confirm that one.

Hope this helps,
