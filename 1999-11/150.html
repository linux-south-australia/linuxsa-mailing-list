Re: Tape operation question

Yes fast forwarding and rewinding for tension (also helps align to the
zenith/azimuth of the heads) is what should be done to tapes whenever you
feel that this could be causing a problem, great for new tapes.

You would not necessarily do this in a backup. Whilst backing up you use the
non-rewinding method so that you get minimal tape travel otherwise you dump
some data to a volume, it rewinds, you fast forward, it dumps, rewinds, fsf
etc. There is a lot of work going on here for no apparent reason (or
requirement).

So to minimize wear on tape and drives, remove this activity by using the
non-rewinding device and rewind at the end of the backup.
