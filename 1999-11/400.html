Re: Apache question/problem

What you're looking for is virtual hosting - as per the Apache docs:

"The term Virtual Host refers to the practice of maintaining more than
one server on one machine, as differentiated by their apparent hostname.
For example, it is often desirable for companies sharing a web server to
have their own domains, with web servers accessible as www.company1.com
and www.company2.com, without requiring
the user to know any extra path information."

I haven't set one up (yet), but it looks pretty straightforward. The
apache documentation covers it pretty well - If you can't find it in
your local documentation, it's available on the apache web site at http://www.apache.org/docs/vhosts
