> > > When I try and connect from my other machine, I recieve the message in
> > > issue.net but then it hangs before printing the login prompt.
> > 
> > It's not likely to be a tcpd (/etc/hosts.allow) if you're receiving
> > the issue.net contents, as it wouldn't be invoking in.telnetd to
> > begin with (which, btw, is what you need to put in hosts.allow).
> > Does it come up with the login prompt eventually (ie, if you wait
> > a minute or so), or have you not waited this long?  If so, it would
> > most likely be either:
> > 
> >   a)  DNS issue (reverse name lookup).
> >   b)  Ident timeout (linux box is trying to probe Windows ident box,
> >       which it most likely doesn't have running).

You can tell if it's a dns thing, by putting the IP and name of the
machine you are connecting from in the /etc/hosts file

ie

192.168.0.3	win.domain.com.ca	win
