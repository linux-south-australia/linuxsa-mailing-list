Re: more spooky stuff on the net!

Alan Kennington wrote:

> Maybe telnet creates it's own "helper" to feed incorrect
> IP addresses to it.

No, telnet should be using gethostbyname.  Try executing this:

---<Cut here for a new monitor>---

#!/usr/bin/env python

import socket
for host in ['www.geekboys.org', 'geekboys.org']:
    print host, '=>', socket.gethostbyname(host)

---<Cut here for a new monitor>---

I get:

www.geekboys.org => 212.78.193.215
geekboys.org => 212.78.193.215

which is as it should be.  Your result is ...?

> Very sneaky access to a Unix socket!!!
> This does not show up in tcpdump -i lo.
> What does it get from there?
> =====================================================
> socket(PF_UNIX, SOCK_STREAM, 0)         = 3

It's a unix domain socket, which (kinda) lives on the file system, not an
Inet socket.  If you did an ls -l of it, it would show up as something like:

ls -l /tmp/.X11-unix/X0 
srwxrwxrwx   1 root     root            0 Nov 17 09:26 /tmp/.X11-unix/X0

However, I have no idea what /var/run/.nscd_socket is.
