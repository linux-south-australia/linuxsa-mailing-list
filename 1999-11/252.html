Re: [off topic] - problems migrating gif -> png in web page

I've noticed that web servers have a default mime type for when they
don't know what to do with a file. Apache often seems to be set to
text/plain, whereas NT/IIS seems to default to application/octet-stream.
So it's not so much that NT is configured wrong here, mroe that it just
plain doesn't know what a .png is. i think. Same solution applies
anyway.
