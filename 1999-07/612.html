Re: Creating ELF Binaries...

David Lloyd wrote:

> When I do:
> 
>   gcc lex.yy.c -lfl
> 
> It creates a file called:
> 
>   a.out
> 
> I *presume* this is an old a.out binary. 

Try typing

  file a.out

it might say something like:

  a.out: ELF 32-bit LSB executable, Intel 80386, version 1,
  dynamically linked, not stripped

By tradition, the default object file produced by cc is named
`a.out' whatever the actual object format.

To set the name of the output object file to `fred' use:

  gcc lex.yy.c -lfl -o fred
