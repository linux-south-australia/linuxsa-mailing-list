Re: Multiple Mail Servers on One machine

Actually I'm using this very method myself.  I racked my brains for a
while then went looking on the net, and Lo & Behold, I came across some
obscure documentation for Linuxconf stating that it can handle multiple
e-mail domains.
After a little careful reading I worked out that they are usign a
virtual pop3 daemon (vpop3d supplied as part of linuxconf) to answer all
incoming pop3 requests and authenticate/reply accordingly (must use
virtual.mail.domain as mail server in client).

eg:
(assume domain is the actual domain name of the mail server)
mail.domain.com.au (regular server)
mail.vdomain1.com.au (virtual server)
vpop3d uses the called server to determine which set of passwd/shadow
files to authenticate from.  From what I've seen it only works with
sendmail tho (Qmail not supported to my knowledge) and a little
tinkering.

you a hand
