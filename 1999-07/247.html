I'm interested in following this up (and I do have the ability to actually
fix it if it's broken :-).  Can you provide a bit more info about the
make/model of the card (like, is it an authentic SB16 or a clone?),
the jumpering configuration, whether it's PnP, the Linux kernel probe
messages that identify the CD, and the various options you tried when
you were trying to make it work?

If you could tell me the version of the OS you were attempting to install
that'd be helpful too.

For what it's worth, you can still install the OS:  If you mount the
FreeBSD installation CD and NFS-export it on the system that *is*
running FreeBSD, you can instruct the Pentium system to do an
installation over NFS. :-)  I mention this in case you're interested
in seeing how it performs on slightly better hardware than a '486.
