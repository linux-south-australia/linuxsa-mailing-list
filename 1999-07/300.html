> 
> Wrong Tool for the job
> 
> vnc is intended as a remote console control program, it therefore gives a
> window into your consoles main X11 display. great for what it is intended for
> but not what you want!
> 
> X11clientprogram--loopback--consoleX11server--VNCserver--eth--windowsVNCClient
> 
> What you need is X11 itself which was designed friom the outset as a multi user
> networked display system to do just what your after. It will be much faster
> than the VNC approach, possibly even faster than the console display, and one
> properly configured linux/unix box can support dozens of "X terminals", be they
> other unix workstations, special purpose X terminals, or windows X11 server
> software.
> 
> X11clientprogram--eth--windowsXserver
> 
> To run X11 programs on your linux box and display the contents on a Windows box
> you will therefore need a windows compatable X server. There are many excellent
> ones out there. Most come with tools to put icons on the windows desktop to make
> it all seamless for users.  Unfortunately it's been my experience that the
> polished ones, like "exceed" are commercial, with limited free trials and a
> purchase price of about $100 per windows seat.
> 
> A great companion tool for windows/X11 is terraterm with the secure shell
> extensions
> 
> Regards
> Adrian
> 
> ps: the X "Server" is the thing drawing on the screen (eg the windows X11
> software, or the XFree86 server controlling yor console), The X client is the

"Q. Can I connect multiple users to the same Windows server, and have
them
each see their own desktop, as with WinFrame, NTrigue, WTS, etc?"

"No. Windows NT has a reasonable concept of multi-user access, but not
where the GUI is concerned.  Basically, you can't do this without access
to the source code of Windows,...." 

<irrelevant windows stuff deleted>

"Of course, if your server is Unix-based, then you have no problem.  You
can run dozens of VNC servers on a single machine." 
 
<and from another part of the FAQ>

"Xvnc is basically XFree86 with the hardware-dependent bits taken out
and
VNC put in their place, so it doesn't have drivers for any graphics
cards. You could have a look at the Xvnc source code to see which bits
have been added to the standard XFree86, and make the equivalent changes
on your X server.

X server to run the viewer.  It's very fast when on the same machine as
the server!"
