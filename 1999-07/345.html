Re: making slackware lilo not probe for cdrom

Hi Jake,

At 15:21 13/07/99 +0930, you wrote:
>1. I cant recompile the kernel (no disk space left)
>2. its a slackware box heavily trashed by an ex-admin
>3. there is no cdrom drive
>4. it takes 15 mins for the kernel to finish probing for the said cdrom
>drive
>5. this is bad :)
>
>I need to be able to tell LILO to tell the kernel *not* to probe.
>

Is it:
1) LILO probing for the CDROM as a boot drive?
or 
2) the the kernel probing for a non-ATAPI cdrom device?
or
3) the /etc/rc.d/rc.cdrom is still active (permissioned as executable)?

If it is LILO (option 1), then remove the reference from 
/etc/lilo.conf and run 'lilo' to write a new boot record.

If it is actually a non-ATAPI (option 2), then the bad news is that 
you'll just have to find some more HDD space. Although I do wonder
how syslogd might be coping if you really are _that_ full ;-)
Can you nuke some old logs (or zip/FTP them elsewhere)?

If it is LILO (option 3), then a simple 'chmod 644 /etc/rc.d/rc.cdrom' 
(or just rm it ;-) will do the trick. 
Slackware's entire init system is kept within /etc/rc.d/.
All the misc /etc/rc.d/rc.* files (eg: cdrom, local, ibsc2) are
called from within the rc.M or rc.inet2. The rc.inet1 contains
all the ifconfig/route info and commands for network init.

Hope this helps,
-Daniel

PS: I'm discounting the possibility of some guru actually
knowing of a specific LILO boot option that will disable
non-ATAPI cdrom probing from a given kernel. However, no such
post forthcoming, I thought it was at least worth mentioning
the easier (or known) possibilities :-)
