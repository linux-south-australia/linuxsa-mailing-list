Re: dumb terminals - 286/386/other

Require a 'null' modem cable for most terminals.

This is because, terminals were originally designed to sit at the end of a
telephone line with a modem link in the middle when connecting to a
host machine.  A null modem cable just swaps appropriate signal lines.
(Small adaptors and cables are available form Dick Smith for example.)

Some modern terminals, are designed to be connected directly and just
require a straight through serial cable... and some modern
(non-PC, eg Sun's) computers
and switches, are designed with serial ports that look like 'modems'.

Pick a line speed, parity and stopbit's.. 9600 n 8 is what's usually
suggested.
Pick your serial port: /dev/ttyS0 (COM1:) or /dev/ttyS1 (COM2:)

[I think you need to run 'stty' or something similar to set the port line
speed
as above... but I haven't bothered...]

Files to look at:
  /etc/inittab - need to add a line which starts 'getty' on port..
                  There might be an example of this at the bottom.
  /etc/securetty - specifies which terminals root can log in on..
                  Add 'ttyS0' or 'ttyS1' if appropriate.
  lilo.conf - (man lilo.conf) there is an option which lets you set a
                  serial device to write and read from.... also you can set

                  the kernal console with the append=" console=1,9600n8",
                  See: /usr/src/linux/Documentation/serial-console..
                  This is useful if you want to see linux booting on the
terminal..
                   .. please let me know what options you use if you
                  get this working as I get
                  the 'LILO' and 'loading linux.....'  messages from lilo,
but
                  no messages from the kernel.
                (Remember to run '/sbin/lilo' before rebooting!)
