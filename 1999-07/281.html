RE: RedHat 5.2 and Adaptec 2940UW SCSI card

While it says there are 0 hosts, you wont get far at all - try to sort that
problem out first (hardware, cabling, jumper settings ?)

Once linux recognises >0 hosts, you should be able to 'see'
/dev/sd[a-g][0-15]    where [a-g] is the host ID between 1-7, and 0-15 is
the partition id on that disk.

eg - if you get linux to see host ID of 4 on the scsi bus, try fdisk
/dev/sdd.   

(I think this is right - ID 1 = sda, or is that ID 0 = sda ??? someone fill
me in here ?)

Once you can fdisk the new disk, create the partition table, and then run
mkfs to create file systems on there. - You can just stick with the default
ext2fs if you want to get running quickly, or you could take the opportunity
to try out one of the new-fangled file systems like CODA, or the journalling
file system that SGI have recently released for linux. ( - re-live the joys
of using OpenVMS ! )
