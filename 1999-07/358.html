RE: MIT-MAGIC-COOKIES

As you would be aware I've been having MIT-MAGIC-COOKIE authentication
problems. After someone told me that it had something to do with X
Security and not allowing certain IP addresses (or something like that)
to work with the X Server, I had a snoop through my system.

A long time ago, I couldn't work out how to get my loopback device
working. Therefore, I had added a manual ifconfig lo 127.000.00.1 to my
the file /etc/rc/init.d which of course had the effect of enabling lo.

However, I've now reworked the system so that the network daemon takes
care of setting up "lo" and it seems to have resolved the problem. I
think the moral of the story is:

"If you're running X make sure the network daemon starts at boot time".
