Phil Pittard wrote:

 > PROBLEM #1:
 > The so called "booteasy" FreeBSD Boot Manager was overwritten when I
 > installed LINUX & despite doing some serious RTFM'ing I can find no
 > reference in the damn book or on the FreeBSD site on how to reinstall
 > the Free BSD boot manager. 

'man boot0cfg' on FreeBSD 3.2 or later should help you out.

 > PROBLEM #2:
 > FreeBSD has quite good linux emulation so I want to play with it &
 > easiest way is to just  cp /linuxDir/linuxfiles /FreeBS
 > howver I cannot get FreeBSD's partition to mount properly. 

Check that the 'ufs' supported by Linux is the BSD Fast Filesystem
version of ufs, not the System-V Release 4 or SCO SysVR3 ufs.  I
suspect your ufs doesn't understand Berkeley ('man 5 filesystems'
if you have section-5 manpages installed;  I've checked a few of 
our RedHat and Slackware boxen and can't find it <grumble>).

If you can't find BSD ufs code for Linux, I'd suggest that you 
mount the Linux filesystem under BSD:  BSD supports ext2fs just
fine.  

You'll probably want to do something like this:

      mount -t ext2fs /dev/wd0s1 /mnt
                           /   \
          wd0 - disk name-/     \-s1 - Slice (partition) 1

Your IDE disks (wd0, wd1, wd2, ...) and SCSI disks (da0, da1, da2, ...)
are identified at boot-time;  use "dmesg" to verify that you have the
right one.

Check the mount_ext2fs(8) manpage for more info.
