installing JDK on remote Debian box

>>>>> "s" == stephen  <twisted@primus.com.au> writes:

  s> i have a question about installing the jdk on a remote debian
  s> (2.0.34) box.
<snip>
  s> is there a way to get the jdk running without upgrading libraries on
  s> the remote box ...

Yes, jdk Version: 1.1.7v1a-2
         Provides: jdk1.1-runtime, java-virtual-machine
         Depends: libc6, xlib6g (>= 3.3-5), xpm4g (>= 3.4j-0)

So I expect that you _would_ need the library installed to get jdk
running.

But is the problem that the Debian box is running only libc5?  In this
case it might already have an older version of libxpm on
it...  Actually there are many possibilities depending on whether it
is a hamm, slink, or potato version, whether you are compiling your
own jdk or using the package manager, etc.  

A strength of Debian/GNU Linux is the ease of remote maintenance using
dselect/dpkg.  Why not get the root passwd for an hour and use dselect
remotely to install (e.g. by FTP/HTTP, etc), and let the package
manager worry about libraries?

Or does the sysadmin want every user to download and compile their own
tarballs for every package (and library) that they need to run?
