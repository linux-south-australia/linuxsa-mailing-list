>> >> >A quick tour of the documentation, and I mean blindingly quick so
don't
>> >> >blink, follows:
>> >> >
>> >> >The standard source for information on flavours of Unix is the "man"
>> >> >command.  Type "man man" for details.  To help find the name of a
>> command
>> >> >which does something use the "-k" option of man.  For example, to
find
>> >> >commands relating to text editors try "man -k editor".  Each page in
>> >> >the manual is stored in a separate file, and sometimes you can find
what
>> >> >you are looking for most quickly by searching through all pages.
>> >> >Depending on the flavour Unix the pages are in /usr/man, or /var/man,
>> >> >or /usr/share/man.  Look around and you'll find them.
>> >> >
>> >> >Linux comes with some pretty good tutorials on different topics.
These
>> >> >are known as "HOWTOs".  There are a lot of HOWTOs, and they are kept
in
>> >> >/usr/doc/HOWTO.  They are usually compressed using gzip, so display
them
>> >> >using zcat and pipe the output into more (or less).
>> >> >
>> >> >There is often a swag of "other" good documentation which lives in
>> >> >/usr/doc.  On some Unixes this might be /var/doc.
>> >> >
>> >> >The GNU people, who pioneered the philosophy free software, don't
like
>> >> >to use man pages for a reason which I cannot fathom.  Maybe they also
>> >> >pioneer the "not invented here" syndrome, which is bad.  Anyway, they
>> >> >like "info" pages.  I urge you to ignore info pages and to never
write
>> >> >one, and to read them as a last resort if you have to.  Info pages
are
>> >> >displayed using the "info" command and they live in /usr/info.
>> >> >
>> >> >Finally--and this is not intended as criticism--but is given in the
>> >> >spirit that there are things everyone needs to know, don't ask
questions
>> >> >on the list until after you tried to find the answer for yourself.
If
>> >> >you indicate what you have tried when you ask a question you'll

>> >> >receive a positive and helpful response.  For example, ``how do I
