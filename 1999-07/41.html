Re: An entry in inetd doesnt work..

>  I've written a daemon and I want it to get invoked from
>  inetd. I added an entry in inetd.conf, updated the
>  /etc/services to have the corresponding port for the
>  service name and I "kill -1"ed the inetd. But when I 
>  "netstat -an"ed, it didnt show any LISTEN at the port
>  where I wanted my daemon to listen. 

	Make sure you included the full pathname to the server
	in inetd.conf.
	Also, if you're using a service name (eg finger, telnet etc)
	instead of a port number, make sure that the service name is
	listed in /etc/services...

	Finally, check /var/log/messages for errors after inetd is
	restarted... If there is something wrong with the inetd.conf
	entry, inetd won't bother adding it to its listen list...

	If in doubt, post the inetd entry to the list....
