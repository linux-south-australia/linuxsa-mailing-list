Re: adding another network card.

> pII 233
> 2 x intel etherpro 10/100mhz pci.
> can someone point me in the right direction.
> i am adding another network card in a linux machine. having trouble
> getting linux to see it.
> linux only sees one if both are in the machine and the machine is built
> from the ground up.
> have used linuxconf. this sets the config but linux still doesn't *use*
> the second card.
> any ideas?
> 

Ensure that you have an alias for the other network card in /etc/conf.modules

In a system that I manage there are two 3com cards. We have the following in
conf.modules
 
alias eth0 3c59x 
alias eth1 3c59x 

So when the system probes for the eth0 device viola! it finds that it should
use the the same driver for the other network card.

My experience is that PCI is fairly deterministic in how it probes the cards
and so I have always had the cards come up in the right order. YMMV, if they
don't always come up in the same order, you might need to specify some
paramaters to each line.

I am assuming that /etc/sysconfig/network-scripts/ifcfg-eth0 and ifcfg-eth1
are set up properly.
