Re: uninstall

syahrul nizam wrote:

 > hi everybody
 > I'd like to know, if there's any good way to uninstall s/ware that
 > have been installed using makefile.

Not easily.

If you still have the source and object code lying around you can
do "make -n install" to see which files it installs and where it puts
them, then manually delete them.  Some makefiles might (rarely) have
a "make uninstall" target too.

You can also simplify it by configuring your application with 
"./configure --prefix=/usr/local/applicationname" at configuration
time;  then removing it is usually a matter of doing
"rm -rf /usr/local/applicationname" (that's what I do when I install
software which doesn't fit into any sort of "package" infrastructure).
You usually end up with /usr/local/applicationname/bin, .../etc, .../lib,
and so on, which helps keep things like config files and local libraries
separate so they don't clutter the rest of your system.  You can symlink
the contents of the .../bin directory into /usr/local/bin if you want
to avoid overpopulating peoples' $PATH.

Generally speaking, the problem you're describing is the reason for
package management tools like RPM.
