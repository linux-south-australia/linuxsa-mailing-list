Re: linking the sockets library

> trying to compile a MySQL client, and i get this error..
> 
> gcc -g -c myclient.c -L/usr/lib/mysql
> gcc -g -o myclient myclient.o -L/usr/lib/mysql -lmysqlclient -lsocket
> -lnsl

Drop the nsl and socket.  They are for a Solaris system.

System V derived systems don't natively use sockets, the use TLI.  BSD
derived systems use sockets.  Both interfaces allow network communications.

Linux is very much BSDish when it comes to the use of sockets.

Since sockets seem to be more prevalent System V vendors tend to provide
compatibility drivers for their OSes.  Hence under solaris to compile a 
socket based application, you must use the '-lsocket -lnsl' libraries. 

Under most BSD based systems neither library is supported.

Hence, your command should be 

	gcc -g -o myclient myclient.o -L/usr/lib/mysql -lmysqlclient

Chances are you either haven't './configure'ed properly or you are trying
to build a third party application that was originally compiled on a SYSV 
style system.
