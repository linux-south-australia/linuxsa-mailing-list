Re: Postgres questions

libpq does in fact use sockets to talk to Postmaster and then in turn
PostgreSQL on port (standard) 5555. Have you got all the env variables set ?
PGHOST, PGPORT, etc.

I am pretty sure that psql talks directly to libpq to get to the DB.

So on a network, you will need libpq on every machine that is going to talk
to PostgreSQL.

The connect routines (all three of them) have pgport, pghost options on them
which are used like this
- passed in options
- environment variables
- built in defaults.

Hopefully this helps.

Rob.
