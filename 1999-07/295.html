Ummm, that is not how I think it works.

/dev/sda is the first SCSI device found on the first buss, scanned in the
order that the cards some up.

So, if you have a disk on SCSI ID 4 on the first controller (either bus
priority order if built into the kernel, I think, or the order you specify
them in /etc/conf.modules--also /etc/modules.conf--which is the safest way
to ensure they come up in the order you want), then, it becomes:

   /dev/sda with partitions sda1, sda2, etc.

The second device, let us say, another disk on the next controller, comes
up as /dev/sdb.

This can be a problem if you add another device to the first chain, eg with
SCSI ID 6, then it will come up as /dev/sdb, while the former /dev/sdb will
come up as /dev/sdc.  

This can be a right royal pain in the fundamental orrifice. Careful
planning is required.
