Netscape version 3 Java support is buggy.  It relies on a "feature" of
older malloc/free implementations which allowed programs to do things
that they weren't strictly meant to do (like freeing the same memory
twice).  The newer implementation of malloc/free is faster, but it is
less tolerant of buggy programs.

The workaround involves replacing either the entire library or just
malloc/free with a version using the older, more tolerant algorithm.
