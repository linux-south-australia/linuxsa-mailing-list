Unfortunately  AFAIK  MS has no TCP kernel for DOS.  There are other
DOS based TCP kernels/libraries (Sunsoft PC-NFS, WU-TCP, PCFTP, etc)
which you can use.  You might, however have a problem with finding one
that
sits happily on one disk.

Another option might be to boot machines with DOS and a TCP kernel from
the network (ie with BootPROMs).  But that is a new thread altogether.
