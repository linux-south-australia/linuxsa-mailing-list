Hi all,
I too have been wanting to recompile a meaner, leaner kernel but I have 
been putting it off coz I'm afraid I might mess it up.
I've read the RH User's Guide and have written down the steps.
If any kernel gurus out there can OK the steps (or even 
give some additions), I'd be grateful. It can even be incorporated into 
the LSADP HOWTO.

I am running RH 4.1.

1.Visit http://math-www.uni-paderborn.de/~axel/Configure.help-2.0  
  (This page provides some clues about configuring the kernel)

2.Make a fresh brew of coffee (preferably Java :-)

3.cd /usr/src/linux

4.mv .config config.orig
  (in case of botched rebuild)

5.mv /boot/vmlinuz /boot/vmlinuz.orig
  (in case of botched rebuild)

6.mv /lib/modules/2.0.27 /lib/modules/2.0.27-orig
  (in case of botched rebuild)

7. make mrproper

8. make config

9. make dep

10. make clean

11. make boot

12. make modules

13. make modules_install

14. cp /usr/src/linux/arch/i386/boot/zImage /boot/vmlinuz

15. (edit /etc/lilo.config to give possibility to boot original kernel)

16. /sbin/lilo

Any comments?
