the serial port,
and dtr toggles automatically, I guess that with minicom it does an exit
without
closing :(.

Do a search in /usr/src/linux/drivers/char/serial.c for 'void
shutdown('.
It looks like you need a ioctl to change the HUPCL (Hangup on Close
termio).
HUPCL toggles DTR.

A bit more research should yield if it is set for normal serial ports or
if you need to
do a short proggy to change the HUPCL flag.
(open(/dev/cua1);ioctl(....);close(/dev/cua1)).
