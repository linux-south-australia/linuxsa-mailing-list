The above was written when Applixware 4.2 was the current version;
4.3 is even better :).

Personally, I think Applixware is great.  I'm not a heavy user of
office-type software, but it works well for when I need it.  The
spreadsheet is pretty good too.  What else is there to say?  It works
fine, and it runs under Linux!

Just recently, I had to print out ~200 reports.  Before printing, I
needed to add an identifier to the title, based on the filename.  To
complicate things, the files used the IBM-PC graphics character set.
Perl + Applixware made it _really_ easy to get all this done (much
MUCH easier than doing it under DOS/Windows).

The Perl script went through each of the files and added the header,
as well as making sure that none of the reports were missing or
misnamed.

I then installed the "MS Linedraw" truetype font into Applixware, and
recorded a macro to put a document into the MS Linedraw font and print
it.  A short shell script to combine a bundle of reports into one text
file later, and the printer was hard at work.

My example doesn't really say much about Applixware's power, but its
support for truetype fonts meant that I could do all the work I needed
to using only Linux.

> StarOffice is available for ftp but its a big beastie.

I hear that it's slow too.  There is a StarOffice mini-HOWTO at
http://www.jnpcs.com/mkb/linux/StarOffice.txt -- the author writes
that the dynamically linked version of StarOffice takes about a minute
to load on his Pentium 100 with 48MB RAM.  My Pentium 66 with 40MB RAM
loads Applixware in <20 seconds.
