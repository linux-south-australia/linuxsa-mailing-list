Re: XFree86 question...

> Hi all.
> 
> I recently had to reinstall XFree86 (version 3.3), but when I
> rebooted, X11 didn't fire up.  /var/adm/xdm-errors told me that X11
> was complaining about the lack of /dev/tty0 (I had tty1 .. tty9, but
> no tty0).  Since I've installed an out-of-the-box system, the lack of
> this device comes as a bit of a surprise, and I was wondering if
> anyone else had come across this, and what the solution was?

I haven't had that problem, but that's probably because I'm running
Red Hat and not Debian :).

The device looks like this:
# ls -la /dev/tty0
crw-------   1 root     root       4,   0 Feb  8  1996 /dev/tty0

And to create it, you can do this:
# cd /dev
# mknod tty0 c 4 0
# chown root.root tty0
# chmod 600 tty0

FYI, /dev/tty1, tty2, tty3, etc are the virtual console devices that
you see when you use ALT-F1, ALT-F2, etc.  /dev/tty0 refers to
whatever virtual console you are currently looking at, so if for
example you are using virtual console number 3, /dev/tty0 is
equivalent to /dev/tty3.
