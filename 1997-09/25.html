NFS installation help

Hi everyone,

I am hoping somebody could guide me with this problem. I must be doing 
something wrong somewhere.

I am trying to install RH4.0 on a recently demothballed 386DX33 computer.
I had a lot of problems trying to get the installation done via local CD.
The installation program can't find the cdrom (which was easily found 
by my other 486 and Pentium Linux boxes). I suspect it has something 
to do with this computer's ide/mb/bios.

Anyway, I forced recognition with:
	hdb=cdrom  
but it still won't work; error msg. in installation log says drive not 
ready.

So I decided to do an nfs installation.

I mounted the RH cd on another Linux computer, edited /etc/exports to 
contain:

	/tmp/cdrom	'IP no. of the 386'

mounted the cdrom and then stopped and restarted /etc/rc.d/init.d/nfsfs

Then, I started the installation on the 386; all goes well until I get 
to the 'nfs setup' page. After entering the IP no. and the 
exported directory (/tmp/cdrom), I pressed 'enter' and got the 
following error msg on the screen:

	Error: mount failed
	       invalid argument

	I could not mount that directory from the server

Error log (alt-F3) says:
nfsmount returned non zero 

Error log (alt-F4) says:
<4> nfs_read_super: not a socket

And that's where I'm stuck. Any help on how to proceed would be greatly 
appreciated.

Thanks in advance.
