Re: tcsh & executables

Andrae Muys <A.Muys@mailbox.uq.edu.au> said:
>On Mon, 8 Sep 1997, Andrew Chesson wrote:
>> whenever I want to run an executable in the current directory, I have
>> to preceed the file name with ./
>
> if you were to put "./" in your $PATH you would always be able to
> execute any file in the current directory.  This however is a security
> hole.
> [ example of a trojan horse ]

I suggest that placing . at the *end* of your path is no particular
security hole.  Andrew's example only works if . is in PATH before the
directory containing ls (ie before /bin).  If . is at the end of PATH
then you are "unlikely" to execute any program in the current directory
other than those which you meant to execute.  At that point, having .
becomes a convenience, not a security hole.
