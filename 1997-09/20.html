Re: tcsh & exes

Just re the argument about paths...
One side effect of having . at the beginning of your path is, as someone has pointed out, is that someone else could put a fake ls file in there...but that is only if your directory is world-writeable, etc. And who here has users directories as chmod 777?
700 in my case, and u script to reset them to that too.. hehe
 
Also, if u have . on the END of your path, then if you make a program called, say, test, and run it, then you may actually run a file somewhere earlier in the path called test. If u are like me, and sometimes put wierd files into /usr/local/bin n stuff, this can cause users hassle when they try to run their little batch/shell scripts etc, and get unexpected results... ;)

Anyway, just my 2c worth (rounded down.)
