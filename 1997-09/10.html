Re: Funny error message during boot up...

On Tue, 2 Sep 1997, Andrew Allison wrote:

> >bevis login: /usr/sbin/sendmail: Permission denied
> 
> The annoying part is all the stuff about "/usr/sbin/sendmail: 
> Permission denied" ... I created a directory called 
> /usr/sbin/sendmail and every user on the system has complete rights 
> in that directory.... but I still get the message...
> 
> I have two questions :
> 
> 1/ What user and process generates this message?
You shouldn't need another machine called butthead, since you are doing a
fine job yourself. :-)

Anyhow, sendmail is a program and it is trying to execute it on startup. 
Since you cannot execute a directory, you get an error message. Sendmail,
like everything else that starts at system bootup, gets run from the
scripts in the /etc/rc.d directory. 

> 2/ How do I fix up my system so that this message goes away?
All you really have to do is comment out the sendmail stuff in those
scripts, or remove that directory you created and install sendmail.
