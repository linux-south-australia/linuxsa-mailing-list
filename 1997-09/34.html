Re: root mail

Alex Garner wrote:

> I have recently set up a rather groovy e-mail programme on a machine, and I
> want to be able to collect the root mail from the server, directly. (as
> this e-mail proggramme can handle multiple e-mail addresses) The problem is
> that root logins are limited to console. I was wondering (maybe using
> /etc/securetty) if I can make it so that I can login as root to coollect
> mail (SMTP/pop3) from, say, a specific IP only (my e-mail machine). I know
> I could use aliases to forward it to another account - but is it possible
> to get it directly?

Do you want to login, or do you want to connect via POP3? It isn't
clear from the above.

If you want to login as root via telnet, then delete /etc/securetty
and use /etc/usertty, e.g.

	CLASSES
	local	tty1 tty2 tty3 tty4 tty5 tty6 @localhost
	
	USERS
	root	local @123.123.123.123/255.255.255.255

If, OTOH, you use POP3, then there shouldn't be any particular
restrictions on root.
