Re: IP Address change

Well gang,
I spent some time and fixed this darn problem and uncovered a few things
whilst at it.
It appears my assumption that 'ifconfig eth0 up' rereads the ifcfg-eth0
file was wrong.  It merely restarts
the previously configured interface.  To force a reread of ifcfg-eth0 it is
required to 'cd /etc/sysconfig/network-scripts'
and then 'ifup ifcfg-eth0'.  My real problem is to do with a bug in the
/etc/rc.d/init.d/network script.
It reads ALL files with names beginning with "ifcfg-eth".  As I had used
joe to do my editing it was reading the backup file
ifcfg-eth0~ as well (after ifcfg-eth0) and leaving the system configured to
that file.
Here's hoping no-one else gets caught in future.
Don.
