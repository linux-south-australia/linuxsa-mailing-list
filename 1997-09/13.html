Re: Linux and Win95 sr2

>>>>> "MS" == Michael Schwarz <michael@merlin.net.au> writes:

  MS> Hi there, Just wondering if there is any way to read-mount the
  MS> fat32 file system (Win95-sr2) from Linux yet. 

Thanks to Geoffrey Bennett for providing me with this answer
initially:

http://www-plateau.cs.berkeley.edu/people/chaffee/fat32.html

Note that this is a kernel patch, so you have to be prepared to
recompile...

Note also that initially I selected the UNICODE stuff in this patch as
a loadable module by selecting `m' rather than `y' during the `make
xconfig' process.  The compile failed in the linking (ld) stage,
because of unresolved external symbols.  (This happened even working
on fresh source which had been just reinstalled by dpkg after I blew
the old source away (rm -r !!))  So I think there was/is a real bug
there which could trouble others, but now that I've made the FAT32
stuff a permanent part of my kernel, I have no more problems.

FWIW, I got: joliet_fat32_nls_patch-0.2.7 (610429 bytes).
