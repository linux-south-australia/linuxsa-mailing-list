  JP> Hi all, I'm new to Linux and am having a few problems
  JP> configuring properly, in particular with XWindows.  When I enter
  JP> 'startx', XWindows starts but the screen flickers, I get 'noise'
  JP> on my screen, and the mouse pointer doesn't move.  I'm running
  JP> with Red Hat 4.2 and XFree86 3.3.1 on a 686.  I've got an 4MB
  JP> ATI 3D Pro Turbo PC2TV Video Card (RAGE II Accelerator).  

Have you tried the XF86 web pages?  e.g.
http://x.physics.usyd.edu.au/3.3.1/Mach64.html
(or even http://x.physics.usyd.edu.au/3.3.1/ati.html)

(It says on the ATI page that "Users of ATI adapters based on the
Mach8, Mach32 or Mach64 (including 3D Rage and 3D Rage II)
accelerators should be using the accelerated servers (XF86_Mach8,
XF86_Mach32 or XF86_Mach64)".)

Which of the XF86 Xservers are you using? 

  JP> I've read the README files and tried playing around with the
  JP> XF86Config file but have had very little success.  I've set up
  JP> with 8bpp, 800x600 resolution and told the XF86Config file that
  JP> I've got 4096 KB of memory.  Anybody got some ideas or know
  JP> where I can go to get some assistance?

I presume you've previously set up the XF86Config file using XF86Setup
(the graphical configuration tool), and that your monitor can support
the refresh rates that your card can produce?

Anyway, see the web pages for specific help.  For example, they'll
tell you that the Mach64 X server does not support 24 bpp modes, while
accelerated doublescan modes are supported.  These resolutions are
available for a 4Mb card: 

Max Dot Clk BPP  Max Resolution
----------- ---- --------------
170MHz       8      1600x1200 
170MHz      16      1600x1200 
135MHz      32       1024x768
