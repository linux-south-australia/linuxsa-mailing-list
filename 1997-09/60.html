Re: X on linux

Andrew Chesson wrote:
> 
> I would like to run a higher colour depth when starting X with startx. I
> have the -bpp 24 in my Xservers file, and when I run xdm, it runs with a
> colour depth of 24 bits per pixel. Why doesn't this occur when running
> the startx script?
> 
There are a couple of ways of achieving 24 bpp.

1) if using xdm do it the way you have done it.
2) create a file called .xserverrc with a single line
	of 'X -bpp 24'.  The startx script will pick it up.
3) set up XFree to start up in 24 bpp by default.
	IMHO this is the best option, so you can still
	start X in 8 or 16 bpp.  
	Include a DefaultColorDepth 24 in the appropriate
	"Screens" section of your /etc/X11/XF86Config.
	If you run into problems, see XF86Config(5).

And a couple of more obtuse ways I am sure.
