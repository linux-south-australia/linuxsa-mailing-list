Re: Red Hat 5.2 Sparc install failing

Dieter Jansen wrote:

 > I've had a chance to retry the install with a couple of other FTP
 > servers and an NFS server running on the Windows 95 box with the
 > CD drive, and the cause now appears clear: all of the FTP and NFS
 > servers I have been able to test do not make use of the TRANS.TBL
 > file in determining the filenames they export.

I don't think any FTP servers use that file;  It exists to provide
a mapping from UNIX-style names to ISO-9660-style names (which look
suspiciously like VMS filenames:  8 characters + 3 character 
extension + semicolon + revision number) and vice-versa.

The presence or absence of that file in each directory on a CD
defines whether the CD has RockRidge extensions.

Does NT grok RockRidge extensions on CDs?
