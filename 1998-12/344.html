make sure you're logged in as root. also make sure you are in /usr/src/linux/
while issuing these commands. From memory the
make mrproper
comes first and then
make xconfig
i installed my rh5.2 from pocket book cd as well and already compiled kernel
twice. i had lotsa problems for the first time, however. basically kernel wouldnt
compile.
i traced it to corrupted source .c files as well as .h headers in my /usr/src/linux/
directory. i then reinstalled source files from the cd and DIDNT DO the 'make
mrproper' command.
it worked! ...yesterday i recompiled my kernel again (didnt bother to use mrproper,
went straight to 'make xconfig') and no worries.
see how you go...
