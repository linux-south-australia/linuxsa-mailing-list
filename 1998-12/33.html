LINUX for the "Third World"...

 I really don't mean to be cynical in this post.

 But I've just now found the *first* explicit
 suggestion that LINUX may do some good in the
 Developing Countries.

 It was to be found on the ELKS site (that lit-
 tle-known gang working on an Embeddable LINUX
 for 8086- and, later, 80286-based computers):

 "Q1.1. What is ELKS? 

 ELKS is the Embeddable Linux Kernel Subset, a project to build a small
kernel subset
 Linux (which will provide more or less UNIX V7 functionality within the
kernel) that
 can run on machines with limited processor and memory resources. The
initial proposed 
 targets are the Intel 8086 and eventually the 286's 16-bit protected mode.
A kernel
 that can run on this kind of hardware is useful for embedded systems
projects,

 for third world deployment where 80x86 x>0 machines are not easily
available,

 and for use on various palmtops."

 (Added white-space is mine...)

 Has anyone stumbled upon any *other* references (e.g.
 from World Vision - like organisations) anywhere to
 LINUX-as-development-tool (in this sense) recently?
