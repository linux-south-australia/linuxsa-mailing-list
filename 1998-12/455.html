Re: netcfg

> What I do first I log as a user, then "su", passwd(as root user), enter
> the "netcfg" command. When it comes to start "netcfg", insteed I get a
> number of error messages from "Tlc....". Why?

Basically you are not permitted to start X applications.  This may be
for two reasons.

1)      DISPLAY environemnt not set.

        Do an 'echo $DISPLAY' to confirm it is set to ":0.0"

2)      Authentication is not allow allowed.

        Two options.

        a) Use host based authentication and open things up doing
                'xhost +'

        b) Do user based authentication.

                Do a 'xauth list' as the user.

                Do your 'su'  (The root is not needed - it is the default).

                Do a 'xauth add' with the appropriate line from the xauth list
                        output.  OIn my machine since I am using ':0.0' I would
                        use

        		xauth add myhost/unix:0  MIT-MAGIC-COOKIE-1  5e71044942703e2f077454451273313f

                and then the user I did the xauth as wiould be able to do
                things.


> Could it be that trying to set up "kde" to connect to the Net made some
> changes or else?

Wouldn't expect so I am sure most of the KDE utils hook straight into
the pppd directly, and not go through the /etc/sysconfig scripts as does
netcfg.
