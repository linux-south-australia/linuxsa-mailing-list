Re: sendmail problem

> It refuses to accept any POP3 sessions from my Windows box that is on the
> LAN behind it, and it won't accept SMTP connections from any other servers
> on the Net.

When you say SMTP connections on the net are you saying that the system
won't allow a TCP connection to port 25?  Or are you saying that it won't
accept mail for your domain?  If you can get the connection, you should
be able to send email manually using raw SMTP.

Sendmail defaults to accepting only your domain name that it resolves at
startup (as some people who have been annoyed by the delay that a poorly
configured network setup will cause to sendmail).  Alternatively it can be
overidden through some of the sendmail.cf options.

With the POP3 issue can you connect to the port and do things manually.
I can give you the POP3 commands to run manually if you want.
