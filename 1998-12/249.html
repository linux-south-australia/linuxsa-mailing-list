Last nights meeting

Last night I took my machine to the LinuxSA meeting and some generous people
set my machine to dual boot NT 4.0 / Linux, and showed me how to change the
video display resolution.

When we set up the dual boot at the meeting we never actually allowed NT to
boot completely, we got as far as the initial welcome screen to confirm that
NT would boot, hit the CTL ALT DEL keys to reboot, and then got onto more
interesting matters with Linux.

However when I got the machine home I tried to boot NT. I got the initial
welcome screen, allowed NT to boot through to the blue screen, where I got
the dreaded "blue screen of death" with the message
inaccessible_boot_device. 

Does anybody know what I should do next? I think LILO has overwritten some
files in the MBR that NT requires, but I'm only guessing.

I'm happy to go back to booting Linux from a floppy, but I really need to
get NT back.

I've included the contents from my lilo.conf file below. NT is on the first
partition, Linux is on the second partition. When I installed Linux I
selected to install LILO at the beginning of the Linux partition and not in
the MBR. But I suspect that may have been one of the things that where
changed last night.


Regards, Steve


======= lilo.conf ========

boot=/dev/hda
map=/boot/map

install=/boot/boot.b
prompt
timeout=30
default=dos
image=/boot/vmlinuz-2.0.36-0.7
        label=linux
        root=/dev/hda2
        read-only
other=/dev/hda1
        label=dos
        table=/dev/hda
