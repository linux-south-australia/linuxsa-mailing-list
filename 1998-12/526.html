On the (faintly related) topic of standardised versus 
configuration files, especially when administering 
accounts for lots of people:  

I saw a nice way of getting people to use standard .rc
files but allowing them to configure the relevant bits.  
For example, with .cshrc:  Create a standard .cshrc and 
place it in a public (networked) place.  The standard 
script executes all the not-negotiable things, then looks 
for ~/.cshrc_own and executes that if present.  If 
no ~/*_own is found, sets up a reasonable set of 
defaults for the optional stuff.  

When an account is created, merely use a symbolic 
link in the user's home directory to reference the 
public version.  (The public version is, of couse, 
not editable by nonprivileged users.)  This allows 
you to make changes with confidence that all 
(reasonable) users will receive the change upon 
their next login.
