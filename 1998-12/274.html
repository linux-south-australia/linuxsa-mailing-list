RE: Last nights meeting

With a bit of digging, and some help from Matthew Tippett it seems there is
a bug in LILO 20 that was fixed last week with the release of LILO 21. The
release is source code only, and as I don't know how to compile the source
code I'll wait until I can get LILO 21 as a rpm file.

My problem was, I needed to remove LILO from the MBR to get NT working
again. Looking through the Man pages I discovered that you can remove LILO
with the command lilo -U which removes LILO from the MBR and then restores
the files that LILO replaced.

Magic - NT boots again, and I can boot linux using a bootdisk. Whoever wrote
LILO should get a medal for thinking of backing up the files that LILO
replaced and then writing them back to the MBR when you remove LILO.

One day I'll work out how to compile source code and then install the
binaries, but until then I'll manage with work arounds and rpm files.
