Re: RFC: POP accounts for my students.

POP accounts..

popd daemon only needs an entry in the /etc/passwd file on order for
people to be able to have access to their email.

sendmail receives mail and stores it (usually) in
/var/spool/mail in a file with their username (after looking
to see if the user exists in the /etc/passwd file).

As an asside, if you wish to change the home directory
so that its not in the standard /home, just change it in the
'/etc/passwd' file.

[When adding you students, ypu might like to hack the 'adduser'
script so that it adds /home/students/$user instead of just
/home/$user.. (call the script addstudent :-) 
.. something to look at anyway.]

All in all, the /etc/passwd file is used quite a bit by assorted 
programs, that's why is need to be world readable.. also a good reason
to use a shadow passwords file.
