Re: Header in Mailing List

On the one hand, it's obvious that there are heaps of distinctive
lines in linuxsa mail which can be used in tools such as procmail
to automatically put mail into folders. I do this myself with
the first line of each e-mail.

On the other hand, one must ask why something that is obvious is
not obvious to someone else. In this case, I remember that
MS e-mail viewing software (like outlook) does not show all of
the lines of the header, unless you click the "Options" tab or
something.

Is it possible that  bigfellow@cyberdude  is using some sort
of MS software to read mail, and wanting to use the built-in
archiving facility of their e-mail readers?

I'm sure that the fact that some mailers don't display the full header
wil not mean that in-built folder archiving will not be able to
be based on the hidden header fields.
