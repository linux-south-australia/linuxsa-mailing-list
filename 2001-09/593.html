My thoughts on licencing:

1. If you want to the source to stay free, and not allow someone to either 
release your software as proprietry, or canabalise bits of it for incorporation 
into a proprietry program, then release it as GPL.  (Linux, GNOME, any redhat 
additions)

2. If you want your code to remain free, but you are quite happy for someone to 
write a proprietry piece of software that links in your code (shared library or 
similar) use the LGPL.  (ie glibc)

3. If you want the source out there, and you don't care what people do to it, 
use the BSD or X licence. (ie X windows becoming the Broken^H^H^H^H^H^HOpen 
Windows dead-end)

4. If you can, avoid the Artistic licence as it has more loopholes than 
Australian tax law.  (ie perl - dual licenced GPL/Artistic)

5. Don't write your own licence, or add clauses to other licences.  This makes 
it very hard for distributors.  (ie sharefonts :(

Some very good discussions on software licencing are listed in the book, "Open 
Sources: Voice from the Revolution" by DiBona et al.  published by O'Reilly.

A good model is that followed by Sendmail etc where they have two versions of 
their main product, a proprietry one which costs $$$, and a GPL one.  The 
changes in the proprietry one are incorporated into the GPL one 6 months later, 
providing a mechanism to make money, but still being generous to the community.
