Re: disapearing disk space problem

Hi,

With the suggestion of running out of inodes as a possible problem, it
might be worth just checking this. Try: df -i
It will list the free inodes.

You mentioned you went to single user mode. This should kill processes
which would leave open files. 

Have you tried checking the filesystem for errors with: fsck -f
It's possible that you have unfreed inodes from a past crash.

You may wish to check that you have the up to date version of the
utilities (df, du, etc). It's possible that these may just have a bug in
them. This is especially true if you have upgraded your kernel. Check
/usr/src/linux/Documentation/Changes if you have.

It's also possible that the drive has bad sectors on your hard disk.
Although very unlikely. If you do, bad blocks show up as used space. The
filesystem will attempt to automatically avoid/mark bad blocks. You can
force bad block detection with the: badblock command.

If none of this works, and you find the solution, I'd be most interested
to know what the problem was.
