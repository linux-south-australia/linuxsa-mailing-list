Richard Russell writes:
> > Sure, but I don't understand why this is an improvement over how
> > apt-get works since the Packages file also contains md5sums for the
> > downloaded packages. Isn't this the same (except you don't need to
> > worry about having build dependencies installed)?
> 
> I suppose the packages file is re-downloaded every time... so there's only
> one place you need to crack... (correct?)

Yes - but if patches for packages are updated then the md5sums also
need to be updated/generated and so they are also vulnerable from a
single place.

> Also, are you saying that ports doesn't do dependencies?

No. Just if I'm going to trust a repository with keeping non-trojaned
source packages I might as well trust them to keep non-trojaned binary
packages too and so not have to have as many development libraries
installed/updated as well as save myself the trouble of compiling
them.
