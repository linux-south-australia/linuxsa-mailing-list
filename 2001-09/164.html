ilox wrote:

> If we can be hit by these mongrels during an install of MS Software
> what hope do we have that we will be safe from any kind of viral attack.

Upgrading Linux whilst connected to the Internet can have similar
problems.  The best solution is to install a pre-patched operating
system.  That's not possible for operating systems that don't
have an open source installer, including Windows and some
Linux distributions.

Generally operating system distributors don't want to release
pre-patched operating systems.  You can understand why from
a configuration control and stock levels point of view.
Red Hat used to ship CDs with the latest patches, but they
took a lot of heat over two CDs marked "RHL x.y" being
different.

Companies like NetCraft here in SA will sell RHL+patches.
Alternatively you can follow the instructions at
  http://cambuca.ldhs.cetuc.puc-rio.br/RedHat7-CDs-HowTo.html
to build your own.

I'm sure you can do the same with Debian and some other
distros, and I'd appreciate being sent a procedure that
I can incorporate into a forthcoming version of the
Remote-Serial-Console HOWTO.
