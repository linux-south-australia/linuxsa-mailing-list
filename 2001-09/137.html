Re: Problems with Netgear PS105 Printserverbox and HP 4000T

Try Ctrl -> Alt ->F2
Login
su to root
type yast
Go to ->System Admin->network admin-> configure remote printers
If you put the name of the remote box in "server name" you will need a line 
in /etc/hosts for it to find its address.
eg  192.168.50.7	Poncho.localhost

lpc can be good for monitoring printers
try lpc -help for a list of options

Remember to restart the lpd deamon when you make changes to the printer setup
/etc/rc.d/init.d/lpd restart 

in the below example I would use

lpr -Pbjc filename


/etc/printcap

bjc|remote printer on Manduck:\
        :sd=/var/spool/lpd/Manduck-canon-bjc:\
        :rm=Manduck:\
        :rp=canon:\
        :bk:sh:mx#0:
#
localfax|points to fax on Manduck:\
	:sd=/var/spool/lpd/faxspool:\
	:rm=Manduck:\
	:rp=faxlp:\
	:bk:sh:mx#0
#
plot|remote printer on Manduck:\
        :sd=/var/spool/lpd/plot:\
        :rm=Manduck:\
        :rp=plotter:\
        :bk:sh:mx#0:

> This is my setup on a RH7.1 for a Netgear 110 to print to
> either a HP2200D (lp) or a BJC7000 (lp0). Basically it is set up
> as if you are printing to lpd on a remote computer.
>
> To print I use
>
> lpr -P lp  file
> lpr -P lp0 file
>
> Note: printserver = 192.168.1.250
>
> # /etc/printcap
> #
> # DO NOT EDIT! MANUAL CHANGES WILL BE LOST!
> # This file is autogenerated by printconf-backend during lpd init.
> #
> # Hand edited changes can be put in /etc/printcap.local, and will be
> included.
>
> lp:\
>
>         :sh:\
>         :ml=0:\
>         :mx=0:\
>         :sd=/var/spool/lpd/lp:\
>         :rm=printserver:\
>         :rp=P1:\
>         :lpd_bounce=true:\
>         :if=/usr/share/printconf/mf_wrapper:
>
>  lp0:\
>
>         :sh:\
>         :ml=0:\
>         :mx=0:\
>         :sd=/var/spool/lpd/lp0:\
>         :rm=printserver:\
>         :rp=P2:\
>         :lpd_bounce=true:\
>         :if=/usr/share/printconf/mf_wrapper:
>
> ##########################################################################
> #####
> ## Everything below here is included verbatim from /etc/printcap.local
> ##
> ##########################################################################
> #####
> # printcap.local
> #
> # This file is included by printconf's generated printcap,
> # and can be used to specify custom hand edited printers.
>
>
