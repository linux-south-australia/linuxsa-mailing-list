RAM strangeness on Athlon box

Hi all,

I have RH6.2 (with 2.2.17 kernel) running on an Athlon box (1 x 650MHz).

[ Oh, and for people who were trying to get video cards working with Athlon
  systems, I finally got an nVidia Riva NTN2 Model 64 which works reasonably
  well.  Nothing else I tried worked properly.]

Anyway, I have recently upgraded it from 256M RAM (2 x 128M), to 512, by
putting a 256M dimm into the last memory slot.  The BIOS recognises that it
has 512M, but linux kernel panics intermittently if I run with "mem=512M" as
a boot parameter.  Interestingly, I can run with "mem=384M" without any
trouble (but without changing the hardware setup), so it looks as though it's
not handling the fact that there are different size dimms in there.  Has
anyone else had any similar problems?

I couldn't find any mention of this on the Web, and the motherboard manual
doesn't mention having to put the memory in a particular order.  I have
tried putting the 256M dimm in the first slot, but whilst it does seem
to help a bit, the kernel still panics, freezes or behaves stragely.
Incidentally, I tried it with a 2.4 kernel and it did much the same
thing.

Any help/ideas appreciated.
