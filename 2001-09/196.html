tcsh builtin exit status

Hi.


I was intrigued by the behaviour of some tcsh builtins:

-----
ls -F foo && echo true
Cannot access foo: No such file or directory
-----

-----
ls-F foo && echo true
foo: No such file or directory.
true
-----

"which ls" => "/usr/bin/ls"
"which ls-F" => "ls-F: shell built-in command."
(note that the second is a builtin)

I would expect that both would not execute "echo true", since the exit 
status of the first command should be zero, so I'm guessing that  
builtins always return zero exit status.

I am running tcsh 6.10, but have tried this on another (unknown) 
version which gives the same result.  Other builtins (such as which) 
also appear to do the same thing.

Perhaps this is intended behaviour.  Perhaps there is no facility for 
passing exit statuses from a builtin back to the (interactive) shell.  
Perhaps the builtins simply don't return an exit status.

If anyone knows about this, or can point out what I'm doing wrong, 
please let me know!
