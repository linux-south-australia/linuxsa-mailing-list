Re: FreeBSD

On Monday, 17 September 2001 at 14:18:03 +0930, Richard Russell wrote:
>> Sure, but I don't understand why this is an improvement over how
>> apt-get works since the Packages file also contains md5sums for the
>> downloaded packages. Isn't this the same (except you don't need to
>> worry about having build dependencies installed)?
>
> I suppose the packages file is re-downloaded every time... so
> there's only one place you need to crack... (correct?)

No.  You only download once.

There are a number of issues here, but basically the tarballs get
downloaded from the main distribution site, and the MD5 checksums get
downloaded from the FreeBSD repository.  To break something, you have
to access both places.

> Also, are you saying that ports doesn't do dependencies?

I don't think he is.  If he is, he's wrong.

As Chris knows, I find apt-get unnecessarily fiddly.  I'm sure that it
does a good job, but it's too fine-grained for my liking.  It also
shows you all the dependencies and potential conflicts, and expects
you to do something about it.  The Ports Collection (called Packages
Collection in NetBSD) does it all for you.  There also seem to be far
fewer conflicts in FreeBSD, which makes the job easier.

time.  Due to my schedule, it may not be on the normal meeting date,
but I'll try to do it soon.
