> I can't seem to mount a floppy on the FreeBSD computer so I am unable
> to get 
> a full copy of the dmesg.boot to post.
> 
> The full line of when the card is probed during the search for PNP devices is:
> 
> CSN 1 Vendor ID: EDI0119 [0x19018914] Serial 0x9d7a85b4 Comp ID:@@@0000 
> [0x00000]
> 
> And the output during the ISA probe is:
> 
> ie0: unknown board _id : f000
> ie0 not found at 0x300

Hmm... Don't know about that one. The ie driver (according to the man
page) is the "ethernet device driver". I'm not too sure about it.

Got another network card? :-)

> As for the floppy issue...
> 
> After using the command 			fdformat /dev/fd0
> and answering y when asked if I want to format it as a 1440K floppy
> I see a whole lot of format/verify steps and then it completes.
> 
> I then immediately try to mount it using	mount /dev/fd0 /mnt/floppy
> and I get an error message saying it has an incorrect superblock structure.

Firstly, I daresay that /mnt/floppy doesn't exist. In FreeBSD, the
mountpoint /mnt is the only one that comes by default (and /cdrom).

Secondly, you've not putted a filesystem on the floppy. From the
fdformat (1) man page:

    "Note that fdformat does only perform low-level formatting.  In case
    you wish to create a file system on the medium, see the
    commands newfs(8) for an ufs file system, or newfs_msdos(8) for an
    MS-DOS (FAT) file system."
    
Have a read of the newfs man page -- that'll show you how to put a UFS
filesystem onto the disk. Then, you should be able to mount the floppy
to /mnt.
