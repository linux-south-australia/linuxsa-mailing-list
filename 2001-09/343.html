cron.hourly message

In the last couple of days the cron daemon on one of my boxes has been
issuing the following messages. It's a RedHat 6.1 system that has been
running almost continuously for well over a year. The only changes
from the original install have been security-related updates:


> Warning: /boot/System.map-2.2.12-20 not parseable as a System.map
> Warning: /boot/System.map not parseable as a System.map


The message is generated through cron.hourly, but does not appear
every single hour. cron.hourly merely contains a couple of
checkprocesses scripts to keep standard daemons running.

Google references to this problem say it -can- happen if a new
kernel is installed and you forget to run lilo afterwards. I was
compelled to reinstall this kernel a couple of weeks ago; after a
bad disk sector hatched in its guts. But lilo (and fsck) was certainly
run at that time.

The HDD in question is about 7 years old and about 85 percent full.
I'm wondering if it's now developed another bad sector, this time
under /boot/System.map ?

Or have I overlooked something blindingly obvious?
