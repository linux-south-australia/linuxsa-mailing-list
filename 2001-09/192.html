Michael Wardle wrote:

>Hi Richard.
>
>>Under RH7.1, when you do an emergency boot, /proc/partitions does
>>not exist, so it is difficult to use fsck.
>>
>>I am trying to fid a switch to fsck or where/how /proc/partitions is 
>>created so I can resolve this.
>>
>
>I just picked up a Red Hat package, and this is mentioned in the 
>release notes (which are availble online at 
><URL:http://www.redhat.com/support/manuals/RHL-7.1-Manual/release-notes/s1-install-changes.html>;):
>
>-----
>Improved Rescue Mode - Rescue mode now attempts to mount the 
>filesystems listed in /etc/fstab (assuming the root filesystem can be 
>found). The filesystems are mounted under /mnt/sysimage. This 
>eliminates a very confusing step for users using rescue-mode for the 
>first time.
>-----
>
>And on most systems, proc should be listed in /etc/fstab something like:
>-----
>none                    /proc                   proc    defaults        
>0 0 
>-----
>
>
>With any luck, this would suggest that proc should be available under 
>/mnt/sysimage/proc or similar.
>
>
>Hope this helps.
>
