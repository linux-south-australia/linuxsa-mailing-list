> loads up a pretty graphical version of something apparently

> windows and memtest, and which is apparently a GUI front end

> appears to do this, but when I actually boot, it remains as
> it was before.  

Below is my lilo.conf (in /etc) on my SuSE 7.2 machine.
All I do to reorder the boot sequence is change the order of the image
blocks below and re-run lilo.

I believe, but haven't tried, that there is also a "default=imagename"
option you can place in the beginning section of the file.

# LILO configuration file
# Start LILO global Section
# If you want to prevent console users to boot with init=/bin/bash,
#  restrict usage of boot params by setting a passwd and using the
option
#  restricted.
#password=bootpwd
#restricted
append="vga=0x0303"
boot=/dev/hda
#compact       # faster, but won't work on all systems.
vga = normal    # force sane state
message=/boot/message
menu-scheme=Wg:kw:Wg:Wg
read-only
prompt
timeout=300
# End LILO global Section
#
image = /boot/vmlinuz.20010823
  root = /dev/hdd2
  label = linux
  initrd = /boot/initrd

image = /boot/vmlinuz.20010904
  root = /dev/hdd2
  label = linux-exp
  initrd = /boot/initrd

other = /dev/hda1
  label = Win2k
  table = /dev/hda

image = /boot/vmlinuz.jul13
  root = /dev/hdd2
  label = linux-jul
  initrd = /boot/initrd

image = /boot/vmlinuz.suse
  root = /dev/hdd2
  label = linux-orig
  initrd = /boot/initrd

#
image = /boot/memtest.bin
  label = memtest86
  optional
