I don't have a copy of the samba source at the moment, so I can't tell for
sure (I have all this stuff in a book that I forgot to bring to work this
morning as well :-( ), but here's a few places to start looking:

* Check to make sure both your windows machine and your unix machine are
using password encryption (either that or they both aren't).
* Try deleting the account from the PDC and recreating it.

If I recall correctly, you were trying to join a Linux box to a Win2K
domain? I think from memory, when creating a machine account on an NT PDC,
you have the option of creating a backwards compatible account (you'd use
this for a Windows NT machine as well). Try enabling that (obviously,
Samba doesn't support the Active Directory stuff yet).

Do your samba logs have any further info?
