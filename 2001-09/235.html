Re: ipchains redirect to another host?

> I've got a computer with a real static IP address, which hosts a few
> services, including a HTTP server. I have another computer in a
> private network connected to the computer with the real IP address
> and I'd like it to be a second HTTP server.
>
> I've been searching google for things like "ipchains port forward"
> and "ipchains redirect" (without the quotes), but all I've found is
> how to redirect to the same computer but a different port. Is there
> a way to redirect to a different port on a different computer or at
> least just a different computer?

You can do this in userspace with a port forwarder.  No kernel support
required.  SSH will do it for you but this is probably over kill.  You
can look at something like http://www.worldvisions.ca/fastfwd/
