Re: problem after powerfailure, ipop3d,ssh timing out

Michael Kratz wrote:
> Does anyone have any idea what this is or how
> I can fix it?? everything else on this box seems to be working fine.

The aim of e2fsck is to get a consistent filesystem.  That
unfortunately doesn't mean that you get all your files back,
merely that you can access the bits of the files that are
remaining.

It looks like you've lost a few blocks out of some essential
files.  If you run tripwire you are laughing.

If not then there are RPM options to check the database against
the files on disk.  You'll need to check the /usr/local files
by hand -- it's probably quicker to rebuild these.

You might consider a UPS.  These have a link to a serial
port on your computer and thus you can close the machines
down gracefully upon a power failure.
