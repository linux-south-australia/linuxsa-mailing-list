I'm hoping someone can save what's left of my hair.
 
I'm using a pr200 w 32 meg ram 17gig Hdd as a internet/mail/file server
for a small network. Using SuSE 7.0
I have set up samba and Ip masq so all computers on the network can
access the net on demand.  Each of the computers have some file space of
the servers Hdd using samba.  No problems.
 
Mail is a pain in the %$%$#
 
I can send and receive mail for root on the server.  I can receive mail
from the other the boxes only if the IP is used instead of the FQ domain
name eg root@server.acc.box will not work, but root@192.168.0.1 will.
 
Using Outlook on the other boxes.  The pop3 times out or comes up with a
cryptic error number about 10 numbers long. So cannot receive mail from
the server.
 
DNS seems to be part of the problem.  Ip numbers resolve but FQDN do
not.
 
Each of the other boxes are listed in /etc/hosts with FQDN and IP and
short name
 
Can anyone point me to a walk through, step by step configuration for
sendmail.
 
I've searched the web, read heaps of stuff, SuSE Database doesn't help
much
