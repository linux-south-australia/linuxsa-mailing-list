Re: OT: Best licence for a new Open Source package ...

To add to michael's response, the BSD license appears to be
best if you are typing to establish an API or file format
or protocol as a standard (eg: OggVorbis, BSD sockets).

So, for example, if you were writing a network file server
with a new protocol then using the BSD license would be a
good idea.

You might want to have a look at the MPL used by Mozilla
as a license that falls between the GPL and the BSD.
