How much more specific than "NT logon domain thingie" do you need? :-)

In fact you can make a Linux (or any unix) box act as a Primary Domain
Controller. It's done using the software, Samba. If you download the
latest version, it comes with a soft copy of the book, Using Samba.

> > Can this be minipulated in linux with a LOGON SCRIPT?

I'm not sure what you mean by this.

You can implement login scripts on your Linux PDC that are basically just
DOS batch files. These are interpreted/executed by the Windows client when
a user logs into the machine.

Using the experimental 'winbindd' software (part of Samba), it is possible
(using PAM) to have the authentication on your Linux box done by a remote
PDC.

I realise that there are a large number of you that don't care, but do
those people that do care actually understand how the whole NT domain
thing works (from an sys admin point of view, anyway)?
