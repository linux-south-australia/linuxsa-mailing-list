Richard Russell wrote:

> On Tue, Sep 04, 2001 at 06:28:42PM +1100, Moring, Peter wrote:
> 
> 
> I've actually been stuck trying to configure this same laptop for a
> client... however, I am using Solaris x86 with XFree86 4.1.0... I can
> get it to kind of work at 640x480, or at 800x600, but at 1024x768,
> the image is partly off-screen, and seems somewhat garbled... I'm
> using 24bpp (though have tried 8 and 16), and have tried nothing for
> the scan rates, as well as 60, 75 and 85, and various combinations...
> 
> What did you end up using (particularly for ModeLine...), and does
> anyone else have any other suggestions? 
> 

I'm using horizontal sync of 57 Khz and vertical refresh of 50 - 90 Hz
to get 1024 * 768.  In this instance the horizontal sync had to be 
specific number instead of a range.
A file to check is /var/log/XFree86.0.log to see if it gives any useful 
infomation.  EDID can be a problem if it receives incorrect info from 
the monitor.  You can disable this by putting:-

Option  "IgnoreEDID" "on"

in your XF86Config file.
