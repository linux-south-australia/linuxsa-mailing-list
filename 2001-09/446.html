Re: Netlogon Via Samba

On Mon, 17 Sep 2001 22:18, Mick Collins wrote:
> Hello Group. I Have 2 questions for you's

You's isn't a word.  It also sounds exactly like "use".  >:-(

> In Redhat 6.0, is there any way I can validate people username's and
> passwords? Like a NT Domain Logon? My Client Machines are all Win 98
> SE

Certainly can.  This can be accomplished using samba.  The relevant 
options in smb.conf are "domain logons" and "security".  Also note that 
you may need to enable encrypted passwords to talk to Windows 98 
clients.  I think this is done using the "encrypt passwords" option and 
running a script to encrypt your password.

(I use exactly these features on my home network.)

> 2) When someone logs onto my server using their username and
> password, can the server map certain network drives for specific
> users?
> As in
>
> Mick Logs On , I get All Privilages
> Sid Logs On, Has Home Dir Privilages
> and so on
> ???????????

You can have a separate logon script for each user using something like 
"logon script = %U.bat".

Please read more about this on <URL:http://www.samba.org/>;, or on the 
samba(7) and smb.conf(5) manual pages on your system.  You may need to 
install packages called samba or samba-server.

Also, once the relevant files are installed, the smb.conf that ships 
with Red Hat usually has lots of useful comments that can help.


Hope this helps.
