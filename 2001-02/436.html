RE: Router connections (which?)

Tuesday, February 13, 2001, 8:10:58 PM, Michael wrote:

> Hi all,

> just wondering which of the following is a better way to connect machines to 
> the net:

> (a)
>
> LAN ---- http/mail/ns machine ----- gateway/firewall machine ----- NET

> (b)
>           http/mail/ns machine
>                 |
>                 |
> LAN ---- gateway/firewall machine ----- NET


> this would be utilising 2 machines. I am setting up a network for my cousin 
> and am thinking about doing it like (a) as the Gateway machine is an older 
> machine so I don't want to put too much load on it. Note: all machines 
> including those on the LAN are using internet routeable addresses. All 
> opinions are welcome,

Well,  because  I'm lazy[1], mine is configured as a) above. But b) is
the  classic  LAN/DMZ/NET setup, and I'd go with that. It is generally
considered more robust (security wise), than a).

David
[1]  I recently put in a separate Firewall machine at home, and rather
than  screw  around  with changing all sorts of things on the existing
server  &  LAN,  I  configured my home net somewhat like a)  (bit more
complicated than that, but similar principle).
