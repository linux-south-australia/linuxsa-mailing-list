Re: Deny List project

I have an even better idea. And it's based on one already implemented.
Hear me out!

1) Exactly Like DNS
 - same protocol
 - different servers
 - different root servers

2) Root Servers
 - root domains:
  par (for the totally spam paranoid)
  hig (for those really concerned)
  med (for the average user)
  low (for people who want protec. against serious stuff)
  non (no protection)

3) Sub Domains
 - for each IP

Now, each IP gets a reference from its FQDN. So:

www.rebel.net.au might resolve in:

  www.rebel.net.au.med

As OK to Receive

BUT

	www.rebel.net.au.pa

As NOT OK to Receive

Now, name servers already exist. This would require the following
things:

* at least 5 worldwide BIP-SERVERS (Bad IP SERVERS)
* a way to resolve the BIP name to the DNS name

So, what would happen is, that a BIP aware MOZILLA would do:

int result=get_denylist_byip(203.20.69.66);

if (result==E_OK) {
	get_whatever_we_were_getting(blah bla);
}

I think relying on single sources such as blackhole.org or ACNS is apt
to overload. ACNS have said that blackhole et al are slow and so forth.

Why reinvent the wheel?
