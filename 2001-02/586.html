Re: Opinions sought on requiring nmbd to be running toovercome a Windows NT bug :-)

Richard Sharpe writes:
> 
>   Samba-Common
>   Samba-Client
>   Samba 
> 
> I am not sure what others are doing WRT this, but it worries me that
> introducing an nmbd only subset may increase the complexity for
> installers and for people moving from a libsmbclient-using
> workstation to a Samba server.

Debian have done a similar thing with:

samba-common
samba
smbclient

which can be useful as sometimes all you want is smbclient.

As you say adding an nmbd only subset will make it more complex, but
with more sophisticated package management systems (such as apt or
apt-rpm) it shouldn't cause problems for the end user as they don't
need to worry about these intermediary dependencies - install/upgrade
scripts will need to get smarter though :-)
