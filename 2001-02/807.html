Java servlets

Hi,

I hear all this talk from Oracle, IBM, et al. about Java-enabled 
databases, and about using Java servlets to 'web-enable' your
databases.

So has anyone done the full Java-based webserver thing yet? I
remember hearing that Linux was *the platform for enterprise
Java deployment (sic), but is it actually being used as such?

I would have thought that Java is the last thing you want 
listening for web requests, purely because it is slow. Is this
a valid assumption, or are they loaded into memory, where they
run just as fast as any other program?

And wouldn't the development time be a bit high? Again, I would
have thought that just writing PHP pages that listen for requests
(like XML-RPC does) would be much quicker, both in dev time and 
run-time speed, than running a Java servlet.

Comments appreciated...I'm wondering whether I have missed a boat, 
and whether everyone else has missed it too.
