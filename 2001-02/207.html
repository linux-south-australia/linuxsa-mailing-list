Re: Sendmail 8.9.3 -> 8.11.2

David Lloyd wrote:
> 
> If it's RedHat, you should be able to get away with something like:
> 
> * rpm -Uvh (the package name)

The only problem with this is that RedHat's RPMs are missing a few
compile-time features (like LDAP routing). I would personally back up
any files you want to keep across the two versions and uninstall old
sendmail before compiling and installing new sendmail.

> And rebuilding the sendmail.cf file in the correct place. I think that
> is /etc/mail/sendmail.cf whilst older version use /etc/sendmail.cf. I
> always forget which version is precompiled to look where so you may take
> an extra 2 or 3 minutes figuring it out. Naturally, the documentation
> should tell you where to put it as well.
> 
> Given that you say the computer is not connected to the Internet or
> acting as a mail server at the moment I've guessed that you are going to
> have to rebuild the sendmail.cf anyway, hence I would:
> 
> * upgrade SendMail before worrying about how to configure it too much
> * once upgraded, make sure you have access to the Internet and the
> /usr/doc/HOWTO or http://www.linuxdoc.org/

There is also some fantastic documentation available with sendmail.
There's the intro and the Operations guide as well as the README files
spread throughout the sendmail source.

Hope it helps,
