Most Unix filesystems are designed to minimise fragmentation without the
need for user intervention (user friendly). It it still possible to have
a file system with a serious amount of fragmentation, though. One of my
mail servers has a partition with 700 odd mailboxes. With individual
messages being deleted and added constantly, fragmentation is quite
high. When it gets to a point where performance is fading, we just move
everything off that partition, format it and move everything back. Note,
though that this is not necessary for most machines.
