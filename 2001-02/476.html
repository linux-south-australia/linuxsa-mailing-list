NT Oddity

Hi!

I did a port scan, using nmap, of an NT server on my network.  It had four
ports open - 

  135/tcp    open        loc-srv                 
  139/tcp    open        netbios-ssn             
  6667/tcp   open        ?
  6668/tcp   open        ?

The ?'s are there because I can't remember what they said - something to do
with IRC (I believe those ports are commonly used by IRC).

Anyway.

I thought this was a little strange, considering I don't run any IRC
servers on my network, and if I did, I sure as hell wouldn't run them on
NT.  So I telnetted to port 6667 of the server.

It did this:

  [root@medusa /root]# telnet jester 6667
  Trying 10.1.1.5...
  Connected to jester.aiew (10.1.1.5).
  Escape character is '^]'.

.. and I pressed enter a few times...  then it kicked me off

  Connection closed by foreign host.
  [root@medusa /root]# 
  
So I rescanned the server - and the ports were closed.

Both 6667 and 6668 were closed.

Should I be worried?  It's in another building, and for some reason, the
VNC server has stopped working - so I'm going to have to use that age-old
NT administration tool - the car - to check it out.

The linux bits of this post are nmap, and I telnetted from a linux box :)

Anyone ever seen these ports opened on an NT server before?
