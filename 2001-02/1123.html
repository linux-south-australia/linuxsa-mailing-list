Samba & IIS

Hi all...

quick question re Samba 2.0.7 and IIS 4.

Samba is sharing home directories to mainly NT clients, using cleartext
passwords. This works fine. We have been using Samba 1.9.18pl1, but recently
upgraded. The home directories didn't break at all. Samba also shares a
directory to IIS on an NT server. IIS now can't map to this directory. It's
kinda weird, as it appears that Windows Explorer can see it, but IIS can't.
Interestingly, if we change the global security setting to "security =
global", then the IIS share works but the home dirs don't. We don't want to
go to encrypted passwords, for various reasons.

Summary:

homes: works on either version of samba, as long as we don't use "security =
share" per user passwords etc.
IIS share: works on 1.9.18pl1, or on 2.0.7 as long as we use "security =
share"

how do we get both of these working at once?
