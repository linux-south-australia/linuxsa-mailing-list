This is an interesting question. Only becauase I caused a double failure
in a production oil pipeline control system [1] because of the
interesting features I'll slurge here:

IT DEPENDS

:)

Some versions of *NTPx do time syncing differently.

In ntpd, if you say 'slew' as an option, it will slowly adjust the
clock.
if you say 'always slew' it will slowly adjust the clock even for bigger
differences, except when those differences are bigger than 128ms [2]

In xntpd if you say slew, it will slew the clock
if you say always slew it will always slew the clock.

Slewing is the process of adjusting the kernel variable that dictated
how many ticks make up a second. This way, the machine can experience
all 60 seconds in a minute, but maybe not 1000ms in every second.

So, if your slewing over 5 minutes, it make take hours (even days). IT
is very platform dependant and very version and protocol dependant.

You would be better off stepping the time on `little boy' to sync with
your server (use an NTP utility to do this) then use ntpd to keep the
time synced.
