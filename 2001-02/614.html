Re: setting up a gateway for a two machine subnet

Dan!

>         I have been asked to set up a gateway machine for a two machine subnet for a friend (I have never
> tried anything like this before). He has a 486 (specs not available to me yet) to act as the gateway
> for a Mac (pre OS X) and a Windoze98 machine. What kinds of things should I be reading about and
> what can I expect the 486 to support. The subnet will be run off a dynacally assigned IP (read
> between the lines). The 98 machine is involved in serving streaming data , tha Mac primarily is used
> for surfing.

You need to read things such as:

* the firewall HOWTO
 - sounds odd, but masquerading and NAT creep into it as well
 - don't ask what they mean; the HOWTO explains it

* ipchains and ipmasqadm man pages

* TCP/IP
 - get a basic understanding of IP especially

Incidentally I assume you mean a gateway to the Internet. If you
actually mean "dial up server" or "remote access server" then lookup the
documentation on mgetty+sendfax (/usr/doc/mgetty+sendfax-[version
number]/).
