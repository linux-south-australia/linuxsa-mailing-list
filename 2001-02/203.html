> With all due respect I'm yet to see a development suite
> under Linux with the same level of functionality that
> Kylix should allow...although I am happy to admit that
> I could be wrong.

Do you know how to:

* program gnome or gtk+?
* program Qt?

In turn, have you ever:

* used GLADE?
* used KDevelop?

Both these widget sets contain drop in components that range from the
simple combo boxes, to whole pre_made default menus, to editors that
will highlight and indent C programs automatically...

Not only that, but providing you have the right GLADE module - really an
XSL translator - you can turn GLADE into:

* gtk+ under C
* gtk-- for C++ (yes, gtk-- is a C++ widget set)
* gtk+ under PERL

And even more...

> I will not be drawn into a discussion about the so-called
> incompetence of Microsoft products...

I am not drawing you into a discussion about that at all. I was pointing
out that under a set of similar circumstances, and with nothing but
licencing cost - and perhaps reliability - Linux would be the most
sensible implementation. If, for example, you wanted a company web
server whose only purpose was to serve static web sites, and it was a
box all on its own, if you had the knowledge to drive IIS + Windows 2000
or RedHat 6.2 and Apache, would you:

* recommend Windows 2000?
* recommend RedHat 6.2 and Apache?
