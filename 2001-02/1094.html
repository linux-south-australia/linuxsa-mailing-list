Re: Replacement Mail Clients

>>>>> "DL" == David Lloyd <lloy0076@rebel.net.au> writes:

  DL> I'm looking for a replacement mail client (currently using
  DL> Netscape Messenger 4.7X)...it must

David!  A person that does as much "posting" as you do certainly needs
top notch tools.  That is why you need (X)emacs gnus.  You can use it
to read mail, news, headlines, etc.

  DL> Understand mime type text/plain, even stuff created by Outlook

Gnus v5.8 is fully MIME aware.  (I do not get "bothered" by HTML
anymore, because gnus quietly strips it out before I read it.)

(But yes, posts to LinuxSA in HTML format are a waste of bandwidth,
etc etc)

  DL> Run under X

Gnus is your friend.

  DL> Operate on click and point principles including filters (I don't
  DL> want to use procmail's filters if I can avoid it)

Gnus supports static or adaptive filtering, scoring, threading,
"washing", pgp, etc.

Gnus can operate with or without lots of clickable buttons.  See the
documentation on `gnus-carpal'.

  DL> Speak SMTP flawlessly

Of course.

  DL> Not be kmail

gnus is not kmail.

  DL> Not be text based - I am looking for WYSIWYG and click, point; I
  DL> only use things like PINE when I don't have access to X

As an added bonus, it runs under X or on a text console.

Sometimes one just has to take the time to sharpen one's tools so that
future work can proceed more efficiently.  I recommend gnus to any
technically minded professional that needs a full-featured mail/news
reader.  It is really worth the time it takes to set up and learn,
because it takes all the hard work out of managing large amounts of
email.  

No more excuses, David: time to give it a try!
