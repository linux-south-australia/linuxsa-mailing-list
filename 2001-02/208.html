> 
> There is no Windows work at all; You just need to make sure that the
> firewall permits Windows SMB protocols through.

You will need to implement something like a WINS system, or train staff
to use \\1.2.3.4 windows shares, because the "network neighbourhood"
type name broadcasts aren't likely to be seen over the routed modem link
to the remote user if the dialup server is not the same box as the smb
server.
