Re: newbie kernel installation question

Michael!

> 1.    unpackage the following into /usr/src directory using following
> command
>         # bzip2 -cd linux-2.4.0-test11.tar.bz2 tar xvf -
> 
> when I do this I get screen output, but when I check the usr/src/linux
> directory it seems to be linked to a directory /usr/src/linux 2.2.14-5 and
> when I look at the changes documentation it refers to kernel 2.2.
> What am I doing wrong here??

Nothing.

/usr/src/linux is generally a link to the real kernel source. Either:

* rename linux-2.4.0 to just linux (man mv)
* execute "man ln", look up the "-s" option and link /usr/src/linux to
/usr/src/linux/2.4.0

> 2.    Using RPM to install the package.  I have the following version of
> RPM -         3.0.4.  Using the command:
>         rpm -u kernel-2.4.0.26.i386.rpm
> it then comes up with an error something like older versions <=3 not
> supported

You have to get the newest version of RPM but before you do there are
some issues. Firstly, if you intend to use it you'll need to rebuild
your RPM database. Secondly, finding it is an absolute totaly pain.

Look up "RPM Version 4" on the mailing list archives. You might want to
also add "lloy0076" to it because I created a huge thread on it ages ago
and there was a link somewhere in it that pointed to an FTP site where
you could get the RPM binaries.
