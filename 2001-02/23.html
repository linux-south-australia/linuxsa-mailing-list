You can have the different IPs on one subnet, differenct IPs on seperate
subnets (like your internal - external example above) and apparently,
there's a way of running two NICs with a single IP address. The first
and last of these three situations could be used for redundancy or just
general performance increase (I've never really looked at this, but I
would think it would be possible for ARP replies to be done in a
'round-robin' type fashion, allowing both cards to have the same IP
address).

I would be interested to see some of the replies you get from the Debian
List.
