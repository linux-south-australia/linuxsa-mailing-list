Re: Sendmail 8.9.3 -> 8.11.2

Michael Kratz wrote:
> 
> > > If it's RedHat, you should be able to get away with something like:
> > >
> > > * rpm -Uvh (the package name)
> >
> 
> Where do I find an RPM for 8.11.2 the only one that I can find for RH 6.2 is
> 8.9.3-20.
> 
> >The only problem with this is that RedHat's RPMs are missing a few
> >compile-time features (like LDAP routing). I would personally back up
> >any files you want to keep across the two versions and uninstall old
> >sendmail before compiling and installing new sendmail.
> >
> 
> How do I uninstall 8.9.3

rpm- e sendmail

> 
> and then how can I do it to a system which has to be running 24/7

The system will. Sendmail will be down for a very brief time.

> 
> could I:
> 
> 1. backup the config files (can u list all the ones that I should pls)

Typically:

/etc/sendmail*
/etc/aliases*
/etc/mail/*
/etc/rc.d/init.d/sendmail (this file usually checks for some of
sendmail's files. You may need to edit the location for them)

After recompiling, sendmail* and aliases* move to /etc/mail

> 2. stop 8.9.3 daemon

Leave this running until you are ready to install sendmail after
compiling it - less downtime. You may also want to test your old
sendmail config on a machine running new sendmail before the upgrade
(going through the whole upgrade process is a better idea, as it will
give you a better idea of what to expect).

> 3. uninstall 8.9.3
> 4. unzip 8.11.2
> 5. compile 8.11.2 (does it install itself??)

Pretty much ( ./Build ). You may want to add additional features. If so,
read about these before compiling.

> 6. overwrite the new 8.11.2 config files with the 8.9.3 config files
> 7. restart sendmail daemon
> 
> will this work????

We're running sendmail 8.11.0 or 8.11.1 on two of our servers here ....

Good luck and RTFM before compiling ;-)
