Would comm work for your purposes?  You would probably want to suppress the
duplicate lines.  That is:
   comm -3 file1 file2

You'd be able to see which lines are unique to each file.  Of course, that's
assuming that both files are sorted.  And the output might be a bit more
than what you're looking for.  (output for the above command is two columns:
column 1 shows lines unique to file1, column 2 shows lines unique to file2)

I don't think comm needs to use much memory to run.  I think it only uses as
much as is needed to keep track of where it's at in each file and the
current lines it's comparing.

Being a Linux newbie, however, I could be very, very, very wrong.
