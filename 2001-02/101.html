It was Sun, Feb 04, 2001 at 10:36:20AM +1030.  Michael Pearson spake:
> * Prashanth N V (nv@sasken.com) wrote:
> > Hi,
> > 
> > Can any one let me know the source for  Adaptec 2940 SCSI Controller
> > Drivers for linux and also i'm installing a Segate 4MM Dat Drive so pls
> > help me with this.
> 
> It seems to be included in linux 2.4.1. Look at:
> 
> SCSI Support->Low Level Drivers->Adaptec AIC7xxx support
> 
> The help for that option contains:
> 
> : This is support for the various aic7xxx based Adaptec SCSI :
> controllers. These include the 274x EISA cards; 284x VLB cards; : 902,
> 2910, 293x, 294x, 394x, 3985 and several other PCI and : motherboard
> based SCSI controllers from Adaptec.
> 
> I'm not sure about the DAT drive though... it should just "work" if SCSI
> tape support is enabled.


Specifically - if the SCSI driver and the tape module (st.o) load
correctly, you can use /dev/st0 (probably) as your tape device.

# find /home | cpio -blah > /dev/st0

The message I get in my /var/log/messages after the st.o module loads
succesfully is:

Jan 31 20:55:40 medusa kernel: st: bufsize 32768, wrt 30720, max buffers 4, s/g segs 16. 
Jan 31 20:55:40 medusa kernel: Detected scsi tape st0 at scsi1, channel 0, id 6, lun 0 

Hope this is useful in some way :)
