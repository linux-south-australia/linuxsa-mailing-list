Squid authentication modules are just standard executable programs.
Squid runs the program, which accepts the username and password
seperated by a space on stdin. The module does it's thing and then
displays either ERR or OK on stdout. Try running the module by itself
from the command line. This will tell you whether it's the module's
config that's broken or whether it's squid's. These types of modules
usually accept their configuration from the command line arguments you
provide, so check these.

You could also run tcpdump whilst testing the module, so that you can
see how it talks to their servers. I would be quite interested to see
how they're doing it (I think it might be radius).
