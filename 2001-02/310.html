One common problem is a missing synlink.
This happens with kppp and other programs tht look for /dev/modem, which 
susuallya symlink to a serial port(at least with enxternal serial
based modems)

Does the messgae sa soemthign like can't find /dev/modem

If so it is probably because thtat file(which is usually a symlink) doens
not exist.

You will need to create it.

ln -s /dev/ttyS0 /dev/modem
or
ln -s /dev/ttyS1 /dev/modem

The first if your modem attached your first serial port, the second if it
is attached to your second serial port.

Of course if the message is somethgin completetly different then this
doesn't apply.

I have no idea what device is used if it is an internal modem, or a usb
modem.
