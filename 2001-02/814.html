RE: Java servlets

Yes I have created a full java-based webserver project!

I used Tomcat 3.1 (also known as jakarta), which is a java application that
processes JSP pages (available from www.apache.org), I connected it to the
Apache Web Server 1.3.12.  
I also connected the JSP's to an EJB server where the EJB's connected to a
threaded Database Connection manager transporting the data to and from the
database via CORBA (if required) to get and save data from an Oracle 8i
database.  I used CORBA (Inprise's Visibroker) as there was a requirement to
be able to connect to several databases simultaneously that were not all
Oracle 8i databases.
The process works well with Apache processing the web requests and passes
any JSP requests through to Tomcat, which then builds and/or submits the
required pages to and from the user.  
I also configured the JSP pages so that they were precompiled which removes
a runtime compilation step. 
The performance is excellent (the backend was a Sun Sparc).
I have also installed the same configuration on a Red Hat 6.2 partition on
my notebook with success, so that I am able to test configurations and code
changes in a small, single environment before uploading it into the Sun
Sparc's Unix environment.

I have had no problems with this infrastructure and find it very tailor-able
to changing needs and load, with no performance issues.
