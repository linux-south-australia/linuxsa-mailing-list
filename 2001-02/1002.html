Re: Log file questions and issues

> Feb 19 14:43:43 <hostname> sendmail[28353]: NOQUEUE: Null connection from
> [207.215.117.200]

Someone connected to your mail server, and then didn't send mail.
They probably wanted to check your mailer's make and model.

> Mailbox vulnerable - directory /var/spool/mail must have 1777 protection

It means that the directory permissions let anyone alter anyone
else's e-mail file.

> Feb 18 05:30:00 <hostname> named[27547]: Cleaned cache of 3 RRsets
> Above: NAMED spews all this crap into my log files and bloats them, 

It is the statistics on BIND's cache performance,

> how can I get rid of it all?? and what does it mean anyway??

You configure BINDs "statistics" logging off.

For a quieter BIND8, add to /etc/named.conf:

  logging {
        category statistics { null; };
        category lame-servers { null; };
        category cname { null; };
   };


> My old man has his machine set to check mail every minute, so it blows the
> log out ridiculously, is there a way that I can tell it to ignore from a
> specific IP address??

No.

If log sizes worries you, compress the old log files.  In most Linuxen
this can be done automatically.  Then any often repeated text will be
compressed in to a few bits.

Personally, I view detailed log files as a good thing, the bigger the
better.  If they use 1Gb then it's still only costing you $40 or less
and you can easily blow that sort of time tracking down a problem that
wasn't logged.

> root@212.83.149.108

> Above: I have seen refusals to connect before but never root@ip.add.ress

DNS held no mapping for the IP address to the DNS name (called the
"reverse mapping").  You can check for a reverse mapping with

  dig -x <ipaddr>
