sort of. The linux source lives in a symobilc link /usr/src/linux. When
installing a new kernel you should always remove the existing
/usr/src/linux

# cd /usr/src
# rm linux     # this is a symbolic link so it will work with rm
# tar xvzf linux-n.n.n.tzar.gz
# mv linux linux-n.n.n
# ln -s linux-n.n.n linux

where n.n.n is the kernel version

Note that you have probably written all over the existing soruce tree you
will need to re install the old kernel sources if you want to go back to
them. Remeber to get the new /usr/src/linux out of the way first. Other
than that I am using the new 2.2.9 and had no troubles compiling.
