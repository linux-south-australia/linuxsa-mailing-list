Re: quick questionair

Toby Corkindale <tjcorkin@steadycom.com.au> wrote:
> I use samba inbetween my PCs.  Thus I can use either a command line
> "copy" or "cp" depending on OS, or else drag-n-drop with a GUI.

This is a good suggestion, although I'm personally of the opinion that
Windows is particularly bad at groking SMB.  In particular I've yet
to find a method which permits me to choose the username and password
that I use to authenticate myself on the remote server at the time that
I connect to that server.  Worse, I think it's very different between
Windows 9x and Windows NT.

Apple got this right in a way that Micrsoft got wrong: On a Macintosh
you type your foreign username and password when you mount the volume.
This is true even for SMB mounts (there is an SMB client for Macintosh)
so I say that Macintosh is better at Windows networking than Windows.

Finally, while I have no doubt it is possible, I have yet to find a way
to use SMB over a vanilla WAN.  I'm sure I ought to be able to use an
IP address, or a DNS resolvable hostname, however the few times I tried
it while idly wondering, it failed.  (If I really cared (which I don't,
being that I no longer use Windows) I'd work it out.)
