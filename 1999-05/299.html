Re: new to dial up users

Ben Donohue wrote:

 > i do hosting but no dial-up users, however i'm looking at starting to
 > have some dial up accounts. i've been searching on my own but thought
 > i'd ask the list for their opinion if i may. can someone point me in the
 > right direction for a cost effective dial up solution using linux
 > software.

mgetty with AutoPPP support can take the calls;  If you fill up a
multiport serial card (anything from a dumb "8 16550's on an ISA card"
type through to a Cyclades or Specialix "smart" RS-232 card) with 
33.6k modems you oughtn't have any trouble.

Test your installation on your onboard serial ports; when you're 
sure it's working, spring for the extra hardware.

If you want to accept incoming 56k calls, you're doomed: You need
primary-rate ISDN to handle 56k.  You'll have to get a Cisco Access
Server (like an AS-5300), an Ascend Max, or a Livingston Portmaster.
The redeeming feature of those guys is that they can handle your ISDN
net connection too; you'd then use the Linux box as a radius or TACACS
server to provide authentication services to the access server;  Another
redeeming feature is that they handle 30 - 60 incoming lines in a 2-RU
profile (i.e.: you don't need to fill up a rack with modems).
