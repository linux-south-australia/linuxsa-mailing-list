Re: Why not a "Machine & Version ID"?

A ready made (almost) solution...
Given that most packages these day's use configure/autoconf...

See the following example...
--save as configure.in---
AC_INIT()
AC_ALLOCA
AC_GETLOADAVG
AC_OUTPUT()
-----------------
run autoconf in the same directory -> generates a 'configure' script
run ./configure
which produces the following (on my machine anyway)
- Output ------------
loading cache ./config.cache
checking how to run the C preprocessor... cc -E
checking for working alloca.h... yes
checking for alloca... yes
checking for elf_begin in -lelf... no
checking for kvm_open in -lkvm... no
checking for getloadavg in -lutil... no
checking for getloadavg in -lgetloadavg... no
checking for getloadavg... no
checking for sys/dg_sys_info.h... no
checking for inq_stats/cpustats.h... no
checking for sys/cpustats.h... no
checking for mach/mach.h... no
checking for nlist.h... no
checking whether getloadavg requires setgid... no
updating cache ./config.cache
creating ./config.status
-------------------
The info pages have more information on how to set up more tests
(the AC_ALLOCA and AC_GETLOADAVG are tests)

It would be too hard to write some more tests that fit into this
schema,  and it would be useful for the entire community!
