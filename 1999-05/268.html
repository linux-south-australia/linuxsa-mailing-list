RedHat 6.0 in.telnetd

Just installed RH 6.0 (kernel 2.2.5) and have had an old problem come back.

I'm running virtual services, using the Virtual-Services HOW-TO as a guide.

When attempting to telnet to a virtual server, the session fails, with the
error "telnetd: All network ports in use."

On the old machine, (RH 5.1 upgraded to kernel 2.0.36), this was caused by
the /dev tree not being filled out correctly.

I have attempted to fill out the dev tree by the following methods, with no
improvement:
	cp -pdR /dev /virtualpath
	hard links
	symbolic links
	tar, copy, and un-tar

I am expecting it has to do with the pty devices, but I'm not sure where to
go from here. Ideas?
