What network card have you got? Is it setup in /etc/modules.conf
correctly?
You're getting the errors because the network card isn't setup
correctly. I'd suggest removing the network card, rebooting and make
sure kudzu removes the existing config for it, then replace the card and
run kudzu again. (kudzu is redhat's hardware detection tool, and should
run automatically at boot)

Once the network interface card is installed properly you need to setup
your networking properly. At a root prompt typing setup will start a
menu driven series of configuration tools, including networking. If in X
you could run redhat-config-network-druid (as root) or go to your menus,
System Tools -> Internet Configuration Wizard.

> 
> I thought that reinstalling would have reset the iptables
> but obviously hasn't.  As a relative newbie I have no idea
> about iptables and have been unable to find any helpful information
> on the net.  I've also tried unsuccessfully to activate the LAN card.

You can control iptables on the command line with the commands...

service iptables start
service iptables stop
service iptables save
service iptables status

Do service --status-all to see the full list.

To start/stop services like iptables for various run levels (see
/etc/inittab for an explanation of run levels) use the chkconfig
command.
i.e.
chkconfig --level 35 iptables on
or more simply...
chkconfig iptables on

Both chkconfig and the service command are the command line equivalents
of running ntsysv, or for the X gui run redhat-config-services or go to
your menus and find System Settings -> Server Settings -> Services.
