Re: Looking for opinions: Qmail vs. Postfix

Simon Lindsay wrote:

> It doesn't use /etc/init.d/ or /etc/inittab to run the services, it uses some
> special daemon thing, which does the job based on the /service/ directory. I
> forget the name of it.

Daemontools (and ucspi-tcp for the inetd replacement).  I actually like it
better than inittab, but I have an issue with it not being able to handle
dependencies with other services (I know you're supposed to take care of
that yourself in the run script, it just kinda sucks having to homegrow
something else).

You're not obligated to run qmail under daemontools/ucspi-tcp, though.
