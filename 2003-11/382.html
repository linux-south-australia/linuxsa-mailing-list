Re: Cron job

Darrin,

You have the general idea of cron correct.  Your problem might be that the
tool doesn't work the way you think it does.  Make sure that you can issue
the command from a shell prompt before setting it in a crontab.  Also,
make sure that your command isn't relying on any environment variable.
You can clear all environment variables (in bash) using:

  unset `env|sed 's/=.*//'`

Do that, and then when you can successfully run your command from the
shell you can probably transcribe the command into crontab.
