Sendmail Aliases

Hi,

We have a sendmail box which receives email and runs it through a virus
scanner and spam filter before delivering it either locally, or sending it
off to the customer's own Exchange server.  In some cases (such as the
example below) the domains are configured as local domains on our server so
that some user mailboxes can be hosted locally, and others (on the same
domain) can be hosted on Exchange.  When we do this, we set up aliases for
the users which are hosted remotely like so...

john@company.com.au:  john@exchange.company.com.au
bob@company.com.au:   bob@exchange.company.com.au

So that the message is delivered locally to our server, then forwarded on to
the exchange server.  What I'd really like to know is if there is a way of
aliasing the same username in different domains to different locations, like
so...

testme@company1.com.au:  test1@exchange.company1.com.au
testme@company2.com.au:  test2@exchange.company2.com.au

If I try to add the above to my aliases file, I get this when I run
newaliases...

/etc/mail/aliases: line 38: testme@company2.com.au... Warning: duplicate
alias name testme

Is there a way to alias based on both domain name and username, or a
facility like aliases which would allow me to do this?  I'm running Sendmail
8.12.10.

Thanks,
Jim.
