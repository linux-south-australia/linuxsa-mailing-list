It sounds like its timing out before the handshake is complete. Have a
look if the chat session is being logged to /var/log/messages and see if
there is a reason logged when it dies. If its to do with a timeout find
the chat script which should contain something like this (minus the auth
stuff as it probably leaves that to pppd)

exec chat -v                                            \
        TIMEOUT         3                               \
        ABORT           '\nBUSY\r'                      \
        ABORT           '\nNO ANSWER\r'                 \
        ABORT           '\nRINGING\r\n\r\nRINGING\r'    \
        ''              \rAT                            \
        TIMEOUT         40                              \
        OK              ATX3M0                          \
        OK              ATDT81689056                    \
        CONNECT         ''                              \
        ogin:           <user>				\        	assword:        <pass>

This is from an old dialup script I used to use, and Im not sure where
esmith is going to hide its equivalent, or indeed if the options are
meant to be set in a config file instead.
