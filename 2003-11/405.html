My question is this:  Since I know that I can use my Linux Distro =
(Redhat 9) to act as (amongst other things) a DNS/Proxy/Web Server - I =
would like to know if I can use a non-static IP address to host the =
domain that I own on one of the four boxes on my home network...

Let me be as specific as I can so that there's no confusion: My ADSL =
connection is through Internode.  My understanding (I haven't confirmed =
with Internode) is that I do not have a static IP, and that every time I =
dissconnect and/or reboot my ADSL Modem/Router I automaticly get =
assigned a different 'Master' IP address by them.

However, I also know that all of the boxes on my Home LAN (4) all have =
static IP addresses permenantly configured - through the Router/Modem - =
because (for example) when I'm serving on MIRC - I know that regardless =
of how many times I've had to reboot the Modem/Router (I've got a crapy =
Billion which I'm about to upgrade) that MIRC always finds the right =
box, and everything correctly connects enabeling peops to get the stuff =
I'm serving without having to adjust Port forwarding on the Billion =
Modem/Router each time I've had to reboot.

So, to get to the point: I 'own' a domain (a .com.au address) and what I =
would like to know is: Can I configure/Port forward (or whatever) one of =
my Linux Redhat 9 boxes, to host the domain that I own so that =
regardless of how many times I connect/dissconect from Internode, a =
person going to my URL ( EG: http:www.mydomain.com.au ) and/or sending =
me an E-Mail ( newbie@mydomain.com.au ) will always arrive at the Redhat =
9 box I'm using to host my domain on - regardless of how many times I =
reboot/reset my Modem/Router's connection to Internode?
