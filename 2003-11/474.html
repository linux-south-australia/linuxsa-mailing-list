Re: IMAP folders question

Kim Hawtin wrote:

> I'm currently using Cyrus 2.0.12 on a 4.3 FreeBSD box,
> but there seems to be no support for sub folders,

Cyrus supports sub folders fine (says he who has far
too many).

> it also appears to be using mbox format to store mail
> in each users home folder.

This doesn't sound right; are you sure you're running
Cyrus?  It should be storing the email under a defined
directory.  What does "telnet imap_server 143" return
as a response?  For reference, you should get something
like:

* OK zen.testernet Cyrus IMAP4 v2.1.9 server ready

In fact, it can't be right - Cyrus uses a common user ID
(typically, "cyrus") to create a "black box" type IMAP
setup, so it can't get access to user owned mailbox files.

> is this a config issue on Cyrus?

Not that I'm aware of.
