As I mentioned at last nights LinuxSA meeting :) the Adelaide Uni
central web proxies run Squid under Linux (authenticating against
LDAP).  I might give a talk about it some time.  Not sure why you
thought they were windows?  Nmap reports them as unknown due to the
firewall.  Visitting http://www.adelaide.edu.ua/ quickly confirms that
they are Squid.

The proxy.pac does not enforce the access controls, it is merely a
central config file which lets your browser where the proxies are. 
Access control is enforced via per subnet firewalling and proxy
authentication.  I've included an example proxy.pac below.  All major
browsers support proxy.pac (Netscape, Mozilla based, IE).  More info on
this at:
http://wp.netscape.com/eng/mozilla/2.0/relnotes/demo/proxy-live.html

Some subnets have direct outbound access, some dont.  It depends if the
department responsible for the subnet wants to pay for the traffic this
will generate.  If you abuse it, obviously they will be less inclined to
pay.

Example proxy.pac:

function FindProxyForURL(url, host)
{
    if (dnsDomainIs(host, ".adelaide.edu.au") &&
        !localHostOrDomainIs(host, "access.adelaide.edu.au"))
        return "DIRECT";
    else if ((isInNet(host, "127.0.0.1", "255.255.255.255") ||
              isInNet(host, "129.127.0.0", "255.255.0.0")) &&
             !isInNet(host, "129.127.123.0", "255.255.255.0"))
        return "DIRECT";
    else
        return "PROXY www-proxy.adelaide.edu.au:8080";
}
