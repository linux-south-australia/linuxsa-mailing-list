Re: [Talk] SCO news - targetting BSD & 'major user', more money to law firm

On Saturday, 22 November 2003 at 13:56:58 +1030, David Lloyd wrote:
>> I find I am not deeply worried about the chance of a negative outcome.
>> Not because it isn't possible, but because the open source community
>> will prevail regardless of the outcome. They have said from the start,
>> "show us your code so we can remove it, we don't want it", and once SCO
>> identifies it (if it is even there, which I doubt), it will get removed.
>
> /me sigh
>
> But this only proves that there IS stolen code in the open source code
> base.

It proves nothing.  SCO doesn't even know which parts of System V are
original and which have been incorporated from elsewhere.  In August
this year they showed "proof" of code in Linux which they said came
from System V.  In one case, it was demonstrably BSD code.  See
http://www.lemis.com/grog/SCO/code-comparison.html for details.

> Which makes ALL installations of open source material suspect. I'm
> not sure whether *this* would actually do us any good at all.

What the case is highlighting is that code theft is possible.  So far
it hasn't shown any concrete examples, and I'm hoping that when they
do, it will point to SCO having misappropriated code from elsewhere.
Take the BSD example above, for example: yes, they're allowed to use
it, but they need to maintain the license statement in the source
files.  If they claimed it was theirs, there's a good chance that the
BSD license statement is missing in that file.
