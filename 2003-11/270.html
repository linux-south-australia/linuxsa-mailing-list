> Logically I should perform all the reformatting on the Linux

It's possible you are dealing with filetypes that require tools that are
not yet available on linux. Until this is known, it will be hard for
anyone here to offer much help. You may also run into font problems,
depending on what your authors have dreamt up.

> 
> SOLUTION 2
> Co-locate a  Linux  box anywhere I can get power and a 
> permanent IP and FTP the files to there. Run Gimp using 
> command line and macros if possible and then a remote X 
> session to finish off, and then FTP the files back to the server.
> 
> This second solution seems to be the best bet at the moment 
> and I am looking for constructive criticism on that. Also some 
> suggestions on suitable places to locate this box (offers 
> invited). If it is in Adelaide I do not really need all the 
> backup, redundancy and high bandwidth available in a web farm, 
> I have three days to complete the task each week.

If a lot of the files are easily convertable on linux via commandline
tools, I'd be writing scripts to recognise and process those files, and
put them in a temporary area that you can get to via your browser. This
would enable you to check the conversions before they got published.

Any remaining files might require you to d/l and work with them on your
own system. Using remote X is do-able if you can get a friendly host to
allow it (security would be an issue), but it assumes the files in
question are editable in linux.

Suggest you post a list of filetypes that you have to deal with.
