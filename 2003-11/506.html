Yup, checking the Makefile again, I did compile it with MPPE.

At around like 35 of pppd/Makefile:
# Uncomment the next 2 lines to include support for Microsoft's
# MS-CHAP authentication protocol.  Also, edit
plugins/radius/Makefile.linux.
CHAPMS=y
USE_CRYPT=y
# Don't use MSLANMAN unless you really know what you're doing.
#MSLANMAN=y
# Uncomment the next line to include support for MPPE.  CHAPMS (above)
must
# also be enabled.  Also, edit plugins/radius/Makefile.linux.
MPPE=y

And around line 94:
ifdef MPPE
CFLAGS   += -DMPPE=1
endif

I remember watching it build with Chris and it did compile with -DMPPE=1.
The first one we used wasn't compiled with MPPE support and pppd really
barfed saying that require-mppe was an invalid setting.  Its not doing
that now.
