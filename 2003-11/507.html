What is in your /etc/ppp/options.pptpd file?  Also which exact version of
pppd are you using?  (Output of "rpm -qa | grep ppp" if you're using rpm, or
the name of the original tarball file if you compiled from source.)

I had a problem with poptop awhile back giving similar errors in the logs to
yours.  Apparently there are two versions of ppp with mppe included.  There
are patches to older versions of ppp (2.4.0, 2.4.1, I think) and a newer ppp
with BSD licensed mppe built in (2.4.3+?).  The problem is that they use
differently named options (i.e. in options.pptpd) to specify accepting 40-
or 128-bit MPPE encryption and auth types to allow.


PPP with MPPE patch would have something like the following in
options.pptpd:

-chap
-chapms
+chapms-v2
mppe-128
mppe-stateless


And the BSD licensed version would have:

refuse-pap
refuse-chap
refuse-mschap
require-mschap-v2
require-mppe


Your configuration may be using the wrong options for the ppp version you
have.
