Remote Graphic Editing

Am I on the right track.

THE PROBLEM
Each week I have a task where I download about 20-40MB of assorted
Quark/PDF DTP files which are converted to PHP/PNG files and re-uploaded
to a Web server (~5MB). 
I plan to be travelling for some time and the download size is a problem
particularly in China. The major bulk is publishing resolution PDFs and
TIFFs which I currently render in Photoshop and crop and resize and save
as PNG8s

SOLUTION 1
Logically I should perform all the reformatting on the Linux 2.4.21-xfs.
Virtual Server using command line tools and never move them around, but
I have had trouble finding suitable software tools that are within usage
agreement. Since the 50 odd files come from 50 odd different authors, I
do not know what needs to be done to them until AFTER I preview the file!

SOLUTION 2
Co-locate a  Linux  box anywhere I can get power and a permanent IP and
FTP the files to there.
Run Gimp using command line and macros if possible and then a remote X
session to finish off, and then FTP the files back to the server.

This second solution seems to be the best bet at the moment and I am
looking for constructive criticism on that. Also some suggestions on
suitable places to locate this box (offers invited). If it is in
Adelaide I do not really need all the backup, redundancy and high
bandwidth available in a web farm, I have three days to complete the
task each week.

Thank you.
