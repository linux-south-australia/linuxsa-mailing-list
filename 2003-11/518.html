something like this can be achieved in mozilla - i do this kind of thing 
regularly. this applies to mozilla suite, but possibly not totally to 
mozilla firebird. i don't use mozilla firebird, so YMMV.

first off, the DOM inspector will give you the "click on an element to 
display properties about it" functionality - it won't display 
exclusively the CSS values, but you can dig through the properties it 
shows to find the relevant CSS stuff. even the computed style (based on 
the actual rendering), which can be very handy. you can install the DOM 
inspector as a sidebar, but that doesn't give you the 
click-on-an-element functionality; you'd have to drill down through the 
page structure to find the node you are interested in.

you launch the DOM inspector from "Tools -> Web Development -> DOM 
Inspector" in the menus.

the next major bit is the evaluating CSS rules as they are typed - this 
can be done via the "edit styles" bookmarklet, which can be found at:

	http://www.squarefree.com/bookmarklets/webdevel.html

you just create a bookmark to the 'edit styles' link (i have the 
bookmark in my personal toolbar so it's more like an actual button). 
when you click it, it opens a new window with the CSS from the current 
page pre-loaded in it; as you type new CSS rules it re-renders the 
underlying page to reflect the new rules.

advantages: it's cross-platform (win, mac, linux), it's open source, 
and it's one of the most (if not the most) standards-accurate CSS 
renderings available. disadvantages: it's not quite the dedicated CSS 
application you were after ;-)

FWIW, mozilla's composer app has some really good CSS tools in it; it's 
getting ported to be a standalone app called nvu (see www.nvu.com) by 
Daniel Glazman, sponsored by lindows (from memory). i don't think 
there's any actual releases of nvu available yet tho.

if you installed mozilla via RPM, you need the 'mozilla-dom-inspector' 
package installed to use the DOM inspector; otherwise, if you've 
installed mozilla from source or an installer, make sure you choose to 
install "all" or explicitly choose the DOM inspector if doing a custom 
install.
