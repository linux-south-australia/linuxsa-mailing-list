Mark Newton wrote:

> I suspect they're just running a search engine which ignores robots.txt
> files - I'm sure there's at least one of them around which doesn't look

It's not wget -- it provides a header that says something like:

 User-Agent: Wget/1.5.3

which can be refused.

Stupid crawlers don't provide a User-Agent and don't read
robots.txt.

Alan's module traps these and is really welcome.  You'd be
surprised how many dead-heads try to index mirror.aarnet.edu.au.

One change that would be useful would be to rate-limit top-level
diretcories, not files.  Then people can download the latest
RedHat release, but get trapped if indexing the whole site.
