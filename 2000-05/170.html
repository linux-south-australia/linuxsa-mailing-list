Re: IPC for Apache module child processes (technical)

On Mon, 8 May 2000, Alan Kennington wrote:
> The Apache throttle module I'm reading for inspiration
> (by Anthony Howe) uses the integer 0x61715f02 as the first argument
> to semget(). 

It has to be unique.  I think for you it won't matter.  I think you can
create the semaphore set (using IPC_PRIVATE) in the parent, and pass it to
the children that you fork.  You only need a pre-agreed id when there's
insufficient common ancestry between all users of the semaphore set.
(Ditto for shm and msg.)
