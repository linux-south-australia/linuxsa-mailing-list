Re: how do you stop site-downloaders?

> Does anyone have any ideas on the least time-and-energy
> way to do what I want with this?
> 
> I've even thought of implementing this sort of access
> limiting as a kernel module in the IP layer - once again,
> this is a lot of work.

One thing that IP-Chains allows you to do (although i don't know the
full details) is push packets to userland to have processing applied
to them.

The theory here would be that on the input chain of your webserver you
could push packets into userspace, work with the data and then push it
back into the IP stack to let it pass back to your server.  

The overhead for a low bandwidth connection shouldn't be noticable and
it would probably be the easiest code based solution.  

I guess one other solution might be to use SSI/PHP/eperl to talk to do 
something at the start of each page-view.

These are initial thoughts and full of cons, but nonetheless I hope it 
helps in some way.
