produce honking amounts of data (eg, squid for an ISP).  less will
chew up loads of VM, bringing your system to a standstill (or so
I've heard :-).  For a less (no pun intended) memory hungry version,
tail -f (or tailf) works as well, although one neat thing I like
about less is to highlight a word using /search, then run it in
follow mode (it will highlight subsequent words).
