I'm not sure can I agree that programs should not be able to send email
without user intervention.  Sendmail would be pretty lame if somebody had
to type Y for every outgoing message.  Anyway, the problem isn't that
Outlook can send automated mail; the determined virus writer can open a TCP
connection to your configured SMTP host and thus avoid what you suggest.
The problem clearly is that people, unknown to the recipient, can send a
message containing a program, and Outlook is configured to execute that
program by default.

"Hi, you don't know me but I've got this program I'd like you to run."

It's all very well for Microsoft to say that scripting is a neat feature --
and it is -- but is it a wise feature?  I'm looking forward to a class action
against Microsoft, which must SURELY be about to be launched, so that this
question can be legally tested.
