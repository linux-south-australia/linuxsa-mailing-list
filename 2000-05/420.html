Re: US congress versus McAfee

Well Javascript is limited by the security model implemented in the browser
(also by the implementation of itself as well).

For instance if a security model says that JavaScript can read from the
local disk, then any Javascript written from then on can exploit that
security model.

Java is subject to the same thing when written as an applet. If the browser
security model let's something happen, then the applet can do it.

Java run directly from the command prompt ie java x.class can have a
security manager attached to it. Basically create a class which is a
security manager and handle what you want. If you don't explicitly handle
something then it can not be done.

Hope this helps..

Rob.
