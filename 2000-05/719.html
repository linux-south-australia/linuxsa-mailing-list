Not sure about the dhcpd.conf file but I believe that you will have to setup
the router up to allow the token ring users to see the DHCP server.
The only reason I know this is I just setup a frame relay network with
Telsta and I was taking to the tech that programmed the cisco routers and he
said that it is possible to direct the traffic over the wan to the one DHCP
server.

Wouldn't it be easier to setup another DHCP server on the second subnet. I
recently set one up on an old 486 that was heading for the bin and used
RH5.2 on 200meg driver with 100meg to spare, works great.
