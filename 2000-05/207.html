PPS: global list of shm segments?

....I forgot to ask something important in my previous mail.

Is there some way in linux to obtain a global list of
all allocated shared memory segments on a running system?

I looked in /proc/meminfo and so forth, and found nothing
but a single number for the total amount of shared memory.
In the per-process /proc directories, I found nothing either.

It would be great if there was a system call to return a list
of all identifiers of shared memory segments, their addresses,
owners, users etc. etc.
It's easy to get such info for any segment that I can
guess the ID for, but that's not so easy in a 4-billion
address space.

I bet this is something you find out in a standard
comp-sci degree.
