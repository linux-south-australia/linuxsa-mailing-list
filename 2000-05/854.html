**start rant***
Just as an aside.... I dont know how many of you have looked at or installed
Corels Linux ( I dont like to diss, Corel have tried to make a User , read
Microsoft, friendly package and succeeded in parts), but the way in which
the libraries are situated and linked throughout the filesystem doesnt seem
to match what most applications are looking for, PLUS it doesnt allow for
upgrading of libraries when I tried to fix the problem (because the library
placement is hashed). Add this to the KDE debian binaries installer, thats
supposed to make it all that much easier (and which cant find the libs to
link to!), and your left with an 'almost' useless system on which its
'almost' impossible to install anything but the simplest apps on......In the
last 1/2 hour since I fixed the res problem RedHat has allowed me to install
all the apps Ive tried with NO grief...nada...zip. Speaks for itself really
:-)

This leads me to think that sticking to some kind of library placement
protocol could really go a long way in the opening up Linux to a wider user
group. Maybe most distributors do and Corel wasnt thinking, bad move really
since theyve got themsleves the title of the new user friendly dist.
**end rant***
