Re: Sendmail and DNS

I have exactly this setup at home. I have a SuSE 6.2 machine acting as
masquerading gateway / squid proxy / mail server / etc.

I run sendmail with the options '-bd -om' (without the -q30m, so it doesn't
retry thet queue) and with the sendmail expensive option set, so that it
doesn't run the queue unless you specifically flush the queue.

Then I set up a crontab for root to run every 10 minutes to flush the queue.
This is set up as a 'user' cron which is installed in my ip-up script and
deleted in my ip-down script.

I also have fetchmail setup to grab the mail from 3 email accounts I have,
and 2 my fiance has, and deliver them locally.

I've had it running for about a month without problems. I've never had a
problem with it refusing mail...
