pgp 5.0 for linux no good!

What a nuisance.
It turns out (according to slashdot, cryptome etc.)
that an elementary programming error made the
randomness of key generation in the unix versions of
PGP 5.0 rather weak.
Luckily I use a long passphrase. So I should be (fairly) safe.

Question:
Has anyone out there tried to update from PGP 5.0 to 6.5.1i?

If so, how have you dealt with these issues?

1.	The command line seems to have gone back from
	the "pgpk", "pgpe" etc. command line form to the
	older "pgp -k", "pgp -e" form as documented in the
	old O'Reilly PGP book by Garfinkel.
	Therefore this must cause difficulties for "mutt".

2.	The definitions of variables in the pgp.key file
	have changed in a non-backwards-compatible fashion.

3.	The software has the word "beta" on it.

It seems to me a little wierd that there has apparently been
a backwards move in the command line format.
A quick look at http://www.mutt.org/ seems to indicate no
modifications to account for PGP 6.5.1.

I assume that I must now modify my mutt and pgp set-up so
that everything adapts to the new pgp, and I'm not sure what
to do about the mutt pgp support - maybe just take out the
version 5 commands in the mutt config file.

Before I start making drastic changes throughout, I'd like
to know if I'm on the right path.

Can anyone confirm that I've got the right picture here?

Cheers,
Alan Kennington.
