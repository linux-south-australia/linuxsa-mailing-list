At 11:46 AM 5/31/00 +0930, Tomlinson, Brenton wrote:
>G'Day all.

>We are in the process of setting up a multi-OS machine for testing purposes
>and we want to use Lilo to choose between OS's. Do we need to have the OS's
>installed on different physical disks or is there a way we can do it by
>simply partitioning the disks. If we only partition then one will overwrite
>the other in the mbr or can Lilo fix or decipher that.

Hmmm, I covered some of this in my Linux course at DSTO a couple of weeks
ago. I must make sure I covered during the one I am giving this week :-)

You do NOT need to have them on separate disks. I have run three different
distros on the one disk, and with a little effort you can set things up so
that you can run lilo from any of the distros and it will create a correct
lilo/MBR setup to allow all three to boot fine.

You might also want a /boot if your disk is larger than 8.4GB (> 1023
cylinders).

Here is the sort of layout you have:

Under, Say RH 6.2, your set up

/rh62 as a symbolic link to /
/tl60 as a directory which is mounted with your TurboLinux partition
/suse as a directory which is mounted with your SuSE partition

You have a similar structure for each other distro, except the symbolic
link moves around.

Then you construct one lilo.conf that you copy to each distro's area 

    cp /etc/lilo.conf /tl60/etc/lilo.conf # for example

The lilo.conf has the following image sections in it:

image=/rh62/boot/vmlinuz...
    label = rh62
    ...         # anything else you need

image=/tl60/boot/vmlinuz...
    label = tl60
    ...         # Anything else you need

image=/suse/boot/vmlinuz...
    label = suse
    ...         # Etc ...

If you use /boot, things will be a bit easier.

I hope this helps ...
