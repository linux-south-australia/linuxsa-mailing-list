A little while ago (just after Corel 1.0 came out) I was stuck in a
situation where I had to install a server quickly and I only had a RH CD,
and a brand new Corel CD. I didn't want to install RH since rpms aren't so
easy to handle in a server situation unless you spend time tuning the
install environment (I'm not saying bad things about RH here, just that
rpm isn't state-of-the-art wrt network updates.) Since I knew that Corel
was based on Debian, I installed it, and then immediately removed the
Corel-specific fancy boot-time stuff.

Then I installed networking and used apt-get to upgrade the entire system.
No library problems, no errors of any sort. It's been working perfectly
for many months now. apt-get is very good at managing dependencies.

I think the difference is that you tried to use the fancy graphical
package mangement tools.
