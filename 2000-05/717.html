RE: Virtual Hosting

Hi,

Watch out for Virtual hosting, as in a purest sense you would be using HTTP
1.1.
This allowing a single IP to serve many websites.

As a result you will find that if the customers web browser does not support
HTTP 1.1 then they won't be able to get to any of your sites via DNS name.

I have seen an additional trick to get around this.
The method is to map a non HTTP 1.1 site to the IP which when the customer
with a non 1.1 compliant browser goes to, they get a list of the sites on
the server, not referenced by DNS name but by directory structure on the
server. They simply press the appropriate link and off they go.


Derek
