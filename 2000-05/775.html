Re: Squid

Hi...

> 	       I need some tip on how to setup Squid for my machine using
> red hat V6.2
> Can anyone help me with a basic squid.conf

Squid comes with a squid.conf which is almost ready to go.  All you need to do
is add ACLs for your domain.  Something like this should work:

This line goes with all the other ACLs:
acl mydomain 192.168.1.0/255.255.255.0

You should substitute 192.168.1.0/255.255.255.0 with the appropriate network and
netmask addresses for your network.

And these lines go after the
# INSERT YOUR OWN RULE(S) HERE TO ALLOW ACCESS FROM YOUR CLIENTS

http_access allow mydomain
http_access deny all

The second line should already be there.  This will allow access to squid from
machines on your network, but not from anywhere else.

You may also want to modify the cache_dir line, which specifies how big the
cache should be and where it should be stored.  If you want a big cache on
multiple disks you could add more cache_dir lines.

Once you've got it setup you'll need to run squid -z to build the cache
directories.

I also like to change the cache_effective_user and cache_effective group for
extra security.
