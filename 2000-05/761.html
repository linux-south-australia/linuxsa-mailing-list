Here\'s the way Xavier is headed (and almost there). A 
Samba TNG machine acting as the PDC. All other file 
servers (NT or Linux) will run Samba as a member of the 
NT domain controlled by Samba. Using the \'unix password 
sync = \' line in smb.conf, I will have the Unix password 
(stored in an LDAP directory) updated. I think this will 
work, because Windows apparently sends plain text 
passwords when changing passwords.

If the unix password sync thing won\'t work, I will hack 
the LDAP CGI scripts I am writing, to allow for a call 
to smbpasswd or samedit and have all password changes 
through a HTTP interface (which is the way the boss 
wants to go anyway).

we use PAM_LDAP, NSS_LDAP, LDAP_AUTH (squid) and soon 
mod_ldap_auth for Apache. Sendmail 8.10.x also supports 
mail routing and stuff through LDAP.

If you have any specific questions, I will happily 
answer them on or off the list (maybe a little time 
delay though).
