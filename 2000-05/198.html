i have had similar experiences here.

One thing that I did once as part of the troubleshooting, was to remove
all the chains from ipchains. This helped - coincedence maybe. So
perhaps the ipchains section of the kernel was gluggy.

Also, part of what IPchains does (having just read the HOWTO) is:
"
     Routing decision:
        The destination field is examined by the routing code, to decide
        if this packet should go to a local process (see Local process
        below) or forwarded to a remote machine (see forward chain
        below).


     Local process:
        A process running on the machine can receive packets after the
        Routing Decision step, and can send packets (which go through
        the Routing Decision step, then traverse the output chain).

"

As I understand this, this means that there is an ability to send the
packets to process which will do routing, as well as the standard kernel
routing.

Also, as I understand it, a packet will traverse the entire ipchains
structure (that is input->xx->output) before arriving at your telnet
program say.

So, consider turning off these extra user-space routing (and network
debugging stuff) in the kernel.

Some thoughts.
