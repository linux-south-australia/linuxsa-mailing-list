Re: Backuping a Linux Box

On Tue, 23 May 2000, Matthew Tippett wrote:
> Now for those of you who use Linux in a more formal environment, I am 
> interested in canvassing opinions as to what is a good backup configuration
> for Linux.

A tape rotation system based on dump works well, using custom scripts of
course.

But, if you use Redhat Linux 6.1 (and maybe others), do not use the
supplied dump/restore tools. They are broken. This can be shown when doing
something that involves piping dump to restore.

There are no new ones on the RH site, there is a later version out there
(try rpmfind.net) dump-0.4b16-1.src.rpm, and the accompanying mt rpm.

This may have been addressed in later versions. I think this is something
that Mandrake have not addressed in 7.02 either.

That dump only supports ext2fs.

In various tests, dump has been shown to be more reliable when backing up,
than cpio or tar. Rsync is something that I would be interested in testing
(for non-tape related backups). Something that supports a full
root partition recovery is important (eg: devices and pipes etc...).
