On Thu, 11 May 2000, Adam Hawes wrote:
> > is connected to telstra. I also have several other IP addresses which I
> > intend to use for the webserver and mail eg 203.38.200.123 and
> > 203.38.200.124 for www and mail respectively now what I would like to know
> > is whether I can set up a linux machine to cater for all three of these ip
> > addresses ?
> 
> Yes, in short.  In detail, it is a little difficult.  I know that Apache is
> able to be configured to respond to multiple domain names.  The Linux kernel
> network drivers are what is configured to ACCEPT multiple IPs into the one
> ethernet device.  I don't know how Apache deals with this tho.

You can tell apache to 'listen' to a given port on a given address using
the listen directive in httpd.conf. The following is from that file:
# Listen: Allows you to bind Apache to specific IP addresses and/or
# ports, in addition to the default. See also the <VirtualHost>
# directive.
#
#Listen 3000
#Listen 12.34.56.78:80

#
# BindAddress: You can support virtual hosts with this option. This
directive
# is used to tell the server which IP address to listen to. It can either
# contain "*", an IP address, or a fully qualified Internet domain name.
# See also the <VirtualHost> and Listen directives.
#
#BindAddress *

These can be found in the Apache html manual that is often installed with
Apache. You do need other interfaces or aliases to be configured on the
host in question.
