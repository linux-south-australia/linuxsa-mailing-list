Re: A simple (!) SSL Question

I did some work using SSL for SMTP transmissions etc.

when u access an ssl secure page u will see 'HTTPS' instead og 'HTTP'
if you client cannot negotiate a level of SSL then you dont see the page
but some form of error
saying secure page cannot be accessed.

simply ssl works as follows (once authentication is made via certificate)

Server <---- RSA  (asymetric) negotiation of symetric key ----> Client
(VERY SLOW thats why it's used for key negotiation

Server <---- DES encryption of data using mutually agreed symetric key --->
Client 

The negotiation phase determines the level of security that the browser and
server can handle
they do step down but not to an open connection. (unless specifically
specified within server setup)

The reason they dont use SSL to server all pages is no matter how fast
symetric encryption is over asymetric one it still takes time to
encrypt/decrypt.
