I looked into this a while back, as I have a 386 that would work well
running seti@home and a couple of other things that i could use it for
(print server, time server, dumb x-term).

I wanted to remote boot it and have it start Linux and a few simple
processed then X... getting it to NFS boot is simple enough (if you
compile NFS root into the kernel). You can either set it to boot of your
normal / on your big computer, or set up a filesystem for it, and
install into that (you can actually do this from the computer you want
to boot off the network if all the NFS shares and permissions are set
OK).

The key is that you NEED to compile NFS root into the Kernel.
