Re: runaway process

Matthew Tippett <mtippett@ticons.com.au> wrote:
> The /proc file system is a wonderful place :).  It allows you to look
> inside the kernel.  So for your problem you can use internal shell
> commands to see what is happening.  If you can run ls and cat then
> you can find almost anything.

And if you can't run ls or cat, but can run the shell, you can *still*
simulate them.

  ls() { echo *; }
  cat() { while read line; do echo $line; done < "$1"; }

and...

  more() {
    n=0
    while read line; do
      test $n -eq 24 && echo -n '-- more -- ' && read n < /dev/tty && n=0
      n=$(($n + 1))
      echo $line
    done < "$1"
  }

and so on.
