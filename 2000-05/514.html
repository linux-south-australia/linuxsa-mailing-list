A simple (!) SSL Question

If I've read my documentation correctly, if you set up Apache or another
web server to use SSL then your data is sent encrypted using your
Security Certificate (purchased from Thawte or someone such) and the web
client can verify that the certificate is from you based on your IP
address or some such thing. And that the data is encrypted by the
Certificate. And that the Certificate is of sufficient length (128 bits
or more), the data is for all practical purposes non-decipherable.

Hence:

* one could use the SSL protocol to serve absolutely any pages and
information you want AND all the information is encrypted...

In addition, what if the web client doesn't understand the SSL protocol
(for some reason)? Does it reject it or start displaying unintelligble
garbage?
