Re: disabling telnet,ftp to a user

Hi...

> We have webserver in isp running webmail,for demo purpose we need to
> create some users for our customers,so they will access the server through
> http,but since we create the users they will also be able to telnet to the
> webserver ,so i wanted to know if by any way i could put restriction for
> telnet ,ftp ,rlogin .rsh ,,,,,, to that perticular user/diff ways of
> restriction available,

You can stop telnet, rlogin, rsh and all that stuff by setting their shell to
something that's not a shell, like /bin/false or /bin/passwd (if you want to let
them change their own password).  This may or may not stop FTP access depending
on which FTP server you're running, and how it's configured.
