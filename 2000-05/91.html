Re: netbios DoS

I get these netbios-ns probes all the time.
These quite often come from a few asian countries
whose webmasters never reply to my hacker reports.
I assume that as in the case of the chernobyl virus,
asian countries were particularly vulnerable because
they don't seem to take computer security very
seriously. It may also be that the language barrier makes
it difficult for the relevant info to travel there.

Either way, I get so many netbios-ns (TCP and UDP port 137)
probes, I just ignore them now.

But of more interest to linux users (esp. red hat derived
distributions) would be the equally frequent port 98
scans. These happen a couple of times a day on my IP address
spaces. Most of these are from Korea, in my experience.

The problem is that these are not at all innocent.
There's apparently a highly virulent DDoS program
being developed (Mstream - see
http://packetstorm.securify.com/ ) which has been found
on linux machines. I don't know if they use port 98 to infect
people's machines (linuxconf).
But according to

http://www.theregister.co.uk/000502-000005.html

this Register article, and the notes:

http://packetstorm.securify.com/distributed/Mstream_Analysis.txt
http://packetstorm.securify.com/distributed/mstream.txt

it looks like it's red hat linux which is the vulnerable machine this time.

I think it's odd that although almost no virus (in the sense of
code infecting applications of boot records) ever seem to
infect linux machines, they do seem to have a particular
vulnerability to infection with DDoS zombies.

Not that I know anything about network security at all,
but if I were an RH linux server operator, I'd firewall
out port 98 immediately, and check that Mstream has
not yet infected your machine.
