It can. I don\'t believe you can make Samba 2.x control a 
Win2K Domain (although it will easily handle WinNt in a 
roundabout kind of way). What you want is Samba TNG (The 
Next Generation), the code is slowly making it\'s way 
into the Samba 2.x codebase. There are still some 
problems, but there are functioning implementations of 
samba TNG out there.

If you are interested, do yourself a *huge* favour and 
read Lars Kneschke\'s TNG FAQ and join the Samba-NTDOM 
mailing list (see samba.org for details on how).

Samba TNG is the result of Luke Leighton sitting in a 
room watching NT servers talk to each other. The guy is 
a machine. Also, other Samba team members(including 
Richard Sharpe ;-)) have done *so* much to help it 
along. It is usable now, but when Samba 3 is released, 
We won\'t need NT server anymore

> Or is there another samba-like 
product that is able
> to do this?

Don\'t think there\'s one for Win2K.
