I realise that this is hardly an ideal solution, but I 
personally don\'t have this problem.

You could try sticking it on another subnet, in it\'s own 
network (itself and a hub) and run tcpdump to try and 
find what types of packets are being sent. There are 
also a great deal of other free tools for network 
analysis (like iptraf - which will be good for this 
testing).

Try disabling absolutely everything in the Linux kernel 
that is not necessary (assuming you haven\'t already). 
There may be a bug or just something which reacts badly 
to something else.

I found a cool remote management tool (sorry, but I 
forgot the name) and it tried to ping every machine on 
the network. It was multi-threaded and could easily kill 
a network in no time. Is it possible that something like 
that has accidentally been installed?

I apologise if you have already tried these things, or 
if I was absolutely no help whatsoever.
