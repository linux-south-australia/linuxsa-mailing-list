Re: ZIP Drive

When I had an internal IDE ZIP 100 on my machine, I just mounted it like
you'd mount any IDE hard drive:

mount /dev/hdc1 /mnt/zip100 -t vfat

However, some disks seem to be formated so that the usable partition is on
partition 1 and others seem to be formatted with on partition 4. If you get
an error message with the above command, try

mount /dev/hdc4 /mnt/zip100 -t vfat

Of course, replace hdc with the actual letter of the drive, according to:

Primary Master: hda
Primary Slave: hbb
Secondard Master: hdc
Secondary Slave: hdd

And also make sure that /mnt/zip100 exists.
