Re: PAM Question

On Tue, May 23, 2000 at 12:32:40PM +0930, David Drury wrote:

 > Addin a PAM does the following
 > Utility --- Checks PAM --- PAM Checks Authentication Server ---
 > Login/Info out
 > 
 > To change the authentication method one changes the PAM, and then tells
 > Linux to reference a different PAM.
 > A PAM provides a standard set of calls such that I add in User Info
 > consultation service of my choice, it knows how to talk to any PAM as they
 > all have the same interface.
 > Am I more or less right ?

More or less.  Except that there is no authentication server, as such --
PAM is implemented by shared objects (.so files) which are mapped into
each process which wants to use PAM, and those modules don't need to
talk to a server or anything (e.g.: a PAM module which consults a 
flat-file would be perfectly workable, so would one which returns "Yes,
you can login" to every request).

Also, you need more than PAM to do a proper login/logout on a UNIX
box.  PAM handles authentication, not authorization -- so it checks
whether you have a valid password, but it doesn't do things like telling
you which UNIX uid your login shell is suppoed to use.
