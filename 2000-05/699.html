Re: Squid and accountability

Hi...

> I am looking for a way to track the web usage of the users on my network.
> Ideally I want something that asks for a username/password before allowing
> access. I don't really want to limit anything as yet, I just want to be
> able to find out which users are churning through the bandwidth.
> 
> Can this be done through Squid or do I need to add another program?

Squid comes with a couple of authentication modules that you can use to get
usernames/passwords from your users.  One uses NCSA style password files, and
another uses the system password file (or possibly PAM... can't remember).  You
can also write your own quite easily.

Authentication is configured in squid.conf along with all the other ACLs.
There's also some info in one of the FAQs about setting up username/password
authentication.

Once you've got the authentication happening, squid will log the username along
with all URLs accessed in the access.log.  You can then write a simple script
that looks at the access.log and works out how much data each user has used.
I'm not sure if someone has written such scripts already.  Checkout
www.squid-cache.org - there's a list of logfile scripts there.
