EQL Module

Hey All,

I've FINALLY got thet EQL serial line equalisation module to work on a dual
modem link. However, I don't think the speed is really up to scratch that
much. I only seem to be getting about the speed of a single modem, although
doing a tcpdump on both ppp links simultaniously shows data on both links.

One other thing about it is that tcpdump on the actual eql device only shows
the data in the outbound direction, is that normal?  To explain a little
better, if I run a tcpdump on the eql device, and then ping a machine at the
remote end of the link, I can see all the echo reqests being sent, but not
the echo replies, although I am getting ping times, so I know I am getting
the replies. If I run a tcpdump from the other end of the link, I can see
the echo replies, but not the echo requests... Just seems a bit odd to me.

Anyways, I was mainly writing to ask if anyone has set up the EQL driver
before, and if they have any hints about maximising the performance of the
link? I have the following setup:

Machine 1: Mandrake 7 (standard kernal), P133
Machine 2: RedHat 6.0 (standard kernal), 486DX4/100
MTU and MRU are set at 900
Port speed is set to 115200 on all ports
All the modems are the same brand and model. Discovery 56K voice modems.
Both UARTs on Machine 2 are 16550As, and both UARTs on Machine 1 are 16550s.

Any thing else relevant?
