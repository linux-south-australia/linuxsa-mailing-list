one solution I've seen is a utility that is called ftom /etc/inet.d that
simply copies any request to the port that it is on, to any other port on
any other computer.

One disadvantage of this is that if it sits on port 25 say, anyone can use
that machine to SPAM as sendmail will see the requesrts as comming from
the copied machine, not the source address. If you apply this with IP
chains rules at the same time, youy won'ty have this problem.
