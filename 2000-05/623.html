Points to think about:


1.	My experience is that NT as a proxy server is a binary state thing. It
either works very well, or its a pig. To use it effectively you need to
use ACL (access control lists) to do accounting and autherisation (who can access the web and who
cant) on a per user basis.
	In the setups that I have seen, this means
installing a MSproxy client thing on the windows boxes. This in fact
replicates your winsock.dll to allow you to do things like port
replication, which allows you to, say, exist on a private
192.168.*.* network with your proxy being the only real IP machine, and
your ICQ will still work. In theory.

2.	Cost, free vs. $100's

3.	Ease or configuration - equal in my book. A
GUI-wizard-follow-your-nose thing, or a text file

4.	Reporting and accounting. Squid will log everything to a text
file, allowing simple scripts to compile info. MS proxy can log everything
into an .mdb, allowing to use Access perhaps, or even ODBC connections to
the .mdb.

5.	Cost. See above

6.	In the recent proxy bake-off, squid came third, after dedicated
poxy hardware.

7.	cost :)
