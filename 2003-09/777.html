Re: [OT] IBM "boot linux faster"

On Fri, 2003-09-19 at 11:36, Matthew Geddes wrote:
> On Fri, 2003-09-19 at 11:02, David Lloyd wrote:

> > What a pile of hogwash. Linux *looks* like it takes a long time to boot.
> 
> It takes longer than it needs to.

Yep.  It's time to rip-and-replace init so that it understands
service and resource dependencies.

The article understates the amount of work involved, as
the init system and hotplug overlap somewhat.

This then helps boot time as a lot of services are
started on a "might be needed" basis.

Note that parallel start of services might slow performance
on a single-disk system, so the init file also needs to
give some hints about filesystem usage (eg, I hammer /tmp
when starting) so services that fight over disk heads can
be started serially.

On the plus side, stating dependencies rather than arbitrary
numbers helps application authors tremendously.
