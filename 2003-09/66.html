Re: SCSI Raid Cards

Brad Wilson wrote:

>RH 7.2/3 had a couple of minor problems at setup time.  You definitely
>need to use the supplied drivers with this version of RH, and it has a
>problem whereby the initrd file created in the /boot filesystem does
>*not* include the installed RAID driver module - so the system won't
>boot up.  It is a known bug and there are various convoluted workarounds
>available, but my simple workaround was to make a boot disk during Linux
>install and copy the initrd from the boot disk to /boot.  Worked a
>treat.
>  
>

There are plenty of known bugs in the Intel Linux support .. my 
favourite is the motherboard that requires an updated kernel for the 
onboard RAID driver to work (model SRCZCR), yet the motherboard 
management software for the same software requires the default kernel 
[1]. Incompatability between components from different vendors is one 
thing, but features made mutually exclusive by driver issues on the same 
motherboard are a bit much.

Contacting Intel support about this issue invariably results in someone 
telling you to visit the web site, or reading from the web site, and 
telling you to check back later.

Without the motherboard management software, I can't control the blue 
identity LED's on the chassis from remote!

John Edwards

[1] Checking the web site, it looks like there is newer software for 
this now, maybe it can work..
