procmail isn't that hard to cope with, especially if you can make other
applications do the hard work for you.

# ----- .procmailrc ----------
:0fw
| /usr/local/bin/clamfilter.pl

:0fw
| /usr/local/bin/spamc

# ----------------------------

clamfilter is a quick script I use to filter incoming email through
clamav (not bad for a freebie) and spamassassin.  All it does is change
the headers a little so all you would need to filter in sylpheed is 


header "X-Spam-Level" matchcase "*****" color 3
header "X-Virus-Found" matchcase "yes" mark_as_read
header "X-Virus-Found" matchcase "yes" move "#mh/mail/Spam"
header "X-Spam-Flag" matchcase "YES" mark_as_read
header "X-Spam-Flag" matchcase "YES" move "#mh/mail/Spam"

Simple enough for you?

Actually, just counting through my spam folder I've got over 200 of the
"critical update" viruses in there since monday.  Now if only I could
filter them at the server side I wouldn't even have to download them.
