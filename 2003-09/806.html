there is a patch for bind9. a paste from my post to 2600au:

----
> Anyone got any ideas on a hosts style trick (all oses) that says all
> 64.94.110.11 addresses = www.google.com?
                                                                                                                            
there is a patch for bind9 that allows you to set "delegation only" for
particular zones. setting it for "com" and "net" means that when you query a
*.gtld-servers.net name server you only want a referral and your patched
version of bind will ignore anything else. hence, if the domain does not
exist and the name server returns a A record for sitefinder.verisign.com,
bind will ignore the A record and return NXDOMAIN.
                                                                                                                            
this way of approaching the problem means that you circumvent the current
implementation no matter which IP address verisign is using.
                                                                                                                            
on the other hand, they could start returning referrals to their own name
servers which would then offer the A record for sitefinder.verisign.com
which would make the patch for bind9 useless.
----
