Re: good workarounds for the verisign typo-hijack exploit?

On Thu, Sep 18, 2003 at 02:45:32AM +0930, Alan Kennington wrote:
> 
> What do you need to do to the named.conf file?
> I've tried this sort of thing with BIND 9.2.2-P1:
> 
> zone "." in {
>     type hint;
>     type delegation-only;
>     file "root.hint";
>     };
> 

Hah! I worked it all out from a clue at slashdot. This is how to do it:

zone "com" in {
    type delegation-only;
    };
zone "net" in {
    type delegation-only;
    };

No more problems:

============================================
root@dog# host dsjhfkdslfsafds.com
Host dsjhfkdslfsafds.com not found: 3(NXDOMAIN)
============================================

As someone pointed out in slashdot though, if verisign does send a genuine
A-record, you're not going to get the benefit of it. Time will tell if this
is a sufficiently bad thing to worry about.



Now another question....
If you have your /etc/resolv.conf file set up to accept some other 
DNS server in addition to your own, then surely the resolver software
will grab the Verisign A-records from there.
Is this true?
And if so, is the only way to avoid it to only stick "good" DNS servers
in your /etc/resolv.conf file?
