Re: Linux, ADSL, TCP, Window Sizes, download times

TCP flow control works independently in both directions
Upload speed has nothing to do with download speed apart from the back 
channel. If this wasn't the case there wouldn't be asyncronous accounts or 
technologies as there would be no point.

Synchronous DSL is useful for uploading at high speeds ie, content providers, 
there really isn't any point in it for downloading at all.

I hope this has cleared it up a bit.
