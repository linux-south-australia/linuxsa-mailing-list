>That depends on how the programmer writes the program:
>
> 1. If the program modifies a GPL licenced program
>  - then it must be released under the GPL itself
>
> 2. If the program modifies a BSD licenced program
>  - then it must be released under the BSD licence
>
>If the program links *against* an LGPL library then you can do whatever you
>like with the program (include keep it proprietary).
>
>This is a legal question.


>You have to answer:
>
> 1. Who actually owns the copyright?
> 2. When writing the program did I/someone use an already licenced library
>    or whole program to implement it?
>
>DSL
>
1. I think  by default the programmer reserves the copyright unless
specified otherwise by a contract (eg with my employer they own the code I
write) (or even with my uni they own the code I write  - they're welcome to
it :P)

2. We have this situation whereby I am licensed to use a tool (OCX) I have
a developers license and there are no royalities attached (expressed in the
license) so I can write what I like and give the binary away I cannot give
the OCX away (even though they get a runtime version of it when it's
packaged) I do use a 3rd party for something which is public domain (not
GPL too embarrased to show the code). AFAIK there is no issue in this.
