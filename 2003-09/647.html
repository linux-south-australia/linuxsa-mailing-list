Re: Stress testing new servers

the "internal slashdot effect" is good for web servers - essentially:

while true; wget -r -l 5 http://test.my-server.com/ > /dev/null &

(note: i'm pretty sure that exact line won't work; the basic idea is to 
retrieve some page(s) of your website indefinitely as fast as you can 
and pipe them into /dev/null. there may have to be an interval and/or 
delete-after-downloading switch in there somewhere.)

of course, you'll need to be ready to `killall -9 wget` after a little 
while, it will hammer the machine ;-)

some places swear by "burning in" hardware (cpu & memory particularly); 
IMHO, it's snake oil (i'm happy to be corrected tho). if you want to do 
that, i reckon you could try some big compile like openoffice.org, 
mozilla, or a few kernels back to back.
