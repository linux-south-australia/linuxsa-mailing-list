> This is a complicated question. You can write closed source software for
> Linux as long as it doesn't include code or link to code that precludes
> that.
>
> For instance you can include BSD licensed code, but you can't include
> GPL licensed code. You can link in LPGL libraries but not GPL ones.
>
> If all the code is written from scratch and does not link in anyone
> elses libraries you can closed source, no problems.
>
> I would be very careful about creating closed source for Linux. You have
> to offer significant value for people to choose closed source over open
> source.
>
> Open source software really is about providing the source when
> distributing. If you are developing the program for inhouse use only,
> then you can keep the source closed and use open source components.
> However then you don't get the benefit of having a development community
> helping you.
>
> Be careful about developing the next big idea in software. One of the
> advantages of open source is that somebody has probably already done it
> or something close. You may be better off building on their work if you
> are happy to leave the source open.
>
> BTW, the scheduled bell ringing can be done by cron and ntp will get the
> exact time for you.
>
