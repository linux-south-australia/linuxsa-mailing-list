technically, even this doesn't stop you using sitefinder, because any queries 
that are sent through it still come back. the best you can do is to disregard 
it when the info is sent back, with iptables/dns hacks/whatever. but even 
_with_ these, you've been sent the IP, which means they've already started 
it.
there is only one way to make sure you don't use it. only one way. and that's 
not to use the Internet. unless you want to have very rigid controls, which 
ensure that nonexistent domain names cannot be queried. if you only ever used 
your ISPs page and software mirror, this might, just might be possible. but 
I'm sure no-one here would accept this as a good solution. basically, the 
only working solution is for verisign to cancel this, and for it to never 
happen again. preferably a legal sentence is placed that ensures no-one tries 
a stunt like this again.
