chris, your understanding of TCP flow control is a little simplistic.

TCP flow control is designed to increase the rate data is sent (using "slow
start", google for it) until you reach the limit of the weakest link in the
connection.

if you are downloading a file (no matter how big) the payload in the packets
you are receiving will be a great deal larger than in the packets you send
acknowledging the ones received. you will also only be sending one packet
for approximately every five packets received.

eg
server -> packet 1 -> you
server -> packet 2 -> you
server -> packet 3 -> you
server -> packet 4 -> you
server -> packet 5 -> you

you -> ack 1-5 -> server

server -> packet 6 -> you
server -> packet 7 -> you
server -> packet 8 -> you
server -> packet 9 -> you
server -> packet 10 -> you

you -> ack 6-10 -> server

as you are sending a lot less data then you are receiving, if your ADSL is
the weakest link in the connection then it is possible to achieve download
speeds up to the full 1500 without saturating your uplink.
