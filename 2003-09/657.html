> test) servers before they go into production.

I'd definitely be setting up the software (i.e., Operating System,
Applications, etc.) and giving them a bit of a thrashing, too. For
example, if you're building a web server, set up Apache and load it up
to production spec (with websites and applications as would be run in a
production environment) and hammer it over the network.

> My current thoughts are to Load up Knoppix ( as nothing on new HDs )
and
> then run  dd if=/dev/cdrom of=/dev/null  for a while.

Meh. That kind of thing's about as useful as stapling your tongue to the
desk, unless you can do it to all the computer's components. On a
network server, minor components (like the network card) tend to be
forgotten :-)
