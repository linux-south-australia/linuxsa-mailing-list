Yep, so do we.

> 
> Stability : Contrary to the belief of some people here Windows 
> Servers can be stable depending on what they are running. The 
> only time our Win NT F&P server gets rebooted is after 
> essential security patches. Linux also requires rebooting 
> after Kernels get updated due to security issues. The only 
> Windows servers I have ever had to administer that required 
> rebooting weekly was an Exchange 5.5 Server and an old NT 
> Comms server that had a dicky Eicon Routing card that used to lock
> up.

Generally agree, but point out that the requirements for security
patching and subsequent reboots is constant on Windows, and occasional
on Linux. Windows still requires rebooting for just about any patch,
certainly most of them.

We don't have to reboot our W2k boxen weekly, but we run into memory
problems if we leave it over a month. The boxes don't blue screen at
that point, we just get lowered performance and unexplained session
deaths... These boxes are running DB, Apps, print and WTS or citrix.

The Linux boxen run mail, file, print, fax, monitoring, RAS, etc. The
usually get rebooted at every major power failure exceeding the UPS
time, or on hardware failure. No security related reboots that I can
think of in the recent past (1-2 years)

> 
> Keyboard/Mouse/Monitor : 2 of our Windows servers and none of 
> our Linux servers have KB/Mouse/Monitor. Administration of our 
> Win Servers is done with VNC.

Ours have the kb etc via KVM. Rarely used.

> 
> Hardware : Any New Server is quite capable of running Win or 
> Linux. We still have some production Win NT servers running on 
> Pentium 75 Servers so your argument falls down there also.
> 
> Backup : not much good backing up just your smb.conf file if 
> it is a file and print server is it. Surely you need to back 
> up the data also ?

Bet on it :)

>
