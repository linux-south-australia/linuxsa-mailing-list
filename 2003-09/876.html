> I'm not sure what you mean by "full-stack-save".  Saving context
> wasn't a problem with any processors you mention, nor with the 68000.

The problems with 68000s couldn't restart an instruction that had stalled 
because of VM issues (eg. page wasn't in physical memory).

Sun-1s got around this by having 2 CPUs one an instruction behind.  If one 
stalled then it would get the page in memory and hand over to the other CPU 
to keep running the instruction.

IIRC there was no state saving issue.  Otherwise how the heck could you do 
interupts?
