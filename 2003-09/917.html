Re: getting e-Smith to find a modem

Best way to do this is run wvdial, it generally finds any type of coms, run 
a connection, if successful take down the initializing string, run e-smith 
setup, when you get to the config section where it says "put initializing 
string in here" do so and all works.
I have been there with ISDN and this sorted it all out.

>- confirm BIOS settings for com ports
>- confirm that nothing else is conflicting with these settings

>- confirm E-Smith is recognising and configuring the ports on boot (dmesg)
>- then check simple things like typo's (a common error is trying to use 
>/dev/ttys0 instead of /dev/ttyS0 and it just aint gonna work :)

>I know most of this is basic stuff, but it pays to start from the 
>beginning and work your way through. sometimes its the most simple of 
>problems, and it can save a lot of shin kicking later!
>
>Also, got this from the E-smith FAQ
>--
>How do I set the modem initialization string for my modem or ISDN terminal 
>adapter?
>
>It is better to store the modem init in the profile of the modem, rather 
>than specifying an initstring at dial time. You can store init strings in 
>the modem with AT&W by accessing the modem via minicom, or any other 
>serial communication program. For example:
>
>    AT&F   AT&C1&D2&Q0%C0   AT&W
>
>This resets to factory condition, sets the required settings (in this case 
>&C1&D2&Q0%C0), then stores the current settings as the reset setting of 
>the modem. (You might need/want AT&F0 and AT&W0 - refer to your modem 
>manual for modem specific AT commands).
>
>The standard reset of ATZ will then restore that condition - no need to 
>change/customize the SME Server setting.
>
>For more instructions on using minicom and other help on modem commands, 
>see http://www.linuxdoc.org/HOWTO/Modem-HOWTO.html
>
>If you really must use a dial time initstring, you can provide it when 
>configuring the dial up settings during the console configuration
>--
