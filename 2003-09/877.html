RE: Benefits of Samba vs Win2K Server

>You neglect to add "and how much of it they are running". For light use
>in the Microsoft-approved way a Windows server can indeed be stable.
>Remember that Microsoft recommend (and teach their MSCEs) that there be
>only one service per server. And that isn't just for extreme sites
>(where it is a good idea no matter what OS) but for perfectly ordinary
>ones with a few hundred users.

We recently had a recommendation on a Terminal Services environment
for 300 users (running full TS desktops via Citrix) and the recommendation 
was as follows:

2 x Terminal Servers (for redundancy)
1 x Domain Controller
1 x Application Server (for 3rd party apps)
1 x Core Application Server (for "standard" apps, ie: Office etc)
1 x File & Print Server
1 x Backup Server

Needless to say the dollar figures were kinda frightening (not least of 
which because of the Citrix licenses...)

PS No doubt the above system would do the job, and very well, but serious
need justification would have to happen before implementing something like
this...
