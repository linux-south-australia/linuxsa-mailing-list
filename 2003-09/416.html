RE: Benefits of Samba vs Win2K Server

>Assumptions:
>
> * money is not an issue
> * resources are not an issue

I take it this is not a real world question then? :)


>What would be the benefit, therefore, of running a Samba file server as
>opposed to a native Win2K file server?

It depends what client OS you are running.  Technically you are better off
running a native W2k (or above) file server if all your workstations are
running W2k or above as the management options through Active Directory,
Group Policies etc make life much easier.

Having said that, when money/resources are factored into the equation, the
benefits of AD & GPOs start to diminish.  Right tool for the right job as
usual...
