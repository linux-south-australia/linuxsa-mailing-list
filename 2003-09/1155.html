Re: VoIP phones need to find a supplier

You can use the Cisco phones with (for example) a SIP server.  Iagu 
networks makes one that runs on *BSD/Linux.  Cisco also have a SIP 
server that runs inside a router.  There are a number of free ones 
available if you search on freshmeat.net

SIP phones can actually "dial" each other without anything in the middle 
(so to speak) but you lose some functionality.  You do need some 
centralizd intelligence to know who the heck is where and how to call them.

CCM is a pretty good thing, but you really do need 50 or more phones.  A 
lot people who run CCM run it with Performance Solution's Vision product 
which is a good selling point.
