Sendmail Buffer Overflow
Severity: Medium

17 September 2003
Summary:

In posts to security mailing lists today, Michal Zalewski released an
early advisory describing a buffer overflow vulnerability in Sendmail
8.12.9 and earlier. An attacker can exploit this overflow to gain
control of your e-mail server. If you use Sendmail 8.12.9 or earlier on
any platform, we urge you to download, test, and install Sendmail
8.12.10 immediately.
Exposure:

Sendmail is the most widely-implemented SMTP server currently used to
send and receive e-mail on the Internet.

In his post to various security mailing lists, Michal Zalewski
described a new buffer overflow vulnerability that affects Sendmail
8.12.9 and earlier. Because of the preliminary nature of his release,
Zalewski omits many details about this new vulnerability. However,
because the vulnerability affects the Sendmail functions prescan() and
parseaddr(), we believe an attacker could trigger this flaw by using an
e-mail containing a specially malformed address.

Zalewski states that an attacker could exploit this flaw locally to
execute code on your Sendmail server, probably with root privileges,
gaining control of your mail server. He also believes that the flaw is
remotely exploitable, but hasn't confirmed this yet.
Solution Path:

The Sendmail Consortium has released Sendmail version 8.12.10 to fix
this (possibly critical) security vulnerability. Administrators using
Sendmail 8.12.9 and earlier should download, test, and install the
Sendmail 8.12.10 patch or upgrade immediately. If you are using a
Sendmail package that was included with your Linux distribution, see
your Linux vendor for their recommended upgrade path.
