AARNet has only up to vesion 3.6.1 for the linux "portable" version of OpenSSH.
It would be really neat if AARNet mirrored fixed software quicker in
emergencies. But I'm not complaining....

So it looks like the only place to get OpenSSH 3.7.1p1 is on one of those FTP
mirror sites. It's still a great mystery to me why security-conscious people
would still be using FTP. It's slow and vulnerable and clunky.  Are there any
redeeming features in FTP relative to HTTP for distribution of software?

---------------------------

"Hot on the heels of rev 1 of the buffer.adv advisory, here is revision 2,
which fixes more than revision 1 did."

But if you look at the advisories:
http://marc.theaimsgroup.com/?l=openbsd-misc&m=106375452423794&w=2
http://marc.theaimsgroup.com/?l=openbsd-misc&m=106375456923804&w=2

additional patching or vulnerability, as far as I can see. Can I therefore
assume that whichever 3.7.1p1 tarball I download is _the_ fixed version?
It sort of makes me wonder if there is an original 3.7.1 and a second,
better 3.7.1. (As usual, reading the slashdot feedback is about as relevant
as a visit to the local loony bin.) My guess is that there is only one
version 3.7.1, but the slashdot comment makes one wonder.
