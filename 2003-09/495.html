When I first heard SCO claiming that the BPF code sample was not supposed
to be evidence of real System V code in Linux, just a demonstration of the
techniques involved. I thought is was a good recovery from a bad mistake by
SCO but as I though about it some more I realised that it was not a good
recovery and nothing could cover up a crucial mistake by SCO.

If the SCO BPF code was merely a demonstration of the techniques involved
why on earth would they want to obfuscate the SCO BPF code with the Greek
letters?  There was no SCO intellectual property to protect and using the
obfuscated SCO BPF code weakened the "claimed" technique demonstration.

In other words the reason SCO obfuscated the BPF code is because they
thought it was their own code. Which adds more weight to Gregs question
about the lack of the BSD license in SCO's BPF source file.
