Re: copying files correctly

Alex,

> Can you please define what you mean by a "filesystem boundary".

If I have the following /etc/fstab:

/dev/hda5       / ext3 defaults 0 1
/dev/hda6       none    swap    sw      0       0
/dev/hda8       none    swap    sw      0       0
/dev/hda7       /mirrors reiserfs defaults 0 2
proc            /proc   proc    defaults        0       0
/dev/cdrom      /cdrom  iso9660 defaults,noauto 0 0


If I go, say, from / to /mirrors I've crossed to a different filesystem or a
filesystem boundary. If I go from /proc back down to / again the same thing.
