That raises a general question of how best to edit web pages in foreign
languages - assuming that one wishes to avoid "authoring tools".
Supposing you want to write HTML/PHP etc. in the manner of a programmer
rather than point-and-click, there must be 3 ways to do it:

1.	Unicode editor method.
	Edit HTML/PHP files directly in some weird encoding such as
	UTF-8, iso-8859-1 (the easy one), UTF-16, UTF-32, UTF-7, 
	UCS-2, UCS-4 etc. (I gather that vim can do UTF-8.),

2.	Preprocessor method.
	Edit an ascii file which is fed into a convertor from some
	semi-readable notation (e.g. a' to represent ·), or

3.	PHP library conversion method.
	Use PHP plus some library routines to do something like:

	$string = "a'a`a^"; print utf8convert($string);
