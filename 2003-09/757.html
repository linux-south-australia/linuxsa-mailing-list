Is that any reason not to make it boot faster?

Incidentally, the machines I'm dealing with at the moment boot in less
than 10 seconds on a ~500MHz Via C3. I could optimise it more, but for
what we're doing, 10 seconds appears to be enough.

Most Linux installations really only need to do the following after init
starts:

 * File system checks
 * Mount file systems
 * Bring up network
 * Start individual services (usually only a need for a few)
 * Start gettys and graphical display manager

If you were to remove all of the other cruft and start X before the
other services (except XFS), you'd have a system that appeared to take
no time to boot at all.

Bloat (or lack thereof) is the reason why Slackware is the One True
Distribution :-)
