Re: Killing a script after a certain length of time

Sam Silvester wrote:

>I've got a program that I want to run automatically, and every now and
>again it will freeze up for a length of time which is a pain...is there
>any way of running a program (perhaps with some other wrapper program)
>so that I can specify that it will be killed if it takes longer than a
>couple of minutes or so? The program itself is written in c, and I'm
>calling it from a php script using the exec() function.
>  
>

If you're using the source to the C program, you could always make some 
modifications so that it looks after itself.

Have a look at the alarm(2) man page, along with signal(2). ie: set the 
alarm to go off in 120 seconds, and then simply exit at the signal handler.
