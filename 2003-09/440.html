Re: Benefits of Samba vs Win2K Server

My gut feeling is that most users (ie the other 99% of the people on the
network) can't cope with ACLs well. They still think Office is part of
the operating system and explaining how to right click and change
security on a file is inviting disaster.

I think that assigning a group for each department and an accompanying
samba share is very maintainable.

ACLs are really annoying for people accustomed to scripting support of
their file systems.


Glen Turner wrote:
> On Fri, 2003-09-12 at 14:41, Kim Hawtin wrote:
> 
> 
>>What are these "extended" attributes of which you speak?
> 
> 
> The extensions allow metadata in addition to the standard
> UNIX metadata to be stored on the filesystem.  For example,
> you could store ACL names.
> 
> 
>>What do you gain from them over the user and groups model in Unix?
> 
> 
> Access control lists allow arbitrary rules.  The real world
> doesn't fit into the UNIX group model.  For example, "all
> the people in the Black Helicopter project with Secret or
> better clearances and Fred the cleaner" is difficult to
> implement using UNIX groups without exploding the number
> of UNIX groups (leading to maintainability issues).
> 
> 
>>Or are other things funky date and time attrs?
> 
> 
> They are generic.  So they might store
>   - an ACL.
>   - a reference to where in the tape library the contents
>     of the file can be found.
>   - a thumbprint image for files which are images.
>   - a precomputed keyword vector for faster seaching
>     of documents.
> 
> Whatever. Personally, I'd like a URL for files created
> with a web browser "save to disk" option.
>
