> 
> MySQL is remarkably robust when it comes to transplanting data files.
> 
> Most of the time you can get away with just copying them into the
> DATA_DIR directory of an existing mysql installation.
> 
> Install mysql, do something like:
> 'mysqladmin -u root -pPassword variables | grep datadir'
> 
> that will show you where it expects the files to be.
> 
> Then create a directory with the name that you want the database to
> be, drop the data files into it, check permissions (ie: chown -R
> mysql:mysql./mynewdatabasedir/), and restart mysql.
