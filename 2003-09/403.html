Re: Benefits of Samba vs Win2K Server

Well, you've got a *n?x/BSD system and all its functionality.

including a plain text file for configuration - no clickyclicky
interface which drives you crazy when you want to create more than 20 shares

configuration backup is easier - just backup the smb.conf and when your
server has crashed for what reason whatsoever your configuration can
easily be restored.

and you don't need a keyboard, mouse and monitor for the server after
you've installed it. just put it in your toilet ;)
