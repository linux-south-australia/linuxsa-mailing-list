Re: Installfest 2002

Curiously, I always do it the other way round, but with exactly the same
reason as you give.

Windows doesn't recognise, nor create, Linux partitions. It doesn't even
create the MS ones particularly well either.

So, I install linux from a linux boot CD, and while partitioning the drive,
make sure I include a primary vfat partition in there.
Format this with mkfs.vfat and remember to specify fat32 (not the default).
Copy the 98lite installation directory from my server onto this vfat
partition, and also copy loadlin.exe and the initrd and vmlinuz into a
c:\LINUX\ directory. (Also copy over whatever drivers n stuff you need).

Now reboot, (i have a DOS boot CD), and install 98lite (or your choice of
windows) from the harddrive.

After that's done, exit to dos and use loadlin.exe to load linux, whereupon
you tell lilo or grub to re-install themselves into the bootsector.

Now you're done, and you didn't have to use MS fdisk at all :D
