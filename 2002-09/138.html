Re: kernels and rpms

Quoting "De Fluiter, Desiree" <dmdeflui@subcorp.com.au>:
> I have heard it said many times on this list, that one should not try to
> install a kernel from an rpm file.
> 
> What is the best way to take an rpm file and install it onto an existing
> linux system?
> 
> Note:  Considering I only have a boot diskette, how can I put a kernel rpm
> file onto it and rpm to download the rpm to the linux box???

There's no problem INSTALLING a kernel via "rpm -ivh ..." but you shouldn't 
UPGRADE or FRESHEN a kernel with "rpm -Fvh ..." or "rpm -Uvh".

Why?  The latter 2 remove the old kernel.  So what if there's a bug in the 
kernel that's incompatible with your hardware setup and you can't boot?  Or 
what if there's a bug in RPM and you're using LILO (not grub) and lilo fails to 
write the boot sector correctly?  etc etc etc. You are then stuffed[1].

The recommended procedure is to install a new kernel, verify you can boot to 
it, and then if you need to, remove the old one ("rpm -e").  Despite never 
having a problem upgrading a kernel this way, I always install, never upgrade 
or freshen.

[1] Rescue mode on RH disk 1 is *wonderful* though...
