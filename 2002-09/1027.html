Icons in KDE:

Right click, create link to application.

You can copy the icons, just rename them when pasting, you will be 
asked. Change execute tab syntax and icon.

Execute tab is most important here are the contents  of some:

    Crossover Office Wine applications:

            The following opens Microsoft Excel under cxoffice 1.2:

            "/home/james/cxoffice/bin/wine" "C:/Program Files/Microsoft 
Office/Office/EXCEL.EXE" --

            (I was less experienced when I set this thing up. I probably 
should have used "$HOME/cxoffice/bin/wine" instead of 
"/home/james/cxoffice/bin/wine" so I could copy the icon to any other 
desk without having to edit the contents of the execute line.


       Linux native:

                The following opens up Adobe Acrobat Reader 5.05:

                        acroread

Shell scripts:
   
            These are easy and fun.

               Here's one:
                    The first line says, I want to use the shell. (well 
kind of..)
                    The second line is anything I would have typed in 
the shell.

                        #!/bin/bash
                        /usr/local/bin/gtaxestimator

        To make it executible, just save the text file and call it 
whatever-you-want-to-call-this-script.sh. Just right click on it, go to 
properties, and click on executible. You can copy and rename the file so 
you don't have to make it executible.

The execute line of the icon reads:

sh $HOME/guesstimate.sh

Because I saved the file in my home directory and named it guestimate.sh


Network share links in KDE:


            In my computer folder with computer like png attached:

                    Right click,  link to URL. Right click, properties. 
Rename so it doesn't say "F:\.desktop" for network share. Click on URL 
tab, click on                         folder to browse for mount point.
