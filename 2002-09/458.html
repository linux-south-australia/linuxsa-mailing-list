I can't do anything for your memory [tm], but mine says that Xserver 
runs the mouse for 'gui' logins.. are you saying its a bug otherwise?

explain to me why the gui isnt the problem. Xserver would normally be 
creating the windowing so login managers xdm|kdm|gdm can run.
did you install gnome or kde? or neither?

now, mouse settings for Xserver would be found in here;
/etc/X11/XF86Config
look for the 'Core Pointer's InputDevice' section.
mine looks like this;


Section "InputDevice"
# Identifier and driver
     Identifier  "Mouse1"
     Driver      "mouse"
     Option "Protocol"    "IMPS/2"      # IM is for intellimouse
     Option "Device"      "/dev/psaux"   # you would want /dev/ttyS0
     Option "ZAxisMapping"       "4 5"   # these two lines are
     Option "Emulate3Buttons" "no"     #   for scrollwheel
EndSection

restart xserver and the mouse should be going..
hit control-alt-backspace to kill Xserver if need be.

the binary 'Xconfigurator' will configure the Xserver as well, from a 
console, also try 'mouseconfig' for, surprisingly, mouse configuration..
(I havent checked if this is on RH7.3 but is is on 7.1)

otherwise, search the newsgroups at groups
<http://groups.google.com/groups?hl=en&lr=&ie=UTF-8&oe=UTF-8&q=mouse+not+working+redhat+7.3>

redhat bugs and errata can be found here;
<http://rhn.redhat.com/errata/rh73-errata.html>;
