>> No. I remember seeing it somewhere, but can't quite remember where.

>> been for another dialer. But, it was called something like keep-alive,
>> which send and echo down the line every so often to keep the
>> connection alive.
>
> This is what you're talking about:
>
> (man pppd)
>
> lcp-echo-interval n
>     If  this  option  is  given,  pppd will send an LCP
>     echo-request frame to the  peer  every  n  seconds.
>     Normally  the  peer  should  respond  to  the echo-
>     request by sending an echo-reply.  This option  can
>     be  used with the lcp-echo-failure option to detect
>     that the peer is no longer connected.

Which the default is to do every 20 seconds anyway, so I'm not sure that
is actually what you are trying to achieve. From the sound of it you don't
want to be kicked off for 20 minutes of inactivity or whatever the timeout
is?

One way is to put into a crontab a job to ping something on the net every
5 or 10 minutes. Something like this in /etc/crontab will do the job:

-------------------------------------------------

# Ping yahoo.com to keep connection alive
*/5 * * * * root ping -c 5 www.yahoo.com > /dev/null &2>1

-------------------------------------------------

That will send 5 ping packets to Yahoo every 5 minutes and send any output
to /dev/null. Note that you might want to implement something in
/etc/ppp/ip-up.local so that this only happens while you are connected.
