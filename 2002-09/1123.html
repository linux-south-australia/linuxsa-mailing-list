On Wed, 2002-09-25 at 23:25, andreb1 wrote:
> Hi. I've had some problems compiling rar, and while searching the web for some
> resulsts, i found some messages in a list with your answers
> Well, i found a solution for the rar problem, since i am not on that list, if you
> could email the solution to the list would be great.
> The solution is:
> 
> 1) edit arcdos.c, go to line 35, comment out:
> //struct timeval {      /* man page said <sys/types.h>, but it */
> //      long tv_sec;    /* really seems to be in <sys/time.h>, */
> //      long tv_usec;   /* but why bother... */
> //};
> 
> Now when you try make, you will see there is an other problem..
> 2) edit tmclock.c
> and add
> #include <time.h>
> after the last #endif(line22)
> 
> try make.
> You will receive warnings, but it will compile..
