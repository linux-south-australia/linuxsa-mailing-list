Apache and DNS

Hi All,
    I am getting a strange error on a website I am setting
up, and could use a bit of assistance.

    I have my home site working fine and can look at it in
the normal manner, ie http://www.mysite.com.au . I have set
up a IP alias on this network and this also appears to work
correctly, but by IP address only since it is not yet
delegated, ie http://203.myIPrange.193 (home machine) and
http://203.myIPrange.250 (the alias). These can be looked up
on my browser and are visible from the  net (as they are
meant to be).

    Now comes the weird part. I have set up my new site in a
subdirectory from the main one apache uses, this is fairly
standard, but I get strange errors when I try to access it.
If I try to access the site using
http://www.mysite.com.au/subdir/index.html I get a DNS
lookup error, but if I just look up http://www.mysite.com.au
it works without a DNS error. I don't understand why there
is a DNS problem if it can find the home site. It gets
worse, if I look it up under IP the site works perfectly, ie
http://203.myIPrange.250/subdir/index.html .

    O.K., this obviously looks like a DNS problem, but I
cannot see anywhere in the config file that would cause it.
DNS is working for mysite, so I don't understand why it
hangs up when it tries to look at the file in the
subdirectory, which also works if you look for it using IP
address. Below is a slightly modified version of my
httpd.conf file. The ServerName in the alias is deliberately
commented out as the name is not yet delegated and shouldn't
affect this problem.

HostnameLookups On

NameVirtualHost 203.myIPrange.193:80
NameVirtualHost 203.myIPrange.250:80

<VirtualHost 203.myIPrange.193>
  ServerName www.mysite.com.au
  DocumentRoot /var/www/html/
</VirtualHost>

<VirtualHost 203.myIPrange.250>
  DocumentRoot /var/www/html/subdir/
#  ServerName www.newsite.com.au
</VirtualHost>

I am using Apache 1.3.24 and RH 7.0. Any help would be much
appreciated. TIA.
    Cheers!
        Andrew.
