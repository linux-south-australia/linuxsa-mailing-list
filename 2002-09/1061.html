I've always used Cyrus. It can be a pain to compile, but if you use RPMs 
for it then that is a moot point. Cyrus has it's own database format for 
storing and indexing mail, and it seems to be both scalable and robust. 
It certainly doesn't have any problems with my 7,000+ message mailboxes 
(of which I have a few). It is happily dealing with the 7G of mail we 
have here at work, which includes a shared mailbox containing 10700 
messages. This is on a (approx) 700Mhz PIII, which also does a lot of 
other serving work.

It also doesn't require every mail user to have an actual account on the 
mail server, which can be useful if you are authenticating against LDAP 
or a database. Of course if you use the password database for 
authentication your mail users *will* need to have an account.

It supports POP3 nicely, also does POP3S and IMAPS (POP3 and IMAP over 
SSL) for good security and supports sieve for server-side message 
filtering - this is something that I couldn't live without these days 
(no more setting and maintaining mail filters on *every* mail client I use).

Yes, I like Cyrus. Use it, it is good. :)
