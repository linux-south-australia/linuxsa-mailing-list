Re: window / display manager performance

On Tue, Sep 24, 2002 at 10:06:36AM +1000, Mcleod, Ian wrote:

 > Why are these Window managers so much faster?  They seem quite capable
 > although I could not run 3D accelerated apps or video applications on them
 > (I thought a box with IceWM or something similar would make a great
 > networked video / music player).
 > What is windowmaker?  Even that seems faster - and there are still plenty of
 > apps even if one can not run Gnome / KDE apps any more.

The window manager is just the bit of software which "paints" borders
around windows and lets you move, resize, iconify, etc your windows.
Basically it gives you "look and feel".

Applications aren't tied to window managers.  There's no reason why
a GNOME or KDE application won't continue to run in the same way when
you change WM's (some apps these days want various bits of the -desktop-
to be running (e.g., you might need the GNOME Panel running), but the
desktop isn't the same as the window manager either).

 > How can I run KDE on one terminal (default) and start another window manager
 > in another console? 

startx /usr/X11R6/bin/windowmaker -- :1  (for example) will work.
