Re: printer friendly web pages ??

On Tue, 10 Sep 2002, Wilkinson,Alex wrote:

> Howdy People,
> 
> Quick question:
> 
> How do you take a a *non* printer friendly web page and turn it into a document that
> can be actually read when printed ?

Use either links or lynx, both of which are text mode browsers that have a
print function, and both of which can export a web page to formatted plain
text or formatted plain html. By plain html I mean without graphics,
flash, applets or other stuff but retaining frames and other html layout
information. (links is a much smarter implementation of the ideas that
lynx pioneered, worth trying if you thought that text mode browsers can't 
navigate the web these days!)

More generally you can usually get results from web technologies that are
friendly towards blind users (which includes all sorts of visual
handicaps, very few people are completely blind - therefore the technical
solutions don't only concentrate on output suitable for handing to machine
readers.) Something as simple as choosing the 'text only' links on
websites can work with your ordinary browser. Disability-related
guidelines usually mandate plain text versions of websites, so there's
often one buried away even on sites that have gone completely overboard
with visual junk.
