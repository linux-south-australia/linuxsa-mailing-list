Re: mandrake 9.0/gcc3.2 vs. gcc 2.9x

Ben,

> What's the True Status (TM) of this stuff?

Because GCC 3.X is a _MAJOR_ version number change I wouldn't trust any
binaries precompiled against GCC < 3.0 regardless of whether they're
purported to work. When a _MAJOR_ version number changes it's allowed to
break binary compatibility and possibly even some source code
compatibility.

I wouldn't trust a GCC < 3.0 binary on a GCC 3.X system in production
unless:

a) I had to (eg it was closed source and the company who build it are
being obstropolous)
b) I had tested it thoroughly
