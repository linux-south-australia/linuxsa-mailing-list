On Thu, 26 Sep 2002, Matthew Geddes wrote:

> You haven't had to do that after a configuration change for a while now. 

About six years, in fact.

> think the smbd and nmbd processes re-read the configuration file every so
> often anyway

Yep. No need for SIGHUPs unless you can't wait for 60 seconds (the
default.) You'll find misunderstandings about this all over the web
because it isn't normal Unix daemon behaviour. However this way it not
only works better but gives an equivalent to the Microsoft administrator's
experiences. However only some changes get noticed by currently-connected
clients. Samba tries hard to avoid any unexpected effects for currently
conected users (including if the config file has a bug in it.)
