Adam,

> Doing a 'touch' or creating the file manually doesn't seem to work, as
> it doesn't get written to, nor does it get created automatically if it
> goes missing.  If you reboot the machine it will begin working again,
> but that is not something I want to do if I don't have to.
> 
> I tried sending the init process a HUP, but that didn't do anything.

Do as Jarrod says, but if you're running a weird distribution or FreeBSD
then it may be possible that syslog is not running as root. In whch case
you'd probably want to take a look at who owns some of the other files
that are still working and make sure that the owner/group and
permissions are similar.

I've never actually seen a syslog not running as root but I suppose it
would be possible :-)
