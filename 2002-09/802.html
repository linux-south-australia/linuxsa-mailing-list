RE: Gentoo Linux

Unfortunately there is no distinction between security updates and general
updates that I am aware of. 

Downloads are the size of the specific packages source code.

Currently we have around 600Mb of current source code which is used when
building a machine [no Graphics or X applications]. For us the source
distribution saves us bandwidth, we don't have to download new packages for
Linux SPARC machines, Also it means we get the security updates for SPARC's
much quicker than usual.

The base system is under 50Mb, then you can just compile the specific
additional packages you want. I.E. Iptables and squid (require dependency
will automagicly be installed). However, given it really likes to compile
its own applications and not install binary versions, lots of temp space and
memory are required to patch a program...

Personally I think it is excellent.

We run it on all new Linux servers we produce, It's nice and clean. 

advantages are:
- No annoying features such as "Mandrake Aroura".
- Excellent "BSD ports" style package management.
- Source code based. [Portable between architectures]
- Optimized code for architectures [Optional]
- Each program when compiled, will configure for the library versions
installed.
	(no more you require version library-2.7.31 for this package to be
installed)
- Make --update world command [update all programs or system to latest
version]

disadvantages:
- Source code based, Long installation times combined with large downloads
- Optimization limits portability [e.g. if compiled for Athlon XP/MP It will
not work on a P4]

Regards, Dale Butterworth
