Re: How to install an additional harddrive in Linux

On Thu, 19 Sep 2002, David Lloyd wrote:

> 
> James,
> 
> > There is a file called /etc/fstab that stores information about your
> > hard disks. If an entry is not in that file for a hard disk, I think
> > that you can't mount it.
> 
> That is plain wrong.
> 
> What if I have:
> 
> % mount -t ext2 /dev/hda2 /mnt/somewhere
> 
> ??
> 
> Provided that:
> 
> a) the special device /dev/hda2 exists
> b) there is a partition there
> c) that mount knows how to mount ext2 filesystems
> d) /mnt/somewhere exists
> 
> fstab is simply the file that tells init which devices to mount, how,
> whether they should be backed up by dump and so forth.

Hmmm, while we are being ultra-correct, I should point out that this is 
not strictly true either. init does no such thing, it simply launches a 
script/program and then sits around reaping children and making sure the 
things in inittab are executed when they need to be.

It is the responsibility of one of the scripts to do an fsck -a and then 
to mount -a.
