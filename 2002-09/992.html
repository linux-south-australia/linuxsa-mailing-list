> 	The window manager is just the bit of software which "paints"
> borders
> 	around windows and lets you move, resize, iconify, etc your windows.
> 	Basically it gives you "look and feel".
> 
> So why are the others so much faster?  For the first time I actually felt I
> was running a fast computer..  It's just so damn quick - like you can see
> those GHz ticking over.  Other Window managers (MS Windows included) never
> seem to behave as though one is actually running a faster machine.

The window manager *just* does the "decorations" and window manipulation 
functions as Mark says.  Some are faster that others because they do less, or 
are because they're written in C instead of Lisp.  

*Desktop* environments run seperately, but often interact with WMs, making the 
whole experience appear to be a single entity, consistent but often slower 
(ie. "eye candy").  You can run KDE's window manager (kwm still?) without 
running KDE if you like.  Or you can run metacity or sawfish (from GNOME) 
without GNOME - they're just WMs.  Or you can run "e" or "BlackBox" or whatever 
under GNOME or KDE or none.
