Re: Samba usernames

On Sat, 21 Sep 2002, Andrew Galdes wrote:

> Hello all,
> 
> The "%U" variable for samba is displaying as "nobody".
> 
> This is the situation:
> 1) Samba is configured for "share" security. 
> 2) The clients are both windows2k and samba clients.
> 3) The clients seem to connect as "nobody" rather then their windows
> networking usernames.
> 
> Is the problem that i am using "share" rather then "user" security?

In share level, there is no concept of actually loggin in; Samba uses the 
"guest user" as the user accessing the file system. That is typically 
nobody. 

Note also, %U gets you the user that they were mapped to, while %u gets 
you the user they requested. You might find that %u will work.
