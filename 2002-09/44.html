-- essence--
    Most but not all gnome apps (applications) will work in the kde 
desktop and vica versa
In the kde desktop  [Alt][F2] will bring up a  Run command prompt, into 
which you can enter the command for the app you want to run.  ie the 
gnome / redhat network tool you have already configured.

-- topping --
Future references to the required tool can then be accessed by using the 
same Run command prompt [Alt][F2] and either entering the command again, 
or scrolling down the commands previously entered or typing the first 
couple of letters of the command and selecting the correct one from the 
options then presented. (Try it, the previous statement  is easier to 
understand with the Run command promt in front of you  )

-- filling --
Failing this, configure kppp the kde dialup tool to dialup whilst using 
kde. Unlike m$ configuring different tools to do the same job does not 
overide or alter the setting of each tool.
    However, be clear as to wether you are configuring different tools 
to do the same job
or configuring the same setting through different tools.
eg. tools such as kuser and userconf both access the same systemic 
settings for users.
kppp and wvdial are different tools and can be configured differently 
for the same job.
