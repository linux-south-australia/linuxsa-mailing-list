I remember installing newer 2.2.X series kernels on RedHat 6.2 and I
don't remember any major problems. If you are trying to upgrade from a
2.2.X series to a 2.4.X series then you might want to visit:

* http://www.kernel.org/

And look for some documentation teaching you the pitfalls of changing
the major number of a kernel. If I recall correctly -- and don't quote
me -- you'll need to upgrade glibc and things compiled against 2.2.X
might not work correctly or at all.

That document might be held at:

* http://www.linuxdoc.org/

And be part of the Linux Documentation Project.

**
My mention of a reboot diskette, was in case you need to install the
new
kernel with the old kernel offline.
**

As Michael Davies has already said, use the install but not freshen or
upgrade option. That way you can hopefully have both kernels installed
and only delete the old kernel when you are absolutely certain you need
it.
