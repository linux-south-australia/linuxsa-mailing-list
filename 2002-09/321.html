The file size could be an issue.

Your kernel might have a 4Gb file size limit, I think you have to ensure
support for large files is incorporated and it may not be.

You could remove the offending file form the FILELIST and see what
happens; given that it doesn't perform the rest of the backup that
shouldn't be a major drama.

If you get REALLY desperate you could mount the RedHat file system via
NFS and use FreeBSD's tar to back it up. It's massively convoluted but
it would work (I think) ;-P
