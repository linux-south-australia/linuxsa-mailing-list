Re: RH7.3 logrotate bug??

Schenk, Jarrad wrote:
  it is using a good percentage of my system ram, it makes everything
> else run really slow as the system needs to swap things in and out of disk. I 
> can kill the process off (which I don't really like doing but is the only way 
> to get a usable system) and everything comes good but soon after, another 
> process called updatedb starts. this uses alot of IO time on the disk and 
> also slows things down. is it safe stop updatedb from running and if so, how?
> 

What's happening here is that cron is running each of the
jobs in /etc/cron.daily (there's also a cron.hourly and
cron.weekly).

The job named 'slocate' runs 'updatedb'.  You can safely
not run 'updatedb' as long as you don't use the 'locate'
command (which finds a file by looking in the database
of filenames built by 'updatedb').  Easiest way not
to use 'locate' is to remove the package, which contains
only that command.

   rpm -e slocate
   rm -r /var/lib/slocate/

The 'find' command provides an alternative to 'locate'.
