Re: Memory use / application optimisation

Ian,

> How does Squid optimise bandwidth use? 

Visi:

* http://www.squid-cache.org/

You may have the Squid FAQs in /usr/share/doc/squid... or
/usr/doc/squid... as well.

> Does it detect what has changed and
> only update the cache where required?

It's highly configurable. In short, yes.

> So whenever someone goes to a website
> - Squid checks to see if the local cached copy can be substituted somehow?

Not "substituted". Your browser talks to Squid. And Squid returns
something to the browser. Your browser thinks it's talking directly to
the original page. Squid attempts to be as transparent as possible.

> Is Squid hard to set up for a newbie?  Are there GUI interfaces for it?

I think there is. Start from Squid's web-site.

It's not really that difficult to setup.

Some caveats:

* best to have a firewall running BEFORE you set up squid
* best to learn how to use squid ACLs before you turn it on

...as I discovered before people do attempt to connect to your cache for
various reasons.

Neither of the above two points are really that difficult.
