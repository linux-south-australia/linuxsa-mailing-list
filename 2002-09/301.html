Re: using dd

Michael Fewings wrote:

> I'm thinking that I can use dd to do this but am not
> sure if I need to partition the new drive first or whether dd transfers
> the partition imformation as well as the other data.
> Present configuration of partitions is:
> 
> /dev/hda2              2016044   1261028    652604  66% /
> /dev/hda7              1517920    219124   1221688  16% /home
> /dev/hda1              2096160   2022400     73760  97% /mnt/win95
> /dev/hda3               497699    288585    183410  62% /usr/local

If the new drive is the same size as the old, then you can dd
/dev/hda to /dev/new_hdx, and that will copy across the partitioning
information.  Otherwise, just create the partitions on your new
disk to be the same size as the old (or slightly larger, in case
the geometries are slightly different), and dd /dev/hda[237] on
to the new drive's partitions.

> If I wanted to use different partition sizes that those listed above
> would I need to partition the new drive and use tar?

Yes, or the dd exrcise above and one of those resizing utilities for
e2fs, if that's what you're using (this is assuming an increase in
size onto your new disk).
