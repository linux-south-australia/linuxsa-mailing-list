Troy,

Did it say "no screens" found BEFORE you ran Xconfigurator or after
you'd answered all the questions? If it said it after then you've made a
mistake in one of the questions and X has quit.

Try looking for its log file in:

 /var/log/XFree86.0.log

If they are not there you can do this:

 startx >& /root/XFree86.0.log

...down the bottom or closer to the bottom than the top I reckon you'll
see "No valid [something]" or other error messages that may help.

Try this:

# X -configure

...and then type the command it asks you to. If it works and you are
satisfied with the results work out which XF86Config file it's using. Do
this by searching for a line similar to this in the above log files:

(==) Using config file: "/etc/X11/XF86Config-4"

Check to see if that is a link to something else:

% ls -l /etc/X11/XF86Config-4

...if it is the first letter you will see will be an "l" and the last
field will have an ascii arrow. You want to fiddle with the real file
not the link. Here's an example of what I mean:

% ls -l plugin.new
lrwxrwxr-x    1 lloy0076 lloy0076 \
  12 Sep 23 15:58 plugin.new -> plugin.trace

(I wrapped the line with a \ because of the word wrap issues). Obviously
you'd have a link called "XF86Config" or something similar.

Once you've done this copy the file:

# cp /etc/X11/XF86Config-4 /etc/X11/XF86Config-4.BACKUP

...and rewrite it with the file you performed in the command above. It
should be /root/XFree86-4 but it may be something else:

# cp /root/XFree86-4 /etc/X11/XF86Config-4
{ answer yes to overwrite it }

...and see what happens now when you `startx'.
