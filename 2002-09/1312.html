network broadcasts

Hi all,

We've got a network situation that looks like this:

  _____________
/             \          eth0 +----------+ eth1
| 192.168.100 |-------------->| gateway0 |---------+
\_____________/ 192.168.100.1 +----------+ 5.4.3.2 |
  (office)                                          |
                                                    |
                                                    V
                                                 _______
(IPs have been changed to protect              /       \
  the guilty)                                   | i'net |
                                                \_______/
                                                    |
  __(hosting)__                                     |
/             \          eth0 +----------+ eth1    |
| 203.1.2.X   |<--------------| gateway1 |<--------+
\_____________/     203.1.2.1 +----------+ 1.2.3.129



That is, our office is in the 192.168.100 segment, with a gateway to the
internet. We have several servers in a hosting facility (the 203.1.2.X
segment).

Unfortunately, one of the servers in the hosting facility runs a 
(horrible) legacy app that we want to connect to from the office. The 
client for this app uses broadcasts to locate the server (you can only 
identify the server with a name, not an IP).

I want to know if it's possible to transmit broadcasts across from the 
192.168.100 segment across the link(s) to the 203.1.2.X segment. If so, 
how would we achieve this? And a more important question, do we *want* 
to achieve this? (i.e. will it flood our links, present a ridiculous 
security risk, other?)

And to keep this linux-related, both of the gateway boxes are running 
linux (one RH7.0, one RH7.2).

Thanks in advance,
