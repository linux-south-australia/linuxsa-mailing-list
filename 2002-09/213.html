In addition to the suggestions others have provided, here are some more.

Adam Smith wrote:
> Filesystem    Size   Used  Avail Capacity  Mounted on
> /dev/ad0s1a    97M    41M    48M    46%    /
> /dev/ad0s1f   9.1G   6.0G   2.4G    72%    /usr
> /dev/ad0s1e    48M    48M  -3.4M   108%    /var
> procfs        4.0K   4.0K     0B   100%    /proc

ad0?  Smells like BSD, which it doesn't appear that you've stated.

> df reports /var to be full.  It's not.  According to "du," var only
> contains 5.5Mb of data.

As David N said, du will only report files that it can get at (i.e.
actually have file names), as it's possible to "anonymous" files.  df
will report disk space usage at the file system level.  The above
situation will clear itself on system reboot or when the process
that has that file handle open closes it (see the current running
"/var/log/messages" thread).

> I ran a " fsck -f " and got the follwing results for ad0s1e:

Did you run this while the machine was in single user mode with the
disk unmounted?  Expect errors if you ran this in multiuser mode,
and *never* run fsck in any of its fixup modes in multiuser (well,
OK, you can, but be prepared to live with the consequences).
