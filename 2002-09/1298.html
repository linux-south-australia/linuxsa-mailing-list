Re: Samba 2.2.5 + OpenLDAP 2.x - Caveats?

On Mon, Sep 30, 2002 at 10:06:18AM +0930, Freeman, Peter (ERHS) wrote:
> Now I'm making the presumption that Samba + LDAP is the right path

It sounds like it could be a Right Path.

> Can anyone with experience in this type of setup comment on any 
> issues they struck while migrating from smbpasswd based systems
> to central LDAP authentication.

These days I don't have many problems.

> What version of OpenLDAP would you recommend?  2.0.x or 2.1.x? 
> Pros/cons for either version?  I notice the schema file packaged with
> Samba has support for 2.1.x.

I haven't noticed any problems with either myself. Typically, I'd install the
later version if I didn't have an existing install. I usually try to keep
everything on the same OpenLDAP major version (eg, 1.x, 2.0.x, 2.1.x).

> Were there any issues in migrating existing users, ie: file permissions,
> profiles, etc?

It should be a straight migration from your old config to the new, with the
exception of the additions to the config file to do with ldap and the config
file options that can be moved into the LDAP directory (login script, home
directory, profile path, etc)

> What is the speed like over a WAN environment for a local Samba box
> to authenticate against a remote LDAP server, over say a 64k link?

I'd personally have a Samba / LDAP box at each end of the link where
possible. LDAP can take care of the replication for you, which is handy. You
may want to have a look at the Samba-BDC HOWTO in the samba source tree for
ideas along this line.
