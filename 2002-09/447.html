> Hi,

> I have compiled apache-1.3.26 with
> "./configure  --prefix=/usr/local/apache --enable-module=so", and is working
> fine. In /usr/local/apache directory, there are bin, cgi-bin, conf, htdocs,
> icons, include, libexec, logs, man, and proxy directories, but there is no
> html, or something like this, directory for html files.
> 
> Where does apache keep its html files by default?
> 
> If I would like to change that directory, say I create html directory in
> /usr/local/apache and use this, what changes do I have to make (in
> httpd.conf, I guess!)?

Damar,
it would have helped to know what version of Linux or BSD you are talking about... they have been putting the apache configs in various places... It looks like you are running a BSD box.. :)

If you change the html directory, you need to change the DocumentRoot in the httpd.conf

By Default (at least under FreeBSD) it reads :
DocumentRoot "/usr/local/www/data"

Change that and you should be done... unless you hit a permission problem..

Make sure you restart apache... :)
