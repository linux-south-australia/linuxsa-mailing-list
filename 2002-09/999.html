I agree with Glen.

I think that what makes Linux so attractive to us is what restricts its
wide acceptance by Joe Bloggs. The lack of a consistant UI. Don't give me
CUI over GUI arguments our users want GUI so GUI they shall have.

The fact we have so many flavours is good, people dont necessarily see that
though and view RH/Suse et al as 'LINUX' - The step we need to take in
trying to have an Enterprise desktop solution is to choose
KDE/GNOME/<whatever> and go from there, the closer the match to windows the
more people like it I guess, I was being nagged by my son to put Linux back
on his system (he prefers the games) but he also wants his win32 based
games, so I got a copy of WinLinux wich is a RH variant with umsdos based
FS and went through the setup on my machine before giving him the CD to
install it (he's 9). The KDE (3.0) looks so nice and with an appropriate
theme looks far better than ANY Win32 UI I have seen. 

So it isnt hard to offer a better looking/performing OS but if only MS
would offer their office products on this platform (they do for OSX) so it
surely cant be that hard and just think how much happier the world would be :P

>Hi Evan,
>
>As far as management is concerned Linux is inferior.  Their
>computing world view is: e-mail client, word processing,
>spreadsheets and presentations.  Microsoft have best-in-class
>products in all of these areas.  Linux options are comparable
>or worse (eg: compare Excel and Scalc).
>
>So I totally understand a manager that might view MS favourably,
>because from where they sit MS is the better platform.
>
>It's up to IT staff that want to run Linux to make reasonable
>arguments if they want to run Linux across an enterprsise.  This
>is easiest where Linux is used for infrastructure applications.
>
>For desktop applications the "technical superiority" argument
>isn't true, so appeals need to be made to value for money.
>Linux may not even compete on value for money -- the inconsistency
>of the user interface adds significantly to training and support
>costs.  One day's additional training for a Linux-based application
>is enough to pay for a Windows OS+Office license.
>
>For enterprises looking to save money there is another desktop
>alternative: Windows OS + StarOffice.  Unless Linux makes some
>significant user interface improvements I wouldn't expect to see
>Linux + StarOffice deployments beyond what used to be the OS/2
>market (banks, etc).
>
>Regards,
>Glen (who uses desktop Linux and OpenOffice extensively)
