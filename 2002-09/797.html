FWIW you can get "auto updating" happening on a redhat box by running
up2date -u from a daily cron job. If your only on the free update
service it only works about 1 out of 3 days though, so its not a
replacement for manual maintenance.

> Personally I think it is excellent.

> We run it on all new Linux servers we produce, It's nice and clean.

> advantages are:
> - No annoying features such as "Mandrake Aroura".
> - Excellent "BSD ports" style package management.
> - Source code based. [Portable between architectures]
> - Optimized code for architectures [Optional]
> - Each program when compiled, will configure for the library versions
> installed.
> 	(no more you require version library-2.7.31 for this package to be
> installed)
> - Make --update world command [update all programs or system to latest
> version]
> 
> disadvantages:
> - Source code based, Long installation times combined with large downloads
> - Optimization limits portability [e.g. if compiled for Athlon XP/MP It will
> not work on a P4]
