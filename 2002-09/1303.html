> Basically this BDC type setup would be replicated for 10 other sites. 
> Is this feasible enough or is there a more efficient way of doing it?

This is feasible, but under 2.2.x, there is no way to set the SID on each 
of the BDCs ATM, since we moved to having that info in the secrets.tdb. It 
used to be in the file MACHINE.SID which you could move around.

There has been some discussion of a 'smbpasswd -L -X S-1-5-21-x-y-z' to 
allow you to do what you want. However, no code has been done so far.

You should see the discussion on some of the issues in the BDC-Howto 
document as well.
