Re: mandrake 8.2 & KDE3.0.3 issues

Hi Ben,
    This is a known issue with the over install of KDE 3.0 onto 8.2

I'm not sure of the answer, but from memory it was something to do with 
setting the paths so that the new installation /opt/kde3 is called not 
kde ie the kde2.2 apps dont work on the kde3 desktop
The freeze time is the app trying to start but not getting the right 
xserver and libraries.

eg. icon for kate needs to call /opt/kde3/bin/kate not etc/bin/kate
