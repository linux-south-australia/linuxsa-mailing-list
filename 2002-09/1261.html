Re: help with grep

"dcd" <azmon@eml.cc> wrote:

> I'm trying to find the file where the PATH was set to /usr//bin
> to both correct it and set my Java path

Usually that's partially set in your profile.  If you're running
bash, check ~/.bash_profile, ~/.profile and ~/.bashrc (the
~ gets expanded to your home directory).  Following the
bash motif, PATH may also be set globally in the file
/etc/profile or directory /etc/profile.d/ .

> currently $PATH in  the  konsole returns
> 
> bash: /usr//bin:/bin:/usr/bin::/usr/local/bin:/usr/X11R6/bin:/usr/games: 
> No such file or directory

You're not just typing "$PATH" and hitting return here, are you?
PATH is actually a variable; if you are doing the above, you
should be typing in "echo $PATH", otherwise you may strike
a variable one day that isn't as friendly, or is pointing to a
bonafide executable.
