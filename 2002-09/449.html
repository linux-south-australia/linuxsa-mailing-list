Re: Where does apache-1.3.26 keep its html files by default?

On Thu, Sep 12, 2002 at 01:53:43PM +0800, D Thapa wrote:
> Hi,
> 
> I have compiled apache-1.3.26 with
> "./configure  --prefix=/usr/local/apache --enable-module=so", and is working
> fine. In /usr/local/apache directory, there are bin, cgi-bin, conf, htdocs,
> icons, include, libexec, logs, man, and proxy directories, but there is no
> html, or something like this, directory for html files.
> 
> Where does apache keep its html files by default?

If you've installed it in /usr/local/apache/, the default location would be /usr/local/apache/htdocs/. You can check this by looking at the httpd.conf file (by default, /usr/local/apache/conf/httpd.conf) at the DocumentRoot directive.

If you'd like to change the Document Root, just change the DocumentRoot directive in the httpd.conf file and restart apache.

Hope it helps,
