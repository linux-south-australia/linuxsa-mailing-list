Aha,

> No. I remember seeing it somewhere, but can't quite remember where. Maybe
> it wasn't one of the options for pppd or wvdial, it may have been for
> another dialer. But, it was called something like keep-alive, which send
> and echo down the line every so often to keep the connection alive.

This is what you're talking about:

(man pppd)

lcp-echo-interval n
     If  this  option  is  given,  pppd will send an LCP
     echo-request frame to the  peer  every  n  seconds.
     Normally  the  peer  should  respond  to  the echo-
     request by sending an echo-reply.  This option  can
     be  used with the lcp-echo-failure option to detect
     that the peer is no longer connected.
