Re: Samba backup suggestions

On Wed, Sep 11, 2002 at 04:39:46PM +0930, Dan Shearer wrote:
> On Wed, 11 Sep 2002, Benjamin Pulkkinen wrote:
> 
> > I would recommend staying away from dump on Linux if you can. Linus
> > Torvalds has publicly stated that he thinks dump is a stupid program, it
> > isn't reliable on Linux and will be even less so in the future.
> 
> dump has to know about fs internals. That's a pretty dated way of thinking
> about things. dump on Linux is more-or-less just dump for ext2/3. "stupid"
> sounds like a good description.

Tar and cpio don't back up extended file system attributes (such as the immutable and appendonly bits[1]) or other file system dependent features like ACLs (for those that support it). Sure, you can write a script to dump this sort of thing to a text file in your backup path, but that's not for everyone. Dump can make life easier in those situations.

Hope it helps,
Matt

[1] - See lsattr(1) and chattr(1) for more info and a complete list of supported attributes
