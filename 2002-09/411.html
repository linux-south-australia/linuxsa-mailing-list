Re: Samba backup suggestions

Ben,

> I would recommend staying away from dump on Linux if you can. Linus
> Torvalds has publicly stated that he thinks dump is a stupid program, it
> isn't reliable on Linux and will be even less so in the future.

Dump is specific to the file system being dumped. Linus has made these
comments about dump.ext2 (which is, conceptually, what most people talk
about). Other file systems, such as XFS, provide a version of dump. That
said, I'd be inclined to use GNU Tar because it's possibly more portable
across file systems.
