> 'net rpc getdomainsid' or some such, a Samba server can pretty much 
> completely take over the role of an NT server. Even profiles work!

What would be funny is hacking up a quick shell script that makes use of:

  * rpcclient / reg_* / rpc_samsuck*
  * smbclient
  * smbcacls

The shell script could read information about printer and disk shares from
the registry of an NT box, as well as grabbing the SAM. By using smbclient
and smbcacls, it could copy all of the data from the disk shares to itself
and share them out. The shell script could do a complete migration of a Windows
NT file and print server to some flavour of UNIX.

Hmmmm. I know what I'll be doing when I next have some spare time... :-)
