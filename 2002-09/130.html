apache virtual hosts in 1.3.x

Im running apache 1.3.26 and am having some problems with virtual
hosting.

According to the apache site, I need something like:

NameVirtualHost *

That should turn on virtual hosting on all the machines IPs,
then several of similar too...

<VirtualHost *>
    DocumentRoot "/usr/local/apache/html"
    Options Indexes FollowSymLinks MultiViews ExecCGI
    ServerName www.steadycom.com.au
    ServerAlias steadycom.com.au foghorn.steadycom.com.au 
</VirtualHost>

This works OK.

But, when I start trying to run 2 different virtual hosts using SSL, and
have the first one doing a reverse proxy to an internal server, the
second one seems to get ignored. apachectl shows:

{foghorn}/etc/httpd/conf # /usr/local/apache/bin/apachectl configtest
[Thu Sep  5 12:01:21 2002] [warn] module php4_module is already loaded,
skipping
Syntax OK

Im wondering if this is a bug, and its just my combination of features
that triggers it? According to the apache site I can use proxypass in a
virtual host, and Im using name based virtual hosts correctly.

Is anyone awear of any gotchas I may have missed with such a config?
