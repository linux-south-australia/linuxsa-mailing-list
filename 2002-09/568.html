Just a note that my friend has not been able to get iHUG to work at all 
on his mandrake 8.2 distribution (whereas others have worked fine), 
anyone else running linux and connecting to ihug? They seem to have a 
weird setup (they're supposedly a pretty poor ISP anyway).

This may be of interest to u, i connect to ozonline, my ppp config file 
looks as follows:

connect '/usr/sbin/chat -v -f /etc/ppp/ozonline'
-detach crtscts modem defaultroute user XXX usepeerdns
/dev/ttyS1 57600

where XXX is the username for the account.. this username has entries in 
the chap and pap secrets files... the chat script just dials the number 
and waits for ppp to start, nothing special.  And I am running 
 2.2.14-5.0 on red hat linux 6.2 (hey it's a 486 firewall box)..

my chat script is

ABORT "NO CARRIER"
ABORT "NO DIALTONE"
ABORT "ERROR"
ABORT "NO ANSWER"
ABORT "BUSY"
ABORT "Username/Password Incorrect"
"" "at"
OK "at&f"
OK "atdt81223431"
"~-^Uppp-~"

my chap and pap secrets files look the same:

# client        server  secret                  IP addresses

XXX       *            PASSWORD           *

obviously XXX and PASSWORD are my username and password (not the real 
ones! :) )... i start ppp by calling "pppd call isp-ozonline" from a 
script...

if u cant get it to work with ihug, just change ISP's... when I joined 
with ozonline, some dood even talked to me about setting linux up to 
connect to their service, which was pretty cool :) (he had no idea what 
he was talking about and was no help, but nice try tho! )

Damien Uern.
