RE: Samba backup suggestions

David Lloyd wrote:

> Dump is specific to the file system being dumped.

dump also backs up extended attributes and sparse files correctly,
which neither cpio nor tar address, so there still exists a very
real world use for dump.  And while people may that sparse files
are no longer used, it looks like tcpflow still uses them.

> That said, I'd be inclined to use GNU Tar because it's possibly
> more portable across file systems.

That's the point.  You shouldn't use dump to transfer files
across heterogeneous file systems/computers.  It depends on
what you're trying to achieve with your backups.
