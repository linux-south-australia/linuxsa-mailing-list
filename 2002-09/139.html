$author = "Anthony Symons" ;
> 
> But, when I start trying to run 2 different virtual hosts using SSL, and
> have the first one doing a reverse proxy to an internal server, the
> second one seems to get ignored. apachectl shows:

you can't do name virtual hosting for SSL sites.

name virtual hosting requires apache to read the HOST part of the HTTP
request to associate a particular connection with a particular vhost.

it can't associate a connection with the right vhost (and hence the right
SSL key) until it reads the HOST part, but it can't read the HOST part of
the HTTPS request until it decrypts it with the right key. catch 22.
