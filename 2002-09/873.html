Re: mandrake 8.2 & KDE3.0.3 issues

Hi Ben,
    From memory

a.     The DCOP issue is exactly that, delete the incorrect link and 
create a new one to the correct place

b. Links problem - if your not going to use KDE2.2 apps
    Then remove the duplicate entries and/or create links to them from 
bin to opt/kde3/bin

        - If you are still likely to boot to kde2
    Create an entry in /home/user/.kde3/share/config/something_rc (Can't 
remember the specific file) 
    add an entry changing the PATH to include /opt/kde3/bin as the first 
entry
eg export PATH=/opt/kde3/bin:$PATH
    This will allow most apps to start from the correct location, some 
apps with direct links may still need to have their own new shortcuts 
created.

c. Console link - remove from kicker(task bar and creatre a link on the 
desktop to /opt/kde3/bin/konsole)
    - apparently the startup on this causes a conflict otherwise
