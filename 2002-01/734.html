Quoting Dale Long <dalel@loftuscomp.com.au>:

> On Thu, 31 Jan 2002 alcaron@senet.com.au wrote:
> > No. Rsync works by downloading a few files out of many, not by patching
> > existing files.
> 
> # man rsync
> 
> ...
> 
> DESCRIPTION
>        rsync  is a program that behaves in much the same way that
>        rcp does, but has many more options  and  uses  the  rsync
>        remote-update  protocol  to greatly speedup file transfers
>        when the destination file already exists.
                                   ^^^^^^^^^^^^^^

>        The rsync remote-update protocol allows rsync to  transfer
>        just  the differences between two sets of files across the
                                           ^^^^ 
>        network link, using an efficient checksum-search algorithm

The checksum is used to determine which files are different, not what the
differences between the individual files _are_. Note the "already exists"
and "two sets of files" (not "two files") in the above.

either.
