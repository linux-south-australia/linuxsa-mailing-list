On Tue, 2002-01-22 at 15:13, Wilkinson,Alex wrote:
> Can anyone see what I am typing wrong here ?
> 
> >sudo find /usr/local/ -name pkgtools.conf -exec cd .. {} \;
> find: pkgtools.conf: No such file or directory
Hmm... after changing the directory and filename so it'll run on my
system (and removing the 'sudo'), mine says:
find: cd: No such file or directory

Hmm...

Why is the ".." there?
Surely you'd want
find /usr/local -name pkgtools.conf -exec sh -c 'cd `dirname {}`' \;
to change to the directory the file is in?
Of course, 'cd' isn't an actual program, so can't run. 'cd' is a command
that is built-in to the shell (probably bash). Running 'sh -c ...'
starts a new shell and changes to that directory, but doesn't change the
current directory of the shell you are running in.

If all you want to do is run a program in that directory, try:

find /usr/local -name pkgtools.cfg -exec sh -c 'cd `dirname {}`;ls' \;

This runs 'ls' in that directory.

> I thought find, finds the file and then exec executes cd on the file found ?
cd is a builtin command, unfortunately.

> ie -exec cd { /usr/local/etc/ } \;
