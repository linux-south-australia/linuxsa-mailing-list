Sounds like duplex/speed, as was mentioned previously.

Anyhow, you'll need to use a process of elimination to diagnose the problem:

1) Chuck a Linux boot disk (Tom's RBT comes to mind) into the WinXP
   machine and make sure that it's not WinXP causing the problem.

2) Use a crossover between the two machines instead of the switch.

3) Disable firewalls on both machines. If ping now works, enable the firewall
   on one of the machines to find the defective ruleset.

4) Find that Linux boot disk again and run a tcpdump on both machines, while
   the ping is being attempted. You're looking for:
   - Whether the ping leave machine A
   - Whether it gets to machine B
   - Whether machine B responds to machine A
   - Whether the reply gets from B to A
If it's a network card problem you'll be able to find out which one is at
fault.

You're not running any routing protocols or similar over the network, are you?
I've observed very weird behaviour when two incorrectly configured
OSPF-enabled hosts attempt to vy for the default route.
