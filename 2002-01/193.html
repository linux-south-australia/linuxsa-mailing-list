Re: nntpd atomic clock sync via intermittent dial up connection

use ntpdate


useage:

"ntpdate server"

ie.
"ntpdate ntp.nml.csiro.au"

and you will get something like this

Dec 31 05:01:02 wac-gw ntpdate[27319]: adjust time server 130.155.98.1
offset 0.018061 sec
you can set it as a cron job with the -s option and it will output to the
logs not on screen.
if it cant find the server it just does this:
Dec 31 20:01:04 wac-gw ntpdate[28495]: no server suitable for
synchronization found
and stops, so if its a non permanent connection it shouldnt grizzle too much
when not connected,
or you could make a little script which checks to see if a connection is
active before running it, then run that out of a cron job every few minutes
or something.
