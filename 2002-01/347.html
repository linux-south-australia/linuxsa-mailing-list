Re: IMac CDFS file format

On Thu, 2002-01-17 at 14:41, Michael Wardle wrote:

> > I'm unable the read a CD which has been burned on an iMac. RH7.2 does 
> > not recognise the CDFS file system. The given file system is not 
> > recorded as format in the mount man pages either.
> > 
> > Has anyone an idea how to extract the data from the CD?
>
> If it's not a standard ISO9660 CD-ROM file system, I'd try mounting the 
> CD as an HFS or UDF file system.  As I understand it, HFS is commonly 
> found on Apple Macintosh systems, and UDF is a format used for packet 
> writing, commonly used on CD-RWs and DVDs (I could be wrong).

Mac's also like HFS+. I believe the support for HFS+ is somewhat dodgy,
though I'm not sure whether it'd be used on optical media such as a CD.

> Is "CDFS" a file system type?  Could it be synonymous with ISO9660? 

I don't think so. I seem to recall creating a few coasters when I first
used the Apple CD Burning program included with the recent iMac's.

> What told you the CD was CDFS?

The Apple CD Burning software, probably. IIRC, there are 3 different
file systems that you can use. Only one of them is "PC"-compatible.

> Do file systems have some sort of header or code indicating their type? 
>   Can the file system type be easily deduced?

Digging this far down is probably a waste of time. Simply take the CD to
your nearest Mac, copy the data to the hard disk and burn the CD again
using a format that is supported by Linux. Unfortunately, it's been
several months since I used a Mac, so I can't remember the file system
that you have to choose from the combo-box during CD creation.

Perhaps this could be clarified by one of our (or is that our only :-)
Macintosh zealot(s).
