Re: PC Authority: Linux as a Desktop is dead

Glen Turner wrote:

> PS: I never could understand the "Linux must have a journalling
>      file system to succeed in the enterprise" articles.  I always
>      though that Oracle and DB2 did disk access to raw partitions.

Perhaps they still do, but the (a?) preferred method of setting up
Oracle these days is to write to a normal "flat" file (or multiple
flat files spread across several partitions), usually with the aid
of logical volumes (eg, striping).  I guess the trade off in
incurring the overhead of the filesystem is made up in that you can
use normal backup methods, ship the redo logs to other boxes, and
so forth.
