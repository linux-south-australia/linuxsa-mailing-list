Re: stranger problem: POP3 server.

D Thapa wrote:

> POP3 server responding to one network, but not to the other, is there any
> thing I can check on this (any setting?)?
> 
> Any pointers would be highly appreciated.

Try a telnet to port 110, and see if it's refusing the connection, or
accepting it then dropping it immediately. Running inetd with it's -d
flag might assist with finding this.

As of December last year, there's a rootkit circulating that contains a
modified tcpd that weirdly denies access to some networks and allows
others, regardless of the settings in hosts.allow.

If it's installed, try:

stat `which tcpd` and check the file change time to see if it's more
recent than the OS.
