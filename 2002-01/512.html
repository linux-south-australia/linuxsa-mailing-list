> Portage does not support code stored locally (you could fake a CVS archive
> I suppose), so grab the first part but be prepared for the 20Mb online

Portage actually does support the use of locally stored source code 
tarballs for each of its ports.  Before downloading source from the web, 
it will look in /usr/portage/distfiles for the file first.

If you speak of actually getting the portage ports directories on your 
system, after the initial (I suppose it would be a somewhat large 
download) `emerge rsync` command to get it on your system, updating the 
ports tree would be relatively lightweight on the downloads, as rsync is 
used to update it.
