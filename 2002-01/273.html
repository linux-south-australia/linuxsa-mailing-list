Re: good maildir webmail

On Wed, 2002-01-16 at 12:12, Anthony Symons wrote:

> Does anyone know of a good webmail package that supports maildir? I had
> a look at imp, but that had to many dependancys and Could'nt make it
> work properly. 

IMP 3.0 is not that hard at all. Now that the dependency on PHPLib is
gone, much of the hard slog involved in getting it going is gone.

It's particularly easy under FreeBSD:

# cd /usr/ports/mail/imp-devel
# make
# make install

I believe there is some Apache twiddling that needs to be done, as well
as SQL configuration for holding preferences data.

Maildir support is generally completely irrelevant to a web-based mail
interface. There's generally a layer between mail storage and mail
display, namely protocols such as POP and IMAP.

Install a Maildir-compatible IMAP server on your mail server and use
IMP. It's not that hard. It's quite educational, too. IMP 3.0 is /nice/.
