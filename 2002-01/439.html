Re: upgrading Fbsd port ?

In the immortal words of Justin Hawkins <justin@internode.com.au>...
> Generally speaking, you can use the -f parameter to pkg_delete to
> force it to remove the package anyway, then reinstall the new version.
> However this can cause problems, since these libraries often change
> between versions, and the dependent software needs to be rebuilt.

Good idea, but it can be messy depending on the differences between
versions.

> However there is a happier solution. Install portupgrade
> (/usr/ports/sysutils/portupgrade) and let it do the work for you. It
> can upgrade the port as above (just less manually) and if you ask it
> to. It can also rebuild dependent ports recursively, which should
> solve most shared library problems.

Agreed, however in this particular case,they are 2 separate ports.  My
advice is to leave the existing one 1.2 installed, and install the 1.3
version.  I had to do this myself to stop things from breaking, and now
I've got fun & games keeping 2 separate versions of linux_base
installed.  For some strange reason staroffice requires linux_base-7 and
linux-opera requires linux_base-62 and there seems to be no happy
medium, so I've ditched Opera and gone with mozilla, however, that's my
problem and when I sort it out I'll be very happy.

Cheers
