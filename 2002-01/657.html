Re: Webservers,

Hi Daniel,

I think http://www.linuxvirtualserver.org is what you're looking for to do
high-availalability web servers with an all-linux solution.

For your backup mail servers, you may find that just having a lower-priority
MX record does the job as well as a purpose-built load-balancing/failover
system.

If you're using FreeBSD, the newer versions of ipfilter have a round-robin
per-ip forwarding feature built in, netfilter in 2.4 linux kernels *might*
have a similar option.

If you're betting that the great majority of your site's viewers are using
Internet Explorer, you can get away with simply using round-robin DNS. When
IE2 or greater gets a list of IP addresses from the DNS, it tries them all
until it finds one that 'works' for some value of 'works'.
This is almost documented in MS knowledge base article Q154442. You can
implement it in Bind 8 by simply creating multiple A records with the same
hostname and different IP address.

Has anyone noticed if Netscape or Mozilla have similar behaviour?
