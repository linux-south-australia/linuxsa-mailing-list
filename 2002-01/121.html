SSH Version Compatibilty & Public Keys

Hi All,
	I'm having a few problems getting public key authentication working
between different versions of ssh.

I'm going from a host with ssh (client):
OpenSSH_2.9p2, SSH protocols 1.5/2.0, OpenSSL 0x0090602f

to a host with (server):
ssh: SSH Secure Shell 3.0.1 (non-commercial version) on
alphaev6-dec-osf4.0f

I've generated a dsa & rsa key on the client.
I've put the public keys onto the server under ~/.ssh2/mykey.rsa &
~/.ssh2/mykey.dsa
I've created ~/.ssh2/authorized and put in it the lines: 
	key mykey.rsa 
	key mykey.dsa

I've created a local config on the server under ~/.ssh2/ssh2_config
enabling public key as an authentication method.

Yet I'm still asked for a password.

Can anyone see what I'm missing? Is it just you can't use public key
authentication between these two versions of ssh? 

Authentication does work between the client and another box running
openssh.
