AFAIK, the most common way of doing this is with Netsaint, a Nokia
mobile, Gnokii and a data cable (and a phone account)...

Basically, Netsaint monitors stuff and can create events (emails, SMS's,
etc) to indicate that some state has changed. If you want it to SMS you,
either directly hook up a phone to the PC doing the monitoring, or use
an email-SMS gateway. Note that the email-SMS gateway won't tell you
that email has gone down, as it can't email the SMS to the gateway... :)
There are a number of similar limitations -- if the machine doing the
monitoring goes down, it can't generate a message to email or SMS you
with.

Note that you don't need one phone per machine -- one machine can
monitor many others, though bear in mind that your monitoring then
depends on that machine, and all the network connectivity between it and
the one(s) it's monitoring.
