Dissappearing IP packets

Hi...

I'm having network trouble, and I'm not entirely sure how to go about
diagnosing it -- I've never had anything quite like this before.

Situation is that I have a headless old linux server inconveniently
located, and a newish winXP desktop (yeah yeah). They are connected by
10/100 ethernet -- each has a RTL8039 card, going through a D-Link
switch. I've never had trouble before with this connection, but now,
when I `ping -c 1000 <winxp>` from the linux box, I lost 40-50% of

random, with a few runs of dropped packets, but mostly random. Here's
the ping -c 1000 summary:

--- 192.168.0.2 ping statistics ---
1000 packets transmitted, 512 packets received, 48% packet loss
