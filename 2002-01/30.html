RE: Autofs VS am-utils for NFS mounting

Personally, I've always disliked the autofs /net/<host>/fs system. It
seems to me that it introduces one more place where you can add an
unneeded dependency (unless it's used strictly for certain purposes,
like backup, or something, and users don't start symlinking files in
there and so on)...

I prefer to have few mountpoints, explicitly mounted in auto.master &
friends (over NIS or as files) so that I can control it. This way, all
mounted FS's appear as local fs's, and there is no dependency on name
that the user sees...

I don't know if this is just gut feeling based on a bad experience or if
it's actually a sensible opinion though... (maybe there's some really
good legitimate reasons why /net/host/ is a good idea...) ...

Can anyone enlighten me?
