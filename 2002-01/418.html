Re: ftp speed observations

On Fri, 2002-01-18 at 23:42, steve wrote:

> I was fiddling with a freebsd  4.4 setup and temporarily set up an FTP 
> server to hold some files. What was quite amazing was the speed, it was 
> blindingly fast compared to my RH7.2 server. As a sort of test I 
>  reinstalled a server version of  RH7.2 on the same machine as the 
> freebsd system and tried both with the same  set of files. The BSD 
> server was about 5 times faster for the same test.

As Toby said, it may be to do with the TCP options. Have a look through
the sysctl(8) variables regarding tcp.

# sysctl -a | grep net.inet.tcp

Compare them to the default options employed by RedHat.

> Redhat is running wu-ftp and BSD was running the native one that comes 
> with it.

That's a fair enough test, IMO. That is if you're attempting to compare 
the speed attainable by each OS in its default configuration.

> Anyone know the reasons for such a big performance diff. I could take my 
> own guess re how the ftp proggys work but it is a significant diff, 
> that's for sure.

Well, wu-ftpd is available from ports. Install it, change the line in
inetd.conf to reflect the different ftp daemon and run your test again.

# cd /usr/ports/ftp/wu-ftpd
# make install
# vi /etc/inetd.conf
# killall -HUP inetd

Let us know about the results. I've not yet installed RedHat 7.2, and I
don't intend to, so I've got nothing to use to test on myself.
