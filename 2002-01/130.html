Re: IPtables / Firewall question

A firewall is only really usefull if you have services that you want
some people to be able to use and not others.

If a port isnt listening then it isnt listening. If a connection attempt
comes in to that port that is not open then is going to get rejected and
thus you arnt hacked (presuming there is no bugs in your tcp stack). If
it is listening but isnt required then it should be turned off. 

You only really need firewall rules on boxes doing routing between
boundrys between areas where your level of trust changes (usually the
edge of your network). Or where you need to run a service that you dont
want certain people to be able to access.

I'd be more worried about making sure you stay up to date with patches
than I would about overly complex firewall rules on internal hosts.
