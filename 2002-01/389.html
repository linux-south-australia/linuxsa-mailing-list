Re: Linux system -> image -> linux system

On Sat, Jan 19, 2002 at 11:28:03PM +1030, Gwyn Morfey wrote:
> So it seems I need to backup, partition from scratch, and then install linux and 'doze in that order. The linux boot still works perfectly and I really don't want to have to reinstall it. Is there any way (ghost? tar?) I can turn the linux partitions into image files, then bring them back once I've repartitioned? I gather that just tarring up /, then restoring it won't leave me with a working system.
> 

[you need to turn on line-wrapping]

creating an image of a partition sounds like a job for dd.
Check out the man page but (very) roughly something like:
  dd if=/dev/hda3 of=linux_partition.img
you might need a count as well (and set a larger blocksize
than the default to speed it up).

Then fiddle your partition back into shape etc (making sure you
don't change the size of hda3) then splat it back on again:
  dd if=linux_partition.img of=/dev/hda3

(note: all care no responsibility blah blah, do a google search
and check out for yourself if other people have done this and
stuffed it up etc etc)
