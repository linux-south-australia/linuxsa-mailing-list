Re: upgrading Fbsd port ?

On Mon, Jan 21, 2002 at 03:54:21PM +1030, Wilkinson,Alex wrote:
> I really would prefer to delete gtk-1.2.10_2 first and then install the new 
> port or somehow upgrade gtk-1.2.10_2 to gtk+-1.3.12 instead of 
> installing both.
> 
> Is this possible ?
>

Alex,

I am afraid it is not possible - it is not due to a limitation of the 
FreeBSD ports system, but is intead due to the difference in the GTK+ 
versions of which you speak.   They are binary incompatible, so programs 
that link to 1.2.x GTK+ versions (ie. the one you have installed) will not 
run with the 1.3.x (to be 2.0) GTK+ versions.

So, the only way to keep your current programs and port dependencies happy 
is to keep your current GTK+ libraries installed.

The later versions of the GTK+ 1.2.x series have been made so that they 
can be installed parallel to the 1.3.x versions, without any conflicts.  
If you want to install the 1.3.x version of GTK+, I would recommend you 
upgrade your current GTK+ installation to the LATEST 1.2.x version, and 
then install the latest 1.3.x version.  This way, both libraries should 
work in parallel without any problems.

HTH,
