Re: Sendmail Config

On Thu, 2002-01-10 at 15:47, James.Wilson@compsm.com.au wrote:
 
> I am running Sendmail 8.11.6 on RH Linux 7.2.  I would like to allow only
> relaying from a few particular machines and deny connections from all other
> not specified.  I am trying to achieve this without the use of the wrapper
> or any firewall rules.  Is this possible using some kind of rulesets in the
> sendmail.cf file.  Any suggestion is greatly appreciated.

You need to use the access_db feature. The default FreeBSD configuration
looks like this:

/etc/mail/freebsd.mc
   FEATURE(access_db, `hash -o /etc/mail/access')

/etc/mail/access
   cyberspammer.com                550 Spam not allowed
   FREE.STEALTH.MAILER@            550 Spam not allowed
   another.source.of.spam          REJECT
   okay.cyberspammer.com           OK
   128.32                          RELAY

This is all documented on the Sendmail website, available from
http://www.sendmail.org/.  You should have looked there /before/ coming
to the list.
