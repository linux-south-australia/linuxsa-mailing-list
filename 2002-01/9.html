configuring sendmail to catch invalid user emails

Hi all,
happy new year and all that..

I have sendmail 8.11.4 on redhat 7.1 and am trying to make sendmail catch
the emails that come through to an invalid user for our domain.
I have already worked out how to do it for virtual domains by adding into
virtualusertable.db;
@virtualdomain     localuser

but if you try and do this for the real domain it sends all mail to this
user for the whole domain as it obviously looks through the virtualusertable
before aliases or actual users (I think in that order)

there is a LUSER_RELAY feature but I quote sendmail.org
"this feature is disrecommended, since it can break ` working'' addresses if
you introduce new local users."
hmmm... I am not the best with sendmail configuring anyway, dont want to
make it harder..

so I thought about just putting all the domain users in the virtualusertable
before the catchall line and it should do them first before sending the rest
for that domain to one user.. then checking aliases to work as normal...
this works in theory although a little time consuming if you have many
users..

my questions are;
is this the best/only way to do this??
is doing this viable? (will it break anything else?)
or could someone recommend me another method to read up on??
