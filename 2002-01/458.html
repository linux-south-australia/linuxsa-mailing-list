Re: [OT] IT Religions -- the ugly side...

Just a quick comment about the (in)ability of users to use complex things..
Sometimes people can surprise you with what they can figure out, if you just
let them do it themselves, instead of butting in with "no no no, here, let
me do it" or whatever.

I caught up with my parents a little while ago, and my mother mentioned that
she hadn't been able to install a new version of some software she used on
her PC.. The instructions said you had to remove the old version first, and
she'd done that through windows control panel.. new version still didn't
install.
So, she removed that version, then searched the harddrive to find related
remaining program directories for that program, and renamed them, then went
thru the registry and searched for and deleted all references there too.
After this, the new version installed fine. End of story.

So, she figured out how to:
* locate relevant files
* adjust configuration files directly
* search for info on how to deal with a problem
* perform undo-able actions (ie. rename, not delete)
* learn from trial and error
* understand concepts involved, rather than blindly following the NEXT>
  button, and giving up if it didn't work.

Now, that sounds very similar to managing a modern Linux distro!
If I gave my parents a copy of Mandrake 8.1, I reckon they'd manage to
install and use it fine on their own. They'd probably get annoyed when
Abiword crashes doing cut-n-paste, or by not getting the same level of
options for their colour printer, but thats not their fault.

So, anyway, if you haven't fallen asleep by now, my point is that Windows
required complex operations to make it perform happily. So does Linux.
Linux will work in a "degraded" kind of way, just like Windows, if you can't
keep up with stuff like manually adjusting things occaisionally, updating
it, etc.

So, don't put down "Average" users too much; I think their main issues with
Linux would be application support and integration, rather than the actual
day-to-day running and use of the system.

-Toby

> > Nor is it true that the objection to HTML is based on editor

> > legibility and (2) the fact that HTML messages can (and do) force the
> > receiver to download images and other trivia which can cause
> > inconvenience to the receiver.

> Take your HTML message to another MUA, and you're sure to find that the
> message doesn't display the way you thought that it would. When you deal
> with plain text, that issue's not there.
>
> > > > Today's savvy home users are editing DV video, creating
> > > > high-quality digital photo albums, playing high end 3D games, and
