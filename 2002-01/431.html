On 2002.01.21 13:39 Dale Long wrote:

> This all assumes that you only use packages within the distribution in
> question in order to completely benefit from it. Which can be a mixed
> blessing.

There's also "autoupdate".  This allows you to build your own
library of RPMs to be installed/maintained.  So it's not only
restricted to the manufacturer's RPMs.

We run a RPM library per Red Hat version.  And then two libraries
per classification of machine (web server, accounting, etc) of
(1) RPMs that must be installed and (2) RPMs that, if they are
installed, should be updated.

We package all software into RPM format, this is really worth
the hassle.

autoupdate understands dependenices very well.  Something autorpm
doesn't do well.  autoupdate also works well unattended.  So you
can maintain a whole stack of machines very simply.
