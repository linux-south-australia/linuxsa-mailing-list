Re: upgrading Fbsd port ?

On Mon, 21 Jan 2002, Wilkinson,Alex wrote:

> I have a situation where I need to update GTK from: gtk-1.2.10_2 to
> gtk+-1.3.12.
>
> However, I can't do a pkg_delete gtk-1.2.10_2 because I have tonnes of programs
> that require it. eg.
>
> "pkg_delete: package 'gtk-1.2.10_2' is required by these other packages"
>
> mlib-1.9.11
> gnomelibs-1.4.1.1
> fnlib-0.5
> ... etc
>
> So it seems as if I have only one option ie
> to build the gtk+-1.3.12 port as well as keep the gtk-1.2.10_2 port.
>
> I really would prefer to delete gtk-1.2.10_2 first and then install the new port
> or somehow upgrade gtk-1.2.10_2 to gtk+-1.3.12 instead of installing both.

Generally speaking, you can use the -f parameter to pkg_delete to force it
to remove the package anyway, then reinstall the new version. However this
can cause problems, since these libraries often change between versions,
and the dependent software needs to be rebuilt.

However there is a happier solution. Install portupgrade
(/usr/ports/sysutils/portupgrade) and let it do the work for you. It can
upgrade the port as above (just less manually) and if you ask it to. It
can also rebuild dependent ports recursively, which should solve most
shared library problems.

man portupgrade
