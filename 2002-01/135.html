RE: IPtables / Firewall question

 
> Hi all,
> Can anyone help me with this security question?
> 
> I have a machine, 'db', that is behind a router/switch with 
> fairly tight firewalling rules. The database is a redhat 7.2 
> box, iptables 1.2.4. The only thing this box does is serve db 
> content to a web server. So port 4567 can be accessed only 
> from ip 1.2.3.4 and ssh from ip 2.3.4.5
> 
> Is adding a stack of iptable rules to this box overkill?

It may be. There are two conflicting security goals here -- depth and
simplicity.

Depth: it's good to have multiple levels of security. Usually, it's
desirable to have different types of security at each level -- it's not
much use having three identical firewalls in a row, as any vulnerability
in one will most likely be present in the others. A quote from Rusty on
this: "I recommend security in depth: combine tcp-wrappers (for
connections to the packet filter itself), proxies (for connections
passing through the packet filter), route verification and packet
filtering."
(http://netfilter.samba.org/unreliable-guides/packet-filtering-HOWTO.txt
) ...

Simplicity: The simpler your setup, the easier it is for you to
understand, and hence, you're less likely to make mistakes. If you're
not careful, you can end up with holes at every level, which means that
rather than the cracker needing to get through _all_ your layers, they
just need to get through one, and letting them choose which one is not
good...


> Or should that question be is iptables the right app for a 
> machine level firewall that isn't routing?

Yes.
