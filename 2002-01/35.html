It looks, as per the message "cannot resolve PTR record for ....", dns 
server of  4-vendor.com is not resolving name/ip properly. But, 
amazingly it looks it is trying to resolve the name from the IP (reverse).

One thing, when I try to resolve the following:

    $dig 4-vendor.com

it did not resolve as expected. If 4-vendor.com is the real domain you 
are trying to send email, you obviously cannot do so until its DNS can 
resume resolving is names/IP addresses.

Hope, it may help, or as error message indicated, it will be okay as 
soon as the required info is resolved itself.

Damar Thapa
