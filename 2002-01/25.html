Re: hex editor for linux

Toby Corkindale wrote:
> 
> i'm looking for a hex editor for linux.. (easy request so far)
> BUT I'd like something that can load and edit Motorola-style dumps.. with
> headers and checksums per line of several words.

Hi Toby,

You probably need to string some tools together, using something
like SRecord (http://www.canb.auug.org.au/~millerp/srecord/) to
pack/unpack the file.  I haven't used it, but I've some of Peter's
other software and it's idiosyncratic but good.

Emacs/XEmacs has a hex editing mode.

Have a read of the documentation for the Emacs package
jka-compr -- you might be simply able to define the
S-record format as a type of compression with the
SRecord program as the compressor/decompressor.

Then define hexl-mode as the major mode for editing
files with extension ".s".

If you use another editor you can probably do the same,
in the worst case you could write a shell script called
"beds" that did something like

  srec_cat  -Output $1.tmp -Binary $1 -Motorola
  bed $1.tmp
  srec_cat -Output $1 -Motorola $1.tmp -Binary

Bed was just one of the many hits I got on
http://www.freshmeat.net./ for "hex editor".
I personally use XEmacs (the devil you know).
