Re: what gui language to learn ???

> Hi there all,
> I was wondering if some one could point me in the right direction.
> I have this private project of my own that has been on the back 
> burnersfor the last 8 years, The problems I have had up to date is 
> whatlanguage to use / learn.

My biggest recommendation is to use / learn a language and toolkit that 
is portable. Of course there is java, but many applications make 
careful decisions about the gui toolkit so it is portable. Gimp, Dia, 
xemacs, etc.

I would take a look at WxWindows http://www.wxwindows.org
GTK is portable too, but I prefer to use classes and methods for GUI 
programming.

If you need fast 3D / 2D, OpenGL all the way.

Most languages are portable, C++ or python will suit most toolkits the 
best.
