Re: gcc

"David Panton" <david.panton@unisa.edu.au> wrote:

> I have code written in gcc which compiles and executes in RH version
> 5.2 and when ported to a machine running RH 6.2 also compiles. On execution
> however I am getting a segmentation fault and core dump.

If you recompile and link the app with gcc -g, it leaves the debugging symbols
in.  When it generates a _new_ core file, you can use "gdb [executable]
[corefile]" (replacing with the appropriate values), then at the the gdb
prompt execute the "where" command, which will give you a stack traceback.
From there it's left as an exercise to the reader.
