Re: root login from remote machine

> You are much better off logging in as yourself in telnet then at the prompt
> type
> 
> "su -" (without the inverted commas)
> then it will ask you for root's password, enter it in
> and presto you have superuser access.

Oh dear.  Telnet will then send the root password
in plain text across the wire.

Best practice is to connect as yourself to the remote
machine using (Open)SSH configured only with certificates

   ssh remote.example.edu.au
   Passphase: *************

and then become root using SUDO (configured to allow members
of the "wheel" group to become root without knowing the root
password).

   sudo -s
   Password: ******

Write down the root password and lock it away in a safe place,
as with sudo you very rarely need it.  You should make the
root password very, very good (that is, essentially random).
You should also set passwords on the BIOS and lilo and
record these also, otherwise anyone can alter your files
by booting the machine from floppy disk or giving an
"init=" string to the "LILO:" prompt.
