fallover prevention for network card

hi all
I've been having some hassles with a network card in a Samba 
server.  Its a Netgear fa310.
It has the rather nasty habit of dropping out, leaving the following 
repeating message in the kernel log:
(btw, running rh6.2 w/2.4.2 kernel)

<date, time, hostname, etc> kernel: NETDEV WATCHDOG: eth0: transmit timed
out
<date, time, hostname, etc> last message repeated 7 times

.........and so on

This has happened 3 times in the last couple days, and appears to happen at
random.
If anyone has an idea as to what could be causing it, that would be great.

What I'm doing tonight is removing that card & replacing it with 2 NICS, the

theory being that both cards will be set up identically, one i/f up, the
other down.
Then if this should happen again, or for some other reason an interface
dies, the
other one picks up & no users have any idea that the server just had a bit
of a hiccup.
Does anyone know how to automate this process, without me having to type 
"ifdown eth0; ifup eth1" at a console?

All ideas / help appreciated.
Thanks
