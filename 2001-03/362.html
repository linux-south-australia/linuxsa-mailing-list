Re: Echoed Status and Content-type Problem

>If you go to this URL:
>http://www.acns-online.com/chuck/stories/op/storiesView/sid/3/
>you will notice at the top of the page the following output:
>
>Status: 200 OK Content-type: text/html
>
>This should not be there.  There is a header() call in the redir.php3
script
>as follows:
>
><snip>
>header("Status: 200 OK");
></snip>
>
>Why is this echoing on the browser screen?  Also, the Content-type:
>text/html is nowhere in the scripts and still being echoed.

There is no such http header as "status". The status is issued as the HTTP
version, followed by the status code and a description

ie:

HTTP/1.1 200 OK

Whereas you are returning

Status: 200 OK

The page is not a valid http response, but your browser (probably IE) is
attempting to render it anyway. I'll also do this if you go to any other TCP
server, ie: http://mail.something.com:25/

It's odd that PHP lets you do this, maybe a previous version had a provision
for an alternate status code before, or the header() function has been
changed to allow a more nph- like control over the output.
