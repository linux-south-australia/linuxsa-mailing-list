Re: DNS Question (Kind Of)

On Thu, Mar 29, 2001 at 07:47:29PM +0930, David Lloyd wrote:
> 
> I have a dilemma. I want:
> 
> * http://www.originaldomain.com/website/
> 
> To redirect to:
> 
> * http://www.anotherdomain.com/website/

David,

When I do redirects, I use these sorts of levels:

-	html file header redirect (called Refresh)
-	Apache config file redirect (called Rewrite)
-	DNS zone file fiddling

The first does file to file redirection.
The second does file _pattern_ (egrep) to file pattern redirection.
The third does IP address to IP address redirection.

> Ideally I'd like to redirect it via DNS so whenever DNS saw
> "www.original.com/website" it actually resolved to "<ipaddress
> of>www.anotherdomain.com". I know it's possible but I can't quite work
> out who I should be speaking to, or what I should say.

I'm 99.999% sure that you cannot do pattern to pattern redirection
by DNS. Only whole IP to IP address redirection.
So what you want is an Apache config file Rewrite, using mod_rewrite.

Cheers, and apologies if I said something obvious or incorrect,
