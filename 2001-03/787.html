Re: WindXP => no more dual boots with linux??

Alan Kennington wrote:

> ----------------------------------------------------
> The utility companies are going to get it again when Windows
> XP ships - not that they've any right to be surprised. Microsoft
> has made some small revisions to NTFS with WinXP, resulting
> in the likes of Partition Magic bouncing straight off WinXP
> installations, splattering error messages around the place.

Small revisions to NTFS. They did exactly the same thing with Windows
2000, and that still dual boots fine. I can't say much about Partition
Magic, as I haven't used a version for quite some time.

> This probably goes for any other utility that manipulates
> partitions and/or wants to twiddle around with Master Boot
> Records, and it surely won't come as a surprise to the vendors,
> as they regularly get whacked by Microsoft file system upgrades.
> Still, it's a bummer for Partition Magic, which only started
> running native on Win2k relatively recently, and WinXP is in
> essence a rev of Win2k.
> ----------------------------------------------------

I'm not sure how the MBR has anything to do with file system changes.
They may be talking about the problem with changing the partition size
in the partition table. This is guaranteed to screw up an NTFS
partition, unless you rebuild all of the file system structures and move
the file data to within the new size. You would most likely get an
INACCESSIBLE_BOOT_DEVICE blue screen on boot. This would only remain the
case until a) someone worked out what the changes were and told the
world about it, b) some company pays microsoft heaps of money to be able
to find out how the new modifications to NTFS work or c) someone waits
for a little while and gleans as much information as they can from
resources such as MSDN.

I very much doubt that this would affect people's ability to run Windows
XP and other systems. It won't matter to me anyway, as I won't be using
XP. :-).
