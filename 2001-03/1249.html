File redirect would be the simplest way -- Browser connects to the original
site, gets sent HTML (or is it a HTTP response? -- I have a feeling you can
do it either way) that automatically redirects your browser... It also
depends if you want to keep using the original address or not... You would
have seen the pages that say "we have moved to http://blah/. You will be
redirected in 10 seconds or click here." or something to that effect? I'm

DNS is "Domain Name System". It only works with Domain Names. Hence, it
never sees the "/website" part of your address.

check http://www.faqs.org/rfcs/rfc2068.html for how HTTP/1.1 works... it
will give you more information than I can :)

in fact, try this:

----
sysrar@mullet:~$ telnet www.w3.org 80
Trying 18.176.0.26...
Connected to www.w3.org.
Escape character is '^]'.
GET /pub/WWW/TheProject.html HTTP/1.1
Host: www.w3.org

HTTP/1.1 301 Moved Permanently
Date: Thu, 29 Mar 2001 14:46:23 GMT
Server: Apache/1.3.6 (Unix) PHP/3.0.15
Location: http://www.w3.org/TheProject.html
Transfer-Encoding: chunked
Content-Type: text/html

<snip>
----

Note the "HTTP/1.1 301" -- this is a HTTP redirect, as opposed to a HTML
redirect, which I'm pretty sure is also possible...
