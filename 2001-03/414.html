Re: Squid port problems

Phil Hutton wrote:
> 
> I have a squid cache that dies regularly (almost every day).
> 
> At some stage the squid child process dies.  There are no entries in
> the cache.log at this time, but the following appears in
> /var/log/messages

Looks like a programming error.  Probably in Sqiud rather
than the kernel.  Recompile Squid to include symbols and
override any configuration that prevents the generation
of a core file.

When it dies you can attach a debugger to the system state which
is frozen in the core file and mentally walk back through the code
from the NULL pointer to see what generated it.

Don't be afraid to do this -- you don't need much knowledge
of programming, and it's one of the best ways to learn.

After some years of doing this and some familiarity with
the CPU architecture the details in the syslog start to
make some sense, but I'd ignore them for now apart from
the line:

> Mar 12 20:40:40 proxy kernel: Call Trace: [sys_poll+336/364] [system_call+52/56] [startup_32+43/285]

which suggests that the NULL pointer was handed to the
operating system's poll() function.
