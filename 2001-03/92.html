IP NAT under Kernel 2.4.x

Hi all

I had reason to install a system using the 2.4.x kernel. (I had to use
Multilink PPP) I have IP NAT working using iptables, but I have not be able
to get realaudio working (the customer likes to listen to music when working
late). I have search the net for an answer and have not found any references
to these type of protocols
Has any one seen any thing in relation to protocols like realaudio and the
like ?

Thanks
    Mike O'Connor

-------------------------- current firewall
commands ------------------------------
# Load the NAT module (this pulls in all the others).
modprobe iptable_nat

# In the NAT table (-t nat), Append a rule (-A) after routing
# (POSTROUTING) for all packets going out ppp0 (-o ppp0) which says to
# MASQUERADE the connection (-j MASQUERADE).
iptables -t nat -A POSTROUTING -o ppp0 -j MASQUERADE

# Turn on IP forwarding
echo 1 > /proc/sys/net/ipv4/ip_forward
