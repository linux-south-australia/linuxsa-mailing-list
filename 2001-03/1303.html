Recursive CVS Update

Hey All,

I'm using CVS over SSH for a web project. I've got a shell script that
runs the following commands:

cd /dir/name/here
cvs -d /html/cvsroot update modulename

Where modulename is a directory in /dir/name/here/

It will update the files in that directory that have changed in the
repository, but won't get the files that I've just added to the
repository.

ie, It will download the ones that I've been working on for ages and
already exist (in the form of the previous version), but won't download
new ones that it already has a copy of.

The only way I've been able to get the new ones via CVS is to SSH to the
box hosting the webpages and do the following:

cd /dir/name/here
cvs -d /html/cvsroot co modulename/dir/containing/new/files/

Which will download the new file(s).

Surely there is a way to do this, isn't there?

If someone could recommend a switch that I could add to my update
script, I'd be most appreciative.

Thanks in Advance.
