Re: SetUID/SetGID stuff

At 10:20 AM 3/19/01 +1030, Phil Hutton wrote:
>Hi...
>
>> This is OK, except group1 members aren't necessarily going to be members
>> of group2. If I set the SetGID bit on the directory, can group2 then
>> read the contents of that directory (which I don't want)?
>
>No, they can't.  If you set the setgid ("sticky") bit on that
>directory, then all files that are created in the directory will have
>group = group1.  I think that's what you want.

Ummmm, the SetGID bit is not the sticky, or T bit. They have two different
meanings.

Indeed, if you set the SetGID bit on a directory that is owned by group1,
then all new files created in that directory will be owned by group1. If
you set the Sticky bit, then only the owner of a file can delete it,
regardless of the directory's permissions. Thus, you often see /tmp with
1777 permissions.
