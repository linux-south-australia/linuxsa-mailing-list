Re: mailing from within a perl script

Dan Kortschak wrote:
> 
> Hi Everyone, I'm in the process of translating a bash script that I wrote
> a while back, into perl. I've sorted everything out except for the
> following:
> 
> echo $number BAMBE processes based on $infile and started at \
>    $start_date, finished at `date`.$'\n'The output files can be found in \
>    $location. | mail -s "BAMBE Notification" $notify
> 
> and I'm sure that there must be a more sensible way of doing it. I've
> looked at making a system call, but it's pretty lonng winded and also
> at named pipes, but that seems like overkill. Is there an obvious, simple
> and elegant solution that I'm missing, and what is it?

I use the Mail::Sendmail
(http://mirror.aarnet.edu.au/pub/cpan/modules/by-module/Mail/Mail-Sendmail-0.78.tar.gz)
module. It's very simple, clean and it works on most platforms (even
those without sendmail installed).

Hope it helps,
