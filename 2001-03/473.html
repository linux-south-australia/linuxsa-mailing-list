Re: Throttling bandwidth on the WAN

On Thu, Mar 15, 2001 at 10:56:59AM +1030, michael wrote:

 > On our WAN, I often need to copy the odd large file between sites using
 > ftp or scp, but I find that my transfer is maxing out the bandwidth, such
 > that other users of the WAN are impacted. I know the BOFH would have a
 > solution to this which does not involve any impact on the file transfer,
 > but is there a way of slowing down the file transfer on a case-by-case
 > basis? (If it was middle of the night, I'd like full speed please..)
 > I'm sure I can do this in the cisco routers, but I'm hoping for a more
 > flexible solution..

The Cisco solution is flexible :-)  You can traffic-shape by access
list;  You can also define a time-based ACL.  So, if you define an ACL
which matches FTP packets over the WAN between 9:00am and 5:00pm and
traffic shape it down, you'll free up bandwidth for everyone else during
those hours, and you won't affect packets which aren't part of your
file transfer.
