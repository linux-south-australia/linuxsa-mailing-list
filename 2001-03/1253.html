iptables query: -m multiport -sport x,y,z

Here's a little technical question that arises in
building a new firewall with "iptables" with linux 2.4.0.
I can't find the answer in any of the manuals I've
looked at.
In fact, the "iptables" program does not seem to conform
to the manuals, especially "man iptables".

Here's what I want to do:

-----------------------------------------------------
iptables -A INPUT -i ppp0 -d 203.38.148.52 \                                                                                          -p tcp -m multiport --tcp-flags ACK ACK \
    --dport 1024:65535 \
    --sport 443,554,2401,7070,8000,8080,8900,11371 \
    -v -j ACCEPT 
-----------------------------------------------------

It's simple enough.
All I want to do is to permit ACK packets from
ports 443,554,2401,7070,8000,8080,8900,11371 on
any host to ports 1024:65535 on host 203.38.148.52.

But this is what "iptables" says:

-----------------------------------------------------
iptables v1.1.2: invalid TCP port/service `443,554,2401,7070,8000,8080,8900,11371' specified
Try `iptables -h' or 'iptables --help' for more information.   
-----------------------------------------------------

Well, without the option "--tcp-flags ACK ACK",
there is no problem.

Can anyone out there who is experienced with "iptables"
tell me what I'm doing wrong? (If anything!)
The order of "--tcp-flags ACK ACK" and "-m multiport"
makes no difference.
