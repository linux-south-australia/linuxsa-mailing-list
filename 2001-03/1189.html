Re: importing fonts

Jerry Boyko wrote:
> 
> Can you import the fonts from your windows font folder to your Iinux
> machine?  If so how do you do it?

The first thing to realise is that font handling in Unix
sucks badly.  Whereas the Mac and Windows have a single
font handling library, each Unix application does it's
own.

The next thing to do is to run the most recent version 
from your Linux distributor.  You can then safely ignore
all the instructions on the web about hacking X Windows
to support TrueType fonts, as support will be included.

Then you can either copy your fonts into somewhere like
/usr/share/fonts/userfonts or mount the Windows drive
(say at /mnt/vfat/c).  Use the utility ttmkfdir to
create fonts.scale.  The utility mkfontdir copies
this to fonts.dir.

Edit the file /etc/X11/xfs/config and make sure that
the directory with the fonts is listed.

Go into text mode (sudo telinit 3), restart the font
server (sudo /etc/init.d/xfs restart), check for errors
(sudo tail /var/log/messages), and then start X Windows
by hand *not* as the superuser (startx).  If X freezes
because the font server isn't behaving properly press
Ctrl+Alt+Backspace. If it works, exit X and re-enter
the usual graphics mode (sudo telinit 5).

The X Windows application now has your windows fonts
available.  See how painful that was -- let me say again
that font handling in Unix sucks.

       -------------------------------------

Now that you are blooded, you'll be pleased to know that
getting StarOffice to read your TrueType fonts is worse :-)

The best description for recent StarOffice distributions
is at:

  http://home.ict.nl/~frputten/ttf.htm

Ignore the text about getting TrueType fonts to work
with X, your distributor has taken care of that.

If you have a PostScript printer (rather than a
GhostScript-driven printer) you'll also need to
package the TrueType fonts as PostScript fonts.
This is described in the StarOffice doco.
