I have a connection problem with Telstra ADSL.

Originally Telstra had set it up on a Window 98 machine (due to them =
being thick and knowing nothing about Linux)

Anyway due to the marvellous abilities of Windows 98 to crash along with =
any proxy server running with such a broadband connection I recently =
moved the connection over to there Linux server (through no help of =
Telstra)

Anyway I got everything working all sweet, sharing the connection =
through the LAN using Jsentry Proxy server (www.jsentry.com) which works =
very well.

The problem seems to be (and this happened on the Win98 machine) is the =
ADSL modem seems to be locking up.  I have seen this problem at other =
locations also!  Are all these modems faulty?   Alcatel is the brand.

Anyway while this is a problem one problem, it inturn causes another.  =
The Linux box (Caldera 2.3) still thinks the connetion is open with the =
PPPD and PPPoE Processes still running.  The Telstra sever assigned =
Internet IP is also still present in the routing tables. =20

I can get the ADSL to connect up again if i kill off the PPPD daemon, =
but the ADSL then assigns it's self to PPP1 interface due to PPP0 still =
being open from the last failed connection.  This inturn makes the =
Internet unreachable.

My main questions are :-

    Whether or not anyone knows if these modems are known for this =
problem?????

    Is there a simple way of having the PPPD PPPoE and routing clear =
when the connection drops?????

   &  if so can i make the ADSL automatically reconnect????


If the above cannot be achieved i think i will have to set the Jsentry =
proxy server to connect the ADSL on demand (start-adsl) and time out =
(adsl-stop) after a brief period. =20
I would rather fix the problem than do this as I can still see future =
problems arising.

Any help would be greatly appreciated if one could also email replies =
too

    dpoole@gogo.net.au


Thanks Danny
