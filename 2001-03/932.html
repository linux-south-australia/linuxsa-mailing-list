Richard,

There wee no messages - and in fact, all of the required modules
are clearly there where they should be.
But there is no mention of these modules at all in modules.conf.

Someone has told me offline that I might look at whether "kerneld"
is present to do automatic module loading.
But the binary for kerneld is not even there!!
Obviously something must have gone wrong in the installation,
even though I specified that all SuSE packages should be loaded.
And the rc.config file has the variable START_KERNELD
set to "y".

The kernel module iptable_filter.o _is_ automatically loaded
by "iptables" commands, but the module ip_tables.o which is
needed (apparently) by iptable_filter.o is _not_ loaded
automatically.

Whether kerneld running would fix this, I don't know, but it's
a bit worrying that apparently there has been some sort of
installation error.
