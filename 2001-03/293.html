Re: iPrimus ping packet loss

> what is single-homed and multi-hommed ?

It's how many upstream providers you have. Single-homed if only one uplink
or multi-homed if more than one uplink.

But system which has more than one physical connection to different upstream
providers can be considered as a set of single-homed clients (doesn't matter
is it physically one computer/router or not) until this system doesn't have
AS (Autonomous System) and not runin' BGP (Border Gateway Protocol). Who
prohibit you to have two different computers with modems which dialing
different ISPs while these computers stay on the same table? No one.

So there actually three different types of systems:
1. Single single-homed system
2. Set of single-homed systems
3. Multi-homed system (with AS and BGP)

Linux allow you to make static routing like: FTP and HTTP traffic will go to
one upstream provider and all another protocols (including ICMP) to another
provider. That's point where you can save some money by utilizing Telstra's
cache (which cheaper) and you will be still considered as single-homed
system.
