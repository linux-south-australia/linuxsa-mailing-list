Re: Inetd and hosts.*

Andrew Halliday wrote:
> 
> Hi all,
>         A workmate and I just had a convo about the relationship (if any)
> between inetd and the hosts.* files.  Is there any?  My workmate swears that
> changes to hosts.* files are implemented even when not running the inetd
> service.  When I was a beginner, I was convinced that hosts.* files were
> implemented at an 'OS' / networking level, but was later convinced by
> someone else that they are indeed just extension config files for
> inetd(.conf), and are therefore useless if not running inetd.
> 
> Was I convinced wrongly?  WHO / WHAT implents the security set out in these
> files??  Are they more communal files like the 'hosts' file?

It's used by TCP wrappers. Any program that is built to support TCP
wrappers will use these files, as well as those who are run by inetd.
