Re: tar '.*' files

Hi...

> You can use .[A-z]* which works fairly well.  Or .[A-z0-9]* if you
> want to include files that start with a . and then a number.
> 
> Alternatively, you may be able to use tar's --exclude function e.g:
> tar cvf test.tar.gz --exclude ".." --exclude "." .*
> but that's a bit ugly.  It's probably a good idea to be in the habit
> of using .[A-z]* , lest you do something recursively nasty as root.

OK, so I forgot the "z" in the tar command, and I also thought you
only wanted the .* files.

In that case, the best solution (as presented by Michael Davies and
Andrew Reid) is along the lines of:
cd /home/benw
tar czvf uni.tar.gz uni/

This will tar up *everything* in the uni directory, and when untarred
a new uni directory will be created (if one does not exist) and the
files will be extracted into that.

If you just want to tar up the contents of the directory (so that when
the tarball is extracted it puts the files in the current directory,
not a directory called uni) then you can use (as suggested by Martin
Sandiford):
cd /home/benw/uni
tar czvf ../uni.tar.gz .
