Or even, to actually be helpfull, rather than witty:

run nmap (you will probably have to google for it) and it will tell you
what ports are open.

Also, check out /etc/inetd.conf

there is listed all the ports that you can connect to. However, there
may be ports open that are not listed in this file. This is because this
file dictates what ports are being listened to by inetd. (inetd answers
the connection, then pases the connection onto another programs).

Comment or delete all ports not needed from inetd.conf, then killall
-HUP inetd.

Run nmap, and make sure all the ports are closed that should be.
