perl rsh call into the right directory

Hi All, hopefully this will be the last thing I need to ask about this
script.
After my script sets up all o the parameters that it needs to it runs
porcesses based on these parameters with the following system call:

system ("rsh $hostlist[($index - 1) % $processors] bambe < $head.p$torso.$index.rc > /dev/null 2>&1");

Although this works (that is the redirections are understood in the wrong,
but adequate context), when the output files from bambe (rather than the
stout/stderr burbling which I'm dumping) are written, they end up in ~/
rather than the $PWD (to clarify this - hopefully, the ~/ tree is mapped
with NFS to the node that you are executing an rsh call on). My question
is: how can I get the rsh call to make the node start the bambe process in
the $PWD of the master machine? In the shell I can 
`rsh node "cd $PWD; bambe etc..."' but if I try to simulate this in the
perl system call I get an ambiguous redirect error. Any ideas?
