Re: Files and redirection

At 02:13 PM 3/28/01 +0930, Nicholas Morrison wrote:
>Hello!
>
>Is there anything wrong with the following command?

Yes. Perhaps you have already detected the problem ... emptyness!

># cat /etc/shells | grep -v csh > /etc/shells
>
> or, for that matter:
>
># grep -v csh /etc/shells > /etc/shells

The shell processes the redirection request first. This opens /etc/shells
for writing and truncates it to size 0 if it already exists.

It then executes the command or commands in the pipeline and arranges that
stdout for the last command points to /etc/shells.

However, by the time the command executes, /etc/shells is empty :-)
