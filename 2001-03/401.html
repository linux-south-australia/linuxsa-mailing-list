Re: Syslogd Replacements

"Andrew Reid" <andrew.reid@plug.cx> wrote:

> Syslog regularly takes up abot 60% CPU, making the server run like a
> dog.
> 
> I believe that Linux Syslog is a bit slow, compared to say, BSD Syslog.

The issue here is more likely due to the amount of logging that
you're doing.  When syslog normally logs to a file, it fsyncs afterwards
to ensure that the write has been commited to disk (otherwise, if a
crash were to occur and your messages were still in memory waiting
to be written, you'd lose them).  Looking at one of the 6.0 servers
that I have here, the man page for syslogd says that if you prefix the
file name with a "-" sign, that should disabling the sync'ing behaviour
(at the cost of the aforementioned crash scenario).  Give that a try
and see what happens.

> Can someone recommend something that is Linux-Syslog compatible (server
> is running RedHat 6.0) that has a lot better performance.

There is "syslog-ng", the "Next Generation" syslogd, which has
interesting filtering options and the like.  But it's likely to suffer from
the same "problems".
