If a kill -9 doesn't work, it usually means that the kernel is still waiting
for something else to happen. On BSDi systems, you used to be able to fix
this with something like (my memory is really being stretched here..):

stty <device being used> flush

This would flush the i/o buffers on the device so that the process could
continue on and catch the TERMINATE signal issued to it earlier. The stty
manpage for linux doesn't read anything like the old bsd one did, but maybe
it's a starting point (there is stuff in there to send an EOF to the input
followed by terminate).

For the nostalgic, this used to be the way to fix 'hung' modems on systems
that most ISP's resorted to a reboot to fix.
