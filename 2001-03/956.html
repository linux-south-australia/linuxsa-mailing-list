> hub would be limiting performance wouldn't it.

Depending on how much data this thing has to move. ie, If it is only
connected to a modem link and has only a few clients hanging off of it,
the 10/100 issue doesn't play a large part.

> to use a $55 hub I would still need 1 network card for the server and would 
> therefore cost me ~$100
> 
> If I were to buy a 100baseT hub it would cost me $45+ $150 = $195 
> to use two network cards in the server would cost me 2x $45 = $90

I'm not sure if you know that you can purchase Cheapie 10/100 PCI
Network cards from most suppliers for around the $25 mark. I wouldn't
trust them with a mission critical application, but for a box that
doesn't move too much data, they're generally OK. (These cards use the
RealTek 8139 Chipset, supported by Linux).

2 x $25 = $50. That's not too bad to my way of thinking.
