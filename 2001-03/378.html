Re: One set of accounts - over multiple computers

Richard Russell wrote:
> 
> > > Subject: One set of accounts - over multiple computers
> >
> > You (probably) want a combination of NIS (or NIS+) and NFS. This is the
> > standard Unix way of doing things...
> 
> as others have mentioned, LDAP is also an option, and it is a more powerful,
> flexible option than NIS or NIS+, as well as being a non-unix-specific
> solution. If you run an LDAP server, many things can authenticate against
> it.

Not just authentication. You can use LDAP for any number of things (we
use it for routing mail between our two mail servers). It is also
apparently possible to do domain name lookups on an LDAP directory,
rather than use DNS (in fact, pretty much all of the services listed in
/etc/nsswitch.conf can be done using LDAP).
