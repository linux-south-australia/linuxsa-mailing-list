Compiling Apache 1.3.19/1.3.12 with mod_ssl

Hi all,

I'm trying to compile ssl support into apache again, but this time, I'm
having a little trouble. I have successfully compiled and run
apache/mod_ssl on FreeBSD 4.2 and RedHat Linux 6.0 and 6.2 before. I
have a different RedHat 6.2 machine, but I'm having a few problems
(well, actually one big one).

I ./configure mod_ssl and point it to the corresponding apache source
and it successfully patches the source.

I then cd ../apache-1.3.xx and ./configure

Whenever the I ./configure apache with the --enable-module=ssl option,
the configure script returns to the command prompt with no error message
after it (successfully?) adds the ssl module. It mentions none of the
other modules and configures and compiles successfully with the same
command line after removing the --enable-modules=ssl option. Because the
configure script ends where it does, the Makefile is not finished and
will therefore not compile.

This happens with both apache-1.3.12 and 1.3.19. (In fact the 1.3.12
source and the corresponding mod_ssl source are the same tarballs I used
on FreeBSD and RedHat 6.0). I've also tried the obvious make distclean,
etc.

Any suggestions?
