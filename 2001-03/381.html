bjohns44@csc.com.au wrote:
>
> How would those of you setup LDAP for a workgroup? Is that kind of design
> discussed in any of the doco for LDAP? Is it just something that you can
> plug in via the nsswitch.conf file? or is PAM based?

If you're talking about a Windows WORKGROUP, you need to look at Samba
or Samba TNG. Both can (actually, I think it's currently not working in
Samba) use LDAP as a password backend. Meaning that they store all of
their accounts information in the LDAP database. This alone is not
enough to keep both Windows and PAM/Unix clients happy. Both SMB and
Unix passwords are encrypted differently, but it is possible to keep
them synchronised using Samba 'unix password sync = yes' option.

> Where do I go to get more specific info, anyone with any experience?

www.samba.org and
www.samba-tng.org

Somewhere off samba-tng.org, you should find a link to the Samba-LDAP
PDC HOWTO and the Samba-TNG-LDAP PDC HOWTO. There's also plenty of other
documentation on how the whole NT domain thing works (from a Unix point
of view).

hope it helps,
