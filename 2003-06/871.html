Hello,

I have recently installed Redhat Linux 7.3 on a replacement server.

I have had several problems but the main one is to do with the GUI display.

I know nothing about the gui display and have been learning heaps but none
of it is helping me to try to find a solution to the gui display problem.

I would be greatly appreciative if anyone would look at the errors I have
got below and could tell me if they may have seen them before and if they
might have some possible things I could try to fix this.  I am hoping
someone may be able to help:

1.	When I logout of my gnome session on the console, it does not return
me to the logon gui screen but gives me a blank/bluey coloured screen.

2.	I need to be able to open an X-Term window (in particular via
Windows NT 4 SP6a or W2K SP3 called Attachmate KEA! X - Version 4.1) on a
Windows box for the applications developers to do work on the linux server
but when I open the Kea! X window I get a blank display.

I think problems 1 and 2 are related.  Another person in my company tried to
help me fix the problem but in the end suggested swapping to KDE and I did
but got the same problem, so I swapped back (I like the gnome gui better).

The only errors I am getting are in the various /var directories, eg

In the kdm.log in the /var/log directory of:

    (VW) VESA(0): Failed to set up write-combining range
(Oxf0060000,0x770000)

Note: this occurs various times with different range numbers in parameters.

Then I see this (it also appears in the XFree86.0.log in /var/log, and the
:0.log file in /var/log/gdm as well):

    SetKbdSettings - type: 2 rate: 30 delay: 500 snumlk: 0
    SetKbdSettings - Succeeded
    AUDIT: Fri Jun 22 15:35:13 2003: 1599 X: client 6 rejected from local
host

Also, in /var/gdm in the :0.log file, I found a comment at the end of the
log saying:

    error opening security policy file /etc/X11/xserver/SecurityPolicy

Strangely, this file does exist and has owner/group/world read only access
on it and is owned by root.

I also found some errors to do with XDMCP on the Kea! X side - it simply
says the XDMCP process times out if you leave the blank Kea! X window open.

I've read the Redhat Linux 7.3 page dealing with the GUI displays and the
only error on the windows site I've found and tried these things, eg
http://www.europe.redhat.com/documentation/HOWTO/XDMCP/gdm.php3 and
http://www.europe.redhat.com/documentation/HOWTO/XDMCP/procedure.php3 and
http://www.europe.redhat.com/documentation/HOWTO/XDMCP/refs.php3 and
http://www.europe.redhat.com/documentation/HOWTO/XDMCP/ts.php3 and
http://www.experts-exchange.com/Operating_Systems/Linux/Q_10325200.html and
http://www.freebsdforums.org/forums/showthread.php?s=&threadid=6221&highligh
t=gdm+configuration and and and the list is way long.
