Success!

However, it is a complete PITA, requires 15M of downloading and about 30M of
disk space, requires creating a text file, can't handle spaces in the
filename (hint -- create pdfwrite.rsp in C:\, rather than in Program Files
somewhere, where you'd expect it should be)... Also, when you actually
print, the Save As dialog box comes up behind the active program (which
isn't standard windows behaviour), and apparently, it will come up on the
wrong user's screen if you use XP's fast user switching or Windows Terminal
Server...

Also, it has a different licence to the GPL. I didn't fully read it (I'm
tired), but it looks similar... Could be an issue if someone wanted to make
it handy-dandy for Joe Average.

Anyway, it works... Now, why did I want that again? Ah, yes, I wanted some
tool that Joe Average could use to PDF-ize his (her) resume without
thinking. I found pdf995 and this RedMon/GhostScript concoction -- pdf995 is
easy, and based on GhostScript, but is nag-ware with popup ads; the
concoction is free, non-nagging and completely not friendly to set up, and
has some annoying UI quirks. I don't think I've found the right solution,
but I have a feeling that someone more familar with windows software and the
packaging thereof could whip something up to make the setup more friendly...
I wonder if someone already has... Hmm...
