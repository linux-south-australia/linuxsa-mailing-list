Re: Using mutt was kmail problem.

On Wed, Jun 04, 2003 at 12:57:32PM +0930, Bill - wrote:
> I am having trouble sending mail in mutt. I am trying to point it to 
> ~/Mail/sent-mail but keep getting error messages. This directory 
> exists.  The error message is /home/magpie/mail/outgoing is not a 
> mailbox. All I want to do is record all sent mail.
>
> .Muttrc
> 
> # Setting
> #
> set pager context=4
> set pager index lines=10
> set pager stop
> #set record=+outbox
> set record=~/Mail/sent-mail
> set editor=joe
> #set editor="kedit"
> #set editor="kwrite"
> #set record="~/Mail/outgoing"

I think you need the double quotes - set record="~/Mail/sent-mail"
I've never tried it without, but I'd expect some sort of error.

Then, it seems that mutt won't create a new mailbox if it doesn't
exist already. I think there's a logical reason for this, but I can't
quite put my finger on it. If the file exists and is of size zero,
however, it will happily write to it. So give it an empty file:

  touch ~/Mail/sent-mail
  
and it ought to stop moaning. Sorry, I forgot to mention this.

I'm slightly confused by your error message as reported; your .muttrc
has the reference to outgoing commented out. Did you mean to type
"/home/magpie/Mail/sent-mail is not a mailbox"? Unless there's
another, non-commented reference to outgoing elsewhere in your
.muttrc, your error message as reported baffles me, so I'm assuming a
typo to resolve the conundrum :-)
