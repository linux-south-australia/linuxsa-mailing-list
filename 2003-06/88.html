I don't think so.  I agreed with you.  You must keep up to date with
current vulnerabilities.  I added to what you said, by asking if you

>>> The beauty of the Gentoo and Debian approach is that it is very easy,
>>> therefore I can and I will keep them up to date with the minimum
>>> effort.(Capability).

>> They make it easy to stay up to date if you don't want to have to know

> I _think_ that is what I was saying.

If what you think you were saying is that you don't want to rely on a
distribution to keep track of security issues, then we are in agreement.

If you do want to the distributor keep track of security issues then I ask
if you can trust them.  Almost all of the mainstream UNIX distributors
have shown a previous lack of trustworthiness.  (e.g. c.f. Morris R, The
Internet Worm.)

I trust CERT, I trust what I read on package sites when I go there after
an advisory.  Distributors might also have security upgrades available
for me, but I'm not convinced it's easier to look their than at the
package developers.  Sometimes you have to get your upgrade from the
package developer because you need a really new feature.  I think I'm
ahead by largely ignoring distributions.
