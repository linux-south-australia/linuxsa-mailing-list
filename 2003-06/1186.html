At 2003/6/30 11:53+0930  David Lloyd writes:
>
> Basically, I find it annoying that there isn't a standard "This
> comes with the distribution so its configuration should be in /etc"
> as opposed to "This isn't part of the core distribution so its
> configuration should be in /usr/local/etc".
> 
> I find it more annoying than illogical.

The general rule for /usr, /usr/local and /opt, is that if a package
is:

- supplied by the distribution then it goes in /usr
- compiled up locally by the admin then it goes in /usr/local
- supplied by an ISV then it goes in /opt

So, a package can be installed in 3 different places depending on who
built it and it doesn't matter if its considered to be part of the
"core distribution". This sort of separation can be useful in that it
helps avoid locally built software for being overwritten by
distribution upgrades.

btw /usr/local/etc isn't mentioned by the FHS, though a request has
been made for /etc/local (which matches /etc/opt), though
/usr/local/etc seems to be the tradition/estabilshed practice which is
what the FHS tries to keep in line with.
