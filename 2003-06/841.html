I think I may have been misunderstood.  You wanted a scheme where the
authentication was transparent, like MS Proxy.  What I was talking about
was NTLM authentication between the *Browser* (read: IE) and Squid,
which then in turn authenticates against a Domain Controller.  IE passes,
if requested, the user's stored credentials across when challanged[1], and
will only prompt the user if these details are incorrect.  I have this running
at a site with the users partitioned into groups based on username, and
it works a treat.  It even supports multiple NT domains, which this site
also has.
 
 
[1]  IE used to do this regardless of who challenged it, so it was popular
     exploit to have a web server out on the 'net snarf NTLM hashes.
