Re: Scripting Problems

On Wed, Jun 18, 2003 at 12:56:04PM +0930, Martin Stacey wrote:
> I wrote a script and it works fine when I run it from the command line as
> root.
> 
> However, when the script is execute by cron for root, the following error
> message appears;
> 
> /usr/local/sophos/mmsmtp-1.2.2/Linux-x86/bin/mmsmtp.out: error while loading
> shared libraries: libsavi.so.2: cannot open shared object file: No such file
> or directory
> 
> Why would this occurr?

Stab in the dark: libsavi.so.2 is in a directory (/usr/local/lib,
perhaps) which the dynamic linker does not search by default; your
root login environment has LD LIBRARY PATH set in order to modify this
behaviour.

Try export LD LIBRARY PATH="/path/to/libsavi.so.2" at the beginning of
your script.
