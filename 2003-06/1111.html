It fails the w3.org validator (http://validator.w3.org/) with missing
DOCTYPE and encoding tags.  If you guess HTML 3.2, it gets 45 errors.
HTML 2.0 gets 82 errors, and 4.01 transitional gets 45.

> Seriously, if you don't have any need for fancy GUI tools, and just
> want a simple page, why would you want anything more than Your
> Favourite Text Editor (if you like to edit text using ed, go for it
> -- ed is the standard text editor).

You certainly don't need fancy GUI tools.  They produce pages which
are generally worse.  SCO's complaint against IBM
(http://www.sco.com/scosource/complaint3.06.03.html) is also missing
DOCTYPE.  When you guess it, it comes up with somewhat more errors.  I
won't comment on the fact that its character encoding is windows-1252,
a non-standard.

I use Emacs and HTML tidy, also available from w3.org.  And yes, I
started using it because people pointed out all the errors in my web
pages.
