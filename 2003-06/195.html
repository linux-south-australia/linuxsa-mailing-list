Since the original query was from a self-confessed linux newbie and
redhat 7.x shipped with both ipchains AND iptables I merely provided as
complete instructions as I could to ensure that he could ensure that no
previously defined firewall possibly using ipchains was running. You
need to make sure ipchains modules are not running before you unload
them. I also gave complete instructions on how to do so and then run an
iptables firewall instead. For the few minutes (or seconds) it would
take to do this I believe your perception of an unprotected system is
unimportant.

Yes, redhat has tools for starting/stopping and getting the status of
every service on the system via the command line. The service command is
such a tool. Every distro has it's own quirks and variations but redhat
appears to have gone down this path towards user friendliness. Actually
the service command is rather intended to be used by the redhat gui's
rather than by command line junkies, but it is useful for newbies
getting to know CLI and is quite safe. Much safer than trying to call
the /etc/init.d/ scripts directly or worse, starting the daemons
manually without the aid of scripts at all.
