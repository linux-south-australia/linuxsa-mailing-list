Not a good idea - you shouldn't use the mysql root user for user applications,
and you certainly shouldn't leave the password blank.

To set the root user's password, from the shell, you can:

$ mysqladmin -u root password 'password-here'

So, if you wanted to set the root password to 'suse', you would:

$ mysqladmin -u root password suse

Note, anytime you use mysql now as root, you need to specify the "-p"
argument, which will prompt you for the root user password.

Adding a new database is quite simple; you can either use phpmyadmin, or do it
via the mysql console:

$ mysql -u root -p
Enter password: <enter root password here>

mysql> CREATE DATABASE newdatabase;
Query OK, 1 row affected (0.01 sec)

Adding a user/password (say, 'james', with password 'leone') to access that
database is also quite simple:

mysql> USE newdatabase;
Database changed

mysql> GRANT ALL PRIVILEGES ON newdatabase TO james@localhost IDENTIFIED BY
'leone';
Query OK, 0 rows affected (0.08 sec)

(Note; we use james@localhost, because any php/perl scripts you're running
will access mysql locally).

That's all there is to it - you now have a database called 'newdatabase',
username 'james', password 'leone' :)
