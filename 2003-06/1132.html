Re: Some Proposals for a Linux of The Future (tm) :)

I haven't really been following this thread, but:

> No I don't want to move user configs out of $HOME because it makes 
> administration harder. Often the /home partition is NFS mounted (or 
> at my Uni, /home/$USER is automounted via NFS on the file server 

This brilliantly demonstrates why one "perfect" unified Linux distro is a pipe
dream, IMHO.  The freedom of systems such as Linux won't ever allow one single
accepted way of doing things; everybody has their own concepts of 'the right
way', and you'll never, ever convince some people.

That said, there are problems with the 5 billion distrib's out there all doing
things their own way; it's fortunate in some ways most flavours have been
based off some of the more mainstream ones (Redhat/Debian).  It's good there
are moves to unify things to some point, but some concepts seem way too
optimistic.
