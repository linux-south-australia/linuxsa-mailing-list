Re: gentoo vs freebsd

On Sunday, June 1, 2003, at 12:34 PM, Richard Russell wrote:
> I understand that FreeBSD 5.0 brings major improvements, though I
> don't know if that means it's equal to Linux.

FreeBSD 5 is apparently quite unstable so it wouldn't be a 
consideration for servers at this point.

> * Practically speaking, how well does the FreeBSD cvsup and ports 
> system
> work? How much attention does it need to keep up to date?

Practically speaking, it would be better to focus on scripting 
installation and configurations, rather than relying on update 
mechanisms. This applies to all machines, no matter their OS or method.

> * Assuming I had a machine to test the process on first, how easy 
> would it
> be to automate the process of updating everything on a number of 
> machines
> at once (perhaps using my own repository mirror, so as to maintain a
> consistent state to update to).

I'm automating the process of installing everything on a number of 
machines. It will effectively function as an update, but without the 
build up of cruft and the implied faith in hundreds of little updating 
scripts getting everything right.

> * Can I create a package that consists of little more than config files
> and dependencies (whether as normal dependencies, or a canonical list 
> as
> described above), and install that on a machine, and have the automated
> update process respect that?

Yes.
