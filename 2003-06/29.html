Re: gentoo vs freebsd

On Sunday,  1 June 2003 at 11:30:59 +0930, Richard Russell wrote:
> On Sun, Jun 01, 2003 at 09:04:13AM +0930, David Fitch wrote:
>>
>>> It seems that FreeBSD includes things like sendmail in the base
>>> system, which is something that irritated me with Debian (though Debian
>>> installs exim by default). I want to choose my own MTA. Likewise with
>>> things like bind... (why IS that -- it seems somewhat arbitrary that
>>> these things should be included in FreeBSD's or Debian's base
>>> repository?)...
>>
>> cos you need an MTA of some sort?
>
> Perhaps you do... still, it would be nicer to let you choose (which,
> in Debian's case, should be close to trivial, and in FreeBSD's case,
> not quite so easy, as the MTA's are part of a different installation
> method...)

FWIW, we've had a lot of discussion about whether to keep sendmail in
the base distribution, and we're still not in agreement.  As David
points out, the system needs an MTA for normal function (for example,
cron sends you its output via mail).  The alternative MTAs in the
Ports Collection should all replace sendmail.  When building the
complete system, you can also exclude sendmail by setting the
-DNO_SENDMAIL variable.  I don't think there's any way to do this when
installing from CD, however.

>> yes you get exim with debian, I prefer postfix so I do:
>> "apt-get install postfix" after installation to replace it.
>
> Yes, I do similar, but it's still annoying. FreeBSD doesn't appear
> to give you a simple way of uninstalling sendmail.

  # cd /usr/ports/mail/postfix
  # make install

That does it for you.
