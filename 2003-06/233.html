Re: Mandrake 9.0 vs 9.1 MPlayer and Xine

I was in the exact same boat as you not long ago. My mandrake 9.0 system was 
set up beautifully, I didn't want to change a thing. However, I wanted to try 
out the new KDE 3.1 and see what it was like and also see what new features 
mandrake had included, so I decided to give it a whirl.

I chose the Upgrade option rather than a fresh install, and it was probably 
the most brain dead easy thing I've ever done. It asked if I wanted to 
upgrade or install over the top of mdk9, I said upgrade, then it just chewed 
away on installing packages for about 1 hr (had one error with timidity 
package conflict), then I rebooted... it kept my mandrake 9.0 kernel as 
default so my nvidia drivers and what not still worked fine, i booted into 
KDE, it kept my theme, my settings, all my icons, the only difference was new 
software!! yay... I then proceeded with my usual tweaks (e.g. freetype 
package from plf) and everything was sweet... I configured it so the mandrake 
9.1  kernel was default and I reinstalled the nvidia drivers (there was no 
difference, but I thought I might as well run the latest).

I have to say, I am very very happy indeed with my current system setup, I 
like the new KDE 3.1 (not much different though), it has drop menu shadows 
(looks sweet!), and fading panel applet handles.. konqueror launches a bit 
faster.. the mandrake control centre is a bit better... but other than that, 
not a lot of difference.

I am very excited about what the next Mandrake will probably look like, kernel 
2.6.x, KDE 3.2 (with proper randr support), mmmm the future is grand!

Cheers,

Damien

P.S: install kmplayer, it's a very good front end to mplayer, also install the 
win32-codecs package from plf. With kmplayer installed you can play movies 
within konqueror! (it embeds into the konqueror frame), pointless but cool :)
