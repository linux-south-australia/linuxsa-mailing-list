You could set it up as a transparent proxy, although it would need some
rules added to your firewall/router to redirect all web traffic through
your proxy.  The disadvantage to doing it this way is that your accounting
will only show IP addresses not usernames, so it's a little difficult to
pin net abuse on an individual as they can just say that Fred Bloggs was
using their PC then.  You'll have the same issue if you do it by machine
name.  

I guess it comes back to why you are using a proxy in the first place,
if its purely as a caching/traffic saving feature, then why bother with
user authentication at all.  If it's for logging/reports for management
then you need user authentication and poor old userland will just have
to be content with clicking an OK button when they surf the web.
