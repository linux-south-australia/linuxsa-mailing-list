Actually, the 'snapshots' feature in some more recent filesystems looks 
really handy for file recovery in the event of deletion or accidental 
modification. Some implementations will even allow a user to retrieve a 
'deleted' file without sysadmin intervention.

The system (at least the proprietary one I've been playing with over the 
last few days) doesn't use any additional disk space for unchanged files 
besides their filesystem entry, it simply points to the same blocks on 
disk as the original file. If the original is changed, it writes the 
changes to new blocks, so the snapshot still points at the old ones. 
Pretty cool, huh?

Some brief detail is at 
http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/snapshots.html
