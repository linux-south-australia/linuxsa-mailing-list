Hello All,

This (perhaps windowzy) question might not be suited to this forum, however
I'm hoping the gurus of the SA open source community may be able to shed
some light on this issue.

We have a windowz machine that has Cisco VPN client software on it. This is
used to dial in securely to some of our clients. Cisco installs a
deterministic network enhancer (DNE) protocol with the normal TCP/IP on the
dial up adaptor and any network adaptor. It appears (from our Sygate
firewall) that it is doing a lot of network traffic action out the dialup,
even when the VPN is not in use. Any attempt to disable DNE whilst not in
use, causes the network & web browser not to work. If we block the DNE via
the firewall, besides bogging the system down, it's the same thing. Upon
closer inspection, the DNE is connecting to various IP addresses like
these...

193.0.0.129 = RIPE Network Coordination Centre in Amsterdam!!
195.129.111.4 = same
210.80.58.42 = Asia Pacific Network Information Centre in Milton, QLD.
(after using whois at http://ws.arin.net/cgi-bin/whois.pl)

Why is this so? The bandwidth this machine uses is precious, what data is
being sent & why? Why is the protocol connecting to these & other various
IP's? We are suspicious & are not sure what's going on. Can anyone shed some
light on this?

Your constructive replies are greatly appreciated.
