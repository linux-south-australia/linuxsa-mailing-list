As Richard pointed out the LSB Sample Implementation does what I think
Damien is asking for. Its built from source using Linux From Scratch's
ALFS software so you can see the various patches and compile flags
used. Note that its a "sample implementation" not a "reference
implementation" - where the sample implementation contradicts the
specification, the specification is the authorative source.

As for a standards body, there is the "Free Standards Group"
(http://www.freestandards.org), which has a number of workgroups -
eg. the LSB, internationalization, LANANA. The FSG is a community
driven, non profit organisation.

A lot of what has been discussed is covered by the File Hierarchy
Standard (included by the LSB), which is currently undergoing review
for a v2.3 release.

> >
> There needs to be a group that is accepted by the comunity at large to 
> set these standards.
> 
> And the standard MUST NOT be Linux specific.

Contrary to its name, the LSB is not intended to be exclusively Linux
specific. The intent is to allow other operating systems to be capable
of being LSB compliant. We've had input from other operating system
developers into the LSB specification.

> Now, how do we get a respected and recognized body in place?
