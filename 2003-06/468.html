RE: [OT] CPU Speeds

"VERY inefficient" at what, though?  

It all depends on the application.  It can probably be fairly safely
argued that a P4 chip or AMD Athlon is more efficient powering
3D-intensive applications (due to complex, specific 3D instructions) in
comparison to an UltraSPARC chip (VIS instruction set - a type of RISC).
However, for sheer number crunching power (such as compression for eg),
a 900MHz UltraSPARC III will whip a 2.4GHz P4.  The same can be said of
the benefits of high FSB clock speeds, L1 cache sizes etc etc.  It all
depends on the application.

My 2c.

CPU speed itself is not that accurate. Processing capability relates to
Raw CPU Speed / Front side Bus speed and architecture ( ie Intel or
Power PC ) and type of instructions CISC/RISC.

You would be suprised at how low the "Mhz" speed of some really high end
computers. X86 Intel based chipsets are VERY inefficient
