Re: Scripting Problems

Martin Stacey wrote:

> I wrote a script and it works fine when I run it from the command line as
> root.
> 
> However, when the script is execute by cron for root, the following error
> message appears;
> 
> /usr/local/sophos/mmsmtp-1.2.2/Linux-x86/bin/mmsmtp.out: error while loading
> shared libraries: libsavi.so.2: cannot open shared object file: No such file
> or directory
> 
> Why would this occurr?

Because your login environment is different from the environment that
cron sets up.  A QDF[1] is to import the appropriate root profile in
the script (e.g. ". /root/.bash_profile" for bash, "source .cshrc" for
C shell, etc.).  A better fix is to run the "env" command as a logged
in root, run env out of crontab, and compare the outputs.  My rough
guess without further information is that you have a custom
LD_LIBRARY_PATH.
