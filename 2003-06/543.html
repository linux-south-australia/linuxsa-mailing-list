Re: Wireless Access Point - 802.11b

> I bought a SVEC WLAP AP. It uses the ATMEL chipset and thus works 
> with ap-tools (see freshmeat).  Thus, you don't need to use the 
> USB/Windows config port at all.  :)

Hm,

Aren't AMTEL chipsets those common programmable <20mhz processors you see in
many, many devices (including iPaq's, etc), due to their cheapness?  Therefore
wouldn't it mean that its the software running on that chipset, and not
specifically that chipset that makes it supported by those utilities?

A lot of the hardware AP's seem to be configured via SNMP, via both public &
private keys, which would make them only dependent on snmp tools (which is why
many are horribly insecure as per default - not only do most people leave the
private key as per default, they leave the public key as per default too,
which may seem harmless, but can be used to sometimes read the private key).

(I don't like hardware AP's, every single one I've used has crashed in some
way under certain heavy/induced network load, many seem to be incredibly
flakey when it comes to outdoor long distance wireless connections, and a lot
seem to have incredibly stupid security issues.  Wish Dlink still made the
DWL-520's, grumble...)
