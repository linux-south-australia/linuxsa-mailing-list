> The insight you need is that IPP printing protocol used by CUPs
> works using URLs, there are differing URLs for printing and admin,
> and CUPS' access controls effect access to URLs.
> 
> So you can have differing policies for ipp:/printers/...
> URLs and a ipp:/admin/... URLs, using pretty much the same
> syntax in /etc/cups/cupsd.conf as in Apache.

Yep. I had a play yesterday afternoon and it appears as though you *can*
set CUPS to authenticate before allowing access to a particular machine
and that it works when printing from the command line on either Linux or
Apple OS/X, but when you try using a graphical application (such as
Evolution or Mozilla), which typically use lpr to print, lpr and the
application hang (presumably waiting for a user password). The Apple
applications seem to just fail and retry at a frequency that causes
CUPSd to assume a DoS attack and turn itself off.

> Access control is described in the manual.

The manual describes how to set it up, I needed to know whether it would
do what I wanted.
