> I also never realized that you have to start X before running
> startkde.

You should have a .xinitrc file in your home directory.  The last line
should be 'startkde'.  For example, this is what I have in my .xinitrc
for KDE:

   kbdcontrol </dev/ttyv0 -r fast		Set keyboard rate
   xset m 6 2					Set mouse rate
   xrdb -load .Xdefaults			Load defaults (if you have them)
   xmodmap .xremap				Map ctrl and alt keys as God intended
   startkde					Start KDE

.xremap looks like this:

   keysym Alt_L = Meta_L Meta_L
   remove Lock = Caps_Lock
   remove Control = Control_L
   remove Mod1 = Meta_L
   keysym Control_L = Meta_L
   keysym Caps_Lock = Control_L
   add Mod1 = Meta_L
   add Control = Control_L

It puts Ctrl next to A, Alt in the bottom left corner, and CapsLock to
the left of the space bar.  I believe there are more modern ways to do
this nowadays, but this has worked for me for 10 years, so I'm not
fixing it.
