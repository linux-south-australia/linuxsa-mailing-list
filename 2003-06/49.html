Re: Apache's Default Return Type

On Mon, Jun 02, 2003 at 11:18:01AM +0930, David Lloyd wrote:
> 
> Is there are reason why most of the distributions in their default
> apache configuration have "text/plain" as their default mime type rather
> than "text/html"?


David,

Once I asked a sysadmin to change the default to application/octetstream
or some such thing, and after 5 minutes I begged to have it put back to
text/plain. It's a really good idea if you happen to have heaps of
files of the form *.c, *.h, *.tex, and other such stuff.
But I guess the old .htaccess file is there to get particular
directories out of a jam.
But personally, I can't see any earthly reason why an HTML file would
not have an .html or .htm extension. To interpret all unrecognized
extensions as text/html just to catch a few mistyped file extensions seems
like a bad idea really.

Anyway...
