Re: Some Proposals for a Linux of The Future (tm) :)

I don't think we really want a Unified Linux Distribution, we just want all 
Linux distributions to be able to work together and for all software to 
install and operate correctly on any of these distributions that conform to 
standards.

The whole reason I started thinking about this was because of the number of 
distributions based on Debian that can easily install software from Debian's 
apt-get repositories. Being based on a stable base allows this sort of thing 
to happen. I started thinking,  if Debian was the "official" linux that 
everybody based distributions on, then everybody could install software 
designed for it.

I think that "best practices" and standards are needed to ensure 
interopability and less headaches when working with various GNU/Linux 
systems. (And also to stop software from being developed for "Red Hat Version 
Y.X", and instead for "LSB compatible distribution version 1.3 or higher").

Cheers,
