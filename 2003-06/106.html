Re: gentoo vs freebsd

On Monday, June 2, 2003, at 07:53 PM, davidn@rebel.net.au wrote:
> Will it tell you everything that *needs* to be updated, or just what
> they've got fixes for?

Gentoo developers don't fix anything. They just write build 
instructions.

The build instructions usually don't change much, if at all, so you 
simply run the ebuild over the updated code from the developer.

Some ebuilds contain patches (eg, the custom kernels) and fixes, so a 
Gentoo developer can add a security fix if the original developer is 
slack or the patch wasn't backported.

To answer Michael:
> So what happens when I patched something in the past (security patch or
> perhaps a compile option specifically for this package) and then
> subsequently run emerge -p world. Are my changes identified in any 
> way, or
> do they get clobbered?

You can set it up either way. Gentoo co-exists with hand-installed 
software nicely, but ebuilds are so easy that it's worth taking 
advantage of some of the packaging options.

On the other hand, the continual compiling of source code does get a 
bit tiring, especially when you're trying to show people a range of 
things that aren't already on your system.
