> 
> I hava a network that I want to try some trippy stuff with.
> 
> I currently have a Linux box acting as a PDC, the LAN consists of 50 odd Win98 
> Machines. The PDC and clients run flawlessly, however Win98 printer ques are

Networking in the Windows print system has always seemed to me as though
it was an afterthought.

> So I want to set up three Linux print servers, at the moment they are all 
> running Samba and sharing directly to the LAN. I want to do it differently 
> though.
> 
> Is it possible to set the print servers as CUPS to the PDC, and share the 
> printers using Samba on the PDC?

Yep. Samba will even be able to print directly to CUPS, instead of
through the BSD or SysV compatibility stuff.

> If this is possible can I then log the print jobs using the PDC, with 
> usernames and pages printed?

Can't remember specifically whether you'll see the usernames in CUPS'
page_log when printing through CUPS via Samba, but it certainly has the
capability. Try it.
