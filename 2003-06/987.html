Yep. It's an access control thing. in /etc/cups/cupsd.conf, you'll
notice a bunch of '<Locationy...>' directives for various directories
within the CUPS configuration system (it's just the path bit on the end
of the URL in your browser).

It'll probably have a line such as 'Allow from localhost' and the
default is to deny all that's not specifically allowed. You'll probably
want to add a line for the host you want to connect from. For example:

  Allow from 192.168.0.1    # Allows from a single host
  Allow from 192.168.0.*    # Allow from an entire group of machines

Just make sure you don't allow more than you need and you should be
fine.

You can then commit those changes with:

  service cups restart
