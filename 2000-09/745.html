RE: making a boot disk

Hi Michael,

The DOS fdisk command will not erase the MBR unless
the "/mbr" switch is used.

Just because lilo remains untouched does not mean
linux would still boot, especially if the linux
partition is placed on another drive.

Lets assume the primary-slave hard drive had the
same partition structure as the original primary-
master hard drive, and the original boot partition
was hda4 (for example).

The new boot partition should be now hdb4, which
does not correspond to the original settings in lilo.

I'm sure the other linux gurus will correct me if I
am wrong.

Regards,
Richard Watkins
