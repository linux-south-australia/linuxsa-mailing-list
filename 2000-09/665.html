Re: Viewing open files on Samba shares

"Freeman, Peter (ERHS)" wrote:
> 
> Does anyone know of some good tools for administering open files/ active
> users etc on Samba shares.
> I'm setting up a couple drives on a Linux box for file sharing which will be
> mapped as a drive in our main NT server logon script.
> I'd like to have the same, and better preferably ;-), functionality for
> monitoring open files/users that the NT server manager has.
> The SWAT tool gives a nice list of open files, but this would get very
> cumbersome with the number of users we have, and it doesn't give you the
> ability to kill off individual files/users.

I use lsof. It's for Linux files, but if you do, say something like:

lsof |grep <unix_path_to_samba_share>

You'll get a list of all the files that are open by Samba.

You could probably use hack up a script to help automate some of the
tasks you'd do often.

Hope it helps,
