Re: controlling which modules load at boot

On Fri, 8 Sep 2000, Jayson Hay wrote:

> Just wondering where I can configure the modules that load when I
> start linux. I have my paralel port zip disk working but need to load
> the ppa module before I can use it. Any hints? I tried editing
> /etc/modules/default but no joy. I've managed to get everything else
> working like samba, perl, apache & stuff but this has me buggered.

You can load any module you want in /etc/rc.d/rc.local or the more eligant
way is to get the kernel to load the modules for you. When accessing the
device without a module loaded you will probably get a error. If the
kernel knows anything about the device chances are you'll get an error in
the log files along the lines of: unable to load module 
char-major-somenumber-somenumber. Find this line then add a alias for the
correct module in /etc/conf.modules (or /etc/modules.conf - which ever
already exists). Ie add a line: alias char-major... ppa 
Upon next use of the device when the module is not loaded, the kernel will
load it for you, cool heh :)
