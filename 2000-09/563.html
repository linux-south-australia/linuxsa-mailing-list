Re: LinuxSecureFest

Hi,

Well, the suggestion of a Security Fest seems to be popular, in that there
have already been 22 people respond expressing interest.

I think that the next steps are to:

  1. Explore the form that this should take. I suggest that we do that on
     the LinuxSA mailing list for now.

  2. Set up an organizing group to get it going

  3. Just do it.

With respect to 1, here are some initial comments:

  1. Have fun.  We should actually launch some of the attacks we may
     have been on the receiving end of.

  2. Should this be Linux specific, or inlude all forms of UNIX, or 
     include Windows NT?

  3. We should have sessions for the newbies as well as those who are
     perhaps more experienced.

  4. It should be an opportunity for hands-on as well as for being
     presented to.

  5. Should we have a cracking challenge?
