You'd need to be able to turn it off though. I mean, whilst it would
help prevent most people from leaving systems wide open, what happens if
you have a box that is running very little (like a firewall). You "know"
that this machine has no security problems, because it isn't running any
services and you are quite sure that the kernel running on it has no
problems. Updating this machine could compromise the security for your
entire network. It would take a simple slip by a kernel developer to
introduce some kind of remote buffer overflow and end up in your system.
OK, this would only be a problem for a maximum of six months, but that's
six months too long.

I know this is rarely going to be the case, but that's what makes linux
so great, it works in more situations than just about anything else.
