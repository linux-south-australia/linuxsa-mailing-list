Re: Boot log query (newbie)

Paul Malcolm Bailey wrote:
> 
> Hi.
> 
> In /var/log I've got the following boot log files:
> 
> boot.log
> boot.log.1
> boot.log.2
> boot.log.3
> boot.log.4
> 
> If I do a file test on boot.log (e.g. "file boot.log") I get:
> 
> boot.log: empty
> 
> So too, on boot.log.1
> 
> boot.log.1: empty
> 
> But if I do file test on boot.log.2, boot.log.3 and boot.log.4 I get:
> 
> boot.log.2: ASCII text
> boot.log.3: ASCII text
> boot.log.4: ASCII text
> 
> So how come boot logs 2-4 get written where boot log 1 doesn't?  And how
> come boot.log doesn't get written either?  Do the boot logs get written
> in reverse order or something?

Each time you boot, various messages (I think only the bad ones) get
logged in boot.log. Every so often, your log files will be rotated
(boot.log, becomes boot.log.1, etc).
