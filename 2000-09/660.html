Re: samba client for backing up to an NT box

Jonathan Churchill wrote:
> 
> Hello folks,
> 
> I would like to backup various linux workstations to a tape drive on an
> NT box. I have heard you can run a samba client on the linux machines
> and mount the smb share. Does anyone have any experience/thoughts on how
> well this works ?

You can do this a couple of different ways.

1	You can have the linux workstations use samba to mount a share on the
NT box and copy it's files to the NT box. Whilst I wouldn't (and don't)
do it this way, you could use this command to mount the share:

	mount //NTSRV/SHARE /mnt/remote -o username=mrman password=secret -t
smbfs

or the way we do it:

2	Configure Samba on the machines you want to back up and share out the
stuff to back up. If your software allows you to specify paths or browse
network browse lists, you can tell it to grab stuff from a UNC pathname
(\\SERVER\SHARE type thing). If not, you can just map the drives (set up
a batch file to map them all for you) and back up that way. There are a
number of reasons why I'd do it this way. Let me know if you want them.
