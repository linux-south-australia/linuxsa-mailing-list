Re: Possible break-in

Sam Silvester wrote:

> So I had a close look through my logs and found this:

Good habit to get into.

> Sep 17 04:04:19 phydeaux PAM_pwdb[1359]: (su) session opened for user news
> by uid=0)
> Sep 17 04:04:20 phydeaux PAM_pwdb[1359]: (su) session closed for user news
> 
> So am I looking at a possible break in???

No; check crontab for an entry that (most likely) has something like
"su news" or just news as the user to run as.  What's happening is
that a process is dropping privilidges from root to run as the user
news.

> I've just had a look at /etc/hosts.allow and /etc/hosts.deny and they are
> still as I want them. Disallow everything and only allow ssh.

Remember that /etc/hosts.{allow,deny} are for use by tcp wrappers, and
are usually set up for programs started by inetd;  ssh isn't normally
run from inetd, so anything in hosts.{allow,deny} won't make a scrap
of difference to sshd.
