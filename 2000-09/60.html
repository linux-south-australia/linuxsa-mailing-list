Question:

While trying a Partitionless Install to an existing FAT32 partition, the
installer fails with an Anaconda error like this one:

     Traceback (innermost last):

        File "/tmp/updates/todo.py", line 1112, in doInstall
          self.fstab.mountFilesystems (self.instPath)
        File "/usr/lib/anaconda/fstab.py", line 428, in mountFilesystems
          isys.ddfile("/mnt/loophost/redhat.img", self.loopbackSize)
        File "/usr/lib/anaconda/isys.py", line 52, in ddfile
          _isys.ddfile(fd, megs)
     SystemError: (2, 'No such file or directory')

What can I do to avoid this error?

Answer:

This error is caused by the Red Hat Linux installer attempting to make a /
("root") filesystem image larger than 2047MB in size. The FAT filesystem has a
2GB file
size maximum, and our Partitionless Install is attempting to exceed this
limitation and is failing.

To avoid this error, simply choose a / ("root") filesystem size of 2047MB or
less. The maximum size needed for even the largest Red Hat Linux installation is
1.7GB,
so you will not run out of space.
