Define what security you have if you have no firewall. You stated
yourself that a firewall will close ports a human has switched off. A
default RedHat Installation (6.2 but others as well) will allow "telnet"
from anywhere and possibly even "anonymous ftp". If a new user had
failed to check inetd but blocked incoming telnet and incoming ftp using
this, or another firewall, they are more secure.

Mind you, they should check inetd's configuration or turn it off during
a normal installation of Linux. I have no doubt someone will disagree
with the above statement. I have given a simple answer to a simple
question; presuming you do agree that installing a firewall of some type
can be part of a wider security policy - the context is that you do
agree - then your question is redundant, rhetorical or academic.

Given that you have actually asked me to define what additional security
is gained by merely installing a firewall, I shall twist that statement
into "I [John Brazel] am not entirely certain that merely installing a
firewall will increase security". Without an understanding of how one
can be compromised, lose data and other security issues I would be
inclined to agree with this statement. With an understanding - or a
desire to learn - about security issues, you will learn that firewalls
of various descriptions can be a useful part of a wholistic approach to
security. Even if only to clean up after human ignorance, forgetfulness
or plain laziness.

If you want to dispute the efficacy of adding a well configured packet
filtering firewall to a network:

* install RedHat 5.1 _default_ and RedHat 5.2 _default_ on a network
* connect one of them to the Internet
* tell me the IP

And I'll find the POP2 server crack. Before you do, you might want to
talk to Alan Kennington about that....

Now, if he'd actually blocked all services except the ones he knows he
wanted to use, it would have been slightly more difficult for a cracker
to crack the POP2 server. Probably not impossible, but more difficult.
More difficult to crack has a corollary which is more secure for me.

I assume that you and others will dispute this point too.
