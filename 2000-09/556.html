Unix can trap it; I don't know what NT does.

[dt@nipnyep /tmp]$ cat tst.c
#include <stdio.h>
#include <unistd.h>
#include <signal.h>

void
handler(int sig)
{
    write(1, "Baby did a bonehead thang.\n", 27);
    exit(1);
}

int
main(int argc, char *argv[])
{
    int  x;

    signal(SIGFPE, handler);
    x = 1/0;

    return 0;
}

[dt@nipnyep /tmp]$ ./tst 
Baby did a bonehead thang.

Note the exit() in the signal handler - for floating point errors,
it may give you the opportunity to retry.
