Patrick,

In fact, one of the best ways to put together a linux
security document of that sort might be to
develop it through a....

	Linux SecurityFest

I.e. a day of seminars and demos, with notes prepared on
each topic. And then the notes get worked up into a
nice online document.

E.g. these sorts of sessions:

1.	Demo of how to successfully hack an OS out of
	the box.
	E.g. run various hacks to "get root" on a
	redhat 5.2, 6.0 etc. box, SuSE 6.2 etc.
	This should show it all from the hackers'
	point of view.

2.	The inetd.conf file.

3.	Firewall scripts.

4.	The /etc/rc.d/* files.

5.	Packet logging/interpretation for pleasure and profit.

6.	Diagnostic tools like fuser, netstat, /proc/* ....

7.	IP packet structure and function. TCP, UDP, ICMP...

8.	Survey of tools for automating security, bastille,
	freesco, FreeVeracity, tripwire, etc. etc.

This is the logical progression from an InstallFest, as
I think someone said already.
