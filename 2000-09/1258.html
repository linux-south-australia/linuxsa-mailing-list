We had a similar problem, and it turned out to be Frontpage's fault.
First it was trying to publish the web using the FP server extensions,
then using FTP. I can't remember what our web guy did to configure it,
but if you can modify files another way, it does take the blame off
samba. You could also try publishing to a local web (stored on your
workstation) and copy the files across when necessary. It can be time
consuming, but it means that I don't have to restore from backup each
time frontpage removes the navigation bars from every single page on the
server and refuses to put them back....
