Ummm, no. The source port is chosen, probably monotonically increasing,
from ports above 1023 when a client places a connection request.

In a client, when you create a socket, you bind it to an address. You
select INADDR_ANY for the local IP and port, with the Dest IP and port
specified by you.

The TCP layer allocates one of the local IP addresses as the source IP, and
selects a local port as the source port, selecting the next available port
number above 1023.

There are cases where you want to use all four of SrcIP, SrcPort, DestIP,
DestPort, you really can only do that if you are keeping state. When you do
that, you gain extra security.
