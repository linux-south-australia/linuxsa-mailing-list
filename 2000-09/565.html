GFCC

Hi,

I have looked at GFCC, and apart from it being a little difficult to
operate, my main criticism of it is that it is essentially a GUI wrapper
around ipchains/libipfwc.

In order to do useful things with GFCC, one has to have a very good
knowledge of TCP/IP and how packet filtering is done.

IMO, what is really need is a GTK+-based application (with the GUI possibly
built with Glade) that provides the following functionality:

  1. Allows the user to say what functions are allowed on the system
      - mail to an from
      - DNS requests outgoing and replies
      - Out going connections
      - SSH incoming
      - What ever else
  2. The tool produces an ipchains script that does the job and
     installs the scipt in the appropriate place

     With a script, and expert can eyeball it if need be to check that it
     will do a good job.

  3. Allow for script fragments to be used for various functions

  4. Use good, paranoid, defaults in all cases.

  5. Provide help on the consequences of all actions the user can take
     to change the setup.
