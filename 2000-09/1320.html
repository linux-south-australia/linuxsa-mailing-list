Re: postfix to listen on a different port to 25

David Lloyd wrote:

> > I'm installing a virus scanner on my linux box, and as part of the
> > configuration I have to set it to listen to port 25, and my regular MTA to
> > listen to a different port (say, 5000). Buggered if I can find how to do
> > this....
> 
> How does this make your system more secure? I could nmap you and talk to
> your postfix directly on port 5000 instead of port 25....

Which is why most MTAs allow you to optionally list the interfaces
(or local IP addresses) that you will listen on.  In this case you'd
obviously only listen on the loopback interface (127.0.0.1).

You use the same tactic with forwarding-only DNS servers to prevent
other machines using your DNS server for nefarious purposes.
