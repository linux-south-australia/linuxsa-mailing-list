Jeremy!

> In order to run the workstation you will need to have a valid routable internet
> TCP/IP address.

If you run behind a firewall using ipchains or ipfwadm, unless there is
a special module loaded the outside world generally cannot validly route
or send packets directly back to a machine sitting behind the firewall.
Classic services that die include ICQ, IRC and FTP; all of these have
modules to fool the outside servers that there is a real, valid routable
internet TCP/IP address.

> port number. Everytime it reports a different number ranging between 21000 and
> 23000 so far.

Probably because it's got some algorithm that picks ports between 21000
and 23000.

> Does anyone know any simple solutions to resolving this problem ?

Solutions can vary. You can, as you say below, chuck freesco and install
a real Linux. Rather than using ipchains' forwarding rules, you could
try using ipmasqadm (I think) to perform proper NAT (Network Address
Translation) which may fix your problem.
