Even better. To prove a point. How about we set up several default
installations, then show how vulnerable they are.

We can even run ethereal or other tools and demonstrate why not to use rsh
or telnet (and the magic password is...).

We can run nessus or nmap to demonstrate other issues.

On some boxes we can run some rootshell exploits.

Then, we can show how to protect against these, now that most people would
have an enhanced idea of why to do so.

It would be good to bullet point general philosophies that should apply to
all systems and platforms.

I would like to see covered issues relating to potential vulnerabilities
of hosts behind a NAT based firewall. And discuss a proxy-only
firewall vs a NAT firewall and what is best for what situation.

Would we discuss trojans, for example Sub-Seven on a Windows box and how
it uses a NAT firewall to its advantage?
