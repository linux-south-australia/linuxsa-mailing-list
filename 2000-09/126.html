Re: my first DOS attack

Daryl Tester wrote:
> 
> That's an implementation issue.
> 
> > Cisco routers (about 65% of all routers?) do not enforce this.
> 
> That's a configuration issue.  Cisco provides the functionality
> within IOS to filter packets on all interfaces.  Whether the
> administrator uses it depends on a number of factors.  It's
> always a resource tradeoff issue.

ROFL.  Checking a source IP address costs 32 table lookups in the
worst case, three in the average case.  So it actually saves routing
resources, as subsequent routers don't have to forward the packet.

> > So Cisco can justifiably be blamed for all the world's DOS attacks.
> 
> Just as anyone who puts an unsecured box (using any operating system)
> on the Internet, from temporary dialin to permanently connect.

Cisco and clueless or lazy ISPs.  Cisco is under siginficant pressure
to alter their default.  The Australian Governement is likely to
sponsor the development of a Technical Standard for Carriage Service
Providers that forces them to check source IP addresses.

> This is an issue with most routers, not just Cisco.

Not so, most other vendors have changed defaults in recent software
releases.
