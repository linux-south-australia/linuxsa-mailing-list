Re: putty with public key authentication

On Wed, 27 Sep 2000, Alan Kennington wrote:

> But with the public key which I send to the artist,
> there seems to be no way to get the key into the software.

The putty author wrote in
http://lists.tartarus.org/pipermail/putty-bugs/2000-January.txt:

  > A possible way to use public/private keys under windows is for putty to
  > hold them (securely) and require a password (phase whatever) when use of
  > the key is required.

  That buys me nothing over using password authentication: I still
  have to type a pass(word|phrase) in order to log in! For me, the
  purpose of key authentication is to avoid this - to avoid having to
  make an effort to authenticate at all. Naturally, I can only do this
  if I consider the machine I connect from to be trustworthy.

  My assessment of public key authentication still stands: if somebody
  else implements it, I'm willing to accept it, but I won't make the
  effort myself because it's no use to me.
