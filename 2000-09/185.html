Re: Extracting a CD's contents to straight iso image?

On Tuesday,  5 September 2000 at 17:44:28 +1030, Justin Hawkins wrote:
> On Tue, Sep 05, 2000 at 04:22:29PM +0900, Richard Sharpe wrote:
>> At 03:55 PM 9/5/00 +0930, Ian Hogben wrote:
>>> Hello, all:
>>>
>>> I've been looking around, and I can't really find an application that will
>>> make a "disk image" of a CDROM. I wonder if it's possible. I'll explain
>>> myself a bit better, though...
>>>
>>> Around 8 years ago, I used a few utilities that did bit-by-bit transfers
>>> from one floppy to another. Didn't matter if it was a Mac, amiga or DOS
>>> floppy, because it copied the format information with it. Is there something
>>> like this for CD-ROMS? I'm wanting to do a straight "don't ask any
>>> questions" dump from a CD to an image file on disk that I can then send to a
>>> CD burner to record onto a new CD.
>>>
>>> Any ideas welcome. :) Thanks!
>>
>> Ummm, standard UNIX. dd the image off, and use cdrecord to write the image
>> back:
>>
>>   dd if=/dev/cdrom of=mycd.img
>>
>> What could be easier.
>
> I have had this not work on occasion. Not to say it doesn't usually
> work, but it definately has failed for me on least one occasion -
> and in that case the CD was successfully duplicated using a piece of
> Windows software :-(

You can always make mistakes.  But it would be incorrect to blame dd
without some really hard evidence.  dd is so reliable because it's
low-level: unless the CD-ROM driver has problems reading the CD-ROM,
it will work.
