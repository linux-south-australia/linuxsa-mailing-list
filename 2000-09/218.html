Re: VMware

I have been running Vmware at work under redhat 6.2 with both 2.2.14 and 2.2.16
for some time now, however my success seems to have been much greater. However,
I have noticed a couple of things I'd reccomend doing to get decent (usable
without too much annoyance) performance.  Firstly, ensure you have at least
128mb of ram in your linux box.  If you have 64, forget it.  I have compared it
to a mates machine at work and with 64mb it's unbarable.  I have 128, and run it
24/7 and get normal performance in linux, and passable in windoze.  Secondly
make sure you have dma enabled on your hard drive in linux, and also in
windows.  Even though its emulated hardware, think of how much more emulating
it's gonna have to do to emulate polling the drive!  Make sure you are running
windows 95, not 98! 95 will do nearly almost everything that 98  will (some
things even better in 95 like alphabetically sorting the programs on the start
menu - where did that feature go? oh it must have been a bug...) and use huge
amounts less ram than 98. If you must run 98 then go to 98lite.net and run 98
with the 95 shell. This is what i'm running now on my home machine (well with
xwin 4.11 and a linux box under the desk, displaying on the windows screen,
which if you want to try this I can go into more detail as well).  IMHO the 98
kernel was the stablest there has been, but unfortunatly was totally rooted by
all the bugs that IE introduced! 98 lite is small & stable (ish :) anyway back
to the point...  Having 2 OSes and one being bloatware ram is the biggest
performace factor.  If you dont have enough in windows it'll have to swap to
much and of course swapping on an emulated drive?  iiewwww.... And if you give
to much ram to windows then linux will have the swap problem and start running
like crap.  In my particular instance I'm allocating 32mb to windows 95c, and
running office 2000 apps in that 32mb with quite reasonable performance, leaving
96meg for linux which runs just fine as well (with a stack of apps :).  After
more than 1 office app in windows it does slow down to much though. Incadentally
the machine is a celeron 500.

If you want linux + windows + windows games, i'd suggest starnets xwin32.  I
have it set to auto run an Eterm on start up and then run xwin from startup in
windows.  I have my display environment variable also set in the global bash rc,
so that I can use the terminal just like I am on the linux box with no special
configuration and get x apps on the windows machine too.  It works like a dream.
(although no 3d support, for that you need to use exceed which is the same deal
but from hummingbird software and with 3d support.  However this is the most
stupidly expensive solution there is, and it kinda takes over your system to!)
I'm gradually swapping to linux apps when I find good ones with a view that
someday I wont need windows at all! :) :) :)  Infact, my windows partition had a
bad fat corrutption (yeah the 2 copys of the fat right next to each other with
no check sums got corrupted!) about a month ago and it was heaps easier to
re-install since most my data & apps were still intact & installed on the linux
box :)
