> I have compiled my kernel with support for the K6-2 but Mozilla is running
> like a dog. I've turned off Java but even the menus are slow in responding
> with a more than average amount of drive thrashing.

I suspect you're running a 400MHZ i386. The kernel developers insist on
using -m586 when it should be -march=pentium. -m586 aligns things
nicely, but the code is still fully compatible with the i386 (except it
may be misaligned). -march=pentium also aligns things, but also allows
GCC to use pentium specific instructions.

I would note to everyone that many binary installs of virtually anything
are compiled for i386's. If you install these binaries on your system
you've probably got a very fast, very expensive 386. You are inevitably
going to ask, but "how do I know?":

* if you compiled it from source and used defaults
 - probably yes
 - grep "march" * may find a -march option

* if you got a binary which didn't specify an architecture
 - probably yes

Otherwise you don't really know. The only way to know is to go get the
sources (easy to do with GPL software) and rebuild it with customised
CFLAGS and see whether it does make a difference.
