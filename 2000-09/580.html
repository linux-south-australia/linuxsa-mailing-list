Re: PHP3 and dl's

I have a question with regards to PHP3 and the way it handles
'extentions' (dynamic libraries).

I have some PHP3 scripts that I want to make as portable as possible.
The current 'testing' machine is running PHP3 complied from source,
while the scripts are being developed under Debian.
 
Those clever Debian people have managed to make use of PHP extensions
so that you can install separate packages (just dynamic libraries
really.)  Once installed, you just need to put a call to dl('xml') to
get access to the xml functions for instance.
 
From the PHP3 source though (with .configure) I can only create a
monolithic binary... which doesn't need the call to 'dl', (and
complains if I have it in there) and needs to be recompiled if I need
additional functionality, like gd, for dynamically producing images.
 
Does anyone know how to turn on PHP3 extensions (from the source
code).
