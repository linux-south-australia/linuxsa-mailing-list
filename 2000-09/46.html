Re: TCP port 12345

Alan Kennington wrote:

> Hopefully the new Internet (of which the current Internet is
> not worthy to compute the parameters) will be a little
> safer for home computers! Barbarians!!

Unlikely, as host security is considered a host function.
The end-to-end security argument makes this the only logical
choice.  We already have too many instances of SSL secured
web servers allowing the plaintext credit card details on
disk to be read.

Network security, like limiting DoS attacks, *is* a network
function and one that is done poorly by the current generation
of routers.

What is likely to make more of a difference is decent remote
managment tools for system administration.  Unfortunately,
UNIX is just as bad as Windows in this respect (for example,
if you change mail software from sendmail to postfix you
have to rewrite the configuration file), especially when both
are compared to Novell NetWare.
