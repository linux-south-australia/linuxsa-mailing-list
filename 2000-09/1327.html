Re: postfix to listen on a different port to 25

>> I'm installing a virus scanner on my linux box, and as part of the
>> configuration I have to set it to listen to port 25, and my regular MTA
to
>> listen to a different port (say, 5000). Buggered if I can find how to do
>> this....

Disclaimer: I've never actually used postfix, but I downloaded the source
for a look.

Postfix looks like it might rely on the smtpd entry in /etc/services to
determine the port.

You could try changing that, but be careful, because it might also use that
value to determine the default port used on *other* smtp servers when
sending mail.

It also appears that postfix smtpd can be run from inetd, which makes it
easy to change the port address, or you could go one better and use
something like tcpserver from the ucspi-tcp package (although given the
nature of postfix vs qmail, this is somewhat like putting a ford carburettor
on a kingswood)
