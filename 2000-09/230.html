Re: VMware

Just a further point regarding VMWare.  I have read a couple of
comments on VMWare that installing Windows within the Virtual Machine
is very slow.  This is because (I think) that VMWare uses a Windows
driver to streamline performance for the virtual machine, and
obviously this is only loadable once windows is actually installed. 
When I used VMWare I found that I could use the Windows partition
that I had alrealy installed natively.  VMWare simply detected the
installed Windows and set the virtual machine to use it's partition
as the main partition.

Also, anyone thinking of using wine should check out TkWine if they
have not already.  It's also still in development (it has to keep
pace with Wine development) but is actually quite handy.  It's just a
TK based GUI frontend for configuring and using Wine.  I believe that
the developer is currently unable to maintain the software, but
something this usefull will no doubt be taken up by someone. (Oh and
the project page is currently at http://pet.au.dk/~dailywine/tkwine/)
