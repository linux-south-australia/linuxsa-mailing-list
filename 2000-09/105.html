Robyn,

My personal philosophy on this is as follows:

Firewall all telnet (23), login (513),
and shell (514) ports to the outside world.
Also firewall ftp except for the gateway,
which should have the user account "ftp"
commented out of /etc/passwd so as to prevent
anonymous ftp.

Then comment out telnet from /etc/inetd.conf
(and do "kill -HUP xxx", where xxx is the pid of
the inetd process).

For access between machines _within_ your network,
use the .rhosts file to specify which hosts/users
are trusted.

The reason for this is to make sure that your
passwords don't appear on the local network even.

Then use "rcp" and "rlogin" (or "rsh") without
passwords instead of ftp and telnet.

It seems to me that ftp and telnet can both be safely
consigned to history.
