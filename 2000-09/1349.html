SendMail Woes

I have a slight problem. I've configured my sendmail with:

FEATURE(`access_db')

And added  all of .bar.com to:

/etc/mail/access.dc

(using makemap hash acess < access and an access file of:

localhost RELAY
localhost.localdomain RELAY
.bar.com OK

)

Hence user@squigly.bar.com, noone@foob.bar.com *should*, if I've read
the README in /usr/lib/sendmail-cf under ANTI-SPAM should get delivered
to me. SendMail doesn't deliver any e-mail with an unresolvable domain
but according to the README, adding the domain or part of the domain to
the access database should let it through).

Except it doesn't. SendMail is reporting SMTP Error 550 (unresolvable
domain) even though it should IGNORE that error and continue delivery to
me.

Any ideas?
