/proc/net/dev

Hi all,

I am using the info in /proc/net/dev to feed data to MRTG for some
ethernet stats. The format for this file on most of our servers is:

column one: network interface
Column two: number of bytes received on that interface

This works very well except on two of our machines, which have another
column between col 1 and col 2. This column appears to always be zero,
but it does break my code. Does anyone have any ideas as to why this
extra column torments me so? Is it a kernel option I have turned on on
those two machines and not the others?

to be a pattern between the various kernel versions and this problem.
