I'm not sure how to fix it. To be perfectly honest, my locale broke when
I upgraded/rebuilt my C libraries. Furthermore, I did something that
fixed my perl installation.

Linux, being a nice culturally aware operating system, has the concept
of a "locale". Exactly what is wrong with the noun "locality" I don't
know. Your "locale" can help you get things like spelling correct
(rather minor), to setting different languages. You can set it using one
of the variables listed (in .bash_profile for example) or on a command
by command basis.

Deep within programs, there is a system call (I think) that is called
"setlocale( [various arguments )". Many libraries don't complain even if
this function call fails because there is a default. GDK complains; so
does PERL. Your system isn't broken; it's just "non-US" unfriendly.
