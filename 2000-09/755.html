HOWTO use multiple pointers

Like a lot of notebook users I want to use a real keyboard and mouse when
convenient, or at least a real mouse. This message deals with the mice.
The combination here is a PS mouse for the built-in twiddlestick thingy,
and a serial mouse for the external one, when it is plugged in. I don't
want any reconfiguration when I plug in a mouse, and with suspend mode and
just short trips away from base one session might last for many days or
weeks.

gpm makes this easy. Add the following to your /etc/gpm.conf, and the
init.d/gpm in Debian, Slakware and Red Hat (and many other distributions
no doubt, but I know that these three use the same file) will set it all
up for you:

device=/dev/psaux
responsiveness=
repeat_type=
type=ps2
append="-M -m /dev/ttyS0 -t ms"
sample_rate= 

The commandline that the init script creates for this will be:

   gpm -m /dev/psaux -t ps2 -M -m /dev/dev/ttyS0 -t ms

This takes advantage of gpm's ability to handle multiple pointers, in this
case a ps2 protcol ps/2 mouse and an ms protocol serial mouse. But what
about X Window? 

The -M gpm switch also turns on -R, for relay. In relay mode gpm relays
all mouse movements through a fifo called /dev/gpmdata, using the
mousesystems protocol.

So, just set up your XF86Config file to have a pointer on /dev/gpmdata
with protocol mousesystems (or use XF86Setup to do it in a GUI way) and
you will have a multi-mouse system.

man gpm.
