Re: What Do Pagefaults Mean?

Quoting David Lloyd <lloy0076@rebel.net.au>:

> 
> I was running some fairly intensive programs using the "time" command
> and got the following results:
>
> 6.17user 0.73system 0:07.25elapsed 95%CPU (0avgtext+0avgdata
> 0maxresident)k
> 0inputs+0outputs (258major+312minor)pagefaults 0swaps
>
> What are major and minor pagefaults though?

Unsure of the difference between major and minor bits is (I haven't read the man page), but a
page fault is to do with the exception raised when a program hits the edge of a memory
boundary. The CPU sends the page fault exception the the OS, which then grabs the next page and
sticks it in RAM and allows the running program to continue. This is how Virtual Memory is
implemented (well, sort of, my description is not entirely accurate ;-)).

It's possible that major and minor page faults are to do with what has to be done when the
exception is raised. Possibly MAJOR = from disk, MINOR = nothing happens, causethe stuff is
elsewhere in RAM.

I dunno, I'm not an expert.
