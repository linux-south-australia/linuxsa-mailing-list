>I am trying to start squid without it doing the "dns_testnames" lookup.
>The comment says use the -D command line option when starting squid.
>
>So does that mean I should type ./squid start -D ??
>
>I have tried several variations and I cannot stop it from trying to do the
>lookup!
>
>Any suggestions ?

Add some names to dns_testnames in squid that it will always be able to
find, or comment out the dns_testnames line in the config file. AFAIK, Squid
doesn't use the system resolver (it may be an option in there somewhere) to
lookup names, so putting some hostnames in /etc/hosts isn't a workaround.

Are you using the RunCache script to start squid? You may need to edit it to
add the -D if you are.
