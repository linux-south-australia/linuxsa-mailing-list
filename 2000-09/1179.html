RE: package madness

> dpkg, rpm, etc have their place... if a pretty standard system is all you
> need (ie you don't need to custom-configure much), then they make life
> easier for the masses.

"They make life easier for the masses."  I expect Microsoft use pretty much
the same argument about Windows.  The point about package managers is that
they remove the need for users to know what components software is made up
from, and what dependencies it has.  That is a terrible thing.  There are
some nice aspects to package managers: RPMS includes the unchanged source
as released by the author, and patches it at compile time -- I appreciate
that.  But I do NOT appreciate dumbing down administrators.  Tar is a
perfectly satisfactory way to install packages; and it encourages you to
think at package delete time.
