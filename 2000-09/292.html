Newbie queries about logs

Hi.

(This query pertains to RH 6.1.)

Under /var/log there are a few different text files, and some of them
have got numbers 1 to 4 appended to the end of their filenames, e.g.

boot.log
boot.log.1
boot.log.2
boot.log.3
boot.log.4
cron
cron.1
cron.2
cron.3
cron.4
dmesg
htmlaccess.log
lastlog
lastlog.1
lastlog.2
lastlog.3
lastlog.4
maillog
maillog.1
maillog.2
maillog.3
maillog.4
messages
messages.1
messages.2
messages.3
messages.4

...and so on.

How come they stop at four?  Are the various log files cleaned out every
month, then replaced by blank text files, or do the file numbers
increase (e.g. one day you'd have boot.log.17 or some such)?  Is four
some kind of magic number at work here?

And is there some kind of Linux system log file writer or process (that
knows about /var/log), or do the various processes (e.g. cron) just
write text files and they happen to stick them in /var/log as a sort of
agreed-upon place to write log files?

And if you write a program that writes a log of its activity (such as a
password script that logs all accesses), would you write the results to
/var/log or is this directory set aside for systems programs only?

The questions I've got here might be pretty trivial or maybe dumb, but
as I said:  "newbie queries."
