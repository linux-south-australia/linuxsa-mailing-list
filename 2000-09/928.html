>The .h files in /usr/include/bits have include statements reading

>directory.  I made a link in the directory to 
>/usr/i386-glibc20-linux/include/asm  which contained those files but it 
>still had the same errors.  Not being a programmer, I don't know if the 
>link is a good or bad thing.  Does anyone have any suggestions?

It is looking for kernel header files, which you probably haven't 
installed. If you are running an RPM based distribution then you just need 
to install the kernel-headers rpm. If you are still having problems then 
the compiler isn't searching in the right place for the files. Make sure 
that /usr/src/linux-x.x.xx/include exists and contains the header files you 
need, make sure that /usr/src/linux points to /usr/src/linux-x.x.xx and 
that /usr/include/linux points to /usr/src/linux/include/linux. Finally, 
make sure that gcc has an appropriate -I option such as 
-I/usr/src/linux/include, so that it finds the linbraries.
