Re: Red Hat 7 Install Failure

Hi,

> This morning I attempted to install Red Hat Linux 7.0 on my Dad's laptop,
> after burning the two CD images we grabbed from AARnet last
> night.  However, when the CD boots, after running /sbin/loader, I get an
> error message that states 'failed to read /modules/module-info' and then
> the install exits (after sending kill signals, and unmounting and such),
> so I have nothing left to do but reboot.
> 
> Would this problem be created from an error in writing the ISOs to a
> disc?  I hope not... would there be any way around this (keeping my
> current discs)?

Take your handy boot disk created during installation, or your LinuxCare
CD ROM (available as an ISO form the linuxcare website). Check the
existance and permissions on /lib/modules/module-info. Does it exist. If
it doesn't, then maybe comment all entries in /etc/conf.modules out, and
you might be able to get the kernel to boot. Its possible that mayvbe the
kernel is looking for the modules in the wrong place for some reason. Try
simlinking /lib/modules to /modules

cya
