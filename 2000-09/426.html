At least all of them are documented as found. You just need to keep up

> >

;-)

> A few times in the last year, people have asked me if they
> could have e-mail accounts (using pop, imap or whatever), and
> I told them that I was not running such software because I
> thought it was a serious security hole.

It can be.

> 
> So I was quite surprised to find that I _was_ running such
> post-office software.
> 
> I.e. I was _not_ using POP2 for anything.

Then why not check these things after installation. The first thing I do
(usually before configuring the network interfaces) is comment out 95%
of /etc/inetd.conf. When I started at Xavier College, the previous Sys
Admin had done a default install of RedHat 5.1/5.2 on 3 servers.
Needless to say the buggy UW IMAPd that was shipped with (5.2?) was
hacked within days. If he had spent the time commenting out all the
unnecessary services, there would have been no problem.

Someone else mentioned that Microsoft Systems tend to be easier in that
respect, which is kind of true. You have your core kernel/software bugs
which can lead to great big security holes, but Microsoft is not shipped
with much extra software. To truly measure security between, say, NT and
Linux, you'd need to take into account NT, MS Exchange, MS Poxy server
and IIS.

> 
> I at no time told the linux installation software that
> I wanted post-office software.
> It was just inserted for my pleasure by Red Hat.

Did you do the custom install and select individual packages? If so, you
would have been asked whether you want to install IMAPd. The help screen
says that it contains the POP stuff also. Having it uncommented in
/etc/inetd.conf is bad, but you shouldn't be installing stuff that you
don't need.

> 
> At least the recent SuSE installation routines do ask
> you which services you want installed, and this gives you
> the opportunity to veto the dodgier ones.

Some versions of most dists do this. I think most of them ask about the
/etc/rc.d/init.d/* services but not the /etc/inetd.conf stuff.

> 
> The ultimate cause of this is probably the same as the
> ultimate cause of MS software vulnerabilities.

No. The two are very different. Microsoft software is closed and Linux
is open. You have the opportunity to keep your system clean with Linux,
whereas Windows you don't.

> Namely, they try to give you as many features as possible,
> because they know that _marketing_ success depends on
> everyone being able to find their favourite feature.
> But the downside of featuritis is that you get
> more opportunities for vulnerabilities, the more features
> you've got.

This will always be a tradeoff. Granted, Most things should be disabled
by default, but they aren't. Perhaps you'd volunteer your services to
Someone like redhat to help make sure these things don't happen?

> 
> I think that RedHat have tried to add as much as possible,
> in the interests of marketing.

... and possibly because they use it themselves and like having stuff on
a couple of CDs instead of a bunch of FTP sites. You have the option to
not install some of this stuff, but choose not to. Please don't blame
the software company for your laziness. I actually have a friend who
will unselect during an install and manually install everything else on
top of that. Whilst this is an extreme, he hasn't been hacked ;-)

> Some of the other distributions are more spartan, I believe,
> at the cost of not keeping the customers happy with all
> the lovely software packages.
> 
> This is the problem with the default installations.

Yes, users expect everything to be done for them and make assumptions
about the security of the installed system.

> They're neat in the short term, but then you have to go through

Or leave them out to start with

> 
> Ah... and my main message in my initial over-lengthy post
> was: if you have an old redhat 5.2 machine, rip out the pop2
