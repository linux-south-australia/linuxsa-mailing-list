problems with XFree4.0.1

Hi Everyone,
	I recently upgraded to XFree86 4.0.1 from 3.3.6. From Franz Sirls RPMs. Now the
keymapping for my consoles has gone all screwy - to the extent that if I go to a
console the only way I can get back is to poweroff. All the packages that were
placed at his ftp site were installed  and because my previous initscripts
package was 4.95.2 (Yellowdog CS1.2) I did not do the various incantations
described in the initscripts warning. The
/proc/sys/dev/mac_hid/keyboard_send_linux_keycodes is supposed to be set to 1
during init by /etc/sysctl.conf which is present. So the situation is that I can
have a meaningful keyboard in X or a meaningful keyboard on the consoles, but
not both. I hope someone has a fix for this.

thanks for your time
Dan Kortcshak

initscripts-5.35-1a
> ls /mnt/cdrom/YellowDog/RPMS/init*

> cat /proc/sys/dev/mac_hid/keyboard_send_linux_keycodes
cat: /proc/sys/dev/mac_hid/keyboard_send_linux_keycodes: No such file or
directory
> cat /etc/sysctl.conf
# Disables packet forwarding
net.ipv4.ip_forward = 0
# Enables source route verification
net.ipv4.conf.all.rp_filter = 1
# Disables automatic defragmentation (needed for masquerading, LVS)
net.ipv4.ip_always_defrag = 0
# Disables the magic-sysrq key
kernel.sysrq = 0
# enables new keycodes on PPC
dev.mac_hid.keyboard_sends_linux_keycodes = 1
