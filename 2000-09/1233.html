Re: Limits of grep?

Matthew Tippett wrote:

> Or yet a different alternatative is...
> 
>   find . -type f -exec grep blah {} \;
> 
> Where find gets each file from the current directory (and below) and
> runs 'grep blah' on each file.

You will need the option which prints the filename after a match,
as by default grep won't print that when you run it over a single
file.

For non-GNU greps (which don't have this option), I've usually
resorted to:

find . -type f -exec grep blah {} /dev/null \;
