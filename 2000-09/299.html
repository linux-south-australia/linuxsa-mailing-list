Re: getting an x-session happening

Re: running an X application on a remote X server.


> I was wondering if anyone can tell me how to get a remote x-session
> running. I have a linux box (rh6.1) at home and another at uni I want to be
> able to run an x-session over my dialup (speed is not an issue) I just need
> the gui for an app.

It is all in the DISPLAY environment variable, and a little bit of X
authorisation.  On the machine running the application, you must set
the DISPLAY environment variable.  On the machine running the X
server, you must set the authorisation to allow the connection.

If you wish to dial in from home and run an application from uni on
your home display:


  +	Set X authorisation.

	Quick and dirty is to say

		xhost +krypton.cs.adelaide.edu.au

	This allows anyone on krypton to use your display!  See `man
	xauth` for fancier settings.


  +	Set DISPLAY.

	In your krypton shell set the display address.
	Of course, you might not know what the reachable IP address
	is.  Look at the output of `ifconfig` on your home machine and
	get the "inet addr" field.  For example,

		setenv DISPLAY 192.168.1.12:0.0


  +	Run your application, and wait b e c  a   u   s   e  it will be
	so slow!
