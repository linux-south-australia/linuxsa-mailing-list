Re: Memory cleanup in Linux (from newbie)

you may find that there are actually still processes running, but you
cant see them.

in a terminal window, try this
ps -eaf

this will show procs running. if netscape is still running, try:
killall -9 netscape-communicator

killall will kill all processes with a matching name [1].
kill takes a PID (number) argument, and can do the same

the -9 option means send a SIGKILL signal, which is the harshist way to
kill a proc. SIGHUP (-1) and SIGQUIT (-3) are nicer, but as a
programmer, you can block those signals. you cannot block a SIGKILL so
it will always work

I commonly find a rouge netscape process running after all the GUI
windows are gone.

also, the command:

top

may be helpfull. it can show you (in a nice way) which procs are sopping
up the RAM and CPU time. Here's an example:
