Re: Imminent exclusion of non-GPL kernel modules?

At 2003/5/12 09:03+0930  Glen Turner writes:
> 
> What I don't understand is why.  As far as I know the 'taint' flag
> is really a tech support warning -- you can't look to linux-kernel
> for help as they don't own the code that might be causing the
> problem.

That's correct - if you submit a kernel oops to linux-kernel and the
taint flag has been set you'll be asked to try to duplicate the
problem without the binary module loaded. If you can't, well then its
probably a bug in the binary module and you should be talking to
whoever supplied it to you.

> Of much more concern to me is the notion that all binary
> modules might be verboten.  This sucks a lot as there are
> some modules where we don't want people stuffing with the
> source code (such as the frequency assignment registers
> in 802.11 drivers).  Even though we'd be happy to release
> source code to the responsible.

Can't people just edit the binary modules directly (eg look at some of
the Tivo hacking that has been done).
