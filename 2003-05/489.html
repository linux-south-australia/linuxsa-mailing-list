mplayer is the go. Its played all the dvd's I've thrown at it 
so far, and avi/mpg's as well. 

> In Redhat I wish to have an icon on my desktop that I can
>  access a samba share on another windows machine that may
>  or may not be switched on at times, can I use fstab to
>  mount a remote share to a folder in the mnt folder ? If
>  the share is not available because the machine is switched 
> off will this cause any problems. 

Add a line (no line breaks) like 

//server/documents  /documents  smbfs 
username=simon,password=yourpw,uid=500,gid=500 0 0

to your /etc/fstab.

Not sure what it will do if the windows box isn't there. Try using the
smbmount command directly to test it first if the machine is off.

Be careful, if it hangs forever when the machine isnt there, you may
need to boot into single user mode to edit fstab and remove the line
when its not there.

I found that every so often on bootup using the line above, it would
hang when mounting from another linux machine running samba, YMMV
