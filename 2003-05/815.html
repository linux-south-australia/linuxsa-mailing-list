Lyx 1.3.1 was Src rpm

Suse 8.0 Kde 3.1.1

Bouquets to the Lyx team for Lyx 1.3.1. I like the Qt interface, and am 
looking forward to playing further with the software. I went with the 
lyx-1.3.1-1suse-qt.src.rpm. 

Using rpm --rebuild lyx-1.3.1-1suse-qt.src.rpm. the compile proceeded to this 
point 
checking for moc2... not found
checking for moc... not found
configure: error: moc binary not found in $PATH or /bin !
Bad exit status from /var/tmp/rpm-tmp.390 (%build)

These programs were allready on my system so I tried to include the path 
/usr/lib/qt-3.1.1/bin/moc, but after several attempts settled for creating a 
link
ln -s /usr/lib/qt-3.1.1/bin/moc /usr/sbin/moc

This worked ok. I had to do this for uic as well
ln -s /usr/lib/qt-3.1.1/bin/uic /usr/sbin/uic

The script stopped on locating the QTDIR, so I did
export QTDIR=/usr/lib/qt-3.1.1

The compile then went ok, and took quite a while to complete.
From here I was able to do
rpm -ivh lyx-1.3.1-1_qt.i386.rpm.
At this point Rpm wanted dvitools so that was installed with apt-get.


There were a few warnings when the program was started, but Lyx is working ok.
Brian Marr
