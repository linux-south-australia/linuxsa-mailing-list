Re: [OT] Dynamic web content

From: "Andrew Galdes" <AGaldes@elders.com.au>
> This may be a simple one for the vb, js, asp, dhtml, perl etc people. I
would like to be able to dynamically change the size if an image by entering
a new value into an input box and hitting the "Calculate" button.
>
> What is the best way to do this? Any ideas?

The pure HTML way:

I assume the image is already on the web server (it's pretty easy to make an
upload form if it's not).  I would have an ASP page that generates a HTML
form where the image is displayed along with input boxes for the
width/height and a standard submission button (Calculate).  When submitted,
the two variables along with the image are passed to Atalasoft's ImgX
ActiveX object on the server side that produces a high quality resized
image, replaces the original, and some ASP to Response.Redirect the user
back to the original form page where the resized image will now be
displayed.  It's easy to make a copy of the original too before resizing.

Caveats: MS technology, many many web hosting services will not allow
objects on the server side, ImgX costs money (although it is an excellent
package).

Pros:  Universally browser compatible, high quality.

You could also use Java 1.1 (need to use AWT :-( and make a client-side
applet sitting in the browser window, but then you have a major hassle of
dealing with security issues when accessing the local hard drive to get the
image (it can be done, but it's a major pain and most web browser security
settings conspire to thwart your attempts).  If however your image to be
resized is stored on the server it's reasonably simple to retrieve it from
the web server and manipulate it using the Image object in Java 1.1.  I've
done this in the past.  Pros?  No time waiting to refetch the resized image
from the server.

Maybe server side Java?  The image objects have nice high quality image
resizing API calls.  I'm sure Perl has modules for resizing images too....

Resizing just using client-side Javascript to chance the image dimensions
will be slow and produce brutally ugly results.
