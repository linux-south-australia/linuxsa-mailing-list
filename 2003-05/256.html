> Ok, to clear this up. The Windows2k PDC logs show that the samba server has failed an authentication test as a standard "Invalid uid and or password" according the the logs - this happends aprox every 30 seconds. 'NTMachine' is a 2K Machine running as a PDC, but the Samba machine is
not a member of the domain, but rather uses the old-style authentication
stuff. Much clearer :-)

If you don't want to do a packet capture, try turning the log level up a
little at a time. You should be able to change the log level without
restarting Samba by changing the smb.conf file, sending a SIGHUP to the
smbd processes, waiting, changing it back and sending another SIGHUP to
the smbd processes.

This problem may not occur if the Samba machine were a member of the
domain, but if the Samba machine is a production server, you may have to
do a little more testing before making that change.

Hope it helps,
