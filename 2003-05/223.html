Quoting David Lloyd <dlloyd@microbits.com.au>:
> 
> > So I don't think your conspiracy theory work this time David :-)
> 
> I guess the end user interface didn't change much, but everything else
> was fixed. Supposedly. I don't think the fixes make an iota of

Sure, the end user may not see a whole lot of difference (except for a proper 
GNOME 2.2 desktop including integrated CD burning etc etc etc), but that's not 
Red Hat's (stated) reason for upping major version numbers.

The "low end" Red Hat product is a proving ground for technology that makes 
it's way to the "expensive" corporate paid support version, so things like 
improving threading performance, and C++ compiler standards compliance, is 
probably more important than 'eye candy' in this market space.

So you could say that RH9 might have been released to prove technology that 
will appear in the new (corporate) RH Advanced Server product(s) that will be 
coming out in the near future.

I think you'll see some more product differentiation soon - versions of distros 
that are aimed at desktop users verses back office systems.  Whether this will 
work (from a business sense) for free software is another thing (i.e. I want it 
all).
