James Leone wrote:

> Can Etheral send captured packets? See below***

It can capture it to a pcap compatible file, and you can switch the
"Follow TCP trace" output screen from ascii to hex (well, my old
0.8.8 version can).  There is a utility that can convert pcap capture
files back into raw binary dumps, but its name eludes me at the
moment (tcpflow? tcpsplice? tcptrace?  nuts (that last one isn't a
program :-)).

> I will try to look for a Linux "G3 fax" front end that allows me to
> specify a phone number, etc.

G3 is that fax data format (think of it like a GIF for faxes), and
it doesn't know about things like phone numbers, so it's only part
of the solution (and that's still a wild guess on my part).  There's
still more digging and splicing to be done, because I suspect the
"PCFAX Adonis-C2" part is a header of some sort, plus you will need
to add the PJL header to that as well before you can even begin to
start sending faxes, and this is *after* the binary part has been
figured out.  It's a bit premature to be throwing data at the thing
just yet.

If you can extract the data portion after the header, then running
the "file" command across the data should reveal if it's in a common
format or not.

have to have an infinite number of faxes, with everything you'd every want
to fax prepared in advance.
