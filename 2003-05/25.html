James,

It's fixed.
I found the horizontal kHz range on the side of the old cardboard
box that the monitor was delivered in - 31-54 kHz.
So I typed that into /etc/X11/XF86Config and finally it worked.
"startx" picks up the new values and does the right thing.
Women always claim that guys should throw out their old cardboard boxes, but
clearly there's some genetically programmed wisdom in the keeping of
cardboard boxes after all. 

Strangely enough, SaX2 refuses to use the values in the XF86Config file.
Even if I stop and restart YaST2, the SaX2 software which it spawns
still uses the old values:
/usr/X11R6/lib/sax/xc.pl, /usr/X11R6/lib/sax/xw.pl, /usr/X11R6/lib/sax/xapi ...

SaX2 must always work out by probing what it thinks the monitor is,
and in this case it thinks it's 31-60 kHz. Therefore SaX2 will be
eternally useless with this monitor unless I can find some
sneakily hidden config file which can tell SaX2 to please not probe
my monitor. Probably SaX2 is not programmed to know about monitors
as cheap as what I buy. But if I chuck out all the cheap junk in my house,
it will echo like an empty cave.

Great... Now the SuSE KDE default wallpaper shows pictures of bombs.
They look just like the bombs the Atari ST used to show when
the OS bombed. (The number of bombs indicated which exception number
had occurred!) I hope this isn't an omen!

Cheers,
