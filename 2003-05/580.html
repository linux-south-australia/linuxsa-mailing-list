Re: umount problem

Brian Marr <cabernet@ihug.com.au> wrote:
> Go easy killing processes, your linux box may have a cardiac arrest. A lot of 
> these things are started in /etc/modules.conf.

Modules.conf doesn't start any process.

Although not an answer to the generic problem (ie how to find out what
process is causing a kernel module to be in use), you can use the -f
option in umount to force the unmount.  Any process with a file still
open will fail at the next access on that filesystem.
