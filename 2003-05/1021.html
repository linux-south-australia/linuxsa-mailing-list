If you stick with the defaults, Gentoo is extremely stable & not at all bug
prone.
If you want to install the latest beta development software though, then you
take the risk.
Many choose to do this, as it's easy to switch back & forth between stable &
unstable source with the 'emerge' & 'unmerge' tools, and so are able to test
quickly whether something labelled as unstable is for them or not.
As a result, Gentoo does have a sizeable bleeding edge community (me
included).

> So if I finally take the plunge and put Gentoo on a sacrifical machine,
> will I be in security heaven because Gentoo is so good at reporting
> problems that other distros don't,

You will be in security (and fix) heaven. User communication with Gentoo
maintainers is immediate, & so any real fixes are immediately added into the
portage tree for testing.
Example: I created a patch/fix for a certain package & e-mailed the
maintainer with it that night - 2 days later it was fixed & in the global
portage tree for testing :) - can't get much more on the ball than that.

Having said that, the real benefit in using Gentoo for me was that
EVERYTHING was compiled by my system, for my system with full gcc
optimisations. The difference in speed & response really is something to
behold in comparison to any generic binary based distro (they do install far
quicker though).
A typical Gentoo install on an XP 1800+ w/512MB ram will take around 3
hours.

> or a security nightmare because
> their packages are dodgier than the average distro's?

Apart from the basic portage tools & install .iso (40MB), Gentoo doesn't
release any packages.
It simply provides a directory tree of ebuild scripts (updated through
rsync) that wget the original source files from the developer's web-site &
then run ./configure && make && make install - with options & full
dependency checks.
So in answer, no.

If you have a moderately fast processor, & a broadband connection, you'll
fall in love with Gentoo.
