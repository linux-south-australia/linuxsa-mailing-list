I'd strongly advise to take the file containing the message and send
only that.  The best thing to do is to create a new folder and store
the message there; that way you don't have to try to work out which
one it is.  I've just tried that: file in a new folder, in this case
called sample, and you'll see something like this:

   === grog@sydney (/dev/ttypa) ~ 3 -> ls -lR Mail/sample/
   total 1

   drwx------  2 grog  wheel  512 May 29 15:28 tmp

   total 1

   Mail/sample/new:
