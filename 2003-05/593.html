Knoppix hard disk install with Debian Kernels

-----BEGIN PGP SIGNED MESSAGE-----
Hash: SHA1

If anyone has installed Knoppix to hard disk and has found problems getting a 
standard Debian kernel to "apt-get install" then I've found a solution thanks 
to a kind and knowledgeable man call Bert Haverkamp.
This, as it turns out, is also the solution to a question I posted in early 
April titled "Setting up Grub on Debain", Grub wasn't the problem apparently.
Anyway I'll quote the wize words of Bert,

>Hi David,

>After spending most of sunday on this, I think I found an easy solution
>Try replacing the knoppix version of ash with the standard ash or dash.
>(installing ash will automatically replace knoppix-ash)
>Then install the vmlinuz-2.4.20-k7 again. Now things should work.

>What happened was that initrd-tools created a initrd-image with the help of 
>ash.  Ash is supposed to start linuxrc and init on this image. However there 
>seem to be an incompatibality between ash and knoppix-ash that gives an error 
>wne running linuxrc.

>Could you try the fix for me and confirm if this fixes the problem for you 
>too? I plan on reporting this to the knoppix mailinglist when I get a 
>confirmation.

>Regards,

>Bert

and after following Bert's instructions I have'

creedy@ferret:~$ uname -r
2.4.20-1-k7
creedy@ferret:~$

Thanks very much to Bert Haverkamp, this will allow me to install the Nvidia 
drivers into Knoppix at last.

Cheers,
