Re: Linux Kernel Bug?

Matthew Western wrote:

> obviously one can't just bring down all machines on the network to upgrade
> the kernel

Actually, no.  A basic system adminstration task is keeping
the software up to date, so you do need a procedure
to do exactly that.

As one example of a low-impact tactic, we use autoupdate
to distribute the kernel RPMs.  When the kernel, glibc, or
openssl RPM is updated the machine schedules a early-morning
"shutdown -r now" in cron.
