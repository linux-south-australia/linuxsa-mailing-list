RE: [OT] Dynamic web content

> Hello all,
> 
> This may be a simple one for the vb, js, asp, dhtml, perl etc 
> people. I would like to be able to dynamically change the 
> size if an image by entering a new value into an input box 
> and hitting the "Calculate" button. 
> 
> What is the best way to do this? Any ideas?

Do you want to rescale it onscreen, without neccessarily creating an
downloading a new image, or do you actually want a server-side process
to scale the image, and then the client simply download and display at
100%?

Option 1: some sort of cgi or php or whatever you find easiest, where
the "calculate" button submits "scale=<input>" back to the server, and
the server generates the HTML, with the appropriate HEIGHT= and WIDTH=
attributes. The browser will (assumedly) cache the image, and re-render
it in the required scale.

Option 2: As above, but when the CGI runs, it executes imagemagik's
"mogrify --scale <input>" (check that usage...) on the image, notes the
filename, and generates the HTML with the appropriate filename in it.
The browser then downloads the new image, and render it at 100%.
