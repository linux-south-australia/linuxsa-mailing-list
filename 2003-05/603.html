Richard,

There are a couple of tiny problems with that line of reasoning.

1.	Linux does use sequential increment-by-one IP ID numbers for TCP.
	(I just verified this for kernel 2.4.20.)

2.	Linux does not use increment-by-one IP ID numbers for UDP.

3.	The security problems described are exclusively for TCP.

Therefore the linux zero IP ID field does precisely nothing about
the problem and linux is wide open to the problem described.
So you get the nuisance of no IP ID in UDP packets for no benefit.

I rest my case. For a short tea break anyway....
