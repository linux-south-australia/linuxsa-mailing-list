Re: P2P Networks

I'll second that, pyslsk is surely worth the effort to get going.
And has come ahead in leaps & bounds in just the past couple of months.
http://www.sensi.org/~ak/pyslsk/

(16 Feb 2003) pyslsk is now available in FreeBSD ports. Install it with:
cd /usr/ports/net/pyslsk
make install clean

If you can run wxPython demos, you may be sure PySoulSeek will run too.


Another to keep an eye out for is mldonkey, it has developing support
(almost there) for gnutella, dc+connect, soulseek, open-nap, e-donkey &
bit-torrent p2p networks. Each network can be individually
enabled/disabled & it comes with a CLI, GTK+ gui & web interface setup
similar to that of Audiogalaxy's where the core daemon is run & access
is given to it via the web interface.
http://mldonkey.berlios.de/modules.php?name=Wiki&pagename=HomePage

Not sure how well Wine runs in BSD (if at all), but another alternative
is to run Kazaa through Wine. This works extremely well.

As far as my thoughts on gnutella go, i'd steer clear of it. It doesn't
scale well, & all you end up doing is becoming a node for everyone
else's stale searches.
