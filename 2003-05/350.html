Re: wacky PC hardware - IRQs and UARTs and PPP

Alan Kennington wrote:
> It seems like the wacky PC hardware organises all those priorities
> in the /proc/interrupts file as follows:
> 0, 1, 2, 8, 9, 10, 11, 12, 13, 14, 15, 3, 4, 5, 7, 8
> from highest to lowest priority.
> Now that's what I call wacky hardware. Coming from a 68k background,

Actually, the order should be:

0, 1, 8, 9, 10, 11, 12, 13, 14, 15, 3, 4, 5, 6, 7


The reason is simple enough - the IBM XT hardware featured a single
Programmable Interrupt Controller (PIC), providing interrupts 0 through 7.
The IBM AT introduced a second PIC which is cascaded to the first via (have
you guessed?) interrupt 2. Note that this means that the original interrupt
2 no longer exists but for compatibility the bus signal assigned to IRQ2 is
routed to IRQ9. That interrupt is often referred to as IRQ2/9.

Yes, it's "quirky" but understandable as it provides for additional
higher-priority interrupts while retaining compatibility with existing
interrupt assignments (e.g. for COM1 and 2 on IRQ4/3).

I should imagine you'll find googles of references for this. Heh, did I just
create a new term?
