> > The problem with the wording is a big one IMO. There shouldn't be 
> > legislation to mandate the use of Open Source over proprietary software. 
> > This is too restrictive and would never be acceptable to a majority. The 
> > mandate should be to avoid lock-in software, ie. open *Standards* should

Ummm, but surely the intent is not to punish Microsoft for being so 
successful at creating a monopoly in the past; rather it should be all

that are important. If MS can compete successfully on that basis, what is 
wrong with that?

> situations where it will fail because the software is still immature.
> NNeither do we want to ignore the possibility of using it where itis
> comparable to proprietary software.

You are getting too far into creating legistaltion on how purchasing 
decisions should be made.

> Where functional equivalence is inspected, there should be a protocol
> for choice that is followed. Like a SOP.

Moreover, the law needs to be simple to be effective.
