Procedure - rebuilding FreeType for RHL9

As usual, I someone finds this useful.

        RE-BUILDING FREETYPE TO USE THE BYTE CODE INTERPRETER
                          (Red Hat Linux 9)

The FreeType library with Red Hat Linux 9 does not have the TrueType
bytecode interpreter enabled.  Apple Computer claims a patent over
some aspects of the operation of the interpreter.

This procedure installs FreeType with the byte code interpreter
enabled.  The font rendering with the byte code interpreter enabled is
said to be better than without it enabled.

Download and install the source code

  $ wget .../freetype-2.1.3-6.src.rpm
  # rpm -ivh freetype-2.1.3-6.src.rpm

Modify the build specification to enable the byte code interpreter.

  # cd /usr/src/redhat/SPECS
  # vi freetype.spec
    %define without_bytecode_interpreter    0
    Release: 6.1

Rebuild the software

  # rpmbuild -bb freetype.spec

Install the rebuilt software

  # cd ../RPMS/i386
  # rpm -Fvh freetype*-2.1.3-6.1.i386.rpm

Restart X to use new library

  Ctrl-Alt-F1
  # telinit 3
  # service xfs restart
  # telinit 5
