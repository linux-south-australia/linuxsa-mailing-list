Greg 'groggy' Lehey wrote:

> SCO has two separate issues, apparently.  That's the first one.  The
> second is a claim that they have incorporated UnixWare 7 code in the
> Linux kernel.

Thanks Greg, I'd missed that.

I read your page <http://www.lemis.com/grog/sco.html>;,
particularly "The Smoking Gun".

The list of possible sources of UnixWare 7 code in Linux
probably needs to be extended to include C header files.

There's one notorious case where a C header file in the
first implementation of an IETF protocol had an incorrect
value and this value subsequently appeared in the majority
of implementations.  So I think the practice of copying
parts of header files without attribution is pretty
widespread and I imagine that Linux has not spared from
that.
