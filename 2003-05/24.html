Now having a look at it a bit more, I've found that if I kill
a program "xapi", the screen lock is removed and I get back
to where I was before.
So now I don't have to re-boot every time I try to config the X server.

(That is a constant problem with SaX2, which stupidly uses X to
configure X - so if it doesn't work, you can't fix it!!
The old SaX used to have a text-only mode. But I guess the
marketing people didn't like its looks so much.
This is the problem with the rush to imitate the MS pointy-clicky
approach to everything. The advantage of unixes is that you can always
get out of every problem - if you're prepared to give up such
luxuries as sleep and social life. The big distros have turned their
backs on the greatest strength of unixes - the text command line.)
