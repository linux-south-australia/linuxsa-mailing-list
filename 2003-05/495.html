> Debian BTS:
> 
>  Pros:
>  - it's natively understands the Debian system
>  - it can be manipulated totally by e-mail
>  - it stores all things in a text backend (no sql servers etc etc)
>  - it behaves just like bugs.debian.org
>  - it's collaboratively managed
> 
>  Cons:
>  - the lack of an sql server is a con too

>

>  - it's UI is highly configurable
>  - it's security settings seem more fine-grained than the Debian BTS
>  - it uses an SQL backend

> 
>  Cons:

Just based on your thoughts above, I would go with Bugzilla.
 
Perhaps you've just about convinced yourself already that you prefer 
Bugzilla.

On interface issues... well, they're just web pages and web pages aren't 
difficult to edit (time is probably the major issue here) and depending on 
what you need the users to do, it could simply be a matter of deleting all 
the irrelevant stuff (complications?) from the existing pages.

Not having documentation is a big negative in my mind.  The sql backend is 
a positive (for both products).

Are you sure you want to restrict your available options to just these 
two?
