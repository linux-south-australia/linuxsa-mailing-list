Re: C++ ifstream buffering from named pipe

On Thu, 2003-05-22 at 16:26, Joshua Cole wrote:
> The problem seems to be that the reading is buffered.  Does anyone know
> how to set up an unbuffered ifstream in C++?  It works if I read the
> named pipe using C style FILE * streams, but I would like to avoid doing
> this if possible.

for istreams, the equivalent to flush() in ostreams, is sync().
or you can operate directly on the streambuf using such
calls as: pubsetbuf() to set the buffer space (0 being unbuffered).
