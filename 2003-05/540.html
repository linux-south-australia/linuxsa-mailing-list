Re: [OT] DSL

Fundamentally there tend to be two main sources of such issues:

- Line quality problems (Telstra tests are the way to chase this one, 
usually... and note that if it gets worse when it rains, for 
instance, thats a clue). My gut feeling is that you do have a 
physical fault developed or developing on the line.

- Telstra concentrator or DSLAM software upgrades that have adversely 
affected the DSL devices' ability to maintain sync. One specific 
thing here is that the supported, and preferable, modulation for ADSL 
connections in the Telstra xDSL network is "G.DMT" mode. The 
ANSI/G.Lite mode also works, but not as well, especially in adverse 
situations. If you haven't already done so, try locking the device 
into G.DMT mode and see if it magically stops dropping out.

You might find some other ideas to try in here:

http://adsl.internode.on.net/faq/padsl-troubleshooting.htm#ts2


... but fundamentally, the only suggestion in the above that hasn't 
been noted by others in this thread already is the G.DMT thing.
