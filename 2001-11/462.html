Re: Routing Question

Hi,

> We have just connected an Internode ADSL service for a client, and we
> have 5 IP’s, and we are trying to think of the best possible way to
> assign two of the IP’s to the other two servers we are running.
> 
> Our linux box is a Redhat 7.2 running

Bleh! Yuck!

> a 2.4.7 kernel. We are using the
> Iptables NAT module to give Internet access to all the other PC's in
> the
> organization. In the server rack, we also have a Windows 2000 server,
> and a Windows NT server. We would like to assign each of these servers
> an IP, however we weren't sure if we should setup Static Routes, or can
> it be done dynamically with NAT ?

Do you have clients that require an internal (192.168.xxx.xxx) IP? If you dont, 
then NAT is not required. 

Turn on forwarding, (echo 1 > /proc/sys/net/ipv4/ip_forward (from mem...)) and 
you should be OK.

Make sure that the Upstream provider routes those IP's accordingly, or you will 
be like a fish out of water.
