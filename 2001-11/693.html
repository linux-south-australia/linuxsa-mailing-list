RE: Windows X server

I know you asked about a "free" or open source solution, but just in
case you cant find one that you like, I'll let you know about a really
good commercial way to get X-Apps onto windows desktops!

check out http://www.tarantella.com !

It is an application that runs on unix/linux and is basically similar to
Citrix, except it runs X-Apps (and mainframe apps too).

Your X-Apps can run from a tarantella native client or in a standard web
browser (using Java).  It also quite nicely maps your local printer so
when you print from a unix/linux app, it will spit it out on your
windows default printer.  It also runs Terminal Server sessions (and
optimises the bandwidth used), so your end users can run both Terminal
Server sessions and X-Windows apps from a web-browser or native client.
Will also map local drives back to a Win 2000 Terminal Server session so
you can load stuff off floppy onto your thin client session.

The program installs itself with very little fuss on to Redhat 7.x, and
you can download and install it for a 30-day free trial.

Lots of companies are using this product (even big ones like EDS)
becuase it is cheaper and more flexible than Citrix (and of course it
runs under unix/linux :) ).
