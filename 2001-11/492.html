> I accidentally deleted the partition information from an 80Gb hard
> drive.  The drive contained a single NTFS partition.

In this case, you have not actually deleted any files.  I find you can
usually just use Linux fdisk to reinstate the correct partition size and
type, and everything is restored.  (IIRC HPFS/NTFS partition type is
07.)

If you don't value your data (or have a recent backup), you can try this
little exercise:
* boot Linux and make a record of your partitions
   (the output of mount, cat /etc/fstab, and fdisk /dev/hd<whatever> are
quite useful for this)
* run fdisk and delete all your partitions.  save.
* reboot from Linux installation media
   (something like a Red Hat install CD would be good)
* use fdisk to recreate your partitions as they were
   (do not proceed any further with installation)
* reboot

(You may need a boot disk at this point.)

I have done this several times without losing data.

If you value your data, however, you may like to try some other method
first!  :-)
