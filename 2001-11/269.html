Re: Help on Mounting CDROM in RHL7.2

On Fri, 9 Nov 2001 11:32, Allan wrote:
> On Fri,  9 Nov 2001 18:34, Chandrasekar V wrote:
> > Hi All,
> >
> > I have installed RedHat Linux 7.2 using the CDRom
> > drive. After Installating I am unable to mount any CD.
> > If I give the command
> > [root@mc5438 /]# mount /mnt/cdrom
> > I get a error message as
> > mount: /dev/cdrom is not a valid block device.
>
> Which is fair  enough - /mnt/cdrom is a directory, you'll find your block
> devices in the /dev/ directory. For a basic mount command you could try
> something like:
>
> 	mount /dev/cdrom /mnt/cdrom
>
> For more information use the following command:
>
> 	man mount
> and
>
> 	man -k mount
>
> could also be of some benefit.
>
> Allan

Not neccissarily.  If you enter a directory as the argument to mount, it 
checks the fstab to see if a device is setup to be mounted to that directory. 
 So if you have set up the cdrom in your fstab file, then the command "mount 
/mnt/cdrom" is valid.  The relevant line in my fstab is:

/dev/hdc /mnt/cdrom auto user,iocharset=iso8859-1,exec,codepage=850,ro,noauto 
0 0

The noauto reference means that you have to execute the shortened mount 
command to mount a cd.

Sekar, you might want to check your /dev directory with a "ls -l /dev/cdrom" 
to see if there is a link to one of your ide or scsi drives, it could be that 
the /dev/cdrom link is bogus (I'm assuming that your fstab line starts with 
/dev/cdrom).
