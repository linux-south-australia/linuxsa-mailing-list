> 	The web server of my company has an IP address.....this IP address is't
> available from our LAN, what have I to do to can "see" our domain from
> internet and from our LAN ???....

OK, you have two separate things here, DNS and IP...

First thing to do is get the IP address visible where you want it.
This could actually mean that you need two separate IP addresses, one
for the LAN and one for the internet... usually, however, if your LAN
can access the internet, it can access the IP of your webserver on the
internet...

tools to use to ensure that IP is configured correctly include:

ifconfig -a
netstat -nr
ping
traceroute (or better, mtr)

Then, you need to get DNS to correctly translate the hostname into the
appropriate IP address... you need to find out where the master server
for the appropriate domain is, and configure this to translate the
hostname you want to the IP you want. If you have two separate IPs,
and you want to use the same hostname to get to either, depending on
where you are, I suggest reading the O'Reilly book, DNS & BIND. In
fact, read it anyway...

tools to use to ensure that DNS is configured correctly include:

nslookup
dig
