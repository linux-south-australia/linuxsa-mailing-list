Re: Editing html inplace with Apache...

Richard Russell wrote:

> I really don't know where to start looking with this one... I want a
> site, hosted with Apache on my Debian server, to be editable by one
> specific person with a windows PC, so they can just go to the relevant
> page, edit it in MS Word (or whatever) and then save it back to the
> original URL... What is the best way to do that?

Caveat:  I haven't dabbled in this particular area for a while, so what
I'm saying here is definitely not state of the art.  A couple of
"transparent" ways - there are Front Page Server Extensions, from both
Microsoft and "Mrs Brisby" (www.nimh.org - a site whose error pages
come up in haiku), and Apache's own WebDav module, which is supposed
to be what modern authoring tools use for collaborative authoring
these days.  The MS FPSE extensions are as secure as any of the products
that they've written, so avoid it at all costs.  The code at Mrs
Brisby's doesn't look like it's been updated for a while, so WebDav
is probably a better bet (take these suggestions with a pinch of salt,
probably about a kilo's worth).
