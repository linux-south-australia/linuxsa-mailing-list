Linux ISDN Problems

Hi Everyone,

We're having some troubles setting up Traverse ISDN Netjet cards under
linux. We are trying to run 3 cards in the one box for DOV dial-in for
three separate sites. However, when we have three cards in
simultaneously computer freezes whenever we try to initialise the Hisax
module.

If the ISDN lines are NOT plugged in, the modules load and work okay, if
we remove one of the cards, two cards initialise and function okay. We
have removed almost every other card from the system, disabled serial
ports and parallel ports so that there are plenty of IRQ's. When the
system boots, in the device listing it says that each card has it's own
IRQ (10, 11 and 12).

We have tried the following:

* Sent back the cards to Traverse for testing (all cards are the Tiger
320 Chipset)
* Changed Motherboard in PC originally using an Epox 8Kta3+ with Athlon
1Ghz, switched to MSI 6337ET Pro Socket A running PIII 866 Processor.
Box has 256MB ram, and two hard disks.
* We've tried Redhat 6.2 and 7.1 we have used a variety of Kernel's with
both these operating systems raning from:
2.2.12 all the way to 2.2.19
2.4.4 all the way to 2.4.12
* We've used all the recommended Traverse kernel patches, and followed
all instructions exactly.

Now we are lost and don't know what to do. Any help appreciated.
