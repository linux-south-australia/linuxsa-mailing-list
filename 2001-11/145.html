Basically you need to use either ipchains or iptables to set up
masquerading.

What this does is let all of the computers on the internal network (which
I'm assuming use addesses starting with 192.168.x.x or something like that)
use the linux box as a gateway.

For example I use the command

/sbin/ipchains -A forward -j MASQ -s 192.168.200.0/24 -d ! 192.168.200.0/24

in this example I've used the network 192.168.200.0 with a netmask of
255.255.255.0 ( the /24 is just a convienient shorthand). What this example
does is make all packets with a source ( -s ) address from the internal
network, and a destination ( -d ) address not in the internal network get
masqueraded.

You may need to turn on ip forwarding to make this work. Look in
/etc/sysctl.conf and change ip_forwarding to 1.

Hope that helps. I've only covered the very basics here btw - you might want
to add some extra firewalling rules to keep all the nasties out.
