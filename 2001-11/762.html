Re: Routing/filtering issue????

Hard to say without knowing what your network looks like... I'll
assume it's as follows:

Internet (via ISP)
	|
	|
	|
  Wilbs' Server
    a.b.c.d
	|
	|
hub/switch/crossover cable
	|
	|
Wilbs' Workstation
    a.b.c.e


So from a.b.c.e (e for short) you can ping a.b.c.d (d for short), and
you can ping e from d as well... from d you can get to the internet...
e's default route is via d (show us your routing table (netstat -rn)
with IP masked, if you like (for both d and e)). Does e actually
forward packets from the network to the internet and back (this is
usually done by ipchains, so if you removed your firewall, you may
have broken that). Is your ISP actually routing packets destined for
your entire subnet back to you? You can tell this by asking them, or
tracerouting back in from the outside...
