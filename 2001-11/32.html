What do you want all that complexity for?  The lack of System V init
scripts is a feature, not a bug.

> Anyway, I'm not serious (well, I am serious about it not having SysV
> init, but not about me being that upset... :) .... (so don't any BSD
> types try to console, convert or cajole me... it's all in fun :)

So what are you looking for?  All your system configuration
information is in the one file /etc/rc.conf, a shell script which just
sets variables.  In modern versions of FreeBSD, this file contains
overrides to the defaults, which are found in /etc/defaults/rc.conf,
and which contain copious comments.  3.2 is an ancient version of
FreeBSD, of course, and it's possible that you don't have a
/etc/defaults/rc.conf, in which case /etc/rc.conf contains the
defaults as well.
