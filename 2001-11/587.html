Also, some companies are going with firstname.lastname or similar.  Is
there anything wrong with using periods/dots in usernames?  How does
BIND handle it if the administrator is my.name.server.com?

> Do some MTA's barf at really long names?

Any good MTA will reject strings that are excessively long, as there
are possible overflow exploits.  I expect the allowable length would be
in the vacinity of 255 characters, so any person's full name should be
fine.

> What good reason is there for standard character length usernames???

Many older Unix systems (especially those not using MD5) seem
unable to have logins/usernames longer than 8 characters.  I am not
aware of any recent MTA than can not handle such usernames, so
the primary concern would be whether your systems can handle them.

(Both of the last two paragraphs were written with only a basic
understanding of SMTP and MTAs.  I might be wrong!)
