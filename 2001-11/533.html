Re: internet cafe monitor/billing software

Andrew Reid wrote:

> The simplest way of doing this would probably be to use IP-level byte
> counters, using either IPChains or the newer IPTables under Linux, or
> if you want to use a real OS, ipfw and IPFilter under FreeBSD.

Simplest, but still less than exact. If windows security isn't also
addressed, it's not hard for a user to change the IP address of their
system and get charged nothing at all.

We did a system for an Adelaide Netcafe based on VLANS and SNMP, for
accurate billing which doesn't trust the client at all while being easy
to maintain for the netcafe owners. One day the code may end up at:

http://sourceforge.net/projects/netcafe/

Linksys, however, have a new Layer3 switch coming out this month that
looks like it will reduce the complexity of a lot of this sort of
billing, and be cheaper than a cisco switches currently being used. Just
allow only a given IP on a switch port.
