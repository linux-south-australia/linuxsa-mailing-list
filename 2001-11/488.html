> network to run on DHCP, but when I try to ping the admin pages...

Ping web pages?

If you're trying to access the admin web pages, you'll need to try
http://yourservername.domain.com:81/
http://yourserverIP:81/

(note that you will usually NEED to specify the protocol and port
number)

Are you able to ping your smoothy box from another host on the network?
Are you able to ping your workstation from your smoothy box?
Are you able to log into your smoothy box using ssh (port 222, I think)?

If you are unable to perform any of these actions, then it is likely
that your
firewall and/or network are not correctly set up.  It may be useful for
you to
supply some more information about your network, and some more specific
error messages if you would like someone to figure out what the problem
is
likely to be.

> Secondly, how do I manually select my ISA modem in smoothwall setup...
> its not detected at probe time.

Did you try any probe or manual configuration option?  Was one
available?

You would be aware that some internal (usually PCI) modems are not
usable
under Linux.  If you're certain that your modem is not such a modem, you
may be able to configure the firewall to look on a certain serial port.

Once you can log into the system, you may like to rerun setup or netconf
(or whatever the setup/network configuration program is called) to see
if it
can be configured that way.

The other possibility is to log in and configure ppp/pppd by hand, but
that's
not an enjoyable task IMHO.


I hope this gives you some ideas.
