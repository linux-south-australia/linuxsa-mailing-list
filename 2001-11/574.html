No, this is because the Exchange protocol is proprietary meaning it's
not published meaning we have to spend large amounts of
time/energy/resources (aka money) to reverse engineer it.

>
> i'm a bit dissapointed.

You could revers engineer the protocol for us :-)

>
> so since we're talking, what's the possiblity of adding an option of
> allowing an LDAP server to be searched when i type in a name in the To
> field.  searching contacts works fine...

The only reason it isn't enabled right now was because it might have
been slow for users with a slow network connection, it will likely be
turned on with an option in the future.

> can you explain how Ximian can use the iCalendar from Exchange Server??  i
> can't see how i can do it....

It can't use the ical data from the server, your Outlook using friends
have to send you an ical event through email, at which point you can add
it to your calendar in Evolution.

Evolution can also send ical events to Outlook users and they can then
add that to their calendars...
