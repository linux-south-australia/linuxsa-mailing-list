Re: Locking Down Network Interface

HALES, Brett wrote:

> I did modinfo -p 3c59x.
> 
> debug int
> options int array (min = 1, max = 8)
> full_duplex int array (min = 1, max = 8)
> hw_checksums int array (min = 1, max = 8)
> flow_ctrl int array (min = 1, max = 8)
> enable_wol int array (min = 1, max = 8)
> rx_copybreak int
> max_interrupt_work int
> compaq_ioaddr int
> compaq_irq int
> compaq_device_id int
> watchdog int
> 
> How do I tell the full_duplex to be off, I assumed it would be a 0 or 1?


If you are loading the driver manually, try:

modprobe 3c59x full_duplex=0

If you have multiple cards, you can only load the driver once:

modprobe 3c59x full_duplex=0,1,0  # For three cards

 
> I am using a 3Com 3c905B network card. I did try using the 3c90x driver
> however this did not work well.


Yes, I think the 59x is the one you want.

 
> The interface did not come up when the machine booted I had to start it
> manually, it also seemed slower when I copied a 274Mb file.


Add the following lines to /etc/modules.conf


alias eth0 3c59x # Change eth0 to whatever you need
options 3c59x full_duplex=0
