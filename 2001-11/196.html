Re: Virtual Linux servers

On Wed, 7 Nov 2001, John Edwards wrote:

> I don't know if you have any specific application in mind

Many (I like this field a lot) but especially abstractions of entire
systems where whole groups of autonomous oeprating systems or
pseudo-operating systems work on one machine.

> interested in unix-style virtual servers it might be worth checking out the
> "jail" program/function in FreeBSD.

FreeBSD jail works well for many things, but isn't really a virtual server
environment. It is a heavily developed chroot environment. A virtual
server environment can be expected to provide comprehensive mangement
interfaces to manage the virtual manchine, but jail doesn't really have
this. You can't just assume that any ordinary userspace program will run,
either. IPC doesn't work in a jail environment, so no MySQL for example.

It looks like FreeBSD jail might become a lot closer to a virtual server
environment though. I posted here a while about Linux capability sets. It
now seems like FreeBSD is following the same track (its a Posix standard,
.1e from memory) and the plan is that this will be useful for fine-grained
control over what a jailed environment can do.

Maybe I'm out of date. Is there anyone here who's done jail on 4.4 and
can comment whether these limitations still exist?
