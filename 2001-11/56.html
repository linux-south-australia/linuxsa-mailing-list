Re: Ext 3 partition imaging

> If I understand it correctly, ext3 is just ext2 with an added file
> containing journaling information.  If this assumption is correct, the
> filesystem is identical to ext2, and should be supported by any program
> that supports ext2.
> 
> I would do a little research on ext3 to confirm this.

I'm not sure if this is correct but it's not required.  You can
convert between EXT2 and EXT3 with a simple tune2fs command which
takes only a short time to run.

So if it only supports EXT2 then you can simply:
1) boot onto a boot cd with tune2fs (RH7.2 has this) and convert the
   partitions in question to EXT2
3) run your Ghost program in EXT2 mode
4) boot onto the boot cd again and run tune2fs to convert back to EXT3
