> I really don't know where to start looking with this one... I want a
> site, hosted with Apache on my Debian server, to be editable by one
> specific person with a windows PC...

I've previously done this using samba.  Just create a share called
\\webserver\web or similar, and ensure that your samba.conf has the
desired permissions.  At first glance, it seems that samba's permissions
override the Unix file permissions, so I can't think of an easy, fully
secure way of providing write access to only one file.  (Of course, you
could map this share to a drive letter if it would make things easier.)

I hope you can find a way that is suitable for you, possibly using HTTP
put/post.
