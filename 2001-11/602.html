Re: Naming Conventions (again)

Justin Hawkins wrote:
> 
> You are talking about two different things here. There is not
> necessarily any correlation between email addresses and usernames. In
> fact, if they are seperate it can make your life a lot easier.

For a small school, I'd make the mail ID and the user ID
the same.  Making them different only starts to help at
bigger sites where the subtle difference between a mailbox
and a login becomes significant.  Bigger sites can also
leverage off directory services, for a small user community
LDAP doesn't have a huge payoff.

Depending upon the way you like to do things either let
people choose something that approximates their name
or make it random (eg: s123456).  Other choices either
degrade over time or can lead to policy issues (again,
the bigger the user community the more likely policy
issues are).

The important thing about the user ID is that it
should be permanently allocated and unchanging.
Similarly the user ID and group ID numbers should
never change and should be consistent within your
network.
