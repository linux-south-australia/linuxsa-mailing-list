Re: [OT] Re: FreeBSD 3.2 ports...

On Thu, 1 Nov 2001, Dale Long wrote:

> How good is FreeBSD at performing a complete system upgrade, from say a
> 3.x series to a 4.x series,on topof an existing 3.x system?
> 
> Any comments or experiences?

Pretty good. I don't think it breaks binaries you've compiled
yourself. FreeBSD kernels usually offer binary backwards-compatibility
back to version 1.x. Also, most of the packages are structured the same
between releases, so you don't have problems with different package names,
different dependencies, etc, etc.

I've done an upgrade from 3.2 to 4.2 and that seemed to go well.
