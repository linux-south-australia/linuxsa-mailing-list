Re: Commentary: The Greatest Flaw In Linux

Jake Hawkes wrote:

> I want to be able to come home, and have the system already updated itself, or announce an update
> thats available, and allow me to do it, and *NOT* break everything.

That's probably not possible, given today's technology.  Rule of change management:
Given a "completely functional" system, most breakages can be traced back to changes
made to that system (configuration, environmental, etc.).  And as very few systems
on a computer run in complete isolation, there is always a risk that updating a
package is going to break another package (e.g. upgrading a shared library, like
libc, which is used by most programs).
