Re: NO ROUTE TO HOST

Ok, so what happens if you try to connect directly to the SMTP port on
the problem server. EG:

[ant@fineales ant]$ telnet 216.136.129.15 25
Trying 216.136.129.15...
Connected to 216.136.129.15.
Escape character is '^]'.
220 YSmtp mta614.mail.yahoo.com ESMTP service ready
quit
221 mta614.mail.yahoo.com
Connection closed by foreign host.
[ant@fineales ant]$

If you get a connection, then you have a local configuration problem of
some sort. If you dont, what happens instead might provide a clue.

Ant
