Re: spam mails

I believe under Redhat 7.2, the relay file is /etc/mail/access.

Make sure you only have your intranet IP range listed. IE if you use the 
private class C range, it is 192.168.1.x  This basically prevents anyone 
outside this range from using your mail server as a relay box.

EG The format of /etc/mail/access is the following:
192.168.1            RELAY
