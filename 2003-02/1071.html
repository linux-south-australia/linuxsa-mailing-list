Hiya,

I've struck something which is extremely irritating in screen - I'm not 
sure if its a bug, or if there's a termcap or something I can use to 
fix it.

I do much development in php - I love using screen, makes it so easy to 
copy and paste between different vim windows.  I've got the latest 
screen installed from fink.

Normally, if I copy and paste (without screen in OS X Terminal.app, 
GLTerm, aterm, etc), it'll see the carriage returns after every line, 
and not copy the whitespace after it.  Just like all the behaviour I've 
seen in fbsd/linux.

If, however, I run screen, it'll copy *all* the whitespace after each 
line, which makes it extremely irritating if I edit a line and I 
suddenly have a huge blank line.

I realise I can use screen's built in copy and paste functions instead 
of my mouse, but I'd prefer doing it the usual way.
