RE: PHP Gurus? page link to change global registered variables ?

With PHP 4.2 onwards there is a configuration parameter REGISTER_GLOBALS
which is off by default.  REGISTER_GLOBALS would assign $a to the value of a
passed through the URL automatically.  This change is a good thing for
security, but it does mean that at some point in your code, you should...

if(isset($_GET['a'])) {
  $a = floatval($_GET['a']);
};

... you may still see examples and older code written with the assumption
that REGISTER_GLOBALS is turned on, but the "right" (and more secure) way to
do things these days is to assume register globals is turned off and code as
above.
