RE: Data security - any thoughts?

Jon,

Have someone enter the key when they access the file, decrypt on the
fly, then discard the key. Make SURE that the key and the file are
transmitted on a secure link. This way, if someone gets the computer,
they don't have the keys to decrypt, and if someone intercepts the
traffic, they still don't have the keys... However, if they got root,
they could modify some code to capture the keys, and then you'd be lost.
Not sure there's many ways around that -- except for making people
decrypt it themselves with the key...

Also, bear in mind that it sounds as if you are going to have a key per
file, as opposed to a key per person... which is ok, but it does affect
the way you give keys out.
