Re: 6 cyberproxy.com mails per minute!!

David Newall wrote:

>That's a disturbing thought, Alan.  An email server that includes the
>original message in a bounce could be used by spammers to relay, by making
>their intended recipient's address be the apparent sender's address.  I
>bet you are on to the latest tactic here.
>
It's a low yield tactic at best, spammers use relays to abuse the RCPT 
TO feature of SMTP to speed delivery without having to pay for 
bandwidth. Because you can't have multiple MAIL FROM commands in an SMTP 
session, a spammer would have to deliver an email once for each bounced 
email.

The anonymity value of a relay is also negated by usually detailed 
bounce messages that will indicate exactly where a spam came from.

The only benefit to spammers that I can see is that it roughly 
circumvents Open Relay Block lists, which is probably handy if you're 
using a spam-friendly ISP that gives you lots of bandwidth cheaply, but 
is blocked by spam lists.
