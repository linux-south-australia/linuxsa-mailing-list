Re: [NEWS] .NET vs J2EE ?

Ian Loxton wrote:
> 
> Ask Chuck: Is .NET a rip off of J2EE? 

Well, at least one of his points is completely wrong:

"1998 Microsoft releases MSXML; in 2001 Sun releases JAXP"

AFAICT, MSXML is an XML parser and XSLT processor implementation, 
whereas JAXP is an API to allow you to use conforming parsers and 
processors in an API-independent, pluggable manner. He should have veen 
comparing MSXML to the likes of Xerces, Crimson, Saxon, Xalan et al, or 
perhaps even to SAX.

"1997 Microsoft releases Microsoft Transaction Server; in 1998 Sun 
releases EJB"

There is more to EJBs than just transactions, but I don't really know 
what else "Transaction Server" is supposed to do.

Most of the others sound dubious as well, although I don't know enough 
about MS products to say authoritatively. It sounds like he has confused 
a vendor-neutral API with an actual implementation in almost every case. 
I don't have dates or product names but I assume Sun had implementations 
for those services long before they came up with the vendor-neutral API.

MS came up with such a great question to ask. ;)
