RE: /var/mail & message forwarding.

Hi,

You could set up a cron job to run the formail command below on the machine
with the finance package.  This will read the mail file in question and feed
it back through sendmail for delivery to a different email address.

formail -Y -s /usr/sbin/sendmail <username>@sageautomation.com.au <
/var/spool/mail/<username>

Jim.

Subject: /var/mail & message forwarding.



I have a finance package running on a Red Hat 7.3 server.  It uses mail to
send a small amount of information to the end user, however more
importantly, it sends error reports and other things like that to me, as a
user of the system.

I have set my virtusertable to forward all mail on the system to our
primary MX, so now whenever mail is sent on that system to a local
username, it's forwarded by sendmail to username@sageautomation.com.  That
works you-beaut, and it means I can pick up automatically generated emails
and things like that.

Unfortunately, I think that for some reason, PRONTO (the finance package)
writes directly to /var/mail/username and I have determined this because it
is the only application that can avoid having its email forwarded via
sendmail.

Is there any way I can prevent this?  The mail remains stored in the mail
files, and I'd rather push the mail to our mail server rather than pick it
up remotely.

Does anyone have any suggestions as to how I can prevent this, or get
around it??
