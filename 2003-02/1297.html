-------------------------------------------------------------
On a related subject, I wrote some Apache software which returned
the status code 503 (Service Unavailable). This code is supposed
to indicate server overload at the very least.

(HTTP codes are here for the curious: 
http://www.w3.org/Protocols/rfc2616/rfc2616-sec6.html#sec6.1.1 .)

But I found that an awful lot of rampant downloaders were
programmed to try again immediately if they get a 503 code in HTTP.
Instead of backing off, they became more aggressive!
So I changed the code to 200 to make them think that they had
already got the file. Then this aggressive behaviour stopped.
So the result is that I'm operating a web server which uses the
wrong status codes because the correct ones receive such a stupid
response by some software.

-------------------------------------------------------------
Now returning to the STMP status codes, I was wondering if there
was some code other than 550 which would cause the calling
software to think that there was no need to make another dozen
attempts as they do now.
