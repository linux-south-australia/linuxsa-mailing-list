Measuring the average speed of an operation looks
something like this in code:

record the start time.
loop N times
record end time.

average time = (end - start) / N

with accuracy increasing as N gets larger.

Knowing exactly how big your run should be to
guarantee  accuracy is dependent on what you are
doing. Maybe instead of trying to figure that out, you
could also average over the total program run time
(spit out an update of this every 10 secs or
something) and see when the numbers converge.
