webapp: optimise image

Hi.

I have a script that I run from cron each night that optimises any jpegs
in a given upload directory to 480 pixels high and 70% quality which
works fine, but I'd like to give the user an interface to send the file
and have it sent back in the same session, resized as above.

I don't know enough perl/php to do this, but if someone could get me
started, I'm happy to tinker until I get it running smoothly..

The command I have in the (cron) optimise script is:

djpeg -pnm -dct float $file | pnmscale -ysize 480 | cjpeg -quality 70 
-optimize -progressive -dct float > $OUT_DIR/$file

Which is an adaptation of the command found in thumbs.sh by Adam Kopacz,
found in his image_pack download at http://www.kLoGraFX.de/ (Policy: GNU
GPL) The script is reducing 380k scans to about 10% of their original
size.

I guess I see this working with a web page that has a browse button on
it that the user can use to select a jpeg on their local system, then
upload it to the web server.

The web server saves the file in /tmp, runs the optimse script on it and
sends the result back to the users browser session with slightly
modified filename.
