Updating 10 machines from source is no fun at all.

I gather that this is not fantastically urgent though, because
it requires a man-in-the-middle attack.
And I assume it doesn't affect my plain "ssh" login on port 22.
This seems to be confirmed at slashdot:
http://slashdot.org/article.pl?sid=03/02/20/1956229&mode=thread&tid=93&tid=172

It seems to be a limited range of web server SSL setups which are affected,
and only under active man-in-the-middle conditions.

So the port 22 SSH doesn't seem to have any problem, and the port 443
problem is not in the "fix in 6 hours or be hacked by worms" category
as some previous OpenSSL and OpenSSH problems were.
If I'm wrong about this, please let me know.
