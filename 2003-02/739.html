RE: pcmcia crashes

Cam,
I've had a few similar experiences with PCMCIA.

One thing that the Linux PCMCIA software does is scan the machine for free 
memory, I/O ports and interrupts.

Most likely the machine is crashing when it's doing this scanning.  (Some 
devices are especially sensitive to this.

My suggestion is: 
- boot up without PCMCIA 
- have a look at /proc (eg.
/proc/ioports, /proc/iomem, /proc/interrupts) to see what's being used 
- edit /etc/pcmcia/config.opts and add "exclude" lines for anything being
used - make sure that only the minimum area is scanned for each type of
resource. 
- at worst your card will be detected and won't be able to load because of 
not enough resources. 
- you could try to then backout each exclude line to figure out which device 
goes nuts and locks hard. (my problem has been IDE controllers - especially 
older ones).
