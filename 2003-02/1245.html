I didn't realise we were talking about C++ (because I jumped into this
thread well past the half way mark!)  For the benefit of those amongst us
who don't read C++, the "int j=0" part of each for statement introduces
a new identifier, j, which is initialised to zero.  I think the scope
of that identifier is the scope containing the loop, and that j should
continue to exist after the loop completes, hence what you really
wrote is:

	switch(example)
	{
		case 1:
			int j = 0;
			while (j < someval)
			{
				someshit();
				j++;
			}
			break;
		default:
			int j = 0;
			while (j < someotherval)
			{
				someotthershit();
				j++;
			}
			break;
	}

Expressed this way it's clear that MSVC is exactly correct in all
reported errors.

Unfortunately I don't have an authorative C++ reference handy to look up
the defined scope of identifiers declared within the initialiser of a for
statement.  I did find http://www.iplusplus.com/CppRef/For_Statement.html
however they might also be wrong.
