Re: File unsorting.

On Friday, February 7, 2003, at 10:24 AM, David Lloyd wrote:
> ..run the { rand(3)-1 } part only ONCE or (wc -l filelist) number of
> times? If the { } part is run only _once_ the comparisons will be the 
> same
> for each one and you'll only get 3! results.

The { ... } block is run for every comparison, and the number of 
comparisions depends on the sort algorithm. If the sort was bubblesort 
(which it isn't), then it could conceiveably run forever since it'll 
never be flagged as being in order. As it's Quicksort, it's done by 
recursive partitioning with a defined runtime of O(n log(n)). So it's 
closer to your "wc -l filelist" number of times.

Ruby is as easy to read as Python and even more concise than Perl.
