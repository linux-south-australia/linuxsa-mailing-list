For starters there is no such thing as a "EOL" char. There is however Carrage Return (0x0A) and Line Feed (0x0D). Unix authors decided that only one char was needed to signify end of line. DOS authors decided that text files should be stored as they would be sent to the printer, because converting them as they send the file to the printer was way to complicated for them.
Line Feed 
          goes down a line, like this. (hope you have a fixed width font for your email client)
Carriage Return brings the cursor back to the start of the line, without going down a line, so it can't be represented in email without TeX or images. Ultraedit is a windows text editor that performs the conversion. Though personally I wonder why you would need to convert it, as any program worth using supports unix text files. Here is a C program to perform the conversion, it will run pathetically slow on Microsoft platforms. It will most likely crash if compiled with a Microsoft compiler, due to thier utterly horrid compliance to international 30 year old standards.

int main (int argc, char argv[][])
{
	FILE * fin, fout;
	char buf;
	char lf=0x0D;
	fin = fopen(argv[1], "r");
	fout = fopen(argv[2], "w");
	while(fread(&buf, 1, 1, fin))
	{
		if(buf==0x0A)
			fwrite(&lf,1,1, fout);
		fwrite(&buf,1,1, fout);
	}
	fclose(fin);
	fclose(fout);
}
