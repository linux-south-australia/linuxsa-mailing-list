Re: Data security - any thoughts?

Jon,
Many moons ago I worked in the health side and am aware of these issues to
a point.
And currently I am in Retail and I think people are more concerned about
their CC numbers than they are about medical records.

Anyway
1. I suggest no storage of flat files be done on the web server. Use some
form of data reader accessing a DB not on the web server and dynamically
construct what is required on demand. 
2. Secondly you should do this whole process using HTTPS, it will ensure no
sniffing of data.
3. Obvious login/authentication mechanisms to access a restricted area
following HTTPS connection.
4. Encrypt the data using PGP ie the DR's public key so that the DR will be
the only one able to read it.
5. The DR can then use their own private key to read the data

You can't do anything once the MRs hit the destination machine short of a
proprietry applet to read/display the information which will use RSA/pgp
encryption based on the DR's pgp key. The onus then is on the DR to ensure
his database is secure.

evan
