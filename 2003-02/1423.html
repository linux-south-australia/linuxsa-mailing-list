Debian Sarge + DHCP

..hmmm...

I have:

---
% cat /etc/network/interfaces
auto eth0
iface eth0 inet dhcp
auto eth1
iface eth1 inet static
	address 10.0.0.100
	netmask 255.255.255.0
---

And...

% cat /etc/dhclient.conf
send host-name "orthanc";
supersede domain-name "stepney.microbits.com.au";
supersede domain-name-servers 192.168.1.12;

lease { 
	interface "eth1";
	fixed-address 10.0.0.100; 
---

(the --- are not part of the files).

Now every time the dhcp renews eth1 goes and loses its IP address and comes back with some ip address BUT NOT 10.0.0.100. I can't work out WHY it is losing its IP address when it is CLEARLY stated that it should be 10.0.0.100. Even if it does pick up a DHCP address it should get 10.0.0.100. dhclient complains that /var/lib/dhcp/leases (the leases) file is corrup but I haven't done anything to it.

My current kludge is:

# dpkg -r dhcp-client

..because you can't get dhcp addresses if you don't have a dhcp-client but surely there's something I'm missing here.
