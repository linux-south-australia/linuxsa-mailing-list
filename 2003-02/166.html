Re: hyphen in tar and BSD ps [was: RE: gzip vs bzip2]

On Wed, Feb 05, 2003 at 02:58:02PM +1030, Richard Russell wrote:

 > Then, to add to the confusion more, there's things
 > like the Java tools and some SCSI tools (cdrecord, from memory) using
 > single -'s with long options (-version and -scanbus respectively).
 > What is with that?

It sounds like you're being confused more by the misapprehension that
there is a real, solid standard for command line arguments, rather than
by the behaviour of the tools themselves.

If you accept that there actually isn't a standard at all, just a few
loosely followed conventions, then the individual behaviours of tar,
ps, ar, cdrecord, and lots of other commands will cease to be even
remotely surprising.

If "user unfriendly" means "at odds with one's expectations," then 
sometimes it's easier to work around the problem my just doing away
with the expectations. :-)
