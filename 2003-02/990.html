> Hi list guru's, just having a problem setting up fetchmail, I've never done it
> before, always just used kmail straight from my provider, but when I run
> fetchmail it refuses to connect, I've really got no idea where to start, hope
> someone can help.

Use fetchmail and procmail to do it, it's nice and simple.
Create a .fetchmailrc like this

poll pop.senet.com.au	<-----check this is correct for your mail server
protocol pop3
username 	<------put your username here
password	<------and your password here
mda "procmail -d %s"

and a .procmailrc like this

PATH=$HOME/bin:/usr/bin:/usr/sbin:/bin:/sbin:/usr/local/bin:.

MAILDIR=/home/rh/mail   <------change this to the correct pathname

DEFAULT=$MAILDIR/rh	<------and this!

The .procmailrc contains a lot more flash stuff than this but in my version
it is all commented out as all mail goes to the one mailbox for reading
with mutt.
