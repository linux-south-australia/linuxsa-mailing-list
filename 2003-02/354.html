Re: hacked

In the immortal words of Matthew.Cook@tekelek.com.au...
> Do a file search on your machine for files with a creation date after
> a known "non hacked" date, ie over the last 7 days.   You should be
> able to pick which files have been modified, you will find ps, ls, top
> and a few others are likely candidates.    Replace all of these off
> your RH7.1 CD. Anything that you dont recognise do a search on google,
> it will help to identify which r00t kit you were hit with.

Most rootkits change the file timestamps to the original file's
timestamp using touch.  this method probably wouldn't work at all.

Simplest option, if it's a standard redhat box, try using
rpm --verify -a
with results similar to
S.5....T   /bin/chown
S.5....T   /bin/cp
S.5....T   /bin/dd
S.5....T   /bin/df
man rpm will give you more details about these listings.  
this should give at least a hint as to which files were modified.

I haven't heard of a rootkit which actually changes the RPM database,
but it's still possible.
