nah.  sorry to confuse.  linux box, sendmail, pop from dialup.  outlook
clients (and the rest)

i wondered if there was any mail clients that could do exactly what you
said.  use a greater than symbol to see if the score exceeded a certain
number.

there is an option in Spam Assasin that turns on X-MailScanner-SpamScore
which doesn't put out numbers it puts out a number of s's which shows what
score it got.  so if a user wants a tighter filter than the server (we dump
our spam to another mailbox for the time being), they can create a rule that
says if message headers contain 'sss', then their mail level would be 3.0
(obviously decimals arn't going to work  :)) and above.
