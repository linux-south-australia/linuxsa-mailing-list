Re: [OT] Cisco reboot record

Richard Russell wrote:

>Quickie for Cisco types: If I wanted to find out the last few restarts
>for a router, how would I go about it (can I do it?)
>  
>
Some of the Catalyst switches will show the last 5 reboots with 'show 
log', but I havn't noticed anything like that for standalone IOS-based 
routers. Look at configuring a syslog server for logging the messages of 
any suspect routers. If someone is using MRTG to monitor the router, 
gaps in the data sample times can also indicate a reboot, along with 
SNMP timeout messages that may have been sent by crond.
