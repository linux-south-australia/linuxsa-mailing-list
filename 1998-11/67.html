hey guys.

We have a linux box (a very small one) mounted on a wheelchair, that we want to be able to boot and control the console with a serial line.

this is the way we see it:

(1) the machine turns on
(2) the lilo prompt is sent to the serial line
(3) the timeout is reached, and the machine boots the default kernel
(4) the kernel boots, and all printk() statements go down the serial line.
(5) the wheelchair controller software starts up
(6) the serial cable is detached (after logging off i guess)

sometime later.....

(7) the serial cable is re-attahced
(8) an operatoir logs on and does stuff
(9) goto 6

(yeah i know - goto's and infinite loops are bad programming practice :))

the machine is not secure - nor does it need to be.

its running RT-linux (real time linux - a patch that is really cool) and controls an AD
