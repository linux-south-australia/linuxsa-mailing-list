Re: Routing table vanishes on reboot

> When my system comes up I add the routes to the outer net, for example:
> 
> route add 192.168.1.1
> route add default gw 192.168.1.1
> 
> How can I make this information persistent so that I won't need to type
> it again on reboot?

Erm, you shouldn't really have to do that. Route should work itself out.

But anyway, just add it to /etc/rc.d/rc.local. If you're init thingo is
funny, and you're rc.local isn't there, just do a "locate rc.local". If
that still doesn't work, try and find your own local file manually.

What you're looking for is one of the files which is automatically
executes every time on boot. It might be rc.d, but most of the time it's
rc.local because most people like to keep their local this-site-only stuff
away from global config files.
