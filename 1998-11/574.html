> 
> Does anyone have access to a memory tester or is there some thing I can run
> in Linux that will test RAM memory thoroughly.
> I suspect a faulty SIMM module, 32Mb EDO x 4 even if I have to take them
> all out bar one and test each one individually. (my mother board will
> support one module at a time).

A not so precise method is to compile a kernel.  It tends to exercise 
memory fairly well, and is quite prone to memory errors causing SIG 11s
(Segmentation Fault).  Run a couple of those back to back overnight and
you might be able to track down which chip/slot has the problem.
