Running pine remotely - summary

Thanks to all who responded to me. The trick here is instead of running an
xterm locally and trying to get the remote computer to display something
in it, use rsh to run the xterm on the remote computer (which I guess
works because xterm doesn't need stdin/stdout) and run pine in that.

Thanks to David, his solution is below:

On Tue, 17 Nov 1998, David Robson wrote:

> You should be able to script this, but basicly this should work:
> 
> rsh <host> "export DISPLAY=local:0.0; xterm -T Pine -e /usr/local/bin/pine"

	- Justin
