>The proc file system is really a window on the kernel.  Do the following
>'cd /proc;df -k .' and you will get
>
> none 0       0        0      0%   /proc
>
>and 'mount' gives
>
> none on /proc type proc (rw)
>
>For a more in depth discussion read the proc man page, picked up by doing
>a 'man proc'.  Note it also it talks about the kcore file.
>
>So in summary look elsewhere for the files that are taking up space.
>A 'du -sk --one-file-system *' is a good start to work out which particular
>directory is consuming space.  My guess would be /var or /home.
