Broken ownerships behaviour in Star Office

I am trying to setup a shared work folder for 2 people to edit files with
StarOffice. I have created a new group called shared, and changed the
group permissions on the file I want to edit:

-rw-rw-r--   1 jhawkins  shared       1499 Nov 18 20:26 test.sdw

If I use pico I can edit this file, save changes, and it doesn't change
ownership.

If I use StarWriter, when I save the file the ownership reverts to
this:

-rw-rw-r--   1 jhawkins  jhawkins     1490 Nov 18 20:56 test.sdw

Is this behaviour broken? 

My theory is that it mv's the file elsewhere for a backup and rewrites it
out, thus giving it a new ownership. Is there any way around this? 

If it is broken behaviour I'll be quick to report it to Star Division :-)
