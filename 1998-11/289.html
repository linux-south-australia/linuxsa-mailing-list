Re: Linux = NT 4.2?

You wrote:

> Why does my linux box (rh 5.0) show up in NT server manager as a NT 4.2
> server? My school's sysadmin asked me and i have _no_ idea why it would show
> up in the server manager, let alone as an NT 4.2 box. (which i don't think
> is possible.)
> 

I'm just hazarding a guess here, but I think it may be either of two
things:

I'm assuming that you mean with samba filesharing and suchlike, btw.
(samba being the protocol which most windows puters use as default to swap
files, mount network drives, etc)

1. The NT server is sending out a broadcast samba request, receiving your
RH samba info, and thinking "Eh? What's this?", and then defaulting to NT
4.2.

2. As before, but I think there's some sort of samba windows bug which
makes RH reply as NT 4.2 for things to work properly. Thus RH is detecting
that it's a blows server asking for its info, and responding accordingly.
