SOLVED! Config Probs, Apache and DNS

Just to keep everyone posted. I took my machine into the LinuxSA and Michael
was kind enough to figure out what I ballsed up. Thanks Michael.

The problem with the DNS was that I forgot to declare my domains in the
/etc/named.conf file. Stupid me... I'll know next time... lol

The problem with apache was that they have changed the format of the conf file.
so instead of looking like:

----------------------------------------------------------------------------------------

<VirtualHost www.bachblue.com.au>
ServerAdmin darryl@bachblue.com.au
DocumentRoot /www/www.bachblue.com.au
ServerName www.bachblue.com.au
ErrorLog logs/www.bachblue.com.au-error_log
TransferLog logs/www.bachblue.com.au-access_log
</VirtualHost>

<VirtualHost eudunda.bachblue.com.au>
ServerAdmin darryl@bachblue.com.au
DocumentRoot /www/eudunda.bachblue.com.au
ServerName eudunda.bachblue.com.au
ErrorLog logs/eudunda.bachblue.com.au-error_log
TransferLog logs/eudunda.bachblue.com.au-access_log
</VirtualHost>

----------------------------------------------------------------------------------------

It has the form:

----------------------------------------------------------------------------------------

NameVirtualServer 210.8.138.25

<VirtualHost 210.8.138.25>
ServerAdmin darryl@bachblue.com.au
DocumentRoot /www/www.bachblue.com.au
ServerName www.bachblue.com.au
ErrorLog logs/www.bachblue.com.au-error_log
TransferLog logs/www.bachblue.com.au-access_log
</VirtualHost>

<VirtualHost 210.8.138.25>
ServerAdmin darryl@bachblue.com.au
DocumentRoot /www/eudunda.bachblue.com.au
ServerName eudunda.bachblue.com.au
ErrorLog logs/eudunda.bachblue.com.au-error_log
TransferLog logs/eudunda.bachblue.com.au-access_log
</VirtualHost>

----------------------------------------------------------------------------------------

This, presumably, is so that you can run a mixture of IP-based and name-based
virtual hosting. For example, you have two domains that have different IP
addresses, but each domain has sub-domains, which don't need an IP address
each. That's what I work out anyway.

And to clear up the record, HTTP/1.1 sends the domain along with the HTTP
request, so it is possible to use more than one domain on the same IP.

Thank you for everyone who helped. The help has been much appreciated.
