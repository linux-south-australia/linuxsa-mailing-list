Re: Sendmail hang

> The problem is that sendmail hangs at boot-time. I remember the solution was
> something to so with DNS, so I put 127.0.0.1 as the last nameserver is
> resolv.conf, and even as the first nameserver, but to no avail. (odd, because
> that worked on another computer I fixed with the same problem!)

The problem is most likely caused by sendmail trying to resolve the
local IP address for its hostname.  To solve the problem, you can
either fix the problem that is causing your DNS server to be
unreachable (misconfigured networking setup?), or you can modify the
/etc/hosts file to include your machine name and IP address.
