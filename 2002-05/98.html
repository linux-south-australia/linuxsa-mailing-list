Re: Samba delay in Netware network - Follow Up

James,

I was involved with a site of 500+ users having slow network response with
Windows 2000 clients and an NT server.  I spent a good week there to
analyse with a packet sniffer what was happening.  After some investigation
on th client side, it was found that the "My Network Places" was being used
with a whole bunch of short cuts.
This compared with two other possible ways to use the network gave some
very interesting results. Network Neighbourhood or mapped drives.
Mapped drives were the fastest. (Even faster if you turn off that View as
Web page crap ie half the amount of data is transfered)
Network Neighbourhood was slower by about half.
My network Places short cuts where about 10-20 times slower!!!!!
Transfering on average 10 times as much traffic.

We (CSC) raised a case with Microsoft and found out that it had been
reported to them by companies like DuPont (read 20,000 desktops with the
problem) and still M$ wasn't going to fix it.

So Windows2000 has issues with browsing the network full stop if using the
new My Network Places. The advice from the M$ engineer was to upgrade to
XP.

Make sure other things that cause delays like name resolution are working
well too.
