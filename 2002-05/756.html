Done that -- worked a treat.

Only complaint I have is that it would be nice if there was a more
intelligent way of dealing with changed config files. Same with FreeBSD,
though it's a bit different.

Debian's problem is that if a config file was script-generated, or
modified by you at all (ie, it differs from the old installed one at
all), then you have to manually check it. It's frustrating when it's
been a long time, and you have to go through each line of say
squid.conf... especially if something's changed about the format...

FreeBSD is more irritating, in that it doesn't know if you've changed it
or not, it just knows it's a config file, so asks you if you want to
install the new or keep the old, or edit... very frustrating when
there's 200 files, most of which you haven't seen before, and it's an
older machine, so you don't know what might have been changed earlier.
At least much of the configuration is kept in one file as shell
variables, and each config file uses this info in its' own format.
