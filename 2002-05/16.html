Re: authentication

Quoting Daniel Cumming <danielc@statetech.com.au>:

> I plan to replace a windows small business server with a linux box. There
> are 8 win2k pro clients. The services it needs to offer are PDC, file and
> print sharing(samba), mail(postfix, cyrus), firewall + internet
> sharing(iptables), proxy(squid, squidguard), webmail(apache, horde, imp). I
> plan to use ssh and webmin to administer it. I intend to do it using
> mandrake 8.2.
> 
> My question is user authentication, what is the options to tie this all
> together? ldap? mysql? password files? what has been used and known to
> work?
> Also could be scalable for more users at other sites?

I'd create a local account for each user and let things authenticate off of
/etc/passwd, as they normally do. From there, you'd only need to create Cyrus
IMAP accounts and make sure that smbpasswd and passwd operate well together.

AFAIK, scalability with /etc/passwd only really becomes an issue at the
thousands of users mark. Also, while fancy, authentication using LDAP or SQL is
often difficult to configure for software which doesn't natively support it
(although, PAM could be used, I think), and is slower than using the default
passwd mechanism.

In the end, it's a question of how scalable you want it to be.
