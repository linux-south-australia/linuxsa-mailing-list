RTF + Winmail.dat + MIME Defang + NAI

Hi,

We have some users who insist on using RTF for Outlook98 as this is required
for them to be able to share / synchronise meetings and Schedules etc.
Without RTF enabled this feature does not work ( ggrrrr ).

Our Mail server is also running MIMEDefang and NAI Antivirus which links in
with MIME Defang to check attachments. Last week it started quarantining
winmail.dat files ( See below )


Here are the headers for quarantined part 1:
Content-Type: application/ms-tnef;
	name="winmail.dat"
Content-Transfer-Encoding: base64
Content-Disposition: attachment;
	filename="winmail.dat"

----------
Here are the warning details:

The attachment 'winmail.dat' contains a known virus.  It has been
quarantined.

We have also had some XLS files and Even JPG files quarantined as having
supposed virii. Files have been checked with the latest Virus signature
files / engines and all are clean.

Any MIMEDefang Gurus out there who can shed light on this.

P.S.  We dont have the desktop AV ( McAfee V4.51 ) doing it's own email
virus scanning as when that was enabled MIMEdefang was reporting any and
every attachment as being a virus - probably due to a doubling up of
attachment disection by Mcafee and MIMEdefang
