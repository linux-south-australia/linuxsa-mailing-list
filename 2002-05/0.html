Re: ldap schemas

On Tue, 30 Apr 2002, Toby Corkindale wrote:

> Did you try this search?
> http://www.google.com/search?q=ldap%20rfc

This only gives IETF schemas, not that useful as the IETF has deliberately
avoided standing on ISO's toes, so the IETF RFCs reference ISO schemas
rather than replace them.

Try
  http://ldap.akbkhome.com/
for a very nice schema viewer.

Note that the ISO schema have some design flaws, and if you apply the 
schema in the obviously intended way you will run into problems.

For the modern abuse of the ISO+IETF schemas see

  http://www.georgetown.edu/giia/internet2/ldap-recipe/

which refers to eduPerson, a schema at

  http://www.educause.edu/eduperson/

The modern schema require an organisationally-wide unique, unchanging user
ID.  In practice getting this installed usually takes more work than
developing site variations on the LDAP Recipe.
