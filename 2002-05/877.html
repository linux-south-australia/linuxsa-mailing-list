FTP Server timeouts

Hi,

Looking for a possible reason for this occurring.

I've set up an PC running Redhat7.2 at home and have run Bastille for RH7.2
on that PC. Have stopped all unnecessary services, and set up Pureftp on
that PC. I am using a non standard Port 6700 and set up PureFTPd with
Authenticated users only, specified passive port range also.

Connected the PC directly to the net and got a friend to see if they could
upload to the FTP. The idle timeout is set for 15 minutes. They connect OK,
but it kicks them off every minute or so with a message in /var/log/ftp
along the lines of " [INFO] - No activity detected for 900 seconds " and
kicks them off. Their FTP client auto reconnects and resumes uploads.

Is it possible that something in the Bastille hardening process could be
causing this. When connected to the net the PC is invisible and does not
respond to Ping/Tracert requests.
