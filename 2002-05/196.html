Re: Perl Question

Quoting "Wilkinson,Alex" <Alex.Wilkinson@dsto.defence.gov.au>:

> Ok, I did this:
> 
> use lib "/usr/local/lib/perl5/site_perl/5.6.1/sun4-solaris/Bundle";

1. Looking at your original question, you're running Perl 5.0005, but whatever
installation script is being used to install the Bundle::* modules thinks that
you have Perl 5.6.1. You'll need to fix this discrepancy before things will
work, as it will cause many more problems down the line.

2. You should never need to explicitly specify a libdir under /usr/*/lib/perl5/.
That sort of thing is done automatically. See point 1.

3. You're specifying a libdir too deep: if, for some reason, you _did_ need to
specify a libdir under /usr/*/lib/perl5, you would use
"/usr/local/lib/perl5/site_perl/5.6.1/sun4-solaris" without the "Bundle".
"Bundle" is part of the module name and will be traversed automatically when a
Bundle::Whatever module is used.

> 
> And now I get this:
> 
> Can't locate object method "connect" via package "DBI" at
> /usr/local/mysql/sql-bench/server-cfg line 238

Weird, but possibly related to point 1, or you just don't have the DBI package
installed.
