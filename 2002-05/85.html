[Musings out Loud] Squid, Netscape and Mozilla

Hmmmmm...

I've just been doing some work on Squid and helping to secure a system
(1) so I thought, I'll run Squid at home. Now, every week I generally
download about 80Mb of data from the Internet, or so my ISP tells me, so
I thought that haing a Squid cache of about 200Mb would be ample.

Therefore, I decided to setup /var/spool/squid (2) and got squid going.
I don't use my ISP's squid as a parent because I'm reasonably certain
that they're running Squid transparently and, hmmmm, ummmm...anyway.
Now, as a user I tend to visit a small number of sites (about 10) which
have lots of changing pieces (3). These sites inevitably insist on using
the most graphics they could possibly find and I swear they all use
graphic expanders (the concept is the same as gzip but it BLOATS the
graphics whilst adding no real value).

I run Mozilla and Netscape with 32Mb of disk cache and 32Mb of memory
cache each.

Interestingly enough, when I started to use Squid it actually has made a
really, really noticable difference...strange. There's only about 1Mb of
cached stuff in .netscape/cache .mozilla/.../Cache and about 2Mb in
/var/spool/squid...

Maybe Squid is managing to find pages more efficiently that Netscape or
Mozilla (I wouldn't put it past it).
