Re: win2k + samba = slow browsing The Newbie thinks he has an answer...

To clarify, even if you don't use Novell, you should still look at all 
of the possible reasons listed below.  



James Leone wrote:

> Hi list,
>
> I had the same experience with Windows 2000.  However, I found out how 
> to get Windows 2000 from disrupting the wonderful job Samba does:
>
> The major reasons are outlined in the following links:
>
> 1.)  If in a Novell 4.11 environment, follow the steps shown in this 
> link: (I have only tested novell 4.11, but the guide was made for 
> Novell 5)
>
> http://www.ithowto.com/novell/clientspeed.htm
>
> I found steps 2, 3, 4,5,9 and 10 to be most critical.
>
> Step 2=unbind tcpip from novell and unbind ipx from the microsoft 
> windows client, click on the red N, go into novell client properties, 
> protocol properties, and make IPX the default protocol (Novell 4) and 
> turn off IP, I think the opposite is true of Novell 5, but I am unsure.
>
> Step 3=if on a network with a hub, set adapter to half duplex 
> only<----------------------------------------------------NOVELL OR NOT!
>
> 4=Disable DFS
>
> Click on Start, type in regedit/ go to 
> HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\MUP
>
> Add a new Dword value DisableDFS=1
>
>
> 5=Use netbios P node
>
> Click on Start, type in regedit/ go to 
> HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\NetBT\Parameters
>
> Add a new value "NodeType=dword:00000002
>
> 9=Disable remote task scheduler
>
> Click on Start, type in regedit/ go to 
> HKEY_LOCAL_MACHINE\SOFTWARE\MICROSOFT\Windows\CurrentVersion\Explorer\Remote 
> Computer\NameSpace
>
> Delete both keys within Name Space
>
> http://www.techtv.com/screensavers/answerstips/story/0,24330,3372700,00.html 
>
>
>
> 2.) If you are using guest accounts on Samba, add "guest only=ok".
>
> That made a huge difference along with adding:
>
