Re: win2k + samba = slow browsing

On Wed, May 22, 2002 at 09:33:11PM +0930, John Rayner wrote:
> 2. I set my win 2K machine to use unencrypted (plain text passwords - of 
> course
>     only any good if security isn't a problem for you).

Of course, if security *is* an issue for you, could always simply add:

	encrypt passwords = true

I also have this, not sure whether it is required:
	
	obey pam restrictions = yes

Also, I noticed this in my conf (added by default samba install):

	socket options = IPTOS_LOWDELAY TCP_NODELAY SO_SNDBUF=4096
	SO_RCVBUF=4096

Perhaps the TCP_NODELAY option here will help with the slow browsing
problems? Again, this is purely a guess as it's one of those "seems to
make sense to me" scenarios.
