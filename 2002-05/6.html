Printing HTML mail from the command line

Hi Folks,

I have been using soupermail to process complex web forms
and send the results as mixed HTML and text emails - with
different "versions" of the info in the HTML and text.  This is
a bit complex at first, but works a treat once you have it set
up right.

I'm now looking to be able to pipe the HTML emails into some
command line program that can render the HTML layout on a
printer (HP PCL) without manual intervention.  In testing I've
been using Outlook Express and manual printing, but we're
now ready to automate this on the Linux server.  Unfortunately
my Linux experience is very much command line focussed so
I'm not familiar with what graphic tools are available...

The web form closely mimics a paper form that is currently in
use, so the client has a strong preference for the "rich" layout
rather than text.

There is also a need to do the same sort of thing with a fax -
HTML email piped to a fax program of some description that
will convert the HTML to a fax image.

Any ideas?
