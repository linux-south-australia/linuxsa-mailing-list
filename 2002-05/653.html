RE: win2k + samba = slow browsing

I had lots of problems with samba not being able to find things on the network
until I did the following (not all at once, it was a slow process of trial 
and error):

1. As mentioned by others, I got rid of NETBUI.

2. I set my win 2K machine to use unencrypted (plain text passwords - of course
     only any good if security isn't a problem for you).

3. I made my Linux box the master browser for the network (part of my 
smb.conf file
     follows):

[global]
    workgroup = WORKGROUP
    netbios name = ZORBA
    server string = Samba Server - Home Network
    hosts allow = 192.168.1. 192.168.0. 127.
    log file = /var/log/samba/%m.log
    max log size = 100
    security = user
    socket options = TCP_NODELAY SO_RCVBUF=8192 SO_SNDBUF=8192
    local master = yes
    os level = 33
    domain master = yes
    preferred master = yes


ZORBA is running 99% of the time so is always the master browser. This 
seems to work
OK for me - no browsing delays and I find everything on my home network 
every time!
