> log on and seems to keep coming up about every 2 minutes, but it doesn't

Okay, I assume that you are on the a virtual terminal.  Most likely the
first one (available via ALT-F1).

gdm is the GNOME display manager (an alternative to xdm).  It is a program
which manages and controls the X windows display.  The messages are getting
emitted by gdm, which is a user level program (and absolutely nothing to
do with the Linux kernel).

> this is how it goes;
> gdm_config_parse: Can't find the gdm user (gdm). Aborting!
> if anyone can tell me what this means and what stupid thing I've done to
> cause it, I'd really appreciate it.

As has been pointed out, an 'rpm -ql gdm' gives you a list of what 
files are associated with the package.  'rpm -qi gdm' will give you a
little bit of a display about what the package is used for.

But anyway, gdm (once started) relinquishes it's root priveledges by becoming
a non priveledged user.  The useris configured in /etc/X11/gdm/gdm.conf.

In this file there is a line

	User=gdm

which defines the user that gdm becomes.  Chances are you have deleted this
user (gdm) not knowing what it is used for.  If you want to get things 
going again, do an 'rpm -e gdm' and then a 'rpm -i gdm*' in the directory that
contains your standard redhat RPMS.

Hope this helps,
