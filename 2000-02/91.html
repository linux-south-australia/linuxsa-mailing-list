> Does anyone have any experience installing a distribution on a machine

No, tho I use one, and so have a little experience fiddling with the
config.

> Now that some modern motherboards with 10/100BaseT/TX onbaord also
> have a netboot option in BIOS, the application that I have in mind is
> where a server with the latest distribution has been installed.. 
> a client netboots (getting all its configuration from the server),
> does a hardware scan, and then installs/reinstalls/upgrades a
> distribution (minimal or otherwise) from the network to the local harddrive.

I don't necessarily see that you need to net-boot for this. You can
achieve this with scripts and/or cron jobs.

> The uses for this range from the PC shop that want a really easy way
> to install linux (or another OS);

That's an excelent idea,I don't know if a Redhat or other install can be
automated, but since the net boot happens by tftp, it shouldn't be too
dificult to create a boot image which starts an install. The Diskless
workstation HOWTO should tell you what you need to know here.

 network devices eg.. thin(ish)
> clients; high reliability server farms; and (my favourite) nodes in
