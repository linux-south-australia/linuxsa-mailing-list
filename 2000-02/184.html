Program works when root, program doesn't work as user.

options.
1) When su-ed to root, it readds roots ~root/.ppprc file, but
~Robyn/.ppprc does not exist, but needs to be created.
2) When su-ed to root, xisp has permissions to write a file or directory
that it doesn when you are you. Hence check permissions on files. Maybe
also check /tmp and sub dirs. Perhaps create a new account, and see if
that person works. Check the man pages for requirements on what files.
3) xisp's permissions have been changed some how (you didn't do a chmod -r,
anywhere in your file structure?). It may have gone from being a SUID/SGID
program to not. Maybe try a reinstall from the original source (rpm, or
source distribution), it may restore file permissions.
