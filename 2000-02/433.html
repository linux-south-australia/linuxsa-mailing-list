Re: fwtk2.1 compiling error.

emyeu wrote:

this error:
> cc -I.. -O2 -DLINUX -s -static -o http-gw hmain.o
> In file included from child.c:14:
> ulib.h:26: X11/Intrinsic.h: No such file or directory

is caused by the fact that it cant find Intrinsic.h

looking at the compile line, the options to look for inckude files, -I
is ".." which means the parent directory.

Basically, its not finding the include files.

If the file child.c has an include line like this:
<X11/Intrinsic.h>

then you will need have that Intrinsic.h file in
/usr/include/X11/Intrinsic.h Of course, you can have the file anywhere
you want, but you would have to provide the correct parameter into the
-I option of your compiler.


> # Location of X include files
> XINCLUDE=/usr/X11R6/include

so here is where the X include files supposedly are. Ok - but your make
file isn't using this option. 


So, I think your missing a step between editing your Makefile.config and
running make. To be honest - I'm not sure how to get from a
Makefile.config to a makefile..
