Partition advice

Hi Folks,

I'd be interested in any comments people have on an "optimal"
partitioning scheme for a Linux box that is to be remotely
operated.  I expect to have only a single 8 Gb drive, but may
be able to add a second 100 Mb or 200 Mb drive now, and maybe
a second 8 Gb later.

I have it in mind to duplicate all partitions so that I can
run overnight partition to partition backups (maybe disk to
disk later).  I assume that some magic involving extra root
partitions may be required to satisfactorily update kernels
remotely (yes, I know this could be dangerous), and also to
allow a reboot on a "spare" root so that the normal root may
duplicated.

Also any comments on how to use LILO to setup reboots to a
different partition remotely would be great!

If you run a remote server, have you found it necessary to
use a serial console port connected to a modem/NAS, or were
you able to reliably use a network connection without to many
physical visits?

Cheers, Dieter.
