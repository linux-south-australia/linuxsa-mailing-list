Re: Logrotate Question

> Isblank does not belong in unistd.h; it belongs in ctype.h.  Any file that
> references isblank must #include <ctype.h>.  Any program that fails to do
> this is either wrong or non-portable.

As I think about this I realise that isblank is already non-ANSI and so is
already non-portable.  Standard C has isspace, which includes vertical
whitespace as well as horizontal whitespace, but provides no macro to test
specifically for horizontal whitespace.

As far as I can tell isblank is a GNU extension.  I don't know when it was
introduced.  I have only seen isblank implemented via ctype.h, which is as
it should be.  If you don't have isblank in ctype.h you need to innovate
(or upgrade).
