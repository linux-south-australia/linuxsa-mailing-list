Does anyone know a way of getting SuSE 6.2 to *please* not kill my other
partitions? My desired partitioning is:

    /dev/hda1        Solaris 2.4
    /dev/hda2        MS-DOS + NW server boot files
    /dev/hda3        Netware 4.1
    /dev/hda5        Linux swap
    /dev/hda6        SuSE 6.2

I can't just install Linux first, as Solaris is a pain in the arse and
won't install if there is a partition towards the end of the disk.
Solaris also doesn't like the end half of my disks (it's not a BIOS
thing).

For some reason (even if I create the partitions before booting the SuSE
install), SuSE insists on creating partitions for me. The problem is
that they are the partitions that I already have data on. It doesn't try
to make them linux partitions, but I can't access the other operating
systems after. It does it even when I leave the partition table intact.
There is no option to tell it to *please* not kill my other data. I
actually don't mind SuSE, but this really pisses me off ;-)

RedHat allows this config.

Thanks,
