> A simple question. I have two hard disks and would like one to be the
> storage and the other to be the operating system. For simple back-up i

You posed no question.  You said what you wish to do; do it.  What's the
problem?

> Also, what is the best procedure to backing up ON THE NEEDED FILES. My aim
> is to be able to reinstall the OS and reinstate these files and basicly have
> the OS as-it-was.

If you are convinced that all operating system files are older than some
date, and that all user and configuration files are newer than that date,
which is generally true, then you can use the find command to generate a
list of older files and a list of newer files.

After installing your OS, but before adjusting the configuration and loading
user files, create a "timestamp" file:

  # touch /timestamp

When you wish to backup all necessary files use find to generate the list
of files to backup:

  # find / -newer /timestamp -print

You might wish to only backup normal files and directory files, that is,
not include device special files, named pipes, sockets and the like:

  # find / -newer /timestamp \( -type d -o -type f \) -print

I rather like cpio for backups:

  # find / -newer /timestamp -print | cpio -o > /dev/tape
