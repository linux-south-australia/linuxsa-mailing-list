I would personally go with LDAP. NIS is easy to configure, it is also
compatible with a number of flavours of commercial Unix. OpenLDAP
(http://www.openldap.org/) is RFC compliant, which means that it is going to
be fairly portable between Novell's Directory Services, Netscape's Directory
services and even Microsoft's ACtive Directory. Obviously these other vendors
have a tendency to "extend" the functionality of the standards, so you won't
get complete compatability.

NIS and NIS+ have had a fair amount of security problems in the past, where
LDAP doesn't appear to have the same. LDAP supports Access Control Lists, so
you can give different users different permissions.

As well as using LDAP for the same stuff you can with NIS, software such as
Samba [bows down to the mighty Samba Team. "For thou hast done some pretty
cool stuff and things"] support LDAP. Netscape and IE as of versions 4 also
support LDAP address books.

A lot of bigger companies are starting to pay attention to LDAP (IBM and
Netscape are very big on it), so we will start to see it used more and more.

My opinions.
