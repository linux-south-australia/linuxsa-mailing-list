> _peer myhost.mydomain.com     parent    3128  3130  [proxy-only]
> >cache_peer myhost1.mydomain.com     sibling   3128  3130  [proxy-only]
> >cache_peer myhost2.mydomain.com   sibling   3128  3130  [proxy-only]
> >cache_peer myhost3.mydomain.com      sibling   3128  3130  [proxy-only]
> >cache_peer myhost4.mydomain.com   sibling   3128  3130  [proxy-only]
> >
> >I don't know if this is the right setting for cache_peer.
> >Any input from squid admins appreciated.
> 
> That's indicating that proxy-only is optional.
> 
> A cache_peer line should look like:
> 
> cache_peer myhost1.mydomain.com     sibling   3128  3130  proxy-only
> 
> if you don't want to cache objects from a neighbour, or:
> 
> cache_peer myhost1.mydomain.com     sibling   3128  3130
> 
> if you do.
> If you don't have any squid neighbours, you may want to use 'parent' rather
> than 'sibling' to an upstream cache, or have no cache_peer lines at all.
> 

Thanks for replying. Since I do not have caching peers, I removed these tags
from my config file and squid works fine now.
