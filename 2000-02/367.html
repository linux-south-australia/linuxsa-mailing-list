Samba behaviour when path not specified and path wrong

Hi,

This message is sent to clarify a something that occurred on the recent
Samba course I held here in Adelaide, and since the answer is probably of
interest to people on this list, I thought I would post it more generally.

If you do not specify a path for a share, Samba defaults to /tmp as the
path.  You can check this with testparm.

However, if you do specify a path, but forget to create the directory (or
misspell it) or you do not have access to the path, then you will get an
error message back from Windows when you try to connect to the share.  This
message is the Following:

   \\server\share

   The network name cannot be found.

This can be most confusing, and will appear in a FAQ that I am producing.
