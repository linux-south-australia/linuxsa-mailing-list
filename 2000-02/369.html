Ethereal and filtering packets ...

I thought that this message had general relevance to LinuxSA members,
although it was sparked by a question on my recent Samba course.

We wanted to filter packets after they had been captured, and select all
those packets in a particular subnet.

For each protocol, you can select packets in that protocol, simply by
typing the lowercase name.  So 

- to select SMB packets, type smb into the filter window and press enter. 
- to select select BROWSER packets, type browser, and press enter.
- to select telnet, type telnet, and press enter.

Now, to select all packets to or from a node in a particular subnet, simply
type ip.addr == a.b.c.d/prefix len, ie

    ip.addr == 10.0.1.0/24

will show you all packets to or from anyone in 10.0.1.0/255.255.255.0

Also, since Ethereal can read in tcpdump capture files, you can capture
using tcpdump on someone's network, mail the dump file to yourself, and
then examine at your leisure.

To capture using tcpdump, use the following:

   tcpdump -i <your interface> -s 1500 -w somefile

Your interface can be anything, including ppp0, if it is up.  The above
works on FreeBSD and other *nixen as well.
