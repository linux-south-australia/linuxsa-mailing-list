RE: packages

I know what it's like to work non-profit orgs. Tight budgets. Here are two
ideas:

"Distributed Computing"
You could use NFS to store stuff on various PCs, (documents on one PC, data
on another, apps on a third, etc) but that might get messy, in a distributed
sort of way...

"Thin Clients"
- boot linux on each client from a disk, like the Linux Router Project
distro
- mount an NFS share to a "server" consisting of the hard drives you pulled
out of the floppy-based PCs
- this shared location would be of a good size, with full XFree86 and any
apps and data installed
- have your floppy-based clients start X from the network share.
The added advantage to this one would be that if all PCs ran NE2000
compatible ethernet cards, all boot disks could be the same (DHCP)
