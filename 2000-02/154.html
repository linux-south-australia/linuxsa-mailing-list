lpd and zero length files

Greetings all,

I'm currently having problems with lpd, on both a server and on a
client (on a different network).  

Zero length control files (cfxxxxx) appear on the printer queue on
both machines every now and again.  I currently just delete them when a
user complains, and restart lpd.  Sometimes this fixes it, sometimes I
have to repeat these steps several times.

The print server machine is receiving print jobs from both SunOS,
Solaris, Windows95 (via Samba) and Linux, but the client machine
receives only local print requests and is experiencing the same
problem. Both machines are standard RH 5.1. The print jobs are then
being sent to HP laser printers (HP4M and HP5MV) via their lpd
socket. (No jetdirect involved.)

All suggestions welcome.
