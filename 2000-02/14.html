request for tcpdump log samples

Could someone with heavy-ish traffic on
their Internet access link please send me
a tcpdump log file?

I realise that this could possibly comprimise
security/privacy. So I would be happy to just
send you my log file analysis program to
run on the log file, so that you could just
send me the PostScript file outputs.
Alternatively, a command such as

tcpdump -c 100000 -i ppp0 -s 58 -o log0.dat

should be fairly safe.
(14 bytes for each pseudo-ethernet header,
and 44 bytes for the IP and TCP|UDP headers.)
Safer woudl be "-s 54" to avoid getting any
TCP payload ever. Or "-s 42" to avoid even getting
any UDP payload, but still saving the port
numbers.

Motiviation:
My purpose is to try to understand
how bursty Internet access traffic is
nowadays,a dn to try to extrapolate this.
In the future, I think that Internet
access traffic is likely to become less
bursty - i.e. it may become a continuous flow
of multi-media traffic.

Since linux users are more likely to have
a real live connection to the real internet,
particularly in the server role, it is
natural to seek this kind of log file
from linux users.

The more busy the site, the more I would
like to have log files.

many thanks in advance for any positive responses.
