> On rebooting I got the RH boot window but could not log on, ie CDE would not
> boot up.

You seem to have chosen run level 5 which is the X window login manager.
Alternatively you've typed startx or some such thing and X has crashed.
The easiest way to diagnose X problems is to start X from the command
line. This means you'll have to boot into runlevel 3 or lower. Here's a
way to force this:

[reboot]
lilo: linux single
{computer waffle}
# vi /etc/inittab
{change the default runlevel to 3}
# reboot

Once you've done that, relog and then do something like:

# startx 2> x.log

or 

# X -probeonly 2> x.log

Read the logs. They might give you a better idea of what is going wrong.

> CDE is essentially network based (whatever that means).

X is a network (client/server) based protocol. The server (the thing
that displays the graphics on the screen) serves the client (the program
that is running). 

> My error log file
> contains the message
 
>         I/O error on display:myhostname:0.0
>         dtsession:connection to server lost - exiting

Knowing RedHat installs as I do, they don't set up X if there's one
slight error. One slight error and RedHat X installations go totally
west. You could try using XF86Setup or Xconfigurator..
