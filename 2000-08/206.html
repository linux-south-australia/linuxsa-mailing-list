Sendmail, chroot, non-root and LDAP

Hi,

I've got sendmail 8.10.x running on a test machine as a non-root user in
a chroot environment. This machine is to act as a kind of mail router
using sendmail and LDAP as defined in the ldap-routing internet draft. I
have had both of these working individually, but cannot seem to get them
to work together. The problem I am having is when I send a message that
is supposed to be routed to another server, it gets stuck in the queue
with the error message:

ldap_open: failed to localhost in map ldap_mailhost: no route

This machine is happily participating on the network. LDAP is working
well. The relevant attributes are in the account entry for the address I
am testing. All of the libs (including the LDAP libs) are in the jail.
sendmail starts. I can 'telnet localhost xxxx' and send mail (where xxxx
is the port sendmail is listening on).

Anyone got any ideas? I'd rather continue to use sendmail than something
else.
