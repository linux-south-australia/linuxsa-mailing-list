In addition to doing what Robert advised you may find yourself against
an authentication error. You'll recognise it as soon as you see it:

[root@localhost /root]# xman 
Xlib: connection to ":0.0" refused by server
Xlib: Client is not authorized to connect to Server
Error: Can't open display: :0
[root@localhost /root]# 

Under my RedHat system (and I suspect everyone else's) the first person
to run the script "startx" owns the server. Noone else by default can
use the server. To allow all users who are on localhost to use it, I
have to type:

[lloy0076@localhost /home/lloy0076]$ xhost +localhost

At which time when I su with the dash it allows my connection.

Please consult:

man X
man xauth
man xhost

There is also an X security howto (I think) under /usr/doc/HOWTO
