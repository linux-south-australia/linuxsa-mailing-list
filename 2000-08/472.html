That problem normaly means that the kernel image your trying to boot is corrupted, not compiled properly, compiled for the wrong
CPU, or lilo was not set up properly. Other than the CPU type, it shouldnt be a kernel configuration problem.

You dont need to remove the "linux" dir to start again... just do "make clean", this then also keeps your configs.
"make mrproper" removes all settings.
The config is stored in the file ".config" in the kernel source root dir normaly "/usr/src/linux/.config".
