RE: IP Chains problem & squid

>I've already got the proxy set up and working. The firewall is a new
>addition. When you say enable port 3128 is that to telstra or from telstra
>to my proxy? So far I have allowed stuff out to port 3128 and 3130
>(icp) but I'm not sure what I need to allow back to my proxy. So far I
>have only allowed port 3128. (plus all the other regular ones like dns,
>ssh and so forth)

I am using IPFWADM and I don't need to tell it what ports to allow back in, 
all I tell it is to allow connections from xxx.xxx.xxx.xxx (internal 
network) to 0.0.0.0/0 (all internet) 3128 and that does it just fine.
I am not sure what ports to allow back in, but if its really bugging you 
ring telstra BPD help desk and ask them, or simply do a dns lookup for a the 
range of IP addresses 10 up and 10 down from the proxy cache untill you find 
all the ip addresses of the proxy server, then once you have found the ip 
addresses allow anything to come in from those IP addresses, yes that is a 
slight security risk but the proxy servers would have their own firewall and 
it is highly unlikely that someone will establish a connection to your 
server through the telstra proxy cache. I am not even sure whether it is 
possible.
