Re: c socket() programming - numbers mangled

> i've tried messing with signed vs. unsigned chars, and htons() and
> stuff, but I'm not getting anywhere. It looks like the upper and lower
> nibbles are being swapped around right?

You only need to convert to network byte order integer data types that
are greater than 1 (sizeof(char)). If it's using sockets to the same
byte order on both sides, you don't need to do it either, it's not good
practice, but here two wrongs (ie not converting it either end) make a
right :)
