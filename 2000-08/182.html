>The correct answer is likely to be any of the following
>
>         grep \$ file-to-grep
>	 grep '$' file-to-grep
>         grep $ file-to-grep     (in bash, sh or ksh)

On the assumption that the original question was trying to distinguish
between the following two lines in a file:

   cat <<EOF > some-file
   This line has a $ in it
   This line does not
   EOF

Then you have to escape the $ in the command line in a fancy way, because $
has a special meaning to grep. Like it does in vi, it means end of line.
Another interesting character in ^, which means beginning of line. The same
is true in Perl.

I guess you can see it coming now:

   grep \\$ some-file
