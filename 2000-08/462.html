> Dreamweaver FTP, I can log in, but get "connection refused" when =
> attempting an 'ls'.
> 
> This is really weird, as I'm running another Linux Box at UniSA (works =
> fine) and I copied the FTPACCESS and inetd.conf files over (same =
> configuration). Nada..doesn't work...Any ideas?

FTP is a stupid protocol.  The default when using a command line
version (active mode) is to make a TCP back connection (from the
server to the client!) when data needs to be send (eg. sending the
result of an ls or get).

There are two solutions:
1) use passive ftp on the client.  Passive ftp is what is what web
browsers use.  All connections are made from the client to the server.
You can also get command line ftp clients to use it by typing
"passive" after you login.
2) find out why you cannot make the back connection required for active
mode.  Do you have a packet filter/firewall between the two machines?
