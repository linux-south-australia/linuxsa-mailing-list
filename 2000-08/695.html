I take it that the only workstation trust account for \\ARROW is on
\\BOW.

> C:\> net view 10.10.10.1
> System error 53
> Network path no found
> 
> This occurs for all systems in the Users domain but it works correctly for
> all the Parent Domains.

It's looking like some kind of corrupt WINS database (which happens far
too regularly). Have a look to see if it all matches up. You could also
try adding static WINS entries into the database to see if that
overcomes your problem.

How do the WINS entries for ARROW get to the other Domain controllers?
Are they running WINS?
Is there WINS replication?


> 
> Now I suspect that it is a WINS/BROWSER issue because the Samba(ARROW) box
> is the only one in its subnet (No NT boxes) 

Yes. It is looking like WINS/Browse list stuff. Perhaps Richard Sharpe
can point out the problem?

You could also try setting local, preferred and domain master to yes and
setting the os level to 1. This way, it will become the master of all it
needs to and lose really badly in the elections against everything else.
You could combine this with the 'remote browse sync' option (I think
that's what it is - I haven't been playing with Stable Samba much
lately).
