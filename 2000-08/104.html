Re: lpr port

Richard Sharpe wrote:
> 
> You will need port 515 allowed through the routers for TCP.
> 
> Do you have anything at the remote end that can capture network traffic?

If not, then add the "log" option to your router's access list:

  access-list 100 permit tcp any any eq 515 log

You can then see the SYN pass through the router and run Ethereal
on the local PC to see if a SYN+ACK comes back.

It's also possible that it all works, but that the printer's
queue name isn't "lp".  It's worthwhile comparing /etc/printcap
entries.  It took me three hours to find this the first time I
encountered it :-)
