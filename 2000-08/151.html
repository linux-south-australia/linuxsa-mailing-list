Re: Urgent Apache Config.

Kieran Farrell wrote:
> 
> Help *8)
> 
> Sorry if I'm asking a commonly asked question. I need to configure Apache to use users home directories. Also I need CGI capability again in users directory.
> 
> ie http://localhost/~kjfarrell
> 

ok, what you need are a few things:
UserDir public_html 

this means that to get at /~kjfarrell/foo.html, foo.html will have to be
in ~/kjfarrell/public_html/foo.html

then, you need to give permission for cgi scripts to be run from this
directory:

<Directory /home/*/public_html>
    Options ExecCGI
</Directory>

where "this may also be "None", "All", or any combination of "Indexes",
 "Includes", "FollowSymLinks", "ExecCGI", or "MultiViews".  "

you also need to tell apacfhe who to handle files with extension .cgi
AddHandler cgi-script .cgi 

note that here you can have anything:
AddHandler cgi-script .freddyisthedevil

then restart your apache server.
su -c killall -HUP apache

the permissions should be thus:
chmod a+x ~/home/kjfarrel   
chmod a+rx ~/home/kjfarrel/public_html
