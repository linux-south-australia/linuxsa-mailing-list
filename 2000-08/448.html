Re: WU-FTPD Problems

> "Leonard Teo, Macs" wrote:
> I'm having problems with WU-FTPD running on Redhat 6.2
> (Zoot). I can connect to the ftp server okay locally
> (using localhost), but when I try to connect with my
> WIndows 2000 box, using command line FTP, WSFTP and
> Dreamweaver FTP, I can log in, but get "connection
> refused" when attempting an 'ls'.

You might want to check the /etc/inetd.conf and
/etc/hosts.allow
/etc/hosts.deny files and see if there's anything in there
which 
might be stopping it.

> This is really weird, as I'm running another Linux Box at
> UniSA (works fine) and I copied the FTPACCESS and
> inetd.conf files over (same configuration). Nada..doesn't
> work...Any ideas?

Connection refused errors have (from my meagre experience)
tended 
to be caused by the tcp wrappers and inetd.conf
configuration issues.

Make sure the ftp line in inetd.conf is not commented out,
and that
you are actually running the inetd service (/sbin/service
inet status)
Also check that you have allowed ftp access in the
/etc/hosts.allow and 
/etc/hosts.deny files.

Hope this helps
