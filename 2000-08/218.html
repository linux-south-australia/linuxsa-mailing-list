Re: default route dropout.

>     I have a Telstra Big Pong direct permanent connection
> which works fine most of the time. The problem I have is

Me too :).

> that from time to time, Telstra drops the connection and I
> have to obviously reconnect. When I reconnect either
> manually or automatically, the system drops the default
> route, which is really frustrating. Once I reset default
> route manually using the route command it is fine, or if I
> reboot the machine and it logs on at startup the default
> route is there and it works fine. So what have I forgotten
> to set up when I reconnect without rebooting. To do a manual
> reconnect I just run the ppp-on script.

/etc/sysconfig is a good place to visit.  In particular 
/etc/sysconfig/network-scripts.  In there you will find a number of
files for each interface.  Namely ifcfg-ppp0 should be the file to
look at.

This file is picked up by the /sbin/ifup script.  The main line
to check for is a line which has

  DEFROUTE=yes

or rather that is what it should have.
