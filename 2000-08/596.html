Re: color allocation problems

Regarding the colour depth an X server is running at:


> Hi Mike, in the section of the dmesg log (attached) describing the vieo
> configuration I can see no info on depth (none that makes sense to me
> anyway). If you could find the argument you refered that would be great.

Try

	man xdpyinfo

Otherwise, you could capture the information output when the Xserver
starts.  From run level three (either pass the runlevel to the kernel
at the lilo prompt, or run "/sbin/telinit") you could start the X
server and capture its output with

	startx >& ~/x_errors.txt
