gone is gone and other tragic stories...

Here's a tragic story with some morals for us all. And some questions
for the next time I'm this careless. 
OK, I've been doing some evolutionary analysis using Monte Carlo Markov
Chain analysis. After about a week of generating a 155MB file of
floating point calculation results (plus other extra extraneous bits).
After finding out the the analysis component of the package is unable to
deal with the output, I decide to use another package's analysis program
(which uses a slightly different format - every line is terminated by
';\n' instead of just '\n'). After stripping off the leading garbage (I
just use the last 100000 lines of data) by:

tail -100000 run8.1.top > run8.1.top.consensable

I then add the trailing ';'s using vi:

vi run8.1.top.consensable
->:1,$ s/$/;/g
->:wq

Now this all takes a long time and vi needs to set up a big swap file on
the /home fs, so play aroung trying to figure out a more space and time
efficient way to do this, generating test files along the way.
Eventually I give up and delete all the teast files... Except instead of
typing in:

rm -f test*

I type:

rm -f test *

All gone - all ~200MB of data and run control files - everything
associated with that set of analysis. I watched in horror at my disk
usage applet moved from full to 75% free on my /home fs.  I'm now
redoing the week's worth of processing.

So my questions are:
Is it possible to undelete inodes (given theat the disk hasn't been
written to in the mean time)? So if I'm a complete idiot again I can
save myself the horror.
Is there a better/more efficient way to do the tailing of lines with
';'s?
