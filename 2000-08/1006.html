Re: speed problems over ppp link

Ian Hogben wrote:
> 
> I've got it. It's fixed. I can stop stressing. :)
> 
> Essentially, the modem init string was bunkus. It used to be "ATS0=0Q0V1",
> but it is now "AT&K3E1S0=0Q0&C1&S0V1". That, and I have been told that an
> MTU of 1500 for a ppp link is too high. So I set it to 500. Dunno what an
> optimal setting is, but it ain't broke no more...

You need to set the init strings to do the following:
 - DCD to follow line signal
 - hang up and reset on DTR low
 - CTS/RTS handshaking

It's probably the last that has cuased your link to work.

Depending on the program you use, you may need to set more
or less verbose response strings.

PPP's default MTU of 1500 should work fine for applications
that prefer throughput over latency.

> # ping -c 3 munnari.oz.au
> PING munnari.oz.au (128.250.1.21): 56 data bytes
> 64 bytes from 128.250.1.21: icmp_seq=0 ttl=247 time=245.0 ms

For diagnosing connectivity, you should ping something on your
ISP's network.  Hosts on AARNet are protected from high levels
of ping traffic: 100% loss to munarri may mean that everything
is working fine, but that only 1 in 1000 ICMP Echo Requests is
being passed to munnari because some idiot has sent 1000 of them
in the few seconds.
