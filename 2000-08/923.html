It's the "'" characters; they shouldn't be in there at all.  Here is a
totally fictitious SMTP transaction with an imaginary mail server (:-) : 

[dt@nipnyep dt]$ telnet mail.safcol.com.au smtp
Trying www.xx.yy.zzz...
Connected to mail.safcol.com.au.
Escape character is '^]'.
220 mail.safcol.com.au ESMTP Postfix (Postfix-19991231) (Linux-Mandrake)
helo dt.picknowl.com.au
250 mail.safcol.com.au
mail from: <dt@picknowl.com.au>
250 Ok
rcpt to: <richard@safcol.com.au>
250 Ok
rcpt to: <'richard@safcol.com.au'>
554 <'richard@safcol.com.au'>: Recipient address rejected: Relay access denied

The 'Relay access denied' seems to be a bit of a misnomer; either Postfix
rejected the address because it was invalid, or it couldn't look up the
domain name with a trailing "'" in it.
