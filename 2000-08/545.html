Check what services are open ie ftp, telnet ,etc these are potential
security problems as Redhat installs with all services active. (TL
Server installs all services off and tcp_wrapper active) Comment with a
# the services you don't want running in the inetd.conf file. Also make
comments in the /etc/hosts.allow and /etc/hosts.deny files (you'll
probably find these empty). 

Run xnmap or nessis against your box and see what breaks. Is your box
located behind a firewall? set ipchains (use gfcc if you like GUI's) to
address your services. I suggest using ethereal to suss out the ports
being addressed so you know yourself that your box is safe. Then run
nessis again.

Let me if you want to know anything else.
