> How exactly would I go around setting up a Linux system to validate Windows
> 9x workstation logins, much like a NT server.  I have set it up once before,

> 
> Basically I have set validate logins = yes in smb.conf, and set master browser
> etc..., I configured the win9x machine to be validated by the domain ['WORKGROUP'
> as I set in smb.conf] but it claims when I try to login with a username + pass
> that is valid on the UNIX machine, that there is no validation server available.
>  I have read the documentation over and over, and can't seem to suss this one
> out.
> 
> Additionally, how can I allow logins on smb, but not as a shell account?
> 
> Help :) (the answer is probably something simple, knowing me heh)

So simple, the Samba documentation tells you how!

You need:

encrypt passwords = yes
domain logons = yes
os level = 65
domain master = yes
preferred master = yes
local master = yes
wins support = yes

to do domain logons for Windows 9x (and NT) properly (actually, not all
of it is necessary, but it makes life a lot easier).

restart samba

/etc/rc.d/init.d/smb restart (for RedHat based systems) or just killall
-HUP smbd ; killall -HUP nmbd for any Linux system with Samba already
running

then touch /etc/smbpasswd

and 

smbpasswd -a [username] to add users.

Have fun.
