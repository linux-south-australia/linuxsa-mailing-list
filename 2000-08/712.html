Re: partition changes

Matthew Mills wrote:

> Of course now my system still boots up ok until it trys to mount the /root and
> other partitions.  then I get a prompt.  I have found that I can manually map
> the /root /var /home and /usr partitions but cannot mount the /boot partition
> due to an error.  I guess this is where I need a rescue disk? I have tried to
> edit lilo.conf but it wont allow me to save the file (no swap file found- which
> I fixed by using swapdev /dev/hdb5) as it cannot find vmlinuz on the /boot
> partition as it is not mounted.
> 
> any thoughts? or is this a reinstall job?

At the LILO prompt, 'linux root=/dev/hdxx ro' will allow you to boot OK.
What sorts of errors are you getting when trying to mount /boot? You
could try copying /usr/src/linux/arch/i386/boot/bzimage to some place
where LILO (and the BIOS) can find it and then point LILO.conf to it,
run LILO and reboot. Oh, and don't forget to change /etc/fstab.
