There are also specialist processor instructions that will
shift left a register until a 1 bit is found and return the
number of shifts.

You want to make sure that writing the "obvious" code isn't
optimised down to this instruction and that moving to a "smarter"
algorithm isn't undoing the compiler's good work.

If the compiler doesn't emit this instruction, then the Linux
kernel tradition seems to be to implement the same procedure twice:
once in C and once in inline assembler.
