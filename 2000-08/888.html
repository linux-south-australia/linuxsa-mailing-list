> So do I.  The default route is not a problem.  I'm trying to add extra
> static routes to one host, which has two network cards and is connected to
> the internet with one, and needs a plethora of routing entries to talk with
> the rest of the network(s).

This isn't too surprising.  DHCP has hundreds of options --
the server has to support all of them, each client will
support a subset.

DHCP is designed to allow configuration-free hosts, not
configuration-free routers, which is what your Internet
gateway is.  Given that it is actually a router, save yourself
the pain and run a real routing protocol: this will populate
the routing table.  Depending if other machines should use your
link to the Internet, allow or deny the advertisment of the
static default route pointing down the Internet interface
(even this is a bit sucky: configuring a BGP connection to your
ISP with an enterprise AS number and having them advertise just
default to you would be better, as you can then automate fallback
links).
