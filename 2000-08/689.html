Re: Duplicate IP addresses

Timothy Aslat wrote:
> 
> This is probably a stupid question, but is there any way
> that a duplicate IP address on a network can be detected as
> an error condition on a Linux/BSD box?

The short answer is no: subnetting prevents a host from
seeing the hardware addresses of hosts outside of it's
own subnetwork, so you cannot monitor the hardware addresses
within a network of subnetworks.

The long answer is that you can use tools like arpwatch to
monitor the local subnet and routers allow you to use SNMP
to query their ARP caches.  If you SNMP query all routers and
the routers run promiscuous ARP then you will have all hardware
addresses in use.

As far as someone gazumping your host's IP address goes,
Linux 2.4 has a change that logs this.  In the meantime,
use arpwatch.

I strongly suggest you avoid the whole IP address allocation
problem and insist upon the use of DHCP for address allocation.
