RE: DNS problem

* >Recently upon visiting several sites I have noticed
* >that the address I am visiting is resolving without
* >the www eg.
* >httP://www.freshmeat.net becomes
* >http://freshmeat.net
* >
* >How is this done?
* >Is this a DNS issue or browser redirection?
*
* All this is, is that they have not set the host www to resolve to
* the same IP address or the same server as the domain freshmeat.net

I think that there is an apache configuration option in httpd.conf that
governs this:

# ServerName allows you to set a host name which is sent back to clients for
# your server if it's different than the one the program would get (i.e. use
# "www" instead of the host's real name).
#
# Note: You cannot just invent host names and hope they work. The name you
# define here must be a valid DNS name for your host. If you don't
understand
# this, ask your network administrator.

#ServerName new.host.name
