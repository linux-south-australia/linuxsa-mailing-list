Re: speed problems over ppp link

Personally, I don't think you'll improve things
with MTU=500. This should make things a bit worse
unless your error rate is above about 5% packet loss.

Here are my settings for the last 18 months on 33 kbits/sec:

ppp0      Link encap:Point-to-Point Protocol
          inet addr:139.130.140.14  P-t-P:139.130.140.1  Mask:255.255.255.128
          UP POINTOPOINT RUNNING  MTU:1500  Metric:1
          RX packets:334496 errors:247 dropped:247 overruns:0 frame:0
          TX packets:337842 errors:0 dropped:0 overruns:0 carrier:0
          collisions:0

I don't seem to have problems with this.

If you have other nodes using the gateway, then
your gateway will have to split all packets into
3 or so for transmission over the link.
Only packets originated in the gateway itself will
suffer no IP fragmentation.

So I think you should stick that 500 back to 1500.
But it's a free country - well...  as long as your
behaviour is "appropriate".

Cheers,
Alan Kennington.


PS. If you run IP over skywave short-wave (HF) radio,
then MTU=100 is better, especially when the error rate
gets to around 50% of packets at 100-byte packet lengths.
