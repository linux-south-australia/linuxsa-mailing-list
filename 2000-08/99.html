Re: lpr port

At 06:26 PM 8/3/00 +0930, Steve Sloan wrote:
>Hi all
>
>I am trying to get a standalone print server (Lexmark marknet XLe) working
>over a frame relay from a RH6.1 box.
>
>We can ping it OK but can't send any print jobs to it using lpr etc. It
>works OK on the local lan.
>I am not sure which port should be open so I looked in /etc/services and the
>closest I could find was tcp 515.
>Is this the correct port I should be opening up on the routers.

OK, 515 is the port used for LPD.

One way to check it out is to telnet to port 515:

   telnet a.b.c.d 515

If you do not get connected, there are problems.

You will need port 515 allowed through the routers for TCP.

Do you have anything at the remote end that can capture network traffic?

Also, can you capture at the local end and check to see if a 3-way
hand-shake is completing when you try to print?

If it you can see the first packet (SYN) going out to the right address,
that tells you a lot, and if you don't see the response (SYN ACK) that
tells you more.
