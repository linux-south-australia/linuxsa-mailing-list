> While using the linuxconf util to mount partitions. i found my server
> unbootable. The system would boot to about 50 percent (no good to me) and
> stop. the error message indicates that the disk is in read only mode and for
> this reason i can't modify the /etc/fstab file. any ideas?


did it say "why" it stopped booting? In most cases, it stops booting,
and drops you into a single user environment becase your disk is in dire
need of an interactive fsck.

> 
> The fstab file is clearly not in good shape and this is visible from:
> "more /etc/fstab". This has happened twice now with the same distro, RH 6.2.

what is wrong with the fstab?

but in order to edit this file, you need to remount in read write (RW)
mode. here is a little trick:

mount -o remount /dev/hda1 (or whatever your root partition is)

This gets around the classic Catch 22: In order to fix the problem, you
need to write to the root partition. In order to mount it RW, you need
to unmount it, but you can't unmount the root partition.
