Yeessssss....

Everything I've read about PGP says firmly that there is a real
session key for the whole message, and this is encrypted once
for each recipient's public key.

Therefore if your message doubles in size, there's something
very non-standard going on.

The reason for only encrypting the session key is not
for space minimization but rather for time optimisation.
It would have taken far too long on Phil Zimmerman's original
slow computer to encrypt whole messages with RSA.
So he used IDEA or whatever for symmetric encryption and
encrypted the session key.
But it has the very nice conseuqence that space
is also optimised in the case of multiple recipients.

That's the whole point - the message might only be a few
bytes (about 16 or so?) longer, which would not lead
to immediate suspicion.
