xhost and xauth

My Problem:  With an xwindows application that will run only with xauth
credentials.
This is only with a window manager xdm or gdm present.

If any xhosts entries exist (type in xhost) the application will not
work
because the xauth authentication sees the xhost entries and claims
"insecure"

Obvious attempt to rectify this is "xhost -" or "xhost - <host>" for
each entry in the
xhost list. Unfortunately the entries returned from xhost are sticky.
I can't remove them.

This smells like a bug. Has anyone seen this?

I have been able to work around this by not using xdm or gdm (comment
out prefdm in
/etc/inittab) .  Login is now via console and a manual "startx".
