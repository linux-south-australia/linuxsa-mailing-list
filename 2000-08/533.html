Re: Increasing Swap Space

On Wed, 16 Aug 2000, Matthew Tippett wrote:

<SNIP>
> The one other useful thing which Michael didn't add is that to make this
> happen on boot, you should add the following to /etc/fstab.
> 
> /dev/sda2       swap            swap    pri=0           0       0
> /tmp/swap       swap            swap    pri=0           0       0
> 

By setting the priorities to the same level the kernel will use the swap
space in a raid 0 style. Using a swap file means that the kernel must
processes the swap request through an extra layer of translation. This
basically means that it more efficient to use the swap partition before
the swap file so one should adjust the priorities accordingly.
