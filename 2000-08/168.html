This is fixed in sox-12.16 package version 4 and later.

I found that it was actually originally reported as bug ID 5669 in 1999.  
I was too specific in my initial bugzilla search so I missed to original 
postings.

An interesting security note from the 
archives, /var/tmp/sox-root/usr/man/man1/play.1 could be created by a 
normal user.  Some of you may have seen on bugtraq a while back that 
troff macros can be used to execute system commands.  This might be a bad 
thing if you `man rec` as root.
