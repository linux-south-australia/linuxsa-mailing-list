crossover cable woes

This is my first attempt at connecting two computers
with
a crossover cable. I have two 3c509 nics, one in each
machine.

I did "modprobe 3c509" and setup the irq and io ports
correctly
(afaik, if anyone can find the actual hardware
settings for a 3c509,
I'd be grateful to know, couldn't find anything on
3com.com.)

I then setup the interfaces on both machines
"ifconfig eth0 192.168.10.5 (.6 for other machine)
netmask 255.255.255.0 up"

Setup the lo interface on both machines aswell as
described in
NET3-4-HOWTO. I was told that seeing as I just want
these two machines
to talk, there is no need to set a route for the eth0
devices, nor
a gateway. I only intend these two computers to
communicate
with eachother, with no external network.

BoxA has ip 192.168.10.5, BoxB = 192.168.10.6

Pinging BoxB from BoxA I get
"ippl: ICMP message type destination unreachable - bad
host from [192.168.10.5]"
On BoxA, I can ping 192.168.10.5 but not BoxB, does
anyone
have any idea why? I don't think it's the nics, I have
a feeling
it's the cable. I bought it as a crossover cable, and
from
what I can tell, the correct wires are crossed over.

searchable archives :)
