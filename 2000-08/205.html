Re: Linux installation via net

On Mon, 7 Aug 2000, Sanjeev Jha wrote:
>While configuring 2nd Linux machine, I found that installation CD is not 
>working (however this CD is working on other machine and
>other CD is working on this machine)

[edited]

>Anyone can help me to find out what to do to install Linux on 
>machine 2, or where I am going wrong ??

1) Remove all hard drives from both machines.

2) Temporarily install hard drive(s) from machine 2 into
machine 1.  Go into the BIOS setup on machine 1 and
reconfigure it for the "new" drive(s).  In most cases you
can simply Autodetect HDD Drive and it automagically happens
for you.

3) Install Linux (using machine 1).  If
also installing X and Gnome/KDE etc, then install as basic
640x480 Generic VGA for now. This will work on any
card/monitor.  Don't bother with any networking setup yet
(except for local loopback).

4) Shut down the new Linux.  Replace all hard drives into
their original machines.

5) Start Linux on machine 2, then do any specific
configuration such as networking setup.
Also reconfigure X for machine 2's water-cooled 
trillion-colour video card and cinema sized
monitor ;-)


Of course the second Linux should be appropriate for machine
2 eg: if it's a 386 then install Slackware or RedHat rather
than Mandrake. 

This technique won't work if the machines are very
different architectures (eg: Intel vs Apple).
