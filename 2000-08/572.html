>
>Linux / LILO don't have the problem. It's actually the BIOS of the
>machine. LILO will boot anywhere the BIOS says exists. Linux will
>support the entire disk (irrelevant of what the BIOS says). Make sure
>that if you have a dodgy BIOS (like mine;-)) you have the Linux kernel
>installed from a partition that is completely below the magic line that
>the BIOS draws as the end of the disk. The rest of linux can be
>anywhere, but When LILO tries to boot the kernel, it really needs to be
>able to access it via the BIOS.

Indeed. Mandrake 7.1 and the RH 7.0 BETA both include a version of LILO
that can use the LBA32 or EDD stuff in more modern BIOSES.

>I'm not sure how GRUB works. It may work better, but it is difficult.

Grub does a two stage load, so the second stage loader has enough smarts to
use its own routines and avoids the BIOS, AFAICS.
