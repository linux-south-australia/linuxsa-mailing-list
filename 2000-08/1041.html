> Just found usign X -probeonly that X is trying to set my default fontpath to
> "unix/:-1".Have no idea why.........
> 

this indicates that X should try to connect to a local unix socket, on
the other end of which is a program that serves fonts.

if X cannot find fonts, it is a Bad Thing, and it will die. Though I
thought you could have mutliple font paths defined.

If your font server is not running, then it might help to start it.
These things are usually started in /etc/rc.d/rc5.d (5 being the X
runlevel).

on a redhat 6.12 system (like mine) you can tell if your x font server
is running by doing

ps -eaf | grep xfs,

or

/etc/rc.c/init.d/xfs status
