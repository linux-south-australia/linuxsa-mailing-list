Re: Directory listing bash script

Paul Malcolm Bailey wrote:
> Richard Sharpe wrote:
> > >for somevar in $(ls); do
> > 
> >                 ^^^^^   This is not portable. It is a bashism. Use `ls` ...
> >                         unless you need nesting.
> 
> So what you're saying here is always put command substitutions in single
> quotes as standard practice?  (For example, $(ls -al) should instead be
> $('ls -al') for portability reasons.  Is that right?)

Wrote quotes.  You want back ticks (`) not single quotes (').  

The difference is that the command in between the back ticks is executed in a
sub-shell and substituted where the command is.  ie `ls` expands to a whitespace
delimited list of files in the current directory.  Whereas single quotes (')
"escapes" the contents to the shell, ie. prevent the shell expanding, or
pre-interpreting the contents.  Not what you want here.

It's a little tricky to start with, but once you understand, shell-scripting is
quite powerful.  Now just modify your syntax slightly into perl and you have
great execution speed too :-)  But that's another thread I'm sure.
