I'm using a cacheing DNS server also. It's kinda easy to set up. When
installing, you can specify a cacheing DNS server in many distro's. My
resolv.conf file looked like this first up:
search safcol.com.au
nameserver 127.0.0.1

My /etc/sysconfig/network-scripts/ifcfg-ppp0 file had this important flag:
PEERDNS="yes"

Then when I connected to my ISP, the resolv.conf file was overwritten to
include my ISP's DNS servers. This worked to my advantage, because I wanted
to know what they were to get things working perfectly. The resolv.conf then
looked like this (unique to each ISP, of course):
search safcol.com.au
# nameserver 127.0.0.1
nameserver 203.24.77.6
nameserver 203.24.77.1

The final step meant that I changed the ifcfg-ppp0 file to this:
PEERDNS="no"
and I changed my resolv.conf to look at my own cacheing DNS server first
like so:
search safcol.com.au
nameserver 127.0.0.1
nameserver 203.24.77.6
nameserver 203.24.77.1

And Bob's your uncle, everything works perfectly ever after (until, of
course, the ISP changes its DNS servers, but that's unlikely...)
