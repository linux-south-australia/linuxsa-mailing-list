sendmail (was telnet)

Hi,

Thanks to those who responded to last nights telnet dilemma. I admit I left 
a fair bit out. The school runs W95 for the kids with a Novell 4.11 client. 
The mail server uses sendmail under Linux. This morning we removed 
telnet.exe from the system which will stop the little buggers maliciously 
sending email through telnet from within the school. To stop them using 
this feature on the other side of our firewall and spamming in, we thought 
of hacking the sendmail.cf file on the line under timeouts.

We found the three main commands to mail in telnet are mail, rcpt and data.

In sendmail.cf, some of the defaults are
#0 Timeout.mail=10m
#0 Timeout.rcpt=1h

and I am hoping that this refers to 10 minutes and 1 hour respectively, 
when typing anything if using a telnet session

If we change this to
0 Timeout.mail=5s	#hopefully 5 seconds

Now for my questions.
1) If we make this change,  will this stop slow typists using telnet to 
send mail through our mail server?

2) can we then do a
source /etc/sendmail.cf
on the mail server without having to restart the mail server, although a 
restart probably is insignificant.

3) or have I completely lost the plot.

I realise there is only so much you can do, but if we can make it that 
little bit harder ....

Thanks
