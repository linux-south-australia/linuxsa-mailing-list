Re: xman query (newbie question)

Geoffrey et al:

> Alternatively, using just 'su' should mean that the DISPLAY variable
> gets passed along.

That's true however there are some security issues with using "just su"
to root. Example:

* I add current directory to path

* I set up a shell script that. among other things, gives me (i.e. my
account) a setuid link to /bin/bash and I place that somewhere hidden
 - fairly harmless but if root creates this it's a setuid 0 shell
account

* I call this script "ls", and except the other bits and pieces above it
behaves like ls...it could simply just pass all arguments to the real
ls...I would probably dynamically remove the current path so my next
ploy would work

* I then complain to system administrator that the programs I create
don't work, or something that is likely to get him to do ls

* sys admin arrives and does su (no dash)

* runs ls

I have just gotten myself a setuid shell account.

> There may be hassles with X authentication, and you can get around
> these with a xhost +localhost, but if you're using Red Hat (since
> version 6.0), when you use 'su' your xauth key gets passed to root's
> session so it will 'just work' (see
> http://www.redhat.com/support/wpapers/newpam/ for more information
> about that).

Yes, this seems to work...provided you don't provide the dash where the
key is not passed to UID 0 by default...
