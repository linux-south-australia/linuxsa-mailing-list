Re: Linux server in a small business LAN (was: Request for ...)

John wrote:
> 
> > It does allow denial of service attacks on the server, but
> > with an ISDN link I wouldn't get too fussed.
> 
> Umm, please explain??? Why wouldn't one be "too fussed" about DOS
> attacks if one has an ISDN link?

Because the link bandwidth is so low, a DoS attack
will hose the ISDN link before it has much chance of
hosing the server.

Conversely, you can hose a gigabit-connected web server
simply by sending it end-to-end small packets.  The
ethernet card interrupt rate is enough to stop the machine
doing useful work.

This explains why cable ISPs like @Home traffic-shape ICMP
packets (such as ping requests) and why gigabit routers
are designed to track IP flows and apply traffic policies to
them.
