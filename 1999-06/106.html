If its an ATAPI cdrom, on Secondary slave then /dev/cdrom should be linked
to /dev/hdd, check this (ls -als), if not then do so (man ln to find out
how), if so, then I just had this problem at home, the CD is going faulty
and not autodetecting, so with lilo prompt use the command to boot to
linux (linux on my machine) with the argument hdd=cdrom ie

linux hdd=cdrom

There should be a way of putting this into lilo.conf, but I've yet to
research this problem.
