Re: FreeBSD vs Linux

Justin Viiret wrote:

 > FreeBSD uses the ports system (http://www.freebsd.org/ports/),
 > which is a sort of source-only package management system.

Not quite true.  pkg_add is also supported;  it can even install
from a HTTP or FTP URL (and automatically takes care of dependencies,
etc).

 > It's cool once you've had a play with it, though perhaps not so
 > easy to keep up to date as Debian Linux's, or Red Hat's.

Install "pib", which is the ports collection management tool 
(/usr/ports/sysutils/pib).  It's an X11 GUI-driven tool which 
scans your installed packages to see if they need upgrading, 
offers point-and-click installation and deletion of packages,
and manages the source code archive you invariably end up with
if you install a lot of ports.

Like I said before, though, I'll be covering this extensively in
a couple of meetings time.  For the developers, the ports collection
is something which has been developed to be reasonably portable, and
a nice little project could be to port the Makefiles that drive it
to Linux.  That would be a very cool thing indeed.
