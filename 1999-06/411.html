> It can be any method really - html, dynamic html by cgi, javascript,

I don't think you will be able to this with raw apache basic auth, you need
to be able to work with the headers.

The trick is that you have to trick the browser in thinking that the
username/password for a given realm is invalid and consequently when it
gets challanged for that realm will request if from the user.

Have a peek at the page, the code should be fairly simple to follow, and 
from there if anyone has any questions have mail the list and I will go in
a touch more deeply.

It is not perfect, but works well for Netscape and IE.
