More ways to use Linux

This seems poetic ...

I have a customer who has a serious problem with a Windows system.  The
networking has given up on this Laptop and the simplest thing seems to be
to get the vendor to reload Windows 95 (one of the earlier Laptops without
a recovery CD), which they will charge $65 for or there abouts.

Since the networking code is not working, it is impossible to get the data
of the disk, or so it would seem.

Linux to the rescue.

Grab a copy of Toms RootBoot disk.  Adds xirc2ps_cs.o to the kit, and
builds a boot floppy.

Boots, modifies /etc/pcmcia/config.opts.  Starts pcmcia, and all goes OK,
then NFS mounts a file system from another Linux system, then mounts the
Windows partition, and copies it all off onto the remote Windows system.

On the way back, Samba can be used to restore selective parts of the disk.

Will it work, well, the NFS stuff sure does.  We get to see in a short
while :-)
