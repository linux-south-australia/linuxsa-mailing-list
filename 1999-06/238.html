ipfwadm doesn't like named

Hi there.
I'm running RH5.0 with a custom kernel 2.0.36-1.  My problem is that
ipfwadm (2.3.0-5) is extremely slow to execute any rule from inside a perl
script called from rc.local.  I've worked out that the problem is only
occuring if I have named (bind-4.9.7-1) loading prior to the script
running.  I have a number of forwarding rules, and of course the
masquerading rule, running from this script and it takes about 30 minutes
for it to finish.. each rule taking between 30 secs. and a minute each to
execute.

Does anyone know what the problem is and how can I bypass it without having
to initiate `/etc/rc.d/init.d/named start` after my firewall script within
rc.local?  I'm only using named as a cacheing server so it is not mission
critical, but still, it is rather puzzling and frustrating.
