Multi-boot configuration

Howdy,
I have a need to create a triple-boot Toshiba 440CDX with:
Linux
OS/2 Warp Server for E-Business
NT Terminal Server, (yuk!)

Could people point me to some howtos on this?
I'd like to do all the booting using lilo, but I don't know if this is
possible, nor do I really know what I need to put in the lilo.conf file.
The thinking behind this is I would like to make the solution as generic as
possible, allowing it to be used for booting other great OSes. I *could*
use OS/2's Boot Manager, Partition magic, NT's boot loader, but that is no
good for people using this technique if they don't have a license for these
products & can't afford a license.

Reading docos & fiddling so far has given the following info:
1)
I need to make sure the lilo partition is in the 1st 1023 cylinders of the
HDD. (No problem, put it in the MBR, the Toshiba does LBA & it has a 6GB
IBM HDD :)
2)
Linux 5.2's fdisk kicks-ass! This gives me excellent  control of what
partition goes where, & it works!
3)
I'm thinking I'll use the following layout:

/dev/hda1 Primary   Linux Native   Linux, /etc, /root, /boot etc
/dev/hda2 Primary   HPFS           OS/2 application code, (seen as c:)
/dev/hda3 Primary   NTFS           NT application code, (seen as c:)
/dev/hda4 Extended
/dev/hda5           Linux Swap           Linux swap partition.
/dev/hda6           HPFS           OS/2 swapper, temp & spool, (OS/2 sees
d:)
/dev/hda7           HPFS           OS/2 data, (OS/2 sees e:)
/dev/hda8           NTFS           NT swapper, temp & spool, (NT sees d:)
/dev/hda9           NTFS           NT data, (NT sees e:)

I'll need lilo to boot
/dev/hda1 (Linux)
/dev/hda2 (OS/2 Normal)
/dev/hda3 (NT)
/dev/hda6 (OS/2 Maint)
