Re: Net Configs

>>>>> "AC" == Andrew Chambers <remnant@acaw.com> writes:

  AC> Hey all, I have recently installed Debian Linux onto my system,
  AC> and in the process was trying to install my SMC Ultra nic.  When
  AC> it came to the appropriate section, I installed the extra
  AC> package for the card, however it failed.

  AC> I was just wondering, now that installation process is over
  AC> with, what utilities etc I could use to determine the settings
  AC> of the card (IRQ, IO etc), 

Typing the following should get you started:
  cat /proc/modules
  cat /proc/ioports
  cat /proc/interrupts
  dmesg

  AC> along with how I would go about actually changing the values for
  AC> the nic in the linux configs.

Assumptions:
  - your kernel is compiled with the NIC support as a module
  - you are running kerneld (type 'ps aux | grep kernel' and look for /sbin/kerneld)
  - your default run-level is 2 (see /etc/inittab)
  - your /etc/rc2.d/S12kerneld is a sym-link to /etc/init.d/kerneld
  - your /etc/init.d/network script is being run and does mention eth0
  - you have a valid /etc/modules file

Then 'man (5) modules' tells you that:

          The /etc/modules file contains the names of kernel modules
	  that are to be loaded at boot time, one per line. Arguments
	  can be given in the same line as the module name.

So add your arguments and try it out.  See also the man pages for
kerneld and associated utilities (insmod(1), rmmod(1), modprobe(1),
depmod(1), syslogd(8), kdstat(8))
