 > Ip based routing wont work in this situation as I cannot predict where the
 > packets are going and route them accordingly.

Understood.  I'd suggest checking out the functionality in ip_filter,
which is a bit like ipfwadm on steroids.  I haven't looked for this 
bit specifically in its docs, but it seems to have everything else :-)

Additionally, FreeBSD's "ipfw" functionality includes a "fwd" keyword
which allows you to specify a next-hop IP address for packets which
match a given access-list clause (oh dear, now I've done it).

Cisco routers provide policy-based routing as well, where you can define
an access-list to describe the packets you're interested in treating 
"differently" and a routing policy to describe what "differently" means,
including the specification of an output network interface or next-hop
IP address.

If you want more info on any of the above, let me know.
