Re: Hacking past Apache user authentication

> How secure is Apache user authentication? 
> 
> Can it be hacked, and entry gained to a directory protected by a .htaccess
> file?

As secure as the medium over which it travels.  The vanilla apache
authentication uses Basic Auth.  Which means the information is
Base64 encoded and sent over the wire (If I Recall Correctly).

Any scheme that uses Basic Auth is prone to this.  The auth testing
page that I pointed out previously didn't use apache auth but was
prone to the same sort of problems.

I have seen/heard of programs that will snoop a network looking for
packets that contain the HTTP authentication header.  So it is as secure
as passing your password over the network in plain text (as in telnet/ftp/
pop/imap and most other protocols).

You may also be open to brute force attacks, I am not sure if apache logs
multiple successive attempts.

In terms of breaking in without snooping/brute forcing I don't think there
would be too many holes in apache (unless there are some killer bugs out
there) with respect to the .htaccess controls. (Standard disclaimer applies)

Something more secure... Hmmm... There should be information on the Internet
on different auth mechanisms and their relative security.  

For *LOTS* of auth options for apache have a look at

	http://modules.apache.org/search

and do a search for 'auth'.  It would at least be a good start.
