Vypre wrote:

> The only thing that I can think of is that the video driver that I am using
> is not quite compatible. In Linux it is called an Alliance Promotion 6422 (I
> think from memory), but Windows 95/98 only describes it as an Alliance

When you run startx, a log is left on the main console.

Ctrl-Alt-F1 will bring up the main console (if X hasn't
died totally hard).

If you are automatically starting X, the error log is
kept in /var/log/xdm-error.log.

To help you debug, a correctly running X server start
under RH5.2 is shown below.

A quick look at the XFree86 web site at:

http://xfree86.mirror.aarnet.edu.au/3.3.3.1/apm.html

suggests using the SVGA server for your chipset.  The
doco also suggests turning off some acceleration
features if you are having trouble.
