QUICK RUNLEVEL TUTORIAL

You can also type "linux 3" to boot normally, but without
X.  This is usually used for web servers.  "linux 5" will
boot into X.

In general, the argument after linux gets passed onto
the kernel, which passes it onto init.  At run level X
init executes the programs:

  /etc/rc.d/rcX.d/S[0-9]+* start

in sorted order.  These programs are usually symbolic
links to programs in /etc/rc.d/init.d/ and do useful
things like starting networking, Apache, Samba, X and
the like.

Without an argument, init looks in /etc/initrunlvl for
the run-level.

For kicks, you can boot with "linux 0" and watch it halt
as soon as it is running.  Run level 4 is unused: this
is very handy if you are developing a turnkey application.

Single user mode is special.  Only root can log in, and
usually only from the console.  This is ideal if you need
to know that very few files are open.  Note that some
Linux distributions (notoriously RedHat) run too many
networking daemons in single user mode and devalue this
assurance.

You can get lilo to kill any kernel parameters.  You need
this for computing labs and other hostile environments where
users can access the console.  You don't want them to boot
into other run levels or pass parameters to device drivers.
For these machines it is worthwhile setting Ctrl-Alt-Del
to shutdown the machine rather than rebooting it.
