Re: Mounting CD in RH 6

Mike Earnshaw wrote:

> List,
>
> Tried mounting a CD that I have just added to a previously installed system
> (the first CD was robbed to repair another machine). As root I changed to
> the mnt dir and entered:
>
> mount cdrom
>
> Response was:
>
> wrong fs type, bad option, bad superblock on /dev/cdrom or too many mounted
> file systems

Linux is detecting your cdrom drive otherwise you wouldn't get the above
message.
Try for experiment 'mount -t iso9660 /dev/cdrom  /<<your mount point>>'
probably /mnt/cdrom.  What happens?  Next, look in /etc/fstab and check
for a reference to the cdrom.  This is the file linux looks at when it
mounts file systems.  It should have iso9660 as file system type.  Which
hardware device is it associated with?  What is the disc you are trying
to mount?

>
>
> When checking the hardware setup through DOS the CD works OK. My setup is:
>
> IDE 1: 7Gb HD partioned (but I forget which is which, I know the 133 is
> swap!), 1702, 1004, 698, 1702, 133 all in ext2
> IDE 2: 3Gb HD partioned 1012 (Fat32) 2048 unused with IDE sec of CDRom
>
> Questions:
>
> 1. Is there a limit to file systems you can mount? If so where is it
> configured?
> 2. I know I should know the answer to this one, but what is a block device?
> 3. I am still getting used to Unix dev structures, nothing like DOS C:\ D:\
> to pollute the mind eh? - but I am having difficulty visualising what dir is
> on what physical drive. Any pointers for easier understanding?
Linux doesn't present hard drives as separate drives like DOS does. If
you were to add another drive and use fstab to mount it during boot it
would appear as a directory that you specify. For instance, I mount my
dos partition on /cdrive, or drive D: which to linux is hdc1 on /drived. 
The names aren't predetermined, you can call them what you like.
