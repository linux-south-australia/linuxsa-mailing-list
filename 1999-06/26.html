mserver - C-Masqdialer not working with bash-2.x

Hey all,

I run a c-masqdialer on my LAN for manipulating internet connections, and it
has been working great until recently i noticed it was broken. Trying to
diagnose the problem i almost tore my hair out finding that nothing had
really changed, so it must be something else.

Just casually looking in /var/log/secure i notice:
Jun  2 00:17:21 pikachu bash[2153]: Remote execution attempt from squirtle
Hmm... Looks like Bash doesnt like remote executions (How c-masqdialer
scripts dial).. and then i remembered i had just updated to bash 2.x
Changing /bin/sh -> bash to point to tcsh has fixed the problem, but im
still wondering why 2.x stops remote bash executions like that. The daemon
is running from root, so there shouldnt be any permission problems..

Anyone know how to enable bash-2.03-7 to allow "Remote executions"?
