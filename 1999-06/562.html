How to use a 10GB disk with Linux and the Devil's OS?

Hi,

I have ordered a Dell I7K system with a 10GB drive.  I will be running
several OSes on the one disk, including Win98, Win95, WinNT, RH Linux, TL,
SuSE, etc (and maybe even FreeBSD if Mark will make me a CD :-).

My immediate problem will be getting Linux and Win98 to co-exist happily,
given the 10GB drive.

I had thoght to use FIPS 2.0 to cut the drive down to size (about 1.6GB,
but that may not be the way to go, given the problems with LILO and
cylinders above 1023.

Since the disk is 10GB, I suspect that I cannot use LBA, in which case I am
left with having to move the Win98 partition as well as cutting it down in
size.

I suspect that what I may have to do is:

1. Cut down the Win98 partition and save it over the network to somewere
else. TomsRtBt seems to support the cards I need (www.toms.net).

2. Create a small 10MB /boot partition on the front.

3. Create an active partition for Win98 after that

4. Install Linux with /boot correctly setup

5. Copy Win98 back over the network via TomsRtBt ...

Does anyone have any comments?
