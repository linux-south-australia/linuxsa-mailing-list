Nick
I've been using jdk_1.1.7-v1a-glibc-x86 from blackdown for ages on several
machines. I installed manually from jdk_1.1.7-v1a-glibc-x86.tar.bz2

I can't find libXm.so.2 on any of these machines but rather 
./usr/X11R6/lib/libXmu.so.6 which has a symbolic link libXmu.so
This is part of the XFree86-libs

You may find you have a similar setup. If so you may simply be able to 
override the rpm dependency checks using --nodeps option.

Alternatively download the tar & install it long hand, its not hard.

Adrian
