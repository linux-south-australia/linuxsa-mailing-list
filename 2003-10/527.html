Re: export in bash scripts

Hi Frank,

On Fri, 17 Oct 2003, Frank Bendon wrote:
> when I run a bash script like:
> 
>    VAR=SOMETHING
>    export VAR
> 
> then:
>    set |grep VAR
> it is not there
> - what would stop 'export' working as expected?

If you run the script something like:

$ ./zz.sh

Then you are running in a new shell which exits when the script
finishes.  At this point the new shell environment is lost.  A bit like
doing this:

$ bash
$ export FOO=BAR
$ ^D
$ echo $FOO     # Probably nothing

You probably want to "source" the file if you want to get stuff into
your shell environment:

$ source ./zz.sh

or

$ . ./zz.sh
