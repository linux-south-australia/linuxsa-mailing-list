On Fri, 2003-10-31 at 13:25, Mark Newton wrote:
> On Fri, Oct 31, 2003 at 01:00:33PM +1030, Jordan Noblet wrote:
> 
>  > I have apache executing bash scripts on a machine and echoing the
>  > results back to the browser in html.
> 
> <shudder>  Are you sure a bash script is the right answer here?
> This sounds like a security nightmare waiting to happen;  I'll bet 
> you a box of doughnuts that you don't handle the passed-in form
> elements securely.

Thats probably a fair bet :) However most other script languages don't
do that as well :(

Check the "test-cgi" bash example that gets installed with apache (or
search using google).

GET data goes into env var "QUERY_STRING", POST data goes to stdin. In
both cases you will need to parse the tuples into something usable
(can't be that hard). The get the data from stdin, a line like
somevar=`cat -` will do the trick.

Security will be a fair sized issue, however if this is just an internal
script then you probably do not need to stress about it.  I would filter
all fields to any non alpha-numerical chars (a..z,A.Z,0..9 type of
thing)
