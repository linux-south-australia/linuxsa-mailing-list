Re: unwanted clock setting on reboot -- UTC mixup

Luke Vanderfluit wrote:

> Every once in a while, usually about monthly I reboot my linux box for
> whatever reason.

And at the other end of the scale, I've just had a set of servers hit
the 497 day uptime wrparound bug.  Where's uptime.viper.net.au when
you need it?

> Is there any way to tell the system to keep it's current time and not
> set the hardware clock to system time on power down?

Well, I'm not sure what distribution you're running (hint hint), but
given that it's Unix it should be easy to find, as it will most likely
be in the shutdown scripts.  Depending on where your start scripts are
located, try running "grep hwclock $(find /etc/rc.d -type f -print)",
then if found modify the appropriate script.
