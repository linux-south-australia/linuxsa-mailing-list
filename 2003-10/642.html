Damien,

> I just thought I would report on my experiences with the autopackage

> 
> http://www.autopackage.org/downloads/gaim/
> 
> It cannot fetch dependencies from the net yet, so all packages in that

It looks like "yet more redundancy" to me.

> Overall, I think it's a very promising packaging format and should make 
> installing software a lot simpler for the user. It is intended to be 
> distribution-independent, and so the very same packages should install
> on SuSE, Redhat, Debian, slackware, etc etc... having a repository of

Under Debian one can do:

 auto-apt gaim

...and auto-apt will just run off and download, preconfigure and configure
all packages necessary to run gaim. It's not ported to apt4rpm yet but the
port wouldn't be too difficult.

WHY introduce yet another packaging format?

 * synaptic
 * apt-get install BLAH
