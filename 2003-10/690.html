RE: ssh with public and private key

Stephen Donaldson wrote:
> Subject: Re: ssh with public and private key
> 
> I have read throuhg some of the replies to this thread. 
> So here's a couple of gotchas. The primary one is if you
> have used the paraphrase to enter a password it will always
> ask for a password. This is common.

you can, if you like, turn off asking for a password, 
in the sshd conf file with the option;

 PasswordAuthentication no

or specifically turn it on, by setting it to "yes"

> Do not use paraphrase, eg just press enter and move on.
> Make sure you made the keys as the user on both ends and not as 
> root in a users folder.
> Make sure authorised_keys is a file and not a folder

also you might like to disallow root logins from external interfaces
and run a second ssh server on the internal loopback device that
accepts root logins, so requiring ssh authenticated access to the
host before getting near a root account, and requiring properly
setup ssh agent forwarding...
