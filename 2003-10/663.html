Re: ssh with public and private key

On Fri, 2003-10-24 at 15:29, Luke Vanderfluit wrote:
> Hi,
> 
> I wish to ssh from a local machine to a remote machine without having to
> enter passwords.
> 
> I have created a private and public key on my local machine with
> 'ssh-keygen'. I have the private key file in $HOME/.ssh on the local
> machine( 'myKey'), then I have copied the public key file-contents to
> the $HOME/.ssh/authorised_keys file on the remote machine,
> 
> according to the man page this should work,
> what am I doing wrong,

Hi Luke,

A couple of things to check:

* Make sure you're using the right protocol for your key. Ie if it's the
old RSA key, use 'ssh -1 host', or if it's the newer dsa key
(id_dsa.pub), use 'ssh -2 host' (and make sure the key's in the
'authorized_keys2' file).

* Make sure the authorized_keys (or authorized_keys2 file) have
permissions of 0600.

Cheers,
Chris.
