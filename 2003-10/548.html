Re: cat command on windows?

Don,

It's such a stupid command syntax though. Who cares if the files are binary or 
ASCII?

fopen("filename","w") does the same as fopen("filename","wb").  From the fopen 
man page:

"The mode string can also include the letter  ``b''  either  as  a  last
character  or  as a character between the characters in any of the two-
character strings described above.  This is strictly for  compatibility
with  ANSI  X3.159-1989  (``ANSI  C'')  and has no effect; the ``b'' is
ignored on all POSIX conforming systems"...

That's why when I used copy that one time I was totally stunned that I had to 
specify what type of file it was. Oh, and in my command I have the /B 
following the file it is meant to apply to, so I dont have /B on the last 
file. From the command syntax below:

 [+ source [/A | /B] [+ ...]]

Cheers,

Damien
