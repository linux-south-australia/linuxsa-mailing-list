> However after changing the scsi card 
> from a aic7xxx type to an aha1542, so that I've got an external 
> connector and a boot option.
> I found that even after changing /etc/modules.conf and running
> pnpdump > isapnp.conf then modifying the result, the dang thing still 
> gives the complaint:
> busybox: init aic7xxx error: no aic7xxx adapter found.
> when starting. So I am wondering how to make it forget it used to have 
> an aic7xxx adapter in it.

This sounds like a initrd problem, that's if your seeing the error very
early in the boot process, and your right no matter what you do, you
will see the error, as the module would have been built into the initrd
file..    you can fix your situation with the "mkinitrd" command..  Now
even I would have to review howto run that command, but first go through
man file for "mkinitrd" e.g. "man mkinitrd".

O.K. here's how I would go about it..  now this will be rough, as I am
not going to try it just to see if it's correct..  Warning if you get
this wrong your Linux will not boot at all, so anything that you do from
my instructions is at your own risk. First make sure that your
/etc/modules.conf file is in order, as mkinitrd will use that file.

Make the image
mkinitrd /boot/initrd-test.img `uname -r`

Edit the lilo.conf file.. add a test boot..   here's an example, of
course your should be different.. depending on your hardware config etc.

image=/boot/vmlinuz
        label="test"
        root=/dev/hda5
   # note the initrd entry
        initrd=/boot/initrd-test.img
        append="devfs=mount hdc=ide-scsi acpi=off quiet"
        vga=788
        read-only

Once you have worked it all out..  DON'T FORGET to run the "lilo"
command when you have finished
