Re: html links to local file systems

Hi all,

With help from other linux users I've come to the following:

By default Mozilla doesn't allow access to the local file system using
the 'file://' protocol.
It is possible to enable this by adding 

'user_pref("security.checkloaduri", false);'

to 'prefs.js'

However, I've followed the advice of 'security pursuant' linux users and
disabled access to the local file system. I.E. with browser using the
'file://' protocol. 
(so I'm not using 'user_pref("security.checkloaduri", false);')

I've followed some other advice and installed apache locally, created
aliases to the local directories and used 'http://localhost',

After some help thinking things through, if I were to load a page from
somewhere that had a malicious script that targeted a generically titled
script on my system, or I loaded a page that had java script that
accessed my local file system, and say sent results to somewhere, that
would be worrying.

I'm glad mozilla has this feature and surprised that Konqueror and IE
allow access to the local file system.
Apparently there are pages that say "Look!  The contents of 
your hard drive are exposed!" and just have a link to C:\ (since 
everbody has a C drive)?
