Re: html forms and bash scripts.

On Fri, Oct 31, 2003 at 04:55:42PM +1030, Jordan Noblet wrote:

 > I should have made it clear that this script is for internal use only.
 
The problem is that scripts written for internal use are almost always
exposed to people who aren't perhaps as deserving of your trust as you
think they should be.

 > Is there something inherently insecure about what I've done so far?  Or
 > is passing form elements to the bash script the "security nightmare"?

Almost everything you can usefully do in a shell involves calling 
external commands.

Ponder, if you will, what happens when you pass one of your form 
variables to an external command.

    /usr/bin/command $variable

Now think of what happens if $variable happens to begin with 
the string ";/bin/rm -rf /", because the user who entered the
data on your form guessed that you were using a shell scripting
language.
