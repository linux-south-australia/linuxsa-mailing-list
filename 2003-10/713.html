Re: ssh with public and private key

Tim Aslat wrote:

>I personally would recommend forcing the use of sudo rather than have
>people su to root, but that's personal preference, not necessarily a
>better or worse solution.
>  
>

Yes, sudo is my personal preference too.

Another alternative found on some older systems in SA is 'stfsu', which 
is kind of like su with no password required if the user is in the 
correct group - the author is on this list.

>At least sudo logs each command to syslog whic hcan be on a remote
>machine, so logs like:
> Oct 27 12:47:27 redback sudo: username :
>TTY=ttyp3; PWD=/home/username/ ; USER=root ;
>COMMAND=/bin/ls
>would have a slightly better audit trail than just "root logged in".
>  
>

It's nice, but the logging is barely usable as intended in a root 
administration environment - many users of sudo use it to run su or 
their favourite shell, defeating much of the value of it's extra 
logging, unless you have a carefully crafted configuration specifying 
allowed commands. It's still better than using su and a shared password.
