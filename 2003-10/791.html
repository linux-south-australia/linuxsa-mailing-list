Re: Gentoo Linux - any good?

Hi All

i have been using it for the last 6 months.

pros:
*customisability & tweakability (compile software with the options you
want)
*faster than redhat8/9
*easy to update to latest packages (1 command downloads, configures and
compiles and updates all the packages currently installed)
*unmetered mirrors for internode and netspacers (possibly others)
*more paranoid security than rh
*forums and docs usually helpful
*no dependency problems!
*can set it compiling ...go home and come to work next day -
semi-automatically update config files and bingo! ... your linux box has
the buxfixes & new packages released in the last 24 hours
*can use the livecd for other interesting things

cons:
*probably want to compile everything from source code
 - openoffice took 36-48 hours + 3Gig working space to build on a
celeron 500 --- but did run faster than binary download available
*want lots of bandwidth and time to initially build packages
*not for newbies
*people say "so it's like bsd's package manager" ... well yes (is that a
bad thing?)

WD
...ain't going back to redhat, suse or mandrake...
