Re: Quick Query

On Fri, 1 Aug 1997, Nick Morrison wrote:

> The setup I've got at my place is a central unix machine, and about
> 1,000,000 dumb terminals coming off it.  What I want to do is have the
> output to one terminal copy itself to another terminal - ie I want to
> 'spy' on another terminal with my own.  I believe the 'tee' command will
> help me out... but from there on I don't know!  Someone help! 

Try to get a package called 'ttysnoop'. Some distributions already contain
it. Follow the instructions to change what is run when someone tries to
login via getty/telnetd. Once it is setup you can use 'ttysnoop' to watch
or use any terminal. As far as I know the latest version doesn't support
PAM or shadow passwords, which can be painful.
