Help with Network card

Hi Guys,
I'm new to Linux and I'm stumped!! Apologies for the long email but I wanted 
people to have as much info as possible.

I'm installing a network card to my Linux box (RH4.1, Asus TX97 m/b). I 
got myself a 3Com 3c905XL PCI card (documentation says it's supported in
RH4.1, using the 3c59x driver).

Installed the card in the computer, and in the BIOS setup, I set the PCI 
slot to auto (INTA#), and also set the OS to non PNP type.

Booted Linux; card was not detected. Boot message gave several warnings:
 
 Warning: Unknown PCI device (8086:7100) Please read include/linux/pci.h
  "                          (8086:7110)
  "                          (8086:7111)
  "                          (8086:7112)
  "                          (8086:7113)
  "                          (10b7:9050)

Linux then continues to boot as usual. 
Examination of /proc/pci showed the 1st 5 warnings above to be due to the 
Intel TX chips (Host bridge, ISA bridge, IDE interface, USB controller and 
Bridge. My PCI video card is also listed here.
The 6th warning is due to the network card:

 Bus 0, device 12, function 0:
 Ethernet controller: 3Com Unknown device (rev. 0)
 Vendor id=10b7. Device id=9060 Medium devsel. IRQ 11. Master Capable. 
 Latency=32. Min Gnt=3. Max Lat=8
 I/O at 0xd400

I thought that the I/O address is a bit large and uncommon. Anyway, even 
with the warnings for the Intel TX chip, Linux work fine...except I am 
without my network card :-)

I tried to pass the parameters:
 
	ether=11,0xd400,eth0 

at the lilo prompt but lilo wouldn't accept it.
So I appended it to lilo.conf and ran sbin/lilo.
It still won't work.

Any help? I'll give more details if anybody needs them.

Thanks in advance.
