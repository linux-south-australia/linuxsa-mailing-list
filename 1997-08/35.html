To have a multi-version undelete I think you would need to have a
journeling filesystem, which ext2 isn't. For the moment if you need
something like 'undelete' there is a replacement for 'rm' that will just
move the file to a 'trash' directory. Like delete sentry(?) on dos.

> > Normally the space that is freed by deleting a

> >good knowledge of the filesystem structure you can recover some or even
> >all of the data, provided it hasn't been reused. 
> It would have to be some really expensive data to justify someone
> sitting
> there splicing all the loose sectors together.

True, but if it was some really important data then there should be
backups :). I have read that Linus Torvalds had to reconstruct his
filesystem containing the latest work on the kernel 'cause it got trashed,
but that was a fair while ago.
