Linux not recognizing xtra memory

Hi folks, me again.

I have discovered that linux not being able to recognize the xtra memory
was caused by a bios setting called "OS/2 Compatible mode".

I have an American Megatrends bios, and since i run OS/2 as well, i
had this turned on. After turning this off, KAPOW, 48Mb - warp factor 9,
swap file usage 0!

Interestingly, turning OS/2 Compatible mode off had no noticable affect
on OS/2! I wonder what this switch is for?

Thanx 2 those who replied.
