> While trying to compile in sound into my kernel
> I used rpm to install a package kernel-2.0.27.src.rpm which
> looked like it reinstalled a kernel over an already beautifully configured
> one.

It shouldn't have.  When you unpack a *.src.rpm package, it will be
placed into /usr/src/redhat/SOURCES and /usr/src/redhat/SPECS.  You
probably don't want the kernel-*.src.rpm as that is used to build the
actual installable kernel packages (eg. kernel-source, kernel-headers,
etc).

My suggestion:
- don't use the kernel*.rpm files.
- rm the stuff you installed into /usr/src/redhat.
- download linux-2.0.whatever.tar.gz into /usr/src and unpack it from
  there.
- cd linux; make config; etc.
