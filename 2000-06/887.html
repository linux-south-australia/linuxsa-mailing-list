Hi,

I did a course on Solaris 2.x System Administration. It wasn't put on by
Sun but by a company here in the UK called Quanta.

The topics covered were
System Administration
	- The superuser
	- Switching users
	- Using Admintool
	- Using Solstice admin suite
	- Documentation
	- Answerbook 1 & 2

User Management
	- New user considerations
	- Group management
	- Adding new users manually
	- Adding new users using admintool

Security
	- Understanding the password file
	- Security files
	- syslogd

Printer Management
	- Printing files
	- Cancelling print jobs
	- Installing local printers
	- Moving print jobs
	- Filters and forms
	- Remote printing
	- Using admintool to add a remote printer
	- Solaris 2.6 - The new printer environment
	- Adding a network printer
	- Troubleshooting printer problems

Filesystems
	- SVR4 Directory layout
	- Adding a new disk
	- Filesystems
	- Partitioning a disk
	- Making a filesystem
	- Unmounting a filesystem
	- Filesystem checking
	- Monitoring disk usage
	- Making room on a filesystem
	- Maximum file sizes and disk quotas

Processes
	- Background processes and Daemons
	- Killing processes
	- Examining system activity and input/output statistics

Timing and Scheduling
	- Setting the system date
	- Scheduling one off jobs and regular job scheduling
	- Security considerations

Backup and Restoring
	- Logical backup devices
	- Backup types
	- The ufsdump command and the ufsrestore command
	- The cpio command
	- The tar command
	- The dd command
	- Disaster recovery
	- Forgotten root password

Booting and Shutting down
	- The boot sequence
	- The boot prom loader
	- Initialisation states ( run levels )
	- Startup scripts
	- Shutting the system down

Packages
	- Installing packages
	- Removing packages


As you can probably guess, I got those heading direct from the literature I
was
given when I attended the course.
The course I attended was excellent. Each person had a machine to themselves
with
which we could practice what the instructor was teaching us.  The machines
weren't brilliant ( Intel's not sparcs ) but they were adaquate.
The course went very well ( apart from the bit where I managed to convert
the
home partition into swap space ).

Overall, a very good course.

Stuart Jeffery.
PC Systems Admin.
CarnaudMetalbox Engineering
