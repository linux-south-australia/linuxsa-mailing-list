Re: Linux and NT

Hi...

> Can a linux box be set up to "log in" to an NT network ala 95/98
> 
> I realise that access to shares etc can be done through samba but I was
> wondering if, for example, you had an NT server that controlled all of the
> networks access (ie www, email, etc) and that access to these services could
> only be obtained by users that were logged in to the system. How would you
> go about getting the NT box to realise that the linux box was logged in to
> it?

Exactly what is it that you want to do?  You have an NT box that provides
certain services.  Options that I can see (possibly it's neither of these):
1 When a user is logged on to the Windows NT network on a Windows box, they have
  access to all the services without entering their password again and you want
  this to work similarly when the user is logged in to a Linux box, or
2 Machines that belong to the same domain as the server should have access to
  services provided by the server.

Corresponding solutions:
1 Tricky.  I suspect you'll run into troubles when there are multiple users
  logged on to the Linux box.  You could try pam_smb (authenticates users from
  the NT domain when they log in to the Linux box), but I have no idea whether
  or not it will do what you need.
2 Try domain level security with samba.  Check out
  http://www.linuxworld.com/linuxworld/lw-1998-10/lw-10-samba_p.html.

Am I close?  If you provide more information I'll have another try...  ;-)
