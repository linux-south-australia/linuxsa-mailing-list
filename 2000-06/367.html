Re: Postgresql vs MySQL

David Lloyd wrote:
> Therefore, I guess my question lies in:
> 
> * speed (given same factors, eg same machine, same hard disks, executing
> the same set of commands)

The MySQL web page lists some performance figures for itself, postgresql and
several other databases.  The executive summary?  All depends on what sort of
operations you're doing frequently.  If you know that you're doing more
inserting than querying go for XYZ.  If you're formatting complex queries go for
ABC. etc. etc. etc.  I think to get a winner you need to characterise your
usage.

> * ease of administration/installation

Hey, just an RPM away! :-)  But if you must compile I seem to remember that both
build easily from the standard condigure; make all; make install routine.  If
not, they should :-)

If you have two pieces of software that are much of a muchness I always go for
the free-est version - in this case postgresql.  Feel free to differ though.
