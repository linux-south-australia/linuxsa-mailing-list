If patch is installed, try "which patch".  This will only work if
patch can be found in your PATH (ie, there are differences between
root's PATH and a normal low level user PATH).  Otherwise, nothing
like a find to suck up CPU/disk cycles and annoy other users on the
box.

Note: rpm -qf file will only work if the package is actually installed
(I'm unsure if you already knew this from the context of your message).
First thing I usually do when I get a new set of RedHat disks in is
run the following (from memory) across the CD:

for file in `find /mnt/cdrom -name "*.rpm" -print`; do
    echo "===" ${file}
    rpm -qlp ${file}
    echo
done > RedHat_6.x_filelist

Then grep/less/vi/whatever are your bestest friends.
