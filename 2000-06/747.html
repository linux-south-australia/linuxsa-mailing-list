Re: HTML formatted mail messages

From: Lindsay Whitbread <linuxsa@dingoblue.net.au>

> <PLAIN TEXT>
> I've noticed a lot of people get cut up about HTML formatted emails. I've
> also heard that a lot of people bin any HTML formatted emails.

I luv html. For everything.

It is true that some preferred mail readers will only deal in plain ASCI,
and hence you do lose quite a large audience.  But the real objection (which
I think is fair enough) is that html is a bandwidth hog. messages are a
conservative double size and more realistically 3 x. This before the usual
temptation of adding grafix. If you're sifting thru 300+ emails per day,
that modem led is on for a lllllllloooooooong time.

Anything else you hear is a religion and becomes subjective rather than
realistic as to which is / isn't 'better'.

Bottom line is, if you're on a mailgroup that prefers plain ASCII,
regardless of reason, it's bad nettiquette to do otherwise. I'd prefer html
any day because I believe it is becoming the emerging documentation
standard, but I'm not about to shove that down anyone else's throat.
