Re: Ghostscript and PDF output

Martin Sheppard wrote:
> 
> On my system, ghostscript comes with a wrapper script called ps2pdf that
> will handle all this for you.

Which is proken, because it uses Letter paper as the default.

My makefile include is attached.  To use it, write a Makefile
containing:

  #!/usr/bin/make -f
  include ghostscript.mk
  fred.pdf: fred.ps

I have a series of makefile includes for LaTex, DocBook,
Word, RTF, XFig and so on.  To produce output I then
say in the Makefile:

  fred.pdf fred.html: fred.xml fred1.fig fred2.fig

It's also useful to have a dummy target that prints all
the documents in a Makefile, so

  make printall

runs off a copy of the project documentation and

  make htmlall

uploads the documentation to the web site.

#
# pstopdf.mk -- Makefile include for creating PDF files
#
# $Id$

# Configuration
PAPER=a4

# Programs
GS=gs

# Rules
%.pdf : %.ps
	$(GS) -q -dNOPAUSE -dBATCH -sDEVICE=pdfwrite -sPAPERSIZE=$(PAPER) -sOutputFile=$@ -c save pop -f $<

%.pdf : %.eps
	$(GS) -q -dNOPAUSE -dBATCH -sDEVICE=pdfwrite -sPAPERSIZE=$(PAPER) -sOutputFile=$@ -c save pop -f $<
