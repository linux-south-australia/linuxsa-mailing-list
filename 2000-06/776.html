Lockup problem

Guys,
I wonder whether you might be able to help diagnose a lockup 
problem I am having on a RH6.1 server.

First, some background.  
I run a Linux server that is part of a Air Traffic Control simulation 
environment,  with a total of 6 servers connected world-wide giving 
users free access to experience interaction between other users in 
either an ATC or Pilot role.  Almost all of the software the users 
require is free, except for the flight simulation component, where 
commercial products, MS FS95, MS FS98, MS FS2000, Fly!, 
Airline Simulator 2 or Aerowinx PS1 can be used.


Communication between pilot and ATC is either done via text or via 
voice over the internet, using a freeware product called RogerWilco.

It is the RogerWilco  base station server software for Linux which I 
think is causing my server to lock up.

Actually, the server doesn't quite lock up.  When this software 
goes belly-up, the server still responds to pings and traceroutes, 
but refuses to answer on any port.  On the console, I can switch 
between sessions using the F keys, but get no response when 
attempting to log on at the console.

My question is, exactly what state is the server in a this point, why 
is it able to respond to ping/traceroute but nothing else, and how 
could this occur.

Because I don't understand how the server can get into this state, I 
dont really know what I should do as the next step in investigating 
this problem.

Any ideas?
