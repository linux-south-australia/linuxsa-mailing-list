This doesn't really help, but may be of interest.

Don't know about Postgresql. But with Mysql, one reason for its good
speed/reliability is the 
fact that it was ALL written form scratch. IE. it wasn't built on top
of old code. They didn't keep old code/modules etc. They wrote
everything using the latest technologies in algorithms, code structures
etc.

Some complex features, that are rarely used, have been left out, again
to improve speed. I think the 2 you mentioned above are examples.

We have been using Mysql for all or clients. Had no problems so far.
