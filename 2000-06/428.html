Windows 2000 is a nasty beast. I boot between Linux, Linux, Solaris, NT
4 (3 different installs), NT 3.51 and FreeBSD. I think Win95 is there
somewhere too. ;-).

I had a problem with Windows 2000 "updating" my NTFS partition that had
NT member servers, PDCs, BDCs and Workstations (for testing the
marvelous Samba TNG) and I found that W2K made perfectly sure that the
NTFS filesystem was no longer backwards compatible. I think I remember
something in the install about "corrupt of unformatted" partitions. I
still managed to make it out OK, though.

Try booting off a floppy distro (TomsRTBT, pocketlinux, cclinux, etc)
and if you can access the data you are after, try dumping it across a
LAN or to another local disk. When all else fails, you could try an
upgrade install of Linux. It may be able to do something you can't

Be careful ;-),
