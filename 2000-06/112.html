Re: IMAP - server side filtering

Lindsay Whitbread wrote:

> I've got imap working, and am succesfully sending and receiving mail
> using sendmail / fetchmail and imapd but obviously all my mail is just
> getting dropped in my inbox.

The simplest way to to get fetchmail to send your mail to different
SMTP addresses, like so:

  gdt@bush.aarnet.edu.au             <-- from chief account
  gdt+linuxsa@bush.aarnet.edu.au     <-- from linuxsa account
  gdt+installfest@bush.aarnet.edu.au <-- from installfest account

If you use UW IMAP and procmail (the Red Hat default set up) then
the name after the "+" is in $1, so you can have rules like:

  :0:
  $1=="linuxsa"
  linuxsa

to put the mail to the account "+linuxsa" into the "linuxsa" folder.

If you use the better Cyrus IMAP then you can't use procmail.  It
has its own filtering language, which IMHO sux big time[1].  However,
it does automatically put the e-mail to "gdt+linuxsa" into the
"linuxsa" folder (if the "linuxsa" folder already exists).

Personally, I'm beginning to simply subscribe the e-mail address

  glen.turner+listname@aarnet.edu.au

to the lists I'm on and have the messages automagically appear in the
"listname" folder.  Having a look through the majordomo files for
aarnet.edu.au suggests that quite a few people already do this.

Glen
