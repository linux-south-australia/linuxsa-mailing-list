Re: Korn shell

either you don't have ksh in your path (do `which ksh`) or there is
something in the .kshrc (or .profile or .whatever ksh uses -- possibly
/etc/profile) file that isn't accessible. Try making sure that all commands
that are to be executed are accessible in the path that they have AT THE
TIME... :) ie, if you set your path at the end of your .profile, the path
won't be accessible at the start, when you run (say) `biff&`... Put absolute
pathnames all over the place, and see if that works...

HTH
