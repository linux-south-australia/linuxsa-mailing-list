Postgresql vs MySQL

Hi!

I am having an internal, philosophical debate as to which one of this
might be better under the following circumstances:

* no licencing issues

I'd be prepared to pay a reasonable licence to each of them hence
MySQL's pay if you make money policy doesn't really make a difference to
me.

* no need for subselects or transactions that are able to be rolled back

The application I would use the SQL database doesn't require either of
these features that MySQL currently lacks

* no need for an OO or procedural internal specialised language

Personally, I'd prefer to leave procedural stuff out of SQL and besides,
why learn yet another language when PERL::DBI can help you out>

Therefore, I guess my question lies in:

* speed (given same factors, eg same machine, same hard disks, executing
the same set of commands)
* efficiency in stored database format
* ease of administration/installation
* security issues

Any comments would be appreciated. If the list thinks this is off topic
- which it might - then please e-mail me directly about the matter.
