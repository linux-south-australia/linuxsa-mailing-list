Re: LILO (again, forgot one small point)

At 01:34 PM 6/12/00 +0930, Damien Wilson wrote:
>Hi...
>
>> For personal reason, I upgraded Win95 to Win98, but noticed during the
>> upgrade process the LILO prompt didn't appear.
>>
>> I get the impression this has been removed from my MBR while upgrading.
>
>Bingo, everytime you re-install or upgrade your OS the MBR gets re-written
>to suit that particualr OS be it Linux, Windows, BeOS etc.

Well, at least Linux allows you to choose not to write the MBR ...

>> My question, Can I somehow replace the LILO to my MBR without having to
>> lose everything on my Linux partition - that is, if it already hasn't
>> been removed. As per normal, Windoze is only showing me what is left on
>> my windoze partition.
>
>Don't worry, your linux partition is still there, to get it back do the
>following:

Yep, it sure is.

> - If you created a boot disk during the linux setup, boot from that and
>re-run lilo by typing /sbin/lilo from the prompt

Yes, that is one way.

> - If you didn't create a boot disk (shame, shame, shame) grab your install
>CD and perform an upgrade of your linux system. This re-writes the MBR with
>LILO without trashing your system

Well, no need to do that :-)

If you used RedHat, then use the following command from the boot prompt
from the RH Install CD:

   linux single initrd= root=/dev/hdax

Where hdax is the device where the root file system is.

This tells it to boot into single use mode, and to bypass the installation
script and to set your root file system to what is on the disk.

Then, since your lilo.conf has the right things in it, simply type:

  lilo -v

And it will write a new MBR with LILO, and since it was already set up for
dual booting, you should be able to reboot and all will be ok.
