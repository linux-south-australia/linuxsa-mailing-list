Firstly, there shouldn't be a requirement to recompile for such a common
driver as ne2000. But, having done that now you need to

depmod -a

this will resolve missing links in your module database (if any)

Second, you deserve what you get. 'insmod' is an internal command not for
public consumption, all it does, is attempt to load _the_ driver. You need
to use 'modprobe', it loads _the_ driver, and all associated modules that
that driver might need. There is _never_ a reason to type insmod. Not never.

The traditional resources for an ISA ne2000 nic card are io=0x300, irq=10
You need to specify _both_ on the command line as the driver will not accept
a single parm.

Finally, if you are using such a card, it is highly likely this animal is a
pnp. In which case, you must first resolve this problem before anything
else. Don't even attempt to install *any* isa type card, sound, ethernet or
whatever, till you fix up isapnp->  on the site below. You'll just have to
trust me on this one, if you have isaPNP anything, you cannot pass go, you
cannot collect $200 until you fix that problem first.
