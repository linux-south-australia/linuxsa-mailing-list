Not one and the same thing, something else happened, and hopefully is of no
importance.

"depmod -a"

should fix this for you.

Technically, modprobe is reporting that the database it relies on,
modules.dep is not current. One or more modules that are contained in that
database are 'stale', the files they point to are NOT the files on your
system.

This occurs quite naturally on any recompile of the kernel, as in make
modules_install. In a redhat system depmod -a is run each and every time you
reboot, specifically to 'get round' this issue. But, regardless, a refresh
of the database will fix the first problem

the second issue is that you _might_ have inadvertently not compiled some
dependent modules, which will make depmod scream. You can fix all that by
using the script recompile->scripts-> on the site below.
