Re: Linux and FreeBSD on the one machine

On Fri, Jun 02, 2000 at 09:16:01PM +0900, Richard Sharpe wrote:

 > I now have FreeBSD and Linux on the one machine in a dual boot setup.
 > However, it seems that FreeBSD does not like to use extended partitions.
 > Can anyone confirm this or tell me how to use extended partitions?
 
The first four partitions show up as s1, s2, s3 and s4 appended to 
the device name (so "slice" 2 on your SCSI disk will be da0s2).

Extended partitions show up as slice 5, 6, 7, etc, etc.  If you don't
have pre-existing device nodes in /dev for them, use the "MAKEDEV"
script provided, e.g.:

     # cd /dev
     # ./MAKEDEV da0s5

... to create slice 5.

FreeBSD also only needs *ONE* "real" disk partition, incidentally.  On
a Linux box you'd use two BIOS partitions, one for filesystem and one
for swap.  On FreeBSD, you set up one as a "FreeBSD" partition;  you
then subdivide that single partition into /, /usr, /var, swap, and 
whatever else takes your fancy when you get to the disk label editor
in the installation program.

That means you can conceivably have FreeBSD, Linux, Linux swap and
Windows partitions on the same disk without needing any extended 
partitions at all.
