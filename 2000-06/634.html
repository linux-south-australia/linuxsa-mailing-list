DocBook Presentation (perhaps)

I'm currently doing some documentation work using DocBook and SGML by
*HAND*. I'm wondering if the attendees at meetings would like to see a
presentation covering DocBook. The presentation would cover:

THEORY
* basic SGML
  - Getting the DTD to behave itself
  - Entity References
  - Formatting expectations

* basic DocBook
  - tags useful for writing man pages (and similar things)
  - tags/template useful for writing articles
  - the Book tag

PRACTICAL
* DocBook Tools
  - DocBook itself
  - Getting DocBook setup on your system
  - Jade/OpenJade and using DSSL Style Sheets to render the SGML

* Structuring Documents
  - Not how you write them, the directory structure.

For those that don't know, DocBook is the accepted standard for writing
documentation for the Linux Documentation Project. It's used because
it's extraordinarily flexibly, but at the same time very standardised,
can be rendered into many formats very easily and relieves the
typesetting burden from the author.

I'd need about 1.5 weeks to prepare for this if it happens. What do you
think?
