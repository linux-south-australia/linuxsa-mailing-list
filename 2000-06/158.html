I had that problem on Netscape 4.7 on the NT box at work.. open a page
with Javascript and if it were left in the background (while i did other
work) for long enough the memory usage would climb until it was all of
physical RAM and all of swap space minus other running programs, and the
thing would thrash until NT died (interesting that NT died, and Netscape
didn't run out of memory and die, even tho all memory was allocated!).

as did leaving the page (back button, link, whatever) and the thing
stopped thrashing. Haven't seen this but with Nutscrape 4.7 on Linux,
but I don't usually leave the pages sitting long enough to do it...
