Socks5 problem

Greetings peoples,

This one's got me stumped, and it seems to have other people on the 'net
stumped as well judging by an internet search.

Situation... Have a firewalled PC and a non-firewalled RH6.0 linux
box. The linux box has the NEC socks5 server running on it. The
socksification works for programs running under sockscap32b (also an NEC
product), and I can ftp outsuide the Uni's firewall. When it comes to
applications that have builtin socks support like ICQ or MS Net meeting,
it refuses to connect, returning an error like:
Jun 23 17:11:17 escher Socks5[22212]: Auth Failed: (129.127.128.64:2303)
in the system log.

Using Ethereal I was able to watch the authentication session, which was
fine, except the User Authentication reply from the client returns an
error saying FRAME TOO SHORT in the socks protocol section of the packet.

The weird thing is that we have an identical configuration elsewhere in
the Uni running on a Sparc running Sparc RH 6.0, and the client works
fine, no errors, so it must be the server, I've carefully checked the
config of evertything else, and they are the same.

Both are installed from RPMs from the RedHat contrib directory.

Has anyone encountered this error before, and/or know what might be
causing the problem.
