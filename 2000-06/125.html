Re: IMAP - server side filtering

Glen Turner wrote:

> Personally, I'm beginning to simply subscribe the e-mail address
> 
>   glen.turner+listname@aarnet.edu.au
> 
> to the lists I'm on and have the messages automagically appear in the
> "listname" folder.  Having a look through the majordomo files for
> aarnet.edu.au suggests that quite a few people already do this.

Just to point out, this only works if the SMTP server where your
pop mailbox resides supports "plussed" (where "+" can be a "-")
users.  Sendmail and qmail support this if configured (and if exim
supports it, I'm sure Dan Shearer will mention it :-).  The pop
server I pull my mail from doesn't understand this, so it thinks
that "dt+foo" is a seperate entity from "dt".  :-(  It also makes
for a handy way to track where a spamster retrieved your mail
address from.
