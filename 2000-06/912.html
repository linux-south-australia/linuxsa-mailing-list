I recommend that you install the ip_mask_ftp module. Using that you don't
need to use PASV mode.

As root:

modprobe ip_masq_ftp

and it should load.
 
> I can't seem to get a directory listing of a clients html directory on
> ftp.senet.com.au
> Using ftp clients in PASV mode and/or c:\ftp ftp.senet.com.au I can login
> fine but I get the following
> 
> 500 Illegal PORT command

This means that your client has told the ftp server to use a certain port,
but when the server tries to connect to you on that port, it can't. Are
you _SURE_ the client is using passive mode? If you use the ip_masq_ftp
module, you shouldn't need passive mode, as I said above.

Incidently, what FTP program are you running? Try as I might, I can't even
find the command in DOS FTP to set passive mode.. and Internet Explorer
doesn't use it either.

> Ok I guess it has something to do with the firewall and port 21????
 
Not port 21. One of the ftp_data ports, but you are correct, it is because
of a firewall and the ftp ports. Is there another firewall between yours
and SeNET?

> However other ftp sites work fine??? Why only SeNet?

Are you doing _exactly_ the same on other ftp servers?
