Re: Web Pages

> 	As a rule I much prefer to start with Composer

Many people (eg me!) find Netscape Composer unusably unstable. It seems to
depend on many things and I can't be bothered finding out what given the
status of the product.

I was reminded by Geoffrey's posting about Ross Williams of his FunnelWeb
package (http://www.ross.net/funnelweb) which is one of several similar
languages. I've been vaguely aware of FunnelWeb and now I've read the
manual I intend to try it for web page maintenance. It appears to be a
nicely designed generic text handling macro package well-suited to
maintaining static web pages. I expect it could be wrapped up into an
Apache module too as a kind of SSI language if someone was keen enough,
but I only need static pages for now.
