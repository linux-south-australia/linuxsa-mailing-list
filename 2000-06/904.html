RE: FIXED RH 6.2 refuses to install properly

* When I get to the login prompt and type in a login name it doesnt matter
* whether I type a valid one or not, and I don't get a chance to enter a
* password it just goes on with loading X, and I cannot stop it from doing
* that.

That's because your system is booting to init 5. It's easy enough to fix;
you'll need to get some kind of terminal up (whether you open up a terminal
under X, or if you telnet into your computer) as root. edit the /etc/inittab
file and change the line that says "id:5:initdefault:" to say
"id:3:initdefault:". Note that all I did was replace the number 5 with a 3.
It's a bugger to find, but easy once you know how. :)

I'd be happy to help further if you'd like more specific assistance...
