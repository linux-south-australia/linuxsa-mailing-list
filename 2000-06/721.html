> i'm currently using the  standard
> /sbin/ipchains -P forward REJECT
> /sbin/ipchains -A forward -j MASQ -s 192.168.1.0/24 -d 0.0.0.0/0
>

Those rules won't affect dhcpd, which don't use the forward chain. Are you
sure that your dhcpd server is running and that it is configured properly?

You could try:

/sbin/ipchains -I 1 input -j ACCEPT -s 192.168.1.0/24
/sbin/ipchains -I 1 output -j ACCEPT -d 192.168.1.0/24

That that does is make the first rule in both the input and output chain to
accept packets from and to your local network.
