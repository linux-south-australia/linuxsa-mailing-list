installfest web page authentication?

Hi all,

Developing a database administration web page for installfest and I have
a couple of questions for you database types out there...

I'm doing stuff like:

      Password for admin functions:
      <form action=admin.php3 method=get>
      <input type=PASSWORD name=pw value=''>
      <input type=submit name=submit value=ENTER>
      </form>

so administration functions (ie deleting/modifying records) aren't
available to anyone.  Using PHP this web page can send the data back to
itself, and implement some security within PHP, but of course the
browser displays the URL as admin.php3?pw=foobar if foobar were the
password used.

How can I use PHP/postgres/apache to implement some form of web page
authentication without resorting to interfacing to Linux user
authentication?  These pages will be guest hosted by netcraft so the
less interaction outside of php/postgres/apache that occurs the better. 
Is there some simple way to pass variables to web pages without them
being displayed in the URL window on a browser?  Don't care about text
being sniffed off the network.

Thanks in advance,
