Linux, Seti @ Windows 2000

I run two identical computers that use an AMD Athlon 550, Asus K7M
mainboard, 256Meg of memory and two Seagate 30 Gig Barracuda harddrives.

One system runs Mandrake 7.02 and is used for all my general computer work,
the other system runs Windows 2000 Desktop, which is used to run my audio
restoration software.

I have Seti@Home Client running on both machines and have noticed that the
Linux box completes a work unit faster than the identical box running
Windows2000 Desktop. The linux box completes a work unit, on average in 7
hours 45 minutes 20 seconds. The Windows box completes a work unit on
average in 9 hours 2 minutes 40 seconds (which is about the same time it
took to complete a work unit in Windows98)

My Linux box is nearly 2 hours faster in completing the same work unit in
Windows.

Could someone hazard a guess as to why Linux performs so much faster with
floating point calculations.
