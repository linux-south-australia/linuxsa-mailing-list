Re: Tests in sh

Eric March wrote:

> I copied a script out of a book, but bin/test is unknown
> How do I do following on Linux?

Apart from Dan's answer (which is correct; test under RedHat anyway
is located in /usr/bin), there is something else I wanted to point
out.  /usr/bin/[ (yes, open square bracket) is linked to /usr/bin/test,
and can improve the readability of your scripts (well, in some people's
minds anyway).  So in your example script where you have something
like:

> /bin/test $# -eq 2 && /bin/test -d $FROM && /bin/test -d $TO || \
> { echo usage: $0 \<from directory\>\<to directory\> ; exit 1;}

can be written as

if [ $# -eq 2 -a -d $FROM -a -d $TO ]; then
  echo usage: $0 \<from directory\>\<to directory\>
  exit 1
fi

(note that && is a shell construct, where as -a is the AND operator
under test).

And ...

>     if /bin/test -d $i # a directory
>        then continue
>     fi

Becomes

      if [ -d $i ]
         then continue
      fi

or (my preferred method):

      if [ -d $i ]; then
         continue
      fi


Readability 101, but this is probably a contentious issue, much like C
source formatting.
