Re: Latex problem - used ... string characters out of 72048

> I have a co-worker who has a 420 page latex document and he want to go
> larger. (It's produced automagically by cxref, to document some code.)
> 
> Unfortunately latex spits with a message like..
> 
> used ??? string characters out of 72048
> 
> It looks like latex has some hard limits. Does anyone know how to
> change them? Do ai have to recompile latex?

Yes.  TeX (the underlying formatter over which LaTeX is built) has
some hard limits.  However, you should not need to recompile to avoid
these, since teTeX distribution 1.0 and greater have tools to deal
with this problem.  See (on a RedHat box, and probably others)

    /usr/share/texmf/doc/index.html

Note in particular

    What's New in teTEX 1.0

	...

	Based on Web2C 7.3. This allows the size of TeX's tables
	to be set in the configuration file texmf.cnf and easily 
	adjusted.


Try running the texconfig program (it uses dialog and runs on a
console).


I have not met this limit myself, but I have used texconfig.
