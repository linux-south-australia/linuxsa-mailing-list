Re: Sendmail filtering

Matthew Geddes wrote:

> If I just wanted to bounce messages based on the attachment file type, I
> shouldn't need to. I could just bounce (or better still, drop) the
> message.
> 
> something like:
> 
> if (grep ($SMTP_attachment_is_here_string,$message)){
>         drop();
> }

I don't have any copies of the virus about to check what type it's
using, but the string you would need to match is something like:

Content-Type: application/octet-stream;
        name="something.vbs"
Content-Disposition: attachment;
        filename="something_possbly_else.vbs"

(which may occur multiple lines, or may not).  And you would have just
deleted this mail message, because I mentioned the appropriate string
in a valid context (outside of a MIME encoded message).  Had you MIME
decoded the message to begin with, then you wouldn't have had this
false positive match.

Grep is _not_ the answer (at least, not at this level).
