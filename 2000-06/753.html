Re: Samba and simultaneous file access

On Sat, Jun 24, 2000 at 09:56:57PM +0930, Mike O'Connor wrote:

 > I read as the smb.conf Man and FAQ which were installed with RPM and none of
 > the locking options seemed to fit my problem.
 
If you want all I/O on files which are used by someone else to be 
enabled, simply turn off all locking, and don't come crying to me when
all your files get corrupted.

If you want something approximating the behaviour of a Windows file
server, set "locking = yes" and "share modes = yes".  These options
are actually the defaults, so I'm not entirely sure why it isn't working
for you :-/

There's also a file in the documentation directory which comes with
Samba called NT4-Locking.reg.  grepping through the rest of the docs,
though, I can't find any mention of what this file is actually for :-/
However, if your SMB client machine is an NT4 system, perhaps backup
your registry then apply the updates from that file to see if it
makes any difference.
