Yep, totally agree with your point. The point I am trying to make is
Mysql is a brilliant DB for its purpose. If you need a DB for web
pages, its the way to go. Its fast. The internet already is slow, you
don't want an overly complicated DB that slows it down even more.

Transactions can be done with Mysql. If your application requires heavy
transaction support and speed is not an important issue, than I would
go for some other DB. Every application needs to weigh its priorities
to make a decision. We have set up many shop fronts for many years,
with SIMPLE transaction support with no problems.

Considering the original poster said 'no need for subselects or
transactions that are able to be rolled back', I still recommend Mysql
;)
