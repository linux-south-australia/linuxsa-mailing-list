Martin Stacey wrote:
> According to /var/run/gdm.pid, gdm was already running (some process number
> is displayed)
> but seems to have been murdered mysteriously
> 
> and the following in my messages file;
> 
> gdm[4546]: gdm_slave_windows_kill_ioerror_handler: Fatal X error -
> Restarting :0
> gdm[4549]: gdm_slave_windows_kill_ioerror_handler: Fatal X error -
> Restarting :0
> gdm[4552]: gdm_slave_windows_kill_ioerror_handler: Fatal X error -
> Restarting :0
> gdm[4555]: gdm_slave_windows_kill_ioerror_handler: Fatal X error -
> Restarting :0
> gdm[4542]: Failed to start X server several times in a short time period;
> disabling display :0
> 
> How can I fix this?

I presume you're not already running an X server on display 0... If you 
are, it would be wise to kill this or reconfigure gdm to spawn on 
another display...

The first thing I'd check is whether X is working at all.  To check, 
perhaps try running something simple like X or xinit at the command 
prompt.  If it's not, the easiest way is probably to run an X 
configuration tool such as Xconfigurator to get X running.

You may also get similar error messages if another display manager (such 
as xdm or kdm, or even another instance of gdm) is running, but again, 
I'd check whether X is working first.

If you're still stuck and looking for clues, perhaps you can get gdm to 
provide verbose output and redirect that to a log file somehow.
