Re: Restricting Telnet access to FTP Server

Hi Sekar,

Obviously, ipchains/iptables would be a good idea if it is
allowing telnets based on where (what IP) the connection
is from.

However, I am also ruthless about never giving usable shells
to accounts that are intended for FTP/POP3/etc only.

Set the shell of non-admin user accounts to be "/bin/false"
(note: you will need to add /bin/false to the /etc/shells file to
make it a valid shell definition).

It is pretty hard to get telnet access without a shell ;-)

Cheers,
-Dan
