Re: WebCams

>There are heaps of Linux webcam streaming programs out there, some
>quite powerful, at least for the task you seem to want them for. There
>are motion detectors (http://motion.technolust.cx/), streamers
>(http://www.ee.up.ac.za/~justin/bttv/) and many more.

If you're prepared to put in a little more effort, have a look at
http://mpeg4ip.sourceforge.net

I use a 3-camera home-brew linux MPEG-1 system with 'motion' and cheap PC-TV
cards, and a 4-camera windows MPEG-4 system that came in a box, with custom
software for its 4-input capture card.

I have to say that MPEG4 is a lot more pleasant to work with, as it's
time-independent nature makes it much more practical for security video and
streaming. Because it's difficult to stream mpeg-1, you can't see what's
happening until after it happens. You can use the V4L video loopback device
to run more than one V4L application, but that just chews up more cycles on
a system that is already running at high load (compressing mpegs in realtime
is kinda CPU intensive..)

motion still has an open-source advantage, I've even contributed patches to
it in the past. For more esoteric security applications it's convenient to
be able to put triggers in the code to make other things happen (disco comes
to mind for some reason :).

motion + mpeg4ip combined would make an excellent security package.
