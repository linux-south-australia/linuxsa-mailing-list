Michael Davies wrote:

>On Tue, 2002-02-19 at 20:50, David Delyn wrote:
>
>>  Hi,
>>Can some one point me in the right direction please.
>>
>>The Problem
>>I am using Mandrake 8.1 which uses rpmdrake to manage software installation
>>I have updated Mozilla using the online package installation from their 
>>site to install v0.9.8
>>before which I removed the Mozilla rpms installed and the associated 
>>packages which rely on it.
>>Now of these packages I only really wanted to replace Galeon for which I 
>>downloaded the latest RPM 1.0.3.2.
>>However, it stalls on installation requiring Mozilla 0.9.8 as a 
>>dependancy before installation can continue.
>>The latest rpm I can find of Mozilla is v0.9.4.4
>>
>>My Question
>>Is there a way to tell the rpm system about tgz installed programs?
>>Ive noticed this before, but as the tgz's and rpm's were independant I 
>>just kept note of the programs.
>>
>
>Simple Answer: No.
>
>Complex Answer: You could futz around with the RPM database but I
>wouldn't want to do it.
>
>The best bet is to stay with the RPMs for your distro.  Do you really
>eed the bleeding edge?  If so, Red Hat has "rawhide" RPMS, which are
>pretty much the latest and greatest, but haven't been tested to work
>with everything else, so you can have the latest and greatest if you are
>willing to risk some instability.  Under Manduck I'm not so sure.  You
>could try their Manduck Users Club so whatever it's called - someone may
>have built new RPMs for some newer versions of software.
>
>The other hackish way is to reinstall your old Mozilla via RPMS so the
>database is updated, and then remove the mozilla files (rpm -ql mozilla)
>and insall your tgz.  Or install both mozilla and galeon from tgz and
>not from RPM at all.
>
>But don't do it.  Install a Galeon RPM that matches your Mozilla RPM. 
>You won't have the latest of everything, but it'll all work.
>
