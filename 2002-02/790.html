Re: NAT Accounting

Adam Smith wrote:

> I'd like to be able to be able to see for each IP address how many bytes
> are being NATted in.  That way I will be able to accurately see how much
> data is being passed in from the Internet, and in turn, find out where
> all our bandwidth is going.

It might not be helpful recommending yet another program to account for
bytes, but where flow/per-ip accounting is required and Cisco netflow is
not available, NetraMet looks like a well-developed solution.

http://www2.auckland.ac.nz/net//NeTraMet/

A quick look at their web site suggests that it now supports dscp as
well, which is cool, because you can get your router to set this value
in IP packets to differentiate local traffic from Internet traffic.
