Re: Printer Drivers

Martin Stacey wrote:
> Does anyone know where you can get drivers for a HP LaserJet 5L for Red Hat?

This printer probably does not need drivers, as it will be able to 
understand the native PostScript output that Linux/Unix machines 
typically send.

Some file types, however, may need to be translated into PostScript 
output.  One way of doing this is to install a product like ghostscript, 
apsfilter, enscript, or a2ps (all being designed for slightly different 
jobs), and configure your printer to use that as an output filter.

The easy way to get all this working on a Red Hat system is to select 
printer support during installation, then run the configuration tool 
(printtool in Red Hat 6, printconf-gui in Red Hat 7) and select your 
printer from the list.

If you don't have printer support installed, you will need to install 
LPRng, ghostscript, and foomatic from your Red Hat installation media.

I've tried to give some background (as I understand it) about how 
printing works, and how you might be able to get it working.  If none of 
this makes sense, run "Printer Configuration" from the System menu. :-)

I hope this helps.
