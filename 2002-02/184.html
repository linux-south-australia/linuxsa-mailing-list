Re: PHPNuke & 'Security Risk'

On Wed, Feb 06, 2002 at 11:37:08AM +1030, Matthew Western wrote:
> and if i have the uname/pass in the config.php isn't it technically possible
> to just download that file if i know it exists?  doesn't seem like a very
> good idea to me....
> 

If the config.php file consists solely of what you have pasted below, then 
there is no problem with having it somewhere accessible by the httpd, as 
long as the httpd will parse it (correctly) as a PHP file.  When parsed, 
the php code below outputs _nothing_ (as it only defines variables), so 
any user trying to access it via the httpd directly will only see a blank page.

If you like, I am sure it would be trivial to modify the source code of 
PHPNuke to allow loading (ie. via include() )the config.php file from an 
arbitrary location, allowing you to put it in a safer place.

> config.php ------------
> $dbhost = "localhost";
> $dbuname = "root";
> $dbpass = "";
> $dbname = "nuke";
> $prefix = "nuke";
> $user_prefix = "nuke";
> $dbtype = "MySQL";
> 
> -- 

It does not seem to me that putting the _root_ user and password is 
necessary here.  All that is needed is a user who can access a particular 
database - this user does not need to be privileged in any other sense.  
You just need to tell the database to give access to a certain user (by 
using the GRANT statement (in MySQL)).
