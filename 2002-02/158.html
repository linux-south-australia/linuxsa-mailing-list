Re: forwarding ports to another machine

>if the machine receives a connection on 'this address' on 'this port' then
>pass the connection on to 'this isp at this address' but only do it for the
>above.

If 'this port' == 25, then make sure you double-check everything before
connecting it to 'this isp'.

Most ISP's now allow relaying by IP address, so that you can easily send
mail with any 'from' domain you choose. If you redirect port 25 on an
machine to your ISP's mail server, that mail server will see a valid IP
address for relaying.

Then you turn some sysadmins proud anti-spam setup into an open-relay on the
Internet, all pointing to your IP as the culprit.
