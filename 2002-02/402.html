Re: Insurmountable problems with PCMCIA and WLAN card on a PC CHIPSmobo and Linux

Richard,
I've had this problem - I have an ISA PCMCIA adapter and some Cabletron 
RoamAbouts (previously I was using WebGear Aviator, but they don't suport 
W2k on my Laptop).

What you need to do is pick an IRQ that is available and then exclude all of 
the rest, also you may need to exclude various memory areas.

You do this (on RH7.1/7.2) in /etc/pcmcia/config.opts:

exclude irq 1
exclude irq 2
etc

include irq 9 
(for instance)

and  include only the one you want to.  This is the only way I have gotten 
it to work.  Also, it's worth getting pcmcia to only test the ioports and 
memory areas you want it to use as pcmcia seems to quite happily stomp on 
things (my older server had some dodgy IDE controller and stomping on that 
caused the machine to lock hard - req a power cycle).
