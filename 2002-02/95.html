Re: recompile modules

> i'm working on a school project that will involve modifying a filesystem
> under linux.  i've chosen to modify the minix filesystem, but i've been
> unable to figure out how to go about recompiling the minix module to
> include the changes i've made to it.  can anyone offer any guidance on
> this issue?

If you don't mind screwing up the kernel's Minix module you could just
edit /usr/src/linux/fs/minix/*

In place and if the Minix module is recompiled in your current kernel
config then your changes will be included.  Please note that this will
destroy the kernel's default Minix FS handling so don't do it if you
actually use Minix filesystems (and keep a backup of the original files
so you can revert back later... can you say patch).

If you're asking about how to turn on the Minix module.  From
/usr/src/linux type "make xconfig".

You'll end up in the kernel config.  Click the "Filesystems" button
there. Scroll down to "Minix FS Support" and select "M" for it's build
option. The files in /usr/src/linux/fs/minix will be build into minix.o
when you type "make modules"

I don't know how to compile it as a module if you're trying to build
from a separate copy of the files.  Look in the Makefile in the dir
listed above and see what's done.  It shouldn't be that hard.

Modifying a filesystem is a pretty big task for a school project - it's
more of an Operating Systems project at University.  What changes do you
want to make, and for what reason is the standard FS not adequate?
