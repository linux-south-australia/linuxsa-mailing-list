RE: Squid questions

>1. anyone know of a good log analyser that will pull the 
>access.log file and
>gimme a report of total transfer to each ip address on the network.

We use SARG here (previously Sqmgrlog), you can set it up to report on 
IP address, tho we use it per user, see below.  It gives you per user/IP 
bytes in & out plus percentage of data used from the cache or outside 
so you can work out how effective your proxy is.

http://web.onda.com.br/orso/


>2. a good how to / easy way to make people log in and then 
>track what sites
>they go to and how much data they are downloading.

Depends on your server(s), we have an NT box as our PDC and use the 
smb_auth program to authenticate users using their NT username/password.
(You could do the same using a Samba box obviously)

http://www.hacom.nl/~richard/software/smb_auth.html

of course you can authenticate using any number of things, LDAP, Radius, 
Squid itself, read the manual, above is what we've had in place for 18 
months without problems
