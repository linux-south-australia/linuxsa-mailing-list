Changing IP address under RH7.2 requires reboot?

I've noticed that in RedHat 7.2 if I try to change
/etc/sysconfig/network-scripts/ifcfg-eth0 to a different IP address and
then:

	ifdown eth0 ; ifup eth0

I get "RTNETLINK answers: Network is unreachable" (sometimes)

if I run

	/etc/init.d/network stop ; /etc/init.d/network start

then /var/log/messages tells me the interface started successfully. 
Irregardless, the interface is still listed (in ifconfig) as having the
old IP address.

Only way I've managed to get it to pick up the new IP address is a
reboot.  Unacceptable! :-)

This has happened to me on every 7.2 machine I've tried it on.  Can
someone tell me what I'm missing?
