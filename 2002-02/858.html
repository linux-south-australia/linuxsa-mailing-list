Daryl Pilkington wrote:

>Thanks Glen for the detailed advice.
>Never thought about the limited write issue.
>Suppose a RAM drive will have to be used.
>
>This is for a production industrial application, so cost isn't the only
>factor.
>Wondering if I can get solid-state memory devices that don't have the
>write issue.
>

This is not as big an issue as it sounds. Good drivers these days 
implement wear leveling, and the issue is erase cycles, not really write 
cycles, it seems.

So, what you do is erase a block, which is something like 16 or more kB. 
Then you write to each sector in turn. With wear leveling, the number of 
write cylcles can be quite high even if the device only supports 250,000 
erase cycles.

Check out wear-leveling. Also, work our how many times per hour or day 
you are going to write to the device, and then figure out how long the 
memory will last.
