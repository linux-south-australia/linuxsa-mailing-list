Re: Flash drives

On Tue, 26 Feb 2002, Richard Sharpe wrote:

> 
> This is not as big an issue as it sounds. Good drivers these days 
> implement wear leveling, and the issue is erase cycles, not really write 
> cycles, it seems.

Sorry, I meant that it is a consideration, not a show-stopper.

It's really only continual writing that presents a threat, but equally it
is a difficult thing to avoid (eg: remembering to set noatime on mount).  
In an embedded application I'd think about partitioning the 'disk' into
two, having a read-only root partition and a R/W partition mounted on
/data or somewhere else where I can explicitly control writes; and having
a RAM fs for /var/log /tmp /usr/tmp and so on.

Depending upon the maintenance scenarios I'd also think about running
Linux using a VFAT filesystem.
