It only resolves there addresses; in some cases not providing a FQDN can 
  cause an address not to resolve. And not using FQDN's can really land 
you in the mud. Consider:

/etc/resolv.conf
search csiro.com.au, david.net.au
(ip addresses)

Now let's say that I have a machine called:

csiro.david.net.au

And that if it sees ANY mail traffic it sends a message to the USA to 
start world war III. Now, let's say that the csiro are running a 
mailserver on:

csiro.mail.csiro.com.au

Now, if I were to type:

telnet csiro 25
ehlo david.david.net.au
^D

I very well might start world war III -- oops I forgot, it's already started, it's just called the War against terrorism.
