I can show the fix for the problem of starting on the lowest resolution 
selected.  Edit the /etc/X11/XF86Config file to reflect the following 
example (I don't personally use mutliple resolutions anymore) :-

# The 16-color VGA server
 
Section "Screen"
    Driver      "vga16"
    Device      "Generic VGA"
    Monitor     "My Monitor"
    Subsection "Display"

# The "Mode" line defines the cycle of what resolution(s) are displayed,
# the default display is the first one encountered.
#        Modes       "640x480" "800x600" "1024x768" #Original Order
        Modes       "800x600" "1024x768" " 640x480" #Preferred Order
        ViewPort    0 0
    EndSubsection
EndSection                                                                  

As for the Virtual desktop size, I seem to remember an entry just below the 
Mode line (perhaps Desktop or Virtual) which also had something like 
"800x600", just change that value to the required dimensions. I don't 
remember experimenting with multiple dimensions in hopes of getting it to 
follow resolution.
