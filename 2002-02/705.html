Missed the conference on DHCP. But downloaded the info from the Linux SA
web-site.

Essentially, I want to set up DHCP on my Linux box (Mandrake 8.0) as a DHCP
server. My goal?

To play GOLF with a friend in Melbourne routing through both our linux
boxes. The theory is this, we both have windows machines, (mine win2k, his
not sure) with our Windows LINKS golf, and both want these routing through
our linux boxes. Theoretically, we ought to be able to achieve a link-up
over direct modem together with the game thinking it is on a LAN. (true I
guess), but in reality, this IP info is  sent received via DHCP to the linux
machines which will talk together (as gateways) to one another, routing the
info back and forth to the two windows machines.

I gather I need to set up dhcp.conf. My lan card on linux box set with IP
all ok. PPP internet all ok.

I can get the win2k to gather info from dhcp, but setting up dhcp.conf have
no idea. Also, I assume I need to specify somewhere some routing info to
direct outward info from the gateway (linux box) through the modem. I don't
know how to do this either.

I do possess the rudimentary knowledge of TCP/IP on simple networks and how
it relates to each machine on that network (inc DNS of which I have set up
for my lan card on the linux box all ok), so with any help anyone might give
me, I ought to be able to configure .conf files for the DHCP<-->win2k
relationship.... 

The configuration would look something like this:
MELBOURNE:
ADELAIDE: 
win? <--> DHCP linux server-->	MODEM <--> MODEM		<--linux
server  DHCP <-->win2k
