Paul,

Stick with the linux options, especially those that *may* be M$ portable.
There is a program for Linux called Xastir which is an amateur radio
tracking program which would have the basics of what you are after.   The
graphical front end to it was written in LessTif (Motif derivative) which
quite frankly sucks.   It is incredibly slow, even on a celleron 600MHz
machine.

Pick a programming language that has fast graphic capabilities, keep away
from heavily interpreted languages you just wont get the graphical response
you are after.   I have some good references for C++ libs for decoding NMEA
0183 sentences, GUI stuff unfortuately is not my field.

As an aside I have been using the program that you mention aswell for the
National RDF (Radio Direction Finding) championships held in Mt Gambier,
for the last three years.   Would definitely like to see an alternative as
I would like to add some features which would help me with my RDF tracking,
ie Automatic bearing updates from my Doppler, manual input bearings etc...
