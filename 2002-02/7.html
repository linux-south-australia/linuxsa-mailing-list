Re: ext3 vs. reiserfs

On 2002.01.31 15:43 jason pernito wrote:

> any idea on the advantage of using ext3 over reiserfs or vice versa

It depends up your application.
The pros and cons are

Reliability: ext2; then reiserfs, ext3; then xfs, jfs.

Speed of recovery: ext3, reiserfs, xfs, jfs; and a long way back, ext2

Ease of backup and restore: ext2, xfs; ext3 and jfs require additional 
tools;
    reiserfs has no dump/restore.

Vendor support: ext2; then ext3; then reiserfs; then xfs, jfs.  This will
    probably change to: "ext2, ext3, reiserfs, xfs; then jfs" at the end 
of 2002.

Easy upgrade path from ext2: ext3

Good small file performance (web proxy, etc): reiserfs

Good large file performance: xfs 
Compatibility with IRIX: xfs

Compatibility with AIX: jfs

Support for heirarchical storage: xfs, jfs

Boot loader support: vfat, ext2, ext3; then reiserfs; then xfs, jfs

If you run a single disk machine for your own use then
I'd stick with ext2 and move to ext3 later.  After all,
you don't have that machine backed up, right :-)  And if
the fs has a bug you're stuffed.

If you are using a server there is a lot to be said for
running the system on ext3 and using the most appropiate
filesystem on the partition with the application data.

Be prepared to change the filesystem later, as xfs will
probably end up being the journalling filesystem of choice.

Note that there's no stats on the interaction between
the newer filesystems and RAID.  Filesystem performance
under RAID can vary much more radically than filesystem
performance to a single disk.  That lack of stats also
tells you something :-)
