Does GNU "cp -a" do any error checking as it copies?

Hi,

I have been looking on the web for information about whether the GNU
version of cp has any safeguards protecting the integrity of its copy.
I haven't been able to find anything yet.  Does anyone know about this?

You would think that it would be fairly easy for something like cp to
do integrity of slabs of data as it does its copying.  Ie, it looks at
a bit of file to be copied, calculates a checksum for it, copies it,
reads it back from disk and calculates the checksum again to see if it
has been transfered properly.  Or something like this.  Does anyone
know if cp does this?  What about cpio?

Thanks,
