RE: Samba PDC Problem

Win NT/2K requires machine accounts to be set up in the PDC prior to
connecting to the domain.  I don't know the low-level details (if someone
does and I've got it wrong, please feel free to correct me), but it goes
something like this--the administrator of the windos domain sets up a name
in the PDC, the new machine registers in the domain with that particular
name at which point the PDC somehow links that machine name with the MAC
address of the NIC.  This is a security feature so someone can't set up an
"imposter" of a fileserver--that name only works with that NIC.  Win 98 just
broadcasts a name saying "I'm here, feel free to connect to me" but NT/2K is
a bit more secure about that.  I'm sorry to say I don't know how to create
an account in Samba (I've never used Samba as a PDC) but hopefully this will
point you in the right direction--try to find a way to register the Win 2K
machine's name in the Samba PDC prior to registering the box with the domain
and I think you'll be all set.  Good luck.
