This was all worked out back in about Feb of this year, and is discussed in
detail in Special Edition, Using Samba. It is also documented in the 2.0.7
or later files in the docs directory.

Samba 2.0.4 and 2.0.5 were wrong in their handling of logon home and logon
path. This is because Windows 9X uses the LAN Manager APIs for getting the
information you want, and the 2.0.4 and 2.0.5 functionality, while it made
things easy for profiles, broke the 'net use /home' statment.

The best you can do with profiles under 2.0.7 and above is to use logon
home (logon path is not used at all for Windows 9X), and to set it to
things like:

   logon home = \\%L\%U\.profile

>Why the f?#k is Samba being a pig about the profile directory?  I will

It's not. This is the way things work under Windows 9X clients. Windows NT
allows you to use logon path and put profiles in a separate share.
