* 	Ok, scripts start with an interpreter line, which is something like:
*
* 	#!/path/to/interpreter
*
* 	If the interpreter (usually /bin/sh) does not exist, you'll get
* 	the above message. *Also*, and this is tricky, if the script has
* 	DOS newlines in it (ie you see lots of '^M's when you edit it in
* 	vi), then the kernel may be looking for the interpreter
* 	/bin/sh^M, not /bin/sh.
*
* 	So:
*
* 	tr -d "\r" < axinstall > o
* 	mv o axinstall
* 	chmod 755 axinstall
* 	./axinstall
*
* 	Try that...
