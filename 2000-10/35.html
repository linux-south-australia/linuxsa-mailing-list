Disclaimer: I like Perl.

> 1. Have a play with Perl!

Perl is not a programming language, it's a pattern matching tool. It
also allows too much freedom for a novice user IMO, you must first learn
dicipline in order to be free....

#!/usr/bin/perl
$myvariable = "Dog";
print $myvariabel;

Does what it's told, but not what you want. Maybe they will get into the
habit of using -w, but maybe not.

My own path was Basic, HTML (no, it's not a programming language, but
it's useful for a newbie to get their heads around the "what I type here
won't be displayed as I type it, but as the renderer interprets my code"
concept), Ada, C, Java and Perl.

I didn't find Java easy, in fact, I like C better because you do things
by combining a simple FEW tools you know well, rather than the Java way
of having to learn a gazillion object methods.

NOBODY should start with Java, because it would lead to the world
willBeTalkingLikeThis. there_are_much_better_ways :)
