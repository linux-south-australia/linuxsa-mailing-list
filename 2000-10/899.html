RE: Script behaving badly

I don't know if this script has been anywhere near a windows box, but if it
has, it could be a carriage return/line feed issue.  Sometimes the ^M symbol
doesn't visibly appear, but they might be there (vi will usually say
something like [dos] at the bottom of the screen if this is the case) and
bash doesn't know what to do with them, so it won't execute the script.  Try
running dos2unix or some other crlf converting prog over it and see if that
does the trick.
