Unfortunately, the gear is getting too cheap so the field is filling
with "value" suppliers for both hubs and transceivers.  One of the
sources of this "value" is less than rigorous testing.

The main problem with an ordinary hub and using two fiber AUI is that
the form factor is usually 24*10baseT + 1*AUI.  So you'll either
need to go for a chassis-based switch with say, 2*12*100/10BaseTX +
8*10BaseF wihich will incur a significant price rise.  Or have a
look for a 24*10BaseT + 2*slot.

I suggest you have a look at the HP switch range.  I recall a
24*10/100BaseTX + 1slot.  The slot will take a 4*100BaseFX card.

If you need to be cheap and nasty, then go with 24*10BaseT and buy
two 10BaseT/10BaseF converters.  Be warned that these have very low
reliability.

I strongly suggest you try and use a switch rather than a hub.
The difference between 3Mbps and 100Mbps to a file server is
quiet noticable, especially if the file server has enough RAM
for a decent file cache.
