Re: C library calls on Linux

> Where can I find a listing of libraries, and what header files require I
> link them at compile time?
 
One of the benefits of the C language is it's minimalist approach.
There are on 22 reserved words (I believe).  Additional
functions/subroutines are made available from libraries at linking
time (after you source has been compiled into object code) or
dynamically linked in at runtime.
 
In order to make life a little easier, there is the standard C library
which is compiled in by default (libc).  All other libraries need to
be specified on the command line to the compiler (-l option, eg -lm
links in the libm library).

System libraries can usually be found in /usr/lib,
eg. /usr/lib/libc.a
and you can get an idea about what functions they provide by listing
their symbols with 'nm /usr/lib/libc.a' (for example).
 
The man pages are the best source of information.. 'man pow' gives you
the infomation your asking about.
 
In order to findout whether a function exists to do what you're after,
some 'man -k topic' (-k is for keyword) searching is quite helpful,
as well as the local linuxsa mailing list, a C book or a C guru. 
(eg. Try and find the 'select' command without using 'select').
