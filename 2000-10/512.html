To bad, you are missing out on superb rcs support under emacs ;-):
  "C-x vi" initialise version control,
  "C-x C-q" toggle check in/out status of file,
  "C-x v=" show changes since last check in,
  "C-x vu" undo changes since last check in,
  "C-x vl" show rcs log for file,
  "C-x vc" delete last checked in version from repository,
  "C-x vs" recursively snapshot directory,
  "M-x vc-version-diff"
... etc


> How do I set up rcs or my environment so that I create a central store for
> the rcs files? At the moment, they are stored in the same directory, or
> the RCS directory in that directory, if the RCS directory exists. That's
> ok, but I have to create a RCS/ in any directory I want to use rcs in. It
> would be neat to have them all in one place - I could tar the whole lot up
> and keep them on another machine for backup.

For centralizing the store, CVS runs atop of RCS.  It might be better
to look for this functionality there.  But what is wrong with leaving
the RCS files in place and doing something like

	cd /
	find . -name \*,v \
	    | tar --create --gzip --file=configs.tgz --files-from=-
