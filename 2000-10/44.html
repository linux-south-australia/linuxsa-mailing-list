Re: C

David Lawrence wrote:

> Perl is not a programming language, it's a pattern matching tool. It
> also allows too much freedom for a novice user IMO, you must first learn
> dicipline in order to be free....
> 
> #!/usr/bin/perl
> $myvariable = "Dog";
> print $myvariabel;
> 
> Does what it's told, but not what you want. Maybe they will get into the
> habit of using -w, but maybe not.

If you want your perl to be more like C, try adding "use strict;" to the
first few lines. This will require that all variables and functions are
declared, and the script won't run unless everything is done correctly. 

This is something that should be used on any perl script used in a
production environment, along with -w.
