Re: telnet slowness

Hi

Telnet & ssh should be the same speed, ssh would only be slower if one or
both of the machines very old and very overloaded.

You could try compression under ssh. This would only be helpfully if the
programs you are running on remote end are very screen intensive.

Do some ping test.

I have a machine in Melbourne which is connected via Telstra Direct (I'm
also connected this way) the ping time is about 330ms which work out about
right.
Modem generally have a ping time to the other end of about 120 to 180ms so
using a modem at each end with the Internet between them should work out
between 240 to 400ms pings



Hope this is of some help

Mike O'Connor
