RE: RedHat 6.2 works OK with Linux 2.4.0-test9

Reginald...

The "test9" in "2.4.0-test9" means that this kernel is not yet ready for
mass consumption. It's close, but you should expect problems. It's there for
people who are experienced with building kernels and are interested in
debugging broken kernels, so that they can try it, break it, and help fix
it. A common saying regarding development (2.<odd number>.x), -pre and -test
kernels is that if you don't already know how to get it, compile and install
it without help, then it's not for you. This is because those kernels are
still in the development and testing stages, and pretty much anything could
go wrong. To build, install and run one of these kernels, you really need to
be able to tell when something is actually broken, and when you are doing
something wrong, because often, something actually is broken. Try compiling
a stable kernel (2.<even number>.x) first, and see how you go. I think the
currest stable kernel is 2.2.17, but don't quote me on that.

As it turns out, your main problem here was capitalisation (zImage, not
zimage), and if you correct that, you will probably have no other problems
(it is, after all, a test9, and should theoretically be close to stable).
However, it's recommended that unless you know how to recover when it all
goes pear-shaped, and nothing works, that you keep away from dev, pre and
test kernels.
