Re: linux for mobile phones - supported?

On Fri, Oct 06, 2000 at 10:42:48AM +0930, W. Sierke wrote:
> 
> As John indicated, the interface to the Nokia phones is proprietary.
> Electrically, an adapter (RS232 to 3.3V level translator) is required, which
> can either be purchased or cobbled together quite easily (I used the
> connector from a 'hands-free' kit).
> 
> The Nokia software for Windows creates a virtual COM port which does
> implement a Hayes-style interface, and performs the necessary translation to
> the Nokia protocol over the physical COM port, however the details of this
> appear to be an on-going problem to decipher for the gnokii team. As you
> will see from the gnokii web-site, Nokia have refused to provide the details
> of their protcol, which is a great shame.

This makes it sound like the situation for Garmin GPS units, and
some kinds of webcams, which I have found do work with linux,
although in a not very comfortable way.
But if I'm in the bush and I need e-mail, I might be happy
to have any kind of connection at all.

The linux Garmin GPS software was written by observing windows
software generating commands and reverse engineering it.
Clearly the gnokii people have done the same thing.
