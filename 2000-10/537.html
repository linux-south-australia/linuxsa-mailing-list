Re: Does GNU cp -a do any error checking as it copies?

"Russell, Richard (DEH)" wrote:

> what if i copy to disk, it goes to the cache correctly, but gets written to
> disk incorrectly. Running any hash will tell you that the cache is correct,
> but when the cache is flushed, the data will be wrong.

If you're going to the trouble of doing MD5 hashes et al, you could also
use the API's present to write directly to disk instead of via the
cache. This may not help with some hardware caching, although I think
there might be some scsi feature to allow direct access. Apparently it's
what a decent RDBMS does when you issue a COMMIT command, again for data
integrity reasons.
