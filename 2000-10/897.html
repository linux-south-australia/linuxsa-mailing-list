Re: Script behaving badly

At 11:08 AM 10/27/00 +0930, Ian Hogben wrote:
>Hello all:
>
>I am trying to get an Axis print server running on my linux box, and I've
>found an installation script that allegedly supports linux. When running it,
>though, I get the following:
>
>$ ls -al ./axinstall
>-rwxr-xr-x    1 ian      ian        108217 Oct 27 07:49 ./axinstall
>
>$ ./axinstall
>bash: ./axinstall: No such file or directory

Look inside at the first line: It should say something like:

   #!/path/to/exe

If it is a perl script, is may say #!/usr/local/bin/perl -Tw yadda

Fix up the path. This is what you would get in the case that the path is
wrong.

>The file exists. So I'm guessing that the bloody thing is referencing

>message is referring to that. This seems like an awfully cryptic error
>message, and I wonder if there is anything out there that can help me narrow
>down the offender. Is there a verbose option in a bash script? Some way to
>step through it line by line? It's over 100K, so I can't exactly scan
>through it to find the offending reference.
