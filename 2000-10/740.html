Re: Sound card conf.

G'day,

    Your errors look as if you are activating the card before the kernel can
access it.

First, rename /etc/isapnp.conf to /etc/isapnp.conf.old  then reboot the
system and start linux again.  This will remove any misconfiguration done
by isapnp.conf

Then run sndconfig with the following command:

      sndconfig --noprobe

Give it the same settings you had under windows (if you run Windows)

   io=0x220, IRQ=7, DMA=0, DMA16=1, mpu=0x300, opl3=0x288

(These are typical sound blaster settings, but yours may be different)

The configuration should leave the following in /etc/conf.modules:

   alias sound sb
   pre-install sound /sbin/insmod sound dmabuf=1
   alias midi op13
   options op13 io=Ox388
   options sb io=Ox220 irq=7 dma=0 mpu_io=Ox300

This should work if the card is already being activated by the kernel
before isapnp.conf can access it.

Hope this helps,

Cheers,   Rick
