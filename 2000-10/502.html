Re: SendMail and Rebuilding Db's

Daryl Tester wrote:

> Sendmail doesn't know how to build these files, unlike the /etc/aliases
> database, and uses them in a "read-only" fashion.  So you're looking at
> an external cause (most likely process).  I know RedHat ship a Makefile
> that lives in the /etc/mail/ directory where these db's normally live
> for rebuilding them; perhaps that's being run nightly?

Close - there is a Makefile as you say, but as far as I know it
is not initiated nightly.  The culprit is more likely to be the
following from /etc/rc.d/init.d/sendmail:

    for i in virtusertable access domaintable mailertable ; do
      if [ -f /etc/mail/$i ] ; then
        makemap hash /etc/mail/$i < /etc/mail/$i
      fi
    done

Have a look in last and see if the timestamps correspond to the
most recent reboot.

It's caught me too - especially when you add some new maps.  I
do hope RedHat will not develop the Billy habit of "helping"
too much!  I suppose it's a balancing act, with so many RedHat
users being relatively inexperienced.

Cheers, Dieter.
