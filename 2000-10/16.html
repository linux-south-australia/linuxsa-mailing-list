Re: Question Re ipmasqadm/ipchains

David Lloyd wrote:

> If I know exactly what IP address and what port some packets will be
> coming from, is there any way I can transparently forward them through a
> Linux 2.2.17 based firewall to an internal, private 192.168.0.0/24 based
> machine? 

ipmasqadm.  From my old firewall config:

# Flush masquerading rulesets.
/usr/sbin/ipmasqadm portfw -f

[...]

##  Forward Netstorm traffic through to nipnyep
##  (6799 definitely, not sure about 6800).
/usr/sbin/ipmasqadm portfw -a -P tcp -L 203.24.77.208 6800 -R nipnyep 6800
/sbin/ipchains -A input -i ppp+ -p tcp --dport 6800 --syn -j ACCEPT -l
/usr/sbin/ipmasqadm portfw -a -P tcp -L 203.24.77.208 6799 -R nipnyep 6799
/sbin/ipchains -A input -i ppp+ -p tcp --dport 6799 --syn -j ACCEPT -l

This was working across a NAT'd setup.
