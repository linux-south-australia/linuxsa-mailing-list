.jsp pages are dymanically compiled into Servlets by the JSP Engine, and
then served by a Servlet engine. The advantage with them is that they are so
much easier to read and maintain than Servlet classes. Servlets tend to have
hundreds of lines like 'System.out.println("_some HTML text_");', and this
gets ugly to say the least, when you have lots of these classes. JSPs turn
it around, so you have a HTML page with some embedded Java program text, and
this is far more readable.

>Roughly severlets are the java euqivilant to cgi bin programs.

Sort of. You can write a CGI program in Java, but it will run painfully slow
(much MUCH slower than Servlets or JSPs, which are actually relatively fast)
due to the start-up speed. Servlets are in-place, and use a slightly
different method of getting the parameters sent to them.
