Signal 11 is a Seg fault.

man -S7 signal
...
       Signal     Value     Action   Comment

-------------------------------------------------------------------------
       SIGHUP        1        A      Hangup detected on controlling
terminal
                                     or death of controlling process
       SIGINT        2        A      Interrupt from keyboard
       SIGQUIT       3        A      Quit from keyboard
       SIGILL        4        A      Illegal Instruction
       SIGABRT       6        C      Abort signal from abort(3)
       SIGFPE        8        C      Floating point exception
       SIGKILL       9       AEF     Kill signal
       SIGSEGV      11        C      Invalid memory reference
...


I think these are commonly programmer problems, eg attemtping to
reference a null pointer or memory space you don't own.

Hardware might cause it, but I would think that would happen only when the
hardware is really faulty and probably won;t run any other programs/OSs or
the programmer has made some assumption about the presence of some
hardware which is not there - ie bad programming.

I think hardware is a fairly _uncommon_ cause for segfaults (signal 11s)
