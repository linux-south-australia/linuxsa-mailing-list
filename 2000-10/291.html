Daryl Pilkington <darylp@senet.com.au> wrote:
> 
> Is it possible because telnetd is running on the router its stopping
> telnet to route?

A router routes IP packets - a layer 3 operation.  telnetd is an
application that listens to TCP port 23 on the local interface(s)
of the machine - even if it is acting as a router, that won't stop
it from routing IP packets that contain TCP/23 telnet packets.

If you happen to be running some form of masquarading (eg ipmasq/nat)
then the above may not be valid, but you havn't mentioned it so I'm
not considering it :-)
