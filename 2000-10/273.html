Re: Just a Quickie

> ...in particular "2>&1".

tells the shell to redirect stand error (2) to the same place as
standard output (1).  If you just use "> my_log.txt" standard out will
go to the file, but standard error will still go the console.

It is generally better to see standard error on the console, but of
course background commands or scheduled commands would rather log
standard error.

Another example might be

	make all 2>&1 | less
