> If you're using the SuSE 6.4 (and possibly other SuSE)
> yast software to configure IP addresses of ethernet cards,
> you might like to know something nasty which I found with it.
> 
> Suppose you have two ethernet cards -
> 
> eth0 connects to the real Internet
> eth1 connects to a private LAN
> 
> Suppose you set up route.conf so that
> the default IP gateway is through eth0.
> 
> Now suppose you change an IP address for
> eth1. It seems like the rule is that YaST will
> use the gateway node on eth1 for the default because
> it is the last interface that you configured.
> 
> This happened to me a couple of times, with the result
> that when I rebooted the computer remotely,
> YAST had changed /etc/route.conf so that the machine
> thought the wide open Internet was through eth1.
> Therefore it replied to all of my packets by sending
> them to eth1, even though they arrived on eth0.
this is easy fixed by putting them in /etc/rc.conf
