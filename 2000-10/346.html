On Wed, Oct 11, 2000 at 01:39:43PM +0930, James.Wilson@compsm.com.au wrote:
> 
> On the topic of RH7 anyone have any problems with compiling 2.2.17 on RH7?

The gcc that comes with RH 7 is effectively a CVS snapshot.  RH named it
2.96 (which doesn't actually exist).

The kernel does 'interesting' things which aren't supported properly by
2.95 and later.

*BUT*, RH have supplied kgcc (which is rally gcc 2.91) which does a 
flawless compile.

So... To do a kernel compile you can do a very hopeful compile by doing
a 

  make CC=kgcc dep && make CC=kgcc && make CC=kgcc modules && make CC=kgcc install && make CC=kgcc modules_install

Which simply informs make to use kgcc (2.91) rather than gcc (2.96).  

Using 2.96, I would have boot failures with a new kernel, with 2.91 it works
flawlessly.
