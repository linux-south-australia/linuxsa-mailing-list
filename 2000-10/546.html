Re: Running "X" in multiusermode...

On Mon, Oct 16, 2000 at 12:02:57PM +0200, H.Schumacher wrote:

 > Oh, i need help. I want to set up a RED HAT BOX 6.2 as X-Server. I want to 
 > get a graphical login on my X-Terminals. Do somebody know , if there are 
 > HOWTO's which help me to setup up my RH Box correctly ? Or, do somebody 
 > know, what i have to configure.

There really isn't all that much to configure.  Just make sure that
the "DisplayManager.requestPort: 0" line is commented-out from
/etc/X11/xdm/xdm-config, make sure xdm is running (hint:  if you get a 
graphical login window when you boot up, xdm is running), and that's
about it on the file server side.  Depending on which release of 
XFree86 you have, it might actually be set up to do what you want in
its "out of the box" configuration.

On the X terminals, set them up to send XDMCP queries to your redhat
box.  If the X terminals are actually UNIX boxes in disguise, you can
do this by running:

    X -query redhatbox-hostname

If they're real honest-to-god X terminals, they'll have a setup menu
option somewhere to configure XDMCP, so enter the hostname or IP address
of your redhat box in the appropriate place there.

Trivia:  In that scenario, the X servers are running on the X terminals,
not the redhat box.  See the list archives for several explanations of 
why this is so, and remember that a "server" is usually a box which has
a hardware resource which it's prepared to share.
