Squid has config options to limit the access to it from a variety of IP
ranges or domains. I suggest reading the very informative Squid web site
documentation. I've found it to be quite complete and informative.

> Option 1:
> 
> Not knowing how exactly how squid cache's work (regarding expiration)
> could it be possible that there is material that no longer exists on
> any particular website, but that migrates between caches?

If you are peering with other Caches, I've seen some confusion occur on
occasions which means that squid thinks another cache holds a document,
and will try and serve it, or another cache does hold a document and
serves it instead of the most recent copy. It all depends upon the config
of the caches, and what the expire time is set to. In the case of a single
cache this shouldn't happen.
