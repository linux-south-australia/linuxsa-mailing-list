RE: postfix to listen on a different port to 25

Got it!

I edited /etc/services to include "smtp2" service description to port 5000.
So I've got the original smtp 25/tcp line as well as an smtp2 5000/tcp. Then
I just edit the /etc/postfix/master.cf file and change the service starting
with "smtp" to "smtp2".

Listens fine on 5000, sends fine on 25. Love it. ;-]

Thanks to all for the info!

PS: for those that care, I've got a firewall rule in place that blocks
access to 5000 from ppp0, so it should prevent people bypassing my virus
scanner.

* Postfix looks like it might rely on the smtpd entry in /etc/services to

* also use that
