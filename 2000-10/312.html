Re: fsck stuff (longish)

Some types of file system damage cannot be reliably corrected.  For
example, if you  have duplicate blocks, that is the same block in two
files: you need to remove those blocks from one file, but which one?

I've seen fsck unlink /bin and then relink all the files that were in it
to /lost+found.  Trouble is, you end up with /bin/sh being renamed to
/lost+found/005612 which makes it shell scripts a bit tricky.
