Domain Logins/Login Path/Samba

Hi all..

I am at the end of my rope trying to diagnose this problem...

I have Samba 2.0.6 set up here.  I want to use roaming profiles with Win98 boxes on the network here.  With samba 2.0.5 it was as easy as setting "domain logons = yes" and "logon path = \\yoda\profiles\%u" and creating a share called profiles that pointed
to the profile directory.

Under 2.0.6 I have done this and Samba INSISTS on usong ~user for the the profile directory; no matter that I have specified EXACTLY where to store profiles.  The profiles share is configured correctly and is browsable.  I can log into Windows and the
logon script executes and I can see all the shares mounted.  I can access H: which is my home dir and manipulate the files there.  I can even browse to \\yoda\profiles\adam and see all the files threre.  I cannot browse to other's profiles or home dirs.

Why the f?#k is Samba being a pig about the profile directory?  I will attach my config file so that you can possibly see where I have gone wrong.
