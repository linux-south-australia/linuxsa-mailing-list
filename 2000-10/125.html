> You won't have any glitches having two kernels on the system provided
> you follow the instructions on how to install them (in
> /usr/src/kernel-source/README). You may fall foul of programs that think
> they're talking to a 2.2.X series kernel when you're running under
> 2.4.0. Because it's a major number release it is allowed to break some
> things. For example, ipchains for a 2.2.X series won't work under 2.4.0
> without getting the very latest source (if it exists)...

Make sure you DONT use the new devfs system otherwise things wont work too
well.
Also, if your using ipchains with 2.2.x youll need to get the iptables targz
to do similar things under 2.4.0
You may also have problems with pppd and may need to get the new version of
that.
There are lots of other utils that also need be upgraded to use 2.4.0... look
in the "src/linux-2.4.0/Documentation/Changes" file for details.
All of the new versions of the utils should be fine back in 2.2.x

BTW... 2.4.0-test1 is very old now... i would getting the latest test version
which is test9.
Ive been running the 2.4.0 kernel for quite a while now (since test2) with no
problems, and havnt yet had a need to go back to 2.2.16.
