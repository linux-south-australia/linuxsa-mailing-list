Re: Help on Samba Clause

With NT you can log in under any name.  NT will request a username and password if
you are logged on to NT as an administrator.

Make sure you have an account on the RH box.
If the password option in the smb.conf is set to NO ie no encryption you must
change the registry file on NT to use plain text passwords.
You can download a registry file from the samba home page that will do it for you.
Just double click the downloaded file and it will alter the
registry file to EnablePlaintextPasswords.

Restart NT
Restart samba

Browse for a share, and try to access the share.
Type in an appropriate name and password and all should be well.

Your troubles sound like an encryption problem, lets get that out of the way first
and let me know how you go.
Any more problems and we can go from there.


Good luck


Steve
