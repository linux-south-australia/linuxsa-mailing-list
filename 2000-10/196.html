RE: JSP Server

>If you want dynamically generated web pages from a database
>then what's wrong with using PHP?  It's free, fast,
>reasonably simple to use, and you can link it to almost
>every database server known to man/woman and microsoft
>(almost).

Because PHP is a great tool for small projects where you want the page
engine (Apache with mod_php) to be doing all the processing and using a
database. JSP will let you do anything Java can do. You can have the JSP
page talking CORBA (or RMI) to various other programs on various other
servers (written in various other languages on various othre platforms), as
well as talk to a database. It's an order of magnitude heavier duty.
However, when PHP really will do the job, and you don't see a future need
for the heavier duty stuff, then by all means, use it. It's similar to the
difference between a Suziki Vitara and a LandCuiser. Different tools for
different jobs.
