Sun 3/60 booting via tftp

So I have this Sun 3/60 here (it lives! well sorta)
I'd  like  to boot it... but there seems to be a problem with the boot
tape, so I figure, I'll boot it off my Linux server.

After  determining that it needed an entry in the RARP table to get an
IP, and then that it wanted to use tftp to retrieve the boot file...
I  tracked down a NetBSD boot file and placed it in /tftpboot (as this
is where the tftp server is supposedly configured to look).

I  then  created  a symlink to netboot (the boot file) called C0A80003
(as  this  is the IP addr of the sun box in hex, as recommended by the
NetBSD install FAQ).

Of  course, it doesn't work. For some reason the server (eric) replies
with an icmp: error (see below).

Obviously I haven't configured something quite right... any ideas?

(possibly  relevant  info:   eric  is  multihomed,  it  has  two  eth
interfaces, the second one (eth1) is the 192.168.0 network, ie the one
the sun lives on.  Dunno if this would have a bearing on tftpd or not)
