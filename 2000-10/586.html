there are a few options:

sudo -- the most powerful of these options... can specify commands,
arguments, users who can run hem, and the users they can run them as...
Quite secure, if done properly (letting all users do all commands as all
users is a big nono :)

adding a user with uid of zero -- this will effectively mean that you can go
`su - <newuser>` and enter that password, and you will then _be_ root...

add a setuid shell somehwere...

there are other solutions also, but I can't think right now :)

have a think about the security implications of these, and make a choice
.... It really depends on what you wnat it for, what's at risk, and what you
can afford to do  (timewise)...
