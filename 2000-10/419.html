Safe way to move/copy between different partitions

I shall shortly be moving around bits of my filesystem from one
partition to another and then later back again.  Some of this will
occur over NFS.  I am worried that corruption might occur in the
process.  Are there any ways to guard against this.

The ways I know about are:

1. cp -a    I've used this before and it works, but is it safe?

2. tar through a pipe.  Again, is it safe?

3. tar to a compressed file on the same file system.  Then do a md5sum
on it.  Move around the compressed tar file and check that the md5sum
doesn't change.  This method relies on the original taring and
compression not having corruption.  Is this a reasonable assumption?
Also, it assumes you have room on the file system to store the
compressed tar file.

4. cpio.  I don't know much about this one, and I don't know its
advantages.

Any thoughts or suggestions?

Thanks,

Mark.
