Something wrong with RH7.0. They released 2.4 kernel along with it which was
unstable for that moment. And RH announced shipping of 2.4 before Linus
actually finished it.

Anyway if you use RH then wait for *.2 version before you will upgrade your
vital system. Because *.2 versions are stable and Auto-RPM works smoothly
then. At least it is so with RH 4.2, 5.2, 6.2.

Also Auto-RPM can be tuned well so it will tell you that new update arrived
but cannot be installed due dependencies - then DIY. If no dependencies will
be broken then it will silently install new package. And probably will put
kernel* onto exception list... :)
