Re: MySQL DB problem....

>> Im trying to copy a database from one mysql server over to another (both
are
>> 3.23.27-beta).
>> The DB is the old style non-iasm db (not sure what its called).

>Postgress has a dump database option, which converts the entire database
>to SQL commands and an import that restores it. I'd think that would be a
>safer way to do it if MySQL has the same sort of options.

Yes, it is probably safest to do this with MySQL as well, although moving
whole database files around usually works.

Simply do :
mysqldump -u root dbname > dbdumpfile

then on the other system:
mysqladmin -u root create dbname
mysql -u root < dbdumpfile
