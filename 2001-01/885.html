Not only that, but you can use LDAP with sendmail to do some pretty
powerful mail routing and Samba/Samba-TNG are both planning on having
the option of using an LDAP backend. This means that you can have all of
your user account information in one place. For example, this is a test
entry in one of our LDAP directories:

dn: uid=testuser,ou=Staff,ou=People,o=Xavier
objectclass: top			#Standard Unix account (not shadow) attributes
objectclass: posixAccount
objectclass: account
cn: Test User
loginshell: /bin/false
uidnumber: 502
gidnumber: 504
uid: testuser
homedirectory: /home/testuser
gecos: Test User
userpassword: {crypt}*
objectclass: inetlocalmailrecipient	#Stuff used by sendmail
maillocaladdress: Mr.User@xavier.sa.edu.au
maillocaladdress: Test.User@xavier.sa.edu.au
maillocaladdress: testuser@xavier.sa.edu.au
mailroutingaddress: testuser@mail.xavier.sa.edu.au
mailhost: mail.xavier.sa.edu.au
objectclass: sambaAccount	#Samba TNG PDC account information. It also
uses a couple of attributes from the posixAccount attributes
ntuid: testuser
acctflags: [U            ]
rid: 44F
lmpassword: NO PASSWORDXXXXXXXXXXXXXXXXXXXXX
ntpassword: NO PASSWORDXXXXXXXXXXXXXXXXXXXXX
pwdlastset: 3A36BF90
pwdcanchange: 3A36BDF4
pwdmustchange: FFFFFFFF
logontime: 00000000
logofftime: 00000000
kickofftime: 00000000
grouprid: 23B
smbhome: \\mackillop\testuser$
homedrive: Z:
script: staff.bat
internet: ok		#This stuff is to allow us to turn a persons squid or IMAP
account on and off (using a patched pam_ldap)
allowmail: ok
mail: testuser@xavier.sa.edu.au

There is a fair bit of duplicated data, but it works.
