Well, I'm not sure:

OBJECTIVE
Use Xlib Primitives to Print

METHOD

1) Create window
2) Capture all Xlib calls (i.e. start drawing)
3) Display on screen if visual set
4) Store in internal-format
5) Stop capturing Xlib calls (i.e. stop drawing)
6) Render internal-format into well-formed and valid PostScript

OUTCOME

Does not dispense with PostScript/ghostview but does use Xlib to draw
graphics.

DISCUSSION

Whilst this will take a little implementation - it could be done in a
multitude of ways - it may be worth considering. One way would be to set
an attribute in all Windows' attributes which said `printable = true' or
`printable = false'. Another way would be to make it less dependent on
internal X Windows attr structures and force any `printable' windows to
be the child of a `print frame' window.

The main point is that there is NO reason a set of Xlib primitives
couldn't be converted to PostScript. The only thing missing, in my
opinion, is the framework in which to do it.
