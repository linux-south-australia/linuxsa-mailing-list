Re: Problem with rpm on MD7.2

> Howdy folks,
>
> I have a weird problem with "rpm" on my Mandrake 7.2 system
> that I just can't figure out, so I'm hoping one of you have come
> across it before.
>
> First, I've reinstalled Mandrake 7.2 about 3 times due to various
> reasons, and have *only* had this problem with the latest install.
> The problem is that I get the following error message about 50%
> of the time when I try to install new packages:
>
> root:/tmp> rpm -i php-4.0.3pl1-1mdk.i586.rpm
> warning: /etc/php.ini created as /etc/php.ini.rpmnew
> unpacking of archive failed: cpio: Bad magic
>
> Where is this "cpio: Bad magic" message coming from? This
> happens when trying to install rpms from my MD7.2 CD, or when
> installing stuff I've downloaded from the net. I've tried
> reinstalling the cpio rpm, but didn't change anything.
>
> Any help appreciated.
>
> Thanks,
>
> Dean

I saw such behaviour of RPM before. It's a problem of RPM itself. As I found
for myself it happened due inconsistency of RPM DB. So try to upgrade to
latest RPM (if not done so) and try to say rpm --rebuilddb

May be this can help - chances 50/50.

I also saw such message when installed RPMs onto system with low memory and
when RPM package was broken (I exclude this one in your case).
