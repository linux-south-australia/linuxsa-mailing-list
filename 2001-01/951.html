RE: Strange multiple beeping

I can't say I've ever had it happen to me on a Linux box, but that's not
that uncommon on Win9x machines.  When an application becomes unstable, it
can do funky things with the i/o buffers.  It sounds like a buffer overflow.
As you put it, it would sound like an invalid key was being held down.  In
Windows, this can sometimes be resolved without rebooting if you can manage
to kill the application.  (Not easy since the mouse and/or keyboard wouldn't
be responding...)

I've also seen keyboard and/or mouse lock-ups after static discharge, which
could sometimes(although rarely) cause the same symptom.  In the latter
instance, only a reboot will solve the problem.

I don't know that I'd worry about it unless it happens again a couple more
times in the near future.
