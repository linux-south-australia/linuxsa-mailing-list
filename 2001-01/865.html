Obviously the easy way to fix some of the hassle would be a second hard
drive to house the cache, mainly to cut down the disk activity, having the
cache & the swapfile on the same drive isnt the best scenario.
Also a memory upgrade to at least 128 would help too, but may not be
essential if you do the above.
A 64mb swap is probably a little underdone too, experiment with it a bit,
all depends what size your hard drive is.

Also if you plan to upgrade to RH 6.2, try compiling a new kernel with
Reiserfs support & format your cache partition in Reiserfs.  I know Reiserfs
does have some issues, but for a proxy cache its hardly a high risk if you
keep the partition separate.

A proxy server which I recently set up is running RH 6.2 with a modified
version of Squid 2.2.STABLE5.
Admittedly it has 256mb RAM & a 2 disk SCSI cache (reiserfs), but it
services 30-40 concurrent users with no apparent performance hit.

For the Squid setup I more or less followed this setup:
http://www.devshed.com/Server_Side/Administration/SQUID/print.html

and used the Squid source RPMs from:
http://www.swelltech.com/pengies/joe/

(i didnt use the kernel from here as I built my own, actually I u/g it to
2.2.18 last week)
