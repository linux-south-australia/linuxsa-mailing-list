Re: Cron logs

Martin Stacey spake...
> 
> My cron logs in /var/log aren't turning over. The log file cron is empty,
> while the log file cron.1 is receiving messages from the cron daemon. The
> other cron log files haven't changed in weeks. I am using RH7.

I've had the same problem, but with my /var/log/maillog file.  The problem
was spasmodic, and seems to have gone away now.  It was only the maillog
file that didn't rotate, the others were all ok.

If you want to research it a bit, check out logrotate's configuration
files, in /etc/logrotate.conf and /etc/logrotate.d/* (under RH6.2, that
is), as logrotated is responsible for rotating logs.

Could also be syslogd.

I don't know why it occurs, but it might be nice for you to know there are
others out there with the same issue :)
