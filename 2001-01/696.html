RE: determining IP address of client

On Wed, 17 Jan 2001, Richard Russell wrote:

> > last | grep 'still logged in'
> >
> > This will show info about all users currently logged in. Add more or
> > less grep options to taste.
> 
> I like that. One of those commands that I knew about, but had never used for
> that purpose before :)
> 
> But what about something more general? I often find that I want to find out
> which process (and which user, etc) is listening on a specific socket. EG: I
> start a webserver on tcp port 18080, but it can't bind to that port as it is
> in use. So I try to find out what process is using it, and who is running
> the offending process (so I can `sudo -u <user> kill -1 -1` them :) ...

Absolutely:

[root@box /root]# fuser  -n tcp 22
22/tcp:                467
[root@box /root]# ps ax | grep " 467 "
  467 ?        S      0:00 /usr/sbin/sshd
 1267 pts/1    S      0:00 grep  467 

fuser can also search in the 'file' or 'udp' space, 'file' meaning of
course 'what process has this file open'.
