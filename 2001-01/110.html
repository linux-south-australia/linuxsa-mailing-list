adding options to modules in conf.modules

Thanks to Richard for the pointer to mkinitrd.  Have 2.2.18
working.....almost.

Just have a small issue with a scsi adapter & network card.
Network card is an Intel etherexpress pro, using module eepro100.
Scsi card is an Adaptec 2906 using module aic71xxx
(incidentally, had the same problem with the 2.2.14 kernel, that's part of
the reason I
wanted to u/g to 2.2.18)

cat /proc/interrupts shows both are sharing irq 11

I've had linux working ok with shared irqs before, but the kernel generates
these
errors right after loading the network card (scsi is loaded well before this
of course)
	
	eth0: card reports no resources
	eth0: card reports no RX buffers

I've checked the eepro100.c & aic71xxx.c files from the kernel sources,
neither 
seem to mention options for setting irq.
I tried adding the following line to /etc/conf.modules

	options scsi_hostadapter irq=10

this didn't change the irq, so put eepro100 instead of scsi_hostadapter,
same thing still.

I'm not sure where to go from here, I've tried changing BIOS PnP settings
etc, no change.

Ideas anyone??
