Re: Linux Server Lockup

Hi...

> Hi
>
> I have a Customer with a Linux Box using Samba and Squid.
> It is Red Hat 6.1 with a Kernal Version of 2.2.12-20.
> The box is a Pentium III 550 with 20Gb HDD and 64Mb Ram.
> It is setup for File Sharing Primarly sharing office files and MYOB files.
> The server is currently locking up, as in a total freeze(no keyboard
> response etc).  It seems to be when specific programs are being run.  Eg
> Today when one of the users was restoring a quickbooks file from 3 floppy
> disks to the shared drive on the linux server, it locked up each time the
> user attempted this task.  When the user did the same step to their Cdrive
> it went through OK.
> I'm not sure where to start looking, do I need to do a Kernal Upgrade?  Is
> there problems with 6.1?
> Any Suggestions would be very appreciated.

Since they are hard lockups, i would start by looking at the machines
hardware.
Check for bad ram, motherboard, overheating, IRQ conflicts, BIOS settings,
etc.
After that, then next would prolly have to be the kernel, disabling daemons,
getting newer versions of software, etc.

Good luck....
