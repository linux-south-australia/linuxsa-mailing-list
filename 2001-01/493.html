When inserting rules each ipchains command has 3 parts which are:

  1) where the rules exists.  ie. which chain and where in that chain
  2) what packets match that rule.  eg. input interface is ppp0
  3) what to do if that rule matches eg. REJECT and log

So your problem is that you have not specified 1 and 2.  

Infact, you don't need to specify 2, which will result in all packets
being matched.  So a rule which matches ALL incoming packets and logs
them would be:
	ipchains -A input -j RETURN -l

BTW this will flood you logs so DON'T do it.

> but also I wanted to reject all SYN packets and +++ATH0 and so
> forth,

You can match on SYN packets (-y) but not on the content of a packet
with ipchains.  

... and yes, rejecting all incoming SYN packets destined for your box
is often done to make sure that no TCP connections can initialised
from external machines.
