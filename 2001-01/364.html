> > because it does not  tranpsort anything.
> 
> Sure it does.  As was pointed out before, you do have a small payload with
> ICMP, which Ping uses to match returned packets with the original transmitted

David,

As I pointed out earlier, this argument doesn't fly --
because there can be no ICMP error in reply to an ICMP
packet, then ICMP can't be used to transport anything
with certainty.

ICMP isn't a transport protocol, its a control protocol
for the TCP/IP suite.  There's lots of these little
control protocols, even in the OSI suite.

The OSI protocol model never claimed to be all-encompasing.
Rather it's a tool for thinking about protocols.  And a
successful one too -- for example the farce that is addressing
in DECnet would have been avoided with more attention paid
to the model.
