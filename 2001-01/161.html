> configure: error: installation or configuration problem: C++ compiler
> cannot create executables.
> 
> Can anyone suggest where to look for a solution to this ?

1)

If you're using RedHat 7.0 and RedHat GCC 2.96, try using kgcc instead.
To override the compiler you could try ./configure --help but I think
CC=kgcc ./configure will override it anyway...if I recall correctly the
main changes in RH's compiler were to do with C++ extensions so this
could be your glitch.

2)

Write up a quick proggie:

#include <iostream.h>

int main(int argc, char *argv[]) {

cout << "Hello World!" << endl;
return 0;
}

And compile:

g++ hello.c -o hello

And see whether that works...

3)

Force the compiler options to `-wall' only:

CPPFLAGS='-wall'

And try reconfiguring...

If it works then one of the compiler options it's attempting to use
doesn't work.

4)

The build directories must be writable by the user who is building the
source. Hence, check to make sure that:

* all directories have the write bit set
* the user that the directories have untarred as is sensible (i.e. it
appears as a name, eg. root, rather than a random number)
