Re: Installing Packages

Hi,

> What are the guidelines for installing a package using the rpm command. By
> this I mean;
> 
> * Do I need to place the package in a specific directory?

No, not necessarily. This is a matter of archive phylosophy. It is
probably a good idea to place all updates somewhere, and all new installs
somewhere in a logically organised fashion (Free BSD's prots collection is
nicely organised, you could pollibly browse this for ideas). By default
GNO-RPM looks in /usr/src/RedHat/.....

Traditionally source files went into /usr/local/src, and there are a
variety of pylosophies on structures for file system, including whatever
you currently have in place.

> * Do I need to reboot my server once the package has been installed?

Depends upon what you install. Generally not, you would simply need to
(re)configure the service then (re)start it but for something like a
kernel then a re-boot is mandatory for you to use it. The documentation
included in the package will tell you the (re)start procedure for what you
installed. You may need to re-read the path in any open shells you
have. This will depend upon your shell tho.

Without details of what package(s) you are installing this is dificult to
answer in more detail.
