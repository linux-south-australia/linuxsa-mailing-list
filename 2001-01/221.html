in which case, you may find that sending it a HUP signal will cause it
to re-read its config file. This is what `init' does, (I don't run
xinit)

to send signals, you use (the badly named)

kill

program. 

$> kill -l
will list the signals you can send

$>ps -eaf | grep xinit
get the process number

$> kill -HUP <pid>
or
$> kill -1 <pid>
or
$> killall HUP xinit

the last one illiminates the need to find the process ID, it finds it
for you.

interestingly, killall on other UNIXs can be *very bad*, as it tried to
kill all process owned by the current user, which when you are root, on
a live production system is *VERY VERY BAD* :)
