Unless you want it only to run in certain runlevels which is probably
what you want with services(assumgn that the services use the network).
You probably wnat to have the servcie only run in run levels where netwrok
services are available, generally 3 and/or 5.

In which case:
/etc/rc.d/rc<x>.d/<'S'|'K'>y><scriptname>

Where:
x is the runlevel you wnat teh script to run in
and y is the order you want ti to run in.
S means run this script at next tiem we go to this runlevel.
K means do't runteh script netxt tiem you enter the runlevel.


So if you wanted to run apache (httpd)
you might have a script called
/etc/rc.d/rc3.d/S85httpd

The 85 (ie the <y> component) specifies what order teh scripts are called
in.
85 woudl be called after 35 which woudl eb called after 10 etc.

So you need to make sure that any servcies your service requires will
have a lower number than your new service.

Ie if your service requires dns then you need to make sure teh script that
starts DNS has a lower number.

The S is required as the first character.
Scripts that start with S will be staarted. Scripts taht start with K are
not started.

So later if you decide you dont watn to run apache for the next
reboos or runlevel change, you can change the S85httpd file name to
K85httpd and when you wnat it back all you ahve to do its change it back.
 
I am not sure what happens with scripts not starting with  S or K
then number.

This works for redhat I assume it works for mandrake.
