Security is very strange thing to learn. Commonly it's a set of rules which
you must obey (remove unused programs, keep up to date and so on). But I
never saw them somewhere... So everyone makes such set by themself and
nobody knows how good it is. For eaxample: It's their decision to have gcc
or not (some exploits need gcc on system). But then it takes time to install

inconvinient. So keeping restrictive policy not good for attacker but in the
same time it's not good for yourself.
