username/password @ escaping

An ftp site requires anonymous login with email address as password. A
command line too requires a string of the format
[username[:password]@]server[:port] ... so I need to have something like:

FTP_SERVER = anonymous:richardrussell@mail.com@ftp.server

Note the two ‘@‘ signs ... The tool parses this incorrectly, and looks for a
server called ‘mail.com@ftp.server‘ ... The site will reject anonymous
logins without a valid-looking email address (ie, it needs the ‘@‘ symbol)
... I tride the usual shell escapes, but this doesn’t work -- it’s not the
shell doing the misinterpreting, it’s the program... Unfortunately, this was
a while back, and I can’t recall what the site or the program was -- I
_think_ I was using wget on a debian archive, but I could be wrong :) ...
but I think you have the same problems with other tools and so on -- is
there a general solution to this problem, or is it specific to the ftp
program and/or server program?
