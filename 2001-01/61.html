kernel recompile & RAID = kernel panic

Having a few problems with compiling a new kernel on a hardware raid setup.
It's a standard server install of RH6.2 (kernel 2.2.14-5.0)
Running a Mylex Acceleraid controller w/ 4 SCSI drives in RAID 5.

I'm wanting to upgrade the kernel to a stock 2.2.18 + openwall & reiserfs
patches.
The actual kernel recompiling I have done enough times b4, however this is
the first time that I have to take booting off a raid controller into
account.
I can successfully compile the kernel & modules, copy to /boot etc, setup
lilo, everything is per normal recompile.
However upon reboot the kernel panics just b4 starting the module
loading/script running routine.
(I have tried both 2.2.18 & 2.2.16-3, the Redhat upgrade kernel for RH6.2,
both with the same outcome)

I can still boot successfully with the 2.2.14-5.0 boot disk I made during
the original install.  I'm thinking it could be something to do with the
initrd.img.
The 2.2.14-5.0.img on the boot disk doesn't want to work with the new
kernel.
Is there some way I can make a new initrd.img to suit the new kernel &
support the raid drive or am I barking up the wrong tree & its some simple
thing I've overlooked?
