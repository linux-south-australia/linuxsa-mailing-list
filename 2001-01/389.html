Using Mgetty Over a Null Modem

Hi There!

(from mgetty man page)
>  syntax -   command value/value [command line switch]
> This option is used if you want to connect to machines via nullmodem cable.

So, if I wanted to connect a Windows machine via PPP to my network I
could:

* setup mgetty with this option
* have mgetty autodetect a PPP connection
* connect Linux to Windows with a null modem cable
* convince Windows to start the connection

And it should authenticate correctly and not have any problems. If
anyone's interested in why I am doing this in such a strange manner, it
looks like I may have to connect a laptop with a serial port to my
network on an ad hoc basis which (not surprisingly) doesn't have a coax
network connector. Even if it did, I have no hub and I'm almost certain
you can't string more than two computers together will crossover RJ45
(is it R or A or some other damn name - I always forget) cables.

Hence, a solution appears to be to connect this computer via a null
modem to one of my computers (probably directly to the gateway), get
mgetty to authenticate said computer, DHCP to assign IP addresses and so
forth and it should work...

Have I missed something? Or does anyone know of any "gotchas"?
