Restarting an ftp send part-way through

Since I've got the commands in my buffer, here's one for the LinuxSA tips
page. ftp can do a lot more than it is usually used for. I left a CD image
copying and went off somewhere else...

Connected.
220 meg.shearer.org FTP server 
Name (192.168.1.13:dan):
331 Password required for dan.
Password:
230 User dan logged in.
Remote system type is UNIX.
Using binary mode to transfer files.   
ftp> mput *iso
mput binary-i386-1_NONUS.iso? y
200 PORT command successful.
150 Opening BINARY mode data connection for binary-i386-1_NONUS.iso.
netout: Broken pipe
452 Error writing file: Success.

   [Bother. Out of space on the destination drive. Fix that
    problem using rm. Get the file size on the remote system (ie how much
    was transferred before the abort) and add one to it. Then back
    on the original machine...]
        
ftp> help restart
restart         restart file transfer at bytecount
ftp> restart 139900929
restarting at 139900929. execute get, put or append to initiate transfer
ftp> put binary-i386-1_NONUS.iso
local: binary-i386-1_NONUS.iso remote: binary-i386-1_NONUS.iso
200 PORT command successful.
350 Restarting at 139900929. Send STORE or RETRIEVE to initiate transfer.
150 Opening BINARY mode data connection for binary-i386-1_NONUS.iso.  
226 Transfer complete.

Did that work? Let's see. On the destination machine:

[root@meg /megdata]# mkdir /mnt/folle
[root@meg /megdata]# mount -o loop ./binary-i386-1_NONUS.iso /mnt/folle
[root@meg /megdata]# ls -l /mnt/folle [looks good, as does a du]

Fixed.
