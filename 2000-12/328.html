Given we have no information as to if Andrew Jose is wanting to use 
rlogin on a public or corporate network, you perhaps should have 
recommended that he doesn't do it. You should know the reason.


Andrew, I would recommend that you install SSH or OpenSSH to hide the 
root password. If you like to think of it ssh is a secure replacement 
for rlogin.

The reason is that with both rlogin is not secure. Any user that happens 
to have a packet sniffer can capture the root password. It will allow 
you to login as root and no-one can reasonable crack the password. I do 
hope it is for a home network and not on the DSTO's network.
