On Wed, 6 Dec 2000, Moore, Alastair (DETE) wrote:

> The only thing about slackware that concerns me is that some software -
> coreldraw for linux, wordperfect office for linux etc. seem to require
> a debian or redhat package manager. 

There are two solutions to this problem.

1) One can use rpm2tgz or rpm2targz (essentially the same programs) to
convert the RPM package into a slackware package, and then it can be
installed using installpkg or pkgtool.

2) Recently (I am not sure when this practice began), Slackware has been
shipping with RPM as a package that can be installed (it is on the install
CD).  You could install the package natively using RPM, but you would most
probably need to throw in a --force or --nodeps in order for the package
to install successfully.


To go off on a tangent, I recently noted in the Slackware.com development
forum that Pat V. posted a method of using Helix GNOME on Slackware, using
the RH 6.2 packages and RPM, so even the updater works.  This may be of
interest to some, and I can forward them his message if they would like
it.
