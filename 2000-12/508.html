Re: Who Owns a Domain (How to Find Out)

you could try

dig -x ip.address.of.interest
and progressively drop bytes

e.g.

dig -t any -x 203.16.230.115
dig -t any -x 203.16.230
dig -t any -x 203.16

Might give you what you are after

Mark

At 10:35 13/12/2000 +1030, John Edwards wrote:
>David Lloyd wrote:
> >
> > I have a pingable, live IP address that has no in.addr-arpa entries. How
> > would I go about finding who actually owns it?
>
>It's not a domain, it's address space.
>
>Try "whois", although the server you need to use depends on where the
>address is located.
>
>ie (using linuxsa IP for example only):
>
>whois 203.16.230.115@whois.arin.net
>
>gives you APNIC's details, so then you do
>
>whois 203.16.230.115@whois.apnic.net
>
>Which comes back with:
>
>inetnum:     203.16.230.0 - 203.16.231.255
>netname:     NETCRAFT-AU
>descr:       Netcraft Pty Ltd
>
>Which is as close as you get to identifying an 'owner'.
>
>There are also plenty of web tools that automate this process for you if
>you're lazy.
>
