Re: LDAP and Schema and ACL's, oh my...

nickm@bhwb.nsw.gov.au wrote:
> 
> I've been reading and reading and reading, but there's something missing
> from every single page I've read..  I started with the LDAP Roadmap on
> http://www.kingsmountain.com/ldapRoadmap.shtml - but I can't QUITE grok the
> Schema and ACL portion of the whole LDAP bizzo.

The schema establishes what data items can be stored in LDAP.
eg: userids, groupids, e-mail addresses

The ACL establishes who can access those data items.
eg: only people in the group "wheel" can modify the userid
field.

The confusion arises simply because the ACLs themselves
need to be stored in an LDAP schema (and also need ACLs
to protect the ACL from unauthorised modification ).
The whole thing then seems to be circular.

Finally, a lot of the trouble people have understanding
LDAP is related to the awful user interface.  Install
kldap to solve that problem.
