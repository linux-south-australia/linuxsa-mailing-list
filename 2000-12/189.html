Re: SCSI order under 2.2.x

On Tue, 5 Dec 2000, Dale Long wrote:

> On Tue, 5 Dec 2000, Dan Shearer wrote:
> > 2.4 has devfs for addressing devices something like the Solaris scheme,
> > only dynamically on-demand (and many, many other new things that require
> > lots of homework to be done.)
> 
> Will this include the ability to statically bind devices to id's, so that
> the removal of a drive, other device or controller in the  'middle' will
> not cause allocations be reallocated down (thus messing up fstab and
> mounting etc...)? I can image this would cause a few headaches with
> software raid.

Yes and no.  The name will include the SCSI ID, so unless you change
SCSI IDs the disk will always have the same name.

There is still a problem with controllers: if you remove a controller
then the names of the device on the other controller will change.

devfs does retain some state to avoid name changes.  However
I wouldn't use this for disks as it complicates data recovery.

Also, you don't seem to be aware that mount will take the UUID
of the partition in place of the device name.  Using the UUID
will always mount the same partition to the same mount point
regardless of its physical address.  See mount and e2tune -l.

> Is there a good and specific reference that addresses these issues
> somewhere in the 2.4 documentation?

Yes, the most readable article on the differences between 2.2
and 2.4 is Pranevich's "Wonderful World of Linux 2.4".
