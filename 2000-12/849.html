This is a Domain Name Server setup problem. When you specify a name such
as www.slashdot.org this gets looked up and translated into an IP address
(64.28.67.48). A page is then requested from the webserver on that IP
address.

You never mentioned how you were connecting to the Internet. However,
putting the lines: 

search dingoblue.net.au
nameserver 202.139.47.98      

Should enable netscape and other programs to be able to translate the
names to ip's, hence fixing your problem (You may want to check the number
- it will be listed as Primary DNS somewhere). You may also want to check
that the mail server isn't actually meant to be mail.dingoblue.net.au
