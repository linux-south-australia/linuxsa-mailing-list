Re: ftp

On Fri, 22 Dec 2000, Robyn Manning wrote:

> Hi again
> 
> So how can I transfer files with reasonable security between me and Aarnet 
> that allows resumption of downloads.
> 
> In English please or should I say laypersons terms.

1. The potential insecurity relates firstly to people sniffing
   your password, because by default ftp uses plain text passwords.
   If you are getting files from AARNet via anonymous ftp the
   username is 'anonymous' and the password is an email address, so
   you shouldn't care about that. A web browser is an example of an
   ftp client that just does this automatically for you when you 
   browse an ftp site, ie one that starts with ftp://. If you use the
   commandline 'ftp' program you have to type it in manually. Programs
   such as wget (non-interactive) and lftp (interactive) are yet more
   interfaces to the same facility.

   If you are getting files that require a username and a secret password,
   then ftp with plain text authentication is probably not such a good
   idea. Do one of the things that previous posters on this thread have
   suggested. None of it is hard.

2. You might be concerned about people knowing what files you are
   transferring. So use one of the end-to-end secure methods, rather
   than GPG-endcoded files. An example of this is ordinary ftp 
   tunnelled over ssh (using ssh -L). Another is scp. Search freshmeat
   for 'secure ftp'.

3. You might be worried about the possibility of someone attacking your
   machine because you are using an ftp client. Don't be.
