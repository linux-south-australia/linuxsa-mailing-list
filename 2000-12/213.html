Multitasking (was: introduction)

On Wednesday,  6 December 2000 at 14:38:20 +1030, Jason Tan wrote:
> On Wed, 6 Dec 2000, David Lloyd wrote:
>>> I thought that pre-emptive meant that the OS decided when a process's
>>> timeslice was up and just did the context switch the program had no say.
>>> (as opposed ot cooperative multitaskign when the program voluntarily
>>> yeilds cpu whenever it feels like it)
>>
>> Yes.
>
> So there is no reltionship bewteen an os having/supproting pre-emptive
> multitasking and fast context switching?

Only insofar as any real operating system uses preemptive
multitasking.

> Obviouly the faster the context switch the better but whether it is
> fast or slow the reason for the context switch, ie whether the
> process was pre empted by the os or whetehr it yeilded or whatever
> other reason, is nto related to the speed of the cotext switch?

There are subtle relationships, which in fact work against preemptive
multitasking.  You don't have to save as much information if you only
schedule when the process is expecting it.  You also don't have to
protect against scheduling at inopportune times.
