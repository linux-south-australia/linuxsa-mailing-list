Re: moving/resizing/imaging /usr

Hi,

> I have run out of space on the /usr partition and I would like to move /usr
> off to a bigger drive. I've been adviced that there may be a problem moving
> /usr with tar, so I was wondering if I can use imaging (ghost/ic3/etc) to
> move the partition to a bigger drive.
> 
> Any advice would be appreciated

I don't know of tar problems if you use the Tar from hell command 
(cd original_directory; tar cpf - .) | (cd new_location; tar xvpf - .)
otherwise, Ghost would work to transfer the partition in raw mode (even dd
would work here), but then you'd need to use a utility like ex2fsresize to
make the partition larger.

WARNING, HAVE A BACKUP IF YOU ARE UNDERTAKING PROCESSES LIKE THIS.
