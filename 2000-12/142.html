Re: SCSI order under 2.2.x

Dale Long wrote:
> 
> Is there a way under Linux Kernel 2.2.x that allows a device to be tied to
> a SCSI id? Eg: /dev/sda to SCSI id 0 and /dev/sdb to SCSI id 1, on a given
> interface. This can be done with FreeBSD (it was fiddly). Or in a manner
> similar to Solaris where the device name is the device/controller/lun
> etc...?

Not really.  The bus is scanned from highest to lowest
address, assigning letters from "a" onwards as it goes. [1]

If you assign your SCSI IDs from highest to lowest then
the existing names won't change when you add a new unit.
 
> Will this be something they include (or have included) in 2.4?

2.4's devfs fixes this, by making the SCSI ID part of the name.
I'm using it now and it works surpisingly well: only
some GUI front-ends break on the longer names and I'm
not a front-end sort of person.

One nasty side effect is that /etc/fstab becomes almost
unreadable as every line is way longer than 80 characters.

Glen

 [1] This makes sense if you remember that larger SCSI
     IDs have better priority.  So the main hard disk will
     almost always have SCSI ID=6, and we want it
     to have the name /dev/sda.
