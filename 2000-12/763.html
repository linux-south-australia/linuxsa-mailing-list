RE: Boot command

y'all may like to take a look at this:

http://216.121.249.151/story/runlevels/

particularly:
------- start of cut text --------------
0       halt
1       single user mode
2       multiuser with no remote networking
3       normal/full multiuser
4       reserved for local use, default is normal/full multiuser
5       xdm or equivalent
6       reboot
------- end ----------------------------

However, note that it's not standardised (yet) and each distribution is
different. I know Debian is usually set up slightly differently to Redhat...

Note also that runlevels are nothing magical. They are simply an ordered set
of scripts that get run with either stop or start as the parameter on
entering the runlevel. What these scripts should be doing is starting and
stopping services... so the only real difference between the runlevels is
which services get started and stopped when. Look at the rc dirs (/etc/rc.d
or something -- can't remember right now and don't have access to linux)...
It's quite possible that on a box that is imperfectly set up, entering level
1 from level 3 may leave you in a different state to if you entered from
level 5, and so on... :)

However, the answer to the original question is that at the LILO prompt,
type 'linux single' ...
