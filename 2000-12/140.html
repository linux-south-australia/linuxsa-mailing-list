On Tuesday,  5 December 2000 at 13:14:51 +1030, Jason Tan wrote:
> On Mon, 4 Dec 2000, Jerry McAlister wrote:
>> Not. The 286 couldn't do multitasking. The 386 introduced
>> protected-mode which ushered in the multitasking OS like Windows
>> and Linux. Windows 2.0 on the 286 was really the old DOS Shell
>> program, a simple ASCII drawn file manager that was dumped from
>> memory when another program was started. Yuck!
>
> Wasn't it a 286 didnt have an mmu therefore could not have virtual
> memory? (not sure about protected memory)

The 286 had segmentation rather than paging.  That's a matter of
semantics: it meant that a page size was variable, and up to 64 kB.
The biggest bug in the implementation was that there was no dirty bit,
so you didn't know whether you needed to page out a writeable segment
or not.

> I would have thought it would be possible to write an os that
> supports multitasking on any cpu - no hardware support required.

Correct.  The real issue is that you need memory protection for
*reliable* multitasking.

> For example the mac could do multitasking (co-operative not
> pre-emptive) way before the 68k family had an mmu.

Sure.  XENIX also ran on the 86, which had nothing in the way of
protection.
