Re: Printer woes --- again!

Printer problem solving.


Have you tried to start and stop lpr ? 

/etc/rc.d/ini.d/lpd stop

There have been times when I have had to restart the printer queue

(as root) lpc

lpc> help

In this example I check the printer queue and remove a file.
I then use lpc to restart the printer and enable the queue

root@Manduck bombay]# lpq
lpq: lp: unknown printer
[root@Manduck bombay]# lpq -Pplotter
Warning: plotter is down: printing disabled
Warning: no daemon present
Rank   Owner      Job  Files                                 Total Size
1st    bombay     269   ...                                  506391
bytes
2nd    bombay     270   ...                                  228517
bytes
[root@Manduck bombay]# lprm -Pplotter 270
dfA270Manduck dequeued
cfA270Manduck dequeued
[root@Manduck bombay]# lpq -Pplotter
Warning: plotter is down: printing disabled
Warning: no daemon present
Rank   Owner      Job  Files                                 Total Size
1st    bombay     269   ...                                  506391
bytes
[root@Manduck bombay]# start plotter
bash: start: command not found
[root@Manduck bombay]# lpc
lpc> start plotter
plotter:
        printing enabled
        daemon started
lpc> stop plotter
plotter:
        printing disabled
lpc> start plotter
plotter:
        printing enabled
        daemon started
lpc>


Recently in order to get remote printing to work I had to move
/etc/printcap and configure another. It turned out that something 
in printcap was causing a problem (there were a number of entries).
The networked printer started ok after this
