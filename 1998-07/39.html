Red Hat Linux does not include KDE.  It includes Gnome, which is
another desktop environment.

The summary (IMO, and not speaking for anyone):

- KDE is free software.
- KDE depends on a library called Qt.
- Qt is not free software (among other restrictions, you need to pay
  money if you want to develop a commercial application using it).
- Red Hat don't want such a major part of their distribution (that is,
  the desktop) to be dependent on non-free software.
- Red Hat want any part of their distribution to be able to be used to
  develop any sort of software, free or non-free.
- Another issue is that some people believe that distribution
  manufacturers are violating the Qt license if they ship it.
- Therefore unless the license agreement on Qt changes, Red Hat will
  not be shipping KDE and will continue sponsoring Gnome.

KDE is ahead of Gnome because it has been around for longer, but there
is certainly a lot of effort going into Gnome.  Red Hat have created a
division called Red Hat Advanced Development Laboratories (RHAD Labs).
From their web page at http://www.labs.redhat.com/:

You shouldn't run into any problems; Red Hat 5.1 is set up so that you
can run a.out, libc5, and glibc applications.  The most noteworthy
exception to this is StarOffice, but there is a staroffice-wrapper RPM
that installs the libc5 version that StarOffice wants, and it puts it
somewhere where it can't interfere with anything else.

I don't know what the Wine problem is -- I compiled Wine-980614 a few
days ago on a RH5.1 system with no problems.

> Also seeing as i'll prolly wish to

Anything you compile will be linked against glibc.  It is possible to
compile things for libc5, but it does involve some work.
