Re: shell

On Thu, 9 Jul 1998, Soichiro Tomo wrote:

> Could anyone explain me the term "interactive shell"? What is the difference
> between interactive shell and login shell?

If you can type into a shell and it responds to your commands, then it is
an interactive shell. An example of a non-interactive shell would be:
nice sh -c "./configure ; make ; make install"

Your login shell is an interactive shell, however the reverse is not
necessarily true. If you just run "sh" at the prompt you are starting an
interactive shell which is not a login shell.

When you start a login shell one or more extra files are sourced. Under
bash these are ~/.bash_profile or ~/.profile, and /etc/profile. When you
logout of a login shell it may also source a file. Under bash this is
~/.bash_logout.
