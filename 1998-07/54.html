Re: PPP server What to use? mgetty? or is there better

> Its all pretty much in the title, for a PPP server what does everybody
> else run to handle the Dial-in access part? I have currently got mgetty
> handling a modem but the dialing user usually has to dial then after a
> while hang-up then dial again until a valid ppp connection is made.
> Does anyone know how to enable mgetty (or something else) to
> automatically answer and start PPP negotiation rather than dialing in to
> the terminal then starting ppp in the shell.

Yes.  Check out the AutoPPP option to mgetty.  It presents the login
prompt as usual, but if a PPP session is detected on the other end it
will start pppd instead of login.
