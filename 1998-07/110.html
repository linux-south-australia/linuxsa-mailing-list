Hi Ben,

That means you need to link it with the appropriate library, that is,
libcrypt:

% cat /etc/redhat-release 
release 5.1 (Manhattan)
% cat crypt-test.c
#define _XOPEN_SOURCE
#include <unistd.h>
#include <stdio.h>
int main(void) { printf("%s\n", crypt("abc", "de")); return 0; }
% cc -o crypt-test crypt-test.c -lcrypt -Wall
% ./crypt-test
de86L5W89BsXE

Nothing to do with header files (although the #defines and includes
fix the problems that -Wall would otherwise warn about).
