> Unfortunately, I can't get it to go!!  Have I missed something

.xinitrc is only used by startx.  The one you were looking for is
.xsession, which is used when you log in through xdm.  BUT the default
/etc/X11/xinit/xinitrc and /etc/X11/xdm/Xsession files (at least under
RHL) are set up so that you can put a file .Xclients in your home
directory (don't forget to chmod +x .Xclients) and it will be used by
both startx and xdm.

In summary: if you want different behaviour for startx and xdm, use
.xinitrc and .xsession.  If you want the same behaviour use .Xclients.

> Also, (the "and ...." part), I have followed the directions
> regarding the date problem with RH 5.0 talked about recently,
> but I'm still having date problems.  Which file is it,
> that tells the system that my clock is set to GMT as opposed
> to CST.  My clock is adding 9.5 hours to the sys clock.

/etc/localtime is a symlink which selects what timezone you are in:

cd /etc
rm localtime
ln -s ../usr/share/zoneinfo/Australia/South localtime
(or Australia/Adelaide -- they are the same)

/etc/sysconfig/clock is a file which determines whether your hardware
clock is in UTC or local time.  There should be a line in there with
either "UTC=true" or "UTC=false".

Other related stuff:

To set your Unix clock, use date.  To set your hardware clock, use the
hwclock utility.  Something like 'hwclock --hctosys --utc' should set
the hardware clock to your Unix time.
