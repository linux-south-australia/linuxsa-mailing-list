Checking UARTs

Hi all
I suspect that I may have a problem with the UART controlling COM2 on my
Linux Box. I get a LOT of errors, and normal FTP transfers from a PC on
ethernet at the other end of my modem link tend to max out at 1.5kps (I
have a 33.6 external modem)
Here's the output from ifconfig ppp0 at this moment:
ppp0      Link encap:Point-to-Point Protocol  
          inet addr:203.29.178.27  P-t-P:203.22.206.19  Mask:255.255.255.248
          UP POINTOPOINT RUNNING  MTU:1500  Metric:1
          RX packets:175203 errors:889 dropped:889 overruns:0
          TX packets:202832 errors:0 dropped:0 overruns:0

I only ever get RX packets being dropped, and the frequency of an error
increases dramatically if I transfer a file via FTP or HTTP from my linux
box (I also have a Win98 box on ethernet to this PC)
My Linux box is a 486DX2-80 with 8Mb FPM RAM, High-speed I/O + HDD Card,
and it is running RedHat 5.1
I also get problems when playing Quakeworld from my Win98 box though the
linux gateway. The picture will freeze, and multiple "Warning: U_Remove on
full update" messages will appear on the screen. I believe this is related,
since the problem never happened when I wasn't using my linux box as a
gateway. I don't know whether the quake problem is due to the
slowness/small amount of RAM in the gateway, or whether it's caused by
something else.

In short, I am looking for a program for linux (or DOS, if it can fit onto
a floppy disk) that I can use to test the UART chips and see if they are
functioning correctly.
Thanks
- Steven Bolbot
