Re: mgetty respawn woes

On Thu, 16 Jul 1998, O'CONNOR, Steve wrote:

> > user2:502:501:password:/home/user2:/sbin/pppd -detach

You most likely need the 'modem' argument to pppd. From
the man page:

       modem  Use the modem control lines.  This  option  is  the
              default.   With this option, pppd will wait for the
              CD (Carrier Detect) signal from  the  modem  to  be
              asserted  when  opening the serial device (unless a
              connect script is specified), and it will drop  the
              DTR  (Data  Terminal Ready) signal briefly when the
              connection is terminated and before  executing  the
              connect  script.   On  Ultrix,  this option implies
              hardware flow control, as for the crtscts option.

One other thing to check is if you establish the link, then power down
the modem on the server end, does the pppd persist? If so and you
have added the modem argument, then you might have a cable
problem (although I doubt it because mgetty with the arguments
you gave it probably wouldn't work with this kind of cable
problem).
