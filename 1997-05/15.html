Andrew Allison wrote:

> Dear colleagues..
>
> There are two flaws which are interfering with my enjoyment of LINUX
>
>
> They concern XFree86 :
>
> 1/ I accidentally specified the first serial port for the pointing
>    device ( mouse ). This was a bad choice... My Windows 3.11 for
>    Workgroups is set up to use the mouse on COM2: This means that I
>   have to swap the mouse around all the time..
>
>   Q: How can I force LINUX to use the second com port for the mouse?
>
>        Is this device referred to as /dev/tty01 ?
>        Do I simply abandon the /dev/mouse device?
>        Or do I have to re-compile this device driver in some way?

Do the following....

rm /dev/mouse
ln -s /dev/tty01 /dev/mouse

Or run
/usr/sbin/mouseconfig (preferred)

>
> 2/ I am 40 years old and I find that the fonts in X are way..  too
>     small to read easily.. I find myself working with my face
> pressed
>     up against the screen .. wearing reading glasses.. This is all
>    most tiring.. I would like to make the default font for X much
> larger.
>
>    Q: How is this done?
>         Is the .Xdefaults file relevant?
>         Which man page contains descriptions of all the codes in the
>
>         .Xdefaults and .xinitrc files?

The Xdefaults file will be the standard (I find the font 10x20 quite
readable (and large)).
A line like

     xterm*font:   10x20

should do it.

But things like the X startup xterms will use the X switch -fn 9x13
switch to the xterm binary.

I can't remember which file they are in, but do a grep xterm .* in your
home directory should find them
(or dump heaps of stuff to the screen :)
