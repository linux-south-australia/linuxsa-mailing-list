Re: Compiling Perl

Hmmm...

> A compiled perl binary is likely to be the bytecode + the interpreter, so it
> will suffer from the same problem Java has - that being that you could
> conceiveably decompile the bytecode back into human readable source (as far as
> possible, afterall, it _IS_ perl :-)

Actually, I disagree. Having taken a look at the 3rd Edition of
Programming Perl it appears that B::C and B::CC reproduce the bytecode
into C itself. That is to say, the bytecode is NOT contained in the
executable but is instead laid out in the C source file.

Apparently, B::CC produces code whose C flow-structure is more congruent
with the Perl flow-structure than B::C.

And again, these are experimental so you should do extra testing if you
intend to use any of these tools; oh, and it does say that the code
produced is "mainly" dependency free but you will need to link in bits
and pieces of the Perl distribution libraries if you don't choose to
link them statically.
