Re: OT:Routing and ftp

On 30 Aug 2001 10:09:29 +91800, michaeld@senet.com.au wrote:

> Thank Matt for the suggestion - unfortunately the linux ftp client defaults to 
> passive mode already (from man page) and I've observed the client sending PASV 
> to Solaris' in.ftpd from the tcpdump.
> 
> Any other ideas anyone?

Must it be FTP? Could you use SCP instead? If you've got any access to
the Solaris box at all, this could be a viable alternative.

Even if you don't have root access on the Solaris box, you could
configure SSHd to bind to an unprivelidged port and get the router
administrator to change the port number.

If it has to be called from a script, avoid the need for password entry
by putting the client's public key into the ~/.ssh/authorized_hosts file
on the server.
