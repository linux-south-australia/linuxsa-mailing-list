Re: DDD (Debugger)

Hi,

I have deleted the context because it is mostly irrelevant ...

I suspect that we have all been guilty of trivializing what is a complex 
skill.

A debugger is a very useful tool, and I use gdb, more because I am 
unwilling to learn how to use another tool (ddd or what have you) than 
because I don't like being mollycoddled :-)  However, gdb will not pick 
up long term memory leaks for example, while a tool like Insure will, as 
will instrumenting your allocation and free routines and then gathering 
traces and analysing them.

I also use printf() and proper debug facilities, a la Samba's DEBUG 
facility plus my own facility to extend what Samba provides.

I also reason about what is going on inside the program.

In addition, I will construct debugging tools when needed as well.

In many respects, debugging is a lot like troubleshooting a configuration.

One needs to approach the problem methodically, know something about the 
system being debugged, be aware of the tools involved, like autoconf, 
configure, make, etc. Debugging the final source can be a waste of time 
if you are not aware that it is the result of a long tool chain and the 
bug is potentially on many platforms.

So printf() has its place, but only as one part of your armory, and only 
  in some places ...
