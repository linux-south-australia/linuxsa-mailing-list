Re: splitting files in to two groups

Hey Ant,

Here's my solution, implement it in perl, or i guess bash for the
masochistic..

get the sizes of all files, and sort in order of size, into an array.

iterate thru the array, starting from the large end {
  if next largest file will fit, move it into a @cd array.
  (ie. pop off one array, push onto next)
  keep track of total size of files in that array
  (eg. $totalsize += size of file moved)
} while the total size is under 650 Mb, and you haven't reached end of
  array

now, create a directory "CD" and foreach(@cd) { move file }; to conclude.

That will leave you with a pretty good result.. It won't be the totally
PERFECT optimum result, but its probably the best you can for that amount of
code and CPU time. Ask me about the travelling salesman problem (if you
haven't heard of it before) and i'll explain why.
