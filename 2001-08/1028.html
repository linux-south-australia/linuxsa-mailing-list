Re: DNS question - new hostnames

> Ben Williams <benw@webmedia.com.au> wrote:
> 
> according to my understanding (and please, please correct me if i'm
> wrong[1]), the TTL field controls how long an external NS will cache
> my zone file info for. therefore, at the worst case, there's going to be
> a delay of $TTL before *changes* to an existing dns entry (eg changing
> CNAME, changing IP) will be visible to every external name server[2].

this bit you have correct. if a name server has queried your server less then a TTL 
period before the change, it will have incorrect data for up to another TTL period.

> but, is this the case for *new* entries?

no. a name server only has cached data if it has queried for the name. I also 
believe that some name servers don't cache negative data (ie. if the query 
doesn't get a result, because the hostname does not exist or some other 
problem). so if the hostname didn't previously exist then they won't have any data 
for that hostname in their cache. however, I can't remember which name servers 
do and don't cache negative data.
