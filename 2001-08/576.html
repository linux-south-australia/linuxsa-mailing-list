Relational databases in Linux

Morning all.

For the last year or so, I have been gradually moving my applications
and data from Windows98 to Linux - starting with Caldera, now using
Mandrake 8.  Eventually, I hope to be able to dump Windows from my
computer altogether.

I have reached the stage where the only times I use Windows are when I
need to work with the half dozen or so databases I have developed
using Microsoft Access.  These range from simple databases for
collections (stamps, books, audio CDs) to more complex ones to do with
my sporting involvement (for example, the membership database for the
SA Orienteering Association).  None of these databases need to be used
anywhere except on my own, single, computer (ie, no networking, or Web
access is involved).

Can anybody suggest anything in the linux world which I could use as a
functionally equivalent replacement for Access please.
Note that I have looked at pgaccess, but it has nowhere near the
features I need yet (if I have to write my own front ends to, say,
PostgreSQL databases using Perl, C, Tcl/Tk etc, the process will take
forever).
