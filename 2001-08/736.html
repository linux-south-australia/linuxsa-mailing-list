Re: 20gb HDD 10gigs to linux 10gigs to fat32 how?

On Sun, 19 Aug 2001, Michael Kratz wrote:
> So what? I would have to use like partition magic or something in win2k to
> partition the drive 10gigs fat32 then 10gigs ext2 then when I install linux
> repartition what would be /dev/hdb2 or would it be /dev/hdb5 because what
> would come up as /dev/hdb1 would be the fat32 partition.

If you use Partition Magic, you choose.  Make that second partition
Primary and it would be hda2,  Make it extended and put a logical drive
covering the whole space inside it would be hda6 (hda5 is the extended
partition).  
My preference would be to create the extended partition
and use RH fdisk to set up the logical partitions you want. 
Alternatively, just leave the second part of your drive as "Free Space"
and let the RH7.1 install do what it thinks best, automatically.

I gather from another post that RH7.1 avoids that problem.  If that is
so, you don't have that worry.
