OK, just try:

   /path/to/syslog

on the command line. It should print out the version, and wait for 
input. You can then do ^D (Control-D) to exit, or type QUIT.

I think the problem is that the log level is not high enough. What 
version of syslogd is the RaQ running? You can check this with 'syslogd 
-v'. However, you might need to put a path on the front.

I seem to recall that for older versions of syslog, it would not print 
debug and info messages by default. I am running 1.3-3 and 1.4-0 on 
different machines, and they both seem to send all messages to the 
syslog file if you have something like mail.*.

You may want to add mail.info,mail.* to /etc/syslog.conf to see if that 
changes things.
