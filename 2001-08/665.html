Michael,

The problem with the pscp is that it uses ssh.
In other words, if you make your server machine
able to do pscp, then it also does ssh - and that
means that the user can get a local shell.
What I really want is an "scp" that does not
require the provision of an ssh shell.

I've started a few times to put together a
restricted shell that only does scp and
nothing else. But all the sources of information on 
this that I've looked at so far are so convoluted and
incomplete that I gave up. 
The idea I wanted was that scp should work perfectly,
but an attempt to log in with ssh should result in
immediate logout with no possibility of any mischief.
