RE: Debian APT source password?

> You can setup the apt server as a normal ftp server with user access
> only (ie. no anonymous access).  Then in the sources.list you can
> specify the useranme and password like ftp://user:pass@server:port/.
> You can do a similar thing for http access (see man 5 sources.list)

or you can use a http proxy and set the http_proxy environment variable
before apt-getting... (http_proxy=http://user:pass@server:port/) ... I've
done this before... obviously, you may want to think a little before putting
your password in plaintext, but ...

>
> I don't know how to (or if you can) get apt to prompt for this
> password interactively but the source is there... go hack :-)


depends what you are trying to acheive... if you have many people with many
machines, and you want to allow them access to your server selectively, you
want some combination of usernames/passwords on the server, and perhaps
ipchains to stop connections from certian IPs... Of course, you could get
really really funky with this if you wanted...

However, if you are trying to allow certain people to apt-get on one
machine, use sudo.
