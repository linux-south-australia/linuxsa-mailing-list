Nope, I don't think so. Certainly, webservers in your own 192.168.x.x
network are attackable by Code red on an internal or gateway webserver,
but not to other private networks.

Looking at my own network, which has nat, and 2 web servers, 1 on the
gateway and one within, I see 145 code red attacks since Sunday on the
gateway and a clean zero on the internal server.

If my boxes were IIs, and attackable, I'd expect the internal server to be
infected from the gateway once it was infected.
