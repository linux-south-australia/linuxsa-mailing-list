Re: Cisoc 827 and IPTables

Hi Steve,

I'm fairly sure that your problem is the "service config" line at the start.
Try either removing it altogether or setting it to "no service config".

It tells the router to look on the network for a config file (at whatever 
TFTP location responds).
It is normally used with the "boot host" or "boot network" commands to 
specify where it
will find that config.

There is a full explanation at:
http://www.cisco.com/univercd/cc/td/doc/product/software/ios122/122cgcr/ffun_r/ffrprt2/frf007.htm#1017708
