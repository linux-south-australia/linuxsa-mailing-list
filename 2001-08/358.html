Code Red II experiment

The majority of Code Red probes I'm getting (over 500 so far
this week) are from dialups. Something I read in MS Technet
a few months ago prompted an experiment with one of my
spare boxes on Monday night...

On a blank HDD I did a default install of Windows 98, which
would have included Personal Web Server etc. Then I did an
upgrade to Windows 2000 Professional. Although this box was
specifically not a webserver, IIS 5 got installed anyway.
According to Technet this always occurs if any MS webserver
program is detected during Win2000 install.

To continue the experiment, I then ignored my CD-full of MS
Service Packs and patches, and simply used my alternate dialup
to go on the Net with Win2000 Pro. Typical "home user with a
new system" scenario.

Ten minutes later, I disconnected from the Net. And then ran
some tests. Yes, the box was infected with Code Red II. In
ten minutes!

I'm thinking the only way to stop Code Red is to release
something that kills the infected systems completely. That 
might force people to wonder if there's something amiss with
their Win2000.
