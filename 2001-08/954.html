Re: Tail Recursive

Heh!

I couldn't see the wood for the trees, and I was falling over the
Lisp/Scheme syntax. Naturally, because Scheme _must_ be tail recursive,
most of the examples use Scheme to illustrate it.

For everyone's interest, a function is tail recursive if there is
nothing left to be done when the function returns.

Hence (--1):

function R(int *x) {

  if (x < 0) return;

  &x--;

  R(x);
}

This function is tail recursive because whenever the function returns
there's no more work to do. Why? If x drops below 0 then it will keep on
returning and the &x-- won't be executed.

function NR(int *x) {

  if (x < 0) return;

  &x--;

  NR(x);
  printf ("boo\n");
}

This function is not tail recursive because when NR finally does start
returning, it will still need to print "boo". But it is still recursive
because it will call itself.

Apparently this is a benefit to compiler/interpreter writers as well as
programmers (1).
