Re: Does anyone know of resellers of 'embedded'-386 boards?

Don't forget CRAMFS (part of the Linux kernel, if you've enabled it)
and ROMFS.. There would be a few advantages for either. ROMFS uses less
kernel memory for the driver, but CRAMFS compresses more.

Then, use TMPFS instead of a ramdisk, since it only uses memory as it
requires it.. leaving the rest for your programs.

So, mount the RO flash on /, and mount tmpfs partitions on /tmp, /home,
/var...

This saves you the need to waste memory by untarring the flash into ramdisk.
