RE: splitting files in to two groups

> I've got 1.3gb of mp3s, that I want to put on 2 cds. I could manually
> split them up and keep re-checking sizes of groups until I have two
> equal sized groups, but does anyone know of a simple way or a little
> util that will do it for me? An automatic way would just be usefull to
> know for probably other things as well.

the simplest way is to get the total size, divide it by 2 (say 600Mb), and
then one at a time, move files from one directory (.../cd1) to another
(.../cd2) until the size of the first directory comes below half the total
(or until the first > the second... etc)...

Of course, if you wanted to have the same number of files on each CD as
well, then you would want to:

sort by size
iterate over files
	copy file from store to first dir
	copy next file from store to second dir
repeat.


this way, you would have an approximately equal size (not quite, mind you),
and similar fiole size distribution...

to get it even closer:


sort by size from largest to smallest
while files left loop
	copy file from store to smallest dir
repeat.


This would give you approximately the same number of files, similar file
size distribution, and similar sizes... :)
