Re: packet capture/modification/reinjection

Correct. The TCP/IP implementation is able to tell the packets are false.

The packets contain a sequence number that must match for the packet to be
accepted. You can find more information on this which will answer your next
question, which is "how do i create a number that will be accepted?"

(There are also various checksums in the packet, but I assume you are fixing
these?)

Toby
