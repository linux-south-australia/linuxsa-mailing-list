Re: Slow "Internet" (was: [OT] Can't Think of a Subject)

We're seeing about 50-150 scans per megabit per second.

Nothing to worry about in capacity terms, so I suspect
any slowdown isn't due to Code Red.  In capacity
terms Code Red runs fine over a modem link (which
is one of the reasons why it spreads so successfully).

However, flow inspection devices like firewalls, some
transparent proxies (depends on the configuration)
and the like are slowing badly when faced with a
mere 250 flows per second (ie: way under vendor
claims).

AARNet chose not to run these sort of devices and
that seems to be the right, if expensive, choice.
Of course, bandwidth is so expensive in Australia
that most ISPs are forced to run a transparent
cache.

Most companies run stateful firewalls, for reasons
I don't understand.

So if you are seeing slower Internet performance,
they are probably the two reasons.
