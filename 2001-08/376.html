michaeld@senet.com.au writes:
> library and the standard math library. Without these two libraries, a

> national language (locale) support and timezone databases.
> 
> >rpm -q --whatrequires kernel-headers
> glibc-devel-2.2-9   ****** But not glibc :-(

I think this make sense. You don't need the header files for runtime,
only for compiling. I don't have a Red Hat box handy at the moment,
but I'm guessing that the glibc package doesn't contain any header
files at all.

> There's no link in RPM's dependencies between kernel-headers (which provides 
> /usr/include/linux) and glibc (except the link to glibc-devel), and no 
> indication of this link in the doco either :-(

The link between kernel headers and glibc-devel is understandable. On
for example my Debian system, /usr/include/linux is just part of the
libc devel package:

cyeoh@rockhopper:~$ dpkg -S /usr/include/linux
libc6-dev: /usr/include/linux

> Hmmm.  I like Chris' suggestion, but the deps don't support it, and I 
> sympathise with Richard's annoyance.

The bug is probably with the way the modules were set to compile.
They should be asking for the root of the source kernel tree and not
assuming that /usr/include/linux matches the kernel being used (which
may be true for a vanilla installation). The makefile/config was
probably setup for an older RH system :-)
