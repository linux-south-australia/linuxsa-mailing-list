Re: compiled NIC drivers and fullduplex

> From what I've managed to find out so far, all options parameters
> for compiled drivers must be specified in the source before
> compiling the kernel.

You can pass parameters from LILO or use an option
to ifconfig.

The source code supports an option 5 called "100BaseTx-FDX"
so in LILO say
   append="tulip=5"

> Reason being that I'd like to force fullduplex mode on the card
> to be in line with a Cisco switch.  The autonegotiation doesn't
> work as well as it should on the 1900 series.

This isn't so.  The fault lies in the DEC Tulip card.  There
was an "AA" release of DEC's 21040 ethernet chip that fixed the
autonegotiation fault.  Given current NIC pricing, best solution
is to discard the NIC.

The problem is so notorious that if you allow KERN_DEBUG messages
to be printed then the entire autonegotiation is printed in
great detail.

John Edwards wrote:
>
> Alternatively, it might be easier to set your Cisco switch to force
> duplex on that port.

This can't be done.  Either both interfaces need to be
autonegotiate or both interfaces need to be set manually.
