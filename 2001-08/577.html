RE: Damn you dhcpd

Hi all,

> > # /etc/dhcpd.conf
> > subnet 192.168.0.0 netmask 255.255.255.0 {
> >    range 192.168.0.10 192.168.0.100;
> > }
>
> While I'm no DHCPd expert, I know enough to think that the netmask
> and/or subnet is incorrect.
>
> You've specified a netmask for a /24 subnet, while the subnet is set to
> what would be a /16.

Yes and no, from looking at the rest of the config, Craig is only using the
first 100 addresses in a 192.168.0.x subnet and the 255.255.255.0 netmask
will be fine. If Craig, however, had machines with addresses of 192.168.1.x
etc then he would need something different as his netmask.

I found that adding the route made no difference to my dhcp setup.

If all you are getting is the welcome message, there is some changes that
you need to make to your dhcpd startup script. It may be different for
RH7.1, but on my Progeny machine, I had to change a value from 0 to 1 to
allow dhcpd to run.
