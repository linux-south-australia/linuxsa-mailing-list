Re: Blocking Code Red ingress to your ISP network

Alan Kennington wrote:
> 
> Now we need something like that for iptables.

Only if you are using Linux as a router.  If you are
using it for web serving you just may as well let
it propogate into the user space web server.

Another solution, similar in spirit to NBAR,
is to use a transparent web proxy on incoming
traffic.  Unfortunately, the results with
Cisco kit asren't as encourgaging there,
although the WCCP-based proxies from other
vendors seem fine.

The problem with both of these approaches is
that it is very easy to write a Code Red III
that presents a random string.
