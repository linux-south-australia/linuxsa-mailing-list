Re: Dell pulls the plug on desktop Linux

Alan Kennington wrote:
> 
> Although I've never tried to buy a preinstalled linux
> system, it seems to me that you usually will want to
> do a lot of configuration, re-installation etc.
> during the life of any machine, and it's easier to
> install and configure linux yourself than to tell
> someone else what you want on it.

Having done a fair bit of installation of Linux on
the AARNet servers (two in most states, three in some),
no.

You want Linux on the machine, even if it isn't set
up exactly as you'd like.  You can configure or
reinstall from there.  And you can do that without
physical access to the machine.

If the box ships with Windows or no OS then you are
stuffed as far as remote deployment goes.  If it
has Linux you can get the colo site to configure
enough of the remote console to get it answering
the modem from single user mode.  From there you
can configure.  If you get them to leave a blank
floppy in the drive you can even remotely reinstall
or upgrade.

> 1.      People trying to put linux on really wierd
>         hardware or trying to make linux do really
>         wierd, off-beat thing, or

If Linux comes with the box then you know the hardware
works with Linux, somehow.

> In case 1, you'd much rather know what is going onto
> your machine, so that you can fix it when it goes wrong.

Alternatively, you might simply want to copy the
/etc/X11/XFree86Config that the vendor supplied.
Similarly for other magic files.  No intimate
knowledge of the hardware needed, although Dell
do provide very good hardware doco.

> The skills required for installation and initial
> configuration are closely related to the skills required
> for ongoing maintenance. So if the user can't do one,
> how can they do the other?

Not at all.  You want to configure ongoing maintenance
using something like autorpm so that you touch only
one machine to maintain all of your machines.  Testing
an RPM and dropping it into the distribution directory
doesn't require a huge amount of skill.

> Saying "Dell installed my OS." is not likely to
> be at all helpful.

Yes it is, because you can then ask Dell tech support.
And to be fair to Dell, their installation of Red Hat
Linux 7.1 is very bog-standard.

> Bundling is bad.

Red Hat Linux from Dell is a zero-price option, so you
save nothing by going for the bare PC.
