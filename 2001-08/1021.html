> I wonder how I can find out what the floopy kernel's config is -- as
> the Red Hat standard kernel already supports serial console.

I don't think you'll need to find the exact same config. Like I said
before, I think that the initrd stuff (and perhaps loopback filesystem
support) is the only thing that the floppy relies on that most people
don't put in a kernel anyway (filesystem support, IDE and/or SCSI support,
etc).

> They drop DTR which then hangs up the modem.  You actually
> want this to happen, otherwise when the user logs out the
> modem call isn't cleared.  The alternative leads to tricky
> situations where the console is waiting for someone to log
> in but the modem is still holding up an old call.  This
> isn't desirable behaviour on your Command Line of Last
> Resort.

I guess if you need to make a couple of phone calls, because you had to
restart, it's better than no access at all. As long as you were guaranteed
that when it restarted, you could get back in, it would be bearable for
most.
