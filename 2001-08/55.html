Re: Robust date parsers

On Wed, 1 Aug 2001, dan@tellurian.com.au wrote:
> On Wed, 1 Aug 2001, John Edwards wrote:

[...]

> The problem with ignoring dates is this: if you have 20k messages in a
> single folder (not uncommon) and if you are using IMAP (not uncommon),
> and if the IMAP daemon sends the result of LIST commands in inode
> order, then you avoid a very expensive sorting operation. If you are
> not using IMAP, but instead something like a Maildir-aware mutt
> directly reading the Maildir, then the same can be true.

Maybe I am missing something here, but isn't there an easier way?

Why not find an MUA that understands IMAP and Maildir (such as Gnus,
maybe some patched mutts), open up the source (IMAP) mailbox, sort it as
you would like, and then tag and move the messages into the new
(Maildir) mailbox.

Come back in the morning when everything is done if you have a really
slow machine.
