On Fri, Aug 03, 2001 at 03:21:34PM +0930, John Edwards wrote:

 > "Freeman, Peter (ERHS)" wrote:
 > 
 > > Reason being that I'd like to force fullduplex mode on the card
 > > to be in line with a Cisco switch.  The autonegotiation doesn't
 > > work as well as it should on the 1900 series.
 > 
 > You should be able to pass options to a module to set duplex on or off,
 > I don't think you can pass options to the kernel without specifying them
 > in lilo somehow.
 > Alternatively, it might be easier to set your Cisco switch to force
 > duplex on that port.

That won't get you out of forcing it on the Linux box too.  

Autonegotiation on switches only works if *both* ends (i.e., the
switch and the thing plugged into it) are set to "auto".  If you 
set one end to force a particular speed/duplex combination, you 
need to set the other end too.
