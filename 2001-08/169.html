new code red worm variant

My theory now on the new Code Red variant is this:

SOmeone in the US got sick of having Code Red probes in
their web server logs.
So they wrote a Code Red variant to re-infect already infected
machines with a new variant.
The new variant only probes addresses in the 203.x.y.z
range - and possibly some in the 211.x.y.z range.
See these for details:
http://www.ipindex.net/c/203/203.html
http://www.ipindex.net/c/211/211.html

The new variant looks different in the log files.
If you have a web server in the 203.x.y.z range, you will
know what I'm talking about.
Now about 90% of all probes to my web server are of the
new X-variant. The old one had all "N" characters in it.

203.231.9.10 - - [05/Aug/2001:07:36:22 +0930] "GET /default.ida?XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX%u9090%u6858%ucbd3%u7801%u9090%u6858%ucbd3%u7801%u9090%u6858%ucbd3%u7801%u9090%u9090%u8190%u00c3%u0003%u8b00%u531b%u53ff%u0078%u0000%u00=a  HTTP/1.0" 404 1372 "-"   

If my theory is right (or even if it isn't), a consequence is
that we in Australia are in for a lot more bombardment than
the rest of the world from Code Red.
It's really mucking up all my web server log files.
And it's going to increase the load on Australian networks.
24 hours ago, Code Red made 1 to 1.5 probes per IP address per hour.
Now that's up to about 3.5 to 4 probes per IP address per hour
in my areas of IP space.

Unfortunately, I can't think of any way to get rid of
this nuisance with packet filtering etc.
The only way I can think of is for someone to write yet another
worm-variant which will go into all infected
machines and install the patch to make them immune!
If anyone out there has the right worm-writing skills, they
could do the world a big favour!
