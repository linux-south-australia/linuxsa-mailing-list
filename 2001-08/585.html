Re: Damn you dhcpd

Andrew Reid wrote:
> 
> On 14 Aug 2001 23:10:57 +0930, Craig Keogh wrote:
> 
> > My dhcpd configuration file is simple. It looks like:
> >
> > # /etc/dhcpd.conf
> > subnet 192.168.0.0 netmask 255.255.255.0 {
> >    range 192.168.0.10 192.168.0.100;
> > }
> 
> While I'm no DHCPd expert, I know enough to think that the netmask
> and/or subnet is incorrect.
> 
> You've specified a netmask for a /24 subnet, while the subnet is set to
> what would be a /16.

True, and non true. It's irking me hearing so many people say this
subnet is incorrect. the 192.168.x.x is a bank of 255 class C addresses.
This meants that the correct mask is 255.255.255.0 True, depending on
what you are doing 255.255.0.0 might me more appropriate, but
technically speaking 255.255.255.0 is the correct mask
