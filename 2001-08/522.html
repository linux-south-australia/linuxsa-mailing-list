Re: procmailrc re-send recipes (which fallback)

Hi all,

Alrighty! I've at least managed to come up with a dodgey comprimise for the
fallback if I get "user unknown" for the prefix users getting filtered.

The following recipe, will send a copy of the message back to master's 
spool file
(although it does still end up in dead.letter too...which sucks a bit).

/home/master/procmailrc:
----------------------------------------------------------------------------
DOMAIN=foo.com.au
ENV_TO=$1

:0f
* ENV_TO ?? .
| formail -i "X-Envelope-To: "$ENV_TO@$DOMAIN
:0fE
| formail -i "X-Envelope-To: UNKNOWN"

:1:
* ^To:.*(NSW|VIC)
! $ENV_TO@localhost

:1e:
/var/spool/mail/master
-----------------------------------------------------------------------------

Any ideas how to can the extra copy going to dead-letter ?
