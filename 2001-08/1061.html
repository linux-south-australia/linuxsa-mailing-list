Re: Tail Recursive

Martin Sandiford wrote:
> 
> Most procedural languages do not implement tail recursion, for example
> the following C function *could* be compiled to run in constant space.
> I am not aware of any compilers that do this.
> 
>     void foo(int n) {
>       if (n == 0) return;
>       printf("%d\n", n);
>       foo(n-1);
>     }

There's been a lot of work in tail recursion with GCC.  This
is to allow languages like Scheme to be compliled down to
C rather than down to assembler or down to the input to the
GCC backend.

This is desirable as the C compiler can then apply all the
traditional optimisations, making life easier for the author
of the Scheme compiler.

GCC 2.9.2 will use the tail recursion optimisation in the above
case.  But because of the design of C there are restrictions, the most
notable being that global variables and local arrays can't be used.

There's a lot of talk about adding a notation like

  __tailrecurse__ foo(n-1);

so that languages like Scheme can be compiled down into
efficient C.

A related problem is the ability of Lisp family langauges
to return multiple values.  This needs to be optimised
too for Scheme-->C-->assembler to be as fast as
Scheme-->assembler.  Currently GCC on ia32 only efficiently
handles the case where one parameter is returned.
