Re: Compiling Perl

Matthew Geddes wrote:
> Perl 5 comes with an experimental feature which allows you to compile perl
> scripts into a binary format. I can't remember whether it's executable by
> itself, or whether it's just bytecode that you still need the perl
> interpreter for, but the man pages will probably tell you more.

A compiled perl binary is likely to be the bytecode + the interpreter, so it 
will suffer from the same problem Java has - that being that you could 
conceiveably decompile the bytecode back into human readable source (as far as 
possible, afterall, it _IS_ perl :-)

The perl compiler might do some obfuscation, but the java experience shows us 
that you can always get back the original source.

So just give up and distribute your source as well :-P
