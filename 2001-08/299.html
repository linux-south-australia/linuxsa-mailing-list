Essentially you want to pipe your mail through a program. Therefore
using the following tools:

* ~/.forward
* .procmailrc
* emacs/vi/some unix text editor

And:

man procmailrc       

|      Starts the specified program, possibly in $SHELL if
       any of the characters $SHELLMETAS are spotted.  You
       can optionally prepend this pipe symbol with  vari­
       able=, which will cause stdout of the program to be
       captured in the environment variable (procmail will
       not terminate processing the rcfile at this point).
       If you specify just this pipe symbol,  without  any
       program,  then  procmail will pipe the mail to std­
       out.

I think you'll have to adjust the program that you listed. I can't quite
work out how it intercepts STDIN although it seems to use the
"spaceship" operatore (<>) to read the file name, which implies that
it's reading from a file name. I think replacing the:

while (<>)

 with

while (<STDIN>)

 is what you need.
