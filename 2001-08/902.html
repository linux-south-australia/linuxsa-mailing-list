Upgrading kernel when using RAID1

***************************************************************
Having slain the evil 'HTML email dragon' the pilgrim kneels
before the mighty 'Oracle of Penguin' and submits his question.
***************************************************************

Hi,

I'm have a RH7.1 system setup using a software RAID1 and 
it's working fine.

/boot  /dev/md0
/      /dev/md1
/usr   /dev/md2
/var   /dev/md3
etc...


lilo --
boot=/dev/md0
map=/boot/map
install=/boot/boot.b
prompt
timeout=50
message=/boot/message
linear
default=linux

image=/boot/vmlinuz-2.4.9
	label=linux
      initrd=/boot/initrd-2.4.9.img
	read-only
	root=/dev/md1

image=/boot/vmlinuz-2.4.2-2
	label=linux-2.4.2-2
      initrd=/boot/initrd-2.4.2-2.img
	read-only
	root=/dev/md1

---------------

The problem is I need to upgrade the kernel to 2.4.9
I've built a kernel with CONFIG_BLK_DEV_INITRD=y 
CONFIG_BLK_DEV_MD=y. Then used 
mkinitrd /boot/initrd-2.4.9.img 2.4.9

The problem is the system then hangs on bootup while 
mounting /  I beleive that the RAID is not starting
properly.
