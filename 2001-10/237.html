Re: RedHat 7.1 locks up during install

Hi,

I don't know what is wrong with your specific computer, but I have some
suggestions.
Firstly, if you take the computer to a LinuxSA meeting, I am sure you'll
have a room full of people willing to try and fix it. A few of them are
really quite clueful and would probably be able to solve it accurately.

In the meantime, I can suggest the two reasons I have experienced that have
caused that same problem before.

1) Bad RAM, or >1 GB ram
  * Bad memory will screw things up for you everytime, in wierd ways.
    However, if your system used to be perfectly stable, you can discount
    this. If, on the other hand, you've recently added or changed your
    memory around, then try swapping it out.
  * Too much memory.
    RedhAt 7.1 will not install on a machine with more than
    1Gb of ram, in my experience. Instead, you have to remove the excess
    ram, install, then recompile the kernel (don't use redhat's source!),
    and add the extra RAM back.
    This too-much-ram problem shows up as exactly what you've described
    during install. But, you have only 256 Mb.


2) Corruption on your harddrive
   If your harddrive is silently corrupting itself when written to, perhaps
   because of a faulty IDE chipset, or more likely, a faulty driver for the
   IDE chipset, then you can see the problem you've experienced as well.
   You haven't posted enough information to diagnose this problem.. What
   motherboard and harddrive do you have?
   Most things are supposed quite well under recent linux kernels, but i
   have seen corruption on AMI Megaraid, HP netraid, and i think there were
   bugfixes to a via chipset of some description too. The via chipset bug
   only occured under certain rare circumstances (i run a via chipset of
   the type, and never experienced problems at all).. but perhaps you've
   hit that one?
    All those bugs were fixed later.. Redhat 7.1 uses are rather early
    kernel version, unfortunately.

Hope this helps

Toby
