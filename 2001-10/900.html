RE: Boot CDRom Creation - Dual Boot XP / RH7.2

Its all pretty flexible from my experiences. It doesnt matter where
anything is as long as there is some way of running it, and that "it"
can find the next bit of the boot sequence. You can use lilo to get past
the MBR stage, and boot the nt loader, you can put the nt loader in a
file and use lilo to boot it from the file. Since NT is the only bit of
the chain thats picky you need to make sure that that its going to be
happy to find and load/run the main part of the OS. This will depend on
what partition it is on, and if it is an invalid one, and even then you
can still cheat the system with good boot loaders. I know grub can swap
bios drives so that you can have multiple MS OSes that all need the hda1
partition, and it can make them all think they have it after boot. I
belive lilo can probably do this as well, but I have been using grub for
some time because it can handle strangeness better than lilo. At home I
have technically invalid partioning (partition that doesnt end on a
cylender boundry I think) and grub works fine and lilo, well... LILILI
or L0L0L0 I think...
