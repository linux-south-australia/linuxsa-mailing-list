Re: RedHat 7.1 locks up during install

Dale Long <dalel@loftuscomp.com.au> wrote:
>On Tue, 9 Oct 2001 davidn@rebel.net.au wrote:
>> Now if only
>> I could get the SB16 working -- the machine hangs whenever KDE starts.
> How did you configure the sound card? Did you use the sndconfig tool?

I configured the kernel for modular sound, sb16 and yamaha opl-3, and
compiled and installed the kernel and modules.  Then I added entries
to /etc/modules.conf for the sound blaster, pretty much as described
in Documentation/sound.  I can't currently give you the exact entries,
so from memory:

 alias sound-source-0 sb
 options sb io=0x220 irq=5 mda=1 mda16=3 mpu_io=0x330
 post-install sb /sbin/modprobe -k opl3
 options opl3 io=0x388

On bootup, when rc.sysinit loads mixer settings, the soundcore,
sound, sb_lib, opl3 and sb modules are all loaded, the card is reset
(as evidenced by a "clunk" from the speakers), and the kernel displays
messages indicating no ISA PnP card found, and then found SoundBlaster
16 (220) IO=0x220 IRQ=5 DMA=1,3 and then SoundBlaster OPL3 IO=0x388
IRQ=5 DMA=0,0.

I was able to get audio from an mpeg video, don't know which player
I used, but when I boot KDE the machine hangs after loading aRtsd,
presumably when it's about to play the kde starting sound.
