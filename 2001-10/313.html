Re: Hot swap IDE

John, 

> Unfortunately most OS's only 'initialise' hard drives at boot,
> usually relying on information transferred to them by the bios. If
> the drive isn't present on boot then the OS won't automatically make
> the right device entries for it etc.

You could also make the IDE driver a module and reload it.  Ofcourse,
your root file system (and any other file systems you want while you
swap) would then have to reside on a device which was not IDE.

> As seen on last week's episode, I use the 3ware IDE controllers in a
> number of systems, which overcome most of the above problems by
> abstracting the IDE interface from the OS. The OS sees a scsi
> drive(s) which is actually made up of multiple IDE hard
> disks. Because the firmware can re-initialise a drive without
> rebooting, new drives can be added to an array on the fly. The
> firmware is accessed by OS-specific software that has a web
> interface.

Sounds interesting.  Where can I get more details or these, where can
I buy one and how much are they?  

> This doesn't help much if you want hot-swap for data portability
> (maybe look at an external USB/IDE casing) but its a definate plus
> if 100% uptime is your goal. Also, the 3ware cards are EOL'd as of
> last week, so get one soon if you're interested :)

USB would be good but I think it is going to be too slow for the
