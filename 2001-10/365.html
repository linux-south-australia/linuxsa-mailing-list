I don't think so, because there is only a very limited amount of tuning
that can be done without knowing your environment. The choice of setting
up an imapd involves choosing your mail storage format. This depends on
many things, including any local MUAs you want to use. Either you decide
that all mail access will be via IMAP, or you agree on a file format
supported by both the imapd and the MUAs. The distro doesn't know your
preferences or skills in this area, and so just picks imap defaults of BSD
file format and mailbox == username (are there any exceptions? I can't
think of any.)

One thing it is possible to say is that for many mail operations, BSD
format is both risky and slow.
