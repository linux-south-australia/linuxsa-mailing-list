Is the boot device in the BIOS set to IDE, instead of SCSI?  Maybe its
trying to boot off a half installed lilo install on the SCSI drive.

If not, then your lilo config is probably mulched.  I've got no idea if
you're able to access X or not, but if so, checkout freshmeat.net for some
GUI lilo config utilities, which can make the task 9 million times easier.

I might be stating the obvious.. but.. its worth a shot.. hda1 = first
partition on your hard drive.  If you want the boot loader installed in your
MBA (where you'd likely want it if you're not using another boot manager)
you want hda, which is the actual hard drive device.

> no linux. Using loadlin I get a kernel panic because it doesn't load the
scsi
> stuff soon enough.

You'll probably need to recompile the kernel.  I've never actually used a
SCSI Linux box, but I'm assuming that you're probably using a default kernel
which loads scsi support via. modules - which of course it cant load,
because the hard drive isn't accessible. :-)
