>If the machine matters you, dualbooting may not be the best option.  An
>alternative is payware vmware, which does completely isolate 
>the guest OS
>from your host OS. www.vmware.com. There are drawbacks to VMware but it
>does work quite well for what it tries to do, if you disregard 
>the sales
>literature.

While Vmware is a great piece of software, and I've had success with it in
the past, I'd imagine that on a laptop running with 64mb, it may not be the
best option, especially when you consider the minimum specs of XP, if not 
RH7.2.  I found Vmware to be adequate running Windows with 64mb allocated on
a machine running 128mb RAM, but personally I'd only ever consider running
the lower end versions of windows (95,98) with 64mb.  When you'd have to 
allocate 32mb to your VM (48 if you're game enough to leave X running with
16mb ;o), it'd result in a rather painful experience one would think!
