This is one of the policies which RedHat and Debian differ on which is
reasonably fundamental.  

All packages in Debian stop and/or start the daemon (if required) when
upgrading/removing/installing a package.

All Redhat packages don't stop and/or start the daemon.  This mean you
have to manually restart everything, which if you are doing on a bunch
of packages means it's probably better just to reboot.

This is a RedHat vs Debian issue not an RPM vs deb issue.  

IMHO I prefer the Debian policy.  They are probably a number of sys
admins out there who run "rpm -F sendmail-<version>.rpm" when there is
a new version of sendmail released due to a bug and think that their
system is now running a bug free sendmail.
