> The permissions may indicate that the file itself is executable, but
> if the filesystem on which the file resides was mounted with the
> noexec flag then you will not be able to directly execute the file
> (though there are ways around it without copying the file).
> 
> If you run mount without any arguments you'll see something like this
> when the cd is mounted:
> 
> /dev/cdrom on /cdrom type iso9660 (ro,noexec,nosuid,nodev,user=cyeoh)
> 
> If this is the case you can remove the noexec flag with something
> like:
> 
> mount -o remount,exec /cdrom
> 
> Regards,
> 
> Chris.
