Re: Linux and VIA Chipsets

I have linux installed on a MSI mother board that I think uses a VIA 694X
chipset - it ertaily uses a 694 chipset, just not sure about the X bit.

I think it is this one (the machine is at home and I'm at work so I cant 
confirm):

http://www.msi.com.tw/products/mainboard/mainboard.php?model=MS-6321

(I have the non firewire, non "hw raid", non scsi version).

It works satifactorily for me and my needs.

I am not sure if all the finer features of the chipset work however.
Eg the board has 4 IDE busses 2 ata 66 and 2 ATA 100. There is a warning
at boot time regarding the disk on the ATA 100 bus.

I have not invesytigated this, as the disk is available and seems to work,
and I dont care about performance really. htis is a promise ata100
controller that I believe linux correectly identifies.

The other ata 66 drive is installed on the non ata100 bus and that works
fine too.

I am using a default rh7.1 from a magazine cover rh distro (not a
an APC pocket guide, the other one - mainly orange cover)install using the
default mp kernel.

I am using a ps2 kb and a usb mouse, both work fine.

This mobo is dual cpu so the dual cpu support of the 694 chipset is being
exercised and does seem to work.

This is all rather general info, if you want specific info please let me
know and I'll try and find out for you, but generally speaking the system
seems to work under linux.

(supply a shell script if you like and ill send you the output).

Jason
