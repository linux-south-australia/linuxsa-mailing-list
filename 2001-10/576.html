Re: Tweaking Linux for performance

On Thu, 18 Oct 2001, David Fitch wrote:

> On Thu, Oct 18, 2001 at 12:33:09AM +0930, David Lloyd wrote:
> > I think I've heard it said that OpenOffice and Mozilla suffer at startup
> > partly because they're written in C++ (and setting up all the virtual
> > tables to support C++ objects takes a lot of time).
> 
> got any references for further reading on that? 

He is quite right.

There is a problem with C++ application startup on systems such as Linux
and FreeBSD that are built on the GNU toolchain. The paper
http://www.suse.de/~bastian/Export/linking.txt talks about the problem and
a specific hack the KDE people did to get around it while waiting for the
linker to improve. I think large C++ programs which do not use the KDE
libraries will still suffer from slow startup because I have not seen
anyone other than KDE with a fix. There is no kinit-type thing with
StarOffice 6.0 beta anyway. I have verified what the paper talks about
enough to satisfy me on Linux at any rate.

But application startup is the least worry in a system with low resources.
If every application took a little longer to start but ran really well I
imagine many users would be pleased. There are much more profitible places
to look first.
