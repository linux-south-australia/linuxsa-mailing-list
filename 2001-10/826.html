Re: Rebooting Linux (And other OS's) Regularly

On Thu, Oct 25, 2001 at 10:42:47AM +0930, Glen Turner wrote:
> Unfortunately, you should also have a scheduled attended
> reboot some time after hot swapping a CPU, memory or disk
> assembly. We've had cases of hot swapping working fine but
> difficulties being encountered at the next reboot.

yeah -- we used to schedule a reboot some time after doing anything
with hardware or anything with the boot process, so we could properly
test it... we had an instance once where an init script hung the
server if it was run during the reboot, but was OK if run manually
later. The cause of this turned out to be (IIRC) a timing issue with
that script and the previous database script -- the database wasn't
ready for it... It's the kind of problem you'd only discover on a real
reboot...
