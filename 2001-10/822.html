Re: SQUID + linux-2.4.5/XFS

Andrew Reid wrote:
> 
> We've been running Linux 2.4.5 + XFS patches on our web cache server for
> a while now. Since that's happened, we've been running into some wierd
> problems.
> 
> Every so-often, SQUID will hang -- hogging 99.9% CPU (according to
> `top'). You can't kill it in any way whatsoever -- the thing sits there
> in `R' mode.

> Has anyone experienced anything similar, either with XFS kernels or 2.4
> itself?

It might be a side-effect of using a journaling filesystem with Squid.

I recently converted 2 FreeBSD caches to linux 2.4 with ReiserFS for the
cache partitions (I kept hearing good reports about the networking code
in 2.4, but I don't see a lot of difference). Hits are a little quicker,
but CPU usage is way up for the same hardware. FreeBSD used to consume
no more than 11% on average, the reiser+linux combo hovers around 30%.

I'm also using diskd which I didn't before, which could also contribute
to more CPU usage.
