As DSL might say, it's important to get your partitions right before you begin.

Any user-generated data I create I put on a separate /home partition (with 
/usr/local symlinked to /home/local).  That way I can blow away any partition 
and install a new/different/updated RedHat/Debian/FreeBSD/Manduck at any time.

So as long as your "data" partitions are the only ones with stuff you care 
about, blow away and start again.  I don't think it'd be a good idea to try and 
install Debian on top of Red Hat (or anything else for that matter).

If you are going to reinstall, go to a journalling filesystem while you're at it 
:-)
