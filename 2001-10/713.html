Re: Mobile phone plans for remote monitoring...

This is pretty straightforward.

Yes, just find a plan that costs as little as possible, in terms of 
minimum monthly fee and cost per sms originated msg. I think we run 
ours on a plan that costs $10 per month and $0.15 per msg (with 
vodafone), there will be plenty of choices here.

We used to run a cobbled together phone-on-a-serial port. Ok, but not 
great, in reliability terms.

These days we use a purpose built device (GSM 'modem') - there are 
several vendors of same, they're not amazingly cheap (several hundred 
$) but they work so nicely that I'm very happy about them. They work 
using AT commands, just like a modem, with some additional AT 
commands for SMS message originate (and receive) handling.

They are just a power pack, a serial port (standard PC thingy) and an 
antenna jack (with a choice of 'internal' or 'external' grade 
antenna).

Lots of bits of software know how to drive them (including that gnu 
software thingy that knows how to drive mobiles, name escapes me 
right now). We wrote our own software in perl to drive it.

We also run these at our remote PoP's for dial-in access to our 
access server consoles. Absolutely wonderful - because it's out of 
band to the Telstra PSTN network, so if Telstra mess up big time, we 
can still dial in via the (non-Telstra!) mobile network we use and 
find out what gives. To do that, you just need a mobile plan that 
permits data dialup, and to ask your provider to enable it (they 
usually hand you a second mobile 'number' that this runs on, on the 
same SIM card). Same low cost plan, as you are dialling in from a 
land line so the phone still never places an outbound call. We have 
them all over the country :)

The one we run is an Siemens unit, called an M20:

http://www.ic.siemens.com/Daten/Produkt/2000/12/14/223055m20t_e.pdf

http://www.mobiledata.com.au/SiemensM20.html

Another popular one I'm aware of is the Falcom unit:

http://www.falcom.de/e/e-produkte.html

While NetSaint seems popular as something to do the actual monitoring 
with, at Internode we've been very happy with 'Mon' 
(http://www.kernel.org/software/mon/), which is nice, extensible (and 
we have done so), and has all the bells and whistles we need here... 
have used it for some years now.

Cheers,
