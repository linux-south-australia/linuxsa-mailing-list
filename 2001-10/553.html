Re: Tweaking Linux for performance

HI All

I spent time with Ian, working this thought, we shut everything down that
was not need and even shutdown KDE/Gnome, and ran blackbox which is about
the lights on memory windows manager around. I also twitched the IDE but
there was not much to gain there when from 2.5Mb/sec to 5Mb/sec.

It comes down to that OpenOffice and StarOffice used close to 50Mb of ram
leaving only 10 or so for the rest of the system. X windows uses close to
7Mb just for the graphics at 800x600x16.

The slow loading time is just a factor of the amount of data to load and the
processor speed.

This leaves very little left for using the software and an interface which
leave a little bit to be desired for a windows user.

There is just no way of making this work any faster, at least that I could
see.
