Re: [OT] IMAP and laptops

On Thu, 11 Oct 2001, Richard Russell wrote:

> when someone has a laptop, and they go away for a while, do things
> with their mailbox (perhaps deleting a few messages, moving a few,
> writing a few new ones and so on) and then they return and
> resynchronise, this has historically taken up to half an hour (even
> with smallish (~50Mb) mailboxes)... Personally, I put this down to the
> fact that our mail server is massively underspecced at present...
   :

What you didn't say was what IMAP server, nor how it is configured. 

If you are using UW impad with the default configuration of BSD-format
mail files, then you can expect poor performance under many circumstances.
If you are using UW with some other file format (which is rare, outside
large UW installations such as UW :-) then you will get better
performance. If you use Courier imapd then expect performance to be very
good for most things. Cyrus works well too, but in my opinion it has
fallen well behind Courier.
