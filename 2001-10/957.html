Re: bsd v gpl licensing

On Tue, Oct 30, 2001 at 01:00:43PM +1030, Dale Long wrote:

 > On Tue, 30 Oct 2001, Dan Shearer wrote:
 > > For nearly all users nearly all the time it doesn't matter. For a few
 > > developers it does matter, and companies built around selling software or
 > > software services. For people who like philosophical debates it does
 > > matter. But if you are a user of *Linux or *BSD it does not matter a bit
 > > to you, and you are most likely using large chunks of code under both
 > > schemes.
 > 
 > It might if Linux was BSD licensed, and MS decided to do their own MS
 > Linux distribution. And implemented embrace and extend. It would be
 > interesting to see what would really happen if that was the case.

The GPL doesn't stop MS from doing any of that.  They can release a 
perfectly vanilla source-code-included Linux distribution (which they
can "steal" from RedHat :-), and they can write a proprietary
closed-source kernel module which they'd sell from their web site,
which implemented the "embrace-and-extend" kernel bits.  Then they 
can "encourage" people to buy the kernel module by releasing Internet
Explorer and Microsoft Office for Linux, as closed-source applications,
with code which only allows them to run if the closed-source kernel
module is loaded.

Result:  An "embraced and extended" Microsoft Linux distribution which
every man and his dog would flock to use, fully GPL compliant.

So -- If this thread /must/ continue, can we please run it without any
silly misconceptions about the GPL preventing Microsoft from hijacking
GPL'ed software?  Thanks.
