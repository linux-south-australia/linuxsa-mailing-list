Re: Rebooting Linux (And other OS's) Regularly

On Wed, Oct 24, 2001 at 08:11:34PM +0930, Dale Long wrote:

 > What are opinions on rebooting a 'generic' server (say file and print, or
 > mail etc...) using Linux (or xBSD) on a regular basis? This would usually
 > involved a call to 'reboot' via cron early one morning every week.
 
You must be kidding.

  newton@olympia> uptime
  5:10AM  up 853 days, 16:27, 1 user, load averages: 0.04, 0.03, 0.02

 > I know the question is very generalize, but the scope of its application
 > and usage is also general.
 > Is it good practice and a necesary 'evil' to keep a box (and potential
 > memory leaks and so on) available? Or is it a lazy alternative to creating
 > a stable and well spec'ed setup?
 
Definitely not.  If you have an operating system which can't continue
to provide service without being rebooted all the time, then you have an
operating system which is in dire need of replacement.  

Think about it:  Why would you induce instability (i.e., reboot) to
protect yourself against instability (i.e., crashes)?

 > What experiences have people had (any OS, any hardware) where it would be
 > better to reboot regularly, or leave it running until the next upgrade?
 
Definitely leave it running.  If you can't leave it running, upgrade it
straight away.

 > Is it better to have a machine reboot regularly during a quiet period (not
 > talking 24/7 access service level here) vs a slow-down and eventual hang
 > during working hours (say middle of the day in a finance departemnt just
 > before new financial year) due to a problem that built up over several
 > months (bad process table etc...)?

If you have a system which slows down and eventually hangs during working
hours, that's indicative of a major and fundamental design fault with
either the OS or the application you're running.

You need to ask yourself the question:  "Do I want to remove this problem,
or do I want to put up with the unreliability and inconvenience of
regularly rebooting for the rest of the system's life?"

If you choose the latter, you might as well be using Windows for all
the difference it'll make.

 > I know there are different requirements for different situations and
 > scopes of work, but I am looking for an overall picture.

Perhaps I'm overly idealistic, but I don't believe there /are/ different
requirements for different situations.  I expect 100% reliability.  The
only explanation for anything less than 100% reliability is a bug, and
bugs need to be fixed.
