Re: changing a Mb value to Kb value

Jason Tan <jason@rebel.rebel.net.au> wrote:
> Just because people choose to write difficult to read perl ins ta
> function of the language.

That's right; but a more reasonable statement, "Just because people
write difficult to read perl isn't a function of the language," is
wrong.  The design of some languages leads to easily read programs.
Cobol springs to mind.  Other languages inspire cryptic code.  APL
is a good example of such a language.  That's not to say you can't
write good code in a cryptic language, or that you can't write ugly
code in a more readable language.  To quote from "Real Programmer's
Don't Use Pascal": Real Programmer's can write Fortan in any language.
