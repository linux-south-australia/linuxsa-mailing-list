Re: bsd v gpl licensing

Mark Newton wrote:

> But the developers who wrote that BSD code don't mind.  Isn't that
> what's important?  They're the ones who chose the license, after all.

Agreed, It is up to the developer. 

Sometimes it's just fun to give away code and see people use it, and the
only way you can truly do this is to use a BSD or public domain licence.

If you can put aside objections to Microsoft profiting from BSD code for
a minute, it would be pretty cool to know that millions of people
worldwide are benefiting from your work, even if they did make a poor
choice of OS.

Once upon a time I wrote a web utility for a proprietary http server and
released it and future upgrades for free. It was very satisfying to find
that the utility was being used on around 90% of sites capable of
running it.
