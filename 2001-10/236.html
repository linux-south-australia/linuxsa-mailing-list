Re: Kernel 2.4.10 compilation on Mandrake 8.0

Joseph!

> I find that on my own machine, if I didn't run lilo after a kernel
> compilation, more likely than not he new kernel would simply hang. This
> occured even with kernels of the same version that I've simply tweaked, no
> file name changes in /boot were done. I can't explain this, but if you find
> that your own kernels are giving you a blank screen, type lilo at the end
> and reboot.

If I read my documentation correctly, the boot loader lilo (and possibly
other boot loaders) - at some stage - must know physically where one of
the kernel stage images is on the disk. When you install a new kernel -
whether it has exactly the same name or not - it may not live exactly
where it was before. 

You could, naturally, be lucky and not have this problem as you have
noted by saying "more likely than not".

Advice: if using lilo and rebuilding kernels, always run lilo after
fiddling with /etc/lilo.conf or anything in /boot :-)
