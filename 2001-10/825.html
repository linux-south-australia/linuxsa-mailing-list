Re: Rebooting Linux (And other OS's) Regularly

> On Wed, Oct 24, 2001 at 08:11:34PM +0930, Dale Long wrote:
> > What are opinions on rebooting a 'generic' server (say file and print, or
> > mail etc...) using Linux (or xBSD) on a regular basis? This would usually
> > involved a call to 'reboot' via cron early one morning every week.

Just a few thoughts:

Cobalt servers run almost indefinately thanks to linux, but have a weird
OpenSSL bug that requires a reboot to make a new security certificate
active. No amount of starting and stopping the web server gets around
this.

The more recent versions of FreeBSD have a "kern_securelevel" option in
the bootup that makes some applications behave differently.

With this set high some more security-concious applications like
ipfilter refuse to re-read their config file, and require a reboot to
make changes. There's probably a way around this with init levels, but
rebooting is easier and has a similar effect on network services :)

Some OS-independent Raid systems cannot report disk status except
through their BIOS interface either.
