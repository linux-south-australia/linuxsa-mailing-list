Re: Tweaking Linux for performance

ilox wrote:
> 
>  The computers are generally running Intel 200 MMX CPU
>  with 64 M RAM and a 4 G drive.

That should be slow but useable.  All UNIX tends to
be I/O bound on modern hardware, so I'd check that the
drives are configured for 32bit DMA.  Have a look in
/etc/sysconfig/harddisks and check USE_DMA, EIDE_32BIT,
MULTIPLE_IO, READ_AHEAD.

You might also want to try a less greedy window manager
like fvwm95.

VM performance under Red Hat Linux currently sucks, the
newer Linux-series of kernels claim to have solved this.
Could be worth building a recent kernel.
