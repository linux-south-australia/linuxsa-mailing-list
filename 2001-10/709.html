Re: Linux vs. Microsoft requirements (was: [OT] Should I beworried...)

At 11:32 AM +0930 18/10/01, Greg Lehey wrote:
>Apart from that, of course, he's just plain wrong.  I believe it's
>still possible to get Linux to run in 8 MB of memory, for some
>definition of run.  I don't think that you'd get Microsoft to perform
>anywhere near as well on the same minimal hardware.

And it's quite practical to run Linux microkernels to allow a 
functional multitasking system to exist inside a single low cost 
chip. Try http://www.netsilicon.com for an example of this.

Try that with Windows XP (not).

Meanwhile those specification comparisons are just nonsensical.

This is because the thing that matters isn't the minimum 
configuration on which the animal will boot up, what tends to matter 
is the minimum configuration for which the -applications- the punter 
wants are running viably fast.

We've all seen statements at the release point for operating systems 
(from lots of vendors) which quote laughable levels of cpu, ram and 
other resources as minima to run the OS. Typically if one doubles the 
ram and goes two notches up on CPU, it all starts to get 'nice'. The 
greatest evil for virtual memory systems is lack of enough physical 
memory to avoid swapping.
