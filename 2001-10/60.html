Hi,

I've done this before. The particular setup used
RedHat6.2 but it should be the same or very similar
on other distros.

What follows is a rough description which lists some
of the main configuration files - you should be
able to find a detailed description on the web
somewhere, and I'm sure others on this list will do
better than me at describing this


> 
> My Lan needs to be able to communitcate with the
> server I have dialed into
> and vise versa.
> 
> [ LAN 1 ]     ------------   SERVER ----------- My
> Server --------------[
> LAN 2]
> 
> Everything needs to communitcate.

There is a package called mgetty+sendfax - you need to
make sure that is installed, and configure a line in
/etc/inittab to work on the serial port where your
modem is. You also need to add an /AutoPPP/ line
to /etc/mgetty+sendfax/login.config to activate PPP
for the connection. 
Other files that will need to be modified include
/etc/PPP/options and /etc/mgetty+sendfax/mgetty.config

Other things to consider include using ipchains or the
latest equivalent to block / allow ports if needed or
to forward on to other subnets. At the least you
probably need to enable IP forwarding. Other issues to

consider include using SSH for security, etc...
