Re: Kernel 2.4.10 compilation on Mandrake 8.0

Hiya,

I think half the trick to getting a successful compilation going is to
know exactly what drivers you need for the services that you are
planning to run (eg NetFilter, networking), and the hardware that you
have.

I've had good experiences with a stock 2.4.12 kernel on Mandrake 8.1
(after you fix up the parellel port bug). I'm trying out 2.4.12-ac3 with
the new preemptive patch.

It is very easy to break something and create a non-working kernel. Go
for a conservative setting then cut it down if you want.

What confuses me is why certain modules don't start automatically when
they should. For instance on my notebook, PCMCIA support will not start
unless I compile it in as a kernel not as a module.

To be honest the only reason whay I do kernel compiles is to try out the
latest and best patches (such as the preemptive kernel stuff) on my home
machine. Even if its compiled on a 686 machine I don't think there is a
significant improvement in perofrmance over the 586 kernel shipped with
Mandrake.

Cheers,
Joseph Tan
