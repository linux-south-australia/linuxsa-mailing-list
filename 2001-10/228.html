> I might be stating the obvious.. but.. its worth a shot.. hda1 = first
> partition on your hard drive.  If you want the boot loader installed in your

Here goes again.

Although it is technically possible (using devfs and devfsd and a
particularly weird mapping) to get a SCSI drive to appear as /dev/hda or
its partitions as /dev/hda# (where # is a partition number), last time I
looked SCSI devices appear as /dev/s[cryptic letters after this], not
/dev/h[cryptic letters after this].

So, given that I know that Robyn is not running a whacko version of
devfs with such a whacko naming scheme the hard drive she wants is NOT
hda...nor any of its partitions. Besides which, how can hda1 be the
first partion on my hard drive which happens to be /dev/hdb?

> You'll probably need to recompile the kernel.  I've never actually used a

> which loads scsi support via. modules - which of course it cant load,

I think I've already said (but in private) that Robyn will have to work
out how to boot using either:

1) a kernel with SCSI support built in
2) a kernel + an initrd with appropriate modules thereon

Given that you'd probably be using the SCSI driver continuously if it is
the boot device it makes no sense to build it as a kernel module (i.e.
it's going to be used almost all the time anyway, so why bother about
unloading and unloading it all the time).
