My SAMBA Quest...

I finally found the solution to my SAMBA problem. I'd been playing with
SAMBA on and off for around 5 months but I've never been able to
successfully connect to it via any Windows machines.

The trick was simply not to enter any log in information and hit enter
then I was able to see my shares on the connecting computer!

This WAS simply not obvious in the documentation since they all
mentioned a "guest" account, and in Windows 2000 I connect to SMB
machines by giving the log in username "guest", and no password.

God knows why SAMBA doesn't require ANY log in information.

Now I can see myself plugging away getting the printer shares working
and making the SAMBA Linux box viewable under network neighbourhood for
the foreseeable future.

Two questions:

- Should the Linux machine be viewable in Network Neighbourhood? All my
server settings for SAMBA are at default.

- Is there a tool availible in Linux (Mandrake/RedHat) to allow you to
browse through computers in a Network Neighbour fashion?
