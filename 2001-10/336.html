Re: best filesystem type for sharing data between Linux and Windows?

On Thu, 11 Oct 2001, Michael Wardle wrote:

> I'm still baffled why one FAT32 partition works, and the other doesn't.

You could always cheat, and create a data partition the same size as the 
working Windows partition and do a 

   dd if=/dev/hda1 of=/dev/hda2

or whatever your partition names are to duplicate it exactly. Then you
just have to delete a whole lot of files on the new data partition and it
should work. This doesn't solve the problem (although I still think
mkfs.vfat might help, the point is what you did looks like it should have
worked in the first place.)

To complete the cheat, you could then use fips or parted to resize the
data partition, assuming it is too big. This is all a bit messy :-)
