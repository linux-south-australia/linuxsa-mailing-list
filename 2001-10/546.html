On 17 Oct 2001, Anthony Symons wrote:
>
> Modify /etc/httpd/conf/httpd.conf
>
> if using apache find the section with:
>
> MinSpareServers 1
> MaxSpareServers 2
>
> StartServers 2
>
> and do something like that. With the default of 150 clients per server
> that'll still give you capacity (in theory) to serve 300 clients
> simultaneously, but you'll save a few meg of ram per instance, and when
> it defaults to 10 instances that should help a fair bit.
>

ummmm
i think your logic is a bit messed up there?

Correct me if i'm wrong, but you can serve 1 client per server,
simultaneously.

2 servers == 2 clients (or requests) simultaneously.

Apache will ramp up the number of running servers as required, however.

The 150 clients is the maximum number of servers to run.. once that figure
is reached, apache won't spawn more servers, even if they are all in use
100% of the time!

(this is to prevent a denial-of-service attack from making thousands of
really slow simultanous requests, thus spawning 1000xApache's and using all
the memory up)
