>>
>> I think what you'll find is you're hitting a limit of Windows not 
>> Samba.
>> The user profile usually contains the user registry entries (in a 
>> .dat file), Temporary Internet Files, various application specific 
>> settings and temp access areas. Therefore the clients need write 
>> access to this area. Furthermore, unless they're all logging in with 
>> the same username, you can't share a single profile over disparate 
>> users. Again, this is a limitation of Windows not Samba.
