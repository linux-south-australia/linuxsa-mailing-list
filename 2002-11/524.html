Re: Why have a web site?

Michael Kratz wrote:

> Check out http://adsl.internode.on.net


Hmm, not too bad, but it still contains crud like [reformatted for clarity]:

<div class="navlinkside">
   <p>
     <em>
       <font color="#FFCC00">Other Sites:</font>
     </em>
     <br>
     <br>

     <a href="..." class="navtextside">Internode Dialup/ISDN</a>
     <br>
     <br>

     <a href="..." class="navtextside">Internode Data Centre</a>
     <br>
     <br>

     <a href="..." class="navtextside">Agile Communications</a>
   </p>
</div>

First, the inner <p> is intially superfluous, it should get ditched. 
Next, the <br>s should be replaced by <p>'s around the links. Some style 
would also be needed to get the <p>'s margins the same as a double-<br>. 
   Lastly, the font tag should be sttched in favour of a class, id, or 
style attribute on the <em>. So, to do it better, you'd end up with:

<div class="navlinkside">
   <p><em class="navlinksideheading">Other Sites:</em></p>
   <p><a href="..." class="navtextside">Internode Dialup/ISDN</a></p>
   <p><a href="..." class="navtextside">Internode Data Centre</a></p>
   <p><a href="" class="navtextside">Agile Communications</a></p>
</div>

This is good, because it will actually make some sense on older 
browsers, but it's still not perfect. What we really have here is a 
heading, and a navigation list. It really should look like:

<div class="navlinkside">
   <h2>Other Sites:</h2>
   <ul>
    <li><a href="...">Internode Dialup/ISDN</a></li>
    <li><a href="...">Internode Data Centre</a></li>
    <li><a href="...">Agile Communications</a></li>
   </ul>
</div>

This then gets styled using something like:

/* style to make our h2 look like the font'y em */
div.navlinkside > h2 {
   font-face: italic;
   font-size: 1em;
   font-weight: normal;
   color: #ffcc00;
   /* etc */
}

/* style for our links */
div.navlinkside > ul {
   list-style: none; /* remove the bullets */
   margin-left: 0;
   padding-left: 1em;
   /* etc */
}

The end result looks exactly the same in a modern browser, and gives 
users of older browsers at least as much (if not more) of an idea as to 
what is going on.

So yeah, DMX's code isn't too bad, but it could be a lot better.
