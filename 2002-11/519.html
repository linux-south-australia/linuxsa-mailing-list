Michael Kratz wrote:

>
> Geepers, that doesn't leave much for design then! :)

Not at all, have you seen the size of the CSS2 spec? :)

> I agree with not using SPAN tags but DIV tags can be handy.

Yeah, there are situations where you just can't get away with anything 
else. Still, avoid {over|mis}using them. A common example is using 
something like:

<div class="heading">My document's title</div>

and using CSS to style that div to look like a heading, instead of:

<h1>My document's title</h1>

and styling the h1 to look how you want. The former is meaningless 
drivel, the latter is a nicely styled heading, even if they look the same.

> Not using tables for layouts, hmm. well not sure on that one, I use 
> tables,
> the only reason I haven't migrated to CSS is because of backwards
> compatiblity, a lot of old browsers don't like the new css tags.

Yeah, but they're all a lost cause anyway. If you follow the best 
current paractices, then your pages will at least still be readable in 
old browsers. Bear in mind that a lot of older broswers don't support 
tables properly, either. In these cases CSS-based layout degrades well, 
table-based layout does not.

Check this out: <http://www.meryl.net/css/>;. 1000 web sites which don't 
use tables for layout. It's a bit out of date, but hey.

> and HTML 4.01 Transitional seems to work just fine with 99% of browsers.

Yeah, but -transitional is just -strict with all the extra, bad, 
proprietry crap left in. If you're not using junk like <font> then you 
don't need to use -transitional.
