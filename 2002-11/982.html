Re: Kernel buffering

On Fri, 2002-11-22 at 16:42, Matthew Geddes wrote:
> This is on the list of things to consider, but I had thought of it before and
> seems a little painful.

It's not so painful.  For writing you need only keep track of the last
block: When it's full you allocate a new block, storing it's address in
the newly filled block's next pointer, and then again in the global
current-block pointer.  You can make the task a little easier by having
a circular chain, and adding elements only when writeblk->next ==
readblk.  Release all blocks when the device file is closed, thus the
algorithm allocates blocks as needed, and releases them all only when
the user process quits.
