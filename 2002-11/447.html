Re: busy inodes

David,

> Just because you can eject removable media when it's mounted doesn't
> mean you should.  In fact you should not.  You should always unmount
> them (with the umount command) before ejecting them.  You might find
> that umount fails with "filesystem in use."  That's not a problem with
> unmounting, but occurs because you are using files on the media you are
> trying to unmount.

I agree.

I've been playing around with the Linux Kernel automounter (the userland
service is called "autofs") and it's very nifty. For example, one thing
one can do is tell it to unmount a filesystem within a certain timeout
(defaults to a minute) if nothing is using that filesystem.

Hence, it makes this situation less likely to happen.
