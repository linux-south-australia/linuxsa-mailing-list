Re: Mozilla 1.0.1-2.7.2 woes

On Fri, 2002-11-22 at 14:06, Graeme Jensen wrote:
> I'm seem to be still having problems with the rpm packages using RH7.2.
> After installing new packages using up2date (messages at the very end) I
> cannot get mozilla to start.
> I get this when running strace mozilla, after lots of output.
> 
> unlink("/home/Lesley/.mozilla/default/ycmeibwt.slt/lock")=0_exit(11)
>
> Can anyone tell me what this means?
Well, unlink is delete, so its deleteing that lock file, and getting
exit code 11, whatever that is. 

Check if that lock file exists, and if your user can delete it or not.
Then delete it yourself anyway since it sounds like its just a stale
lock.
 
> This is the second time I've had this happen.  Last time I tried deleting
> and reinstalling the mozilla packages.  I ended up with a corrupt rpm
> database and losing the GUI.  I've now reinstalled the whole system (RH7.2)
> and want to get my system up to date.
> Here are the messages I got after running the up2date command and
> successfully installing packages.  I don't see anything here that indicates
> potential problems for mozilla though.
> 
> warning: /usr/share/ssl/openssl.cnf  created as
>         /usr/share/ssl/openssl.cnf.rpmnew

This is normal - its just not trashing your config, while providing an
upgraded template for you.

> error: unpacking of archive failed on file
> /usr/lib/gcc-lib/i386-redhat-linux/3.0.4/libgcj.a;3ddcbc02:cpio:MD5sum
> mismatch

This is bad. This means that your RPM is corrupted, or at least the file
was after it was written to disk for some reason. I belive if the RPM is
corrupted then it'll warn you before it starts installing - not 100%
sure on that though. Could be a hardware problem, or obscure bug that
just you are getting? 

Try grabbing the rpm manually, checking its md5sum, and installing that.
