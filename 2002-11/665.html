Yep, it's a quote. Similar to a double quote ("), but stricter. It
causes the shell not to do any interpretation on the contents -- it
treats the contents like a single argument.

man bash will tell you more.

as egs:

---
# VARIABLE=blerk
# ls a file $VARIABLE
a: No such file or directory
file: No such file or directory
blerk: No such file or directory
# ls "a file $VARIABLE"
a file blerk: No such file or directory
# ls 'a file $VARIABLE'
a file $VARIABLE: No such file or directory
#
---

Note the differences... (there's a bit more to it than that, but you
can read that in the man pages)
