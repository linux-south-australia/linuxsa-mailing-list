Re: LDAP issues

Andrew Galdes wrote:

> Can someone help out with the following examples?
>
> dc=companyname,dc=com,dc=au cn=andrew
>
> So it might look like:
>
> ldapsearch -H ldaps://ldapserver.companyname.com.au
> dc=companyname,dc=com,dc=au cn=andrew


Okay, you probably don't want to use SASL for authentication, so use a 
"-x". You probably also want to specify the search root with -b, which 
will probably be "dc=companyname,dc=com,dc=au". if you need to login, 
specify the username/dn with -D, and get it to ask you for a password 
with -W.

So use either:

ldapsearch -x -H ldaps://ldapserver.companyname.com.au \
   -b "dc=companyname,dc=com,dc=au" \
   cn=andrew

for no authentication, or:

ldapsearch -x -H ldaps://ldapserver.companyname.com.au \
   -b "dc=companyname,dc=com,dc=au" \
   -D "cn=andrew,dc=companyname,dc=com,dc=au" -W
   cn=andrew

To log in as "cn=andrew,dc=companyname,dc=com,dc=au".
