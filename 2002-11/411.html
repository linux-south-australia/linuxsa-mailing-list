Re: Html email gets a reply

Pigeon <jah.pigeon@ukonline.co.uk> wrote:
> Sorry. Totally disagree. The vast majority of home internet
> connections are via dialup and are therefore too slow for anything but
> text.

The interesting thing about his comment is it ignores the "T" in HTML.
HTML is text.  It contains no images, music, or any media other than
text.  HTML is text with markup that provides some basic formatting and
expression which otherwise is missing from plain text.  It's true that
HTML can express loading of all those media that I claimed it contains
none of, but you can (or should be able to) turn off loading of graphics,
sound and such.  You don't need to enable cookies, so email tracking
need be no problem, it's searchable, and, as I said, it's very expressive.

Aren't asterisks, slashes and underlines for expression all so last
millenium, particularly when HTML lets you emphasise in <b>*bold*</b>?

At the end of the day, all of the arguments that I have seen concerning
why not to use HTML are shallow attempts to disguise poor use of the tool
as defects in the tool.  My answer to that would have to be a fifty line,
ascii art signature.  That, and that the arguments, coming as they mostly
do from grey beards, is "crusty old fart."
