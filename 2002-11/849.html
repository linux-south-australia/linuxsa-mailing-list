apache weirdness

I have a pinup-board for sharing files between the two locations that my
laboratory is in (ANU and Adelaide Uni, and restricted to clients within
those domains). Recently (I'm not sure exactly when this happened since it
isn't used all that frequently), the server has responded with Forbidden
when trying to acess the page - commenting out the `Deny from all' line
recovers access, but we don't want global access to this page.
I've had a look through the Apache 1.3 docs (running 1.3.26 under Woody),
but it appears to be as it should, and it _was_ working before.

Any ideas?


the relevant section of httpd.conf:

#pinup-board a bit private
Alias /pinup-board/ /home/ftp/pub/incoming/pinup-board/
<Directory /home/ftp/pub/incoming/pinup-board/>
    Options Indexes MultiViews
    AllowOverride None
    Order deny,allow
    Deny from all
    Allow from molecularbiosciences.adelaide.edu.au
    Allow from molecular-biosciences.adelaide.edu.au
    Allow from anu.edu.au
</Directory>
