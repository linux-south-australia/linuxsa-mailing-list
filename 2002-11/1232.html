RE: automating netbios names for domains

>Is there a way to automate adding netbios names from win clients to a
>SAMBA box.  It appears that the machines names have to be 
>added manually
>to the passwd file.  Is there a script, or other way of doing this.  We
>have about 250 netbios names to be added to a redhat box (+users homes

If you have all the machine names in a text file the easiest way is to 
run it through an if statement with smbpasswd.

You should be able to do something similar with the users & their home
directories by dumping the usernames into a text file and using a script
to create the home dir, chg permissions, blah etc.  (Very nice to show an
NT admin that you can create x number of users + home/profile directories
in a matter of seconds, I still cringe when it comes to user management on
NT boxes, to say it sucks is an understatement.)

If you'd like some examples let me know, we have a custom command setup 
in webmin for our samba boxes for adding machine a/c and users.  Its simply
a web frontend with a couple of dialogs to interface the CL script, but
it satisfies the GUI addicts.... ;)
