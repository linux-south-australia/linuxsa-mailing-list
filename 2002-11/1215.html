Re: apache reaches MaxClients setting loading a simple page

1. It is possible that you've been under DOS attack
check   netstat -an |grep SYN_RECV 
If so - consider enabling SYN COOKIES for basic protection

2. Too many concurent connection ( max in you conf is 250 )- it just 
warns you
If so - compile apache with bigger "maxclients" :
untar, then edit

vi +/HARD_ /usr/src/apache_1.3.27/src/include/httpd.h
change the value - #define HARD_SERVER_LIMIT 256
then compile

( httpd -V gives the compilled value)

hope it helps :)
