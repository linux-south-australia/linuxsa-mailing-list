Re: Identifying which packages various files belong to

Quoting Andrew Lord <andrewlord@internode.on.net>:
> Sometime ago, mention was made of a shell command that could be used to 
> identify which package a particular file belongs to.  I'm in Mandrake 9.0 and
> 
> trying to install ktelnet the usual way (ie rpm -ivh ktelnet*) but it fails 
> with the following:
> 
> error: failed dependencies:
> 	libDCOP.so.1 is needed by ktelnet-0.7b1-13mdk
> 	libkdecore.so.3 is needed by  etc etc
> 	etc
> 	etc
> 
> I just want to identify which packages these files belong to so I can install
> them.  Can anybody help me please ?

I can't speak for Mandrake, but under Red Hat you would do the following:

To find a file:

rpm --redhatprovides libDCOP.so.1 (as long as you have rpmdb-redhat*.rpm 
installed)

or to install apt-get like:

up2date ktelnet-0.7b1-13mdk

which will install the package and any outstanding dependencies.

Why is this relevant?  Mandrake was originally based on Red Hat, so something 
similar may be available or implementable.
