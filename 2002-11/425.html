Re: Html email gets a reply

Justin Hawkins wrote:

> HTML is bad for email. It's too easy for webbugs to be embedded in
> message, too easy for you to be maliciously tricked into visiting
> somewhere you didn't want to go (goatse.cx anyone?) and I think worst of
> all, most HTML mail readers use the local web browser rendering engine.
> The bugs in those are bad enough for general surfing, but in your

how they can be bypassed makes you wonder why Microsoft bothered in the
first place (ooh, I know!!  "token effort" read "Microsoft Security
Initiative").
