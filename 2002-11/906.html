DHCP problem

Hi all,

This is kinda off-topic, being on a Solaris machine, but it's close
enough to be relevant (IMHO).

I've got a machine running Solaris that uses DHCP for its primary
interface. All was running OK until yesterday, when I got a large
number of messages like these:

Nov 20 16:24:26 <host> /sbin/dhcpagent[394]: [ID 410337 daemon.error] IP
address already in use by <XX XX XX XX XX XX>
Nov 20 16:24:26 <host> /sbin/dhcpagent[394]: [ID 220039 daemon.warning] dhcp_acknak: dhcp_bound failed for dmfe0

The MAC address I XX'd out above, strangely enough, is the MAC address
for the very interface it's trying to get an IP address for. It looks
to me as if it's saying "I can't use that IP address because someone
(me) is already using it". I've contacted the DHCP server
administrator, and the IP address is not being used by anyone else,
and the logs there say that my host is rejecting the address offer.
What else could be the problem here? I don't know enough about DHCP to
know what's going on.

Also, I've tried setting the address manually, turning dhcp off and
just using ifconfig. It appears to be configured, but I can't ping,
and the route table doesn't include anything except the loopback
route.

Any ideas?
