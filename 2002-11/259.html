> 
> If you (for example) had a machine with two network interfaces and were
> trying to talk to one from the other. The source IP would probably be 0.0.0.0
> (which is similar to the loopback address on many systems):
> 
> [me@moocow tmp]$ ping 0.0.0.0
> PING 0.0.0.0 (127.0.0.1) from 127.0.0.1 : 56(84) bytes of data.
> 64 bytes from 127.0.0.1: icmp_seq=1 ttl=255 time=0.217 ms
> 64 bytes from 127.0.0.1: icmp_seq=2 ttl=255 time=0.141 ms

Yes, something else I did not consider is that 0.0.0.0 means this machine, 
so the response you see above may be the Samba server requesting the IP 
address of the Local Master Browser, and being told that it is the LMB.

One of the fastest ways to telnet to your own host is:

  telnet 0
