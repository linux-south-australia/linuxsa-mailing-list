Re: Compiling problem

Quoting Michael Selge <mselge@hansenyuncken.com.au>:
> Trying to install Mnogosearch on RH 7.1 and I'm running into errors. My SQL
> is installed and working. When I try and compile I get this output
> 
> checking for MySQL support... no
> configure: error: Invalid MySQL directory - unable to find mysql.h
> 
> OK..so I then do a search for mysql.h
> and it finds it here
> /root/php-4.2.2/ext/mysql/libmysql/mysql.h
> 
> so I change the compile option to
> 
> ./configure --with-mysql=/root/php-4.2.2/ext/mysql/libmysql/
> 
> and I still get an error
> 
> checking for MySQL support... no
> configure: error: Invalid MySQL directory - unable to find mysql.h under
> /root/php-4.2.2/ext/mysql/libmysql/
> 
> 
> What am I doing wrong ??

Configure is just a script, so you can read it and debug it yourself - but I 
think your application is wanting MySQL, not php's interface to MySQL.

Ahhh, hang-on, what RPMs do you have installed?  To build other software that 
depends upon MySQL, you'll need the mysql-devel-*.rpm package installed - that 
will give you /usr/include/mysql/mysql.h (hopefully :)  Install that and you 
shouldn't even need to specify a prefix for configure as /usr/include is 
somewhere configure checks automagically.
