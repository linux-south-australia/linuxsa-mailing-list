The point of the program no-ip.com give you is that it's a background daemon that only renews the IP when necessary.
It doesn't need restarting every time you dial-in. You run it once.... then forget about it.
You only need to cron it if you're paranoid.
I've only had it stop on me once in nearly two weeks of uptime after dozens of dialins.
I've used other dynamic ip hosts that provided a perl script to be run at each dial-in. 
Depends who you host with, and whether it's for commercial sites as to the costs.
