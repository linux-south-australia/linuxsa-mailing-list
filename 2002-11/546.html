Databases, Perl and DBI

I'm musing here, but I was thinking:

"Wouldn't it be nice if I had a DBI wrapper that intercepted any SQL
statements and changed adjusted their syntax from one SQL database to the
other?"

...something a little like the _po_ library in Gnome.

I see this useful as a preloaded DBI module in Apache where one might want
to use Bugzilla, which seems only to speak to MySQL [from what I could
find], against Postgres and not MySQL.

I'd envisage something like:

use DBI::My2Pg; 

...which then intercepted any calls to the DBD drivers for MySQL and
transparently reworked them into calls to the DBD drivers for Postgres.
And fixed differences in the SQL as best it could.

I wonder if this has been done yet.
