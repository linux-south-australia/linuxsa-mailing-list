Luke,

    Unless I am misunderstanding you the "PATH" in your above example is 
what you are searching for in the specified file that follows.  The 
specified file that followed was of course "." or "$HOME".  Now  getting 
back to your statement above about the man page not showing that you put 
the path before the file is not true using grep the way you are, so that 
is why it would not be in the man page.  The proper syntax for grep in 
your case is `prog <options> <what you are searching for> <what you are 
searching in>` , and an example of that would be: `grep -r PATH $HOME`.  
That would return to me any file that contains the word "PATH"(case 
sensitive) in the directory of "$HOME" and in any other directory below 
that.  I hope I didn't misunderstand your statement above.  If I did 
then just ignore me (:
