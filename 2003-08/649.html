Re: [Monday] Good news for all IT workers who know a lot about humangenes

Are you sure that you're not taking that quote out of context? Its 
incorrect to think of human evolution progressing while other species 
remain static.
This probably seems obvious, but would mean that we probably wouldn't 
find a redundant gene present in a human 'from' a mouse.

The other thing is that DNA doesn't really code how we're made, as such. 
It codes for proteins, which in turn build other structures, which 
combine with other proteins (recursively) again and again, to eventually 
produce what we recognise... thus, the operation of DNA is massively 
more complicated than mass-media would have us believe (particularly 
when they're telling us to accept/buy/eat the hacks that they call 
'genetic engineering'), and it's entirely possible that the 'junk' code 
is vital to the correct operation of DNA.

FWIW, I remember editing a MSDOS binary file (a game - in a text editor 
*sigh*) when I was about 10, so that it wouldn't work (a single 
character change). When I undid what I had done, and saved the file 
again, I couldn't understand why it didn't work... dunno if this is a 
simile or not  ;-)

-Gus
