I've updated my web page.
http://www.lemis.com/grog/SCO/code-comparison.html.

In a nutshell:

- For about 16 months, there was a copy of UNIX malloc() in the Linux
  tree, in the ia64 branch (where nobody used it much).
- It was removed because people thought the code wasn't good enough.
- There's no question that it was derived from System V.4.
- However, the differences from, say, the Seventh Edition are minimal.
  In the FreeBSD project, we'd give the sort of work (changing from
  the 7th Ed to SVR4 versions) to a junior kernel hacker, and we'd be
  upset if he didn't come up with code identical to SVR4 without
  seeing it.
- This code is over 30 years old.  It was present in substantially the
  same form in the Third Edition (20 January 1973).
- BSD and most commercial UNIXes have replaced it with something
  better years ago.
- This is the first time we've seen evidence of SCO's "Enterprise
  quality" code.  I think it backfired.
