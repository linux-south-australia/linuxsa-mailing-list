Re: [BUGA] Re: pf/altq under FreeBSD?

> Actually, you _can_ do this with NetBSD - there is a sysctl variable
> (net.inet.tcp.mss_ifmtu) which will force the MSS to be the MTU of the
> interface.  This will work ok for packets originating from that
> machine but if you are using the machine as a gateway/router then you
> need to put a NAT rule in (even if you done translate the address..)
> that will clamp the MSS for you.  All this is builtin to the OS.

Sure you can do it with Linux, but the patch allows you to apply setting the 
MSS to anything you can describe in netfilter rules, so potentially packets 
to a single port on a single host.
