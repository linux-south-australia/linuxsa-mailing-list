Oracle of Oz,

> also, the 128 is a swap partition, the other 100 is two swap files,
> and it seems the files are significantly slower... is there perhaps a
> way to make linux use the partition to its fullest before using the
> files?

man swapon tells us:

- --
- -p priority

Specify priority for swapon.  This option is only available  if swapon 
was  compiled  under  and is used under a 1.3.2 or later kernel. 
priority is a value between and 32767. See  swapon(2) for  a full
description of swap priorities. Add pri=value to the option field of
/etc/fstab for use with swapon -a.
- --

man 2 swapon tells us:

- --
Swap pages are allocated from areas in priority order, highest priority
first. For areas with different priorities, a higher-priority area  is
exhausted  before  using  a  lower-priority area.  If two or more areas
have the same priority, and it is the highest priority available, pages
are allocated on a round-robin basis between them.
- --

Therefore, find your /etc/fstab and have something like this:

/dev/hda5  none  swap    sw,pri=100  0  0
someswapfile none swap sw,pri=50 0 0
...

As for your other questions, it makes no sense to swap pages out to disk
if the operating system doesn't need to. It can't really guess what you
want to do.
