> Using netstat -anp displays:

I forgot to mention you can use the --inet option to just get the
internet
parts of that, and not the Unix sockets...

ie netstat -anp --inet

> So it looks like xinietd is running but not listening on the port.

In that case, you need to get xinetd to reload its config files...

/etc/init.d/xinetd status (to check if it's really running)
/etc/init.d/xinetd reload (to make it reload the config)

Then try ftp again, and try netstat -anp --inet again to see if there's
any
indication that xinetd is listening on the ftp port.

> I have also chkconfig xinetd on and resarted the service. 
> Still not good.

check it's running with the status line above, or `ps -elf | grep inet`

If it's running and still doesn't work, it's time to try something
different
-- I'd reccomend briefly unplugging your network, /etc/init.d/iptables
stop
(ie flush your tables and set them to ACCEPT), and trying again. If ftp
is
running, it should work now.
