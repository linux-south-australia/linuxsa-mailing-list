Re: Xf86 stumbles on new kernel

On Fri, 2003-08-08 at 10:04, Brian Marr wrote:
> 
> 
> Suse 8.0 box.
> 
> I am having trouble booting into X with a new 2.4.21 kernel from kernel.org. 
> 
> Going back further up the chain....lilo normally shows vga = 791. If I leave
> this parameter in lilo, I get a blank screen as the kernel boots up. If I
> remove this line, the boot process is displayed on the screen in larger than
> normal text. It seems the display may be something like 860x640, although I
> cannot confirm this. The result is X fails to start.
> 
> Going back yet further mk_initrd displays the following.
> 
> Concatenating splash picture size 1024x768 to initrd... done.
> 
> Can anyone throw some light on the boot process and vga ?
> My backup system continues to boot into X ok. I need
> the new kernel to use these settings.

Under your kernel source tree, the Documentation/fb/vesafb.txt file
lists the vga= parameters for the VESA framebuffer driver. Try picking a
different vga mode and setting that in your lilo.conf. It sounds like
the VESA framebuffer driver is built into your kernel, otherwise you'd
probably see an error message instead of a blank screen.

Hope it helps,
Matt
