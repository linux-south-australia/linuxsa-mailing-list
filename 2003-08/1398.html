*OR* 

Let Internet Exploder do it's own searching.  Get this.  Internet
Explorer 
will check the local zone, eg for blackwood.sa.edu.au it will attempt to

resolve wpad.blackwood.sa.edu.au.  If it doesn't find it, it will
attempt 
to resolve wpad.sa.edu.au and so on and so forth until it finds a server

willing to supply a wpad.dat file.  What this means is, if you run a 
proxy, you need to compensate for this by creating CNAME's of your proxy

server to make sure that IE will "catch" the right proxy server.  

I'm not sure if this is the way Internet Exploder works anymore, but it 
certainly used to!
