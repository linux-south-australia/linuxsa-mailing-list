Re: How to tell whos calling a function or accessing a file

On 15/08/2003 11:32 AM, jonathan soong wrote:
> Hi
> 
> Dunno if anyone can help me here,
> I am using the Horde framework for mail (PHP)
> and i think its doing a whole bunch of includes on different files etc.
> 
> I was wondering if there is an easy way to trace what files are being 
> accessed without trawling thru code..
> 
> (i guess kind of like lsof - i want to know what was just opened, not 
> what is currently open)
> 
> ?

<facetious>
grep -iE "include|require" /usr/local/lib/php/horde/*
</facetious>

on a related note, for a long time i wanted to be able to tell what file 
was *including* the current file, and thought i couldn't get access to 
that - then i realised that $PHP_SELF refers to the script doing the 
including, and voila.

e.g:
master1.php: <?php include("slave.php"); ?>
master2.php: <?php include("slave.php"); ?>
slave.php: <?php echo $PHP_SELF."\n"; ?>

`php -q master1.php`: master1.php
`php -q master2.php`: master2.php

handy!
