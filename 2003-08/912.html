Re: How to tell whos calling a function or accessing a file

On Fri, Aug 15, 2003 at 11:32:59AM +0930, jonathan soong wrote:
> Hi
> 
> Dunno if anyone can help me here,
> I am using the Horde framework for mail (PHP)
> and i think its doing a whole bunch of includes on different files etc.
> 
> I was wondering if there is an easy way to trace what files are being 
> accessed without trawling thru code..
> 
> (i guess kind of like lsof - i want to know what was just opened, not 
> what is currently open)

The -u option to ls tells it to use the access time of the file
instead of the modification time. So you can, for example, use 
ls -lut to display a long-format listing sorted by access time, and
see which files were accessed at or after the time that you ran the
command you suspect of accessing them.
