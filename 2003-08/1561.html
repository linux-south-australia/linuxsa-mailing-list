have a read of http://razor.sourceforge.net/ if you want to understand
Vipul's Razor more fully, but basically it tries to identify spam by
comparing the checksum of the message with those of spam reported to the
service. 


> 2. If it actually needs to do this can it connect via port 80 or 443 ?

the daemon for the service is listening on port 2703 at those IPs. if you
have your own server configured you might have it running on a different
port which you can specify in the config.

the error you are seeing is basically the client not connecting to the
server it is trying. is there a firewall between you and the internet?
