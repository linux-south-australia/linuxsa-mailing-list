Re: copying files correctly

> Hi all,
> 
> Just want to know if there is a better way of doing what I am about 
> to do.  I am transferring most of the system and other files to 
> partitions on a new drive.  I have to use an independent 
> installation to copy files using cp -dpR /* and then a boot disk to 
> get the system up and then set up lilo and so forth.  It does work 
> but is there a better way?

I just mount the new hard drive's partitions under /mnt, on the live system (usually init 1'ed), 
then:

find / -xdev | cpio -pm /mnt

cpio tends to copy things over nicely, particulary /dev stuff.
