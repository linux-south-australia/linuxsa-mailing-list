Re: What may be a silly SAMBA question

hi

while i'm no samba guru, i can tell you that it is perfectly safe to write to 
a samba share no matter what the file-system.

The samba protocol provides what you might call a file-system abstraction 
layer, so as long as your kernel can speak samba correctly (which it can), 
its the computer on the other side which has to be able to write to NTFS. I'm 
assuming that the other computer is a windows machine (not a linux box with 
an ntfs mount with writing enabled shared through samba!!!) - and i think 
it'd be safe to say windows knows how to write to an NTFS partition :-)
