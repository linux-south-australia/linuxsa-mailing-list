Ok, here's my better and more correct explanation:

4000

 - on a directory

 Any files created in that directory will get the directory's UID. So,
 if I create a directory:

 # mkdir /tmp/LLOY0076
 # chmod 4777 /tmp/LLOY0076

 And "julie" makes a file in said directory, the file will be owned by
 LLOY0076

 This can be useful when students submit exams and stuff.

2000

 - on a directory

 Same as "4000" but works with the directory's GID.

1000

 - on a directory

 Is the sticky bit. Any files created in the directory will only be able
 to deleted, renamed or moved by the owner of the file.

You can add the numbers together:

7000

...of course you'd want to use the other permissions as well. A
directory with 7000 is probably not useful.
