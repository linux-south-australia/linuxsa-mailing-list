Re: >1 web servers on 1 ip address

Michael Martucci wrote:

> Is there any way that iptables can look at an incoming packet
> and see that it is meant for www.server1-example.com, and
> forward :80 to :80 on server1, likewise if there's an
> incoming packet for www.server2-example.com forward :80 to
> :80 on server2.

No.  Packets are routed by IP address, and you have mapped
them onto the same IP address.

> I know about apache virtual hosting, but that's not what I want.

Set up Apache virtual host on one server that will handle all your
traffic[1].  Use ProxyPass and ProxyPassReverse to map the other
web server into the virtual server's space.

Otherwise, yes, use squid as a reverse proxy to accomplish the same
thing, except at the gateway point.


[1]  Yes, I'm aware there's some irony in proposing a solution you vetoed.
