>1 web servers on 1 ip address

I'm setting up a couple of web servers, in separate UML processes, but as
far as this question goes, the network can be considered like so:
 
 (        )     ---------             ----------
( Internet )---| Gateway |------+----| Server 1 |
 (________)     ---------       |     ----------
                                |       ----------
                                +------| Server 2 |
                                        ----------

The gateway has the one IP address available. I know that port forwarding
can make any requests to (e.g.) gatewayIP:80 be served by (e.g.) server1. Is
there any way that iptables can look at an incoming packet and see that it is
meant for www.server1-example.com, and forward :80 to :80 on server1, likewise
if there's an incoming packet for www.server2-example.com forward :80 to :80
on server2.

I know about apache virtual hosting, but that's not what I want.

The only solution I can think of, is that any requests to domainname:80 are
forwarded to server1onLAN:80, then Apache picks up from there, and if the
HTTP request is not one for a domain name hosted by server1 but for server2,
then it sends back a HTTP-Redirect to (e.g.) domainname:8080, and the subsequent
request from the browser to domainname:8080 is forwarded to server2onLAN:80.

 Something tells me to have a look at squid... Hmm..
