Re: low usage of swap?

> also, the 128 is a swap partition, the other 100 is two swap files,
> and it seems the files are significantly slower... is there perhaps a
> way to make linux use the partition to its fullest before using the
> files?

Adam,

     Have a look at the man page for swapon(`man 8 swapon`). 
Specifically have a look at the -p option.  You need to edit your fstab
and set the priority for all swap partitions to the same priority. 
That should speed things up quite well.  I think in your case it will
never utilize 100% of your swap strictly because there would never be a
need.  The applications you are running do not require the memory that
you would like them to use, which is a good thing (:

A good article for linux swap can be found at:
www.xenotime.net/linux/doc/swap-mini-howto.txt
