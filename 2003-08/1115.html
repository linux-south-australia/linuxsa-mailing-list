I know Greg disagrees with Bruce about the source of
the code.

My view is that it is impossible to tell.  If you start
with the BSD code there is only one place for the lock to
be obtained.  Without some statement from the author, it's
reasonable to assume that both System V and Linux started
with the Ancient/BSD code and placed the lock in the same
obvious position.

Having been bitten by the clean-room Linux reimplementation
of the Berkeley Packet Filter, I'm surprised that SCO
hold that up as copied code.  Yeah the case statement
implementing the virtual machine looks the same, but it's
really the obvious way to implement the instruction set.
