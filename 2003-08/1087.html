On Wednesday, 20 August 2003 at 16:03:02 +0930, Ian Loxton wrote:
> SCO showed part of the Linux code it claims
> ownership to, at the Las Vegas convention.
>
> Bruce Perens says the code SCO claims to own
> was released earlier under a BSD license.

> http://perens.com/Articles/SCOCopiedCode.html

Bruce is wrong.  This code comes from System V.4.  Seriously.  I hope
marcelo can account for how it got there.

I put an analysis on
http://www.lemis.com/grog/SCO/code-comparison.html, but it's wrong.
The code in question existed in a similar form in Research UNIX, but I
have no doubt that the version from which atealloc() was derived is
System V.4.  In particular, the 

	s = mutex spinlock(maplock(mp));

replace the System V construct

	s = splhi();

That's not there in the Research UNIX version (or in System V.3, for
that matter).  Also, the comments wrap differently in System V and
Research UNIX.

I've done a fairly detailed analysis (in German).  Tomorrow I'll
translate it into English, but you might see it on
http://www.heise.de/ first.
