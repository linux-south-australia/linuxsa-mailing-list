Pigeon,

> A similar number of packages, as binaries with a little drop of
> control info, compressed, in Debian stable, takes 7 CDs.

i do a base install of the 7 CD set, then copy all the .deb files
to the local apt cache in an apt-move mirror, then install the 
stuff i need and then run;

  apt-get update && apt-get upgrade && apt-move update

so all the updated packages are pushed into the mirror
so that other servers can get access to those packages
across the local network. it works well, except if you 
are tracking bleeding edge .deb mirrors =)

if a server grabs files that are not on my local .deb mirror
i just ssh them back to the mirrors /var/cache/apt/archive
dir and apt-move mirror, so i only have to download them
once =)

its not ideal, but it works well enough where all the servers
are running from the same distribution =)

package it and put that in another mirror and install as needed =)

but the source is on the CD, and i only need to download the 
updated source package once... rebuild, package, mirror ...
i have two configs, routers/gateways and servers.
