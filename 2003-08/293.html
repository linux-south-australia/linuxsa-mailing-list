> But people complain about "vi" having modes and those people are
> *emacs* people.

Same word, different context.  The trouble with vi is that you have to
change modes frequently.  Emacs normally does not change the mode for
a specific buffer.

> And just how many modes does emacs have? 

Many.

> One for each meta key?

No, you're thinking vi.  In Emacs, you set a mode by running a LISP
function.  Once there, you forget about it.

The idea of modes is to tell Emacs how you want to edit the buffer.
In James' case, he wanted to have automatic line wrap.  That's not the
sort of thing you can accept all the time (in a C program it would be
painful, for example), so it's implemented in a specific mode.  On the
other hand, you may find it very convenient when writing C programs to
have { indent correctly and insert new lines where appropriate.  You
might also find it useful to have the Return key ("Enter" for PC
types) to cause an automatic indent to the appropriate column.  You
wouldn't want either of those when writing a mail message.
