The next time I build a backup box, FreeBSD snapshots + rsync will 
probably be how I do it.

Like all great new open source technologies, the documentation is poor:

http://www.freebsd.org/doc/en_US.ISO8859-1/books/handbook/snapshots.html

In theory (and I'm making the leap of faith that FreeBSD's snapshots 
work like other filesystem snapshot systems, I havn't had a chance to 
try it yet) it will let you do a rotating backup schedule (limited to 20 
simultaneous backup instances per filesystem) where only the changes in 
files will take up disk space.

ie: you make a snapshot called hourly.1, and mounting that filesystem 
lets you see your main filesystem as it was when you created it. Repeat 
for hourly.2, daily.1, daily.2, weekly.1, etc  by renaming the snapshots 
with a script run from cron that also deletes the snapshots that are too 
old (or turns the last daily one into the newest weekly one etc).
