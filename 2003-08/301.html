RE: ftp server setup

> That command does nothing for me. It runs and then returns me 
> back ready for another command. There is no display of anything.

That means FTP is not running.

Take a closer look at what netstat -anp does (use the man pages -- the 'p'
can also be useful). It's handy.

OK, now you need to get ftp running. You can either use xinetd or standalone
-- your choice. Judging from what you told us about the xinetd config for
the ftp server, it appears that xinetd itself isn't running. If you'd like
to use xinetd, you'll need to start it -- easiest way to do that in RH is
'chkconfig xinetd on', or something like that... You'll also need to
manually start xinetd with: '/etc/init.d/xinetd start' ...
