> > Ok, let me give a bit more detail, i have a whole stack of files that
> > have a mixture of upper and lower case, and i need to change the file
> > names, all to lover case......does this expalin what i want to do a
> > bit better
> 
> # for f in `ls`

The problem with this is that you will be screwed if there are any files 
with spaces in their names. It also violates Newall's rule: Don't create 
processes you don't have to!

Better is:

  for f in *

> > do
> > mv $f `echo $f | tr [A-Z] [a-z]`
> > done
