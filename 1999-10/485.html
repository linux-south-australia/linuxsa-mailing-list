Re: linux uses..

I believe some UNIX systems allow a process to get dedicated access to
one or more processors in a multi-processor environment. If you could
hack the APIC IRQ handler so as to pass "your" interrupt (say IRQ 5) to
processor number 2, and have a dedicated process running on it, that'd
be cool. Processor number 1 runs everything else, leaving processor 2
able to sit in a busy-loop waiting for input.
I wonder if that would help.
