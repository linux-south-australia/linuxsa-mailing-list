>The problem I have now is connecting to the LINUX box (RedHat 6.0) from the 
>NT box (4.0 sp4). The other way around works just perfect. Using TkSamba 
>0.8 I'm able to browse and use the resources available on the NT boxes. But 
>trying to connect to the linux box I get the error: The password is invalid 
>for (Linux box). So my question is:

This is likely due to one of the following reasons:

1. The account you logged into on NT, or the account you are using to
connect to the share, are not on the Samba server

2. Since you mention SP4, it could be that the NT system is sending
encrypted passwords and you are not using an smbpasswd file.  You should
either apply the registry updates that are in /usr/doc/samba-$VERSION
somewhere, or you should set up an smbpasswd file.

3. There is a small possibility that NT is sending password to Samba in all
upper case.  I will check this out, and if it is, you need a statement like
'password level = 8' in the global section of your smb.conf file.

>Which parameters do I have to set in the smb.conf file to get access to the 
>linux box? Do I have to set som spesific rights on the /home directory and 
>its sub directories?

Well, you might need to do that after you sort out the password problem.
