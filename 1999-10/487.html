 LOL....

 Okay... so is anybody going to have a shot a building on of these stupidly
 expensive dual processor ECMs???

 I think we are starting to prove that for uses like an ECM, a linux machine
 (well not a x86 anyway) is just not right for the job.

 Really, there is no point in using a multiply use kernel (linux) and system
 (x86) for a single use task. It would be much easier to get a slower
 processor like a AVR (atmel) or 68xxx (motorola) to do the job and then
 interface with a linux box.
