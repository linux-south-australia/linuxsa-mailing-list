On Sat, Oct 09, 1999 at 07:26:56AM +0930, Andreja Zivkovic wrote:

> What do I need to add to /etc/conf.modules to make the module load at
> bootup?

OK, I'm assuming the sblive module is called sblive, change it to whatever it
really is. :) All you need to do is alias char-major-14 to sblive, using the
following incredibly non-intuitive config line in conf.modules:

alias char-major-14 mad16

<grin> This will ensure that any calls to the sound module load your sblive
stuff on demand.

I use this stuff at home to get a very dodgy OPTi 931 pnp card to run, with
the following lines:

options sound dmabuf=1
alias char-major-14 mad16
alias mixer0 mad16
alias audio0 mad16
alias midi0 mad16
alias synth0 opl3
options sb mad16=1
options mad16 irq=7 dma=0 dma16=1 io=0x530 joystick=0 mpu_io=0x330 mpu_irq=9
options opl3 io=0x380
post-install mad16 /usr/local/sbin/ad1848_mixer_reroute 14 8 15 3 16 6 && /usr/bin/aumix -w 75 -s 75 -c 75

Let me know if you need any further help.
