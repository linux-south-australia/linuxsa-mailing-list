Re: postgresql problems with large text fields

On Wed, Oct 20, 1999 at 05:39:25PM +0930, Daryl Tester wrote:

> > The article mentions that if PostgreSQL is used for storing
> > large text fields -- e.g. articles posted by users -- this
> > corrupts the database because of fields over-writing other fields.

> I (IMHO) can't see why people store large amounts of text data
> (or binary data, for that matter - what was that pointless map web
> site that Microsoft used to go on about?) in relational databases
> in the first place.  I still have enough faith in Postgres that
> I will leave my home/business accounting system (such as it is)
> based on it, and enough faith in systems in general to do regular
> backups.

The reason that you could / would store large things in the database
is that it may be just an efficient file system as your regular file 
system. If you all your data is stored in the Database backups <can>
become much easier. ie you do 1 nightly backup of your database as it 
will hold ANY data that has changed.

The ext2 file system that we all run on our linux boxes isn't too 
different to a relational DB when you think about it.
