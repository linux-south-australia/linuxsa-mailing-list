Re: RAID array setup

On Wed, 27 Oct 1999, Steven Tee wrote:

> Has anyone succesfully got a software raid array working on the 2.2.x
> kernel.
> 
> I am trying to setup a RAID-1 array on a Mandrake Linux 6.1. (2.2.13
> kernel with raidtools-19990824-0.90)
> 
> When I run 'mkraid /dev/md0', all I get is the following messages:
> -----
> handling MD device /dev/md0
> analyzing super-block
> disk 0: /dev/hdb1, 6193498Kb, raid superblock at 6193408Kb
> disk 1: /dev/hdd1, 6193026Kb, raid superblock at 6192960Kb
> mkraid: aborted, see the syslog and /proc/mdstat for potential clues.
> -----

To use raid 0.90 you'll need to patch the kernel. There currently isn't a
patch for 2.2.13 but should be soon.

If you want to use 2.2.12 you can grab the patch from:

ftp://ftp.au.kernel.org/pub/linux/daemon/raid/alpha/raid0145-19990824-2.2.11.gz

to apply unzip it, then run (in the kernel root dir):
patch -p1 < raid0145-19990824-2.2.11

The patch will have 1 reject, say no to both Q. This is because a problem
has already been fixed in the 2.2.12 kernel (notice the patch is for
2.2.11).

I you had it working with the default mandrake kernel this is because
redhat has already applied the patch :)
