Ok, try this:

+  newton@gizmo> telnet www.aph.gov.au 80
   Trying 202.14.81.230...
   Connected to wopared.aph.gov.au.
   Escape character is '^]'.
+  GET / HTTP/1.0
+
   HTTP/1.0 200 OK
   Server: Microsoft-IIS/3.0
   Date: Wed, 27 Oct 1999 15:11:21 GMT
   Content-Type: text/html
   Last-Modified: Thu, 14 Oct 1999 04:09:30 GMT
   Set-Cookie: EGSOFT_ID=203.20.76.6-2258699200.29302925; expires=Fri, 31-Dec-2010 00:00:00 GMT; path=/
   Content-Length: 5245
   Connection: close
 
   <!DOCTYPE HTML PUBLIC "-//IETF//DTD HTML//EN">
   <html>
   [ lots of stuff deleted ]

(you should type the bits with "+" signs next to them).  

Verify that both of your computers think that www.aph.gov.au is the
same IP address.  Also verify that the Last-Modified timestamp in the
returned HTTP header is the same for both.  You migth also like to 
remove your cookies.txt file, restart your browser, and start again,
because the site sets a cookie which might be used in a conditional
to determine whether you should get a HTTP redirect (a "Location:" header).

Check the returned HTML too to make sure that it matches up with what
your "bad" browser is seeing.  If the telnet shows one thing and the
browser shows something different, then it's definitely a browser problem
and almost certainly related to caching.

Did you restart Netscape when you removed the cache directory?  Remember
that it also caches in memory, and nothing you do to the cache directory
will affect that...
