Converting Access and other database files

G'day,

At the meeting last night, I mentioned that you can import data from
Access databases into other database systems such as MySQL or
Postresql, but it's more straight forward than discussed, and this
pretty much applies to ANY type of database.

On the non-Linux platform computer, export the data from the database
as a comma (or TAB or Pipe, etc.) delimited file.  Darn near any
database program (regardless of age or platform) are able to do this. 

Next, follow the steps for MySQL or whatever database you use on the
Linux system to first define the fields, and then import the data from 
the comma delimited file. 

If you're in the situation where you'll need to update the information
from the same format, you can write a short script file that will make
it painless to do future updates, deleting the current database table
and importing the updated information from the delimited file.  The
one I use takes up all of 4 lines - the client uploads the file to a
specific directory, and the script takes care of the rest.

For ODBC type databases (such as Foxpro), there's a nifty program
called dbfdump that will dump the database to a text file, and you can
specifiy the delimiter character. Then the procedure is the same as
above - import the data into the Linux database of choice.

You can find dbfdump in DBD-XBase-0.147.tar.gz - it's on the Perl CPAN 
sites.
