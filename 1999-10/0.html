Re: iso 8895-1 typing with US kbd

On Thu, 30 Sep 1999, Alan Kennington wrote:

> Question:
> How do I get foreign accents etc. with a US keyboard?
> 
> More and more lately, I'm finding that I need french and
> german in e-mails.
> Linux mailers receive the west european iso8859-1
> alphabet okay. But I can't see how to actually type in
> accents with my US keyboard.
> 
> Is this application dependent?
> Or does XFree86 have some set of 
> application-independent mechanism for entering
> those foreign characters?
> 
> On solaris, I think you use a "compose" key to
> compose these chars. But I don't see such a key
> on my US standard kbds.

I'm not sure about others but I use the following over my normal keyboard
map. It add support for the euro (with the correct font mapping) and
converts printscreen to compose. Also remaps the win95 keys and a few
others. Just dump it in something.kmap the run loadkeys something.kmap

Ie Printscreen ' a give á

------------------------------
# EURO and CENT
# "Alt" is replaced by "AltGr" else you will not have euro sign !

keycode  18 = +e                +E              currency
keycode  46 = +c                +C              cent
keycode 100 = AltGr
        alt     keycode 100 = Compose

# Extra 105 windows keys

keycode 125 = Decr_Console
keycode 126 = Incr_Console
keycode 127 = Last_Console

# Delete & BackSpace support

keycode  14 = Delete           BackSpace
keycode 111 = Remove       BackSpace
        alt     keycode 111 = Meta_backslash
        control keycode 111 = Control_backslash
        alt     control keycode 14 = Boot
        altgr   control keycode 14 = KeyboardSignal
string Remove = "\033[3~"

# Home & End support

keycode 102 = Home
keycode 107 = End
string Home = "\033[1~"
string End = "\033[4~"

# Many a keyboard signal to CTRL + ALT + END
control alt keycode 79 = KeyboardSignal
control AltGr keycode 107  = KeyboardSignal

# Latin0 composing table
# "Print Screen" is useless with linux, so it is replaced by "Compose"
