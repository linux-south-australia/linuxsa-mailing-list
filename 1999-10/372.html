This is a known bug in the ping client. If you look closely at the output
you can see that the problem occurs when the response time is greater than
1 second. The problem is that each packet that ping sends out is slightly
different. If a packet takes more than 1 second then ping sends out another
packet before it receives the reply. This packet has a different contents
and so ping compares the earlier respose with the latest packet it sent and
finds a difference.
