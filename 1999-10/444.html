RAID array setup

Hi.

Has anyone succesfully got a software raid array working on the 2.2.x
kernel.

I am trying to setup a RAID-1 array on a Mandrake Linux 6.1. (2.2.13
kernel with raidtools-19990824-0.90)

I have configured the /etc/raidtab file as follows:

-----
raiddev /dev/md			0
	raid-level 			1
	nr-raid-disks 		2
	nr-spare-disks 		0
	persistent-superblock 	1
	chunk-size			4

	device			/dev/hdb1
	raid-disk			0
	device			/dev/hdd1
	raid-disk			1
-----
	
When I run 'mkraid /dev/md0', all I get is the following messages:
-----
handling MD device /dev/md0
analyzing super-block
disk 0: /dev/hdb1, 6193498Kb, raid superblock at 6193408Kb
disk 1: /dev/hdd1, 6193026Kb, raid superblock at 6192960Kb
mkraid: aborted, see the syslog and /proc/mdstat for potential clues.
-----

I have checked the syslog file and can find no related messages the
/proc/mdstat is as follows:
-----
Personalities :  [3 raid1]
read_ahead not set
md0: inactive
md1: inactive
md2: inactive
md3: inactive
-----

I have done some initial hacking around in the mkraid.c file and the
error is occuring during an ioctl call to the /dev/md0 with the
SET_ARRAY_INFO command. The errno response is 22(EINVAL)
