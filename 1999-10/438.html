sendmail sets X-Spam based on RBL, etc

If you use sendmail at your network boundary then you
might be interested in this.

Sendmail can be configured to reject messages from
IP addresses in the RBL.

There are other RBL-like databases, but as these
increase the number of false positives you really
have to accept mail from addresses in those databases.

There is a sendmail patch that will look up all the
spam databases and set a new header

   X-Spam: <db>: <reason>

Your users can then send messages with an X-Spam
header to a "Spam" folder or to /dev/null.

I've put it at:

 http://www.adelaide.edu.au/~gturner/software/sendmailrbl/

It's not my work, but synthesised from the work of a few
people.  I've packaged it up in RH6.1 i386 RPMs and an
SRPM.

I'll probably roll it into the main sendmail source in
time for 8.10, with an option like:

  FEATURE(`xspam', `rbl.maps.vix.com', `RBL')
  FEATURE(`xspam', `dul.maps.vix.com', `DUL')
  FEATURE(`xspam', `relays.orbs.org', `ORBS')

Like all mail servers, if you are going to flog the poor
thing, then install a caching name server on the machine.

Note that with the current code you are stuffed if
you appear in one of the listed spam databases.  I'll
fix this.
