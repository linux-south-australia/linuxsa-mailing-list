Re: Apache & .htpasswd

> If I have "AllowOverride All" for a particular directory, all I get is a 
> permission denied from the browser.

I doubt it's related to AllowOverride.  I'd say it's more likely to be
the permissions of either the directory or the particular files within
the directory (assuming you haven't used Auth* and Require.)

Directories *must* include the execute bit in order to be able to access
any file within it.  Take note:

  drw-rw-rw-  21 user  group  2560 Oct 15 12:37 www

Nobody can access any file from this directory.

Take note:

  d--x--x--x  21 user  group  2560 Oct 15 12:37 www

Anybody can access any file from this directory *if* they know the name
of the file.  Nobody can list the directory, but they can access any file
that they know the name of.

Non-directories *must* include the read bit in order to be able to be
read.  Take note:

  --wx-wx-wx  1 user  group  1003 Oct 11 12:62 www/index.html

Nobody can display this file, although anybody can change it's content.
Even after they've the changed the content, still nobody can view it.

Take note:

  -r--r--r--  1 user  group  1003 Oct 11 12:62 www/index.html

Anybody can display this file.

In summary: These are good permissions for directories and non-directories:

  dr-xr-xr-x  21 user  group  2560 Oct 15 12:37 www
  -r--r--r--   1 user  group  1003 Oct 11 12:62 www/index.html
