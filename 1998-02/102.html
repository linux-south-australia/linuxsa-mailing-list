Hi Staf,

I think your problem is that you are trying to connect to a BSD box.

This was discussed on the LinuxSA list back in November/December last
year.  Steven Ellison wrote:

> I find this difficult to believe as both systems have been around for a
> long time!  I know that APANA in South Australia are using FreeBSD as a
> terminal server for several people using Linux as well as a lot of other
> OS's.

And Tony Galliford replied:

> Well, having had exactly the same problem myself (linux box into
> pasa.apana.org.au), I find it quite easy to believe.  It might work
> with some particular configurations or config file tweaking or
> something, but I ended up solving the PPP problem by changing to
> SLIP.

Since then has anyone else been successful in solving this problem
without changing to slip?

A message I quoted back then from Mark Newton said that the problem
was with Predictor-1 compression.  Staf, you might like to try using
the "-pred1comp" option to pppd, which tells it to not accept
Predictor-1 compression (if you do try this, make sure to let us know
what happened).
