Wherefore art thou, libX11.so.6?

Somewhere in the Linux world there are those with an intimate understanding
of how libraries operate.

I wish that guy would stop hiding!

I have a program (not compiled on my machine) which isn't happy:
$/usr/src/utilities/xcalc/XCalc
XCalc: can't find library 'libX11.so.6'

for which:
$ldd XCalc
        libX11.so.6 (DLL Jump 6.0) => not found
        libc.so.4 (DLL Jump 4.6pl27) => /lib/libc.so.4.7.5

Yet, in /usr/X11/lib:
lrwxrwxrwx   1 root     system         13 Feb 13  1997
/usr/X11/lib/libX11.so.6 -> libX11.so.6.0*
-rwxr-xr-x   1 root     system     590068 Jul 23  1995
/usr/X11/lib/libX11.so.6.0*

Ok, so let's check out:
$ cat /etc/ld.so.conf
/usr/local/lib
/usr/X11/lib
/usr/i486-linuxaout/lib
/usr/packages/interviews/lib/LINUX

and a manual scan of /etc/ld.so.cache shows it does indeed contain the
text libX11.so.6 and /usr/X11/lib/libX11.so.6.0.

Any suggestions?
