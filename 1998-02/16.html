Ok these are good questions.

The problem seems to be an mis-understanding about linux and devices in
general.  The simple answers:

Are runlevels set up for definate purposes?  Some yes, some no.

Can you set up runlevels to load specific drivers?  Yes.

Is that what they're for? No.

Should I already know all the answers?  Of course!  Didn't you solve the
					middle-east crisis before breakfast!?

What is a runlevel?  Well I've already answered this question on this list, 
	so I'll just point you to it.  
		http://www.uq.edu.au/~cmamuys/documentation/sysVinit.help

Now to deal with devices:

Under DOS device drivers are kept in a link-list style structure that is 
assembled as each driver is loaded.  As a result attempting to unload a 
device driver under DOS normally corrupts the chain, and crashes the system.  
Not surprisingly this product of brain-dead design isn't used in linux.  
Under linux devices are either compiled into the kernel, or loaded dynamically 
as modules.  These modules can be loaeded and unloaded at will (although there 
are some dependencies between them, ie. vfat requires fat to be loaded first).  
In fact if you use kerneld you can have the modules automatically loaded when 
needed, and unloaded again when they aren't being used.  So you don't even 
have to change runlevel to load anew set of drivers.

I suggest you read the man pages for
	modprobe, insmod, rmmod, lsmod, depmod.
I also suggest you read the Modules HOWTO.
