Re: TCP buffer size

At 06:22 PM 7/22/00 +0930, David Newall wrote:
>> If you're trying to impose "block boundary" semantics, remember that
>> TCP/IP is a _stream_ of data, and you need to impose your own semantics
>> on top of that
>
>Are you sure about that?  My recollection is that block boundaries are
>preserved over TCP.

Daryl is correct. TCP is a stream protocol. Block boundaries are not
guaranteed at the receiver. It depends on a number of factors, like whether
or not TCP_NODELAY or whatever is set, as well as the window size, the
sending size and the receive buffer sizes and what size reads the receiver
does.
