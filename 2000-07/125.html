Patrick,

If you use the same IP address, you have no problems either. Of course you
can use the two different IP's and set up eth0:0 to use the second address,
you can set up apache to server by ip or by URL.

Using virtual hosting with apache (and single IP), you can set it up to
handle the domains, the request is going to.

The packet, to the server, will look something like this

GET www.aaa.com.au/index.html HTTP/1.0

blah blah blah

The server reads this packet to determine where index.html lives. If you
have www.aaa.com.au mapped to /home/aaa/ then the server will take
www.aaa.com.au/index.html and read the file /home/aaa/index.html.

Same for www.bbb.com.au.

The Server can happily use the same IP address for aaa and bbb, because the
client is only after the address of www on either aaa or bbb. Once there it
stills sends the standard HTTP header as shown above which is interpreted by
apache.

Good luck.

Rob.

P.S. This is assuming you are going to be using apache of course 8-)
