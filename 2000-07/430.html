Run tcpdump on the linux box (tcpdump -i eth0) and watch what happens
when you ping from the Windows box to the linux box. If you send it to
the list, it will probably help. I have found that (particularly with
Windows machines and PnP cards) that Windows thinks the card is using
one IRQ and is actually using another.

In this case, tcpdump (or ethereal) will show that a ping request is
being sent from the windows box to the linux box and the linux box is
being sent back. The actual reply never reaches the Windows operating
system, because it thinks the LAN card is on a different IRQ. If you
ping the other way (Linux -> Windows), you will get a request sent and
no reply.

Try this and see how you go.
