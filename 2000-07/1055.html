Re: Install Red Hat/Windows

Okay Ben, I'll give it a go.

First.  My suggestion to getting you set up is to use fdisk under
DOS. 

1)  Access the second physical device (important, don't want to kill
windows)
2)  Remove all current partitions on second device (since you can't
access them anyway I assume there is nothing there to loose)
3)  Create a *Primary* partition 1.5 gig in size
4)  Exit fdisk saving your config

You should now reboot windows and be able to format D:

This now means that there is 1.5 gig of unpartitioned space on the
second hard drive.  The redhat install should automatically use this
space, and if not the install process will let you set it up (just
don't delete any FAT partitions.  (Side note: If you have a Pentium
or higher, have a look at Linux Mandrake 7.1.  It's installer is
quite good for beginners).

Now for the terms as I see them (anyone is more than welcome to
correct any errors):

Primary partition - is the first place windows looks for a partition
on a physical device.  You should always have a primary partition on
a hard drive.  Windows formats it into one logical drive location (ie
c:).  Can be an active partition (ie bootable)

Extended partition - Under windows, this is where extra logical
drives can be created.  Can have multiple logical drives (ie d: &
e:).  I don't think it can be active (ie bootable).

Logical partition - essentially a mapping of an area of drive space
(either a primary partition or a part of an extended partition) that
is assocciated with a volume label (like c: or d:).

Active partition - Is a partition that is bootable.  Dos only allows
one on a drive.

Although it is possible to get around the default setup standards the
standard setup for a physical drive is to have:
  One Primary partition - Active
  One Extended partition - containing one or more logical partitions
  (with the extended partition optional).

Whoooh.  Hope that a)  Makes sense
                   b)  Answers your questions
                   c)  Fixes your problem

Any more questions, just post 'em up.
