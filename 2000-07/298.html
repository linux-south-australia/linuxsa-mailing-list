On Mon, Jul 10, 2000 at 08:50:39AM +0930, Conrad Gittins wrote:

 > didn't register as a zombie and consumed all processor time.  It may
 > also be a java bug.  The only
 > solution I knew of was to reboot which isn't really satisfactory.  Has
 > anyone seen this before and is there another solution (apart from

The only way a process can be unkillable is if it's in the middle of
an IO operation (its status will show as "D" in ps).  That condition
is usually short-term, unless the IO it's doing is to a file on an 
NFS server which is not responding, or a tape, or some other very slow
device.

[ "the only way" is not strictly accurate above, there are other things
  which can place a process into device wait, but the summary above will
  suffice for this message ]

When you say "unkillable", what steps did you take to kill it?  Remember
that it's possible for a process to intercept signal 15, so simply typing
"kill <pid>" isn't necessarily sufficient.  Did you try "kill -9 <pid>" ?
