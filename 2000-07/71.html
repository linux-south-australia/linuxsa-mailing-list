Re: IDE vs SCSI (was: Performance with Raid 0 using IDE drives)

On Monday,  3 July 2000 at 11:27:43 +0930, Ben Hughes wrote:
>>> Anybody else notice that this comparison is skewed by the fact that the
>>> SCSI drives are 10,000 rpm while the ATA drive is only 7,200 rpm?
>>> Surely this would effect the latency...
>>
>> It has an even greater effect on throughput because the speed of disk
>> transfer is proportional to rpm.  Why?  You can't transfer a track in
>> less time than it takes the track to pass under the read/write head.
>> A drive spinning at 10000rpm is approximately 40% faster than one
>> spinning at 7200rpm.
>>
>> Just to be quite accurate, modern drives have large on-board caches,
>> which permit them to return from a write immediately.  Read operations
>> don't benefit quite so much unless you read in the order that the
>> drive electronics predicted (assuming the drive does read ahead, which
>> is likely.)
>
> It's true that this is one of the major factors, but the other is density.
> If you increase the density then you can read more information in the
> same amount of time. I've just bought some Fujitsu ATA drives that
> are 5400 rpm and give better performance that older IBM 7200 rpm
> SCSI drive.

For some definition of "performance".  Higher densities improve
sequential access; higher rpms improve random access.
