Re: Apache Forbidden Problem

Hi
I'd make two suggestions, check file permissions. Apache user runs as
user/group www,www or nobody,nobody and so all files need to be made chmod a+r
and directories made chmod a+rx

Also you http.conf needs lines like ths (if you don't also use
access.conf, and srm.conf. Note if you do, then this will be diferent into
where this is located.

<Directory />
    Options FollowSymLinks
    AllowOverride None
    Options None
</Directory>

also

DocumentRoot /usr/local/apache/share/htdocs

also if you've got any allowfrom and deny rules, check where they say
these should be.

A good reccomendation if you are running RedHat, get the
apache-1.3.12-2.i386.rpm from the redhat 6.2 RPMs folder, and install
it. Its http.conf is well documented and will enable you to build a
working apache server with ease.
