Re: rpm that can unzip the xfree4.01 rpms

Hi there.

I assume that you mean that when you try to install
the rpm using something like:

rpm -i XFree86-4.0.1.rpm 

or whatever it tells you that XFree86 version 4 is
already installed.  If this is the case then you
should try:

rpm -U XFree86.......

instead, as this command will update the package to
the new version.  (Just in case check the man page for
rpm to see if that should be a capital U or lowercase
u).

If not the problem, could I have more details?

Hope this helps.

(Oh and personally I don't like the idea of renaming
rpms.  I think it might mess up rpm's ability to
determine dependencies - as in "This package needs
XFree86-4.0.1 to run" when you renamed it
XFree86-Better-4.0  or something like that).

Damien Byrne
