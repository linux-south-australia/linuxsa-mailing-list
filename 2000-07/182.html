Again, that is a lack of development libraries. I notice that you don't
show "./configure". If the package you are compiling has a file named
"configure" in the main directory of that package, type "./configure"
and it should tell you a little bit about what's missing.

It might say, for example, "gtk+ library version 1.00 needed but
missing. please install gtk+ version 1.00 development headers" or some
similar thing....

To find out if your system is REALLY, REALLY broken try compiling the
hello world:

#include <stdio.h>
int main(int argc, char *argv[])
{
  printf("Hello world!\n");
  return 0;
}

with gcc -o hello hello.c
