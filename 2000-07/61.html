Re: IDE vs SCSI (was: Performance with Raid 0 using IDE drives)

On Mon, 03 Jul 2000, David Newall wrote:

> Just to be quite accurate, modern drives have large on-board caches,
> which permit them to return from a write immediately.  Read operations
> don't benefit quite so much unless you read in the order that the
> drive electronics predicted (assuming the drive does read ahead, which
> is likely.)

And, you have to be wary of what 'on board' cache really is. Trouble is,  what
the oem doesn't tell you is he loads his firmware code into this area leaving
(typically) only half for real cache - i/o operations. What's on the box, is
rarely what's under the covers.
