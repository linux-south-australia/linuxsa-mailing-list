Read the make file Richard.... ;-)

It runs Lilo and set's the system.map for you. The new kernel is installed
as your default boot. So every time its buildt a kernel it will be installed
as "image=/boot/vmlinuz-2.2.16" or what ever I call it in /etc/lilo.conf.

I can always come back up. Because the first thing I ALWAYS do after an
install is create a second boot option in lilo.conf using the install image
so I have a working kernel to boot up if I make an error in the build.


ie
-----------------------------------------------------------
boot=/dev/hda
map=/boot/map
install=/boot/boot.b
prompt
timeout=50
image=/boot/vmlinuz-2.2.16
        label=linux-2.2.16
        root=/dev/hda9
        read-only
image=/boot/vmlinuz-2.2.12
        label=linux-2.2.12
        root=/dev/hda9
        read-only
image=/boot/vmlinuz-2.2.10
        label=linux-2.2.10
        root=/dev/hda9
        read-only
image=/boot/vmlinuz-2.2.5-15
        label=linux-2.2.5-15
        root=/dev/hda9
        read-only
