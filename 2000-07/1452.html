RE: Parallel Port Sharing

* I know that there are ways to get Windows printing to a UNIX printer
* using Samba (and possibly other less well known methods), but is there a
* way to allow a Windows machine to speak directly to a printer port on a
* Linux box? I probably should be more precise:
*
* * Linux Box 1
*
* * Windows Box S (for Sux)
*
* Is there any way I could set things up such that Linux Box 1's second
* parallel port appeared as a parallel port on Windows Box S (so that the
* driver on WBS could speak directly to it)?

We use a little proggie called "ip-p2p" (IP Peer-To-Peer) here that lets a
windows machine print to an IP printer. We use it to print to a networked
Ricoh photocopier/printer and about 5 AXIS devices. Works like a charm.

If you can't find it on the 'net, I'll make sure it isn't copyright then
Email it off to you, if you like...
