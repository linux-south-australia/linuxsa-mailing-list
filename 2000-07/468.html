Re: Can IPChains do this?

> Voiceserver software running on a box listens on TCP port 3782 for 
> any new client connections (not configurable) .  The voiceserver 
> software has no logging facilities, so....
> 
> Can I set IPChains to initially capture any incoming requests from 
> clients on that port, log it thorugh syslog, then allow the request to 
> get to the Voiceserver software?

It sounds like you want to log TCP packets with the SYN bit set

 /sbin/ipchains -A input -l -y -p tcp --destination-port 3782
