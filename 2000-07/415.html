At 01:20 PM 7/11/00 +0200, 752304 wrote:
>Hi,
>
>Using the Bourne Again shell I prepared the /etc/profile and the
>~/.profile. But to my surprise only /etc/profile is activated when I create
>a new terminal. ~/.profile can be activated after doing source .profile.
>According to my knowledge this should be done automatically. Where did I go
>wrong (creating a .bash_profile didn't get me any further)?

Bash looks for the following:

   /etc/profile as you have found

then is looks for and executes the first of the following it finds:

   ~/.bash_profile
   ~/.bash_login
   ~/.profile

You can find all this in the man page for bash. if ~/.bash_profile exists,
then .profile will not be executed.
