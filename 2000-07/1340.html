On Tuesday, 25 July 2000 8:23, Matthew Tippett wrote:

>I also feel that rather than 'give us a Linux version' we should push for
>something more like 'let us work with you'.  Assist the ATO defining
something
>like a BAS XML schema to allow *us* and others to be free to develop
software
>with output that can be easily validated by the ATO.

Sorry to burst your bubble here, but the ATO already _have_ an XML schema
for the BAS.  The intention seems to be that your accounting package
generates the XML file, and the e-Record software then parses the info,
encrypts it, and handles the transmission to the ATO (this is what I
gathered from  skim read of ATO Docco (URL below)).  So I think its really a
Linux version of the e-Record software that we'd need.

If you go to
http://www.taxreform.ato.gov.au/publications/1999/nat3030a/nat3030a.pdf
you'll find a document describing the system they've put in place, and the
XML schema required to interact with it.  At http://203.23.51.169/ you'll
also find the ATO's "Registered Software Facility" which has a lot of
information of relevance to software developers,

Jim.
