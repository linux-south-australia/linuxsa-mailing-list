for IDE drives:

Primary Master - /dev/hda
Primary Slave  - /dev/hdb

Secondary Master - /dev/hdc
Seconadary Slave - /dev/hdd

If SCSI then the "hda" would "sda"  etc...

If you have your primary master (ie /dev/hda) setup with windows (or other
o/s) 
and you have a swap partition for Linux plus a root partition, then your
devices will be:

Windows partition = /dev/hda1
swap partition = /dev/ha5 (redhat sets it to 5 - but you can change it)
root partition = /dev/hda2

the alpha enumaeration is a,b,c and d is for the HDD and the numerical ID is
for the partitions on that HDD

ie if you had 3 partitions on the secondary slave it would be /dev/hdd as
the HDD 
and the partition is /dev/hdd1,  /dev/hdd2,  /dev/hdd3.
