> >       The question I want to ask is which is the best distribution for the
> >       server ? I was simply going to install RedHat 6.2 and tweak it
> >       according to my needs, but there now appears to be a plethora
> >       of options available (e.g. Mandrake, Turbolinux), and I thought

I personally recommend using what you are comfortable with. No matter
how good a distribution is, you will still have to make it do what you
want. Most dists are quite general in the audience they play to, so you
need to manually turn off all services and things before you leave. I am
creating a semi distribution for myself to install servers the way I
like (with nothing enabled in inetd.conf ;-)).

In the past (don't know about current) dists like Caldera didn't allow
you to easily select which packages to install (or better - not). We had
a problem here when I started where the last guy had installed the dodgy
UW-IMAPd (by selecting the defaults) and didn't disable it's use.

> >       It is a commercial environment. The server does file serving, intranet,
> >       database handling (small amounts), firewalling to internet, fax serving,
> >       and anything else I can think of later :)

Linux can do this and more ;-). If you are comfortable with RedHat,
stick with it on the production machines and use something different
when you get a non-critical machine to play with.

Feel free to ask questions on any specific troubles you have.

Hope it helps,
