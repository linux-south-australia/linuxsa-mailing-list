Re: GPG on different machines

Subba Rao wrote:
> 
> I have installed GNU PG on 2 different machines. The different user accounts on these
> machines are identical.

Huh, do you mean that the user names differ (say "gturner" and "gdt")
but the accounts are actually owned by the same person?

>  Very likely, I plan to have 2 sets of keys for each user account,
> one for each machine. Are there any good practices regarding the number of keys a user
> may have?

There are two notions: ether one, or one for each role of the user.

I suggest you go basic: have one key for the e-mail address that the
user uses to send their e-mail.  For me, this would be:

  Glen Turner <glen.turner@aarnet.edu.au>

which you will notice is actually different to either of my login names.
This doesn't cause any problems.

So create the key pair on one machine and copy it to the other machine.

Good practice is to limit the validity of the key.  I'd go for
one key per calendar year.

> Once the keys are generated, where do I keep the public key?

Everywhere that cannot be subverted easily.  I just use my web page,
that's where most people looking how to contact me will go.

> Is there a public server for the GPG public keys?

The PGP key servers work fine.
