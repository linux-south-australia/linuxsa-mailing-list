On Mon, 24 Jul 2000, Mark Newton wrote:

> http://twig.screwdriver.net
> 
> For those who ask about the availability of replacements for Exchange's
> calendar, contact list, etc.

This sort of approach ignores standards, a bit like having a web-based
mail server that doesn't use IMAP to talk to where the messages are
stored. Yes it will work but it doesn't allow you to talk to the same
message store with non-web clients, nor does it give you a modular system
where you can replace components without affecting the whole system.

A list of Calendaring RFCs is at http://www.imc.org/rfcs.html#calendar and
Internet Drafts are at http://www.imc.org/ids.html#calsch.

This is a very difficult area, mostly for non-technical reasons. I'm
fairly sure this topic has been covered on this list already, maybe
someone with a searchable archive can point to a message summmarising what
is available right now in terms of calendaring and scheduling.
