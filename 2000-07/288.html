Re: Partition woes

At 12:49 PM 7/7/00 +0930, Martin de Koning wrote:
>I'm running a Win98/RH6.2.2 box with a new 10gig hddd, which has 1245
>cylinders.
>
>Yes I have the LILO boot problem.
>
>However I don't want to trash my Windows install (I'm a gamer) and I
>know Windows requires the first primary partition in order to boot :)

How much space has Windows taken up?

>However according the archive for this mailing list, in order to get
>around the LILO problem I have to create a small first primary partion
>mounted as "/boot". I'm guessing its not possible to in anyway run
>Windows on the same drive then?

Only if you use the current crop of Linux distros (except Mandrake) and
your mobo does not support EDD.

If you use Mandrake 7.1 and your mobo supports EDD, you can boot from
anywhere on the 10GB disk ...

>I was thinking I could put in a old 200mb HDD as the primary slave, put
>the "/boot" partition on it and somehow configuring LILO to still
>overwrite the 10gig MBR and get it to work that way?

Yes, you can do this. It will work.

>I don't quite understand how putting the "/boot" partition fixes the
>problem :)

What you need is for vmlinuz and all the files that LILO uses in booting to
be below cylinder 1024. If you put it on that small HD, then you have no
problems.
