Paul Malcolm Bailey wrote:
>I'm a Linux newbie and need some advice on how to list directories --
>*only directories* -- in any directory in my filesystem.  [...]

Not quite what you're after, but near enough to be interesting:
	du

shows you the disk usage for the current directory, but also 
lists each subdirectory and its disk usage.  Combine this 
command with the cut command to extract the second field, and 
you've yet another way to handle this problem:

	du | cut -f 2

You can ask du to limit the depth it searches: 

	du --max-depth=3

and can ask du to start from any place in the filesystem 
(file access permissions permitting, of course).  For 
example, to start searching from the root of the filesystem 
tree:

	du /
