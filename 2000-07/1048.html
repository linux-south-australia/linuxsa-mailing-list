Re: misbehaving internet connection

Ive also had this problem and what seems to cause it, is the fact that pppd
tries to set your default route to the remote IP addr, but cant becuase a
default route is allready set.
The easy way to fix this prob is to simply do "route del default" before
trying to connect to your ISP.
Unless you are on a lan that does need a default route to be specified, you
can add that line to /etc/rc.d/rc.local so that the default route is removed
on every boot.
