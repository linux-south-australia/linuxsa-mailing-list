On Wed, Jul 19, 2000 at 07:39:52PM +0930, Jason Tan wrote:

 > >     find . -name '*.xls' -o -mtime -2 -exec cp '{}' /mnt/zip \;

 > 
 > Well actually it sounds more like a find file thign to me, not a general
 > file selection file dialog, fiel selection dialog, even if they allow
 > wildcards, still typcially only search in a single dir - the *nx find
 > command is not analogous to a file selct dialog imo it ia analogous ot a
 > find file command.
 
It is precisely analogous to a file select command, because the output
can be automatically pushed into other commands -- Either by modifying 
the --exec parameter, or by replacing it with --print and enclosing the
whole thing in `backticks` or $(command substitution parens).

A file find utility just finds files -- In a GUI it means you end up 
with a screen full of icons.  Yeah, great, terribly useful guys.  The
expressiveness of the command line is in the fact that that output can
be put to other uses which are far more flexible and useful than simply
displaying it on the screen.

 > Of course you dont have the ability to execute a command with the current
 > mac(and I dont hink the current windows) fiund file command, but the
 > mac(and I assuem the windows) find file have the flexibilitu requried to
 > find the files - from a gui.
 > So the gui is not incapable of what you wanted to do, the interface for
 > most of it - the hard bit is there.
 
The problem is that going the extra step, actually taking that output
out of the window it's displayed in and doing something useful to it, is
deemed to be "too complicated for the users to understand" by those who
do GUIs.  And the third party add-on solutions which do come close to 
doing what we're talking about here by themselves almost universally do
it by providing some kind of scripting language:  A command line in sheep's
clothing.

 > You could of course applescript the whole thing - and you can write the
 > applescript in a gui I believe you can even just tell it to record what
 > you do and it will repeat it back which I am pretty sure you can do in the
 > above case with enough generality to do what you want(eg bring found items
 > window forward, select all, copy to blah) - though I have never done so.

Which really just proves my point.  The original poster was talking about
completely replacing the command line.  AppleScript is an add-on afterthought
to the MacOS which provides it with hidden command-line functionality.  The
fact that quite a bit of AppleScript's functionality (but by no means all 
of it) can be accomplished by dragging around icons doesn't alter this one
little bit:  Indeed, it's the bit that AppleScript *doesn't* let you do 
by dragging around icons which is important.
