Re: Annoying Problem

> I run a local forwarding name server (BIND) with the following setup:
> 
> options {
>   directory "/etc/named";
>   forwarders { 203.20.69.2; };
> };

Is that 203.20.69.2 your ISP? What are you trying to do here? Why are
you using the forwarder statment?

> zone "." in {
>   type hint;
>   file "db.cache";
> };
> 
> My hosts.conf contains:
> 
> order hosts, bind
> multi on

Don't use this file anymore, it has been superceeded. Use
/etc/nsswitch.conf

> My host.conf contains:
> 
> 127.0.0.1 localhost localhost.localdomain

You should also have your local IP address here if you have one ie.

192.168.0.1	mymachine.domain mymachine

If you don't have one and you only use a dynamic PPP the don't worry
about it

> My resolv.conf contains:
> 
> domain localdomain
> 0.0.0.0

Try putting in an IP address that named is actually using. I suspect
that 0.0.0.0 is invalid as it usually means 'the default route'

> [I can tell when a name lookup is going to fail by looking at the system
> monitor applet and the time it takes]

That's a pretty dodgey way of assuming that name service is happening.
You could get a bind stats dump from named, or use tcpdump to watch
the packets.

Better than guessing.
