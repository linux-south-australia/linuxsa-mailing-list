> >> The only real problem is loading lilo and vmlinuz. These are loaded
> >> using the BIOS.  If your BIOS in not capable of EDD, then you must have
> >> /boot on a partition that resides below cylinder 1024.
> >>
> >> If you are using the 4GB disk for that, you have no problems at all.
> >>
> >> Once Linux is running, it can access any part of those 10GB and 17GB
> >> disks.
> >>
> >> It is all a matter of setting up your lilo.conf.
> >>
> >> image=/boot/vmlinuz   # This is /dev/hdc
> >>    root=/dev/hdc1
> >>    ro
> >>
> >> other=/dev/hda1
> >>    table=/dev/hda
> >>
> >> Would be a reasonable setup.
> >>
> >> A stock standard RH6.2 should be able to manage it.
> >>
> >> However, if you have a BIOS that does EDD (one since about 1998), then
> >> Mandrake 7.1 will install a version of LILO that can boot from anywhere
> >> on a 17GB or 10GB disk, so you could put your root file system on the
> >> 2GB unused on the 10GB disk.
