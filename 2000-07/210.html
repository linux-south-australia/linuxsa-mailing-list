non-ascii domain names

Concerning the recent Melbourne IT announcement of
chinese-character domain names -
http://www.melbourneit.com.au/ver2/html/investrelations/company_news/newsstory.cfm?newsid=71

doesn't this imply that all of the BIND software in the world
will have to be updated to some sort of unicode alphabet?

Does anyone have any info on this?

My guess is that they will just use UTP-8, since it
extends ascii in a backwards compatible manner.

This is going to make it really wierd to type in web addresses.
Can you imagine typing in chinese characters into the
browser's URL window? Or Tamil? And will we have to type
Hebrew and Arabic backwards?
Shouldn't we be typing Chinese and Japanese vertically
down the page anyway? (Ignore the last question.)

Surely this requires the whole software to be unicode-based
throughout.
And this suggests a thorough rewrite of big gobs of the
linux distributions.
(I believe the Chinese did this with Bluepoint linux -
http://www.bluepoint.com.cn/ , although reputedly they
did not make their OS mods open source, thereby disobeying
the GPL. But I bet they didn't incorporate Tibetan.)


==================================================
But my real question is about BIND.
Is there a new unicode version that we can obtain
for linux?
And is this really the future?
Or will the new domain name spaces be for 
particular regions of the world only?
Maybe the DNS will be regionalised, and we won't be
able to access web sites outside our language bloc.
