Re: Are they serious?

Changing your root username, possibly with the twist of creating another 
non privileged user called root, is rather pointless from the point of 
view of security.

Common points of attack, in a modern system with shadow passwords, are:
  buffer overflows in network daemons
  badly written or configured network daemons or CGIs
  local buffer overflows
  sniffed passwords
  stolen password files run through crack
  core dump executables or daemons that contain part of the shadow file 
   in memory

None of these points of attack is defended by renaming root.
  buffer overflows in network daemons
    no passwords involved
  badly written or configured network daemons or CGIs
    no passwords involved
  local buffer overflows
    no passwords involved
  sniffed passwords
    username and password are given, who cares if username != root?
  stolen password files run through crack
    crack user with uid=0, not username=root
  core dump executables or daemons that contain part of the shadow file 
   in memory
    same as for stolen password files

The cost of renaming root is fixing up all the scripts and configuration 
files this may break.  This time could be better spent providing real 
security, not just creating obscurity.
