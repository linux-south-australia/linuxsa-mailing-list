pppd & Dial on Demand

Hi Guys

I have a small linux box which I am using as a file and print server. This
box 
is also my router/gateway to the Internet for my Windows workstations.
I have set up ppp via the use of scripts, ppp-on, ppp-off etc. 

I am also using ipchains as a forwarder, which is not blocking anything.
When I want to use the Internet I manually bring up the connection using 
./ppp-on, giving the windows boxes access to the Internet with no problems.

Bringing up the connection manually is a pain, so I thought I would set it up 
so it would connection on demand.

I have modified the /etc/ppp/options file so it includes the options of 
DEMAND and IDLE 120. I then start ./ppp-on and pppd starts nicely and waits
for 
the required traffic to initialise the dialing process.

I do a ping 203.25.189.1 on the linux box and the modem dials out and I get 
a response, two minutes later it hangs up and everything looks fine.

BUT, when I do a ping or start the web browser on the windows machine, 
absolutley nothing happens, the browser sits there and waits, like it's
waiting for a 
connection.

I am confused, the /etc/ppp/options file looks like; 
-- 
lock 
noauth 
demand 
idle 120 
--
or do I need to modify the network settings in Windows, I wouldn't think so 
because it works ok when I manually establish the connection to the Internet.

Is there something else that I need to configure/modify ?

Regards
