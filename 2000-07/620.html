Re: How: mv x.tom x.dick x.harry -> y.tom y.dick y.harry ?

> For the example "x.x.tom", "x.x.dick.old" and "x.x.harry", what we
> really need is a command that matches "x.x." and then returns the
> remainder.  In c there is a string routine, "strstr" that basically
> does this, but I don't know of a corresponding shell command.

There is something very similar and I have used it to do what you are
describing. The pattern matching part of these operators is only bash
2.0.

This is an example of how it works from O'Reilly's "Learning the bash
shell" (On sale now at NetCraft Australia!)

---
#!/bin/bash

example="/home/alex/book/long.file.name"

echo ${example##/*/}
echo ${example#/*/}
echo $example
echo ${example%.*}
echo ${example%%.*}
---

This produces the following output...

long.file.name
alex/book/long.file.name
/home/alex/book/long.file.name
/home/alex/book/long.file
/home/alex/book/long

This should give you something to get started on. 'man bash' if you
need more info.
