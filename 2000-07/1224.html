Re: TCP buffer size

Ted Fernando wrote:

> In Linux 5.2 the max send and receive buffer size for TCP is 64k using the
> GETSOCKOPT and SETSOCKOPT commands. My program is going to require a larger
> buffer size, is there a way to increase the buffer size beyond 64k in Linux
> 5.2?

What exactly are you trying to achieve here?  You can send large
amounts of data down a TCP connection; witness the last time you
downloaded a GIF > 64K in size.  If you're trying to impose "block
boundary" semantics, remember that TCP/IP is a _stream_ of data,
and you need to impose your own semantics on top of that (eg,
newline for variable length text data, blocks prefaced with sizing
information).  If you're expecting to be able to write(LARGE_AMOUNT
_OF_DATA), then at the other end read(LARGE_AMOUNT_OF_DATA), then
expect to be disappointed as well.
