Bash Script Error

Hey All,

I've written a little script to check whether squid is still running, as I
am having a problem of it silently dying regularly. Once set up, it will
be run from cron. Script is:

--------------------------------------------------

#!/bin/sh
OUTPUT="`ps aux | grep \`cat /var/run/squid.pid\` | grep -c squid`"
BAD="0"
if [ "$OUTPUT" == "$BAD" ]; then
  /etc/rc.d/init.d/squid start
  echo "Squid was just restarted.\n Time is `date`"
fi

--------------------------------------------------

But, when I run it, it give the following:

--------------------------------------------------

./chksquid: [: ==: binary operator expected

--------------------------------------------------

I presume it is talking about the if statement. Anyone know what I've done
wrong? I am using exactly the same syntax on another machine in a script,
and it is working fine.

During operation, the $OUTPUT is '1', ie, there is 1 squid process
running. When it dies, it seems to do so without removing the pid file,
which is why I am checking that.

Any help would be most appreciated!
