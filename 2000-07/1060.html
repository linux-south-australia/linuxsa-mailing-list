> I have PostgreSQL on CD, but checking the HOW-TOs and other docs
> around the place, it seemed to be somewhat confusing to get going,
> and if there were easier databases for a first-timer to install and
> run, I'd probably go with that in the first instance.

Don't worry about the CD, a good integrated, easy to upgrade
PostgreSQL comes with your redhat 6.2 CD. It's dead easy to install,
it's a _real_ database, and you won't have to re-learn PostgreSQL when
your contract requires a _real_ database.

If you are building a web server, or any database that is going to be
looking at more than 100,000 hits per hour, I'd go for mySQL.

Here's how you do it...

1) Install all the PostgreSQL packages when you install the system,
   or,

2) Install them manually with:

	'rpm -ivh /location_of_RHCD/RedHat/RPMS/postgresql* '

3) Start the Database server with:

	'/etc/rc.d/init/d/postgresql start'

4) Switch to root then to the postgres user with.

	'su -'
	'su postgres'

5) Create a new user with

	'createuser <username>'

   Follow the bouncing ball.

6) Change back to your username and make a database

	'createdb <databasename>'

7) Play!

You will find the postgresql documentation is actually quite good. Start
with the tutorial in /usr/postgresql-?.?.?/tutorial and have a look at
the web site.

If your starting from scratch, you're much better off learning
PostgreSQL.
