Re: Performance with Raid 0 using IDE drives

> I am experimenting to find the maximum performance I can get out of my
> Linux Box using IDE drives.
> In the HowTo they say 1 drive per controller. Does this mean active
> drives?

IDE does not support overlapped I/O.  While you are waiting for the first
disk to seek to the appropriate track, or to spin the desired sector under
the head, you cannot issue a command to the second drive.

Suppose you tried to put a mirrored (aka raid 0) set on a single IDE
controller then every write would have to complete on one drive before
it could even start on the second.  This would obviously reduce write
performance by 50%.
