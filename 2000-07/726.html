trouble with autofs

greetings to all,

I apologise for the repetition of request but this is regarding the worng
subject I wrote previously i.e. trouble with telnet instead of the current
subject.

I installed redhat-6.2 and managed to get nis client and autofs(partly) 
working.
There is a strange problem. I have a machine called "hamir" and I
named the newly installed machine as "basant"

now hamir happens to have two filesystems which are exported

------------------------------------------------
#	(my exports on hamir )
/a/hamir/home/hamir   basant(rw,no_root_squash)
/a/hamir/home/hamir1  basant(rw,no_root_squash)
------------------------------------------------

my fstab on hamir is as follows
-----------------------------------------------------------------------
#	(fstab on hamir)
/dev/hda4           /a/hamir/home/hamir     ext2    defaults        1 2
/dev/hdb            /a/hamir/home/hamir1    ext2    defaults        1 2
------------------------------------------------------------------------

on basant I'm using autofs for mounting homes automoatically. my
auto.master is this way
--------------------------
#	(auto.master)
/home   /etc/auto.home  
--------------------------

and the indirect automounter map for /home i.e. auto.home is as follows
--------------------------------------------------
#	(auto.home)
hamir   -fstpye=nfs     hamir:/a/hamir/home/hamir
hamir1  -fstpye=nfs     hamir:/a/hamir/home/hamir1
--------------------------------------------------

now /a/hamir/home/hamir gets mounted at /home/hamir but the other fails.
i.e. /a/hamir/home/hamir1 doesnt get mounted at /home/hamir1.

I tried mounting /a/hamir/home/hamir1 using the following command
------------------------------------------
mount -t nfs hamir:/a/hamir/home/hamir1 /t 
------------------------------------------

and the filesystem actually got mounted on /t

I'm unable to figure out what is going wrong with automount. Why cant
automount mount /a/hamir/home/hamir1 on /home/hamir1.
