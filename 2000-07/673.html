Re: LinuxSA mailing list subject line [THINGOS]

The subject is the topic.  If we're talking about the list then the list is
(at least part of) the subject.  If we're talking about something else then
the list is not the topic and putting it in the subject is an abuse of
meaning.  You could as well put it in the from header, which is also
usually displayed.  That would be arguably less of a crock as the subject:
the message is in some sense from the list, even though the relevant RFC
says quite clearly that the list should not rewrite the from line.

I have trouble accepting that we should bastardise the standards to appease
the Microsoft users amongst us, particularly as Microsoft have something of
a reputation as king of bastardisation.  Must we join their ranks?

I think we should include a mandatory text/alternative html section in all
emails from the list, with the html incorporating javascript that is known
to crash Microsoft's HTML viewers.

For the record, I agree with everybody who calls to squash this topic of
discussion now!
