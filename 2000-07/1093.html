Re: Database Query

> What do you mean that postgres user does not have a password?

It really means that the user has a "*" in the password field in the
passwd file and a "!!" in the password field in the shadow file. That
way, no matter what password is spplied no-one can log in. It's simply
a user for the demon to run as rather than root.
