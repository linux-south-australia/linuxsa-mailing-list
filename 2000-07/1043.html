RE: misbehaving internet connection

Thursday, July 20, 2000, 10:28:09 AM, Richard wrote:

Regarding Chariot dialup:
> Any ideas?


I  connect  via  Chariot,  and  it  works fine (SuSE 6.4 and prior to that
RedHat 6.0).

Here is my Options file:

noauth
crtscts
lock
modem
asyncmap 0
nodetach
lcp-echo-interval 30
lcp-echo-failure 4
idle 600
noipx

However, I'm using wvdial to dial chariot.
The /etc/wvdial.conf file is as follows:

[Dialer Defaults]
Modem = /dev/modem
Baud = 115200
Init1 = ATZ
Init2 = ATQ0 V1 E1 S0=0 &C1 &D2 S11=55 +FCLASS=0
Area Code =
Phone = 82525049
Username = /* USERNAME */
Password = /* PASSWORD */
Dial Command = ATDT
Stupid Mode = 1
Compuserve = 0
Force Address =
#Idle Seconds = 86400
ISDN = 0
Modem Type = Analog Modem

And /etc/ppp/peers/wvdial:
noauth
name wvdial
replacedefaultroute

This seems to work quite well indeed.

I also have the pap-secrets file populated with my login details.
