> this code will run as is, but how does one force the g++ linker o ignore
> multiple definitions?

A more portable solution might be to use ANSI C.

If you don't want the variable to be global accross files, then use the keyword
'static' in the declaration eg:

static int foo; // linker does not export foo to other files

On the other hand, if you really want one global variable shared between files
you should declare it in one file (without the keyword 'static'), and in other
files that access the variable use the 'extern' keyword eg:

extern int foo; // tells linker to look in some other file for foo

Hope this helps, hope it was a variable and not a function ...
