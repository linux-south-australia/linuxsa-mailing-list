Netscape blocks on resolver queries. I've finally managed to pin point
where it hangs. If Netscape is using any reolver function, it won't
paint to the screen AT ALL. Depending on the speed of the DNS servers
you're making a query to/with, this can be quite a long time.

Furthermore, it hangs the Java Virtual Machine which apparently throws
the towel down and crashes Netscape in the process. 

> Anyone more exact about this observation?

It's been irritating me for quite a while. Whilst I've only taken a
precursory glance at the resolver libraries, I'm sure a multi-threaded
web browser should be able to:

* resolve DNS
* accept input from the user (eg the user realises that
"www.linuxzsa.org.au" is wrong)
* paint the screen at the same time

I'm not convinced that resolver queries HAVE to be blocking.
