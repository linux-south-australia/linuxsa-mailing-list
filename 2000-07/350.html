Re: Opinion: "Hidden Cron Jobs"

David,

I think a cron entry is rather like any other configuration file that an
application might require.  For example, the man subsystem should regularly
update it's whatis database, and I don't have a problem if it chooses to
insert a crontab.  Cron is a standard feature of the operating system and
I can't think of a reason to not use it.  (The subsystem should document
such jobs, and perhaps it might be quite obscure.  That's another issue.)
