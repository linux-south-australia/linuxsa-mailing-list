Re: Script Analysis help

David Drury wrote:
> 
> Hi,
> 
> i want to do some quick analysis of an apache log file. Essentially I want
> to search for certain string of data in the log file, and count or sum
> it. Ie, count the number of lines that a string occurs on, and also count
> the number of bytes transfered. For the purposes of what i want to achieve
> (quick statistics), I think a script of some sort would be helpful. I know
> most of what I want to achieve, but I'm short a command to achieve
> grep "some data" access_log* | cut -d (and some other options) |

How about 'nl' to number the lines and then cut the first field (the
line number).  That would produce what you wanted.. You could then go
from there to make the output in the right format for your app (like
piping it through 'tail -1' to get the last line (highest number).
