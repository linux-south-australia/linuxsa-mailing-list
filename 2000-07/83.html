On Mon, Jul 03, 2000 at 12:38:22PM +0930, Matthew Geddes wrote:

 > Does anyone know offhand whether it is easy to set up bind to return
 > name->ip mappings based on the address of the client? My copy of
 > O'Reilly's bind book doesn't appear to have much and I didn't see
 > anything in the BOG.

Are you trying to do this so that hosts inside your firewall see a 
different set of IP addresses for mail and web servers to the ones 
hosts outside your firewall see?

If so, why not set up a fake nameserver on the inside?

FWIW, it is possible to run two copies of named on one machine, with
one of them listening to one IP address alias and the other one listening
to another.  If you're looking at BIND 8.x, look at the "listen-on" 
phrase in the "options" clause of named.conf:  If you give two instances
of named different config files which specify different listen-on
clauses, they shouldn't conflict with each other (and if you think I just
mistyped "named.boot", perhaps it's time for you to upgrade to BIND 8.x
:-)
