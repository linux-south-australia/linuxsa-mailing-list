Re: best way to write a large document?

shaun wrote:

> the way i see it is that people have been using standard unix
> applications for years to write huge documents/ books, if they can do it
> so can i. I would like to find out the options though.

groff is very standard across most flavours of Unix. It converts very
nicely into postscript (which makes it look quite professional). The
best bit is that it is really quite simple to get started with. You can
use any text editor to create and I'm sure there's one out there that
does syntax highlighting (vi or emacs / emacs or vi).

LyX is also quite good. It's very good for technical papers.
