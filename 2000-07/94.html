SendMail and /bin/mail woes

Hi!

I've struck a bit of an odd problem. I have setup my sendmail.cf using
these macros:

OSTYPE(`linux')dnl
MAILER(`local')dnl
MAILER(`smtp')dnl

This produces a valid sendmail.cf file in that SendMail doesn't complain
when I load it. Now, the line for the local delivery agent is:

Mlocal,         P=/bin/mail, F=lsDFMAw5:/|@qrmn9, S=10/30, R=20/40,
                T=DNS/RFC822/X-Unix,
                A=mail -d $u


i.e. mailing to myself on my localhost should do:

[piped message] | mail -d lloy0076

However, I'm getting a really odd error. Sendmail believes that it is
sending mail to me and doesn't complain. However, the mail doesn't
appear. I've done some preliminary debug work and discovered that
/bin/mail doesn't work. In fact, it has a propensity to thrash my hard
drive until I manage to find its PID and kill it. It looks like it's
recursively calling itself.

Furthermore, I get these erros in /var/log/ :

Jul  3 16:18:32 localhost sendmail[8516]: QAA08516:
to=lloy0076@localhost.localdomain, delay=00:00:00, xdelay=00:00:00,
mailer=local, stat=unknown mailer error 1

Any ideas of why my local mail system has just "died"?
