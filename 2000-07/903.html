Annoying Problem

I run a local forwarding name server (BIND) with the following setup:

options {
  directory "/etc/named";
  forwarders { 203.20.69.2; };
};

zone "." in {
  type hint;
  file "db.cache";
};

My hosts.conf contains:

order hosts, bind
multi on

My host.conf contains:

127.0.0.1 localhost localhost.localdomain

My resolv.conf contains:

domain localdomain
0.0.0.0

YET, localhost.localdomain and localhost still cause a nameserver lookup
which fails and then finds my correct address. That is to say, the VERY
first time that I look for localhost or localhost.localdomain, BIND is
involved. I can tell because of the system monitor and the amount of
time it takes to look it up.

[I can tell when a name lookup is going to fail by looking at the system
monitor applet and the time it takes]

Once it's found it, it works really quickly.

Specifically it seems to be a Netscape problem but I think it's system
wide....any ideas?
