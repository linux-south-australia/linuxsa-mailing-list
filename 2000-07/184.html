Re: Hosting (Newbie Dumb Question)

Darryl Ross wrote:
> 
> On Tue, 4 Jul 2000, Daryl Tester wrote:

>> Why's that?  Sendmail handles multiple domains fine (or is this a
>> configuration thing?).

> Depends what you mean by multiple domains... Do you want fred@company1.com
> to be the same as, or different to, fred@company2.com ? If it doesn't
> matter and they can be the same, then sendmail will work fine (you just
> need to tell it to accept the extra domains as local).
> 
> If you want them to be different, then you need to set accounts up for
> fred1 and fred2 and tell sendmail to deliver fred@company1.com into
> fred1's box, and fred@company2.com into fred2's box. This is a VirtualUser
> setting.

See my subsequent post - I have sendmail delivering to a "virtual" POP3
server that has seperate user accounts (with their own passwords that
don't use /etc/passwd) for each domain.  When a user retrieves their
email, they connect to the appropriate IP address; the POP3 server
recognises the source IP address of the connection and uses the
appropriate domain setup.
