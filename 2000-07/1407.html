Re: cron question

At 11:24 AM 7/26/00 +0930, bmonten wrote:
>Hi,
>    I've set up a cron job to do, amongst other things, a system backup to
>tape every night. I'm doing the backup part with:
>echo "Daily Backup starting at `date`">>/var/log/dailybackup.log
>find . -depth -print|cpio -ocv -C10240 -O /dev/nst0 > /var/log/daily.backup
>echo "Daily Backup finished at `date`">>/var/log/dailybackup.log
>
>As this is a 'root' cron, the output of the job is, by default, mailed to
>root. The problem is that the backup part produces so much output, the mail
>file fills up pretty quick. I still want the output from the rest of the
>cron mailed, but I would like to suppress the output from the backup. How
>can I do this while still writing the /var/log/daily.backup log file ?

I think that this problem is due to the backup cron job sending output the
stderr, which is not redirected above. 

It may help to change the '>' redirection to '>&' so that stdout and stderr
go to the same place.
