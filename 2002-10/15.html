Re: linux distro on small HD

Hi.

Mine is using Redat 6.2. It didn't start as my headless server, it used to
be my desktop but when I built a new one I couldn't bring myself to kill
it, so it went down to the basement...

Which Distribution is a good question, and can spark off some heated
discussion on linuxsa. I think that if you can hunt down some more memory,
it doesn't matter much - you will be only loading core components, and any
of the big three (RedHat, Mandrake, Debian) will fit the bill. 

Without the extra memory, things might get interesting - I don't know for
sure, but I doubt that the current RedHat and Mandrake will install with
8mb ram. What you need is the distribution with the leanest memory and
disk footprint of install and operation, but I don't _know_ which that is.

I _suspect_ the answer may be Debian. Debian certainly has a lean install.
Others on this list may be more specific, and some time on the web will
help inform your decision.

Network issues for a Mac user?

Mac users don't usually have to spend much time futzing with the network -
it just works. Having a local TCP/IP network will challenge that status.

* Setting and using up a TCP/IP network.

* Lack of multihoming in OS9. - There is only one 'active' TCP/IP
configuration available. You can get around this by using a local
linux box as your internet gateway.

Someone else already suggested versiontracker for niftytelnet.

If you do decide to use the linuxbox as your gateway, you should switch
your linuxbox access from telnet to ssh for security reasons. (I'd do that
anyway)

Regards,

Michael
