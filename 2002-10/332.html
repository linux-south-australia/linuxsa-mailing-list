Re: Ghost 7.0 no good. Kernel Panic

On Fri, Oct 04, 2002 at 09:06:02AM +0930, Matthew Western wrote:
> PS:  perhaps ghost is not the thing to use here, is there alternatives?

Whilst less user friendly, I like to use tar. You just create a tar copy of
everything on the hard disk (except /proc/*) and store it elsewhere and then
use a boot disk with sfdisk, mke2fs and tar to restore it. It takes a little
while to set up, but it makes thing easier if you ever want to make changes.

Alternately, if the disks are in the same machine and are the same geometry,
you can try something like:

 dd if=/dev/sourcedisk of=/dev/destdisk bs=1024k

where sourcedisk would be the name of the source disk and dest disk would be
the destination disk.

Hope it helps,
