Re: Neomail under RedHat 7.3

"Jordan Noblet" <jordan@binarylogic.com.au> wrote:

> We tried doing it this way: /etc/procmailrc

> :0
> | chmod 660 /var/spool/mail/* ; chgrp mail /var/spool/mail/*

Bear in mind that I'm rusty on procmail recipes (haven't used it
in a while), but I don't think you want to run the above.  Unless
you have another rule that's doing the writing, this says pipe the
email into chmod/chgrp.  Also, there is no "i"gnore flag on the
rule, so if chmod/chgrp have errors (which they may do on a
wildcard, especially as procmail won't be running this as root),
then it will probably discard the email.

Try changing it to something like (warning: untested):

:0ic
| chmod 660 $ORGMAIL; chgrp mail $ORGMAIL

:0:
$ORGMAIL

> If I delete a users mailbox then send a piece of mail, it is delivered
> (owned by group mail, 660 permissions - fantastic!)  If I send any more
> mail, the logs look exactly the same, it says the mail was delivered,
> but it never arrives.

Enabling procmail's logfile might help here, but I've certainly forgotten
how to do that.
