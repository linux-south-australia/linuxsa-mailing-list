Re: Problem with PHP / MySQL RPMs

Quoting Michael Selge <mselge@hansenyuncken.com.au>:
> I have a problem getting PHP and MySQL to work together and was hoping
> someone can point me in the right direction.
> PHP itself was already installed on the server ( Redhat 7.1 )
> php-4.1.2-7.1.4
> 
> I have then downloaded the following RPMs and installed them
> 
> MySQL-3.23.51-1.i386.rpm
> MySQL-client-3.23.51-1.i386.rpm
> MySQL-shared-3.23.52-1.i386.rpm
> php-mysql-4.1.2-7.1.4.i386.rpm

Hmmm.  I have an RH7.1 install with MySQL and php running from rpms.

Let's see:
myhost:~ > rpm -qa | grep php
php-devel-4.0.4pl1-9
phplib-7.2-1
php-4.0.4pl1-9
php-mysql-4.0.4pl1-9

myhost:~ > rpm -qa | grep -i mysql
mysql-3.23.36-1
mysql-server-3.23.36-1
mysql-devel-3.23.36-1
mysqlclient9-3.23.22-4
php-mysql-4.0.4pl1-9

The big difference that I can see is your MySQL packages - the RPM filenames 
are the wrong case for Red Hat 7.1.  Are they Red Hat versions (see the output 
of rpm -qi MySQL)?  To get the linkage between MySQL and php to work correctly 
you're either going to have to install a set of RPMs built by the same supplier 
(ie Red Hat) or built from source.

I can't verify any further because most (all?) ftp servers appear to be 
flattened with RH8 downloads :)
