For this reason, my firewall script includes:

modprobe ipt_REJECT
# Reject ident requests (as opposed to drop, speeds up connection to
# irc)
iptables -A INPUT -p tcp --dport 113 -j REJECT --reject-with tcp-reset
iptables -A OUTPUT -p tcp --sport 113 -m state --state RELATED -j ACCEPT

You still do need an ident server for DALnet, though I do wonder why
they bother anymore--most DALnet users (that I encountered, anyway) use
mIRC, with it's faked ident server that only purpose is to allow you to
connect to the network. Ident was originally designed for multi-user
machines where it would be needed to determine who was using it at a
particular time and is therefore accountable for it's actions (for
example if the machine was used to do something illegal). At least
that's my understanding.

I think maybe it would be a good idea for ident to be handled by the
isp, or just killed off completely (not many people seem to use it these
days (that I know of) except to be able to connect to DALnet anyway).
