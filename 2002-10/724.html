samba name registration issue

Howdy all,

I've just installed RH8.0 on a box intended just to serve as a samba 
file server. I got it all set up and running fine, did a few tests on it 
(including a reboot test to see if it would come back up without any 
problems) - all good. I then moved it to our server room, only to 
discover no free ports on the hub - so I moved it back and hooked 
everything up again. Now, on booting it up, it no longer registers on 
our windows network under the name i've given it ("frogfoot"). 
/var/log/samba/log.nmbd give the following info:

[2002/10/14 09:16:31, 0] nmbd/nmbd.c:main(794)
   Netbios nameserver version 2.2.5 started.
   Copyright Andrew Tridgell and the Samba Team 1994-2002
[2002/10/14 09:16:31, 0] 
nmbd/nmbd_nameregister.c:register_name_response(109)
   register_name_response: server at IP 192.168.100.90 rejected our name 
registration of FROGFOOT<00> with error code 6.
[2002/10/14 09:16:31, 0] nmbd/nmbd_mynames.c:my_name_register_failed(40)
   my_name_register_failed: Failed to register my name FROGFOOT<00> on 
subnet 192.168.100.128.
[2002/10/14 09:16:31, 0] nmbd/nmbd_namelistdb.c:standard_fail_register(290)
   standard_fail_register: Failed to register/refresh name FROGFOOT<00> 
on subnet 192.168.100.128
[2002/10/14 09:16:33, 0] 
nmbd/nmbd_nameregister.c:register_name_response(109)
   register_name_response: server at IP 192.168.100.90 rejected our name 
registration of FROGFOOT<20> with error code 6.
[2002/10/14 09:16:33, 0] nmbd/nmbd_mynames.c:my_name_register_failed(40)
   my_name_register_failed: Failed to register my name FROGFOOT<20> on 
subnet 192.168.100.128.
[2002/10/14 09:16:33, 0] nmbd/nmbd_namelistdb.c:standard_fail_register(290)
   standard_fail_register: Failed to register/refresh name FROGFOOT<20> 
on subnet 192.168.100.128

So those "failed to register/refresh name" errors are the problem. It 
works perfectly if I explicitly set "netbios name = frogfoot2" in the 
smb.conf.

it's a totally vanilla RH8.0 install, with samba-2.2.5-10 according to 
rpm -q. it's running server-level security, with both the RH8.0 install 
and the samba config set to authenticate off the local windows domain 
(which seems to work fine - i can samba in with no hassles with the 
netbios name set to "frogfoot2").

has anyone got any ideas what could be causing this problem?
