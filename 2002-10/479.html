On Wed, Oct 09, 2002 at 09:32:43AM +0930, tnomail@stuarths.sa.edu.au wrote:
> Hi all
> 
> Any novell/linux gurus out there?  Novell has a really cool imaging system that comes with Zenworks based on syslinux.  When ever the machine boots it boots to a linux partition checks the server to see if any work is to be done and if not reboots to windows.  All was going sweetly until I upgraded from Zenworks 3.2 to 4. (pause for dramatic music)
>

Not me. I know very, very little about Novell (I'm running Netware 5.0 or
something at home). I have given a few suggestions though. :-)
 
> Problem 1.
> When I boot to the linux partition I recieve this message.
> 'linux' : terminal not known

Check the /etc/termcap file and the termcap man page. There should be an
entry in /etc/termcap for linux. If not, you may be able to copy it from
another linux box.

> Just playing around last night i got the same message to appear by using a 'setterm -term linux' command.  My linux knowledge is pretty limited but growing!
> 

You may even be able to change the above command to use another term type
(like vt100) depending on what the terminal needs to be able to do.

> Problem 2.
> If i go to run the imaging tool (img)I get this error message  
> 
> bus error

Usually indicates a hardware problem. Are you having this problem on multiple
machines? If so, it may just be a corrupt version of the imaging tool.

Hope it helps,
Matt
