Sue,

> Whaat info goes in what field?

1) Log in as "root" in a terminal or a console
2) Type:

# modprobe ne2k-pci

3) See what happens
4) If nothing happens (which is a good thing) or if it prints out a
message that looks like it found something (it may say, for example,
"Found foo card, IRQ X at IO 0xABC"), then try:

# ifconfig eth0 x.y.z.a netmask n.m.s.k

What where x.y.z.a is an IP address and n.m.s.k is a netmask.

If you don't understand that, try:

ifconfig eth0 192.168.0.201 netmask 255.255.255.0

...see whether the interface comes up (type ifconfig with no parameters
at all) and test it.

 5) To specify options to modprobe you can:

modprobe ne2k-pci io=XYZ irq=XYZ

...where the meaning of io and irq should be obvious.

As Darryl Ross has already inted, if you get the above working put a
line similar to:

alias eth0 ne2k-pci

...in a file called "modules.conf". If you need options then put some
lines similar to:

alias eth0 ne2k-pci
	options ne2k-pci io=XYZ iqr=XYZ

If you want to find out about IRQ's and interrupts then look at the
files in /proc called "irq" and "interrupts" (their case may be
different).
