RE: Crontab not running a process as background.

On Mon, 2002-10-14 at 17:57, Joseph Burford wrote:
> 
> I know what commands the script is making because I wrote it :-) What
> preplexes me is that the script itself runs from crontab ok, but the
> programs it is calling to run in the background die. Only when called
> from crontab, when run from command prompt it works as required.

Cron has only a certain subset of your environment when it runs - you
may be assuming something is set or available when it isn't.

See 'man 5 crontab' to see what environment variables are set.  Or just
log to a file (like in lloy0076's email) to see what is going wrong -
ie. use "/bin/bash  -kx script.sh >& /var/log/script.sh.output" in your
crontab instead of just "script.sh".
