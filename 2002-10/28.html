Re: Spam assassin - any good?

On Tue, 1 Oct 2002, Mcleod, Ian wrote:

> Does anyone have any success / failure stories to share about Spam Assassin?
> It looks like a *great* idea if it works..

Over the weekend I tested an installation of SpamAssassin at message
accept time. This means that SA is run against each line of the message as
it is accepted from the remote system, and the connection is immediately
closed with an error.

Advantages:

   - saves paying for an entire message to be downloaded and processed
     later
   - helps get people off spammers lists because it will be marked as 
     a failed delivery
   - uses less system resources

I did this with Exim 4 with integration work from Marc Merlin. Seems happy
so far.
