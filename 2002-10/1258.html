RE: Routing problem.

Tuesday, October 22, 2002, 8:08:22 PM, Drury wrote:


> Given the affirmative to the previous, then what kind of route commands do I
> have to type ?

> Route add <Internet route>
> Route add <172 route>
> Route add <10. route>
> Route add <192.168 route>

> does this seem right ?

Yes, except for the Internet route, you probably want to do something
like:

route add default <ip-of-adsl-gateway>

You *may* need to add this after dialing up your corporate connection.

The corporate routes should be made persistent by adding a -p after
the route command, eg:

route -p add 172.16.0.0 MASK 255.255.248.0 <ip of ppp gateway>

However, I'm not sure how well Windows handles routing to a ppp
endpoint or if you'll get the same ppp endpoint everytime you dial up.
