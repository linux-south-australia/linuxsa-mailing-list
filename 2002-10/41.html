Re: Spam assassin - any good?

Another possible alternative is shunning. 

You block the spammers ip address and notify the net block owner. I mean
block the address from sending email to you, redirect all web request
from that address to an explaination page with instruction on how to
contact you. It is necessary to block all ports requests to and from
that address because some spam contains web addresses for futher
information promotional material, confirmation of a valid email address.

If you dont get any response from the spammer or the net block owner,
you may consider blocking the entire netblock. It is a RFC requirement
for postmaster address to be read by humans or something to that effect.

I have read about this method before somewhere on the web about 18
months ago. I assume that there is a catch. ISPs may not be able to
follow this path but there is nothing that says that you must carry any
traffic on your private network as the end user.
