RE: VMWare and mandrake 9

Hi Bryan,

I posted a solution to this here last week. Try upgrading your bios. The
problem is that your bios is not enabling sse / simd support on bootup and
windows doesn't do it either. The cpu detection code in linux is correctly
identifying the cpu and that  it has simd / sse so it tries to enable it,
which it can't do under vmware.
After upgrading my bios, this problem went away and the machine also ran
alot faster.
SSE 2 was introduced into the AMD chips from the 1 ghz models, so older
bios's don't know to turn it on.
