Excessive memory usage blah

I have a Red Hat 7.3 server which is running an ERP package "PRONTO,"
Apache (which does very little,) and RRDTool has a cron job that runs
every 2 minutes that updates some statistical graphs.

Yesterday I configured RRDTool to graph the memory and I've since
noticed that of the 512Mb of RAM on board that machine, it is
excessively and frequently high, hovering around the 500Mb mark. 'top'
reports the following memory statistics:

  4:00pm  up 66 days,  2:52, 10 users,  load average: 0.18, 0.12, 0.07
86 processes: 84 sleeping, 2 running, 0 zombie, 0 stopped
CPU states:  2.3% user,  1.9% system,  0.0% nice, 95.6% idle

buff
Swap: 1052248K av,   25844K used, 1026404K free                  246540K

500Mb in use?  Blah!!


Pressing SHIFT-M and sorting the processes by % memory allocated tells
me that very little memory is in use.  Of all 86 processes, the highest
any ever reaches is 0.4%, and that is usually a PRONTO process.  There
are only ever a maximum of 16 users on the system at any time, the mean
being around 6 to 8 concurrently.  Many of these users are idling.

The point of activity at which this system is at highest load is daily
at 8pm when a script runs that tgz's the PRONTO database and uploads it
to a Samba server.

What could be the course of this excessive memory usage?  Do I have a
memory leak?  Are there any reasonable utilities out there with which I
can more accurately interrogate my memory usage?
