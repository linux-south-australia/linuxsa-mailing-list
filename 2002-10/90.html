Re: RH7.3 logrotate bug??

Matthew Thom wrote:

> Looking at this example and working through it logically
> I would think that the following command would delete all
> the files listed in the file 'tempfile'. If the find could
> find them all then the same selection criteria will delete
> them (someone please correct me if I am wrong). 
> 
> rm /var/log/*.[0-9]

You'll find that the "*.[0-9]" is expanded by the shell,
and that the shell's command line has a limited length,
well under 11MB.

Try

   find /var/log -name '*.[0-9]' -exec rm -f {} \;

the file glob '*.[0-9]' is expanded by the find command,
not by the shell (which is why it is quoted).
