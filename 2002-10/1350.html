Re: Portable server-side Java rare-to-impossible [LONG]

Dan Shearer wrote:

> Does this mean then that ports of Sun's versions of Java tend to be
> specific to a particular toolchain version (as well as to an OS
> architecture and a hardware architecture?)

Maybe not specific to a particular toolchain version, but definitely 
specific to particular a toolchain implementation. For example, pthreads 
are used for the "native threads VM" for both FreeBSD and Linux. The 
porting process was made considerably harder by not only the differences 
in the implementation and capabilitiesof the two pthreads libraries, but 
also becuase (AFAICT) HotSpot actually messes with the internals of the 
pthreads library. This obviously differs significantly between FBSD and 
Linux.

In fact, it won't really be possible to use HotSpot under FBSD until 
-current's libc_r (the FBSD pthread lib) is finally MFCed; -stable's 
libc_r just isn't up to it.
