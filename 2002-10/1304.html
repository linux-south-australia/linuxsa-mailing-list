> > way of spitting an RDP client onto a webpage in a manner that will
> > facilitate accessibility from a range of different platforms, I'd love
> > to hear about it.
> 
> Why a web page? Running something like a remote-access/thin-client type
> application, I'd usually prefer to run The Native Version of that application
> on each platform. Ie, rdesktop on unix/mac and the ms client on the Windows

Basically, I'm talking about the "what if". What if I decide that I
need to pull up my desktop and do this, that or the other when I'm at
a client's site, or if I'm stuck at a hotel or in the airline lounge?

Half the time, I *can't* run a little application to jump on into my
desktop environment; platforms, connections, local system policies
(amongst other things) all come into this.

Whilst I know that it's not an entirely fool-proof solution, it seems
to be the best solution for doing what I need it to do.
