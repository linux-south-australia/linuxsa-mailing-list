Re: Excessive memory usage blah

Adam Smith wrote:

> Yesterday I configured RRDTool to graph the memory and I've since
> noticed that of the 512Mb of RAM on board that machine, it is
> excessively and frequently high, hovering around the 500Mb mark.

That's really good, it means that the memory you have
paid for is being used to the utmost.  After all, if
you have excess memory above that required by applications
why not use it to cache disk blocks and improve system
throughoput.

Toto, we're not in Windows world any more :-)

> Pressing SHIFT-M and sorting the processes by % memory allocated tells
> me that very little memory is in use.  Of all 86 processes, the highest
> any ever reaches is 0.4%, and that is usually a PRONTO process.

So your applications have more than enough memory.  Good,
they won't swap to disk and slow things down.

> The point of activity at which this system is at highest load is daily
> at 8pm when a script runs that tgz's the PRONTO database and uploads it
> to a Samba server.

That sounds right, lots of disk blocks to cache at that time.

> What could be the course of this excessive memory usage?  Do I have a
> memory leak?

Does "ps" show a signle process's memory growing subtly
but continually?

> Are there any reasonable utilities out there with which I
> can more accurately interrogate my memory usage?
...
> $ cat /proc/meminfo

A very good utility.

> MemTotal:       514180 kB

> Buffers:        105452 kB
> Cached:         247312 kB

> Any ideas?

Have a read of the chapters about about virtual memory
and disk caching in a good operating systems fundementals
