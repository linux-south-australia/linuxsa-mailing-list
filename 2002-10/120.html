Re: IMAP and Windows clients

Dan Shearer wrote:

> You are quite right for realtime synchronisation. Where I use IMAP
> mailstore synchronisation is in two circumstances:
> 
>    1) Maintaining an identical backup
> 
>    2) offline updates (for a laptop) or remote site replication (you've
>       got a server at home)
> 
> Neither case requires realtime synchronisation. In both cases a filesystem
> with rsync is a natural solution.

Probably off-topic, since the discussion is really about
performance.

There are some nice IMAP-to-IMAP synchronisation tools about
as well.  These allow mail users without access to the mail
server command line to sync their IMAP folders.

I use one of these to sync the IMAP server on my laptop with
the AARNet IMAP server, then configure the laptop mail clients
to use the lsptop IMAP server.

The SMTP server on the laptop forwards to the AARNet SMTP server.

The result is nice disconnected operation using standard
protocols.

You can combine this with LDAP authentication to get nice
disconnected corporate authentication as well.  Scarily
another AARNet staff member can use a WLAN at a hotel to
e-mail my laptop's authenticating SMTP which will then wait
until it is reconnected to the Internet to forward the
message.

My point is that there is a lot to be gained by not using
back door protocols like rsyncing the mailboxes.
