RE: [OT] Watchdog eyes broadband limits

there isn't really a 'script that suits users in australia' as there is many
different applications.   but FYI, here's a sample .conf file.  from the
manual i might add :)

incoming all |in|eth0|all||
outgoing all |out|eth0|all||

or for a range (ips changed obviously):

10.1.1 total in |in|eth0|all|10.1.1.0/24|
10.1.1 total out |out|eth0|all||10.1.1.0/24

or for a specific IP:

10.1.1.2 total in |in|eth0|all|10.1.1.2|
10.1.1.2 total out |out|eth0|all||10.1.1.2

as i said i'm not sure if a newer version that i have supports protocols.
it's based on iptables so it could be done, it's a matter if the person has
got around to it.


for bandwidth throttling check out traffic shaping.  from memory i used a
think that comes with linux called tc

> >How does IP accounting work?
>
> install ipac-ng.
> make sure your iptables is running
> configure conf file (simple)
> do a fetchipac -S to do a first run.
> schedule a fetchipac every so often (1 hour is good)
> do an ipacsum to throw up a list of what's been where.
