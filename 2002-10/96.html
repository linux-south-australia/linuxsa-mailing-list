Re: OT: Hub or Switch

John Edwards wrote:

> A switch is better for security

Depends upon the switch.  The IEE802.1D standard says the switch
should flood packets if the MAC address table fills.  You need
a switch which allows that behaviour to be defeated.

Unfortunately, this turns what was a sniffing problem into
a denial of service problem (send lots of packets with different
source MAC addresses, take out the network switching).

So you also need to enable port security on each port with a
host attached.  This takes the port down if multiple source
MAC addresses are presented.  Some vendors require the port
to be manually reset if this occurs -- a pain in an enterprise
environment.  Other switches have a five-minute timeout.

If you are buying a switch to enhance network security
then you need to choose carefully and also accept a
higher administrative cost for the network (every installed
switch will require configuration).

Note that high-end switches can also enhance security one
other way.  You can have lists of allowed MAC addresses
(IEEE 802.1x allows this list to be in a RADIUS server).
So you can maintain a list of all the MAC addresses allowed
to connect to your network and refuse service to (and alert
network operators to) other MAC addresses.

You can maintain the RADIUS server from a restricted-access
web interface using standard ISP software.  Thus every MAC
address on the network to be tied to a user name and phone
number but it is relatively easy and cheap to add more machines
(such as visitors) to the network.  This removes the anonymity
of network usage, an important tool for detecting network
intrusion.
