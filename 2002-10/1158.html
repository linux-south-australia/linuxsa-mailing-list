Re: Is *nix and Linux inherently secure?

On Mon, Oct 21, 2002 at 11:47:06AM +0930, David Lloyd wrote:

> Most of the time it's been administered by administrators. The more
> users (ie mums and dads who have no idea about computers or security
> but want to use word processors and such) we get the less secure we'll
> become.

Not necessarily. There are still plenty of stupid administrators out there
(in both the Windows and the UNIX world). As for the "users", there's plenty
that can be done to minimise the risk of break-in at the distribution level.
User's should be able to install a distribution and have it set up fairly
securely. Take the example of packet filtering being set up by most
distributions (like RedHat).

Here are a number of broad areas where open source may have an advantage:

 * Code review
 * Used by those that wrote it
 * Architecturally better, in many cases, than other legacy operating systems
 * standards (even things like the FHS help indirectly)
