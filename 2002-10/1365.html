Re: Is *nix and Linux inherently secure?

On Fri, 25 Oct 2002 10:00:24 -0700
James Leone <LINUXCPA@netscape.net> wrote:

> Here is my opinion as to why Linux is inherently secure.  It is formed
> from a quote from Steven Crocker "Internet Pioneer", being interviewed
> about the recent DDOS "attack on the Internet."
> 
> http://www.computerworld.com/securitytopics/security/story/0,10801,75350,00.html
> 
> > As part of tightening up the basic DNS system, we need to deploy the
> > DNS security protocol [DNSSEC, a security protocol intended to
> > improve data origin authentication] and create a wider set of
> > implementations of BIND [the Internet Software Consortium's Berkeley
> > Internet Name Domain server software used for DNS]. I hasten to add
> > that lack of diversity is not actively causing any harm, and the
> > main reason for wanting diverse implementations is general good
> > practice.
> >
> So, by implication, if there were 5 open source type Apache projects, 
> "scalper" would not be as much of an issue, or perhaps, it wouldn't
> have been created.
> 
> Enter the diversity of the desktop. With choice of desktop
> environment, web browser, email clients, and applications, coupled
> with a kernel that requires a password and complicated compilation
> process, it would be a miracle if anyone could write a piece of
> malware that could scale this diversity.  Apache is one of the few
> Linux applications that has enough presence and exposure to spread a
> piece of malware systematically, and so, malware has been made for it.
> 
> Just my two cents, plus sales tax.
> 
> James Leone

This is true on the application level _and_ the kernel level to a lesser
extent. Are any two Linux install (sets) indentical ?
