Quoting Andrew Galdes <AGaldes@elders.com.au>:

> Hello,
> 
> How do i tell "mutt" or "mail" which smtp server to use. The system i am
> using isn't running an smtp server. And yes, i have checked man pages.

AFAIR, mutt and mail don't connect to a mail server. They invoke the sendmail 
process itself, which then handles the mail in its own way. Note that I'm only 
vaguely remembering this - I may be wrong.

I think there might be a sendmail clone which simply pumps the mail upstream 
without queuing, or you might be able to configure mutt to use a different 
process.

Note that sendmail doesn't have to run as a server with open IP ports - you can 
manually run the queue yourself, or tell it to run the queue every 30 minutes 
or so.

FreeBSD, from memory, has two sendmail configurations that it uses - one as 
a "true" mailserver, and one for handling outgoing local mail.
