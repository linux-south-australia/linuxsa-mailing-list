Re: Spam assassin - any good?

Mcleod, Ian wrote:
> Does anyone have any success / failure stories to share about Spam Assassin?
> It looks like a *great* idea if it works..
> 

I run it at home, and it seems to work well. I've set it up using the 
Spam Assassin's spamd daemon, spamass milter and sendmail. All incoming 
mail gets processed by Spam Assassin and if it scores higher than the 
threshold, it gets marked as being spam-positive.

The only thing I have the Spamass milter doing is adding a bunch of 
headers to the message with the details. People who have an account on 
my mail server can then filter the spam-positive messages out as they 
see fit, either by moving it to a "spam" mailbox, deleting it, or by 
bouncing it outright. This can be done on the server side using 
cyrus+sieve or on the client side using the email client's own filters.

The sendmail + spamass milter + spamd combo work really well, I've had 
no crashes from any of the three. The load on the machine is negligable: 
with several hundred messages per day, on a 900Mhz Athlon TBird which 
also does a bunch of other things - the load average is still three 0.00's.

> Urghhhh - I hate spam..
> 

Yeah, ditto. I figure my setup saves me from at least twenty or thirty 
spam per day. I have the threshold set to 8 (up from SA's default of 5) 
and while maybe up to five false negatives get through every day, I 
rarely get any false positives.
