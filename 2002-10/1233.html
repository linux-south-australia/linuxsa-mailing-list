Re: FTP reporting incorrect time

Craig,

> For some reason ftp clients logged into my mandrake 8.2 ProFTP server 
> are reporting file timestamps that are exactly 9 hrs 30 mins wrong 
> (adelaide's timezone)

Someone's timezone is set to UTC (or GMT if you're old fashioned like
me). I've seen this problem occur in Mandrake 8.2 before.

Check to see what /etc/localtime is a link to _and_ make sure that it
contains *something*. A friend of mine used a GUI tool to fix the
localtime and it sort of trashed the file that /etc/localtime was
looking at.
