32 bit colour rather pointless?

People,

I have heard many times that running X in 32 bit is rather pointless.
Indeed I was upset at first that my LCD laptop screen did not seem capable
of 32 bit colour - but the more I learn the more it appears that this makes
no difference.

David Lloyd kindly explained to me one night how 32 bit colour is *really*
only useful to professionals who require absolute colour accuracy and
matching.

So is 32 bit colour pointless for desktop users and really only for people
who use X or Linux for professional work - and then only with a decent
monitor?

On Wed, 30 Oct 2002 18:23:22   Nebular wrote:

"Since the freeware servers cannot do "packed pixels," they must burn 32
bits
of memory to represent only 24 bits of color information (some folks refer
to
this problem as "32 bit color," which is very misleading). We can do packed
pixels at TrueColor depth, which improves memory usage and data transfer
times. For chipsets that have hardware overlay capability, "32 bit mode"
refers to 24 bits of color -TrueColor - and 8 bits of overlay"
http://www.xig.com/Pages/Benchmarks/AGPBarCharts.html

And another more indepth explanation on this page:
http://www.xig.com/Pages/Ed-24bitPackedPixels.html

The above simply put, is that the only difference between 24 bit and 32 bit
is extra data for overlays. I don't believe XFree86 factors those 8 bits
into
the colour depth setting.
