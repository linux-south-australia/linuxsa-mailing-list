Quoting Matthew.Cook@tekelek.com.au:

> >Lazy programmers can also use maths functions to save writing
> >up lookup tables :)
> 
> But when a maths functions take 10 clock cycles and a lookup takes 4 which

This sort of thing is useless without proper context. How often is the function 
being used? Is it a bottleneck? Will writing a lookup table take up too much 
memory / take too much time / break other things?

It's better to use a math function the first time around then optimise it later 
if necessary, unless you know before hand that writing a lookup table is 
absolutely necessary.
