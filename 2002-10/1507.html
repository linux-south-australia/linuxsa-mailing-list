UDP vs TCP (was: Re: NTP Servers)

Mcleod, Ian wrote:
> Why is UDP less secure and how does UDP differ from TCP?  I know TCP is
> meant to be more reliable or something?  Universal Datagram Protocol (?) is
> very different?  Forgive my ignorance here if I am right off - but I would
> like to know how UDP is less secure and why UDP is used over other protocols
> at times..
> 

I am not sure about security, but UDP tends to be used where packet 
order, and delivery is not important - and/or packet overhead is. It is 
lighter weight (smaller headers, not so much (any?) handshaking/acks, no 
resends) than TCP - it doesn't guarantee delivery/timeout etc. so if you 
are less worried about all packets being received, and more about 
latency/response then you use UDP. (e.g.. position updates in games etc. 
If one update gets missed/delayed just use the next one, rather than 
waiting around for the first one - use UDP.
e.g.2. If downloading a file you want all packets to get through, and 
you want to be able to reassemble them in order - use TCP.)

Barney.

PS. I may be a bit vague as it has been 5-6 years since I last did a 
networking course, but you get the drift (I hope :) ).
