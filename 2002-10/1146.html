Assuming that:

1. There is a strong risk of you having no server at all if you hang
during install (or do so more than once, thus making the button-pushers
sick of you)

2. There is lots of MHz (I didn't bother following the hardware link you
posted)

3. The OS is complete enough to be locked down using packet filtering and
stable enough to run server processes without problem, and you can
eventually install a reasonably trusted version of sshd even if it is
painful due to Sun's policies

then,

I suggest you use the currently installed OS as a locked-down host for one
or more user mode linux sessions, running whatever distro you prefer. You
can NAT the external address(s) onto internal private networks. All your
system administration will then take place within the target distro(s) you
choose to run.

This gives a sealed box much more authentic than a chroot environment
gives you, and should make the underlying OS mostly transparent. 

And not having to reinstall might save you serious grief.
