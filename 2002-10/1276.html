RE: Linux powered water irrigation systems?

have you ever read or played with this kind of stuff?  i've done a little on
PLCs talking to stuff like this in the field in my last job.  turning on
pumps, valves, taking readings etc etc.  basically it looked like this.

PC (running SCADA* software of some kind, we had a winblows one and another
on running on Sco Unix.)
|
|
PLC (programmable logic controller)
|
|
|  (loads of physical wires running out to all the solenoids which are on
the pumps/vales/etc/etc to drive them)

i didn't have a whole lot to do with the PLC to field stuff, i was on the PC
end taking info from the PLC and doing reports, etc etc..  interesting
stuff.  we did one system with about 7 PLCs up in Moomba for Stantos.  they
we're connected via radio links.

the reason i woffle about all this is because, i'm sure sure that networking
cable or TCP is designed to do this kind of stuff.  BUT, don't let me
discourage you.

it's all feasable, if you have taps or tap controllers that are networkable
with TCP (ie able to give them an IP address or something).  there's no
reason that it can't be done, but the industry standard is not TCP, put it
that way.

Matt

*SCADA - Supervisory Control and Data Aquisition
