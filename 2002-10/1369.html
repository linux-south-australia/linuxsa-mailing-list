To be honest I haven't had much experience actually using these distributions
but I have put a LOT of time into looking at the alternatives for computers
with small hard drives.  

>would you have a reccomendation for a fullest/smallest (i
>know, i know, it's a trade off) console build?

As it was pointed out to me there is no reason why you couldn't use a popular
distribution such as Debian (or others) and choose minimal install options.  I
looked into Debian and it has a network install obtion where minimal packages
are installed and then other packages are added (untill you fill the hard
drive!). See http://www.debian.org/distrib/

I personally wanted to try and install on some minimal hardware (drive space,
RAM, processor) without a CD drive so I came across

Debian network install 
http://www.debian.org/distrib/floppyinst (20 floppies)

"You must have at least 12MB of memory and 110MB of hard disk space. For a
minimal console-based system (all standard packages), 250MB is required. If you
want to install a reasonable amount of software, including the X Window System,
and some development programs and libraries, you'll need at least 400MB. For a
more or less complete installation, you'll need around 800MB"  
-http://www.debian.org/releases/stable/i386/install

ZipSlack (Slackware compatible) (100MB) (fits on 24 floppies or Zipdisk)
http://www.slackware.com/zipslack/

www.peanutlinux.org
Then as I mentioned (600MB installed for 9.3) there is Peanut Linux which is a
good balance between functionality and size, although its main drawcard for me
is X-Windows and KDE.  Not bad for a 220MB download

There are plenty of smaller versions out there but it really depends on your
application.  I have copies of these distro's plan to do some more thorough
testing when exams are over, let me know if there is anything else I might be
able to help with.

Hope you don't mind me copying to list,

Nic Thiele
