As I said originally Ian, use nullidentd from
http://freshmeat.net/projects/nullidentd/?topic_id=150 to answer all
identd requests. Firewalling and identd is a special case since the
gateway box just recieves a request on port 113 but has no idea which
machine on the lan caused the request so cannot forward it because all
lan machines appear as the gateway's IP to the wider internet.

As for security problems, from that url.....

nullidentd is a minimal identd server. All identd requests are responded
to with the same (false) answer. It is intended as a very small
(possibly secure) daemon to run on a firewall for connections to servers
that use identd responses. 


So I hope this helps.
