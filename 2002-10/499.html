Glen Turner <glen.turner@aarnet.edu.au> on Wed, 09 Oct 2002 11:26:48 +0930 wrote:
> tnomail@stuarths.sa.edu.au wrote:
> 
> > Problem 1.
> > When I boot to the linux partition I recieve this message.
> > 'linux' : terminal not known
> 
> Syslinux is a boot loader for Linux from MS-DOS formatted
> disks.  If you boot from a MS-DOS floppy you should see the
> partition as a drive (you might need a CONFIG.SYS with LASTDRIVE=Z).
> 
> The Linux kernel doesn't care about terminals much, so it looks
> like the kernel has booted.  It's probably then running a shell
> and a shell script and one of the commands there has produced
> the 'linux: terminal not known'.
> 
> There is a terminal type called 'linux' that the kernel
> can provide.  If you've got a command line, type 'tset -q'
> to see what the shell thinks the terminal type is.
> 
> Have a look for that script from MS-DOS.  Don't change it
> from a DOS editor (DOS lines end in CR LF, Linux lines in
> just LF, so a DOS editor will stuff a Linux shell script).
> You might want to copy the partition somewhere and alter
> the script to undo the redirection of stdout and to print
> the script commands being executed (set -x, from memory).
> 
> > Problem 2.
> > If i go to run the imaging tool (img)I get this error message  
> > 
> > bus error
> 
> Very bad news.  This indicates an unrecovered error in the
> program source code.  There's nothing you can do about this
> in the short run except to try not to trigger the bug.
> Hopefully, addressing Problem 1 will make this go away.
> 
> > Any pointers would be greatly appreciated, particularly as
>  > school comes back on monday and I have 200 computers down!
> 
> I'd be backing out the change, the MS-DOS format of the
> partition could be handy if you need to do that manually
> (write a MS-DOS boot floppy to alter the ZenWorks installation).
> Alternatively, see if there's a floppy disk option for ZenWorks
> (that could be why they used syslinux in the first place).
> 
> Best of luck,
