Re: DHCP timeout on network interfaces on boot? Post boot 'ifup'?

This is a bit messy, and there is quite likely a better way but a quick
hack would be to move /etc/rc.d/init/network to
/etc/rc.d/init.d/network2

Then create a new network file (with executable permissions) that
contains:

/etc/rc.d/init.d/network2 &

You might not need the full path, I havnt tryed it. But that way the old
network file (now network2) will get forked and run in the background so
that the system will keep booting up while network2 sits and waits.
