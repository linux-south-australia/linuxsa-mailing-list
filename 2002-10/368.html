NFS Homedirs & X Logins

Hi all,

I have set up three Linux boxes on my network.  One will eventually
become my gateway and firewall (and it's working well ATM).

I have set up another (my primary fast box) as a NFS server for /home/*
running RH7.3.  I have the second box configured to mount a user's home
directory whenever a user logs in.  This works OK on the console, and I
can access all my files.

The whole thing falls in a heap when I try and log in with X.  It won't
let me. I get the message:

kdm[2876]: can't update authorization file in home dir /home/adamh

And it returns to the login prompt.  I figured it was because it was
squashing the root user, so I added no root squash to the export line
for /home and restarted the server.  Didn't help!

I presume that the error is talking about .Xauthority.  I've looked and
the file is owned by adamh:users and is rw by adamh.

This used to work (before I reinstalled the server), so what could be
wrong with it?
