-----BEGIN PGP SIGNED MESSAGE-----
Hash: SHA1

G'day.

There's a great package called x2vnc
(http://www.hubbe.net/~hubbe/x2vnc.html) which basically lets you use a
Windows + Linux box just as if they were dual screen.  Right now I've got my
keyboard & mouse plugged into my linux box, and if I move my mouse to the
left of the screen, it'll jump up onto my windows screen (running a vnc
server), and I can use the mouse and keyboard here just as if they were
directly plugged in.

Its *great*, having two mice & keyboards is a pain in the ass.

Thing is - its not completely ideal, as the reason I have a windows machine
is for my games (big EQ fan, grin), and as soon as I load these, I lose my
VNC connectivity, so I have to plug my keyboard and mouse back into my
windows box.

What would be nice is being able to run something on the windows box which
would let me have the keyboard & mouse plugged in here as normal, but if I
move the mouse to the right of the screen, it appears on my Linux desktop.

(In other words, same idea as whats x2vnc does, just the windows machine
"controls" the Linux box, versus other way around).

x2vnc can't do this though... anybody got any clues?
