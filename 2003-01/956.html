Re: test files

jonathan soong wrote:

> I was wondering if it was possible to create a file of size 2325566K 
> that wasn't actually empty, but was filled with junk data.
>

Use /dev/urandom in preference to /dev/random.

/dev/random is 'high quality' random data, whereas /dev/urandom is more 
like the junk you're after.

from the RANDOM(4) man page:
       When the entropy  pool  is  empty,
       reads  to /dev/random will block until additional environ­
       mental noise is gathered.

This makes it undesirable for large amounts of data that you would 
probably like generated in a short amount of time. Using random rather 
than urandom also slows down some cryptographic software like SSL web 
servers that host images and large files, but I guess it depends on your 
level of paranoia.
