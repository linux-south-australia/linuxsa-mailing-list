> I have an IMAP server that's not behaving in the manner that I'd
> expect it to, and a fairly unintelligent MUA. Neither of them run on

Get rid of groupwise :-)

> What I want to do is run a program of some kind on my IMAP server that
> will redirect localhost:143 to imapserver:143, and at the same time,
> log the session data (as in, all the commands and responses) to a
> file, so I can try and work out what the IMAP server is doing.
> 
> Can anyone suggest an application?

Yeah, don't do it this way.  Use ethereal (a packet sniffer) and use it to 
reconstruct the session.  Need to move the server and client to a hub and/or 
turn on port mirroring on the switch (if you can).

This way you're not getting in the way and you don't have to modify 
anything.

Aside from that a good application is ANY linux installer that rids you of 
groupwise ;-)
