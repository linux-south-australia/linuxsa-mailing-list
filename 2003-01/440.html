> What I want to do is to replace a particular text string with another in every 
> text file in a directory, and also in any text files within any 
> subdirectories.  Recursive replacement.
> 
> From the archives I've found that the following works well from the command 
> line for such replacement within the current directory:
> 
> bash $    perl -pi 's/targetText/replacementText/g' *
> 
> But it doesn't deal with files in any subdirectories.  For that matter it 
> doesn't deal with file names either. Does anybody know of a simple 
> modification to this perl line that will enable recursive replacement of text 
> within files as well as file & directory names, one or either ?

How about (untested):?

find . --exec perl -pi 's/targetText/replacementText/g' {} \;
