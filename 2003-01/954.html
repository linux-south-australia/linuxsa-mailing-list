Re: test files

On Thu, 2003-01-23 at 15:41, jonathan soong wrote:
> Hi, this is an easy one
> 
> I needed to create some junk files to test some network drive 
> performance and stability.
> 
> I was planning on using "dd" to create arbitraty sized file (500K to 
> 20Meg)..
> 
> Now when i did this using: dd if=/dev/zero of=new_file seek=100 count=1 
> bs=2325566
> 
> I get an empty file of size 2325566K.

It's not empty, as it's size is 2325566K. It is full of '\0's though. If
you want something other than '\0', try one of these:

 /dev/hda
 /dev/sda
 /dev/random
 /dev/kmem
 /proc/kcore

In the above examples, all but /dev/random have a finite amount of data
you can read. There's probably lots of others in /dev.

You can do something like:

  dd if=/dev/random of=/tmp/crud bs=1024k count=20

For a 20MB file.

Or, something like this:

yes 'testpattern' | dd of=/tmp/crud bs=1024k count=20

Hope it helps,
