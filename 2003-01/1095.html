Re: Dos and Samba

Ah, this type of thing brings back memories - mainly bad :-)

Kieran wrote:
>
> I have some computers that are just going to act as public access for a
> library. The library software is dos based, so I thought I would find 3 of
> the crudiest computers and load dos on them and network them up to my
Samba
> server that is acting as a PDC.
>
> Is it possible to log on to the server in dos and map a network drive so I
> can run the library software?
>
You'll need the tcp/ip stack for dos (I think it was the wfw 3.11 one?) plus
the drivers for your particular network card, plus the redirector (the bit
that lets you map drives). It was all quite primitive back then (people used
to make a living from getting all this stuff to work properly in a network).

They used to run a non-routable protocol (netbui? netbios?) but I dont think
Samba does that, so you'll need the tcpip stuff. Authentication and security
might be a bit of an issue too.

The other option is PC-NFS.

It's a long while since I've done any of this though...........

Good luck
