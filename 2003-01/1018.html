Re: MAIL ISSUE

Ronke,

You are correct: you should use IMAP because this keeps the mail on
the central server.  Microsoft Outlook works nicely with IMAP servers,
although I suspect that some of the collaberation features, such as
shared diaries, will only work with an Exchange server.  However for just
email, a unix server running sendmail and IMAP is an excellent choice.
You should also look at LDAP for a shared directory.  More important, the
IMAP protocol transmits the username and password in clear text, which
means it can be read by anyone who can monitor transmission between the
staff and your central office (ie any 14yo cracker.)  You can solve that
by using IMAP over SSL, which I think Outlook supports.  I believe that's
what Outlook calls "encrypted password".  I don't know how to set up an
IMAP server to support SSL.  I suppose it's a simple configuration option
on modern IMAP servers, but do be aware that the default is plain text.
