not necessarily the answer you are after, but, I have found that most
php scripts I have downloaded were designed to work with
register_globals On and the default php install now has register_globals
Off for added security when passing variables.
I havent checked the websight scripts at all, so it could be something
totally different, but it is a simple thing to try;
edit your php.ini, find register_globals and change the Off to On, save
and reload the page(s) in your browser, if it wworks, thats the problem,
leaving you two options, leave register_globals on or re-write it all to
work with it switched off.
I do believe there are also some lines of code you can insert into each
page to emulate register globals on, even when its off, check their
website for that one.

BUT, before you do anything, you should read about the possible security
implications of using register_globals, it should be on the php.net
website about here;
http://www.php.net/manual/en/security.registerglobals.php

alternatively you could post some of the errors to indicate where it is
going wrong?
