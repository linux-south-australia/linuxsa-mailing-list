Blocking Streaming Audio using Squid

Hi,

Got a problem in properly blocking users from consuming too much bandwidth
with Streaming Audio ( Especially now that we are in Cricket season ).

I have added the following lines to Squid.conf ( squid-2.4.STABLE6-6.7.1 )

acl BLOCK urlpath_regex -i
\.(mp3|avi|mpg|mpeg|asf|wmv|ra|rm|ram|asx|asf|afx|divx|m3u|viv|vivo|vo)$
http_access deny BLOCK

which work fine in some instances such as if a user trys to access
http://www.abc.net.au/streaming/triplej.asx

But if the ASX is spawned as you go to a site then it fails to block it.
http://cricket.wmt.hostworks.com.au/cricket
This site here Spawns Windows Media Player and starts streaming.

Without resorting to blocking the site altogether can I prevent ASX files
from streaming. Is it a better option to try and find what Ports are used
and just block them in Squid or at the Firewall. All Internet access
currently goes out Via Squid - No Direct http
