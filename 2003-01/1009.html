RE: testing the same data between two disk?

depending on how the mirroring works, you could try md5summing /dev/hda1
etc, but if even one bit is different on the fs (eg there might be a
flag that says which one is which...), then it won't work. If you want
something more than just an md5sum of each file, you could try using
something like veracity -- I think that checks timestamps and all that
too...
