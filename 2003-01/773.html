Re: Samba Virtual Hosting?

On Fri, Jan 17, 2003 at 10:21:01AM +1030, Benjamin Close wrote:
> Hi All,
>    I have one box with two IP's via a ethenet aliases on a Redhat box.
> Is it possible to setup samba so that one ip address appears to be a 
> different box to the other ip address?
> 
> Ie:
> 
>    eth0 192.168.1.2
>    eth0:1 192.168.1.3
> 
>      192.168.1.2  \\Carly
>       192.168.1.3   \\Ben

there are two ways of doing this;

- use xinetd to execute smbd for each request and get xinetd to listen
  on each IP, then pass smbd a different config file.

- run two instances of smbd each with a seperate config file, each
  listening to a different IP

the catch list;

- set *one* of these to a higher "os level" than the other,
- set them both up to listen to the same IP network mask list,
- set them up the same workgroup name (there seems to be horrible lag
  otherwise),
- if you are starting two separate smbd process make sure you create or
  modify the appropriate script in /etc/init.d/.. + /etc/rcx.d/...
