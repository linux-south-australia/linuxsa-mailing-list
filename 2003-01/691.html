I don't know a whole lot about the -650 cards, but I'd strongly suggest you
going to a 2.4.x series kernel.  My experiences with wireless on 2.2.x have
been flakey at best.

Bit of information for orinoco/cabletron/enterasys users (I'm guessing the
- -650 is a prism, so it won't apply...)

If you're keen to run something like kismet (kismetwireless.net- awesome
tool which will let you "stumble" on networks) you'll need to patch your
card with snax's diff to make it rfmon capable. I've got this running great
on my powerbook, running 2.4.20 and pcmcia-cs 3.1.34 (there are 3.2.1
patches, but I had problems with iwpriv, 3.1.34 runs perfect).

Usual way to go around this:

1. Grab latest 2.4.x tarball, 2.4.20.  Make sure you compile this *without*
kernel PCMCIA, but with wireless cards selected (but don't actually select
any wireless cards, not even as a module).

2. Grab pcmcia-cs 3.1.34, and the 3.1.34 rfmon patch from
airsnort.shmoo.com.  Patch as usual, compile, install.

3. I've noticed 3.1.34 sometimes doesn't put files in the right place, off
memory I had to copy over the init script (and make a few changes so debian
liked it), the stuff in etc/ to /etc/pcmcia (and again, make a few changes),
and copy the modules into the right place in /lib/modules/2.4.20/ (haven't
got my powerbook with me now, so I can't remember exactly where).

4. Restart pcmcia, throw your card in, it *should* come up.  Try running
iwpriv <device>, and see if you've got a "monitor" there down the bottom.
