Re: Bridging question

On Tue, Jan 07, 2003 at 09:54:42AM +1030, Andrew Reid wrote:
> On Mon, Jan 06, 2003 at 10:23:47PM +0000, Kim Hawtin wrote:
> 
> [ ... ]
> 
> > if think back to why bridging was used "in the old days";
> > - mixed media interfaces
> > - join together ethernet segments that were too long
> > - "we cant afford a router!"
> > - "its 1985 and we don't know what routing is yet!"
> > 
> > i think that if you are using bridging thatn you may need
> > to ask yourself why you are using it...
> 
> Bridging is still useful. Consider:
> 
> * Building A is connected to building C via building B. Building B
>   gets knocked down. Nobody wants to pay to re-cable building C, so
>   one installs a wireless link between building A and C. Bridge
>   between the wireless link and the ethernet in building C, and the
>   network appears as one big happy LAN again.
> 
> ...not that I had much success with that exact scenario recently, mind
> you. Some FreeBSD quirk, or a wi(4) shortcoming, or I was trying to do
> something that wi(4) wasn't designed to do.
> 
> Anyway, bridging is still useful every now and then.

bridging was an intermediate step introduced to patch a hardware
media limitation.

i would suspect that a routed network would make more sense in 
the example above.

especially that the groups of people working in building A and C
would be working in different tasks or groups and requiring different
services/servers, the need to be on the same logical ethernet segment
does not make a great deal of sense ... IMHO

but they will need to access common resorces. and the only thing
that will need to have access to both networks is a samba server,
the rest can be accomodated by sensible network design... and routing.
there are ways and means to get a samba server to talk "locally" on
distant networks ;)
