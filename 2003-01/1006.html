***Assuming Sendmail *is* installed on the system...***

OK,first of all it really helps to take the HOW-TO at hand:
http://www.redhat.com/support/resources/mail_news/mail.html

And the main website helps as well:
http://www.sendmail.org/

If you want the easy and fast way to do this, get a tool to help you (I
often recommend Webmin: http://www.webmin.com ) as this decreases the time
required to get it up and running. It helped me get things set up real fast,
after which I could tweak it at my leisure to get the exact results I
wanted.

Very short and rough howto tho:
1) Create an /etc/aliases file with all the email addresses and accompanying
user accounts. It already exists, so it should be easy to add your own to
that
2) Edit /etc/sendmail.cf
If you want your ISP's SMTP relay to handle outgoing mail put it's IP (or
name) behind the line that starts with DS
If you want a specific domainname to be used, put the name behind the line
that starts with #Dj$w.  (i.e. replace foo.com wth your own domain) and
remove the # sign in front.
These should be all the changes you'll need to do initially to
/etc/sendmail.cf
3) go to /etc/mail
4) Edit local-host-names to include the domain names for which you want to
receive email, one domein per line
5) (re)start sendmail
6) issue a newaliases command

You should now be able to send and receive email using this machine...
(Sendmail comes almost ready configured on RedHat...)

Still, for a novice I'd advice getting webmin and dig in to the specifics
later...
