apache-2.0.43 and mod_ssl

Hi,

Following my last post (where I tried to compile mod_ssl as module), I now
have recompiled my apache with the following (thanks to David de Groot):

./configure --prefix=/usr/local/apache2 --enable-ssl=shared --enable-so --en
able-mods-shared=ssl

After compilation, I have completed the following steps to generate keys and
certificate:

1.    openssl genrsa -des3 -rand file1:file2:file3:file4:file5 -out
server.key 1024
2.    openssl rsa -in server.key -out server.pem
3.    openssl req -new -key server.key -out server.csr
4.    openssl x509 -req -days 999 -in server.csr -signkey server.key -out
server.crt

On completion of  key and certificate generation server.crt and server.pem
is copies to the directory as indicated in ssl.conf and
Virtual_Host_default_:443 is updated as per my host detail.

My ssl.conf is attached herewith.  I have the following problems:

1.    Server does not listen on port 443, despite the fact that ssl.conf
file is Included in httpd.conf.  It listens only when I put it in httpd.conf
file.

2.    Server cannot repond on https://myHost, only on the normal port ie 80
or even 443 with no SSL.

I have tried this both on RH8.0 and Slackware8.1 with the same result. Have
I missed some steps?

Any pointers would be highly appreciated.
