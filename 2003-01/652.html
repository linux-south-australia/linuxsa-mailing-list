Re: Parental Controls on community systems

On Wed, 2003-01-15 at 21:28, Ian Loxton wrote:
> I want to provide a Linux system for the community to try out and it
> would have to be under the same restrictions. No way for the user to
> mess about with the system AND strong parental controls so that no porn
> could be accessed and no wallpaper changed. 

I always thought a better way was to allow free reign on
the system but simply "reinstall" the entire OS very
regularly to return to a known "good" state.
This works for linux as well as M$ of course.

But options in this case:
make it so you login then only run a web browser - so
not even a window manager
do something tricky like mount as much as possible
read-only so changes etc can't be saved.  This requires
crafty partitioning and sym links.  I've done it once
before under solaris.

Neither of these actually stop someone surfing to a porn
site, but I don't believe you can stop that anyway.
It would stop them changing the background though.
