Re: MAIL ISSUE

On Sat, 25 Jan 2003, davidn@rebel.net.au wrote:

[...]

> More important, the IMAP protocol transmits the username and password
> in clear text, which means it can be read by anyone who can monitor
> transmission between the staff and your central office (ie any 14yo
> cracker.)

Most IMAP servers can be set up to only allow authentication by
passwords that are not sent in the clear (ie CRAM-MD5/CRAM-SHA1).  The
problem is that outlook express will only authenticate using passwords
in the clear, or using SPA (aka NTLM v1), which I believe has serious
man-in-the-middle attack issues.

Even worse, although you may authenticate using, say, CRAM-MD5, and
believe your passwords are somewhat safe from casual eyes, all your mail
will be sent to you in clear text and can be easily sniffed, somewhat
obviating the need to sniff passwords :(


> You can solve that by using IMAP over SSL, which I think Outlook
> supports.  I believe that's what Outlook calls "encrypted password".

IMAP over SSL is better in that neither passwords nor mail are sent in
the clear.  Outlook Express has this option on the "Advanced" tab on the
account properties window ("This connection requires SSL" or similar).


> I don't know how to set up an IMAP server to support SSL.  I suppose
> it's a simple configuration option on modern IMAP servers, but do be
> aware that the default is plain text.

I've only used courier-imap with ssl support.  I had to edit a config
file and use a supplied script to generate a server certificate, very
easy stuff.

I believe that Cyrus imapd has ssl support also.
