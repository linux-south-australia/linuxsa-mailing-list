In the majority of cases, Windows machines are *not* cracked by people
"breaking in" because they lacked a firewall stopping incoming
connections reaching a machine.  Windows machines are cracked because
the user is tricked into running trojan code, or an operating
system/application flaw is exploited via ie/oe and/or AV software is not
up to date.

Once a machine is compromised, trojan code is free to make outgoing
connections to the world.  Mass-market 'firewall' NAT routers do not
block any outgoing connections out-of-the-box.  This is by design, so
that things "just work."

You can have a super-whizz-bang Linux firewall with lovingly
hand-crafted rulesets.  You can reject packets from naughty people
trying to break in to services that aren't even running on your machine.
In the end, you're still going to let web and email through, ie/oe are
still going to be compromised and your firewall didn't protect you like
you thought it would.

Don't get me wrong, firewalls have their place.  But firewalls do not
equal security.

Admittedly, a firewall configured to block outgoing connections except
those specifically required will prevent many trojans from delivering a
keystrokes/backdoors from the infected machine.  However, this is still
not a solution in itself.  If anything, the "my firewall will protect
me" belief causes people to be lax about the important things:

 - http://windowsupdate.microsoft.com configure Windows Update to notify
or automatically download updates unless you have a really good reason
not to.  (Control Panel, System, Windows Update tab)
 - Keep AV software up to date
 - Mozilla firebird
 - Spy-Bot

 >However, you may as well fix both these possible problems:
 >  - Running Windows Update should take care of OS vulnerabilities and 
 >running an updated virus scanner should get rid of Trojans...
