RE: Visual Basic Replacement for Linux

Good suggestion but check out Boa Constructor
http://boa-constructor.sourceforge.net/. It is an IDE (written using Boa)
which develops WxWindows apps using Python as the programming language. I
have installed it and worked through the tutorial on both Windoze and RH9.
It's surprisingly good even though the version is only 0.2.3. The fact that
a complex app like itself can be developed using it tells you something. Of
course it's open source.

Timothy,

application type: Configuration Management
Single User: yes
GUI focused: Yes
execution speed: Ease of use is more necessary than speed
userbase: gui background, knows what a button is
Hardware: sane/recent x86
Platform: linux

My first thoughts were learn C and choose KDE or Gnome/GTK.

But if you have a windows background you may (initially) be better off
design/coding in your native enviroment.

Then I'd suggest choosing a platform independant language with a
crossplatform GUI toolkit.

I'd suggest wxWindows as the GUI, because it has been around for about
10 years and is well rounded and quite stable. It works well on Unix,
windows and IIRR MacOS also.

As others have sugested, have a look at the IDE; IDLE with the language
python as your language. There is also perl and java. But you would like
to keep the rest of your hair =)

Python has bindings/libraries to all sorts of useful server
functionality. MySQL, PgSQL, mod_python for Apache, LDAP, XML-RPC, SOAP,
SNMP and then theres GTK, KDE, just to name a few.

There are lots of packages about now that use python for all sorts of
things. There are also plenty of Python book around too.

Have a look at;
  http://www.python.org/

GUI stuff;
  http://www.wxpython.org/what.php
  http://www.wxwindows.org/
  http://wxguide.sourceforge.net/

If you end up putting your configuration stuff onto a web environment,
try mod-python;
  http://www.modpython.org/

Also if you're into any funky graphics programming etc check out
pythong and SDL. Which are cross platform too =) all the links are on;
  http://pygame.org/

cheers,
