W32.Novarg not using MX records

Anyone sitting near an MUA or MTA today has probably seen this worm:

    
http://securityresponse.symantec.com/avcenter/venc/data/w32.novarg.a@mm.html

What's puzzling is that it doesn't appear to be using MX records to 
propogate. We use a system where the final MX for mail is not visible in 
the DNS, yet the Worm is delivering directly from infected systems to 
port 25 on this box.

The A record for the domain points to a different host that gives 
unreachables on port 25. This particular host is reachable from the 
Internet, but is not directly accessed by clients. The only real clues 
to it's existence are that it adds itself to the Received line of mail 
it delivers to disk, and bounce messages for undeliverable user mail 
will originate from it.

I've seen this behaviour before - when receiving spam, but it's 
perplexing to think that a worm running on an end user's system has been 
able to figure this out - unless it's going through the headers of 6 
month old mail from when the delivery host *was* an outbound SMTP relay 
for users..

Anyone else noticed odd behaviour with this worm?
