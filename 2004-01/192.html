Re: imap not working - authentication


On 05/01/2004, at 4:13 PM, James Dean wrote:

> The subject says most of it. New Debian server, uw-imapd, with 
> standard mailboxes. xinetd is listening on 143, and everything does 
> appear to be working, but I can't log in. Password rejected.
>
> A telnet conversation with the imapd server goes something like
>
> jupiter:/etc/pam.d# telnet localhost 143
> Trying 127.0.0.1...
> Connected to jupiter.
> Escape character is '^]'.
> * OK [CAPABILITY IMAP4REV1 LOGIN-REFERRALS STARTTLS LOGINDISABLED] 
> jupiter IMAP4rev1 2003.338 at Mon, 5 Jan 2004 18:38:45 +1030 (CST)
> a001 login test test
> a001 NO LOGIN failed
> a023 logout
> * BYE jupiter IMAP4rev1 server terminating connection
> a023 OK LOGOUT completed
> Connection closed by foreign host.
>

 From the UW-IMAP release notes:

"Support for the LOGINDISABLED IMAP capability.  If the IMAP server 
sends
  LOGINDISABLED as a capability, the client code will never attempt to 
send an
  IMAP LOGIN command."

So it would seem you may need to disable the LOGINDISABLED feature.
