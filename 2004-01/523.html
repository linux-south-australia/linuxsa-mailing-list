> Internode recommend the DLink DSL-300+ (modem) and DSL-504 (modem +
> router). I was planning on getting one of them, but:
>
> 1. Are there any special problems using these modems with Linux?
> (A quick web search suggests "no")

The 300+ in bridging mode works just fine for me, and with redhat 9.0 you
should have access to roaring penguin's pppoe software (`man adsl-setup`)

> 2. The following url suggests security problems with the DSL modems:
> http://xforce.iss.net/xforce/xfdb/10465
> Is this problem still current? Is it easy to fix?

I am sure it still is as it was with mine but I solved it by simply
telneting into it and changing it by following the instructions here:
http://archives.neohapsis.com/archives/bugtraq/2002-10/0355.html
 
> 3. The router in the DSL-504 provides a firewall. Is this worthwhile?
> (Or are the Linux software firewalls so good that it's not).
> 4. Is it easy to put in a firewall under Linux? I have Red Hat 9, 
> and I seem to recall something about firewalls during installation.
> The following url talks about Linux ADSL security, but it is nearly 3
> years old and refers to RH 6:
> http://www.skaht.com/linux/adsl/

RE 3&4:
I prefer iptables and I have not used the 504.  Iptables primer and examples
are here:
http://www.redhat.com/docs/manuals/linux/RHL-9-Manual/security-guide/ch-fw.h
tml#S1-FIREWALL-IPT
OR alternatively you can use the built in firewalls by accessing I think
`/usr/bin/redhat-config-securitylevel`.

Last but not least if you really want to understand iptables then I would
say read this:
http://iptables-tutorial.frozentux.net/iptables-tutorial.html
It is quite long but well worth the read in my opinion, and it also has
examples.

Chris-
