Power saving - slowed cpu still eats battery

Hi -

I'm trying to set up a laptop with a 2Ghz P4-M cpu to run at lower
speeds most of the time, to save battery. I'm running kernel 2.6. I'm
using the p4-clockmod scaling driver.

With the "performance" governor (echo "performance" >
/sys/devices/system/cpu/cpu0/cpufreq), cat /proc/acpi/battery/BAT1/state
says present rate: 20498 mW. With the "powersave" governor, the machine
gets noticeably slower, but the discharge rate is still 20109 mW.

Dmesg says:
cpufreq: P4/Xeon(TM) CPU On-Demand Clock Modulation available
spurious 8259A interrupt: IRQ7.
cpufreq: CPU0 - ACPI performance management activated.
cpufreq: *P0: 2000 MHz, 22000 mW, 250 uS
cpufreq:  P1: 1200 MHz, 9800 mW, 250 uS

...so I'd expect to be able to roughly halve my power consumption.

I've tried the "speedstep-ich" and "acpi" drivers as well - no
significant difference. "speedstep-centrino" won't modprobe.

cat /proc/acpi/processor/CPU0/throttling says "<not supported>", which
is a bit strange.

For what it's worth, the machine's a Toshiba TE2100, and everything else
(including swsusp!) works fine.

Can anyone help?

Thanks,

Gwyn.
