LILO does this correctly and simply by determining the mirrored devices and
writing to both of them.  The RedHat-recommended upgrade from LILO to GRUB
breaks this.

Confusingly, RedHat had managed to make initial installs work with GRUB and
RAID-1 but subsequent updates failed.

We no longer install GRUB on any of the 20-30 systems we maintain.

Install LILO, uninstall GRUB, in /etc/lilo.conf, instead of a line like
boot=/dev/hda use a line like boot=/dev/md0

At 08:38 AM 21/01/2004 +1030, Simon Lindsay wrote:

>Hello all, I ust had an interesting issue with my home server.
>
>Did an apt-get dist-upgrade from k12ltsp (basically rh9) to fedora. After
>removing a couple of packages it complained about, it did it no problems,
>which is pretty cool considering that before I've always booted off redhat cd
>to upgrade. After using debian, having to use the cd sucks.
>
>However, I then installed the latest kernel, which failed, and I had to 
>figure
>out why. It seems that grub-install doesn't support raid devices, and my root
>and /boot are on the raid.
>
>After some searching I discovered that to fix it you boot off the rescue 
>disk,
>chroot, start grub itself (grub --batch) and manually do "boot (hd0,0)" then
>"setup (hd0)" and off it goes again.
>
>It wouldn't be so bad if the grub-install just failed and didn't change
>anything, but my grub was screwed, so it must try, and then die ;-(
>
>This describes in more detail how to fix it, with command output.
>
>http://bugzilla.redhat.com/bugzilla/show_bug.cgi?id=55484#c10
>
>Just thought it might help someone in the future.
>
