Re: Visual Basic Replacement for Linux

Really depends on what the app is ..

For a big database management app, then turning the whole problem on
it's head and re-doing it as Apache/PHP/MySQL is worth exploring.

If it is a graphically intense application, with lots of realtime
controls, then :
- Glade for layouts / then use Python or PHP or C to glue it together.
- KDevelop if you love C++, the qt lib and the KDE look

Or for something really so different that no IDE will cut the mustard -
use SDL to create an interface from scratch.

There is a lot of noise about Eclipse for Java, and Forte for Java, so
if you are into Java, these might be worth a look at.

--------------

I personally hate IDE's - they are OK to get you started, but they just
get in the way after a while. A few XTerms and Vi sessions always gives
you more power than any IDE.
