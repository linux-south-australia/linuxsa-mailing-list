Re: PHP and Register Globals is EVIL

On Mon, 19 Jan 2004 20:00:31 +1030
David Lloyd <lloy0076@adam.com.au> wrote:

> 
> Hmmph,
> 
> I just spent an age working out why my sessions were randomly dying.
> 
> Here's the situation:
> 
>  1. Have register globals on
>     -> use the phpinfo() function to check
> 
>  2. Make a snippet of code like this:
> 
> ,..
> session_start();
> 
> $uploaddir="";
> 
> if (!isset($_SESSION["uploaddir"]) { 
> 	$_SESSION["uploaddir"]="something";
> }
> 
> print "$uploaddir";
> ...
> 
>  3. Under the normal scope of things, "$uploaddir" wouldn't go
>  stomping on
>     the SESSION hash
> 
>  ...however it DOES when register globals is on and produces some
>     particularly difficult to find bugs
> 
> SO:
> 
>  "If you are using PHP's session handling, it is best NOT to call your
>   user variables the same name as a hash vale in the session hash as
>   this could make your PHP application fall over if register_globals
>   is set on."
> 
> Hopefully this helps someone out there.
> 
> DSL

Perfect timing too, as I've just started on an app that will use
