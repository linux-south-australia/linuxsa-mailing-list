> >Quick question,
> >
> >Is it possible to filter/open ports on a firewall based on the
> >application in Linux ?
> >
> >Example - I have a program that uses random ports above 1024, so would
> >like to be able to give this program (and only this program) free run on
> >random ports over 1024.
> >
> >For those who have used many Windows based firewalls such as ZoneAlarm &
> >Norton Security, you'll be familiar with this idea.
> >
> >Cheers,
> >Rick

well, if you have root access when you run the application you might be able 
to do what I've done.
I wanted to have all ports stealthed to the outside world (aka ppp0) when I 
have no servers running, but open port 80 when I have apache up.
so I have all ports stealthed by default, and in rc.http, when apache comes 
up, there's an iptables command to allow port 80, and when it goes down, a 
iptables flush port 80.
for a variable port thing, have them all stealthed, then run the app, and use 
netstat or something to see which open ports are allocated to it, then 
iptables open them (this will only work if it's done any time the open port 
list changes, so if it happens a lot, you're either running an iptables open/
close port daemon of some sort, which has delays, or it won't work
