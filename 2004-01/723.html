Possibly, but not highly likely. You'll find that slackware probably
won't either. Many general purpose Linux distributions have a tendency
to use a reasonable sized ramdisk and quite a generic kernel. You'll
often end up having trouble on machines with less than 12MB or 16MB of
RAM.

What are you planning on running on the laptop? On a machine with that
little hardware, I'd build my own. It sounds like a danting task, but if
you know a bit about how your operating system works, you should be
fine. There's a replacement for glibc called uclibc
(http://www.uclibc.org/), which is designed to be quite small. This
coupled with busybox and friends (http://www.busybox.net/), you'll have
a fairly standard-looking linux command line (with a few notable
omissions) in a few MB at most. It'd only take 20 minutes on a fairly
average machine to compile the two from source and a bit longer to
transfer it onto the box (or less if you stick the laptop HDD in your PC
temporarily).

It's possible to compile XFree86 4.3.0 to use the Linux Vesa Framebuffer
kernel driver. Compiled, it's around 700kB and is suitable for remote X
connections, etc. You can run local X applications, but you'll need to
build library dependencies against uclibc, etc, etc. This is where it
starts to get a little time consuming.

Or you could install NetBSD or OpenBSD. Both of which seem happy to
install/run on a machine with that little RAM and disk. NetBSD in
particular happily installed on my MIPS DECStation, which is a 20MHz
MIPS-based box with 8MB RAM and 200MB of hard disk space. It used to run
Ultrix, but I fixed that :-)

Hope it helps.
