Hi

>I've got uw-imapd, postfix and procmail all going fine but
>imap performance is pathetic.  According to what I've read
>the problem is more than likely mbox and large mail files.
>  
>
We just moved from sendmail/mbox/imapd to postfix/maildir/courier-imap

Our load dropped from an average of (roughly) over 1+ (frequently up to 
5!), to an average of 0.5 (sometimes up to 2)..
On the same hardware..

Courier-Imap seems very good, we've been in production 3 weeks now, and 
no problems. I can't comment on
Cyrus, but i did do a lot of research before selecting Courier - it 
seemed (to me) to have better documentation.

>Is it as simple as installing the new imapd and running
>'makemaildir' in user home dirs?  any catches?
>(there's only 2 users and I will backup first etc)
>
Yup just make sure you do the makemaildir as each user (or chown em 
properly)...
You can use the mailbox2maildir scripts found at 
http://batleth.sapienti-sat.org/projects/mb2md/ to convert
existing mboxes... it works well..  (unless users have named mail 
folders weird names)..
