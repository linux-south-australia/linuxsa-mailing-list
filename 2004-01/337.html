Well, encouraged by all the useful tips on this thread, I converted my 
mail from mailbox to Maildir tonight, changing from UW-IMAP/POP3 and 
dovecot (IMAPS) to Courier-IMAP (imap, pop3 and their ssl equiv).

On the SuSE 8.1 machine, I had to compile my own Courier-IMAP as there 
are no SuSE 8.1 rpms for it, and I had to compile a new version of 
procmail (as 3.15.1 has known Maildir issues), and then I converted all 
the mailboxes to Maildirs (the mb2md perl script does not work on SuSE 
8.1 - but it does on SuSE 9.0, so I used a different conversion script 
that was somewhat more manual).

All in all though, it's gone fairly well. Only real issue is that the 
pop3 users seem to download all their mail on the first pop (which for 
some is several thousand messages). The imap users are all ok, the 
conversion script kept all the necessary flags to show which messages 
were read, etc.
