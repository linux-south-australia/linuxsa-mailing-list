smbmount of a subdirectory of shared directory

Hello,

I am using the following command to mount a shared directory from a
Windows-2003 server:

	mount -t smb -o username=myname //winfileserver/b/mydir
/mnt/mydir

The b is shared but not mydir. This command results in an error stating
that there is no such share. If I issue the following command it
succeeds:

	mount -t smb -o username=myname //winfileserver/b /mnt/mydir

Under Windows 2000 or newer you can mount any sub directory of a shared
directory.

In other words the command:  net use H: \\winfileserver\b\mydir will
have connect H: to the sub directory.

Is there a way to have smbmount connect as in my first example?
