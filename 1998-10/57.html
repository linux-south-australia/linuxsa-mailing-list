It won't be.  It currently is because it's new stuff, and it's using
new libraries which if you're not doing development you're unlikely to
already have.  You're likely though to have older versions of those
libraries, and upgrading them may break other stuff.

BTW, it's not a good idea to use --force unless you know what you're
doing (using --nodeps for an install is okay, though the program might
just not work once its installed).  If RPM is warning about conflicts
during an install, it's probably because you need to be using -U
(upgrade) instead of -i.
