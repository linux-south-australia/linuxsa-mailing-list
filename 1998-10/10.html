Re: linux / win95 dual boot using lilo

Prakash Sabapathy wrote:

> G'Day :
>
> On my computer I have
>
> /dev/hda : My linux hard-disk
> This drive is partitioned as :
> /dev/hda1 : The entire linux system
> /dev/hda2 : The linux swap space.
>
> lilo used to boot of the first hard-disk ( /dev/hda ) and straight
> into Linux. I have now added the second hard-drive and I would like to
> configure Lilo to give me the choice of which OS to boot into
> at boot-time.
>
> Just for information :
> /dev/hdb : My *new* WIN95 hard-disk
> Linux recognises it as WIN95 - FAT 32 and has labelled
> the partition : /dev/hdb1
>
> My /etc/lilo.conf file reads (at the moment):
> boot=/dev/hda
> map=/boot/map
> install=/boot/boot.b
> prompt
> timeout=50
> image=/boot/vmlinuz-2.0.31
>         label=linux
>         root=/dev/hda1
>         read-only
>
> What do I need change to existing system ( or lilo.conf file ) to
> allow lilo to either boot linux or WIN95.
>

Try adding to your lilo.conf,

other=/dev/hdb
        label=Win


then run "lilo"

Note: Which ever section you put first (other or image) ,
will be the system which is booted by default.

Another little trick it to add a message:
 In /etc/lilo.conf, add "message=/boot/message", (before the other and
image sections) edit a file
/boot/message to put whatever boot instructions you want your users to
see.
