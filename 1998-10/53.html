Hi All. Just got a quick question...

I'm running Redhat 5.0 and I need to run a web server and mail server for 3
domains. I've got the DNS server and Apache as the web server all set up and
running correctly. I've even managed to connect to my ISP with it and get
the masquerading to work. For those who can't remember me posting a few
weeks back, I'm running a permanent 33.6K modem Internet connection.

What I'm not so sure about is setting the Sendmail up. I can get it to
receive the mail for the users. That part was easy. I've also got the other
domains I host listed in the aliases file. What I want to know, is how can I
get another computer to be able to log in using POP to retrieve the mail?
When I try to connect to the server using Outlook 98 on my Windows machine,
the Linux box won't allow the connection.  Is it possible to have the use
the same user name for more than one domain but have different accounts for
them?  eg, admin@bachblue.com.au and admin@ross.wattle.id.au... And is it
possible to get it to use IMAP instead of POP?

Also, Sendmail seems to be stuck in a loop... I sent a test message on it,
and it seems to be going round and round in circles trying to do something
with this e-mail. I just deleted the file in /var/spool/mail which was
160Mb!!  And that wasn't the first time I've deleted it either. What's the
file I need to delete to stop the loop?

One other thing. It currently processes the mail every 2 minutes. Is there a
way I can get it to process it as soon as it comes in?

Thanks HEAPS in advance.
