Yeah, I'm pretty sure that happened when Apache went up to the 1.3 dist

(as all the 1.2xx ones I have still use the standard /conf).

It is actually looking for ALL the conf files in ./etc

(it was on mine anyway)


They also have a shared 'server root' directory that is a bit annoying

to work with (esp. if you've had your own way of doing virt hosts with
1.2)


However, I have actually managed to play with the 'Configure' util,

before making the package, to make it install as per normal.

(It still has all these superfluous dirs, just doesn't use them)

It just takes a bit of cafeful re-pathing.


At least they didn't make the little starter script mandatory,

"/usr/sbin/httpd -f /[location of conf]/httpd.conf" still works

like a champ :-)


ANOTHER QUICK TIP:

Another notable thing I found with Apache 1.3x was that the

syntax for ".htaccess" files has one subtle difference.

In a auth popup .htaccess file ('<<limit get>require valid
user<</limit>'), 

where the line entry for <bigger>AuthName could previously be done

as such:

...

AuthName My Auth Window

Authtype Basic

</bigger>...


You now have to use quotes (for that field only):

<bigger>...

AuthName "My Auth Window"

Authtype Basic

</bigger>...



I said it was subtle ;-)

But it had me wondering if my srm.conf (or which one) was set right,

because it just kept failing.
