If it says "permission denied" - you need to 'chmod 755' the script.

If it says "Internal Server Error" - you need to read the Apache error_log
for more details on the error.

To be any more help than that, I/we need to know what went wrong ;-)

Also, if you just set up the Apache, the you might not yet have the "ExecCGI"
flag on you script-dir config. It is in the srm.conf or maybe access.conf.
