Depending on how you configure your srm.conf and access.conf, the
.htaccess

is capable of overriding many webserver settings.

(eg: you can set a special error doc/URL for 404,403,etc errors within

the dir you put it in)


It works recursively from whatever directory you put it in.


They are ALSO capable of (and most commonly used for) setting access

controls on given directory trees .

(eg: those 'popup login windows' you get in the browser)


HOWEVER, in regards to your problem:

>>>>

<bigger>>as my  web server, and when I try to load up a users homepage,
eg, lynx  >localhost/~darryl I get an error saying permission denied.
Reading >through >the  .conf file I noticed it mentions a .htaccess...
help!!!   </bigger>


I think it is as simple as not having a 755 file permission on

the "/home/darryl/" directory.

(something I've noticed that most shadow suites don't do anymore when

you add a user)


In fact, by default, Apache _usually_ has the .htaccess overrides 

set to be capable of hardly anything (in srm.conf I think). So it is

really unlikely that it would be your problem.


Check the perms on that home dir, that should solve your problem,

but I'd still read up on how htaccess works.

IMOHO I think anyone who works with Apache should get around

to learning the .htaccess system anyway. It comes in damn handy ;-).
