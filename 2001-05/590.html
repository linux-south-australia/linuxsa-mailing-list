At 10:29 AM 5/17/01 +0930, Nicholas Morrison wrote:
>> On 17 May 2001 09:55:47 +0930, Nicholas Morrison wrote:
>> 
>> > Does anyone know of a browser for Linux that supports NTLM?  I've
>> > read about it at squid.org, and apparently it's a typically MSly
>> > revolting protocol...  squid has a module which will enable the
>> > squid *server* to request NTLM auth from clients, but I can't
>> > seem to find anything about clients being able to talk NTLM :(
>> 
>> You could install SQUID on the local workstation, even if you 
>> don't have root account (you can install it on an unpriveliged
>> port).
>
>So - squid will talk with NTLM to an upstream proxy after accepting
>'normal' authentication from a non-NTLM speaking client?

Hmmm, it might work, however, can you bypass the proxy? :-)

You would tell squid that the NTLM server is the NT Proxy in this case.
However, there is NTLM[v1] and NTLMv2, and the squid NTLM proxy auth stuff
might not know anything about NTLMv2 ... If your NT proxy and its auth
server (the PDC, most likely) insists on NTLMv2, you may be SOL.
