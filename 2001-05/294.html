squid ACLs

Hi all,

I've got a few ACLs set up under squid as a crude form of filtering. 
Basically, If the user is a valid user, they can access whatever they 
like as long as the URL they visit doesn't contain any of the strings 
given to squid. This is more effective than I thought it would be 
(obviously, no form of censorship is 100% effective). The problem I have 
now, is that the kiddies have found out that the proxy server doesn't 
check the CGI string in a HTTP POST. This means that they can then go to 
any of the banned sites using something like http://www.safeweb.com. 
Now, I could just ban that site, but word has it that there are a number 
of similar sites and it's hard enough to keep track of the sites I need 
to now. So, my question (finally) is:

Does anyone know a way of getting squid to check the CGI string in a 
HTTP POST request?
