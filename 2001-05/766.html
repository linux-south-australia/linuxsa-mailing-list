this wont work as it rightly says there is no table called user in the db
called crud.. the user tables for mysql are in the mysql database...
so use mysql (the database) tables...

I had a lot of trouble using the GRANT options (as advised by Andrew Reid)
even reading from a MySql book which recommended a

GRANT all ON crud.* to user@domain.nic IDENTIFIED BY 'theirpasswd'

so I edited the tables directly and created users there, in the user and db
tables, cross referencing this user to that db.. and that worked for me..
the only problem with that was the encryption on the passwd but we found a
tool (cant remember what it is but its in the mysql documentation) to
encrypt it..

but reading further.. mysqlaccess is the user tool.. so do a man mysqlaccess
and it should help you out..
