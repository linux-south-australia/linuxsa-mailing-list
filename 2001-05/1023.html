Re: FTP /noshell setup on REDHAT 7.0/7.1

On Wed, May 30, 2001 at 09:47:13AM +0930, Anthony Symons wrote:

 > I belive that the requirement for ftp is that the shell returns 'true'.

Incorrect.  

The requirement is that the user logging in through FTP needs to have a 
login shell which is listed in /etc/shells.

 > So we set the shell to /bin/true which just immediatly returns true and
 > exits. Thus ftp with no shell.

Check /etc/shells;  /bin/true is probably listed there.  You can set
the shell to any other value while still allowing FTP if you add that
value to /etc/shells.
