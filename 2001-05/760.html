The open sourced drivers tend to log events of a problematic nature - I don't
know about the Realtek drivers, having never used the cards (or likely to), but
I do know that the Intel pro drivers used to stall quite freqeuently whenever they
felt like it.  The "open sourced drivers" comment above was in reference to Intel's
own Linux drivers, which also stalled but never reported the problem.

If you want to find the problem, then it would be advisable to run tcpdump -n at
both ends of the link (I've seen switches foul up traffic before, so running it at both
ends helps reconcile what you're sending with what you're receiving).  The "-n"
prevents reverse name lookups, which may or may not be part of the problem, but
doens't hurt.
