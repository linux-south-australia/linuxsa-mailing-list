Re: cron jobs for services

On 25 May 2001 19:04:56 -0400, Corey Gilmore wrote:

> Whats the best way to monitor a process (that doesn't generate a pid file)
> and if its not running, restart it?

I suppose you could use 'pidof' which gives you the PID of the process
you specify on the command line.

For example:

--
#!/bin/sh

if [ `/sbin/pidof xemacs` ]
then
    echo "Xemacs is running!"
else
    echo "Xemacs is not running, starting..."
    xemacs
fi
--

This will check if Xemacs is running, and if it's not execute it. Try
adapting this to your circumstance.

It's not going to help if the program crashes and runs at 99.9% CPU
Usage, but it will help if it's not running.
