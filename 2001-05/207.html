two web pages is easy -- if a web server couldn't run 2 (or more) web pages,
you would have to have a new webserver per page, and that would be insane on
even a trivial site. How you do it? You put two html files in the root
directory of your web server....

two web sites is trickier -- you eitehr have two separate machines, or one
machine and two IP addresses (with two webserver processes running, binding
to the same port on different IP addresses -- and you can have one or two
interfaces to acheive this), or you have one machine, one IP address, and
two webservers on different ports (ie http://www.yoursever.com/ and
http://www.yourserver.com:8080/), or (finally), you have one machine, one IP
address and one webserver with virtual hosting...

The virtual hosting option is what you want (I think). You generally have
two DNS records pointing to the one IP address, (eg www.firstsite.com and
www.secondsite.com), and you set apache up so it knows about both names, and
it uses different serverroots for each name (read up about it -- I haven't
done it, I only know you can do it)
