Re: why "./" for cmd in current dir

"Freeman, Peter (ERHS)" wrote:
> 
> I'm trying to explain to someone via email why they need to use "./" before
> an executable
> file in the current directory (which isnt in their path of course)
> 
> just wondering what everyone's logical explanation would be

You can add . to your path and not need it, but... its for security so
that if someone puts, say a trojaned ls in some dir, and root does an ls
in that dir, you will still run the original version and thus it can
defeat this kind of attack. And if you want to run something in your
current dir, you know to explicitly do the ./
