>Does anyone know of a browser for Linux that supports NTLM?  I've
>read about it at squid.org, and apparently it's a typically MSly
>revolting protocol...  squid has a module which will enable the
>squid *server* to request NTLM auth from clients, but I can't
>seem to find anything about clients being able to talk NTLM :(

OK, I am not familiar with how Internet Exploder and the NT proxy use NTLM
authentication, but I do know a bit about NTLM authentication and how SQUID
uses NTLM, and they are an entirely different kettle of fish :-)

Squid accepts a username and password from the client and submits that
username and password to an NTLM authentication beast using one or other of
the libraries available out there. Squid needs, AFAIK, the plaintext
password to perform this trick.

I suspect that the NT proxy server performs pass-thru authentication, and
it is probably a standard NTLM challenge/response approach, where the
server sends a challenge (randomly generated), the client (IE) responds by
hashing the user's password, and then encrypting the challenge with the
user's password hash (two different methods available here), an returns it.
Server (the NTLM auth server) performs the same calculations, and if all
match, then the client is allowed to continue.

If this is the case, then the client will need to implement a lot of NTLM
client stuff, which is by no means impossible, since Samba does it.
