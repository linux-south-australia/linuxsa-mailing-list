We finally tracked down what happened.  The version of 'super' we installed
was a corrupted version.  Have since gotten the correct version and now all
works as it should.  The rpm that we had installed lists 'super' as a strait
755 root.root.  The correct version has it as 04755 root.root.  That's why
when we set 'super' chmod 04755 before it broke it totally.  Thanks for all
the help.  I guess you should always get the latest package DIRECT from the
source. :)
