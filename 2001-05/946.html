Re: Determining network traffic on an ethernet link

"Davis, Harvey" wrote:
> 
> Hello
> 
> I was wondering if there was a software solution for determining the netwrok
> load across a small (three nodes) lan.

Possibly not ideal, but you could use ipchains accounting rules. That
way you could measure different types of traffic (DNS vs HTTP for
example) as well as the total traffic for each host, etc, etc, etc. All
of the counters are accessible from /proc, so you can easily have a CGI
script return a nicely formatted list of real-time traffic data.

I believe there are also some "proper" tools available.
