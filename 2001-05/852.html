We use Amanda, configured for "tar" format.  There were some
issues with dump/restore formats, but I don't recall the
detail.

It was a mongrel to initially configure, mainly because of scarce
documentation.

Biggest gripe so far is that some distros don't put amrecover
on their rescue CDs, so if you lose a root disk recovery
is a nightmare.  If amrecover is on the rescue disk then
the total disk failure recovery scenario is:

 - install new disk
 - boot off rescue CD
 - fdisk
 - reboot off rescue CD
 - mkfs -t ext2 -b 4096
 - tune2fs -U UUID -L LABEL
 - amrecover
 - lilo
 - reboot off recovered disk

which isn't too bad and certainly a lot better than reinstalling
the OS.  Note that you need a paper log containing:

  partitioning information, for each disk N record the output of
    fdisk /dev/hdN
    p
    q

  disk label, for each filesystem NA record output of
    e2label /dev/hdNA

  UUID, for each filesystem NA record output of
    dumpe2fs /dev/hdNA

None of this is described in the Red Hat system adminstration
guides, which sucks as preparation for disaster recovery is a
fundamental sysadm activity.
