Re: Redhat 7.1 point of note

Stephen Donaldson wrote:
> 
> Just a point, it seems although Redhat 7.1 lilo and exceed the 1024
> limit, it can only do so using FDISK!

Bear in mind that a disk exceeding the 1024 cylinder limit
can't be booted on a machine with an older BIOS.

So it's probably not wise to use this feature in a
corporate environment, where you tend to move disks
around a bit when problem-solving.

> IMHO Rehat 7.1 is exceedingly good 

The major thing lacking out of RH7.1 is the Logical
Volume Manager.  I don't understand why this isn't
in the kernel by default and why the installer doesn't
do users a favour and automatically LVM the Linux parts
of the hard disk, especially in the "server" installation.

> (ReiserFS excluded sigh)

SGI have just released a RHL7.1 ISO image that uses their
XFS filesystem, so if you need journalling that might
be worth the look.  Looks ideal for use as a NFS/SMB file
server once any bugs are shaken out.  It also looks
good for notebook users, where abnormal shutdowns
tend to be more common than with desktop PCs.

XFS is a modern filesystem aimed at the high-end of the
market, so getting it into Linux is a real boon.  It
already has the infrastructure support for high end
features like Acess Control Lists and Heirarchical
Storage Management.
