RE: two network at network neighbourhood

> I have two network which is 192.168.1.x and 192.168.2.x
> the problem is the first network can't see all the PC at the 
> 2nd network

what is the subnet mask of the machines on these addresses?

If you are using a pure tcp/ip network, these machines will be 
unable to talk to each other unless they are on the same subnet.

the easy way is to use a subnet of 255.255.0.0 for both subnets,
but this depends on why you have two different subnets in the first
instance, if its because you have more than 254 PCs, then fair enough,
but if its for security reasons, I wouldn't recommend changing the 
subnet, but thats another issue altogether.........

your samba server can be set up with dual interfaces (either two
physical cards with an address on each range, or one card with
two virtual interfaces)
this can then be set up in smb.conf for samba to listen on these 
interfaces
however, even if your server is setup to see both subnets, that 
doesnt mean that PCs on each subnet will see each other, only the
server
