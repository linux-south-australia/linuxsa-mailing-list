Re: hda - irq timeout

I was running a firewall/router for a long time on a very old 100mb alps
hard disk (which strangely enough was still running when I replaced it
last year.) anyway it did it reported pages and pages of these errors on
the console, although it never corrupted any data and always worked. So
I dont know what the prob is in your exact case, but if you never find a
fix for it you can at least be sure that linux can deal with it and your
data will be OK (provided the drive doesnt damage the data itself for
some reason). I suspect the drive is meant to raise an IRQ after an
operation or something like that, but in some cases it doesnt and linux
has a sort time out waiting, times out, resets the bus and keeps going.
