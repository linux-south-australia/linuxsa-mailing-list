Re: why "./" for cmd in current dir

The PATH environment variable is a colon separated list of directories
that are searched for a command if you didn't specify the path.  That is,
if you don't type /path/to/command, or whatever and wherever the command
is, then the command is found by looking in the directories listed
in PATH.  If you don't have . (or a nul directory) in PATH, that is,
if it doesn't contain  ":.:", or start with ".:" or end with ":." (or
contain "::" or start or end with ":"), then the current directory will
not be searched.  In that case you have to specify the current directory:
command won't work, but ./command will.
