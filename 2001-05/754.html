Ssh creates a new key for each session, and since it uses large keys
this can take a long time -- not on a Cray Y-MP, obviously, because
they are very fast machines, but on the sort of machine that you and
I get to play with it can take, well you said it, a couple of seconds.
There are two modes for running Ssh: In one mode it is started by inetd
every time somebody tries to connect, and before it can present a login
prompt it has to create the key.  In the other mode it runs as a daemon
and accepts connections for itself.  In daemon mode it creates a new
key on startup, thus when somebody tries to connect it can present
the login prompt immediately.  In daemon mode it also creates a new
key every session, but that key is for the next session to come in,
not the session that it just accepted.
