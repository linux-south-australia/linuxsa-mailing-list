Re: Socket Programming

OK, you seem to be using TCP.
Did you see this in the send man page?

       <snip>
          If  the  message  is  too  long  to pass atomically
       through the underlying protocol,  the  error  EMSGSIZE  is
       returned, and the message is not transmitted.

It shouldn't be too hard to make a little while loop that keeps
retransmitting in smaller sizes as long as you are getting a EMSGSIZE error.
I suspect you will find it is directly related to the size of the TCP
socket's send buffer (which is adjustable, but only up to about 64k max i
think?).

Let us know how you go.

Toby
