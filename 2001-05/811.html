Sendmail Question

Hmmmm...

I recently fell over with RedHat 7.1's default Sendmail install. It
appears that it uses TCP Wrappers and I had:

# cat /etc/hosts
127.0.0.1 localhost.localdomain localhost

# cat /etc/host.deny
ALL: ALL

# cat /etc/hostsallow
ALL: LOCAL

Now, "LOCAL" means any name with _no_ dots in them. And note that
"localhost.localdomain" has a dot. In /var/maillog I had a message
saying that TCP Wrappers had disallowed the connection.

Has this always been RedHat Sendmail's behaviour? (i.e the behaviour of
sendmail when distributed by an RPM made by RedHat)...
