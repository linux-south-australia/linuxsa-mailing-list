Title:

    iptables: the brand-spanking new linux 2.4.x netfilter/firewall.

Abstract:

    This talk traces the path of an internet packet through the linux
    2.4.x kernel. Includes the new netfilter architecture, NAT, packet
    mangling and connection tracking.  Also some notes on the
    diffserv/QoS/CBQ/scheduling software, and some basic hints on
    writing iptables firewall scripts.

Provisional summary:

- the literature, references
- Internet protocol basics: packet formats
- netfilter architecture overview:
    - the 5 "hook lists"
    - the 3 standard tables: filter, NAT, mangle
    - the 5 standard chains:
      INPUT, OUTPUT, FORWARD, PRE_ROUTING, POST_ROUTING
    - input routing, output routing
    - backwards compatibility to ipchains, ipfwadm
    - layer 2: device drivers
    - layer 4: TCP, UDP, sockets etc.
    - queueing disciplines (Qdisc), classes, class-filters
- location of netfilter in the kernel source tree
- the long and winding road(s) of IP packets through the kernel
- the net filter API
- iptables scripts: simple examples
- /proc/net access to net filter
- general banter and garrulity

Notes:

- This will not be an iptables script-writing tutorial.
  There are plenty of script recipes on the net.
  This talk concentrates instead on what it all means.
- Queueing disciplines, diffserv, QoS etc. will only be mentioned
  briefly.
- The main focus will be on elucidating the paths of packets through
  the IP layer, and relating this to iptables script writing.
