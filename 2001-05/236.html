Re: X over ssh and colour emacs-nox question.

On Tue, 8 May 2001, Andy Zivkovic wrote:

> Hi,
>
> Can emacs-nox do colour when I turn on font-lock-mode? it's obviously not
> essential, but it's useful. :)

I don't think so... but xemacs is also a character mode app, and it
supports color on the console. I just tried it. That may be an option.

If for some reason you want to run it in an xterm, but not use the GUI
version, just:

unset DISPLAY
xemacs <filename>

But reading the below I guess you don't have admin rights on the machine
you want to run your editor on.

> It's possible to send xwindows data through a ssh connection, isn't it?
> Would it work even if I'm sitting at a computer behind a masqueraded
> connection, or would I need a 'real' ip address? Also, would sshd at uni
> need to be configured to allow it, or is there something I can try when I
> log in try enable it, if it isn't already working?

The remote sshd may deny it.

Running Xclients over any modem connection is pretty painful though. Try
it though:

ssh -X username@uni.server.edu.au

If the remote host disabled X11 forwarding, you will get a message telling
you so.

If it doesn't, run an X app:

xeyes &

and see what happens.
