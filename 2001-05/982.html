Re: Found out why Connect.com does not handle PPP from Linux ...

Jason Tan wrote:

> I _think_ that some terminal servers have software that allows a hand
> login and then starts ppp at their end, or you start it manually - cant
> recall which.

Most terminal servers have this functionality, since they are "terminal"
servers after all, where a terminal is traditionally a unix box. In an
awkward (for tech support) period around 1996 a lot of ISP's were
upgrading to terminal servers from older unix dialup pools, and it was
important to maintain backwards compatability (trumpet winsock,
anyone?).

AOL still use a system not unlike this today. On dialup, a telnet
session is created back to base, with a remote server handling all of
the interactive component of the connection. It could just as easily be
telnet to a unix login prompt.

Some terminal servers even have the feature to do a basic ascii menu for
dialup terminal sessions, and others offer the ability to create a
virtual serial link across an IP network for legacy equipment like
pokey/slot machines.
