Re: Official Release of Linux XFS

Alan Kennington wrote:
> 
> The trouble is, of course, that if I want to change the FS on
> a 24/7 server on the net, changing the FS is not something
> one does in a quick midnight swap-over.

Easily done with a second disk.  You can tar the disk being
replaced, switch to single user mode, use the incremental tar
option to update the disk again, change /etc/fstab to point
to the new filesystem, and go back to multi-user mode.  ie,
basically the pre-LVM procedure for resizing partitions.

Do watch the tar options -- check ctime/mtime, ownership,
permissions, and symlinks.

Finally, transactional filesystems don't work optimally
in all cases.  You do need enough idle time for the
journalled transactions to be applied.  So there's likely
to be a elbow in the performance graph where the I/O rate
falls significantly beneath that of the standard filesystem
when a large number of files are being updated.  In short,
I'd load test news server and maybe web proxy performance.
