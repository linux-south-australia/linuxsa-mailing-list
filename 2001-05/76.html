Re: Official Release of Linux XFS

Alan Kennington wrote:

> People have been saying, though, that Reiser FS was just a stop-gap
> while waiting for ext3fs and XFS.
> So I'm regarding it more as a learning experience than anything.

Fair enough. I haven't seen any problems with ReiserFS myself. It would
be nice if it supported ACLs, but apart from that....

> The trouble is, of course, that if I want to change the FS on
> a 24/7 server on the net, changing the FS is not something
> one does in a quick midnight swap-over.

No, but it can be quite painless. It's possible to do a great deal of
preparation beforehand (we recently implemented ReiserFS on a couple of
live mail servers, a proxy and two heavily used file servers).

> By the way, it seems to me that I've never heard of the MS OSs
> having any journalling file system.

I believe NTFS was supposed to have journalling capabilities (according
to the propaganda I've read). It's just that it's a pretty crap
implementation.
