Re: Internet Problem

Jason,
I had a thought 
quoting Dave Rowlands
"The new 2.4 kernels support a new feature of TCP/IP called ECN. Not all
routers support this yet and cause the packets to be dropped. It appears
that the routers used for Telstra ADSL don't support this, so running a
vanilla kernel 2.4 installation of linux over ADSL can cause problems with
connectivity. I found this snippet on the LinuxSA mailing list:

"Basically, there's a TCP option available with the 2.4.x kernels called ECN,
which most routers grok but some don't ... obviously the router or whatever
in front of the telstra machines I was trying to get to was throwing away
any packets with ECN on. So, all I had to do to fix the problem is switch
it off in the kernel config, or touch a file in /proc:"

ECN is screwing BP customers with 2.4 kernels. I am wondering if this is
the case.

echo "0" > /proc/sys/net/ipv4/tcp_ecn"

At what point is ECN introduced into the packet? and is it retained after
NAT massages it?
