RE: tape backup solutions

> How have you found Arcserve? I was looking at it at one stage 
> to provide
> multiplatform backup, ie:
> 
> +--------+        +--------+        +----------+
> | Novell |------->| Linux  |<-------| Linux W3 |
> +--------+        | Backup |        +----------+
>                   | Server |
>                   +--------+
> 
> Essentially centralising backup to the one machine, which palms it off
> to tapes. I'd imagine there may be problems (or 'issues') 
> when you want
> to restore something from the WWW server which is half way through the
> tape. Don't know for certain, however.


Arcserve handles this well, though we don't have any Novell servers.
(I used to admin a NW4.1 server env. with Arcserve, ran well there also)

We started using it here with the Linux box as the main backup location and 
also backing up our NT box (main PDC) using the client.
This worked very well, but since we've moved all our data to the Samba
server and are only using the NT PDC as login/proxy authentication, we
couldnt be bothered wasting tape space on it.  We just back it up every
now and again cause nothing changes (apart from users, & could restore 
them from Samba if need be)

The one thing I love about Arcserve on Linux is the web client.  It beats
VNCing into the NT box to configure it :o)
