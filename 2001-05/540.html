Re: why "./" for cmd in current dir

"Freeman, Peter (ERHS)" <Freeman.Peter@saugov.sa.gov.au> writ:

> I'm trying to explain to someone via email why they need to
> use "./" before an executable
> file in the current directory (which isnt in their path of course)

So that they can execute it - you're telling shell/exec where to
find the executable (because it isn't in their path).  DOS et. al.
includes the current working directory (even with PATH unset), which
is a wonderful security flaw.
