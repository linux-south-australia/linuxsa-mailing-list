Samba Problem (longish)

Well  since  everyone  else is mentioning Samba, I thought I'd jump on
the bandwagon myself.

I  am  running a SuSE 6.4 server which was running Samba 2.07 (default
from  SuSE). This box acts as the master browser and domain controller
(well to an extent anyway) for my little lan.

For  some  time  now the Win2k clients have been intermittently losing
drive  mappings  to  the  samba  machine. (however, TCP/IP connections
don't drop out).

So, SuSE released Samba 2.08 rpms last month, and I upgraded. However,
this has not solved the problem.

Any ideas on how to fix this would be greatly appreciated.

Below is my smb.conf for your learned perusal ;)
(Note:  the  netlogon share is not used by Win2k as I've set the logon
script to run from the profile directory in Win2k).

# Global parameters
[global]
        workgroup = VIKING
        netbios name = ERIC
        interfaces = eth0
        encrypt passwords = Yes
        update encrypted = Yes
        map to guest = Bad User
        keepalive = 30
        logon script = Startup.bat
        domain logons = Yes
        security = user
        os level = 65
        preferred master = Yes
        domain master = Yes
        wins proxy = Yes
        wins support = Yes
        kernel oplocks = No
        socket options = TCP_NODELAY

[netlogon]
        comment = The Domain Logon Service
        path = /etc/smb/netlogon
        browseable = No
        writeable = No

[homes]
        comment = Home Directories
        writeable = Yes
        create mask = 0750
        browseable = No

[printers]
        comment = All Printers
        path = /tmp
        create mask = 0700
        printable = Yes
        browseable = No

[share]
        comment = Shared Area
        path = /share
        valid users = +share
        read list = +share
        write list = +share
        force group = +share
        writeable = Yes
        create mask = 0665
        force create mode = 0665
        directory mask = 0775
        force directory mode = 0775

[varlog]
        comment = var log
        path = /var/log
        valid users = +share
        writeable = No

[backup-www]
        comment = usr local httpd for backup purposes
        path = /usr/local/httpd
        valid users = david
        writeable = No

# -- End of Conf file
