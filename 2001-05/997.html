RE: Help on SSH

Tuesday, May 29, 2001, 1:10:36 PM, Chandra wrote:
> ---
> what is the method to do a secure Telnet?
> I have installed SSH on my RedHatLinux 6.2 PC with the following rpm's.

> openssl-0.9.5a-3.i386.rpm
> openssh-1.2.2-1.i386.rpm
> openssh-server-1.2.2-1.i386.rpm

Firstly, you should update these to the latest.
OpenSSL I believe is at version 0.9.6
and OpenSSH is at 1.2.9 something

> Also  I  have  downloaded  Secure  Tera  TermPro and installed on my
> windows  PC.  If  I run this program it asks for the kind of service
> which  has to be enabled. I click on SSH and the TCP port#22 is also
> enabled.  After  I enter the Host Name and Click Ok I get a SECURITY
> WARNING message.

Yep, sounds right.

> The message says " There is no entry for the server "mc5878" in your
> list of known hosts. The machine you have contacted may be a hostile
> machine pretending to be the server.

> If  you  choose  to  add  this  machine  to the known hosts list and
> continue, then you will not receive this warning again

> Add this machine and its key to the known hosts list

>           Continue              Disconnect"

> If i say click on Continue, Again I get some Warning Message.

There's  a  tick box that says, add this host or some-such. Tick it and
press Continue.


> "The  host  and its key cannot be added, because no known-hosts file
> has  been  specified.  Restart  Teraterm  and  specifu  a read/write
> known-hosts file in the TTSSH Setup dialog box.

Ah, but first, create a file called ssh_known_hosts and put it in your
terra-term  directory.  Then in ttsh/terra term, go to the setup menu,
click   on   "SSH"   and  where  it  says:  "SSH  Known  Hosts"  add:
ssh_known_hosts   to  the Read/Write field. Save your settings and try
to  connect  again  (don't forget to tick "Add this host to your known
hosts" or whatever, and click Continue when prompted.

> Can any one correct me where am i going wrong and what more needs to
> be configured in this regard.

See above.
