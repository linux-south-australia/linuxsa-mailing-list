Re: Central Authentication Server

Corey Gilmore wrote:
> 
> At work we currently have about 20 PC's dual booting win98/linux (rh 7.1),
> and 5 mac's running OS8.6.  What I need is some sort of method to allow
> ANYONE to login to any of those, and have several drives mapped.  The
> drives would be on a server running NT4, and several linux servers with
> kernels 2.4.2 and 2.2.14.

Your best option is probably Novell's NDS.  It's changed
name and the server has just beeen made free (beer).  They
have an authentication module for Windows and UNIX platforms
can authenticate against NDS using LDAP or NIS.

The NDS server runs on Linux.

Have a look at Samba and netatalk for file and print
sharing.  If you set the "non-encrypted password" registry
setting for Windows then Samba can authenticate using
Pluggable Authentication Modules (and thus against LDAP,
NIS, NDS, etc).

All of this is much more work than it should be.  That's
partly because of a succession of security models from
Microsoft, none particularly good until Win2000; and
partly because that company's documented strategy or
limiting the growth of file share and authentication
on free platforms by undocumented protocols and restricting
the availability of documentation.
