Re: Library issues ...

Richard Sharpe wrote:
> 
> Hi,
> 
> I am trying to fix some problems in OpenJade because of the documentation
> work I am doing. I have managed to get the C++ code to compile (after
> fixing some problems) despite not being a C++ person, however, I am having
> linking problems.
> 
> I have the installed RedHAt OpenJade and OpenSP libraries on my machine in
> /usr/lib, as well as the libraries built from sources in /usr/local/lib,
> and I have /usr/local/lib in /etc/ld.so.conf (and have done an ldconfig).
> 
> The linking phase seems to pull the libraries from /usr/lib and thus I get
> lots of unresolved symbols ...
> 
> Can anyone point me to resources on this subject or suggest ways to fix the
> problem?

ldconfig -p | grep <libname> will show where ld is getting the libraries
from.

it is possible that your Makefile is listing many places for libraries.
The -L argument sows what directories the linker will search for. If it
lists /usr/lib and /lusr/local/lib then I assume if it finds them in
/usr/lib/ first, you will get linking errors.
