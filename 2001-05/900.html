Re: dot.plans

On Sun, May 27, 2001 at 11:48:24PM +1000, Andrew Pullin wrote:

 >     Although I didn't look into how it was done, when I was
 > at uni a friend had a .plan that appeared to run a bash
 > script.

The usual way is to make .plan a named pipe, and have some kind 
of script "listening" to the pipe.  The script will wake up whenever
a process (including fingerd) tries to open the pipe;  whatever it 
writes to the pipe will be read by finger.

Other antisocial finger tricks include symlinking .plan from 
/usr/share/dict/words, or to /dev/zero, or other similarly large
data sources.

 > Again I don't know exactly how it was done, but
 > depending on how finger works, I am not all that convinced a
 > scripting .plan file is very secure either, but it can be
 > done.

Some fingerd's will only open .plan if it's a plain file (which
nicely defeats all of the tricks listed above).
