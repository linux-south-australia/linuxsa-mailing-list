Re: Problem with postfix: virtual user black hole!

On 27 May 2001 00:29:32 +0930, Darryl Ross wrote:

> I'm seeing a problem where any mail that is sent to a virtual domain on the
> server 'vanishes' as if it went into a black hole. I tried using 'telnet
> myhost smtp' and it does the right thing about accepting mail for local
> users and virtual users and rejecting mail for non-existent or non-local
> addresses, but after it says it has queued the message for virtual users,
> along with a message number, it vanishes. I can't find it anywhere in the
> queue directory, 'mailq' says there is no mail in the queue, and it doesn't
> turn up in the users mailbox...

Open up two terminals side-by-side. In one, run (as root) 'tail -f
/var/log/maillog'

In the other, open up telnet and perform your little test. See what's
happening in /var/log/maillog. If that gives you no joy, I'd recommend
you check where Postfix is (mean to) be delivering the mail to.

The 'vanishing into a black hole' part may be a result of either
insufficient storage, or Postfix trying to deliver the message to a
non-existent folder or server.

Failing that, there is the Postfix website which contains some good
documentation and links.
