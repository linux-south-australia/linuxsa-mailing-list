FTP stopped by user's SHELL

Hi All,

I have just changed the default shell for users on our
mail server (which is also the FTP server for the users' home dirs).
We don't allow telnet access and the only ones who try
are hackers anyway.

I set all the users' shell to a shell-script which explains
that telnet access is not permitted and logs the attempt.
(which works great for that purpose)

However, I have just noticed (and confirmed in the ftpd man pages)
that this totally voids the user's ability to FTP to the server.

The man pages say: "The user must have a standard shell"

Does anyone know why this is necessary?

Does anyone know how I can define a custom shell without
messing the ftpd around?

Failing all that, is there an easy way the hack the ftpd
such that it no longer cares? ;-)

Any suggestions are welcome.
