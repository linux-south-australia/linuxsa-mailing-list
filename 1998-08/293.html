RE: twin processors

hi guys.

i run a dual pentium (albeit p1 90) redhat system.

it is very easy to do.

in the source tree (/usr/src/linux)

in the "Makefile", at the very top, is a section that looks like this:
# For SMP kernels, set this. We don't want to have this in the config file
# because it makes re-config very ugly and too many fundamental files depend
# on "CONFIG_SMP"
#
# NOTE! SMP is experimental. See the file Documentation/SMP.txt
#
# SMP = 1
#
# SMP profiling options
# SMP_PROF = 1

uncomment the SMP = 1 line.

then, do make config;make dep;make clean;make zImage; (blah blah blah), and voila - your SMP kernel

then, if your feeling brave - try recompiling some of the processing stats programs (eg top, uptime, etc)

that way you can see what your cpus are doing.


here are the web pages I used exclusively to learn the above:
SMP FAQ
http://www.redhat.com/linux-info/ldp/FAQ/SMP-FAQ-2.html

ProCPS utilities (top etc)
http://www.cs.inf.ethz.ch/~rauch/procps.html

regards
