Module Problems solved (sort of)

Hi Geoffrey,
On Tue, 11 Aug 1998, Geoffrey D. Bennett wrote:

> The next silly question: have you loaded the scsi modules you
> compiled?

I would have thought the file modules.dep would have handled this:

stargate{mael}:45> more modules.dep
/lib/modules/2.0.34/scsi/ppa.o: /lib/modules/2.0.34/scsi/scsi_mod.o

/lib/modules/2.0.34/scsi/sd_mod.o: /lib/modules/2.0.34/scsi/scsi_mod.o

/lib/modules/2.0.34/scsi/scsi_mod.o:

/lib/modules/2.0.34/scsi/scsi_syms.o: /lib/modules/2.0.34/scsi/scsi_mod.o


Anyway, I loaded the modules by hand:

/sbin/insmod sd_mod.o 
/sbin/insmod scsi_mod.o

This went OK, but...

/sbin/insmod scsi_syms.o
scsi_syms.o: couldn't find the kernel version the module was compiled for
 
But I continued with:
/sbin/insmod ppa.o

[/lib/modules/2.0.34/scsi]
stargate{root}:42> /sbin/lsmod
Module         Pages    Used by
ppa                1            0
sd_mod             4            0
scsi_mod           7    [ppa sd_mod]    2
3c59x              4            1 (autoclean)
sb                 6            0
uart401            2    [sb]    0
sound             16    [sb uart401]    0
 
stargate{root}:50> mount -t  msdos /dev/sda4 /mnt/zip

And voila! I have my Zip drive mounted.

Thanks for the suggestion Geoffrey.

I supposed I should put something in /etc/conf.modules so that I don't 
have to insmod the modules manually.

Any suggestions on what these would be?
