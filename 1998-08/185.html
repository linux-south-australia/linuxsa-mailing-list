Re: Problem connecting to my ISP.

> To do this properly you need to run:
> To leave minicom without exiting or closing the modem link try <ALT-Q> then
> 
> #>pppd -d -detach /dev/cuaX &
>  
> X is either 0 or 1, 0 if the modem is COM1 and 1 if the modem is COM2.

Don't forget to add defaultroute if this is your connection to the
Internet.  Also, "-detach" makes pppd not background itself, and the
"&" makes the shell background pppd, so the two cancel each other out
(modulo job control), so I use:

pppd /dev/cuaX defaultroute

("-d" is for debugging; not needed if things work first time :-)
