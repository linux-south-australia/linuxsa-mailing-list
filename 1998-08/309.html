RE: printing man pages

Another way that saves on paper is to use a package like a2ps
and do
man <manpage> > file
a2ps -P<your-printer> file
this could easily be done on one command line or written into a script
a one liner may look like 
man <???> | a2ps -P<your-printer>

The only reason I write it to a file first is that by default a2ps uses
the filename as the title. If the one liner is used it always puts
"stdio" as the title.

a2ps is described in the printing-howto and the Cyrillic-howto for more
information.

> PPL,
> 
> Doing a 'man ls | lpr' is one way, but the output is usually lacking
> in the looks department.
> 
> A slightly nicer way to do this is to let man format the page in
> a nice way for printing.
> 
> Doing a 'man man' indicates in typical man page fashion that using 
> the -t option formats the page in postscript.
> 
> So hence 
> 
> 	man -t ls | lpr
> or
> 	man -t ls > ls.ps
> 
> Will give nicely rendered output.  In the first case (assuming your
> print filters are set up properly) your printer would print the
> appropriate page.  In the second case you can keep a copy for looking
> at first with ghostview or similar, and then printing.
> 
> The program that really does all the work is groff.  Have a look at
> the man page for groff for more information.  For those die hard- 
> do it under any Unix-I love the command line-type people the actual
> commands that would result in the same thing would be...
> 
> 	/usr/bin/groff -Tps -mandoc /usr/man/man1/ls.1
> 
> Or alternatively if you are interested in having a look under the hood
> check out the raw form of man pages.  Text preformatted versions are
> in the /usr/man/cat? directories.
> 
> Hope this adds to you understanding.
