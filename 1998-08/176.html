Hi Mike,

There are a few problems with the above config:
- the domain-name-servers value should refer to a host, not a network
  number
- the subnet-mask doesn't match the subnet number
- the broadcast-address doesn't match the subnet number/mask

Here's the config I use, but you'll need to find out the correct
values for your dns, subnet number, subnet mask, and broadcast
address:

option domain-name "mycompany.com.au";
option domain-name-servers a.b.c.d;
subnet a.b.c.0 netmask 255.255.255.0 {
  option routers a.b.c.1;
  range a.b.c.100 a.b.c.104;
  host myhost { hardware ethernet aa:aa:aa:aa:aa:aa: fixed-address myhost; }
}
