Re: DHCPD Server

Mike
I've got 6 in production, There all very similar to this one
which works very well for an intranet, it also sets up
everything for windows nt logon into a samba server as
well.  (I've changed the clients name to "CUSTOMER" nothing else is
altered)

> [root@server /root]# cat /etc/dhcpd.conf
> server-identifier server.customer.intranet;
> shared-network CUSTOMER {
>   subnet 192.168.3.0 netmask 255.255.255.0 {
>     range 192.168.3.150 192.168.3.200;
>     default-lease-time 172800;
>     max-lease-time 172800;
>     option subnet-mask 255.255.255.0;
>     option broadcast-address 192.168.3.255;
>     option routers 192.168.3.1;
>     option domain-name "customer.intranet";
>     option domain-name-servers 192.168.3.1;
>     option netbios-name-servers 192.168.3.1;
>   }
>     host editor{
>       hardware ethernet 00:40:05:67:A8:2F;
>       fixed-address 192.168.3.64;
>     }
>     host editor2{
>       hardware ethernet 00:40:05:67:76:DB;
>       fixed-address 192.168.3.65;
>     }
>     host director{
>       hardware ethernet 00:00:E8:18:00:AE;
>       fixed-address 192.168.3.66;
>     }
> }

also need a host route which I do with a quick & dirty
entry on the end of /etc/rc.d/rc.local
> # route to enable dhcpd to serve windows clients
> /sbin/route add -host 255.255.255.255 dev eth0
