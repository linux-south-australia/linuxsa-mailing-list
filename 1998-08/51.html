Re: Strange gcc behaviour

> just a quick one ...
> 
> why does this work ?
> 
> int a,b,c;
> 
> a = 56 + 3,
> b = 29 * 2,
> c = a * 2;
> 
> (those are really COMMAS up there , NOT SEMICOLONS)

The comma is an operator (as well as a function argument separator).
You use it just like: "a, b", where a and b are expressions.  The
comma operator evaluates a and b, then returns b.

Therefore, the above example evaluates "a = 56 + 3", which returns 59
and has the side-effect of setting a to 59.  It then evaluates "b = 29
* 2" which returns 58 and as a side-effect sets b to 58.  Finally it
evaluates "c = a * 2", which returns 118 and as a side-effect sets c
to 118.  The right-most expression returned 118, so the result of all
the above is 118 (but this value isn't used and gets discarded).
