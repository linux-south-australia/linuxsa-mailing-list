> I use the computer at school, and from home. At home, I use a different IP
> from at school. Normally, i go into x windows and use netcfg, but i need to
> reboot, because I don't know how to make the changes go into effect without
> rebooting. could some one tell me please?

With PCMCIA, the easiest way is to pull out the ethernet card and put
it back in again.  For those who don't have PCMCIA ethernet cards, you
can run:
/etc/rc.d/init.d/network restart
Or if you're in the netcfg program: click on the interface, click
deactivate, then activate.

> also, I would like to make 2 scripts, to make this easier. how do I make
> them? As far as i know, you make a normal text file that does the commands
> you want (and I think you need "#!/bin/bash" as the top line, or something
> like that) then run run some command to make it executable? Well, if that's
> right, what command do I run to make it executable?

That's right.  The command to make it executable is:
chmod +x yourfile

The commands in the shell script need to replace
/etc/sysconfig/network (for the hostname & gateway configuration) and
/etc/sysconfig/network-scripts/ifcfg-eth0 (for the interface
configuration).

BTW, you can switch netcfg PPP configs by switching between different
copies of:
/etc/sysconfig/network-scripts/chat-ppp0
/etc/sysconfig/network-scripts/ifcfg-ppp0
/etc/ppp/pap-secrets

> Finally, at school i use a gateway, but at home i don't (that's why I don't
> just switch my home network to use the same IP numbers). In Windows95 i just
> put the gateway machine number in the gateway box. In Linux, i just do
> "ifconfig eth0 gateway a.b.c.d" right?

No; you can use the network configurator to do this, or the commands
are:

route del default
route add default gw a.b.c.d
