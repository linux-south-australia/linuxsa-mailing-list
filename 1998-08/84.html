First, although you can move the mouse between screens, you can't drag
windows between screens, and each window can only be on one screen.

As for what you need...

XFree86 doesn't yet do multiple screens.  Apparently Metrolink have
donated their multi-head code to the XFree86 team, but it will be a
while before it appears in a released server.

Xi Graphics (http://www.xigraphics.com/) sell a multi-head (up to
eight screens) version of their Accelerated-X X server
(http://www.xigraphics.com/products/mxdata.html) for US$375.  There is
a list of supported boards available at
http://www.xigraphics.com/board/mx/bdindex.html.  I have no experience
with this product.  I once considered purchasing it, but it's waay
expensive, and I couldn't be sure that it'd work with my hardware
beforehand.

Metrolink's (http://www.metrolink.com/) X server
(http://www.metrolink.com/metrox/ess.html) supports up to 4 screens as
is (multihead is not a separate product).  Red Hat used to supply
Metro-X with their Boxed Set (up to and including RHL 5.0), but not
anymore.  The cost for Metro-X is US$39, and it supports the following
cards:
- Matrox Millennium 
- Matrox Millennium II 
- Matrox Mystique 
- Matrox Mystique 220 
- NOTE: Only one Mystique or Mystique 220 (not both) may be used in
  the combination. 

I use Metro X all the time with a Matrox Millennium and a Tseng
ET4000W32.  It's not supported, but it's working fine.  I'm running
whatever version came with Red Hat Linux 4.2.  I have no idea whether
later versions would work with this combination of video cards.

One of the issues with multiple screens is that the video cards must
not conflict.  For example, a VGA card (or similar) is expected to be
located at a particular address in memory.  If you have two of these
cards, there will obviously be problems if they both expect to reside
at the same location in memory.  On my Milennium, there is a switch
that lets you disable the BIOS.  That way, the ET4000 can be used to
boot the system unless X starts and turns on both cards.

In theory, a Millennium plus any other card should work, but perhaps
this unsupported feature of Metro X has disappeared in later versions.

I once tested two Millennium's plus the ET4000.  That worked okay too
(and it was pretty cool having a 2428x800 desktop), but there was a
bug in the version of Metro-X I was using that would cause the server
to lock up under particular circumstances.  The bug should have been
fixed by now, but I haven't tried a later version yet.
