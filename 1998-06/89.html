Re: Sendmail problem

Doing a killall sendmail may not be the best thing to do.

The best way to to stop start a service is either use

	/etc/rc.d/init.d/sendmail restart 

or for the die hards.

	/etc/rc.d/init.d/sendmail stop 
	/etc/rc.d/init.d/sendmail start

The reason that this typically better is if there are any
lock or info files that keep the system running are cleared
and then re-written.

When you did a reboot the sendmail stop command would have
been executed by the rc script catching an
/etc/rc.d/rc[06].d/K??sendmail file.  

Lucky the system does what it is supposed to do :).
