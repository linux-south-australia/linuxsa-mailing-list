> 1> Since upgrading to redhat 5.1 (from 5.0) everytime I login at the
> console I get a line like:
>     PAM_pwdb[596]: (login) session opened for user benjsc by  (uid=0)
>     PAM_pwdb[596]: LOGIN ON tty1 by benjsc
> 
> It only ever appears on the console. How do I get rid of it! (it bugs me
> :P)

Have a look for the line in /etc/syslog.conf which sends the messages to
/dev/console and comment it out with a '#'. Red Hat 4.2 had a line like:
kern.*          /dev/console

Run "killall -HUP syslogd" to get syslog to reread its config file.

It may be a keyboard problem. Some older keyboards (and newer I guess)
don't like certain keys pressed together. If there is happening you can't
hold Ctrl+Alt+F3. Different keyboards have different limitations.

This sort of problem can often be seen under old multiplayer dos games.
Trying to fit 2 ppl on a keyboard & getting it to work properly meant
finding 2 sets of keys which don't conflict.

> 4> Has anyone had experiance setting up 4dos & dosemu's autoexec.bat,
> config.sys redirection?

I've set up a few systems like that. Plain dos is unbearable, at least
with dosemu I can alias unix commands to something similar. The trick is
to add a couple of lines to your /etc/dosemu.conf:
EmuSys EMU
EmuBat EMU

Now whenever dos tries to access autoexec.bat it gets autoexec.emu. Ditto
config.sys -> config.emu. In the *.emu put the setup you want to use under
dosemu. It is pretty much the same as a normal dos config except you use a
couple of drivers that come with dosemu instead (and not the dos
equivalents). To get 4dos add "shell=c:\4dos\4dos.com c:\4dos /e:512 /p"
to the config.emu.

Note that the .emu extensions are really only useful for when dosemu has
direct access to the dos partition (and boots from it). You could just use
a dosemu boot disk. Once you are happy with the boot disk you can copy it
to a disk image under /var/lib/dosemu. After you have finished booting
from the diskimage run "bootoff" to allow access to the floppy normally.

This is my prefered method since I like to let dosemu access the dos
partition as a fake network drive through the "emufs.sys" driver. That way
it is possible to access the partition from more than one program at a
time.

I can mail you the files I for either of these methods.
