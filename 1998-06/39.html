Since RedHat 5.1 is out, there is 2.0.34 kernel rpms. Maybe that will be
easy for you. grape the rpms (kernel*.rpms) and follow the
updates.readme at www.redhat.com to upgrade your kernel. If that still
doesn't read your fat32, you might want to recompile the kernel via 
cd /usr/src/linux
make xconfig; make clean; make dep; make zImage

>

> be displayed after an error message

the file should be patch-2_0_34 if I am correct. Do gzip -d patch...gz
onece, and apply the patch.

If you got 2.0.32 (the default at redhat 5.0), you should get patch for
2.0.32 at fat32 for linux site. (There is a webpage delicated for Fat32
support at linux)
