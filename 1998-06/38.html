Okay - can I just check that you had the previous kernerl (2.0.33) before
hand?
The patches will only upgrade from one kernel below.
eg. patch 2.0.32 will make a 2.0.31 kernel into 2.0.32
2.0.34 will make a 2.0.33 kernel in 2.0.34 kernel

I think you are getting that message because it can't find the file it is
looking for, which could be the case if you are patching from the wrong
source kernel version.

The second thing is - do you have the full kernel source included?
do a 'du -sk' in the /usr/src/linux directory, and if it seems suspiciously
low (eg. under a few Mb) then you probably only have the header include
files installed.
