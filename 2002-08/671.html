Re: a dumb question about pppd

On Mon, 26 Aug 2002, Greg Wallace wrote:

> The question is: how do you get pppd to disconnect the modem? At the moment
> I either pull the plug from the wall or crtl-C pppd (which I leave running
> in the foreground) and then rm /var/run/ppp.pid to be able to restart pppd.
> Neither option is particularly elegant.

There is a difference between ppp taking down a link and the modem
disconnecting. Although you said the latter I think you might have meant
both, or at least the former. This should all be covered in your pppd man
page. For example, part of the story is given by this description of
pppd.conf entries:

       idle n Specifies that pppd should disconnect if  the  link
              is  idle  for  n seconds.  The link is idle when no
              data packets (i.e. IP packets) are  being  sent  or
              received.   Note:  it  is not advisable to use this
              option with the persist option without  the  demand
              option.  If the active-filter option is given, data
              packets which are rejected by the specified  activ­
              ity filter also count as the link being idle.
