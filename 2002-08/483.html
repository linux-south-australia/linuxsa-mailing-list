> > Watching this thread, and considering options I have to wonder - in the
> > case of cd copying is it really that hard to use the command line?
> > Normally I wouldnt expect "normal" users to use the command line, but
> > the concepts seem way simper than when you use a non-intuative gui or a
> > buggy app.
> >
> > Really, all the user needs to do is ensure cdrdao is installed, then
> > learn
> >
> > [ant@ant ant]$ cdrdao scanbus
> > Cdrdao version 1.1.5 - (C) Andreas Mueller <andreas@daneb.de>
> >   SCSI interface library - (C) Joerg Schilling
> >   L-EC encoding library - (C) Heiko Eissfeldt
> >   Paranoia DAE library - (C) Monty
> >
> > Check http://cdrdao.sourceforge.net/drives.html#dt for current driver
> > tables.
> >
> > Using libscg version 'schily-0.5'
> >
> > 0,0,0: PIONEER, DVD-RW  DVR-104, 1.20  <--- Your device is 0,0,0
> >
> > cdrdao --help explains things including getting more help quite well.
> >
> > Then:
> >
> > <insert source cd>
> > cdrdao read-cd --device 0,0,0 image
> >
> > <insert blank cd>
> > cdrdao write --device 0,0,0 image
> >
> > optionally add:
> >
> >  --speed 8 (or whatever speed you want)
> >  --overburn
> >  --simulate
> >  -v 1 or 2 (verbosity if interested).
> >
> > That'll copy most cds OK.
> >
> >  --driver generic-mmc-raw
> >
> > Will do some more copy protected cds.
> >
> > Then rm image and rm data.bin
> >
> > Sure, you might need something like this as a cheat sheet, but if you
> > gave it to an ex-windows user I'd say they'd have a pretty good change
> > of understanding it and getting it going.
> >
> > cdrecord isnt too hard for audio either:
> >
> > cdrecord dev=0,0,0 speed=8 -audio -pad -v *.wav
> >
> > Can be simpler than compiling programs that dont work properly from cvs
> > anyway...
