Re: database connection

Why make it difficult for yourself?

1) Install Microsoft Access
2) Work out the structure of the tables
3) Recreate the tables in MySQL
4) Dump the MSACCESS data into a CSV (comma separated) file
4) Get that into MySQL using LOAD DATA
5) Get MyODBC
6) Setup MyODBC
7) Utilise from there using ODBC

I would strongly, strongly suggest not doing things the hard way simply
because it avoids Microsoft. Simply "I don't want to use Microsoft" is a
little too flimsy a justification for my liking and any *person* will
tell you that if you can access the source it will be much easier than
reverse engineering the binary.

If you can access the database using Microsoft Access then use the tools
at hand.
