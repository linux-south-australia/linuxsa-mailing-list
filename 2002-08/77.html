RE: link errors

Problem solvered - my mistake.

I was thinking about my library in the wrong way.
I was thinking of it as a collection of functions
which its not, its a collection of translation units.
Which means if my translation unit global.cc has forward declarators,
the linker will demand these be resolved.

Its pretty obvious now.

Thanks to all who pondered my problem and to those who provided feedback.
I can also see that if I'd supplied the right source code it would have been
obvious to others too.
