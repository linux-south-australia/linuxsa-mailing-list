RE: Apache - Virtual Host - Configuration

Shane,

> Does anyone know if there is someway to have Apache report 
> which Viertual Host definitions are considered when a
> request comes into the server?

The first virtual host is the default handler. For example if you punch
in a request using the IP of a box that has multiple virtual hosts,
you'll get the first virtual host.

Assuming you have the ServerName directive set you could use that to
check as Apache will generally report the servername back when you
generate an error.

If you formulated a request such as http://falsedomain.com/missing/ An
error would be generated and it should have the servername listed, ie.
Apache/1.3.26 Server at falsedomain.com Port 80.

This may be a way of checking, other than that log each virtual host
into a separate log file and see which one is getting hit when the
errors happen. 

HTH.
