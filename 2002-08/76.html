Re: [svlug] OpenSSH source may be trojaned, unless... (fwd)

This follows pretty closely on the back of a pretty serious hole in
OpenSSH around 20 June time frame.

Even the OpenBSD site now reads "One remote hole in the default install,
in nearly 6 years!", (ie the OpenSSH hole).  The only reason it doesn't
read "Two remote holes..." is because no-one has yet published an
exploit for the latest OpenSSL issue.

Starting to sound a bit like "Our chocolate contains less cockroach
parts than our competitors!"

OTOH, this issue seems to have prompted a DARPA/CHATS funded review of
the OpenSSL code, so maybe this is a good thing for the longer term.
