Re: Squid Entries

> That won't stop people from connecting to the port, however, which means
> that if there is ever a bug in the ACL logic (or some other code path
> within Squid), then you're wide open to the outside world.  So,
> theoretically, you should also have that ACL backed up with firewalling.

you can set squid to ONLY bind to a particular interface:

http_port 10.0.0.1:8000 

as an example.  this means if you're running NAT/MASQ then no one can 
connect to the port squid is listening on.

then you can also select the outgoing address:

tcp_outgoing_address somevalidipaddress
