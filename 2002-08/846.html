Re: network booting terminals

Darryl,

> If I try the RTL8139 (100Mbps) floppy boot it finds the DHCP server,
> downloads the kernel, boots the kernel, but gets stuck mounting the root
> filesystem via NFS. The error I get is "nfs: server 172.17.33.250 is not
> responding. still trying". Strange, as it worked a few moments ago with
> the 8029 card and if I put the 8029 card back in it does work.
> 
> Next I try an Intel EtherExpress Pro/100 PCI card. With the Etherboot
> floppy (using the correct image) it gives exactly the same symptoms as the
> RTL8139 boot. It's almost like the NFS server doesn't like 100Mbps cards!

This may have nothing to do with it, but I suspect it's a card
compatibility problem.

When you connect anothe computer to the NFS server does it run at
100Mbps or does it degrade to 10Mbps? If it degrades (but still works)
this may give you a hint to your problem.

I've noticed that bootp/dhcp seems more prone to just "crashing" on card
incompatibilities.
