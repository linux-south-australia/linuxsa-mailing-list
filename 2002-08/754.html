I think he was referring to a Vorbis decoder implemented in silicon, 
rather than in software which runs on a general-purpose CPU, like that one.

In any case, I wouldn't have thought hardware decoders at this point 
would be that important, especially given that the vast majority of Ogg 
Vorbis users will be using a software decoder of some sort. Once it has 
gained some inital momentum, then you will start seeing hardware 
decoders coming out. Also, with Moore's Law and all that jazz, I'd 
suggest that hardware decoders will be less and less important on the 
long run anyway.

Luckily I started re-ripping my entire CD collection a few months ago as 
Oggs instead of MP3's, so I'm not terribly bothered by the situation 
either way.. :)
