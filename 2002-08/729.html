RE: Apache Problem

Andrew Pullin wrote:

> So the problem seems to be looking past the subdirectory.
> The subdirectory has "read by all" permissions so it _should_
> work, but so far it doesn't.

This doesn't mean what you think it means.

> I am not sure if changing the permissions to "execute by all"
> will make a difference (as Martin on this list suggested)
> but I will try it anyway.

The "execute" permission on a directory allows the appropriately
authorised user/group/other to access the files within it.  The
read option only allows you to view the file *names* within it.
You have to have execute access on all directory components
through to the path to get at the final component, not just the
"immediately above" directory (I believe this is covered in an
Apache FAQ, but it's applicable to all Unix files, not just Apache).
