Well, yes and no. You are right, once the new user has been pointed to apt
for rpm (in a RedHat newbie case) their troubles with dependancies are
just about solved. As you suggest, the Synaptic gui on top of that makes
things pretty straight forward.

Trouble is, how are they going to find apt for rpm and synaptic if those
packages aren't in the default install?

If newbie is the classic escapee from Windows, where dependancies are
usually shipped with the software installer (or so ubiquitous that we
probably already have them or know where to get them), this is a major
PITA. Is there any sign that RedHat will adopt apt?

The issue we haven't mentioned here yet, is ease of keeping the system
security patched. Without a RH subscription, this can be a tedious and
manual process that will most likely not be reliably carried out. Apt of
course changes that as well. In fact, apart from the verification of the
authenticity of the package provider (which is missing in apt for rpm from
what I can see, don't know about debian) the process is almost as easy as
the Mac OS X software updater, which is a very clean and smart system.
