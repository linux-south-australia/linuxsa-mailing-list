>>>>> "Lai" == Lai kokcheong <dot_lai@hotmail.com> writes:

    Lai> How to get a useful information from the system dump 'core'
    Lai> file ?  I tried to use gdb but it said the core file is not a
    Lai> valid binary file ?
    Lai> I want to from what is the fault of the error : -
    Lai> Segmentation fault (core dumped )

[...]

You will need to start Gdb with the program that caused the core file,
and the name of the core file, eg:

gdb /usr/local/bin/netscape ./core

If you don't know the name of the program causing the core dump, you can 
always use "file ./core", which results in output:

core: ELF 32-bit LSB core file of 'netscape' (signal 6), Intel 80386, version 1
