Caching DNS HOWTO and RPM broken

FYI,

There was a DNS-based denial of service attack posted to
BUGTRAQ before the weekend.  This is actively in use.

The RedHat caching-nameserver-*.rpm and the DNS HOWTO
both incorrectly and insecurely set up the BIND DNS
server for caching.  The exploit script posted to BUGTRAQ
makes good use of these machines.

I contacted the author of the HOWTO last year about
the lax configuration, but he couldn't see the point
in fixing it and complicating the HOWTO.  Needless to
say, I'll be trying again now that a published exploit
exists.

If you are using a caching DNS server as a standard host
through an ISP or in a corporate environment you need:

  options {
    forwarders {
       10.0.0.1;
       10.0.0.2;
    };
    forward only;
    
    allow-query {
      localhost;
    };
    allow-transfer {
      none;
    };
  };

where 10.0.0.1 and 10.0.0.2 are the addresses of the
DNS servers supplied to you by your ISP that usually
live in /etc/resolv.conf.

Configurations that attempt to contact the root name servers
directly will begin to fail as ISPs/enterprises install access
lists to prevent their customers machines from being used as
multipliers in denial of service attacks.
