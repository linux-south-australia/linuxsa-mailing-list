> My bios recognises "Legacy keyboard" and "Legacy mouse". Whereas WinNT
> recognises both without demur, Linux recognises the keyboard but will NOT
> accept the mouse.
> 
> Is this a kernel compile problem?  What is the solution to whatever the

This is not a kernel problem. The kernel has no knowledge about mouse
types. All it handles are the ports. The normal serial driver works on ps2
ports which most likely you already have compiled in.

A few quick Q. Does the mouse work under the console (in gpm - ie move the
mouse in the console, does it show the cursor?)

If it does then it may be a conflict between X and gpm. Try disabling gpm.

Secondly, have you set the type to ps2 in X windows. Ie in your XF86Config
file (/usr/X11/XF86Config under redhat) you should have something like:

Section "Pointer"
    Protocol    "PS/2"
    Device      "/dev/mouse"
EndSection

And /dev/mouse should be linked to the ps2 port
(I think it's /dev/psaux - can someone confirm this?)
So: ln -sf /dev/psaux /dev/mouse 
should link it.

There is a whole range of options for mouse protocols if ps/2 doesn't
work. Do: man XF86Config 
and look for the pointers section.
