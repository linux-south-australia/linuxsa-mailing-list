Hmmm, I guess I have to disagree here.

One can stare at source all day, and not learn much.  However, by actually
implementing something, once can go so much further.

For example, by implementing things in Ethereal, I have learned so much
more about TCP/IP.

By writing a multithreaded SMB server, I have learned so much more about
SMB and threading code under Linux.

By actualling writing some Perl code that needed AUTOLOAD functionality and
tree structured, dynamic and recursive objects, I have learned so so much
more about Perl than reading heaps of code.

However, having the code as a handy reference when you are programming is
so useful :-)
