On Wed, Aug 25, 1999 at 08:21:58PM +0930, Andreja Zivkovic wrote:

> I run a web server on my linux box, and would like to get stats on what
> pages are accessed and how often. I'm not really interested in what country
> the user came from, what borwsers they use, etc, but I would just like
> something that works and is rather simple to set up.
> 
> Can anyone suggest something?

For something that takes data from logfiles, Thw Webalizer is very good. We
use it for generating stats for web clients; it's exceptionally pretty, and
does the job very quickly. It also keeps state, so you can run a logfile
through it, then run the next one after a rotation with no trouble. I've got a
big huge perl wrapper (as per usual :) ) around it that handles rotation,
renaming of logfiles by date and bzipping to backup storage. I can't remember
the URL, but it has a freshmeat entry.

Oh, and although you're not that interested in them, it does generate all of
the by country, etc, stats. It also decodes search engine keywords from the
referrer, which has quite a high Geek Cool factor, too.

If you find anything better, let me know, I like playing with these things. :)
