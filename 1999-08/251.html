Re: Shell Scripts and variables

> there are several ways by I'd use this:
>  export DATE=$(date '+%Y%m')

That is technically wrong.  The script requests Bourne Shell as indicated
by "#!/bin/sh".  Bourne shell does not support $(command); Korne shell does
and it is probably from there that Bash took the feature.  It's a pity that
Linux uses a full featured bash for /bin/sh.  This promulgates the mistaken
belief that Bourne shell incorporates features which, in fact, it doesn't;
and with the attendant result that people learn to write non-portable scripts.
