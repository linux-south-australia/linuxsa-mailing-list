Fetchmail

I gather form fetchmailconf help options that fetchmail will collect
mail from a pop (or other) server and forward it to a listener. I
surmise that one kind of 'listener' is sendmail.

I have asked it not to flush the messsages from my ISP because I can't
trust that it will work properly yet. However, why would I have a SMTP
transaction error? A blind guess would be because fetchmail is fetching
the mail but because I'm not forwarding it to a proper listener, the
Simple Mail Transfer Protocol kinda buggers up.

I also attempted to get it to send it to a listener called "/dev/null"
[i.e. the null device] and still the same results. Here's my current
.fetchmailrc:

# Configuration created Tue Aug 24 16:49:14 1999 by fetchmailconf
set postmaster "lloy0076"
set bouncemail
set properties ""
poll senet via "pop.senet.com.au"
 with proto POP3
       user "lloy0076" there with password [thecorrectpassword] is
lloy0076 here options keep warnings 3600
    smtphost  /dev/null

What the above log indicates to me is that Senet's pop server believe
that I am me and are prepared to send me mail, but the protocol falls
over somewhere. Have I identified where it's not working properly, or is
there something more sinister going on?
