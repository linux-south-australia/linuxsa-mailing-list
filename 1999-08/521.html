There's two ways to debate this point:

1) THE ABSURD

Even Windows doesn't leave them mounted ALL the time. Is Windows such a
wonderful system it automounts floppies BEFORE they're in the drive?

2) THE SENSIBLE

Unix is about choice (as always). And some choose to have an automounter
automount them. See my score above...however for those who don't, here
are the various options:

 a) Mount it in normal place with the default filesystem (ext2 for my
floppy and iso9660 for my CD)

NOTES: Windows does this very well. Most of the time it also gets it
right, except of course it doesn't cope with ext2 at all well.

  b) Mount it somewhere else with the default filesystem...in fact
ANYWHERE in my directory tree

NOTE: Windows is DREADFUL at this. Firstly you can't easily put it into
the directory structure (you can map it using the map command in File
Manager if it exists), and even then it's not altogether obvious how. In
addition, assigning a drive a different (non default) letter isn't
altogether too obvious either.

If some misbehaved program decides that the CD is on drive G: for
example, mapping that in Windows is a nightmare. If a unix program
decides that the CD should be mounted at /halu/halu/nonsensemountpoint
making a symbolic link to the real mount point is a cinch!

  c) Mount it in the normal or somewhere else with whatever filesystem
my kernel / modules support...try that one with 

NOTE: Windows hates most other non-microsoft file systems. Need I say
more? If the Windows automounter doesn't like the file system on the
partition or floppy or cd  you tend to be a lithe stuffed. Linux and
unix aren't like that. Linux at least supports most file systems
especially re: development kernels.

I have always conceptualised it like this:

Windows lets you do absolutely EVERYTHING. Consequently at every step it
asks you "are you sure this?", "are you sure that?"....additionally you
can very easily bugger a windows system up if you don't know exactly
what you're trying to achieve.

I find all these "what ifs" and stuff [read CRAP] really annoying and
patronising. If I have said empty the trash I expect the computer to
empty the trash without demur. If I'm not supposed to empty the trash I
expect it not even to GIVE me the option. 

UNIX has always had an administrator (root) around. On a big system this
person is a separate entity or even group of people. However, ROOT is
supposed to give the user as much access as needed and that's it. So the
user doesn't need to worry too much about whether he/she can do this, or
that. If the user can't it won't allow the user to do it. 

Consequently, I have set up a second account on my system. I'm very
confident in doing everything very quickly on that account because the
only data I stuff up is my own. I can't wipe systems partitions, I can't
rm -rf /, I can't do a lot of things. I have protected myself totally
because I literally am not capable without su-ing or relogging of
stuffing up.

I personally prefer this approach. It's not patronising, effective and
to me more organised.
