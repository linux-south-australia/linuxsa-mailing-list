> efax: 59:40 Error: can't open pre-lock file /var/lock/TMP..01120:
> Permission denied
> efax: 59:40 failed -> fax1088.001
> efax: 59:40 done, returning 2
> 
> I have used both (chmod 775 /var/spool/fax) and  (chmod ugo=rwx
> /var/spool/fax) to try to set permissions.
> 
> Any suggestions appreciated

If you're not running efax as root, then I would assume it can't write
to /var/lock (as per error message). I don't think the permission
problems exist with /var/spool/fax. Try adding the owner of the efax
process to a group that has write access to the /var/lock directory.

I'm not sure if modifing the permissions on the /var/lock directory is
a good idea or not, but it will probably fix the problem.
