On Mon, Aug 02, 1999 at 11:09:36PM +0930, Adam Crawford wrote:

> does anyone know how i can issue a shell command when i use 'adduser' ?
> ie. I want to create a public_html directory and set it with the right
> permissions when a user is added.

Adduser's quite distribution dependent, from memory - you see different
scripts floating around everywhere. Probably easiest to write a shell script
wrapper around adduser, i.e.

---<cut>---
#!/bin/sh

# adduser wrapper script, call with username as parameter

/usr/bin/adduser $1		# use the correct path here
mkdir /home/$1/public_html
chmod 755 /home/$1/public_html
# more commands here

# eof
---<cut>---

Then just call the wrapper instead of adduser. For extra credit, you may alias
adduser to the wrapper in your .bash_profile, and then promptly forget about
it. :) 

This, of course, is only /really/ useful if you have an adduser that accepts a
username on the commandline, and also only really useful when you add one user
at a time, without more complex argument handling. Without this, it may be
easier to just modify your adduser script (likely to be written in shell, or
perl?) to run the extra commands.
