RE: [LONG] SuSE 6.2 linux - first impressions

I took the easy way out!

Am using SuSe (could Suse be a play on words in German - as the english
pronunciation = sweet in German??) X server under Redhat since Suse provides
much better support for some of the newer released vid cards.

I believe this is the real benefit about linux, you can grab bits from just
about anywhere. 

The various linux distributions are basically an exercise (bar a few tweaks)
in window dressing as the core Kernels remain the same. Some are good in
some areas, weak in another. As so many opinions exist no-one distribution
will ever offer the ultimate solution. I don't think there is any such a
thing as the best linux distro.

Alan's discourse is very suitable to highlight this, and it does help put
redhat/Suse etc into perspective.

Whilst I disagree that Redhat is going MS like, my personal opinion is that
a month or so delay in release would have produced a vastly superior product
as the time would have been spent to add  extra polish and remove some of
the quirks. Admittedly, I did experience some MS like frustration for a
while but once problems fixed, this dissipates and doesn't hang around like
a Gates'ian foot odour.

I am running lots of brand new chipsets in my hardware and the crappy
drivers avail will of course lead to more issues than are usual.

I am German by descent so I guess I should be partial to Suse, but to be
honest have not gotten around to trying it as yet. Redhat has to be admired
for the manner it penetrated the linux market. 

Regards
	Tarek

serve as a good point for how RedHat develop drivers for linux.
