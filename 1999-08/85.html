Re: Elm - No locks avlbl!

On Thu, 5 Aug 1999, SaiGKrish wrote:
> What is this problem? ... The mail directory is NFS mounted.

NFS requires a companion, lockd, in order to provide file locking.
For reasons which I don't care to look into, lockd is often not
provided on free unixes.  Look, it's only a guess but probably
correct.

You don't have to use 'flock' or 'locking' file locking.  Unix
often uses 'file locking', which is to say you use a second file
to indicate a lock.  If the second file is there then the lock
exists; if it isn't there then you create it and thus hold the
lock.  Elm will use that sort of protocol; it's a compile time
option.
