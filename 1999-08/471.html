Nope.

> Does anyone else think that this is a design flaw?
> In my opinion, the first LILO prompt should
> give you a choice of disks/OSs to boot,
> and only _then_ should it proceed to
> access the various disks about the place.

You are right in that you can't run OS 1 without accessing boot.b &
map.  It's not a design flaw; LILO is broken into two parts (the bit
that goes in your MBR on disk 1 and the bit that goes in /boot)
because the MBR is not large enough to hold all of LILO.  The only
thing that the first stage loader does is load the (bigger) second
stage loader, which is the bit that does the prompting etc.

Revised version:
1. LILO first stage runs out of MBR
2. LILO first stage loads LILO second stage
3. LILO second stage offers you to run either OS 1 or OS 2.
4. LILO boots OS 1 or OS 2.

If you want LILO to work even if you remove the second disk then you
need to install LILO on the first disk only (ie. put /boot/boot.b and
/boot/map in /mnt/dos/linuxboot or similar).
