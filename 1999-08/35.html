Re: adduser

Justin Viiret wrote:

> Adduser's quite distribution dependent, from memory - you see different
> scripts floating around everywhere. Probably easiest to write a shell script
> wrapper around adduser

Alternatively, you might have a distribution that
creates the user's environment based on a "skel"
directory (/etc/skel on RH6.0?).

You then set up the directory in .../skel (including
a .../skel/.virgin file) and have an /etc/profile.d/virgin
containing something like:

 [ -f $HOME/.virgin ] && /usr/local/etc/virgin $HOME

where the /usr/local/etc/virgin executes all the files in
/usr/local/etc/virgin.d and then removes $HOME/.virgin.


This is quiet useful for when the users stuff up their
ownerships and permissions, as inevitably happens.


Most companies that use UNIX extensively set up
two scripts for each product:

  /usr/local/etc/virgin.d/product
  /etc/profile.d/product

One to initialise the user's environment for the first
time and the other to initialise any environment variables
for each login.
