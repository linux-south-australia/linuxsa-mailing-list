The linux figures were obtained 3 ways: Netscape, ftp & ncftp and all there 
gave figures consistently below the download perfromance of Netscape in NT.
I have set up the pppd to use hardware flow control. I figure that some of my

> 
> Jesudas Mathew wrote:
> 
>  > I have found that Linux gives me much slower download speeds using PPP than
>  > the same computer running Windows NT.
> 
> How are you measuring this?  If it's via the indicator at the bottom
> of your Netscape window, then forget it because the indicator totally
> fabricates its throughput measurements.
> 
> Use the command-line ftp client to transfer a large file under Windows,
> then the same large file under Linux.  I will be very surprised if you
> can see any appreciable difference.
> 
> If you *do* see a difference, check that you're correctly using hardware
> flow control on your serial port under Linux (crtscts is the pppd option
> to use, there are checkboxes in graphical PPP configuration clients).
> Also compare the modem init strings that are being used to make sure
> the stuff you've set up in Linux is turning on the same compression
> and error correction options you're using under Windows.
