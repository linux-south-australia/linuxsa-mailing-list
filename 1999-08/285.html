Authentication of WEB users

Hi all,

I am building a WEB site which authenticates a user (username & password)
and allows the authenticated user to see an WEB page.
This works fine, but I would also like to execute a script as the
authenticated user.

My problem is, environment variables are available for the authenticated
username but not the password or equivalent.
So I have needed to prompt the user to re-enter the password on execution of
this script, which is run by the push of a button.
In the NT world, the authenticated WEB user is also authenticated to NT, so
allowing the same authentication to be used for both WEB and File system. I
realise this is not a good security model to use as an example and I don't
want to get into a discussion on the shortcoming of NT security.

What I would like is to have the credentials of the authenticated user
maintained, allowing me to use the authentication at the file level as well.

Can the authentication given to a user within a WEB browser be used to
execute scripts?

I have tried this on Apache and Netscape, both have the same problem..

Any thoughts?
