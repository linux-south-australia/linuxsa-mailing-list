Re: Slow PPP Performance

> The linux figures were obtained 3 ways: Netscape, ftp & ncftp and all there 
> gave figures consistently below the download perfromance of Netscape in NT.
> I have set up the pppd to use hardware flow control. I figure that some of my
> network/ppp settings must be sub-optimal but can't figure out which ones!

So how long did it take to download the same file from the same
location using these three programmes under the two diffrent OS's? As
Mark mentioned, you can't use what the various programmes tell you,
you must use time as your benchmark. Get a stop watch if you must, but
you can't expect reliable results from these programmes.

Fill this table in:

Programme	OS	DL Time

Netscape	NT
Netscape	Linux
NcFTP		Linux	
WSFTP		NT
ftp		Linux
ftp		NT

Don't forget to bypass your proxy server for these tests, and always
grab the same file from the same location. The best way to do this is
to grab a large file that you know exists somewhere on your Internet
Provider's network. Something at least a Meg or two.  Don't let any
network lag between your provider and the rest of the Internet skew
your results.  Remember that alot of ISP's do transparent proxying.
You don't very often get the actual file you are requesting from the
actual location you are downloading from, even though you think you
are.. 

I'm not doubting that you have a problem, but if you are getting the
data the way you say you are, then you won't know if you do.
