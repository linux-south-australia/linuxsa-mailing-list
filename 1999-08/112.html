>Umm, well the way I *used* to run loadlin.exe was not using the cd at

>run redhat, and every time I have seen a linux system, the vmlinuz file is
>in the /boot directory so you would reference /boot/vmlinuz on your linux

Hmmm, but if you are running loadlin, which you must be doing from DOS,
then your kernel file MUST be in the DOS file system.  Loadlin, last I
looked, did not understand a Linux file system.

However, once you are booted, if Debian keeps things in the same place that
some other distributions dom then you should find your kernel in /boot as
vmlinuz which will be a link to the full name.

Why not skip loadlin and use lilo. It works great.
