> I could code in C, C++ and various varieties of basic, and a little bit

> At university I was brought into contact with all sorts of unix
> machines, and I learned that just because I knew Linux, didn't mean I
> knew how to use any unix machine. I was forcibly taught how to use
> things from the point of view as a user, not a an all-powerful root
> user. I also learnt how to code in C 'properly', instead of my
> self-taught style. While admitedly both methods would give the same
> result, you've got to admit that if you were like me and self-taught
> yourself how to program by reading others' (often dodgy) source  code,
> then you can pick up some bad habits.
> Even if you don't, you will still probably be coding in a way that is
> indecipherable to others. When you are forced to work together with
> other people, you start commenting your work better, and you realise how
> to make things such that they can be intergrated better.
>
> I'd say that doing a proper course is definately a good idea.
> I did 90% of my programming from home on a 486 with redhat 4.0 (and then
> 4.2, etc..) linux on it, but I knew that I had to make good code that
> would still compile and work the same on the uni boxes running ultrix
> and solaris. People teaching themselves to code from a book, on their
> own machine, will never experience this variety.
> However, I learn better from source code and books than I do from a
> lecturer, but the combination is best of all.
