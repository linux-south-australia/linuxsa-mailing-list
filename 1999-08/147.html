Re: mkboot (long)

On Mon, 9 Aug 1999, Kevin Waterson wrote:

> 
> I am tying to create a boot.img file using mkboot.
> Of course this is not going smoothly
> 
> [root@vendy trees]# ./mkboot
> 2000+0 records in
> 2000+0 records out

This indicates that they are using a 2MB file for the image. If all your
files that you want to put on the disk are larger than this it won't work
(as you have discovered). run du on the directory to see how large it is.

You can often get away with 3MB (depending on compression). Hence, see if
there is an option to up the image size.
