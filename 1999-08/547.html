Re: INPUT chain

Subba Rao wrote:

> For the modem interface, I setup the following rules,
> 
> $ ipchains -A input -i ppp0 -p 21 -j DENY
> $ ipchains -A input -i ppp0 -p 23 -j DENY
> $ ipchains -A input -i ppp0 -p 80 -j DENY

"Inconcievable!" (paraphrased: I Do Not Think That Parameter Means
What You Think It Means ...)

-p specifies _protocol_, and should be tcp, udp, icmp or the numeric
equivalent thereof.  I think you were really after port.  Also, REJECT
is much friendlier than DENY, otherwise the connection from the outside
world will appear to hang due to lack of ICMP message going back in the
other direction (ie, it puts the "refused" in "telnet: Unable to connect
to remote host: Connection refused").

To wit, you want (nay, need) something like:

    ipchains -D input -i ppp0 -p tcp --dport 23 -j DENY
