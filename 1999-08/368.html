I very much suspect that you will find that it was a corrupt header and not
a virus/trojan horse.

The email was text only (albeit some crap formatting from a cut and paste of
a web page might have been present) so there was no executable present. In
addition it would have had to be pretty clever to persist around until it
finally reached a linux OS to deposit it payload.

Whilst not guaranteed to catch all virii I am running the absolute latest
patch for VET 10, simply because we were badly CIH'd not so long ago.

simply by tidying up the header.

I know it did cause to be a real pain in the neck for everyone and I
apologize. For some reason our MS Exchange server here at work messed it up.
