Re: wierd routing

Toby Corkindale wrote:


> For instance, some places have internal 192.168.x.x networks, and so
> place a rule on their firewall to block any data from or to those IPs
> from going in or out of their internet gateway. Thus, that router on
> that ISPs network would appear to be unreachable.

Which is exactly the point :-)

Using a prviate address range for core routers is a neat and simple
way to stop hax0rs from launching denial of service attacks on
your core routing infrastructure.  An access list on a core router
interface is bad news, so it is better to make sure that no external
traffic can reach it, only pass through it.


> Or, what if someone nearby to that router assigned that IP to their box?

This isn't possible, to reach a core router you have to transit a
border router.
