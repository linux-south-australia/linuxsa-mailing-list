Re: Netscape Startup Page (Still!)

Red Hat starts up the file in /usr/doc/HTML/index.html by default
when you start X - you don't even have to click on netscape.  I think this 
is only the default behaviour for the not so good windowmanagers like fvwm2, 
though.

If this is the problem, the script which calls netscape is
/etc/X11/xinit/Xclients with netscape mentioned somewhere near the end of 
the file.  You could simply remark or remove this line to stop the default 
behaviour.  If this doesn't work, keep in mind that there is sometimes also 
a version of this file (renamed to .Xclients) in a user's home directory, so 
check there too.
