RE: Telnet disconnect in RH 6.0

By default, Redhat doesn't terminate telnet connections after a time delay.

So:

- Are you telnetting TO or FROM the Linux machine?

- If you're telnetting TO it - it would usually be the client that decides
when to disconnect you.. which client are you using?  Check its options.

- If you're telnetting FROM it - what are you using to telnet with?

- If you're using the linux machine as a gateway to another machine - ie
using the IP Forwarding mechanism - you have to set it up with an ipchains
command.  Not sure of it though :)
