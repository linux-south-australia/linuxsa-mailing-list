Re: LILO problem

> I am having a problem with LILO. I was messing around, and accidentally
> formatted my primary partition. I restored it from a backup, but now LILO
> doesn't load. I can still run linux from a boot disk, but is very annoying
> waiting for the kernel to decompress.
> 
> Does anyone know how I can restore LILO without re-installing Linux?

Try booting off the boot disk, and mounting your primary partition somewhere
in the Linux filesystem (anywhere but root).  Go to it, and use lilo with
the -C {file-name} option.

The filename is the lilo.conf file from your remounted root partition.  You
will probably have to modify it temporarily to let it point to your (real)
root file system, because this will not be mounted as root.

Hope this helps.

To summarise, simply use lilo to tell to adjust where it looks for the
kernel image on the disk - restoring the disk (most likely) would have moved
the file to a different spot, so the values stored in LILO at bootup would
not point to a valid kernel.
