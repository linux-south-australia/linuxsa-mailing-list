Re: sendmail frontend

You could try using linuxconf (installed with redhat distro) as this
actually allows users and domains to be entirely separate.  
ie: fred@weird.com is not the same as fred@strange.com
it takes a bit of fiddling (RTM for more info) but it works rather
nicely.

Be warned though, it needs to make it's own sendmail.cf file in /etc/
before any of it's sendmail features will work.
