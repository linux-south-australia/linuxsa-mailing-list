Re: Proxy Speed.

Hi...

> Ok, interesting, the HTTP performance is bad, but I've just compared
> ftp-ing 2 16MB files from the same site, one via Adelaide Uni's proxy, the
> other via my proxy. The speed was the same.
> 
> So the question now is, why is HTTP downloads of pages so much slower ?

Is it that HTTP is slower, or that on the huge file a small latency at
the beginning has less impact?

Telnet to the proxy server and type:
GET http://www.adelaide.edu.au/ HTTP/1.0

Make sure you hit enter twice at the end of the request.  You may want
to substitute the URL with something that you know isn't cached.  See
how long it takes before the reply comes.  It may be that squid is
waiting for something before it sends the request - perhaps an ICP
response, a DNS lookup or something like that.

If you send me the squid.conf I'll take a look at it and see if
there's anything obvious.
