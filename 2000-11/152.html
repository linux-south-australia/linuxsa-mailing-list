Re: Linux IP Routing Question

Matthew Geddes wrote:

> How about just having all aarnet traffic sent to a static route. The rest would be sent through
> the other route. If you have to do it by port, you might be able to use ipchains and port
> forwarding....

People considering routing tricks into AARNet should be warned
that we will shortly be checking the source addresses of packets
against the forwaring table.  So asynchronous route will stop
working: that is, you need to send traffic into AARNet following
the same route that AARNet sends traffic to you.

This is tough -- we're sick and tired of ISPs that won't
check source addresses at their end, so we're going to
check them at our end.
