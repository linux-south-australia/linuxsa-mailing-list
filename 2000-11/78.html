IMAPd vs Sendmail syslog messages

Hi all,

I have a machine which I constantly poll using fetchmail, in IMAP
mode, to see if there is any mail present.  This causes the mail log
(via syslog) to fill up with mostly useless messages.  If I turn off
mail logging (via /etc/syslog.conf) I don't get messages from
sendmail, which I do want to log.

syslog doesn't seem to have the ability to filter log messages based
on the process.  You can only filter based on facility (eg mail, news,
cron, auth) and priority (eg debug, info, notice).

Short of :
1) modifying the IMAPd source to stop logging to syslog (there
   doesn't seem to be a runtime option for turning this off) 
2) using a different syslog such as syslog-ng (see
   http://www.balabit.hu/products/syslog-ng/)
what can I do to solve this problem????
