Re: Scripts walking the tree

> Interesting -- When did maintainability become a property of a language,
> rather than the way in which it is used?

I think it is a property of both -- it's harder to write maintainable
assembly code than COBOL, and it's harder to write maintainable COBOL
than C. Of course, it is possible to have assembly code that is easier
to maintain than some C code, but you get the drift. Surely you've
heard someone say "Perl is a write-only language", meaning that it's
easy to write, but hard to read. Admittedly, someone could write an
easily maintained Perl script, but they tend not to (particularly due
to TMTOWTDI), and this tendency is influenced by the design of the
language.

rr

PS As an excercise for the reader, imagine a modification of C where
there is no whitespace allowed, and everything was one line. You would
have a lot of trouble maintaining a program written in this imaginary
language (C--?), simply because its syntax is so ugly and unreadable.
Hence, it would be a largely unmaintainable language.
