Re: Samba question

Timothy Aslat wrote:
> 
> I know this isn't strictly linux but I was hoping someone may be able to shed
> some light on this subject.
> 
> I need to be able to retrieve account information from an NT domain controller
> on a Linux box.
> 
> Is there some way I can, given a correct username, retrieve group and real-name
> information from the NT server?   Something like the information finger returns
> would be about right.

What you are after is quite difficult (until the Samba team creates some
shared libs). you may be able to achieve something close to what you
want using rpcclient. I believe Stable Samba comes with rpcclient. You
could try:

rpcclient -c 'enumusers'

to get a list of users.

You could get more and more complex with something similar to:

for i in `rpcclient -c enumusers`
do
	rpcclient -c "samuser $i"
done

I haven't tried this and haven't used stable samba's rpcclient utility,
but you get the general idea.

Hope it helps,
Matt

P.S. If you were using Active Directory, you could do standard LDAP
search.
