Re: Sending mail

David Lloyd wrote:

> * use PERL and some Unix commands
>  - essentially you'd use PERL::DBI (or access PostGres internals without
> using its API [1])
>  - format the results and pipe it to /bin/mail or directly to SendMail
>  - reference "command perldoc", "http://www.cpan.org",
> "http://www.postgres.org/ (or whatever PostGres's home page is)"

The Pg module for perl is quite reasonable and is much less nasty to
beginners than the C api. By using perl and DBI, you can change database
engines quite easily if you decide to move to another database engine in
the future. The Mail::Sendmail module makes sending mail far too easy,
so you should learn the perl socket api and SMTP and talk to the mail
server directly ( another joke ;-) ).

Hope it helps,
