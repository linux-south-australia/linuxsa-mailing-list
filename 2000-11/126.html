Samba and registry patch for login server

Hello, all:

I am setting up a registry patch for my workstations to configure them
automagically to use my samba server as a login server. I'm almost there,
but I'm missing one last key. I wonder if anyone knows of the location...?

REGEDIT4

[HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\MSNP32\NetworkProvider
]
"AuthenticatingAgent"="SAFCOL"

[HKEY_LOCAL_MACHINE\Security\Provider]
"Container"="SAFCOL"

-=-=-=-=-
The above is the .reg file that I have so far. As it is, I have set
user-level access control as well as the name of the domain that the
workstation logs in to. The only thing left is the little check box that
sets the workstation to automatically log in to said domain. I've searched
through the registry as well as google, but I can't find the registry entry
that governs this bit.

Any ideas? Does anyone have a better idea of how to automatically set 40+
workstations to log in to my domain by default?
