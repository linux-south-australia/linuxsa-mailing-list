On Wed, Nov 08, 2000 at 09:45:37AM +1030, Dale Long wrote:

 > I would think that what Perl is would not matter so much to Larry Wall, as
 > long as it does what you want and solves your problems in the best way.
 
<shrug>  I think it matters to Larry Wall a lot.  Have a look at his
writings on the matter (www.perl.com).

Perl is a compiled scripting language.  

 > How would you classify C or other programming languages that call
 > libraries and DLL's (and maybe exec the occasional other program)?

Basic, standard compiled languages.  The use of libraries is just a 
factor of how the executable is linked (at run time for shared libraries,
at compile time for static libraries).

 > I guess the classification of a script is one that mostly uses command
 > line tools and commands.

Not generally.  Perl is a scripting language, and you hardly ever use
command line tools in perl scripts.

The main factor which defines scripting languages is that they tend to use
high level abstractions so that you can write rather complex programs in
a quick-and-dirty fashion.
