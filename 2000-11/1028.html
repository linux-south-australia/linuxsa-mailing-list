How to Break Sawmill

Hmmm!

1)
Get the source for 0.24

2)
Get rep-gtk-0.8

3)
Build it under the original RedHat 6.2 libraries and headers

4)
Decide to upgrade glibc/gtk/gnome to the latest BEFORE you need
glibc-2.0 (which is painful to install because you either have to cope
with dual libraries or rebuild anything which speaks pre-glibc)

5)
Get pgcc and then athlon gcc

6)
rpm -e sawfish

7)
BANG! Problem occurs - gdk_color_free or some function goes from int
functionname(char some_type *) to int functionname(const char some_type
*)

8)
Hack rep-gtk (which shows the problem) to make the function declaration
the same - the only difference, note, is the const declaration (I should
ponder what difference this makes)

9)
Everything builds again

10)
sawmill just crashes - it kind of runs, but then can't find itself
(literally, it saws "cannot find sawmill" which is totally dopy because
it does load up)

The solution appears to be get librep, rep-gtk and sawmill 0.33 and
rebuild. It's working again, and I don't have to cope with the clunky
Enlightenment [1] look :-)
