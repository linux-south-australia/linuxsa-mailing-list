Re: Telnet Not Working under RH 6.2

On Mon, 27 Nov 2000, Franco Principe wrote:

> another question re my install (just about all working properly now).
> 
> I am unable to telnet to the linux box. Message received is Connection
> Refused. What have I overlooked in getting this working ?

 
Connection refused is normally a sign that inetd is not running and hence
listening on the port for connections. Check that inetd is running and if
it is check /etc/inetd.conf and make sure that the telnet line is not
commented out. (ntsysv will help get inetd starting each boot up)

A final reminder, remember most systems won't let you login as root via
telnet.
