Re: Samba Sharing

"Freeman, Peter (ERHS)" wrote:

> Setup the share on the Samba server, give it guest access, & make it
> writeable.

I would recommend against giving guest access to your shares. I would use
public=yes if necessary. Better still, make the Samba box a member of the NT
domain (assuming here that you're keeping it) and assign everyone to be a
member of a group which you then give access to the share. Each user will need
to have an account on the Linux box, but it need not be valid. Even if you are
using shadow passwords, the following should suffice for a user:

user1:x:1000:1000:Test User:/usr/tmp:/bin/false

(or if you wanted to get tricky, you could store these accounts elsewhere -
like LDAP other PAM-usable backends). You can also add share-by-share
directives such as 'write users', etc.

As well as Samba share permissions, you can also tweak the Unix file system
permissions to taste.

If you need any help, feel free to e-mail me off list.
