Re: Upgrading

The upgrade process from even 7.1 is iffy at best.  Mostly because of
7.2 having a slighty different Directory structure/heirachy.  A quote
from Mandrakeforum.com :

> Adrian Saidac wrote "It looks like the 7.2 is build from groud up.
> Many things are missing; many are completely new. I would like your
> comments. Thanks, Adrian S.". 
>
> Here is part of the answer. Note also that we are basically filling
> two CDs with downloadable 7.2. Not built from the ground up, and no
> real deletions, but a lot of changes, including moving directories
> per the requirements of the LSB.

Mandrakeforum.com is a great place to have a poke aroung for
installation isuues and upcomming features (it's kinda like
Slashdot.org for Mandrake users).  Also MUO (MandrakeUser.org) is
another good spot.  Anyway, I had some trouble upgrading 
from 7.1 to 7.2 so I would say back up your data before trying
anything (good idea anyway).  

Also, if you do try to upgrade I think it's advised to use
"liveupdate"  it's an executable file on the root of the install CD. 
You have to run it from X windows (and I would suggest running it
from something a little resource friendly) but it's meant to do an
upgrade better than booting from CD.

Good Luck.
