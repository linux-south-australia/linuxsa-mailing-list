Re: Telnet Not Working under RH 6.2

Franco!

> I am unable to telnet to the linux box. Message received is Connection
> Refused. What have I overlooked in getting this working ?

Are you logging in as root? By default, the superuser (root) cannot
login via telnet. Try logging in as a user.

> (additional info: Fresh RH 6.2 distribution, mainly system install
> defaults.)

Is inetd running at all?

* ps ax | grep inet

What does that say? If it's not running then use ntsysv as root to get
it running (or linuxconf or do a man chkconfig).

Do you have a "telnet" line in /etc/inetd.conf? If you don't then e-mail
me back. Something like:

* cat /etc/inetd.conf | grep telnet

Should turn up something about telnet.

Is telnet installed at all?

* find -name in.telnetd* 

Do that from the root directory...or

* locate telnet

OR

* whereis in.telnetd

OR

* which in.telnetd

The `find' should turn something up...

And check /etc/hosts.allow and /etc/hosts.deny (man hosts.allow)
