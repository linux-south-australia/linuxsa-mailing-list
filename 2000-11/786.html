Re: Scripts walking the tree

Ian Hogben wrote:
> 
> * probably with perl:
> *
> * find /home/shared -type f -print | perl -lne '($f = $_) =~ tr/A-Z/a-z/
> * && rename($_, $f)'
> 
> Oooo. That looks pretty excellent. Quick question, though: will it handle
> filenames with spaces, or does it need some quotes around the variables...?

no quotes needed...
although, it might not quite do what you want... if you had a file named
/home/shared/FOO/BAR then it'd try and move it to /home/shared/foo/bar,
which isn't a single-step process... it'd probably be more like:

find /home/shared -type f -print | perl -lne 'm@(.*/)([^/]+)$@; $f =
$1.lc($2); $f ne $_ && rename($_, $f)'

even better would be to use the File::Basename module, but this will do
for now...
