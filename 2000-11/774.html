>Things I haven't ironed out are updating the list of files to change when
>the parent gets renamed (for f in ...), and filenames with high ascii

>  for f in `find /home/shared/`
>  do
>    mv $f `echo \'$f\' | tr A-Z a-z`
>  done

Hmmm, OK, I forgot about spaces in names, among other things ...

Try:

     mv "$f" "`echo $f | tr A-Z a-z`"

Which deals with names with spaces in them correctly. Unfortunately, if you
have files names with ';' in them, you will still be hosed and may need to
carefully check the contents of $f ...

For walking the tree, try something like:

  if [ -d $d ]; then $0 $f; fi

But that assumes that the script takes as an argument a directory that it
will recursively process ...
