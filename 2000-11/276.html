Re: Proxy Speed.

>The problem is that using this proxy server, data pulled is about 5 times
>slower to download as using the Uni's proxy server direct. This is being
>implemented as a cost saving exersise, but performance is an issue.

The first thing I'd be asking is "what's in the uni's proxy server", since
it's your benchmark for access speed.

How many users does/will your proxy support, and how fast is it when used as
a normal web server?
More ram is the easiest way to speed up a proxy cache, but not the cheapest.

One of the simplest things you can do to speed up access is to make sure
that the proxy isn't wasting resources caching stuff that it shouldn't be,
like cgi-generated pages etc.

Security may be a bigger concern in your scenario than performance, when I
was at Adelaide uni [1], other department's proxy servers were a great way
to get a faster/unrestricted access :)

John Edwards

[1] Although back 'then' proxy servers tended to have 'allow by default'
access permissions..
