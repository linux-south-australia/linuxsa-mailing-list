Re: Scripts walking the tree

Dale Long wrote:

> I often get the feeling that filenames and the characters they contain are
> not handled in a consistant manner through out all Windows APIs (for a
> given version of Windows).

They aren't, hence the famous "feature" of Windows web servers where
you could bypass security and access the source to cgi-equivalent
scripts by appending a space to the end of any URL.  Apparently
spaces at the end of filename were equivalent to a filename sans
trailing space, but it threw the parsing logic.  This has been fixed,
but is still undocumented behaviour (but BugTraq and co. have it
written up).
