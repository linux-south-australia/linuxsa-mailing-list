Re: LT Winmodem!!!!! EEK!

On Fri, 3 Nov 2000, Jason Tan wrote:

> If you own/have legitimate access to a piece of hardware, why would it be
> illegal to write software to use on it or to run it?
> For example why would it be illegal for you to write a driver for awin
> modem if you owned one and wanted to and could be bothered?

I'm no expert there, sorry, but I understand there might be some issues
with reverse engineering. Hopefully the mfr would be willing to provide
the specs if you wished to make a driver, but they might not want there
proprietry code to be released. With a winmodem, the CPU and software is
used to replace a lot of the hardware features. I'm no modem expert, but
that might involve all the error correction and compression routines. I
immagine there is a lot of work involved in producing such a driver. 

I immagine it is quite difficult to write a driver for a black box
(modem) if you can't get info from the mfr. The only way would be to
somehow peek into how windows comunicates with the modem (with USB this is
probably possible???) or you have to actually look at the windows software
and decode it and duplicate it... Which I'm sure is violating somebodies
intellectual property. 

If winmodems become the norm, and I daresay MS will be pushing for this if
it causes problems with linux, then I guess more modem makers will produce
drivers... Or we'll all buy lucent.  

In the end it will be a while before it affects me. I'll be surprised if I
have to upgrade my 33.6 modem for a long time. I still only get 24000 on a
good day, and it's not likely to change in the near future :-(
