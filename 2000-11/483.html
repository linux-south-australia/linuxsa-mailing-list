Asunto: Open Source Databases


There's an article titled "As the Tables Turn" at
http://www.phpbuilder.com/columns/tim20001112.php3?page=1 examining tests
done with the latest MySQL and Postgresql using the SourceForge
application. It is a pretty careful comparison over many tests of the
latest from both development teams. The conclusion seems to be that
Postgresql has overtaken MySQL significantly by all measures the author
could find, including speed, under all loads. Most people don't have
extreme database needs though!

Dan's opinion: I suspect this will remain the case until the MySQL team
complete their effort to turn the product into an RDBMS. At the moment
they are adding RDBMS features and my guess is that they have temporarily
strangled themselves on the pervasive old non-RDBMS design. Unless the
Swedes give up - unlikely, but possible - we'll probably see a new version
that begins to match Postgres in features and hopefully speed.

(For the philosophically-minded, would Postgresql be as good now if MySQL
didn't exist?)
