> I have downloaded the source for OpenOffice from AARNET twice now and
> came up with the same problem. The symptoms are:
> 
> [root] # tar -xzvf /usr/local/saved/openoffice.source.609.tar.gz
> tar: oo_609_src/api/com/sun/star/form/component/DataForm.idl: Cannot
> change ownership to uid 118949, gid 6014: Invalid argument
> 
> I've read the manual and info pages for GNUTAR and I'm not sure how
> you'd stop it from attempting this owner change...

AARNet doesn't change the content of the files it mirrors.
If you believe that a mirroring failure has occured, you
can report the fault to mirror@mirror.aarnet.edu.au.

When tar is used as root it attempts to retain the original
extracted file's ownership.  This is what most people want
(think about suid files).

Untar the archive as another user (as you should be doing
anyway, in case the tar archive can conatin a file like
/etc/passwd).  tar won't then attempt to set the ownership
to the undefined user and group

You might want to contact OpenOffice and suggest that they
force file ownership to bin:bin in their distributed
tar archives.
