Re: runing apps in the background

Jeremy Ervine wrote:
> 
> Is it possible to start an application with some sort of flag so that it will
> start and immediately go into the background.

Lots of people have mentioned "&" which will run it in
background.  But a daemon is more than a background-
running task.  It is also unattached from the controlling
terminal (so when you logout it doesn't exit) and sends
messages via syslog rather than to the controlling terminal
or bit bucket.  Finally, there is also the convention that
it re-reads its configuration upon "kill -HUP" and puts its
process ID in a file in /var/run.

Some of this you can influence by the way you start the
program, but some of it needs to be in the program
proper.

Since you are running Red Hat, have a look in
  /etc/rc.d/init.d/functions
and the "daemon" command.  This does most of what you can
expect without help from the program.
