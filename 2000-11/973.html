I suspect if you had an ipchains rule that said (in English):

REJECT ANY OUTGOING TO PORT 25 ON PPP1

after

ACCEPT ANY OUTGOIN TO PORT 25 ON PPP0

you'd achieve the same effect....

Now, I think this would be:

ipchains -A output -i ppp0 -s ${MYIP} ${UNPRIV_PORTS} -d any/0 25 -j
ACCEPT
ipchains -A output -i ${THEOTHERONES} -s ${MYIP} ${UNPRIV_PORTS} -d
any/0 25 -j REJECT

Naturally if you have a default DENY policy you could possibly get away
with not specifying the latter rule.
