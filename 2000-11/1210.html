> Is anyone else with bigpond, and know what the appropriate
> string is to use in 'pap-secrets' (assuming that they use 
> pap).

They don't use pap.

Step one, configure your ppp/pppd to use chap - then edit your
chap-secrets file - create an entry for ppp@shastanets.com (not
ppp@shasternets.com) with your password with it.  Also include
your own username@bigpond with the same password in that file.

Next set the hostname/username in your ppp/pppd config to be
"yourusername@bigpond" (sans quotes).

> Some of the documentation alludes to using 'user@vic.bigpond.net.au'
> as the user id.  

It's wrong.  username@bigpond is the correct login format - that
should have been provided to you by your Telstra ADSL installer
along with your password.

> Looking at the debug messages, I saw a request from
> 'ppp@shasternets.com' for a CHAP authentication request.

That's an incoming chap request - CHAP is a two-way authentication
handshake, this example is CHAP in one of its simplest forms, that
is, it is expecting your password to work for it on your system.

Bizarre, but true.
