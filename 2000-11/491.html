Seriously, you have said:

* you use RH 6.2
* SendMail
* Postgres

Now you have a number of options:

* use PERL and some Unix commands
 - essentially you'd use PERL::DBI (or access PostGres internals without
using its API [1])
 - format the results and pipe it to /bin/mail or directly to SendMail
 - reference "command perldoc", "http://www.cpan.org",
"http://www.postgres.org/ (or whatever PostGres's home page is)"

* use PHP as a standalone
 - yes, you can use PHP as a standalong language
 - do something similar above [2]
 - reference "http://www.php.org/ or whatever PHP's homepage is"

* use a structural language
 - my question is why?
 - unless you're looking for heaps of speed [3], the development time
taken to write it in PASCAL, C or C++ could be excessive
