RE: up2date

Tuesday, December 11, 2001, 10:13:10 PM, Robyn wrote:

> Hi all,

> When I try to run up2date to get my RH updates I get the following error

> [root@kanga root]# up2date
> Xlib: connection to ":0.0" refused by server
> Xlib: Client is not authorized to connect to Server
> No interactive mode available
> Please specify either -l, -u, --nox, or package names as command line 
> arguments.
> Usage: up2date [OPTIONS] <packages>

> Any ideas what this is referring to?
> RH7.2
> Kernel 2.4.13
> Duron700
> XFree86 4.1

Did you login as a normal user and then su to root to run up2date?

If so, you need to run:  xhost localhost
as your normal user to give root permission to use the local X
display.

Alternatively, just run: up2date --nox
which runs it without X support.
