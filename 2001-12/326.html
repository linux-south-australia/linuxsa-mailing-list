Re: Crontab files not working

Quoting Chandrasekar V <vcs7173@yahoo.co.in>:

> I have installed RHL7.1. I wanted to schedule a backup
> process of some files. I have written a small script.
> If I run this script seperately it works fine. If I
> add the same script in crontab, the script does'nt
> works. I Login as root and edit the crontab file. I
> use the command
> [root@canopus /]# crontab -e
> 30 09 * * * /root/scripts/dbackup
> 
> If I list the crontab file its something like this
> [root@canopus /]# crontab -l
> # DO NOT EDIT THIS FILE - edit the master and
> reinstall.
> # (/tmp/crontab.9418 installed on Thu Dec 13 20:02:07
> 2001)
> # (Cron version -- $Id: crontab.c,v 2.13 1994/01/17
> 03:20:37 vixie Exp $)
> 30 09 * * * /root/scripts/dbackup
> 

Firstly, change the crontab line to:

30 09 * * * /root/scripts/dbackup &> /root/scripts/dbackup.log

and also add some logging code to dbackup. You'll then be able to cat or tail
dbackup.log to find out what went wrong.

Most likely it's an environment problem. The script may be trying to call 
programs in an ambiguous manner instead of using an absolute path. IE, using
'fooprogram' instead of '/usr/local/bin/fooprogram'.

Also, have you set execute permissions on dbackup? Does it have
'#!/usr/bin/something' as its first line?
