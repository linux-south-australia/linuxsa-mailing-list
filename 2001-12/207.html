Re: Stopping all disk activity?

On Mon, 10 Dec 2001 23:29, Gwyn Morfey wrote:
> Hi -
>
> I'm running Mandrake 8.1 on my 64mb laptop, and I'd like to be able
> to do simple word processing (or even text editing) without having
> the hard disk spinning. I'm finding that, even using vi on a single
> shell without X running, the disk is accessed every 20 seconds or so.
> I've killed all the processes I dare without effect, and being a
> fairly recent windows refugee, I hesitate to kill much else without a
> proper understanding of what it does. Can anyone help?

Some processes do logging, like (syslogd, klogd), a faulty bit of 
hardware or software may give an occasional error message that these 
processes will log. Killing them usually stops the logging. Though it 
is better to close them via the init scripts. On my system they reside 
in /etc/init.d.
[/etc/init.d/syslog stop]

Also some processes run automated tasks at set intervals, like crond, 
anacron, etc. These may also cause regular hard disk accesses and can 
be stopped in the same way [/etc/init.d/crond stop]

The system also may be swap activity going on, have you stopped the 
system accessing any swap devices. I think you would call /sbin/swapoff 
with some options in order to do this, although I can't for the life of 
me remember these options.

Look throught the HOWTO's to find out how to do this properly though, 
there are some good ones on minimising power usage on laptops.
http://linuxdoc.org/HOWTO/
http://linuxdoc.org/HOWTO/Laptop-HOWTO.html

Whatever you do, don't run SETI. :-) I made that mistake once, it took 
my old laptop from 1.5 hours up time to about 20 minutes.
