>
> I just needed to confirm that FreeBSD can't go onto a logical parition
> (it seems to have the appropriate addressing to address them, (eg
> ad0s5 etc) but the install program won't let me do it)

Well, it depends on your definition of "logical partition".  FreeBSD
isn't a hermit crab which lives in other people's partitions, it uses
its own.  It can address Microsoft "extended partitions", but they're
for Microsoft file systems.  So if you have used up all the partitions
in the partition table, you're out of luck.
