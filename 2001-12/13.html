Howdy all,


Problem Summary:
----------------------------

In summary the subject says it all.=20
I am getting extreme Internet slowness using UserPPP to do NAT on a =
FreeBSD-4.4-RELEASE Gateway box. =20
e.g. I am getting ping times of 16000 ms and higher (pinging Internode's =
name servers). ;-( That sux!
I would like to see if anyone can recommend a sequential set of possible =
mechanics to look at to solve this issue.


Details:
-----------

I have a 486DX2 that I use as my Gateway machine that connects to =
Internode.
It is running FreeBSD 4.4-RELEASE with a GENERIC kernel.

I use UserPPP to dial up to Internode and also for the use of NAT.

When I dial up with the gateway box, all is fine and NAT works fine =
(from the gateway I can get approx 230ms pings to Internode)
However, as soon as I doing anything (browsing, ftp etc) on any of the =
other boxes (FBSD-5.0-CURRENT, Win2k,Win98  and RH 7.2)
both the gateway and the other boxes come to a very slow halt.

Symptoms are:

* ping times go up like a rocket and it can take up to 5 mins to get up =
a homepage.
* there are request time outs on all other boxes, but the gateway.
* there are source quench errors [see below]
* there are no buffer space available errors [see below]
