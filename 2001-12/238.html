RE: up2date

> [root@kanga root]# up2date
> Xlib: connection to ":0.0" refused by server
> Xlib: Client is not authorized to connect to Server

Up2date is trying to open a window on the x server ":0.0" (your
console), and it doesn't have permission. (this is a pretty normal error
-- try running an xterm, and you'll get the same thing)

> No interactive mode available
> Please specify either -l, -u, --nox, or package names as command line 
> arguments.

This means that up2date is suggesting you use one of the command line
options mentioned above -- try reading the man page :)

> Usage: up2date [OPTIONS] <packages>

That's a usage synopsis :)
