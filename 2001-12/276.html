RE: up2date

> In Konsole in kde I change to root and run up2date, it then 
> starts a gui 
> frontend and lets me log in to the RedHat network and I can 
> get my updates. 
> What is it that I don't have permission to do?

> > > [root@kanga root]# up2date
> > > Xlib: connection to ":0.0" refused by server
> > > Xlib: Client is not authorized to connect to Server

Your client (up2date program, run as root) does not have permission to
connect to your server (the Xserver on :0.0, running under your login --
it actually runs as root, usually (always, I think), but allows or
disallows connections according to xauth (or xhost) information)

This is exactly the same as running an xterm up -- you'll get the same
error. To allow connections from another user, look into xauth (which
provides a magic key to authenticate against the xserver) and xhost
(which removes the need for the magic key at all).

The difference with Konsole in kde is probably that it does this for you
automatically... I'm not sure...
