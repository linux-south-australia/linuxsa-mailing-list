Re: Sendmail probs

>> Sending mail internal to the
>> outside world works fine, as does sending mail to an account on the
>> same server. I don't yet have a domain name registered, but I assume
>> that some should be able to send mail to <account>@ip.address and it
>> should work anyway.
>
>I'm not so sure about that... "@domain.com" does not mean "at the ip
>address of domain.com", it means "at the (ip address of the) computer
>listed as the MX in the DNS record for domain.com" ... To be honest, I'm
>not sure what the exact semantics of @ip is... Try looking in the RFC,
>or in the sendmail doco to be sure that you are really doing what you
>think you are doing...

Actually, in the absence of an MX record, @domain.com does just that, send
to the A record for that domain.

Also, when emailing to an IP address, the correct syntax is user@[1.2.3.4].
This used to work quite well in the trusting world of open relays, but
you'll now find that most admins haven't setup their relaying/locals rules
to allow it, and it doesn't make much sense on a server supporting more than
one domain anyway. There are windows shareware mailservers out there that
don't know what to do with an email address like it.
