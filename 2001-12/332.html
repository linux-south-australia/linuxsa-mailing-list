Re: Mgetty & /AUTO_PPP/

Hi.

Yes, you're on the way, but first, you don't need to install mgetty or ppp
from source, the standard redhat rpm's have done the job since oh, 5.x I
think. I know that this did not work with the mgetty rpm on RH 4.2, but
that is quite old now...

The login config that comes as standard has this:
/AutoPPP/ -    a_ppp   /usr/sbin/pppd auth -chap +pap login debug

on the 'net, (search google for mgetty and AutoPPP) you'll find a howto
that says it should be:
/AutoPPP/ - - /usr/sbin/pppd file /etc/ppp/options.server

With the following in /etc/ppp/options.server:
-detach
asyncmap 0
modem
crtscts
lock
proxyarp
ms-dns xxx.xxx.xxx.xxx (local dns ip address if you have one)
require-pap
refuse-chap
login
pppd_host_ip:remote_link_ip (fill in the numbers)

and you will need a /etc/ppp/pap-secrets that looks like this:
# Secrets for authentication using PAP
# client        server  secret                  IP addresses
* * "" *

and a /usr/sbin/pppd that has the following permissions:
 stat /usr/sbin/pppd
  File: "/usr/sbin/pppd"
  Size: 104876       Filetype: Regular File
  Mode: (4750/-rwsr-x---)

Users that dial in must have a local id in the linux password file, you
can disable the shell (make it /bin/false) if you do not want to allow
shell access.
