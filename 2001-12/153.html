A quick way around this if your clients are win9x/2000 is to edit the
"hosts" file on each desktop machine - you can deploy this via a login
script if they all login to a domain.  The hosts file is normally in
c:\winnt\system32\drivers\etc in win2k and
c:\windows\system32\drivers\etc in win9x. The format of this file should
be self explanitory.  This file should be used by windows in preference
to any DNS server.

Perhaps not the best way but it should work...
