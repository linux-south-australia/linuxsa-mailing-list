Re: IP Masquerading and DNS

On Wed, 5 Nov 1997, Matthew Tippett wrote:

> Another problem is that if the attacker is using OOB packets they
> would possibly be spoofing their IP address.  This means that they could
> get past most of the IPFW stuff quite easily, unless of course you deny
> everything.

AFAIK to send OOB packets you need to have a working tcp connection to the
host. This means that the ip address should be correct otherwise the
syn/ack handshake to start the tcp connection won't work. There are
probably ways around this provided the attacker is sitting on a gateway
between 2 networks and spoofing the address to the other side: 

Faked IP <-----> attacker <------> target

This significantly lessens the problem of the ip not being correct. Most
lusers don't have control of a major box on the 'net.

If there is a problem with windows machines getting nuked with OOB then
set up a firewall so that no tcp connections get forwarded to the windows
machine. Something like:

ipfwadm -F -a deny -D winhost/32 -P tcp -y -W ppp0

This will deny any tcp packets with the syn bit set to winhost coming over
ppp0.
