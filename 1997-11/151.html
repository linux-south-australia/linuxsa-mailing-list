Here is the information I was thinking of:

Mark Newton wrote on Nov 23, 1996:
> 
> Geoffrey D. Bennett wrote:
> 
>  > Someone mentioned to me the other day that Linux (sometimes?) won't
>  > talk PPP to FreeBSD due to a buggy PPP implementation under Linux.
>  > Are you able to give me some details on this or a reference?
> 
> Not really, other than the fact that I'm not sure we can completely
> blame Linux -- That much is not conclusive.
> 
> FreeBSD comes with two PPP implementations.  One of them is essentially
> identical to the kernel PPP that comes with Linux, and uses the same
> userland pppd program as a front-end.  I'm not aware of anyone actively
> using that implementation due to the fact that the other implementation
> that comes with BSD, "ijppp", is so much easier to use for dial-on-demand
> applications.
> 
> ijppp is implemented entirely in userland.  It works by attaching itself
> to an IP-tunnel network interface then encapsulating and de-encapsulating
> packets between the interface and its controlling terminal.  That's the
> version that I believe almost everyone is using using with FreeBSD.
> 
> Now, both versions support Predictor-1 compression.  When predictor-1
> is enabled, however, Linux PPP and FreeBSD ijppp cannot talk to each 
> other.
> 
> I was planning on performing some tests between the Cisco at work and
> ijppp to determine if I can talk to a Cisco with pred1 enabled; That's
> one of the items on the list of things I'll do if I ever manage to achieve
> more than six consecutive days in Adelaide :-)  Depending on the outcome
> of that test I'll be able to get a better idea of whether it's ijppp-pred1
> or pppd-pred1 which is at fault.
> 
> My gut feeling is that it's actually ijppp that's the problem here, because
> pppd is so well-tests on essentially every other UNIX platform capable
> of running PPP.  It's also worth pointing out that ijppp's config file
> can be hacked to make it work with Linux PPP by inserting the following
> lines in the section describing a link to a Linux machine:
> 
>    disable pred1
>    deny pred1
