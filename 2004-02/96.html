you could turn up the heat a little with;

for loop in `seq 1 100` ; do make clean dep ; make -j bzImage modules ; done

that'll make it grind its little MMU and put some VM on disk. =)
but your source tree would have to be on a writable medium...

which would thrash CPU but not much memory or disk I/O.
