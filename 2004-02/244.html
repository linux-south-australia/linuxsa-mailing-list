On 06/02/2004, at 9:43 PM, folorunso eniola wrote:

> How can I get a Cafe Billing software that runs on Linux OS?
> I need it to run on the Server and client Terminals.

Given development support (I haven't got time to maintain /anything/ at 
the moment), I'd probably happy to GPL some work I did a year ago for a 
backpackers up in Sydney.  It uses samba logins authenticating against 
a mysql db, and a windows/linux client that does user logouts and 
timing based on a daemon written in Ruby.  It originally was developed 
on FreeBSD (and used ipf/natd/dummynet for data counting/limiting), but 
later migrated over to Linux (iptables/iproute2).  There's a PHP based 
interface for generating cards, manually checking/kicking users, adding 
machines, calculating income, management.

It's not at the point where I'd like to release it, though - many ugly 
bits, and installation will prove rather interesting as it uses custom 
samba/pam modules :-)  Works well, though - been running since it's 
implementation without any known issues.
