Intermittent network fault

I've got a wee problem which is starting to irritate. Intermittently, all 
users on my lan lose connectivity through the server/gateway.

* Internet dies, whether direct connection or through Squid
* Samba services are not available to the PCs
* Putty sessions to server close
* I can ping the internal private NIC (eth1 192.168.1.1) but not the public 
(eth0 192.168.2.1) or anything beyond. "Reply from 192.168.1.1 Destination 
unreachable"

After approx 5 minutes connectivity is restored. The problem is maybe 
occurring every 20 minutes or so.

I'm using Debian 3.0 sarge
ADSL to INternode using DSL-300 connected to eth0
SAmba 3.0.1-2
Squid 2.5.4-3
Shorewall firewall 1.4.8.stable-2

I think there might be a firewall issue. Ie all traffic from the loc zone 
to fw (local zone to firewall in Shorewall speak) seems to stop. I'm 
getting quite a lot of logs like the following.

Feb  3 00:08:20 arofs kernel: Shorewall:newnotsyn:DROP:IN=eth1 OUT= 
MAC=00:06:29:6f:7b:60:00:02:44:68:50:a3:08:00 SRC=192.168.1.100 
DST=192.168.1.1 LEN=40 TOS=0x00 PREC=0x00 TTL=128 ID=38493 DF PROTO=TCP 
SPT=1251 DPT=80 WINDOW=63406 RES=0x00 ACK FIN URGP=0

Any hints greatly appreciated.
