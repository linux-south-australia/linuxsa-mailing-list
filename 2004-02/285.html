Re: Ports creation ?

Andrew,

You don't have to do anything that complicated, and you pretty much
answered the question in your last paragraph.

If 'firebird' has an option (from the command line, or config script) to
run on another port, then you should be able to run 
another copy of the software..

The software will 'open' the port for listening 
(it's a system call) when the program is run.  (Just be aware that posts
below and including 1000 are 'protected' and usually only be used by
programs run as root.)

Use to command 'netstat' to see which ports are currently in use by the
system.

Cheers,
Paul
