On Mon, Feb 02, 2004 at 07:59:32PM +1100, Brad van Elsen wrote:
> Im running Mandrake 9.1, using fetchmail to access my
> ISP's pop3 server, I am running postfix and have
> followed the instructions on how to install this.
> 
> Everyting works fine except the mail gets delivered to
> /var/spool/mail/USERNAME. 
> 
> I cannot get the windows clients to see this mail
> using pop3, even though it is connecting to the server
> OK, but reports there is no mail.

...you're connecting to your Linux box with windoze POP3 clients, on a
LAN, to get at mail which you have already pulled from the ISP?

The POP3 server "ipopd" looks in /var/spool/mail/<username> as
standard. The username/password that the POP3 clients require to
access it are the username/password pair that <username> would log in
with as a regular user.
