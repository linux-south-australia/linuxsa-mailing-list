You need to focus your efforts in the right direction.

I believe that an organisation such as Optus will not see the benifits
of Open Source on the desktop in the short term. Short term is the only
thing that management cares for. 

Open Source has best left its mark in the Infrastructure. Thats servers,
network, internet services (mail, web, instant messaging, databases, etc)

Cisco moved 1000 print servers to Linux in 2000 globally IIRR, 
saved $1M per year.

There has been a trend to middleware applications running on web servers
with db servers on the backend. These middle ware apps are often
Perl/PHP/Python apps talking to MySQL/PostgreSQL/Oracle.

Going middle ware server means the client desktop does not need to be a
hooj bloated windows desktop. A good web client  is all thats needed.
This is where the thin client was supposed to take over the world.

You can lock down the desktop and remove unecessary cruft, therefore
using less memory. Touch screeens make so much sense for POS terminals,
and a similar thing applies to sales desktop system i helped spec in
London. Cut down gnome desktop with a couple of keyboard shortcuts to
run a couple of apps like a web broswer, a print queue thingy,
login/logout, shortcut mozilla with a given URL for addressbooks, serach
etc..

setting these things up in the first place can be a bugger, but the
longer term IT support is less of a problem and Administration ca oft be
done remotely.
