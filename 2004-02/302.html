Worked a treat!  I recall now that I've had no success in the past on the very 
odd occasion when I've tried rpm -e and this explains it.  Curious that the 
package version doesn't need to be specified in the -e context, but I guess 
it doesn't need to be either, since there'll only ever be one version of the 
particular package installed.  

When performing most rpm installations, I have used general instructions 
printed in my old Redhat 6.1 pocket book where it indicated rpm -Uvh --force 
[package] to force an install and rpm -e [package] to uninstall.  I just 
naturally assumed the version should be included in the command.  I see now 
in man rpm (looking more closely!) that -e is used with 'package name' while 
-Uvh etc is used with 'package file'.  Thanks for the highlighting this.
