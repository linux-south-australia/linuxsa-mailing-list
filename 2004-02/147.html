Re: Linux Group and User permissions

Craig Chandler <craig.chandler@internode.on.net> wrote:
> my thoughts are to create a "webedit" group of which the "newuser"
> is a member of change ownership of the relavent directories and files to
> the webedit group and give the group permissions read and write access
> and user (apache) permissions read and exec access

That's more or less the right idea.  Two things: Apache doesn't need to
own the files (and there's a security argument that says apache shouldn't
own them.)  Apache just needs to be able to read the files, and execute
the cgi's, so give read and execute access to other and give ownership
to some other user.  Also, in order to be able to access a file in a
directory the user must be granted execute access to that directory.
That means your webedit group needs execute permission on all directories
that you want them to be able to create files in.
