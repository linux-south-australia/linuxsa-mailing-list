Re: Samba and multiple domains

On Tue, 29 Apr 2003, Brett Hales wrote:

> Hi,
> 
> I need to configure samba to support clients coming from multiple
> domains and authenticating against multiple domain controllers depending
> on the domain they are coming from.

Are you not able to use trusted domains? Since you will most likely be 
setting up a member server, this might be the easiest way to go.

The only other way to do this ATM is to use different interfaces and run 
multiple copies of Samba, each bound to different interfaces. 
 
> Eg.
> 
> Client 1:
> 
> Connects using DOMAIN1\USERNAME ---> \\servername\share
> 
> Client 2:
> 
> Connects using DOMAIN2\USERNAME ---> \\servername\share
> 
> The logic I need to embed inside of the smb.conf is.
> 
> If the user is from domain 1 then go to password server x and if the
> user is from domain 2 then go to password server y.
> 
> Has anybody solved this problem before.
