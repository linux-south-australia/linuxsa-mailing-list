2. smrsh

> /etc/smrsh/wrapper: error: no majordomo user

From:
http://new.linuxnow.com/docs/content/Majordomo-MajorCool-HOWTO-html/Majordomo-MajorCool-HOWTO-2.html

Sendmail Restricted Shell

If Sendmail is using smrsh, then the only programs that can be executed
are those under the /etc/smrsh/ directory. Perhaps the best solution to
run the wrapper from the aliases file is to create a symbolic link from
/etc/smrsh/wrapper to /usr/lib/majordomo/wrapper.

cd into your /etc/smrsh directory, then:

[root@yada smrsh]# ln -s /usr/lib/majordomo/wrapper wrapper

Hope this helps.
