Re: [NEWS] "Solaris on course to merge with Linux"

davidn@rebel.net.au wrote:
> 
> Until everybody agrees on a common platform, which means compatible
> hardware and operating system, there will always be the haves and the
> have-nots.  Windows is closer to that common platform than anything
> else available today.

Surely that is POSIX-like OS's more than it is Windows. I can take a 
program written in standard C which uses POSIX APIs and compile it on a 
number of different platforms and architectures (including Windows, 
IIRC). Windows can run on i386 and the Itanium. Your typical UN*X 
software will run on a hell of a lot more architectures than that.

Anyway, you don't need a common platform, you need a common API. Look at 
GNOME. Does it matter if it is running under Linux, FreeBSD, Solaris, 
HP-UX, Tru64, QNX or whatever? No - it looks and works the same on all 
those platforms.

The day we have a common platform is that day that every single computer 
in the world gets r00ted by the latest SQL Server worm or Outlook virus.

The *last* thing we need is a common platform.
