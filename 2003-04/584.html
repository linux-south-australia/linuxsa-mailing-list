Re: Postgresql

Hi, from memory the short answer is:

su postgres
OR
su postgresql
or su whatever the postgres user account is on your system.

You need to log on as the postgres user account. Below are some links, 
which I searched for briefly which mention changing to the postgres user 
in order to run postgres administration commands. I think pgsql will 
work from then on. The docs are annoying in not mentioning this step 
explicitly. A person wouldn't read these docs below if using a distros 
version of postgres rather than installing from source.

Oh, and I used to forget to not be in roots home directory when doing this.

http://www.postgresql.org/docs/view.php?version=7.3&idoc=0&file=installation.html 

http://www.postgresql.org/docs/view.php?version=7.3&idoc=0&file=creating-cluster.html

Cheers, Stephen.
