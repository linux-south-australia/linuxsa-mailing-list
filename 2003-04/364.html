Re: dns confusion

Adam Smith wrote:
> 
> It proves to be very annoying when I change a DNS entry on a remote DNS server
> and then I'm forced to wait for TTL's to expire before I can start using it
> again :/

Doesn't BIND have a lookup cache file, usually located in the directory 
specified in it's config file with

   options { directory "/foo"; };

?

Deleting the file before starting should fix it.

Alternatively, if you are using forwarders, then you may be getting the 
cached response from those nameservers.
