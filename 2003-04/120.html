On Sat, 5 Apr 2003 08:50 pm, Brian Marr wrote:
> I moved away from the nvaudio driver on this Suse box when I got a kernel
> panic. Alsa has proved a better option.. Still using the nvnet module for
> networking though.
> Brian Marr

I am using ALSA, I don't think NVIDIA supply an audio driver for the nforce2. 
This machine is running a whole bunch of binary drivers: Lucent Winmodem, 
Nvidia Network Card, and Nvidia Geforce 3 card.

Btw I just tried unloading the nvnet driver and it makes no difference with 
the burning problem. Haven't gotten another kernel panic yet though. I can 
reproduce the burning problem consistently:

1) Start burning something in k3b, e.g. an iso
2) Attempt to start any fullscreen game, e.g. frozen-bubble

At this point the game will not start, although it will be in the process 
list. If you attempt to kill said process it will not die, however the 
request to kill seems to be queued and the program ends up dying once the 
burning is finished.

3) wait til burning finishes, or cancel the burning

At the very instant the burning stops, frozen-bubble will come up.

This is the strangest problem I have ever seen. How on earth is burning and 
switching to fullscreen mode related?
