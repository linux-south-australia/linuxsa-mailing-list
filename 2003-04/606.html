spamassassin + Courier + maildrop on Debian

Hi all.

I run Courier MTA with maildrop and mailfilter (as opposed to procmail).
I recently got pissed off with having around a hundred spams per day, so
installed spamassassin.

In case anyone else wants to do it without thinking too much, this is
what I did (there was very little documentation that I could find):

on debian, installing it is just "apt-get install spamassassin razor"

(this includes installing razor, the distributed spam database)

then edit /etc/courier/maildroprc (assuming you already have maildrop
set up) to look something like this:

exception {
        xfilter "/usr/bin/spamassassin -P"
}
if (/^X-Spam-Flag: *YES/)
{
        exception {
                to "/home/richard/Maildir/.SPAM/"
        }
}

At the moment, I still accept spam, but it goes into the SPAM folder --
I'll change that to actually reject spam when I'm happy with it. Also,
I've only just installed it, so I wouldn't quite trust this 100% yet...
I haven't had any spam in the last 10 minutes to test it with, either :)
