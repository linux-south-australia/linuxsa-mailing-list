We have a M$ DHCP server which I'm replacing with a linux one.  I thought
perhaps I could also use the samba WINS on the same box which I have all set
working fine in test mode off the main network.

In /var/cache/samba/wins.dat i see all the entries which we're created
dynamically.  After much reading of the samba mailing list archive the
general consensus is to just add them to the wins.dat file.    Someone said
they have to stop nmb add the entries and restart nmb to stop them getting
overwritten.

I've been playing around and added a new entry to the file and waited a few
minutes hoping that it was possible to add a new machine to it without
restarting any services, but this didn't work.  after a restart it worked.

Surely this is not how it's supposed to be done?  in the M$ WINS server i'm
trying to get rid of I simply chuck new things or change things in the
static mappings and it's instantly available to the clients.  This does not
appear to be the case with how I'm doing this?

Had a good poke though the samba docs as well and found where it mentions
