Re: Slow modem.

On Thu, 24 Apr 2003 09:29 am, Edge, Martin (SSABSA) wrote:
> People,
>
> I'm using pppd thru' a generic modem, 56K running at it's top speed
> (according to the gui config), but it's chronically slow.
> Is there anything I can check to see if the coms port is running correctly,
> or any other config file that might be affecting this speed?
>
> Running Red Hat 8(2.4.18) on a p4 2.2GHz with 512MB.

You could try altering the file /etc/ppp/options.  I found that adding the 
line 'mtu 350' to the top line in that file did incredible things for the 
speed (and reliability of connection) of my 33.6k modem.  This sets the 
'maximum transmission unit' to 350 bytes and, while smaller than the usual 
transmission packet, this enabled my connection to internode to work 
beautifully.  I was truely amazed at the increase in throughput of my modem 
(close to double its earlier pathetic performance).  Perhaps unbelievable, 
but true.
