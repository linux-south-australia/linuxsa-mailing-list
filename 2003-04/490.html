Running it from a read-only compact-flash card might qualify.

> Sorry...to clarify....can parts of a Linux installation be read only?
> 

Yep. Just install the RO parts onto separate partition(s) and mark them
as RO in /etc/fstab. This is fairly common practice if you are
net-booting, I believe. Usually at least /usr is mounted RO, / can be as 
well provided /var, /tmp and other writeable areas are mounted RW 
somehow. Sometimes applications expect to be able to write to /etc, 
which is a broken thing to do, so that needs to be taken into account as 
well.

Some OS's have additonal security-related features. Eg, I know
FreeBSD has the concept of an "immutable" flag for files and
directories. When combined with a high security-level setting in
rc.conf you can flag files and directores as being unwriteable by
everyone, including root. This allows you to have more fine-grained 
control than marking whole partitions RO.
