Y2K: Red Hat Linux groff and libtiff are non-compliant

The groff patch is necessary for man pages.

libtiff can be used by graphics, fax and printing applications.

The groff*.rpm and libtiff*.rpm files are on the usual mirrors, including:

  http://mirror.aarnet.edu.au/patches/redhat/

                   Red Hat, Inc. Bug Fix Advisory

Synopsis:               new libtiff, groff packages available
Advisory ID:            RHBA-1999:061-01
Issue date:             1999-12-31
Updated on:             1999-12-31      
Keywords:               libtiff ras2tiff groff year 2000
Cross references:       
---------------------------------------------------------------------

1. Topic:

New libtiff and groff packages are available to fix minor
date-related issues.

2. Relevant releases/architectures:

Red Hat Linux 6.x, all architectures
Red Hat Linux 5.x, all architectures

3. Problem description:

According to the groff changelog:
      * Y2k fixes.  Don't assume that the current year precedes 2000.

According to the libtiff changelog:
     * Fixed apocalypse-inducing y2k bug in contrib/ras/ras2tiff.c

Note: the newer libtiff libraries no longer support LZW compression
of TIFF images, due to patent issues. LZW decompression is still
supported, though.

4. Solution:

For each RPM for your particular architecture, run:
    rpm -Uvh <filename>
where filename is the name of the RPM.
