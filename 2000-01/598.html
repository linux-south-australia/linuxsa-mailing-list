On Mon, 17 Jan 2000, Mike Andrew sagely said:
> [Postscript] is used as the internal conveyor in linux for porting a
> print document around the network until printed. In every respect it's
> purpose (under linux) is the same as GDI printing in windoze.

It's a pity the unix community hasn't created an X based printing standard.
Both Macintosh and Windows use the same commands to render on the printer
as the screen, so printed reports are no harder to write, nor any different
than, application screens.  It's crazy that X programmers have to write
application displays using X primitives, and printed reports using
Postscript, Epson, PCL, or whatever.  Even just Postscript is one too many.

I suppose somebody could write an X server that renders on the printer;
your X application would display reports on DISPLAY=myprinter:0.  How hard
would it be?
