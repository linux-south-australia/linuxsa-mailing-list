Sean Burford wrote:

> > My RH 6.0 2.2.5-22 system tells me that it needs over 300 megs of memory
> > while I have PPP / netscape /  IRC and a few other small applications
> > (terminal, etc) open.  What is going on?
>
> What tells you that you need over 300M of memory?  Rather than update
> everything, lets find out what needs fixing.
>
> Run top, and press M to sort processes by memory usage.  Does anything
> show up as using more than a few meg under the RSS heading?
>
> >From memory: If top or cat /proc/meminfo says you are using all of your real
> memory (300M? :), but not your swap, that is normal.  Linux uses memory
> which is not needed by applications for file system caching.  If your
> applications need more memory, the file system cache size shrinks.
>
> On the other hand, my favourite source of updates for Redhat is
> ftp.au.redhat.com (which is on mirror.aarnet.edu.au).  Just grab the
> rpm's that you want, not the whole thing.  It shouldn't come to more than
> a few meg once you know what you want to update.  My second favourite
> source of updates is the APM Linux Pocketbook CD's.
>
> Sean
