Re: networks

Robyn Manning wrote:

> Can you connect Linux to Windows using tcp/ip instead of samba?

Not really, the only way to connect to Windows at all is through SMB.

>
>
> When I start smbd and nmbd do I have to do anything else in Linux to
> connect to Windows 98 (I've got a smb.conf file that is correct)?

to connect TO Windows 98, you need to use smbclient or smbmount (you will
need smbfs compiled into your kernel for smbmount). Check out the man pages
for these two.

>
>
> What program/protocol do I need to have running on Windows 98 to connect
> via samba to Linux?

You will need a protocol and a client on Windows 98. The protocol should be
TCP/IP and you can just use the Microsoft Client for Microsoft Networks. You
install this by clicking add in the network control panel, then select
client, Microsoft, MS CLIENT......
