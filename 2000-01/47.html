No :-)

Samba is a much better alternative.  You can use the profile
facility to map drives.

MARS-NWE only does NCP version 3.  That is, no NDS.  I'm
confused by you saying you run a NetWare 3 server -- they
aren't Y2K compliant.

MARS-NWE is also broken in RedHat 6.1 -- the startup scripts
need fixing.  Such a basic failure suggests that the level
of support (and certainly of regression testing) for IPX just
isn't there.  I can't imagine that they'd ship a Samba RPM with
the same problem.

The easiest way to get IPX to work is to use linuxconf to
set

/etc/sysconfig/network.

IPX="yes"
IPXINTERNALNETNUM=""
IPXINTERNALNODENUM=""
IPXAUTOPRIMARY="on"
IPXAUTOFRAME="off"

/etc/sysconfig/network-scripts/ifcfg-eth0 

IPXNETNUM_802_2=""
IPXPRIMARY_802_2="no"
IPXACTIVE_802_2="no"
IPXNETNUM_802_3=""
IPXPRIMARY_802_3="no"
IPXACTIVE_802_3="no"
IPXNETNUM_ETHERII="817f8600"   #  <-- your numbers here
IPXPRIMARY_ETHERII="no"
IPXACTIVE_ETHERII="yes"
IPXNETNUM_SNAP=""
IPXPRIMARY_SNAP="no"
IPXACTIVE_SNAP="no"

MARS-NWE has its own log file /var/log/mars-nwe.log.  You need to
create it before it will contain log messages.  It also isn't
rotated properly.

I've sent patches for the scripts and rotation to Red Hat, having
wasted half a day getting the bloody thing to work.  But if I
didn't have to do it for work, I'd have ditched it for Samba.
