> > The symptoms are that the printer light flashes but nothing prints. I can have it > flash by using lpr, printool or cat file >/dev/lp0, but I have nothing printing.
> 
> OK, so the print file is hitting the printer in a trashed state.
> 
> This might be because of data corruption (thus the module fix)
> but this is unlikely.
> 
> It's more likely that the job has trash characters, such as a
> FF or 0x04 inserted into the print stream.  Check that
> 
>   /var/spool/lpd/lp/postscript.cfg
> 
> has
> 
>   PS_SEND_EOF=NO
> 
> to suppress the 0x04.
> 
> Similarly, general.cfg should say, amoung other things:
> 
>   export DESIRED_TO=ps
>   export PAPERSIZE=a4
>   export ASCII_TO_PS=YES
