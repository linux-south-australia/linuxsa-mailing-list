> UNIX can use pseudo terminals called pttys. It uses the file system that
> the error is about. There are a number of ways to avoid this problem:
>
> 1) COMPILE the file system that doesn't exist into the kernel or as a
> module (advanced)
> 2) UNSET the pseudo tty support in the kernel and recompile it
> (advanced)
> 3) REINSTALL a stock kernel...it should exist in a normal kernel so I
> don't know why you got the problem but just in case you have rearranged
> things
> 4) DISABLE it in /etc/fstab
>
> Step 4 is probably your best short term fix. Edit /etc/fstab with vi or
> emacs or ed even and get rid of the one with /dev/pts file system.
> Either delete the line completely or prefix the whole line with a hash.
> This will stop Linux from trying to mount the file system which will
> stop your error.
>
