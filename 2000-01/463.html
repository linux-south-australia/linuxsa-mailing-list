Re: zombies

As was pointed out last year on this list, at least several times, (and
all present in the archives, I presume...), zombie processes are a
normal part of the operating of the O/S.
A process exists in its zombie state when it terminates, but it's parent
process has not yet acknowledged it's death, by doing a wait,waitfor, or
similar command. (This is neccessary to collect return codes, etc.)

The processes actual code and memory pages, etc. are all unloaded, and
it is not considered a candidate for being swapped to. Thus, they don't
load your system down in any way. They do use a PID up, but since you've
got a lot, this generally isn't a problem.

If you had a (very) badly written program, that kept spawning child
processes and never checking on them, then theoretically you could fill
up your process table. However, as soon as this parent process died, it
would take all the zombies away with it. Users have a maximum process
limit anyway, so a user would not be able to bring your system down by
trying to exploit this.
If you do see heaps of zombies appearing, then do a ps list with the
parents being shown in a tree, and you should see the offending app.
