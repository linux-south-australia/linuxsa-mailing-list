> with the following partition table...
> 
>   Device Boot    Start       End    Blocks   Id  System
> /dev/hda1   *         1       827   6642846    c  Win95 FAT32 (LBA)
> /dev/hda2           828      1653   6634845    5  Extended
> /dev/hda5           828      1024   1582371   83  Linux
> /dev/hda6          1025      1033     72261   82  Linux swap
> /dev/hda7          1034      1653   4980118+  83  Linux
> 
> .. where the last partition was added using fdisk.

[snip]

linux is not bootable beyond the 1023'rd cylinder. this is a bugbear for
ever larger HD's but something everyone has to live with. The entire boot
section of linux must reside below 1024, else it wil fail.

What you need to do is create a small partition below this value (hda4 is
suggested, since hda3 on your system is an extended partition)

everything inside /boot  must be on this partition. (that's not quite true,
but everything = no stuffing about)

I predict that what we will all begin to see from new distributions is 3
(three) partitions, used by linux instead of the current two, for this
reason.
