If you need both UNIX logons and SMB server logons authenticated against an
NT server, then you do need something like PAM_SMB or PAM_NTDOM for UNIX
logons, but you would still probably be better off with security=server
with Samba.

Please be aware that PAM_SMB has some deficiencies:

1. Does not handle encrypted passwords
2. It uses SMBlib, which has some buffer overflows in it that I have not
had a chance to fix.
