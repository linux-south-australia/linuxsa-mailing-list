I think the problem here is that people want the document source, not a
rendered (cf compiled for C source) document.  This make PDF, PS, DVI
unsuitable.  In the case of responding to tenders, having editable
source can greatly reduce the amount of work required by the respondent.

If the problem was providing an electronic version that can be viewed or
printed, then PDF or PS would be a good choice I think.  PDF and PS
format files are quite easy to create with existing commercial or open
source tools under windows.  They can also be printed and viewed with
free (speech) and free (beer) tools on many platforms.

To output a PS file under windows, you simply print to a postscript (eg
Apple LaserWriter) printer connected to FILE:

You can then convert this PS file to a PDF file using ghostscript or gnu 
ghostscript (ps2pdf.bat).

You can also purchase Adobe Acrobat, which includes a driver that allows
you to print directly to a PDF file.

This works with pretty much everything that can print to a printer under
windows.
