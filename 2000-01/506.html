Re: mutt/pgp failure

On Thu, Jan 13, 2000 at 08:14:14PM +1030, Alan Kennington wrote:

 > has anyone succeeded in using the SuSE 6.2 version
 > of mutt/pgp?
 > I just tried to use the pgp within mutt on both
 > a laptop (travelling with me in San Jose), and
 > a K6-2/400 at home, and it failed identically
 > on both. 
 
Put this in .muttrc in addition to the pgp_default_version bit:

set pgp_v5=/usr/local/bin/pgp
set pgp_v5_secring=~/.pgp/secring.skr
set pgp_v5_pubring=~/.pgp/pubring.pkr

Mutt finds pgpk, pgpe, pgps, etc by appending k, e, s, etc to the value
of $pgp_v5.  If you don't set it, it goes looking for commands
called "k", "e", "s", etc, which is why you're getting "command not found"
errors reporting them.
