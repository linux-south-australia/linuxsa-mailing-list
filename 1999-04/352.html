Re: glibc and staroffice

>>>>> "R" == romana  <tpnet@ozemail.com.au> writes:

  Romana> i have an odd problem...  i wish to install glibc 2.1.1, as
  Romana> several upgrades require it (gtk etc) but when i upgrade
  Romana> using the rpm, staroffice no longer works, saying the
  Romana> ld-linux.so files are not correct. i need staroffice, but i
  Romana> wish the new glibc libraries..any ideas?

Hi Romana,

Your problem is not so odd... Check out the debian-user mailing list
archives at http://www.au.debian.org/Lists-Archives/ for Mar and Apr 1999.
Look for "Subject: Star Office 5 & Potato/Glibc2.1??"

Here is a summary of what worked for one user (I'm sure you can
convert it to Red Hat equivalent commands):

========================================================================
1) I got the libc deb from slink. [slink == Debian 2.1, which uses glibc 2.0]
2) I manually unpacked the archive (ar -x libc???.deb; tar xvfz data.tar.gz)
3) I edited the soffice wrapper:
   a) In the section where it sets the LD_LIBRARY_PATH (I believe it is the
      *) option in the block that starts with "case $sd_platform in"), I
      added the path where my glibc2.0 is located to the BEGINNING of the
      LD_LIBRARY_PATH.
	 LD_LIBRARY_PATH=/home/myuser/glibc2.0/lib:$sd_inst/lib:$LD_LIBRARY_PATH

   b) The last line of the soffice script, I changed as follows:
	exec /home/myuser/glibc2.0/lib/ld-linux.so.2 $sd_inst/bin/$sd_binary "$1" "$2" "$3" "$4" "$5" "$6" "$7" "$8" "$9"

It took a *WHILE* (8-15 seconds) for it to load the first time (I run a
P200 w/128M RAM), but it did load, and seems to function properly.  When the
program first loads, it gives me a message box that says, "Error opening
document /home/myuser/Office50/bin/soffice.bin: Nonexistent object.  Filter
not found."  I click the OK button, and it works.
========================================================================
