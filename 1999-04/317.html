Re: The MindCruft Benchmark: The Samba Team responds

> For the un-initiated, DU (Tru64 UNIX now, boy that name sux :-) employs a
> unifed buffer cache, which means that file system caching and VM compete
> for all of memory. The default behaviour of DU is to allow 100% of memory
> to be used for file system caching. 

This sounds vaugly like Windows 9x, where the default file cache is 50% of
RAM, and there is no control pannel section to change this. You have to
inserk the line

MaxFileCache=MemoryinKB

into the [VCache] section of system.ini to change this (even W3.11 you
were able to change this in the 386 control pannel). From experience for
average use, 1/8 th of RAM up to a max of 8MB if plenty for every day use.
