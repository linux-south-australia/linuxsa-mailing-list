WARNING-pm4 & ext2

people: be careful using partition magic to resize ext2 partitions. i did it
last night to get another 150 meg for linux (as opposed to dos/win98).
all went fine, and dos progs such as ldir & lread can read the partition
fine. however:
rebooting into linux gives  several pages of
  "ll_[somethingerother about inodes] only supports 4096 (1024)"
and refuses to run anything (mount, bash, login, etc.)
booting 'linux single' from lilo comes up with a bash prompt, but no files
can be found. (eg ls, cp, cd, etc...) [tab] command completion doesnt work,
either.
has anyone had this problem?
does anyone know of a way to restore the inode size to 4096?

(looks like ill have to free up space on another drive, move everything
over, recreate the partition, and move everything back. oh, dear.)

so BE CAREFUL!
