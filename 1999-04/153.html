RE: SAMG meeting Friday 9/4/99 - The Clipsal C-Bus Automation system.

Hi,

I worked as one of the service tech's on this system..
It's worth a look, we at that stage, were toying with the idea of a java
interface, running on Linux.
Putting houses on the net to be controlled via a WEB browser, I don't think
Clipsal continued with this idea.

The system is very dynamic and programmable, allowing firmware upgrades of
the modules via the external client program.
The system is relatively easy to install and configure, based on low voltage
communications between modules, utilising ordinary twisted pair cable.

The system allows , via a serial connection, full control of lighting, fans,
switches, IR sensors, Light dimming, security, Light sensors , etc.
All this has the ability to be controlled / scheduled via a client on a PC.
It has the ability to connect multiple building together via modems, so you
can control multiple buildings from the location.

Think of it, press a button in the US and turn on a light in your lounge
room in Australia.
This can be done now, this doesn't need the interaction of any dedicated
computer after it has been set up.
The Web activation idea was an extension to this..

The programming of the modules is very open and flexible, but I understand
this is under licence by Clipsal.

Overall a very cool product.
