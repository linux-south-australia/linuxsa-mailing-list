Re: Linux in real time (was More Meeting Information)

Daryl Tester wrote:

> Dispense with the OS, and program to the metal.  :-)

What is usually done is "dispense with software, program
the metal".

It is usually simpler to provide hard real-time guarruntees
with electronics than with software.  FPGAs make it pretty
straight-forward.

You can then use a non-real-time OS to initialise the
FPGAs and to provide the less time-constrained functions.

Of course, adding specialist electronics implies writing
device drivers, and this is a lot easier in an OS with
loadable kernel modules and a huge selection of sample
source code :-)
