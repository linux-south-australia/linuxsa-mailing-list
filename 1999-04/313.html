Re: Syslog and logrotate

On Fri, 16 Apr 1999, Mark Telford wrote:

> Dear Peoples,
> 
> I have a problem with the syslog process stopping on a linux RH 5.1 box. The
> box has the mundane task of taking syslog messages from all the major
> systems on the network and placing them in /var/log/messages. Logrotate is
> set to rotate the logs daily ie messages becomes messages.1 etc. 
> This all works fine except for at around 4am every morning syslog stops.
> Does logrotate stop it? How can I check when logrotate will actually
> execute? 
> We reboot it once it stops and off it goes for the next 24 hrs.
> Does anyone know how to keep syslog going without rebooting the machine at
> 4am.

If by 'stops' you mean syslogd is still running but not doing anything,
then yes the logrotate program is probably causing it - after you move
syslog log files around you need to send syslogd a HUP signal 
(killall -HUP syslogd) to get it to check and reopen log files.
