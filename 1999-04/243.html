ou write:
> I have two questions about substitution in bash.
> 
> 1. Firstly, with a for loop can you use arithmetic substitution for the loop
> index?
> 
> I thinking of something like:
>     for i in "1 to 3" 
>     do
>       # whatever
>     done
> 
> instead of:
>     for i in 1 2 3
>     do
>       # whatever
>     done
> 
> The second way is fine for short lists, but not for longer ones.  I could
> use a while, but I'm curious. 

Generic sh:   
	i=0
	while [ $i -lt 3 ]
	do
		# Whatever
		i=`expr $i + 1`
	done

Using Bash extentions:
	i=0
	while (($i < 3))
	do
		# Whatever   
		i=$(($i + 1))
	done

> 2. Secondly, can you simulate arrays by using substitution in variable names?
> 
> I'm thinking of something like:
>    for i in 1 2 3
>    do
>      A$i=whatever
>    done
> 
> instead of:
>   A1=whatever; A2=whatever; A3=whatever
> 
> I realise there are lots of references out there, but I'm being lazy.  I
> would welcome anyone directing me to reference that has some examples.

Bash has arrays anyway, but you can do this using eval:
    for i in 1 2 3
    do
      eval A$i=whatever
    done
