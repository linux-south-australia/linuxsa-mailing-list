Meeting topic? [was: Compiling Problem]

> A friend of mine doing the same programming subject as me gets the
> following error when he tries to compile...

Is there any interest at all in a meeting topic something
like "writing software using C under Linux"?  Say covering:

  - using gcc
      - header files
      - type and variable scope
      - storage allocation and thread-safe code
      - good and poor function interfaces -- compare tcl.h and string.h
      - gcc has some nice features -- __inline__ v #define
      - design for test

  - ld
      - multiple C source code
      - static v dynamic
      - creating a dynamic library
      - linking other languages -- calling conventions
      - runtime load and link -- installable modules

  - make
      - makefiles
      - automatic creation of makefiles using cpp
      - configure
      - X does it differently -- imake and mkmf

  - debugging with gdb
      - testing a function
      - why did my program core dump?

  - finding hotspots with gprof

  - documentation
      - texinfo
      - sgmltools

  - building a coding environment
      - (X)emacs
      - productivity, motivation and environment

  - distribution
      - creating a tar.gz
      - creating a rpm
      - standard files names
      - home web site and mirroring
      - mailing lists and bug databases

I realise that this is "bread and butter" to quite a few of
us, but this seems to be the area that the more generic
books on C don't mention.
