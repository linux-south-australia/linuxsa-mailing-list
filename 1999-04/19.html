Re: dhcpd and DNS

The default route of 0.0.0.0 is telling the system what to with packets that
it knows nothing about (ie external ip addresses that can't be resolved as
local by the subnet mask.)

Using the network address is for the network you are using, remember
multi-hosting, you are required to tell the network module how to get out of
this machine and onto the correct network via which interface.


I realise the I haven't mentioned DHCP, but hey, they same thing applies,
you need to tell the machine how to handle the packets. The Default Route is
critical.

Is this as clear as mud ? If this information is wrong, hopefully somebody
will tell me to pull my head in!
