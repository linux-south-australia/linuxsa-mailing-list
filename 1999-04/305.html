Re: Funny entry in /var/log/messages

Mohamed Ismail Mohamed-Ibrahim wrote:

> I got this entry '-- MARK --' in my log file this morning a few minutes
> after booting up. I've never seen this before. No other evidence in
> /var/log/secure or ps. What gives??

Someone has either explicitly or implicitly enable the mark facility
in your /etc/syslog.conf file (look for something like mark.*).
According to my manpage for syslogd, the "-- MARK --" message is emitted
every 20 minutes by default.

> They occured exactly 20 minutes apart but I haven't got any new entry
> since then.

If no further activity is occuring in that particular log file, it
is probably queuing it up for the "last message repeated x times"
message.  If you login again (or any other activity that will log to
that same file), it will force syslogd to write the pending messages.
