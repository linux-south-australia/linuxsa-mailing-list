Re: COPY FILESYSTEM

Brian Marr wrote:
> 
> To all
> 
> I have filled my Linux partition (460Mg hdc1) with no room for additional
> software etc. I decided to follow the Hard Disk Upgrade Mini How-to and
> copied my filesystem using
> cp -ax / /hdc5
> to its neighbouring 3.4 Gig hdc5 partition. It is mounted under /main in my
> filesystem. I added /proc after as this stores memory. I then told Lilo to
> boot hdc5 and initiated it with
> /sbin/lilo.
> All seemed well. All the files are copied under /main. The system is
> working.
> However the filesystem still loads as before and looks as before.
> I probably need to change the root filesystem to /main.
> I hoped to use
> chroot bash
> However Bash does not seem to recognize (unknow command) this no matter what
> I try. Am using RH 5.2.
> 
> Any suggestions would be appreciated
> 
> Brian Marr

if you copied all of / to your new hdd, try booting off of that instead
of your older one.

try adding / changing stuff in lilo.conf. im sure there is a command to
do it to another drive, but i can't remember it.

alternatively, use loadlin:
copy your /boot/vmlinuz-* file to a directory on a dos filesystem,
(floppy?) and run
  loadlin vmlinuz root=/dev/hdc5
from dos
