Re: X fonts

Glen Turner on the aus.computers.linux newsgroup suggested editing my
XF86Config file, that made a big difference. For the benifit of others
I've included the relevent section of the edited file below. Thank you
Glen!

    FontPath	"/usr/X11R6/lib/X11/fonts/misc:unscaled/"
    FontPath	"/usr/X11R6/lib/X11/fonts/75dpi:unscaled/"
    FontPath	"/usr/X11R6/lib/X11/fonts/100dpi:unscaled/"
    FontPath	"/usr/X11R6/lib/X11/fonts/misc/"
    FontPath	"/usr/X11R6/lib/X11/fonts/Type1/"
    FontPath	"/usr/X11R6/lib/X11/fonts/Speedo/"
    FontPath	"/usr/X11R6/lib/X11/fonts/75dpi/"
    FontPath	"/usr/X11R6/lib/X11/fonts/100dpi/"
    FontPath    "unix/:7100"

EndSection

I downloaded and installed xfstt, it works OK if you remember to start it
before you launch X. I managed to find a rpm version, which helped. I'm
running Linux on a dual boot Linux/NT machine so I just copied across the
true type fonts from NT and they worked fine. I haven't tried to print a
document yet but at least the desktop looks much better.

Thanks for the help.

On 4 Jan 1999, Phil Nitschke wrote:

> >>>>> "SR" == Stephen Reynolds <spr@ozemail.com.au> writes:
> 
>   SR> Why do the fonts in some applications display so poorly when
>   SR> running X?  
> 
> Without knowing more about your system, I can't say _why_ this is
> happening.
> 
>   SR> In particular I'm referring to the fonts in Netscape,
>   SR> WordPerfect and similar X apps. Some menus are all but
>   SR> unreadable and the small fonts in the Netscape email client are
>   SR> awful. While Terminal apps like Pine and MC display fonts well
>   SR> in xterm.
> 
>   SR> This isn't a problem with a particular machine, it seems to be a
>   SR> common problem on all the machines that I've installed RedHat
>   SR> linux on.
> 
> Many users on the mailing lists that I read reported being much
> happier with the fonts when running xsftt:
> 
>  Description: TrueType Font Server for X11
>    xfstt means "X11 Font Server for TT fonts".
>    TT fonts are generally regarded to be the best scalable fonts
>    for displays. Applications needing scalable fonts that are to
>    be displayed on a screen benefit most.
>    This server will allow X11 applications to use the exact same
>    fonts as the TrueType fonts used on most Windows Machines.
>    NB: This package contains NO FONTS. They MUST be obtained separately
> 
> I use (among others) the TTF files that were supplied with a version
> of win95; these seem to make many netscape pages look better.
> 
> Another thing you can do is install more fonts for your Xserver.  My
> debian box includes this list:
> 
> ii  freefont        0.10-8         Freeware font selection for X11
> ii  gsfonts         5.10a-1        Fonts for the ghostscript interpreter
> ii  sharefont       0.10-7         Shareware font selection for X11
> ii  xfonts-100dpi   3.3.2.3a-8.1   100 dpi fonts for X servers
> ii  xfonts-75dpi    3.3.2.3a-8.1   75 dpi fonts for X servers
> ii  xfonts-base     3.3.2.3a-8.1   standard fonts for X servers
> ii  xfonts-pex      3.3.2.3a-8.1   fonts for minimal PEX support in X servers
> ii  xfonts-scalable 3.3.2.3a-8.1   scalable fonts for X servers
> ii  xfstt           0.9.10-3       TrueType Font Server for X11
> 
> If your machines are networked, you can install lots of fonts on one
> box and turn it into a font server for the others.  Then use something
> like this to list the fonts:
> 
>   fslsfonts -server <machinename>:<portnumber>
>   e.g. fslsfonts -server my.computer.net:7100
>
