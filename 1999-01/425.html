Re: cybercafe issue

> 
> I'm thinking to set up a cybercafe and definitely will be using linux as 
> the server/router/firewall for the local system. The workstations will 
> be using Win95/NT for user convenience. My problem is, how to automate 
> the users access to the net, e.g user A pay for an hour access, so after 
> an hour, his access to the net will be block automatically. I also think 
> to use some sort of local messaging program such as talk to warn the 
> users of his access time to the net. E.g. if the user A have 10 minute 
> access time left to use the net, the message system will warn the user 
> about it, so it won't be to harsh to cut his/her access instantly 
> without warning. Can these idea being implement to a system with linux 
> as a server and Win95/NT as the client?

Should be reasonably easy.

Ensure that each system has a fixed IP address.  Each system routes through
a single Linux box.  Alternatively force them through a proxy.

On user time out generate a MS windows popup though samba.  This informs
them.  At the same time use firewalling rules to block the user IP access.

So in short, the solution should be yes.  In practice it might take a bit
of glueing together.
