I know this probably isn't much help, but I actually use a directory on my
win98 partition to download files to while I'm running linux, which is 90%
of the time.  I often (accidentally) untar/gzip them to the windows 
partition, and then delete the tree I've created, then put it in the right
place on whatever linux partition..  in other words, I'm creating and
deleting files with 'long' and 'short' filenames on my win98 partition all
the time, and I've never had a problem.

One interesting point though, and I'm not sure exactly how relevant this
is.. when I'm running wine (which admittedly isn't often), the standard
'open' dialog boxes stuff up the long file names..  giving me things like
"prog~fbu" when you'd expect either "progra~1" or "Program Files".  This
seems wine-specific at the moment... but doesn't wine just use the
standard linux fs functions to access the windows partitions?
