> dont need the X server, that is on the unix box already, just the client

Hi.

I think that what you need is the X server, not the clients.

X clients are programs, like netscape, xcalc, gimp, and so on, that use X
library calls to perform actions on the screen.

X servers are operating system dependent programs that implement the
library calls that client programs use, so, if you have:
    (S) a server machine, running Linux, with the programs you want to run
    (C) a client machine, running NT/95/98, where you want to sit and watch
all those programs being run then you'll need:

The X client programs (netscape, xcalc, gimp, ...) installed on the server
(S)
An X server installed on the client (C)

After that, open a remote session(telnet, ssh, ...) from C to S over the
net, redirect the display from S to C (export DISPLAY=C.your.net.add:0)(ssh
does that automatically).
And then just run the programs you want.

Hope this helps. If not, just type in more details.
