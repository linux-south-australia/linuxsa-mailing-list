Hi,

At 12:14 AM 1/26/99 +1030, you wrote:
>Yeah, there is a solution to this problem, but not in Javascript!
>
>You can use the Expires and/or Pragma headers to instruct the client
>not to cache the document
>
>eg.
>
>#!/usr/local/bin/perl
>
>print "Content-type: text/html", "\n"
>print "Pragma: no-cache", "\n\n"
>
>and/or
>
>#!/usr/local/bin/perl
>
>print "Content-type: text/html", "\n"
>print "Expires: Tuesday, 25-Jan-99 01:00:00 GMT", "\n\n"

Hmmm, my understanding was that this just prevents the browser from caching
the document.  Thus it must fetch the document each time the user goes
back, and does not prevent the user from using history or the back key.

One technique I can think of is to actually modify the page location using
JavaScript in some way.  If you do View -> Page Info in NS, you will see
that each page has a Location property.  You can change this property in
JavaScript, and I have used that before, on a timer, to force a new page to
load after a delay period.
