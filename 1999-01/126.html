Re: easy modem question

David,

The modem problem is not a simple drop-out -- that would have been
easy to deal with.
The problem is the one that I described in detail in about
10 e-mails to linuxSA group a few weeks ago (end of December),
whcih is that the communication seems to be buffered for
7 seconds out of every 8, adn then all 7 seconds worth are let
through in just 1 second.
Someone else got a similar problem with a 33k Discovery
internal modem at one end. So that's why I suspect the modem.

The problem is that I have no way of knowing _where_ the buffering
is occurring. The loss rate seems to be 0. That's what's really curious
too. Packet loss is easy to comprehend.
I wouldn't be at all surprised if the "ppp" kernel module
does have a bug in it. I don't think that it is irreverent to
suspect that some linux code could have an error.

I'm going to try another modem now.
The problem would have been a hundered times easier to diagnose
if the itnernal modem just had a few TX/RX lights on the outside,
as my ethernet card does.
