But the use of xhost is a quick way of achieving something which
can be done in a lot more secure manner.  The use of xauth is a lot 
more secure.

The problem with using xhost is that it is host based, meaning that once
a trust is established any user on the bigbox can run applications on the
littlebox.

Using xauth allows you to only allow trusted users on bigbox to run 
applications on littlebox.  

On littlebox run 'xauth list'.  You will get output like 

	littlebox:0  MIT-MAGIC-COOKIE-1  590a082e0e5424563a3c6717271a5a65

connect to bigbox and then run 

	xauth add littlebox:0  MIT-MAGIC-COOKIE-1  590a082e0e5424563a3c6717271a5a65

Which will add the key for the user that runs the command on bigbox.  
When an X connection is attempted the application will send the 
key.  

This means that only the people who have the key can open windows.

Although it is a two step process you can automate it fairly easily
using rsh in a script.  If there is an NFS share that contains your home 
directory you will find that the xauth bit has been done (since you are
sharing the same .Xauthority file).

Althernatively you can use SSH and have the whole process done 
automatically.

Note that if you use telnet the DISPLAY variable will be set as part of
the connection.
