Netscape sessions

G'day, 

I have a question that probably has a simple answer but I haven't been able to
find anything on Deja News about it.  

When I use Xemacs, if I select a URL, Netscape is either loaded and the URL is
fetched, OR the currently running Netscape is sought out and the URL is sent to
it and it fetches the data.  Way neat.  

Both Pine and XFmail have options so you can do similar by adding the command
to start the web browser of choice.  

However, simply using the command: "/usr/bin/netscape %s" seems to ignore any
currently running session of Netscape and launches yet another session, which
complains that there's a lock file in use and it won't save bookmarks or cache
entries.  I also tried using the command "netscape %s".  I have a feeling
there's a switch I need to use, but typing "man netscape" tells me there isn't
a man page, and I didn't find anything in the /usr/doc/netscape directory that
enlightened me.  

Does anyone on the list have the answer?
