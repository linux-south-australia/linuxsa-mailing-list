Subject: Updated LRP-CIPE package for VPN

Sorry this has been so late. An updated packaged version of CIPE for Linux
Router Project (LRP) 2.0.36 final kernels is now available from
http://www.linuxrouter-encryptiontunneling.offbeam.com.au.

It is hoped the site is useful for those interested in implementing a VPN
solution through using and adapting a proven set of software and
configuration files. 

Comments made by CIPE author Olaf Titz on the previous FAQ-HOWTO were used
to correct typos and make comments indicating that CIPE module name used
is is appropriate to CIPE version series 1.0.x

Please feel free to make links to this site, if you are interested.

The site is physically located in Australia, which is free of legal issues
with regard to us of encryption technology. The site stops all ICMP
requests on its doorstep (goodbye to ping and traceroute revelations), is
very choosy about which port requests it allows in from from the outside
and has a nameserver that lies about its internal structure (internal
machines use a different nameserver). Am I being too much of bastard?

                           LRP-CIPE Package site
                                      
   This site only makes sense in the context of projects documented at
   two other sites: the Linux Router Project (LRP) site at
   http://www.linuxrouter.org and the Crypto IP Encapsulation (CIPE) site
   at http://www.inka.de/~bigred/devel/cipe.html.
   
   The purpose of this site is to provide a compilation of a CIPE package
   with modules for kernel of current LRP package, together with a
   detailed FAQ/HOWTO. The site is deliberately located in a country
