There is often confusion between the connection speed (i.e. the data
rate over the telephone line) and the serial line speed (sometimes
called DTE speed, i.e. the data rate between the computer and modem).
When my modem connects, it outputs:

----- 

CARRIER 24000

PROTOCOL: LAP-M

CONNECT 115200

----- 

CARRIER 24000 says that I've connected with a data rate of 24kbit/s.

CONNECT 115200 says that the serial line speed between my computer and
modem is 115.2kbaud (actually, it's an internal modem so it doesn't
have a serial line at all, but it still likes to think it does for the
purpose of communicating via the /dev/ttyS3 driver).

By default only the DTE speed is reported (at least on my modem).  I
had to set ATW1 to get both DTE speed and line speed reported.


AFAIK the reporting on "CONNECT" is the same for all Hayes-compatible
modems - anyone know any different?
