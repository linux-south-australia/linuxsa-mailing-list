Re: Problems with e2fsck and netscape

Much as I hate to respond to myself, I think this one is needed...

On Sat, 2 Jan 1999, Tony Galliford wrote:

> On Sat, 2 Jan 1999, Michael Doyle wrote:
> 
> > G'day
> > 
> > Problem 1, when rebooting I'm getting an error message advising that an
> > unchecked file system is being mounted, I have tried e2fsck -b 8193 /dev/hda
> > and get a reply of bad magic number, I have know idea what I'm doing here,
> > any advice welcome.
> 
> A couple of pointers:
> 
>   o  You'll need to e2fsck the partition, not the whole drive, that has
>      (or might have) the filesystem with problems.  eg if you have linux
>      installed on the second partition of the first IDE drive, try
>      ``e2fsck /dev/hda2'' .
> 
>   o  You won't need the ``-b 8193'' unless the superblock on the filesystem
>      is corrupted.  Try without it first.

And the biggest one of all (how could I have forgotten this?!?!)

   o  **NEVER** run e2fsck on a filesystem which is mounted read-write.
      Only run it on an unmounted filesystem, or a filesystem mounted
      read-only.  If it's your root filesystem, do ``mount -n -o remount,ro /''
      before running e2fsck
