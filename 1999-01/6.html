X buttons

e.g. User1 in X windows cannot use the X button to close the "x-term".
       User2 can!
Now the only difference using "ls -al", would be that User1 has a file
called ".kmid_collections", while
User2(which can use the X) has two other different files (but not the
.kmid_collections) and they are:  ".kss.pid" and "file.ps".

Whatever the reason for User1 not to be able to close "xterm" only with
"kill", I don't know. But one thing is for sure: -creating the user "joebag"
I discovered that if you enter in an "xterm" in X windows the command
"usekde" it will create and ".Xclients" file with the following content:
    #!/bin/bash
    # add any local customizations here, before startkde runs
    /opt/kde/bin/startkde

Next time you login, you will be taking to a desktop environment very
similar to the one that was presented by Matt at one of the meetings!
