IP Forwarding Timeouts

Hi All

I have managed to set up a small network at home using Redhat Linux 5.2 and
a Windoze box (never going to use WinGate again either!).  Everything is
working fine as far as I can tell, except for Telenet sessions from the
Windoze box (through firewall).

It works, it's just that if I leave a tail -f <file>, just to see what's
going on, it timesout.  Read man ipfwadm which states

-M     IP  masquerading administration.  This category can only be used in
combination with the -l (list) or -s (set timeout values) command.

and goes on to state

-s tcp tcpfin udp
    Change  the  timeout values used for masquerading.  This command always
    takes 3 parameters, representing the timeout values (in seconds) for
TCP sessions, TCP sessions after receiving a  FIN  packet,  and  UDP
packets,  respectively.   A  timeout  value  0 means that the current
timeout value of the corresponding entry is preserved.  This operation is
only allowed in combination with the -M flag.

This is my current command

ipfwadm -F -a m -S 192.168.0.0/24 -D 0.0.0.0/0

Okay, so my questions are:

1)	Is there a default timeout value for telnet sessions going through a
firewall?

2)	Where would I find the default values for timeouts? (cat <something>?)

3)	What's the syntax necessary to ensure that telnet sessions don't 	timeout?

TIA

Kirsty Rae
