easy modem question

Topic: EISA modems

Whereas both of my Discovery 33.6k EISA modems fail in the same way on
my 486DX-66 fail in the same way for my 24 hour/day permanent link
to bigpond, it seems to me that I need a completely different sort of
modem. I think there may be a faulty design here -- or some
subtle interaction with the ppp code or the ISP's router etc. etc. etc.

Anyway, I've tried finding a cheapish 33.6k EISA modem at
the shops in Adelaide, and all they seem to have in EISA modems is
the Discovery, and a $150 Mitsubishi modem, which is way
overpriced, I think. It seems that all new internal modems are PCI.

I think that this is a linux-specific problem because
people using the other main OS for PCs generally woudl not be
using an EISA-only PC anymore.
Linux users are much more likely to be running an old computer
over a permanent modem link.

Of course, there may be anohter option -- could I maybe use
some sort of serial port card to give me a high-speed serial
connection of 115000 bits/sec (or whatever) so that I can
use an external 33.6 modem?
I imagine that an old PC will only go up to 19.2 kbits/sec.

Some of you must have come across this problem before.
How do you get a high-speed external modem to work with
the old UARTs?

Cheers, and merci d'avance...
