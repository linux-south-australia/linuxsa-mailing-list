Re: extern C keyword

Benjamin Close wrote:

 > It is in a seperate file from the rest of my source code and there is a
 > header file fred.h containing:
 > 
 > int fred(void);
 > 
 > Should I make this:
 > 
 > extern int fred(void);

No, but if fred.c contains:

     struct rock *barnie;

You might want to put:

     extern struct rock *barnie;

into fred.h so that you can use the global variable "barnie" in other
modules.

You don't need extern for functions;   You use it for global variables.
