Re: kernelcfg

Hi Nick,
I suspect that you will need to rename the file accordingly ie.`mv
/boot/module-info-2.0.34-0.6 module-info-2.0.34`.

If you are actually running kernel 2.0.34 (it appears so from the kernelcfg
message) then you will need to also recreate the symlink for
/boot/module-info ie. `rm -f /boot/module-info ; ln -s
/boot/module-info-2.0.34 /boot/module-info`.

Of course these procedures will have to be done while logged in as root.

The problem most likely arose as a result of a kernel/kernel-modules update
being installed.
