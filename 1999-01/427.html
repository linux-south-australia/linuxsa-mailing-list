firewall initialisation

Linux OS question.
Topic: how to make firewall start up on boot-up

In the literature that I've read, I have not seen how to
create some sort of configuration file to set up the firewall
on linux 2.0.36 (readhat 5.2).

I have written a script which calls "ipfwadm" about 10 times
to set up the firewall, and that works fine.

But if the machine re-boots, it will re-establish the internet link,
and will be without firewall.
Obviously I can put a line in some boot script to invoke the
10 "ipfwadm" calls.

But is there a more reliable and better-engineered way of doing this?
I was hoping there would be some file like "ipfwadm.conf"
in the /etc directory which would be read at boot time to
do the firewalling.

Question: Is there such a thing?
