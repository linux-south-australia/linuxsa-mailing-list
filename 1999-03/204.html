Re: graphics

> OK strike one on any info for sound programming. How
> about graphics? Any good links for this? All I need to
> know is how I can designate a pixels colour in (x,y)
> format or a line in (x,y)(x',y') if possible, and then
> wipe the screen.

My suggestion here is use Open Source to it's full advantage.
Find a simple utility that does something similar to what you
want, and learn from the code.

Sound, a utility like 'play' should get you started.  For graphics,
a X program like 'xload' should do fine.  They make good starting points,
and form there you can dig a lot deeper.
