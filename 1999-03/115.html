Re: Modem problems fix - Re: pppd delay (fwd)

justin@tardis.mx.com.au wrote:

 > > >> The dreaded +++ sequence is not a likely cause, by the
 > > >> way, because it needs an idle time on either side.
 > > >
 > > > Not on all modems.  And sometimes the manuals lie about this too.
  [ ... ]
 > Which introduces an interesting piece of trivia - modems without the guard
 > time delay around the escape sequence are open to a ping based DoS attack.
 > If you know their dialup IP address you can tell ping to send them a
 > packet of data with the +++ in it, when it replies it will switch itself
 > into command mode and kill their connection!

Solution:  Put "ATS2=0" in your dialout modem init string.  If you're
using PPP on a line there's no rational reason to permit a keyboard
escape sequence in the first place, so turn it off before you dial.
