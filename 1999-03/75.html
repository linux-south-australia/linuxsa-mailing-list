RE: Info on SQL

> 1) Any comparison or write-up on these two things available ?
> 
	There are plenty of comparisons on the web. From my own experience :

	- MySQL is a little bit faster overall. MySQL is about 10% faster
doing writes, updates and simple queries compared to a properly configured
PostgreSQL site.
	- mySQL is MUCH MUCH faster than a poorly configured PostgreSQL
site.
	- PostgreSQL acheives better performance when doing very complex
queries, compared to mySQL.
	- MySQL tends to drop the connection after an extended period.
PostgreSQL will keep the connection between idle clients and the server open
forever.
	- PostgreSQL has built in transaction handling (commit / rollback,
etc), as well as some nice feature supporting OO, such as inherited table
defns, etc. MySQL leaves these features out in the name of  extra
performancce.
	- A properly setup PostgreSQL server will survive power cuts during
database updates without any problem, MySQL is generally not as robust as
this.

> 2) Is there any difference or is it the same ?
> 
	- MySQL is a bit picky about what sort of fields you can use for
indexes and things. If you want to take, say, an existing Oracle / Sybase /
DB2 database and port it to a free SQL, then PostgreSQL makes your life
slightly easier than mySQL because it supports more of the 'standard SQL'
constructs.
	- mySQL on the other hand has some proprietry, but very nice,
extensions to the SQL command language.

> 3) If yes, which is much more reliable and better ?
> 
	- PostgreSQL easily wins this comparison.

	Hope that helps ...
