Re: Setting a path

Matthew Mills wrote:
> 
> Hi, Still learning the ropes and am having probs with installing apps in
> Linux in that they often get installed into /usr/local/bin or
> /usr/share/bin  which are both not in the standard PATH on a Redhat5.2
> installation, I have been searching on how to change my PATH, have tried
> ENV PATH=<paths appeneded here> but that only temporarily adds the path
> . There seems to be no SETPATH command under bash.
> Why is it that many/most apps install into a path that isnt listed in
> the path variables of a standard redhat5.2 Installation.
> 
> My system is a standalone workstation with 2 users (root and my account)
> 
> thanks in advance for what is probably a simple problem.  :-)
> 
> Matt

The PATH is just a shell variable, so you need to set it up each time
you log in. Global setup scripts for bash are /etc/bashrc and user
specific are at $HOME/.bashrc Put a line in there that goes something
like: PATH = $PATH:/usr/local/bin

Semicolon separates entries, $PATH just dumps what was already in the
variable (end result we append usr/local/bin to the old path) Also check
out the bash man page.
