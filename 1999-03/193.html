Something is wrong with my printer ?

>>>>> "Boo" == Boo Jiun, Ng <bjng@cs.adelaide.edu.au> writes:

  Boo> Hi!  I'd installed RH5.2 and everything is ok until I
  Boo> recompiled the kernel. The PrinterTool couldn't detect my
  Boo> printer on /dev/lp1. Please help!

In the kernel 2.2.x documentation, it warns that if you enable
plug-n-play on the parallel port, /dev/lp1 might change to /dev/lp0.
It did for me.  All you need to do is edit /etc/printcap, change
/dev/lp1 to /dev/lp0, and then run

lpc restart all
