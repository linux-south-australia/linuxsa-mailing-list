Kirsty Rae wrote:
> 
> Hi all
> 
> I've been playing around with SAMBA lately and have basically got it to
> work by reading the relevant docs and HOWTOs, but I'm now getting a new
> error in syslog which I don't understand and would appreciate an
> explanation of....
> 
> netbios-ssn/tcp: bind: Address already in use
> 
> I'm pretty sure that it has something to do with the following line in
> inetd.conf
> 
> netbios-ssn stream tcp nowait root /usr/sbin/smbd smbd
Yep, I'd say that's got something to do with it!
Why are you trying to run samba thru inetd though? It's a standalone
daemon, AFAIK.

> 
> I have commented this line out for now (my updated understanding is that
> I don't actually need it) and done a KILLALL, but I'm still getting the
> error message.
Okay, firstly, have you told inetd to reload the config file? If not,
that'd be the problem.
killall -1 inetd will do it for you, but it'd be neater to do a kill -1
`cat /var/run/inetd.pid`
man inetd for an explanation.

> 
> Any SAMBA gurus like to explain what's going on?
Try that first of all, and if the error is still occuring, check your
smb.conf and see if you're trying to bind to a valid interface/port...
You can specify non standard ones, but I don't think you would have done
this by accident/unknowingly. Are you running with a
standard/out-of-the-box smb.conf?
