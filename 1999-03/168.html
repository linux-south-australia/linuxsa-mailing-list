On the contrary, the modem is expecting no delays between characters. If
you read the manual, I believe some say that some modems require a 3
second delay both before and after the +++, and the +s themselves must
be entered within one second. This applied to my 14.4 modem anyway, when
I was still using BBSs, and I don't know if it applies to newer ones,
because I've never checked.
So, the above line should be
sleep 3; echo "+++" > /dev/modem; sleep 3;
except that the sleep afterwards is perhaps unneccessary, and possibly
the one before is as well. IE> you're back where you started ;)

I beleive that the locking problem is more likely what is stopping you.
The actual .lock file is only there to indicate that the file is locked,
rather than to make the locking itself. I think that's handled at a
kernel level. Are you trying all this as the root user, by the way?
