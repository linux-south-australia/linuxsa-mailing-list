> Toby - Thanks for your reply. It's kind of hard to know where to begin
> reading when you're a beginner, you don't get any error messages and the
> problem is that the application just doesn't do anything!!!
There often are error messages - it's just that they are stored in a
messages area rather than popping up automatically all the time.
Have a look at the files in the /var/log/ directory, in particular
'messages'.
Try using tail -f /var/log/messages, as this will display the last 10
lines of the file, but then keep monitoring it for more details. I
usually edit the /etc/syslogd.conf file so that the same info that goes
to messages also gets pasted to /dev/tty12. (ie. hitting Alt-F12 allows
me to view what's going on)
You may find something in there that helps, but in this case probably
not.

> 
> Is your default route correct? (ie. pointing to your ppp interface, with
> a gateway?)
> 
> This sounds promising ... How would I check / fix that? (Sorry, I'm not at
> the machine in question at the moment so can't play around myself ....)
Okay, type "route" while dialled up, and see what comes up.
Things ending with pppx as the interface are what you are looking for.
There will definately be one there, as a static route to the gateway
machine that you are dialled up to. However, there should also be a
default route at the bottom of the list. This should be pointing to
ppp0. If it is absent, or if it set to eth0 or something, then you will
need to change it. `man route` will help, but essentially it's 
route del default; route add default ppp0
See if that helps.

If things are still broken for WWW, then can you see if you can telnet
or ftp or ping outside hosts?
try `ftp ftp.redhat.com` or something - see if you get anywhere.
try `ping -c 16 139.130.4.5` and see what you get.

If you CAN use ftp and ping to get out, but not WWW, then I'm at a loss
as to what is going on.
However, I think you'll find that the default route might be your
problem. If so, to correct it add a line saying 'defaultroute' to your
appropriate ppp options file
