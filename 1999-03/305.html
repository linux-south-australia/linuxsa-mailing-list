Re: Socket programming question

On Wed, 17 Mar 1999, O'Connor, Steven wrote:
> According to the man pages, [accept] should return a nicely filled
> in struct sockaddr_in, but in my test cases, I get the structure
> filled in with 0's all the time. (My test cases are connections
> from localhost)

In addition to Mark's nice reply, you need to ask yourself which
IP address did you use when you connected to the socket?  When you
make an outgoing connection you normally use the IP address IPADDR_ANY
(all zeros).  The address of the outgoing interface is substituted
when the packet is actually sent.  For localhost, the interface
address is 127.0.0.1, however perhaps the network layer optimizes
and leaves it as 0.0.0.0.  (I would think it shouldn't, but perhaps
it does.)
