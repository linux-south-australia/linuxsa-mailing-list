> I'm still getting this error.
> 
> [michael@hobbes michael]$ esd -port 5000 
> initializing...
> - accepting connections on port 5000
> unsupported sound format: 33
> fatal error configuring sound, /dev/dsp
> 
> BTW I have a Sound Blaster Pro II, which IS configured using sndconfig.

Hmm, I wonder... Does esd *require* 16 bit sound?  IIRC Sound Blast Pros
are only 8 bit sound (in stero).  If you can have a look at the source
code and see if you can get what tests result in it complaining.

The other thing you might want to try doing is having a look at how the
application is communicating with /dev/dsp and the kernel.  Use strace 
to get the output.  Gut feeling is that an ioctl is being issued and 
is returning with EINVAL (invalid value) - rationale is that it is 
trying to change the operating mode of /dev/dsp and geting a failure.
