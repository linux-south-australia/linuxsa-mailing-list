Re: fsck

Unfortunately, the bad area was about where /var/log was sitting, so
'less'ing messages caused the error. Eventually, I used 'partition magic' on
the drive, which did identify hda1 as a e2fs partition, and reported a
'critical' error. The help on 'critical' started bandying words line
'catastrophic' and 'unfixable' about, so I grabbed a spare disk from the
cupboard and rebuilt. Despite the disk errors I was still able to do a save
to tape, so anything vital should be able to be resurrected.

thanks everyone for the help
