RE: X window server for windose

You need to start an instance of xdm running. Before you do edit the 
xdm-config and Xaccess files.

In xdm-config you will find comments that indicate the last line needs to 
be commented out if you want to have xdm manage session on a remote x-server.

Depending how you want to connect to the linux box you will need to edit 
the Xaccess file. The default setting in Xaccess is to respond to broadcast 
request only. If you want the machine to response to queries (direct 
requests) you will need to edit the file appropriately. The embedded 
comments should guide you.

One last thing. Make sure you have and entry in the /etc/hosts file on the 
linux machine matching the ip address and hostname of the xwin32 box. If 
you don't have this you are still likely to get permission denied with 
correct settings in the xdm-config and Xaccess files.

You may have problems with xdm if gdm or kdm are running. I not sure why 
but I have had an entry in /var/run for xdm-pid which was actually caused 
by gdm and stopped me starting xdm.
