Re: hostname and defaultroute questions

Michael Kratz wrote:
> 
> hi,
> 
> two questions:
> 
> 1. what actually determines the hostname of a machine,

on a redhat machine, the hostname is actually determined by
/etc/hostname

This is what the machine thinks its name is. Actually it can be
addressesd as many different names.

> ie. say the machine
> has eth0, eth1 running and both of these have different host names and IP
> addresses can the machine have its own, ie.


when you bring up eth0 and eth1 they will have different IP addresses.
the "hostnames" you talk about for these interfaces are probably what
you see when you do a ping or netstat. These hostnames are what your DNS
client has found. Your upstream providers (sometimes) have names for all
there ips, so this may be what you mean here.

> 
> machineA.domain.tld 10.0.0.1 (mail, etc. appears to come from this host)
> eth0.machineA.domain.tld 10.0.0.2
> eth1.machineA.domain.tld 10.0.0.3
> 
> if so then where is the setting to set the ip address of the system? if not
> then which one should be assigned to the system??

if your machine has real-world IPs (the above example shows you dont -
but perhaps you do) then you need to set up the DNS for your machines. 

> 
> --------
> 
> 2. if I have a config like below:
> 
> LAN <-> eth0(machineA)eth1 <-> eth0(machineB)ppp0 <-> NET
> 
> how do I set the default route for machineA to pass NET traffic onto
> machineB?? I am thinking that it is a default gateway switch in ifconfig, am
> I correct?


Yes. You set the default gateway on machineA to be machineB. That way
when machineA has a packet with destination address that doesn't match
anything in its routing table, it passes it to the default gateway. The
default gateway on machineB should be the default gateway on your
upstream provider.



> 
> ----------
> 
> Yes these two questions are related, ie.
> 
> eth0 machineA 10.0.0.2
>      machineA 10.0.0.1 (not sure if I can do this thats why I'm asking)
> eth1 machineA 10.0.0.3
> eth0 machineB 10.0.0.4
>      machineB 10.0.0.5 (not sure if I can do this thats why I'm asking)
> ppp0 machineB 10.0.0.6


I'm not sure why you want 3 IPs for every machine, but it is possible.
This is called interface aliasing. you actually call the interfaces
eth0:0 eth0:1 etc.
