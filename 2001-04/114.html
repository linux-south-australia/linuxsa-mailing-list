Re: Server Shares and worms

Adam Dixon wrote:
> 
> Is there a way to stop completely or even minimise the spread of a worm via
> a share which is read and writable to everyone? even the worms that may be
> infected on a  connecting computer?

Take a leaf from the unix security model. Executables and the
directories they're run from should not be writeable by normal users.
This may not be possible if you're using Win9x sharing, but you could
always share such directories as read-only. You might also face the
difficult task of educating users about the dangers of executing
anything that lets you double-click on it.

If you have a server with file shares, you might want to discourage
using normal Windows workstation shares, so that only shares with
appropriate permissions are used.
