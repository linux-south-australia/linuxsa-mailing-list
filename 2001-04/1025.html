Re: Struggling with gv (GhostView)

Hi Richard.

Yes, I have noticed that too. My printer has a paper substitute setting
Letter=>A4, which I know is a fudge, but it saves running around to see
what is beeping!

Just about everything to do with ghostscript is default to letter. I
change the important scripts like ps2pdf so sanity can prevail.

have you tried calling it with the -media A4 switch:

      -media <media>
              Selects the paper size to be used. Valid values are
              names  of  paper  sizes  that appear in the list of
              page medias as given by the medias resource.

and you could try adding the following to your .gv file:

       pagemedia
              Sets the paper size.  Allowd values are as given in
              the above list of paper-sizes.   Specifying  "auto
              matic"  causes  gv to attempt to derive the correct
              paper-size from document structuring comments.
