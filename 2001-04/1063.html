Re: Why use DocBook versus Word

On Tuesday, 24 April 2001 at 11:56:53 +0900, Richard Sharpe wrote:
> Hi,
>
> Well, I found one good reason for using DocBook ... I can modify
> things ...

I thought MSword would also allow you to modify some things.

> DocBook produces PDF by converting the SGML to TeX and then going to
> PDF using a tool called pdfjadetex ...

In fact, DocBook just defines the format.  jade and friends are just
one way to format it.  Another is the gmat package from O'Reilly,
which I think does a better job.  They're all painful, though.  gmat
is available in the FreeBSD Ports Collection
(/usr/ports/textproc/gmat).  I don't know where you can get it for
Linux.

> There is a problem with TeX: any << or >> on the text gets turned
> into chevrons because the living god, Don Knuth, requires that you
> code them as <{<} or >{>} if you do not want a ligature between
> them.

TeX is a royal pain to write in.  DocBook is a different pain.  I
still prefer troff, but the writing's on the wall.
