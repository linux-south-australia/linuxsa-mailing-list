> on linux server(Turbo 6.0).Samba is working fine but
> my query is I want to share my windows drives or
> directories (NT-workstation) to my linux server.
> 
> when I am trying to connect to my localserver by
> giving the command #smbclient -L localhost -N it is
> not showing the shares.It is showing the total network
> PC's
> and when I am trying ot mount with the smbmount
> command to mount the wondows directory.... #smbmount
> \\\\winntserver\\share /tmp it is asking for the
> password, so when I gave the password it is not
> working it is giving the error as...session setup
> failed:ERRDOS - ERRnoaccess and SMB connection failed.
> 
> so please any one give the solution...I am very
> thankful to them....


Instead of using smbmount, try something like:

mount //SERVER/SHARE /mount-point -o
username=mgeddes,password=mypass,workgroup=WORKGROUP -t smbfs

Obviously exchanging the example values for username, password,
workgroup, servername, sharename and mount point for your own.You will
need smbfs support in your kernel.

Hope it helps,
