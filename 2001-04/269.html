Re: installing rpm src files HOW??

On 10 Apr 2001 22:02:26 +0930, Michael Kratz wrote:
> I have been trying for about 4 hours to install a src rpm of squid and it 
> just wont work. I think what I am supposed to do is, rpm -i squid.rpm then 
> cd to /usr/src/redhat/SOURCES or SPECS and the do rpm -ba squid.spec

When I last did this, you have to first build the package. As it stands,
the file contains the source for the application (in tar.gz, I beleive)
and information for RPM on how to compile it etc.

> But it does not want to work, it just lists the commands of RPM, how do I 
> install the rpm???

You'll need to use rpm --rebuild to create an binary RPM.

rpm --rebuild squid-2.2-WHATEVER.src.rpm

You should see a heap of messages fly by as RPM uncompresses and
compiles the package. If all goes well when it's finished compiling, the
binary RPM should be put in
/usr/src/redhat/RPMS/i386/squid-2.2-WHATEVER.rpm

Then just install it with

rpm -ivvh /usr/src/redhat/RPMS/i386/squid-2.2-WHATEVER.rpm
