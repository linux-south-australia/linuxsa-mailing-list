Re: LVM and Software RAID (was: Re: New hdd installation)

On Monday, 16 April 2001 at 20:02:26 +0930, michael wrote:
> On 16 Apr 2001, Andrew Reid wrote:
> 
>> On 15 Apr 2001 16:59:52 +0900, Richard Sharpe wrote:
>> 
>>> What you need is LVM or MD. Then you can create a larger volume than the
>>> one drive, and extend the filesystem on-line, and away you go.  LVM is
>>> probably the more flexible here.
>> 
>> LVM is not something that I've had a lot of dealing with under Linux.
>> I've read the HOWTO briefly, but I'm still a little unsure on the whole
>> subject. More reading is the go, I think :-)
>> 
>>> Of course, your reliability goes down, because if you loose either
>>> parition, you have lost the whole volume.
>> 
>> Can LVM be used with Software RAID? Can I create two volumes that span
>> across two disks each and then use Sofware RAID to do mirroring?
>> 
>> +----------------------+     +----------------------+
>> |    Group 1 (12GB)    |     |    Group 2 (12GB)    |
>> |  +-----+    +-----+  |     |  +-----+    +-----+  |
>> |  | HD1 |    | HD2 |  |     |  | HD3 |    | HD4 |  |
>> |  |     |    |     |  |<--->|  |     |    |     |  |
>> |  | 6GB |    | 6GB |  |     |  | 6GB |    | 6GB |  |
>> |  |     |    |     |  |     |  |     |    |     |  |
>> |  +-----+    +-----+  |     |  +-----+    +-----+  |
>> +----------------------+     +----------------------+
>> 
>> Can the two groups mirror each other using software RAID? If the two
>> volumes appear as normal devices, which I believe they do, I'd have
>> thought that this is possible.
>> 
>> Can anyone confirm or suggest reasons why this would not work?
> 
> Whilst it _might_ work, wouldn't it add too much overhead to do the
> lvm+raid all in software? For me, the tradeoff is redundancy versus speed.
> It takes a lot to beat a good daily or better backup and hardware raid
> mirroring. The promise h/w raid controllers are not dear and work with ide
> disks. I get the impression that raid5 is not terrific on them, (search
> ars-technica.com for more details) but we only need raid1 here.

I believe it does work.  I've been told it's very inefficient, though
I haven't proven it myself.  On the other hand, Vinum under FreeBSD
does exactly this, and it's not inefficient: it beats many hardware
RAID setups.  http://www.vinumvm.org/ for more details.

Note also that RAID-1 is a lot more expensive than RAID-5: you can
need up to nearly twice the disk space.
