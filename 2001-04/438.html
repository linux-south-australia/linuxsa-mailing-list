Telstra Bigpond ADSL weirdness: Update

Hi people,

Thanks for the suggestions from those who emailed me - I fiddled around a
lot with mtu settings, etc, but couldn't find anything that would fix the
problem. I also tried the 2.4.x kernel series' kernel-mode PPPOE support,
too - it works well, but doesn't fix the problem. Not surprising, really,
since all it changes is the way the PPPoE stuff is done, not the TCP layer
stuff which I'm pretty convinced is the problem. I'm still surprised that it
works under Windows and not under Linux - I don't see anything different in
the Windows TCP/IP configuration that gets used apart from the fact that I
always seem to get assigned an address from a slightly different class C.
I'd appreciate any more suggestions. :)

Is anyone out there apart from me actually *using* Telstra ADSL with Linux
and is able to get to their web/pop3/smtp servers?  Could I request your
/etc/ppp/options and the details of what you're running, if you're in that
enviable position?

My /etc/ppp/options is currently:

plugin /usr/lib/pppd/plugins/pppoe.so
name pviiret@bigpond
defaultroute
mtu 1452
mru 1452
persist
lock
noipdefault

Thanks,
    jv.
