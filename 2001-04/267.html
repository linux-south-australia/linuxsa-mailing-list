Re: routing metrics

> Could someone please explain what the Metric value is on ifconfig? Does it
> represent the order in which to try to get a packet to its destination? So
> if the the destination is unreachable on the first interface, it will try
> the next interface based on metric order?

No.

I believe that the routing metric is not used in a any routing decisions.
Instead, the first rule that matches will be used. The metric is used by
routing protocols/programs/algorithms, such as RIPd, to determine what order
the entries should appear in the table.
