On Wed, 11 Apr 2001, Richard Russell wrote:

> > How do I allow Users to reboot a machine?
> >
> > I am setting up a machine without a keyboard/monitor/mouse and
> > would like to
> > be able to let users reboot the machine via telnet or a web browser. any
> > ideas on how I can do this?
> 
> Rebooting is pretty much the same regardless of if your machine is headless
> or not...
> 
> I'd suggest one of the following, depending on your security needs:
> 
> * sudo access to `shutdown -r now` for the appropriate users on telnet
> * a reboot user, so people can just log in as `reboot`, type the password,
> and the machine reboots (this is just an /etc/passwd entry)
> * chmodding the binary to be setuid
> * making a script setuid (I think this is less secure than the above one)
> * giving them the root password -- then they can just shut it down normally
> 
> the web access option is similar, you just make sure you can authenticate
> the user in whatever way you want (or perhaps risk a DoS attack by leaving
> it open) and have a CGI or something caused by either a button click or even
> just by visiting a URL (eg http://mymachine/shutdown.pl could simply run a
> script that writes a HTML page that says it is shutting down, and triggers a
> shutdown using one of the above methods..)
