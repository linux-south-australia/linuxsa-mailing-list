Re: hostname and defaultroute questions

Jake + Michael!

> on a redhat machine, the hostname is actually determined by
> /etc/hostname
> 
> This is what the machine thinks its name is. Actually it can be
> addressesd as many different names.

Actually, the hostname is set in /etc/sysconfig/network. I've checked
with two other RedHat 6.2 distributions and /etc/hostname does not
exist. To quote from /etc/rc.d/rc.sysinit:

# Read in config data.
if [ -f /etc/sysconfig/network ]; then
    . /etc/sysconfig/network
else
    NETWORKING=no
    HOSTNAME=localhost
fi

And further on it does:

# Set the hostname.
action "Setting hostname ${HOSTNAME}" hostname ${HOSTNAME}

If you fiddled with rc.sysinit and read man hostname you could put the
hostname in /foo/bar/nonexistent/alienhostname...

> > then which one should be assigned to the system??

Or cope with /etc/hosts files...

> Yes. You set the default gateway on machineA to be machineB. That way
> when machineA has a packet with destination address that doesn't match
> anything in its routing table, it passes it to the default gateway. The
> default gateway on machineB should be the default gateway on your
> upstream provider.

And you do that in /etc/sysconfig/network-scripts/ifcfg-eth[number] on a
RedHat machine. I think it's fairly much the same on Mandrake, SuSe,
OpenLinux and so forth. I'm not sure about Debian. 

Looking in the ifup script I think the setting is GATEWAY=[the ip] if
you want to manually set it. If your gateway were to appear via ifup
ppp0 (i.e. it's an ISP link) then it might be easier to setup your PPP
connection to do the gateway stuff on its own.
