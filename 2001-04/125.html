Re: fsck

>fsck.ext2: No such file or directory  when trying to open /dev/hda1
>The superblock could not be read or does not describe a correct ext2
>filesystem. If the device is valid and it really contains an ext2
>filesystem (and not a swap or nfs or something else), then the superblock
>is corrupt, and you might try running  e2fsck with an alternate superblock:
>     e2fsck -b 8193 <device>
>
>I'm just a little reluctant to try this - I've got this funny feeling I've
>come across it before (when RH4.2 was new) and the block size wasn't
>actually the problem.

Do it.  You have a corrupt superblock and this will use a backup copy, 
restoring it to where it wants it.
