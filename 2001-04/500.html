Re: Slackware 7.1 and multiple interfaces

Richard wrote:

> Is it just my in-experience, or is Slackware pretty poor when it comes to
> support for multiple interfaces?
>
> The netconfig tool will edit /etc/rc.d/rc.inet1 and insert only the
> information on one interface, and will quite happily wipe out any other
> info you might have in there.
>........
> My suggestion would be that the primary interface be left in
> /etc/rc.d/rc.inet1 with everything else being done in /etc/rc.d/rc.local
...
>
That's precisely what I do on a machine here that has six interfaces. (on an
earlier Slackware, but still the same issue) While I dont normally use the
netconfig tool, if you look at rc.inet1 it talks a lot about the network
"card" - note the singular. And of course there is only one of each of the
required commands in that file.....

In particular, commands such as scripted "dial up" links etc need to go in
rc.local too. Sometimes this means you have to be a bit careful about the
sequence of commands, so that an interface is "up" before you go to use it -
a problem that doesnt occur with the interface done in rc.inet1.

cheers

Steve
