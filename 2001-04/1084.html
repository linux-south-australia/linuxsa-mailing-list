Re: samba again

At 08:35 PM 4/25/01 +0930, Robyn Manning wrote:
>Hi all

Hi ...

>More questions
>
>In Network Neighbourhood on Cliff's computer if you click on the Samba share 
>it asks for the password for IPC$, and therefore can't access the printer 
>share either.

This means one of two or three different things.

1. You are using a version of Windows that insists on encrypting passwords,
while Samba is not set up for Encrypted passwords.

2. You do not have a guest account

3. The person accessing the machine does not have an account on the server,
and you do not have 'map to guest = bad user' set.

4. The password used to log into Windows does not match the password on the
server. This can be avoided if you force the Windows machine to log into
the domain, in which case, you know that the passwords are the same.

>- I have made cliff, robyn and root Samba users and set their passwords. 
>- I used the Win98....reg file to set plain passwords on Cliff's computer.

OK, but the explicit entry for Cliff, below, suggests that Cliff does not
have an account, or his password on the server is not the same as that on
the client.

>Following is my smb.conf
>
>[global]
>        workgroup = STARFLEET
>        server string = Samba Server
>        log file = /var/log/samba/log.%m
>        guest account = guest
>        password level = 4
>        security = share
>
>[homes]
>        comment = %U's Home Directory
>        browseable = no
>        read only = no
>        writable = yes
> 
>[robyn]
>        comment = Robyn's Home Directory
>        browseable = yes
>        path = /home/robyn
>        writable = yes
>        read only = no
>        valid users = robyn, cliff

Ummm, why have separate entries for Robyn and Cliff? If it is only for
access purposes you can use a group and set the group correctly for them.
You are going to need that anyway, since Samba enforces UNIX permissions as
well as (read after) SMB access controls.

>[cliff]
>        comment = Cliff's Home Directory
>        browseable = yes
>        path = /home/cliff
>        writable = yes
>        read only = no
>        valid users = cliff, robyn
> 
>[printers]
>        comment = %p Printer
>        path = /var/spool/samba
>        printable = yes
>
