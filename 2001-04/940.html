Or unless you use something which implements the rsync algorithm. Apart
from the original rsync rcp replacement, there are two interesting
implementations, rproxy (an Apache module thingy) and something without a
name as far as I know which is designed to be built into the Debian apt
system. The idea is that you can cache streams of complicated binary
things which are somewhat similar, such as dynamic web content and the
contents of a Linux distribution.

The rsync algorithm is idea for this sort of application. It hasn't been
widely adopted yet but it could easily be so, and may yet be too. It has
the potential to reduce some kinds of dynamic traffic by up to 80%.

Internode or any other with-it ISP might like to consider offering
incentives for customers to run rproxy, or to connect to their rsync://
server.
