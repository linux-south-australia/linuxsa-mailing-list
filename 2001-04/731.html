On Mon, 23 Apr 2001, James Mclean wrote:

JM> didnt have VI! no serously, for my needs it didnt have enough features, (i 
JM> wanted proxy, httpd, ftp etc...) but of course, freesco may be fine for your 
JM> needs. but DOS boot disk? Yuck!

some of those features can be added using freesco's package installer, i
think it's called instpkg... you use listpkg first to get a list of
package names that are available to be installed (either from freesco.org
or from a site you give on as a cla), and then instpkg it... i've added
ssh, a telnet client, a nice editor and updated httpd using it...

and as for a boot disk, if you have dos installed on the pc, just boot to
freesco, run move2hdd and then you can add router.bat to your autoexec.bat
and it runs wholly from the hard drive...

JM> with 8mb ram, dont touch squid as a proxy. there is tinyproxy, which doesnt 
JM> cache, it only anonymises(sp?) the connection. try not to run ftp, unless it is 
JM> *explicitly* needed.
