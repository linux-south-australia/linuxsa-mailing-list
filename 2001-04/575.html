You could try using tune2fs and dumpe2fs to locate a valid superblock if one
exists. Maybe try a read only badblocks to see if it is in anyway readable.
e2fsck searches for superblocks according to the block size, for 1k blocks
they start at 8193, for 2k its 16384, 4k is 32768 but there is also the

-B blocksize
    Normally,  e2fsck will search for the superblock at
    various different block sizes in an attempt to find
    the  appropriate  block  size.   This search can be
    fooled in some cases.  This option forces e2fsck to
    only  try  locating  the superblock at a particular
    blocksize.  If the superblock is not found,  e2fsck
    will terminate with a fatal error.
    
Wouldn't hold your breath though, clunks are usually terminal........
