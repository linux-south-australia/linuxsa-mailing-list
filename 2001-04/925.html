inetd.conf has been replaced with the un-appropriatly named "xinetd"

xinetd has two configuration sections :

/etc/xinetd.conf
/etc/xinetd.d/*

xinetd is a little more flexible and modular than inetd.conf as it allows
for user based permissions now.
