Re: Getting Slack ... to make a bootdisk ...

At 04:45 PM 4/21/01 +0930, Dan Shearer wrote:
>On Sat, 21 Apr 2001, Richard Sharpe wrote:
>
>> One of my favourite tricks is to copy /boot/vmlinuz onto a floppy with dd.
>> 
>> This will make a bootdisk for many Linux distros ... 
>> 
>> However, Slackware's vmlinuz is almost as bloated as Tru64 UNIX's is, and
>> is over 1MB in size, so this trick does not work.
>
>There are a Slackware boot/root/rescue images in the source tree, maybe
>this would be a better starting point for you.
>
>I reckon the Slackware user profile tends towards the "build your own
>kernel" end of the spectrum. Most Slackware users I know prefer Slackware
>because it doesn't have a package manager, doesn't use PAM, doesn't have
>something like linuxconf or colas. So trying to get Slackware to do
>precisely the same things as one of the other distros out of the box is
>probably a bit futile. There are benefits to this approach, see Rocklinux
>for the ultimate extreme.

Well, these things should work because they are the common legacy of Linux.
Now, as it turns out, the bare.i kernel is small enough that it will work
as I expected, and if you build your own kernel under Slackware, it too can
be small and will boot from a floppy as I expected.

As to building your own kernel, I have been doing that for years under
RedHat and Mandrake ... and the flexibility offerent by PAM for doing
real-world things is very useful :-)

Sounds like the Slackware users you profile would be happier with ULTRIX :-)
