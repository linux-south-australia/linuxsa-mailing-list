On 7 Apr 2001, Andrew Reid wrote:

> They do. It's called 'OpenSource'. There would be legal ramifications if
> they chose not to. Admitadly the location of this code can be harder to
> find when glancing over there website, but I assure you it does exist.
> That's one of the main pillars of the OpenSource Development Ethic. You

I've had trouble locating the source tarballs of the Ximian-modified GNOME
packages.  The download page on their website points to
ftp.ximian.com/pub/ximian.  There is a 'sources' directory there, but it
only contains tarballs of the helix-install program.  All the other
directories lead to binary packages for various distributions.

So, as far as I can see, generic source tarballs of the core GNOME
packages (a la Ximian) are non-existant.  Sure, there are SRPMS available,
but the format is rather awkward to work with if one is not on an
RPM-based system.  Note that I am not whinging at Ximian, I am quite sure
they are doing a splendid job in producing a polished product for the
number of systems they already do support.

But I'm sure source tarballs would be much appreciated by many.
