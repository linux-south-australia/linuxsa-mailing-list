Re: Sendmail weirdness

Jake Hawkes wrote:

> Tim Aslat wrote:

>> I'm guessing here, but could somone be trying to use my mail server as a spam
>> relay, or is this some mis-configuration on my part?

Mis-configuration on your part, but in a different way.

> Daryl, any thoughts?

Mostly of a lewd nature.

But for this problem, it's using something other than the "Approved
Method" of modifying sendmail.cf files - something like linuxconf
or webmin.  Something has configured the receiving sendmail (tux1)
to be non-RFC compliant and reject empty SMTP From messages in the
stupid belief that it reduces spam (as someone else once put it,
so does rejecting any email that contains the letter "e" - it breaks
legitimate as well as illegitimate traffic), which is the mechanism
by which bounce messages are delivered.  The easiest solution is to
configure Sendmail using the m4 files supplied with it (sendmail.mc,
et. al.).

I believe this is a Sendmail FAQ entry.
