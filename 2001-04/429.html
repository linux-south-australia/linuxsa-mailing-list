Re: Routing Problem.

>Hello Group. I would like to do something that I personally think is
>impossable. I would like to route traffic through 1 ip like this.
<snip>
>Say I wanted to hit with dynamic dns www.mookey.mine.nu to box 3
>And also with dynamic dns www.starlightchildcare.mine.nu to box 2.
>
>Could I some how route the information for the domain
>www.starlightchildcare.mine.nu to box 2  say 192.168.1.2and
>www.mookey.mine.nu to box 3 say 192.168.1.3

Presuming that you're talking about web sites, yes, this is possible if you
expect all clients accessing the service to be using HTTP/1.1 compliant
browsers. This includes every browser released since Netscape 2.0

You could run a http proxy server on the gateway, and proxy to the different
sites based on the host: header.

If this isn't possible, you could set one site up as port 80 (redirected to
box 2) on the gateway, and the other on port 8080 of your internet gateway.
Then when the www.starlightchildcare.mine.nu system receives a request for
http://www.mookey.mine.nu/$1 you can use mod_rewrite in apache to redirect
to http://www.mookey.mine.nu:8080/$1
