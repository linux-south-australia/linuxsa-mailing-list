Bigpond ADSL problems fixed: summary

Hi people,

The Telstra Bigpond ADSL difficulty I was having has been fixed, thanks to
some web site reading courtesy of Milton Yates. The reference is:

http://eltoday.com/article.php3?ltsn=2001-04-17-001-14-PS

Basically, there's a TCP option available with the 2.4.x kernels called ECN,
which most routers grok but some don't ... obviously the router or whatever
in front of the telstra machines I was trying to get to was throwing away
any packets with ECN on. So, all I had to do to fix the problem is switch
it off in the kernel config, or touch a file in /proc:

echo "0" > /proc/sys/net/ipv4/tcp_ecn

And it's fixed! Voila:

[justin@valhalla justin]$ telnet pop-server.vic.bigpond.net.au pop3
Trying 24.192.1.30...
Connected to m15.bigpond.net.au.
Escape character is '^]'.
+OK m15.bigpond.net.au POP3 server (Netscape Messaging Server - Version 3.6)
ready Thu, 19 Apr 2001 12:31:05 +1000

This is what most of us thought - the problem didn't seem to be routing or
path mtu, since everything else worked.

Many thanks to Milton and to everyone else who offered assistance. :)
