Re: New hdd installation

At 05:07 PM 4/15/01 +0930, Jeremy Ervine wrote:
>Hi everyone,
>
>I have the wonderful task today of installing a new 40gb hdd into our server
>along side the other 30gb.
>
>We have our first 40gb drive partition as 33gb as a data partition mounted
>as /data, and the rest is for system, swap and proxy.
>
>What I would ideally like to do is put in the new 40gb drive and allow for
>it to give /data another 40gb space so we don't have to bother moving data
>all about the place and restructuring the way our server is setup. Does
>anyone know if this is at all possible. It is a feature I use on Novell
>servers all the time and I have never found a way of doing it with Linux
>yet.

What you need is LVM or MD. Then you can create a larger volume than the
one drive, and extend the filesystem on-line, and away you go.  LVM is
probably the more flexible here.

Of course, now that you have a normal EXT2 file system on /dev/hda?, you
will have to do a little work ... 

Of course, your reliability goes down, because if you loose either
parition, you have lost the whole volume.
