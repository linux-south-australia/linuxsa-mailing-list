Re: X Question

> > If I run an x-client on a FreeBSD machine (eg xterm) and set its display
> > to my x-server on my Linux/Solaris/HP-UX machine, it should work
> > shouldn't it?
>
> Yes, of course.

You also need to allow you local. X server to allow connections from the
remote X client machine, otherwise it will refuse the client connection. It
also depends on how you are logging into the remote X client machine. If you
do it via [open]ssh then you would make sure X forwarding is enabled, and
set the DISPLAY variable on the remote X client to be localhost:10.0 for the
first connection, etc. I believe that if you do allow X forwarding, that the
DISPLAY variable is set automatically.

If you are connecting to the remote X client via telnet, or directly in on
the console, you might want to manually export something like
'DISPLAY=x.server.com:10.0' and then on the X server machine play with
'xauth'.

> > independent protocol?
>
> Well, the platform is X :-)
