Re: Why use DocBook versus Word

>>>>> "DSL" == David Lloyd <lloy0076@rebel.net.au> writes:

  >> Personally I think Lyx is by far the best tool to use for large
  >> documents bar none.

  DSL> If you are writing from scratch, LyX can be quite reasonable,
  DSL> however the versions I've seen don't support DocBook. In
  DSL> addition, if the document already pre-exists I'm not sure how
  DSL> well LyX handles an import. 

At home, I currently use LyX 1.1.5.  Under Help->Extended Features
there is a chapter (4.1) which discusses using LyX with SGML-Tools.
Chapter 4.4 discusses using LyX with documents under version control.

I'm not saying LyX is the perfect solution for this particular
problem, but perhaps Richard Sharpe could get some ideas from there?

<opinion> 
  YourAmigo uses LyX for our internal documentation.  We would not use
  M$ Word, no matter how stable someone promised it was.
</opinion>
