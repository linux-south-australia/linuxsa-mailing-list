Re: "root" vs. "su"

> When I log in as user then "su" to root, I get "bash <command>: command
not
> found", however logged in as "root" all works as expected.

Are you using 'su' or 'su -' ?

The first command only gives you the shell of the user you su to, it does
not set up all the environment variables, such as the path statement, which
is what your problem in this case it.

Try using 'su -' instead.
