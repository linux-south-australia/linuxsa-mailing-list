Re: Does anyone know how to get StarOffice to display EPSes ...

Richard Sharpe wrote:
> 
> I have been trying to use StarOffice 5.2 for presentations.
> 
> It looks fine, but cannot deal with EPSes, except perhaps on printout.

That's right, StarOffice doesn't contain a PostScript interpreter.

Only real option is to use GhostScript to convert the image
to a 72dpi bitmap.  GhostScript 7.0 has a lot better anti-aliasing
then the current GNU GhostScript, so it might be worthwhile.

                ----------------------------------

A plan B would be to define a PostScript printer with a 4:3 paper size
in StarOffice and print the file to that.  Then convert the file
to PDF (again GS7.0 is better) and use acroread of a recent xpdf
to do the presentation.

You'll need to set up GS and xpdf to use the fonts available to
StarOffice (ie: the superset of X and StarOffice).  Usual Unix
font nightmare here.

I use the plan B for presentations with the standard Acrobat font set,
but mainly because some sort of PDF reader is the only software you
can reliabily anticipate being present in uni lecture theatres
(MS PowerPoint might be there as well, but PPT presentations aren't
backwardly compatible so you can still be caught out badly).  The
result is probably too stark for a corporate non-technical
presentation.
