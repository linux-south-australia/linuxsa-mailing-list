Apache 2.0.16 => woe+travail

Just in case anyone out there was thinking (as I did)
that it would be a good lark to get the latest
Apache source version and whip it up into a nice
spanking new server.

Well, if you value your remaining hair, for get it.
Here's my account of woe+travail.
http://www.topology.org/linux/apache.html

I finally gave up on Apache 2.0.16 and whipped together
version 1.3.19 instead.

The latest Apache version (admittedly a beta) is incompatible
with PHP (the latest 4.0.4pl1 version), and in fact seems to 
require every one of the 300-400 Apache modules to be
rewritten in a quite non-trivial way.
The worst thing is the almost complete absence of
documentation, which required me to practically read the
entire Apache 2.0.16 source. It's quite a monster.

Since I've just wasted about 80 hours on this,
I thought I'd try to save the time of anyone else
who thought they might like to try it.

On the other hand, if anyone out there _can_ get
Apache 2.0.16 and PHP 4.0.4pl1 to work together,
I'd gladly know about it.

Cheerio,
Alan Kennington.
