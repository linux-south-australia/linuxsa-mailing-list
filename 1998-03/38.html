This will probably turn out to be one of those stupid questions again,

I have a working ppp dialin script, but there's a minor snag, it won't
redial.

this is a copy of the ppp-on-dialler file (yes I'm using slackware and
an x dialler interface won't work for what I need)

#!/bin/sh
#
# This is part 2 of the ppp-on script. It will perform the connection
# protocol for the desired connection.
#
exec /usr/sbin/chat -v                                  \
        TIMEOUT         3                               \
        ABORT           '\nBUSY\r'                      \
        ABORT           '\nNO ANSWER\r'                 \
        ABORT           '\nRINGING\r\n\r\nRINGING\r'    \
        ''              \rAT                            \
        'OK-+++\c-OK'   ATM1L1H0                        \
        TIMEOUT         30                              \
        OK              ATDT$TELEPHONE                  \
        CONNECT         ''                              \


the ppp-on script consists of this entry with the appropriate variable
definitions at the start of the file

exec /usr/sbin/pppd debug lock modem crtscts /dev/modem
115200            \
        user $LOGINNAME remotename $REMOTENAME
\
        noipdefault netmask $NETMASK defaultroute connect $DIALER_SCRIPT



if anyone can give me a solution to enable the script to redial on no
carrier and busy flags I would be much appreciative.
