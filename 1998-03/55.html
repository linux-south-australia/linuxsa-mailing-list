setting up a VPN for an NT network using linux...

> Hi all,
> 
> I have a need to allow Windows 95 clients to log into an NT File
> server using VPN and PPTP over the internet.... I want to do this
> using our linux box which is currently acting as our internet
> gateway/firewall... is this possible?
> 
> Detail:
> 
> We have an NT box, which is our main file-server and logon server
> within our building... people sit down at their windows 95 computers
> in the morning, and log into this server in order to have access to
> their home drives and shared client directories... this is all done
> using straightfordard Windows NT networking.  However, we also have a
> linux box which is acting as our gateway to the internet... this is
> primarily used as a proxy server for providing desktop web-browsing
> services and ftp services to each of the windows 95 desktops with only
> the linux proxy having a valid internet ip address (internal ip's are
> made-up).  However what we want to do now, is to allow for people to
> log into our NT server over the internet, via the linux internet
> gateway, so that it would be *as though* they had logged in via
> Windows NT RAS or from within our building itself, and therefore were
> able to print to the printers on our intranet, or access their files
> on the file-server... as I understand it, it could be done if we
> replaced the linux box with NT Server running Ms-Proxy server and then
> ran Routing and Remote Access Server on this machine... this would
> allow win95 clients to log in as though they were making a RAS
> connection except that it would be using an IP number rather than a
> telephone number... however, I want to avoid using NT if I can, so is
> there anyway that this can be done using the linux proxy?  Is there a
> linux product that could be used to allow people to log into the
> intranet over the internet and be active as though they had logged in
> on a local machine on the intranet?
> 
> The reason that we want to do this *rather* than simply using RAS to
> dial-in to the NT server using a normal telephone line is that often
> the people dialing in will be doing so from interstate or overseas,
> which makes the telephone connection hugely expensive... we want to
> avoid this if possible, and thus utilise the internet as the medium
> for connecting to our network...
> 
> Anyone have any ideas?
> 
> thanks and regards
> 
> David Buddrige
> Network Administrator
> Optum Group Pty. Ltd.
