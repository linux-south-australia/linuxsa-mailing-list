Hi David,

Presuming you have an IDE hard disk, try putting in the boot disk, and
typing at the boot prompt:

linux single root=/dev/hdXX initrd=

(where hdXX is whatever your root partition is)

Once it boots, you can then edit /etc/lilo.conf and rerun /sbin/lilo.

> when I boot, it comes up with "LI" but goes no furtehr - obviously
> failing in an attempt to run lilo... anyone got any ideas?

This is a LILO error that means:

LI  The first stage boot loader was able to load the second stage boot
    loader, but has failed to execute it. This can either be caused by a
    geometry mismatch or by moving /boot/boot.b without running the map
    installer.

Not that that is very helpful, but maybe you were interested :).
