Re: ifconfig

> OK, I've done a bit of IP-Aliasing, but it's all a bit crude. I get things
> going by sticking things like this in /etc/rc.d/rc.local, but I'm sure
> there's an 'official' way of doing it. I'm using redhat (SysV).
> 
>      #setting up IP alias interfaces
>      /sbin/ifconfig eth0:0 203.33.18.193
>      /sbin/ifconfig eth0:1 203.33.18.194
>      #setting up the routes
>      /sbin/route add -host 203.33.18.193 eth0:0
>      /sbin/route add -host 203.33.18.194 eth0:1
> 
> How do I do this 'officially'??

You can of course use the netcfg tools to do it, but here's what the
config files look like if you want to do it manually:

/etc/sysconfig/network-scripts/ifcfg-eth0 (this is the main device)

DEVICE=eth0
IPADDR=a.b.c.d
NETMASK=255.255.255.0
NETWORK=a.b.c.0
BROADCAST=a.b.c.255
ONBOOT=yes

/etc/sysconfig/network-scripts/ifcfg-eth0:0 (this is the first alias)

DEVICE=eth0:0
IPADDR=a.b.c.e
NETMASK=255.255.255.0
NETWORK=a.b.c.0
BROADCAST=a.b.c.255
ONBOOT=yes

The differences are:
Add :0 to the filename (the next ones are :1, :2, etc),
Add :0 to the DEVICE= line,
Change the IPADDR= line.
