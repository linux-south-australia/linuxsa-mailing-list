Re: Who sed what?

Mark Pulford wrote:

 > I'm no sed freak (far from it) but the following should do what you want
 > (bash+sed):
 > 
 > for I in *; do sed -e 's/\/dir\/\///g' < $I > TEMPFILE;mv TMPFILE $I;done

Ugly!  You are, of course, aware that sed doesn't actually think that "/"
is a metacharacter?

The "s" command takes a search pattern, replacement pattern and an
optional set of flags separated by some delimiter character.  Most
people use "/" as a delimiter, but there's nothing to say that you
*must*.  sed will use whatever character follows the "s" command.

So to rewrite the expression above for slightly better readability:

     sed -e 's:/dir//::g'
