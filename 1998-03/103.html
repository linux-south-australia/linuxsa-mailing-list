On a slightly related note, it's not that difficult to get 
four channels of analogue input by wiring stuff up to 
the joystick port.  

http://www.hal9000.net.au/cgi-bin/temperature features an 
application of it:  The readings the page delivers are obtained
by reading the X1 and Y1 channels from the joystick port, which
has been wired to a pair of 200 Kohm thermistors.  No additional
circuitry required.  Software consists of a three-line perl script
which reads /dev/joy then applies a linear interpolation function
against a set of calibration data that was gathered one bright and
sunny afternoon a few weeks ago (the readings are probably very
approximate:  I doubt the thermisters are perfectly linear, and
only two calibration points were used.  "temp1" is ambient room
temperature, "temp2" is the temperature taken from a point adjacent
to the outlet grille on the room's air conditioner.  If they're close
to each other the air conditioner is probably turned off).

Note that in this application X2 and Y2 are still available as
additional channels.
