The thing i like about clone() (well from what i've read) is that the
forked process has the same data segment as the parent. The processes can
maintain their own synchronousy without the overhead of external ipc, and
i need process communications to be fast and almost instant. 

The only other way is sysv ipc memory sharing which is extremely messy,
BSD socketeering was is slow, pipes which are slow and only one way, and
streams? which i haven't used before but are probably very slow too. 

I could be wrong, but as i'm aware, clone() is the only system call that
can do this?.
