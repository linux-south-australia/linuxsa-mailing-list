Re: 2 Unrelated questions re networking.

> Question 1.
> 
> I nfs exported a directory to another machine.  This was
> achieved by editing /etc/exports then running
> /usr/sbin/exportfs -a.  I then easily mounted the directory
> on the other machine.
> 
> How do I stop exporting the directory once finished?  On other
> UNIX machines, I can call /usr/sbin/exportfs -u <the directory>
> to stop exporting the dir, but under Linux, exportfs -u does
> not work.

Under Linux, exportfs only gets the nfs server to reread its config
file.  The -a option is just supported for backward compatibility.  To
stop exporting a directory, remove it from /etc/exports and run
exportfs again.

> Question 2.
> 
> [...]
> 
> By adding regularly used machines in my /etc/hosts I can obviously
> use the machine name.  It  appears to me that the nameserver
> is not being called.
> 
> In my /etc/resolv.conf i have lines of the following :
> serach <domain name of the isp>
> nameserver <appropriate ip # here>
> 
> search <domain name of work>
> nameserver <appropriate ip # here>

You should be using only one pair of those lines at a time.  If you
have more than one nameserver line, the second is only ever looked at
if the first nameserver doesn't respond.
