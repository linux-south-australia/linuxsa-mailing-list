In the immortal words of <regenerate@hush.com>...
> Apparently system() and other calls don't pay attention to this.
> There are ways to break out of open_basedir

This would have to be used in conjunction with safe_mode, which can be
turned on a per directory basis.  No harm in locking users down as much
as possible.  I would suggest adding
php_admin_value safe_mode 1
to the same virtualhost config as the open_basedir, although I see
nothing in the documentation about it not affecting system(), exec(),
etc.

> Can this be overriden in an .htaccess file.

Nope, it can't.  At least according to the documentation
