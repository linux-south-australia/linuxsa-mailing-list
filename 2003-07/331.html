Re: shell question

Chris Harris wrote:

>If we're talking linux (and I presume we are, 'cause it's Linux SA.
>duh.), then if you can find the shell's pid then /proc/<pid>/cwd will be
>a symbolic link to that user's shell's current working directory.
>
>Does anyone know of a more 'unix'y way to get this, out of interest ?
>  
>

If the user is running an application, then you may be able to use 'ps 
e' to return their environment, ie:

ps aeww | grep ^username | grep PWD

there's still some assembly required, and it doesn't help if a user is 
sitting at the shell prompt or changing their working directory once in 
the application.

It's also worth knowing for the software developer that 'ps e' exists, 
since it can be otherwise tempting to use the environment to pass 
private data like passwords to an exec'd application.
