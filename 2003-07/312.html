> An easier solution exists in this case.  Ryan said that the DNS servers
> are on static IPs elsewhere.  He could setup a dynamic DNS host record
> with dyndns.org, no-ip.org, et al - say, quack.dyndns.org for eg.  Then
> in the static DNS server that he has access to,  set up a CNAME record
> for the *real* hostname.  Ie:
> 	quack.mydomain.com.	IN CNAME	quack.dyndns.org.
