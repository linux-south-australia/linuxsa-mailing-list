 > > not be distributed under any license except the GPL.

 > Simply not true.  If you come up with a bug fix you can distribute it,
 > say as an english description, or maybe a unified patch with 0 lines of

No -- Read the license;  Section 2(b) is quite explicit.  If I derive
"in whole or part" from a GPL licensed program, my derivation must
be covered by the GPL when it is distributed.

A bugfix derives from the code that contains the bug.

I don't know how you can read that clause in a way that suggests I 
can distribute a non-GPL Linux kernel bugfix.

 > Obviously you can't distribute it in any truly useful form,

I can't distribute it *AT ALL*.

Are you starting to see why the GPL is restrictive for developers?
