RE: Linux knows too much

> One advantage I do now remember is that if you have two passwords that are
> identical, you will still end up with different hashes... Hence, you can, if
> you want, use the same password for two users, but people casually browsing
> the shadow file will never know that. Still, I have a feeling that there is
> more to it than that. Perhaps the salt is used to pad out the plaintext
> password, rather than using 0's to pad it with... Hmmm... Not sure.

That's what the salt does - make the hash different for the same password.

I think you guys are over thinking the problem.  

Assume I'm a normal user .. everytime I change my password I have to enter 
my old one.  So passwd always have the previous old password in clear text.

All you need to do is encrypt the old plaintext password with the new 
password and store that as well.  If you do this and store the OLD hash and 
salt then you can recurse through the list back to the oldest password 
you've kept.  Risk is low if you think about it.
