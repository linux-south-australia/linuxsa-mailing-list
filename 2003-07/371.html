Re: Web-based SSH client anyone?

why not just run SSH on port 443?

or if you need to get through a proxy, you could use a funky program called
bouncer (I forget the URL), that you can run on your machine you configure
it to connect via the proxy on port 443 then specify a local port, then ssh
to your localhost on the specified port and bouncer tunnels the connection
across HTTPS.

Your end SSHD needs to be running on port 443 for this to work though. I
know it works, I've done it before :)
