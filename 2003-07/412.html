It's not unique to London either:

$ host -t ns www.microsoft.com
www.microsoft.com is an alias for www.microsoft.akadns.net.

$ whois akadns.net@whois.opensrs.net
[whois.opensrs.net]
Registrant:
 Akamai Technologies, Inc.
 8 Cambridge Center
 Cambridge, MA 02139
 US


But wait, there's more:

$ host download.microsoft.com
download.microsoft.com is an alias for dl.ms.georedirector.akadns.net.
dl.ms.georedirector.akadns.net is an alias for 
download.microsoft.com.d4p.net.
download.microsoft.com.d4p.net is an alias for a767.ms.akamai.net.

Don't be too smug about Akamai using Linux either, they have deals with 
Microsoft to support Windows Media Services and distributed .NET 
services. It would be surprising to see MS release Linux versions of 
such software just because it is Akamai's current platform of choice. 
Conversely, I imagine it's more than challenging to install Windows on 
13,000 Linux servers remotely..
