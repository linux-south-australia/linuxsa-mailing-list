Using a printer connected to a parallel port in IBM Thinkpad 600X/ SuSE Linux 8.2/HP Laserjet 4 

I am out of town using my laptop but borrowing a printer that has a 
parallel port connection, but I can't get it to set up or print. I have 
done my work so far in Linux, but have had to reboot into Windows 2000 
to print.

Working with it yesterday, I gathered some information:

* I use Kernel 2.4.20

* I have the parport, parport_pc and lp modules loaded

* Windows 2000 says that the parallel device is an Intel 82371AB/EB PCI 
to ISA bridge (EIOM?)

* I have tpctrl installed, but when I run  "tpctl --rp"

I get:

resource state:    ioaddr        irq#           able?         mode/power
parallel port:      0x3bc         IRQ0         disable       comp

It looks disabled. But when I go into Windows 2000, and access the 
Thinkpad Controls, it says it is enabled.

Windows 2000 also tells me that the device takes up these addresses:

03BC-03BF.

But when I try to run  tpctl --rp 0x3bc IRQ0 enable comp

I get:

tpctl: Error scanning options.  Try 'tpctl --help'.

tpctl --help reads: 
[--rp|--resource-parallel[=(<port>|LPT<#>|IRQ<#>|enable|disable|(COMP|EXT|EPP|ECP))]]

Seems like I did that ?

I also tried running: setpnp 0d io 0x3bc-0x3bf irq0

I saw that after searching google,  but I just get this:

usage: setpnp [-b] [device #] [resources ...]
    or setpnp [-b] [device #] {on|off}


SuSE also allows you to pass parameters to the parallel printer in the 
control center and I have entered this under parralel port details:

0x3BC,Ox3BF

Here is what they say about the wizzard:

"Parallel port IO:
If you have an uncommon parallel port card, it may not be possible to 
automatically detect it. In such cases, you must specify its IO 
addresses. Enter IO addresses separated by commas for all parallel ports 
here, from the first to the last. Example: 0x378,0x2f8
If your parallel port works, leave this field untouched.
Changes will be saved immediately after you press OK."


Any ideas?
