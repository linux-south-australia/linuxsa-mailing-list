The Gtk+ and QT toolkit are both themable. I know that you can override
the Gtk+ user themes if you try (hard) but the reason why the themes are
set by the user are good reasons.

I think that Gtk+ 2.0 and QT 3.0 are both comparable beasts. QT is
almost exclusively written using C++ style programming while Gtk+
typically is written in C with a host of other not-so-official but well
documented interfaces.

My advice: pick one, learn its intricacies and hope the sharks don't get
you ;-P
