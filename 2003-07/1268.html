> > Dave,

> > > > Important tips of the day:
> > > >
> > > > If you have a Promise TX2 ide controller, and you want to install 
> > > > SuSE 8.1 on a drive attached to it, I found the following steps 
> > > > useful:
> > > >
> > > > 1. Download the driver disk from the Promise web site 
> > > > (www.promise.com)
> > > >
> > > > 2. Configure your BIOS to use PnP for the IRQ assignments, DO NOT 
> > > > use manual IRQ assignments.
> > > >
> > > > 3. Use Grub, lilo had a cow.
> > > >
> > > > 4. On an SMP system, you must specify "noapic" as a 
> > kernel argument 
> > > > or you will get "ide_dma_lostirq" messages and the system 
> > won't boot 
> > > > properly.
> > 
> > The problem, as far as I can see, is the actual BIOS on the 
> > card itself won't let me boot off a device unless an array's created.
> > 
> > If I've got a RAID Array defined, it boots off it fine, but 
> > if I haven't (because I'm using software raid), no go.
> > 
> > It's not a Linux problem specifically; doesn't matter which 
> > OS I have on it, or if I reinstall the MBR from a rescue CD.
