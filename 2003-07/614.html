we have three development servers that were built about
three years ago. they are running redhat 6.x.

we have not upraged any software, save security fixes
in that time.

now, we need to upgrade the hardware. we will move the 
uw-scsi disks into a new server with faster cpus, more
memory and add more disks. and power it up and keep going.

as these machines are only compile servers, this really isn't
a problem. even our other servers run samba, web server, mail,
ldap are in the same arena. 

the features on these services dont really change that often 
and appart from the fact that we are hitting the end of life
of the hardware doent mean we need to upgrade the software...

we are not a common business model, but our actual software 
requirements haven't really changed in four years. we do
specialised/embedded software development... but we still are
on the pace.

there is a point that business will need to update their
software and hardware. but more often than not, most don't
really need to, if it works in the first place.
