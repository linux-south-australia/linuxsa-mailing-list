Re: Wierd /usr/bin/passwd on RH7.3

Ryan Verner wrote:

>$ cat /usr/bin/passwd
>donga --stdin mdk
>
>('donga' does not exist at all on the system).
>
>.... anybody got *any* clue what the hell donga is?  Is this a normal RH
>thing?  G
>

--sdtin is part of the standard passwd binary.

My guess is that someone changed the existing passwd binary to 'donga' 
and then wrote a shell script to change the password only for a user 
called 'mdk'. Somehow this has ended up on the system you're in front of.

Does that user exist on the system? Perhaps ~mdk will provide some 
clues, or maybe that particular user had a history that necessitated 
some ugly hack? More than likely there's a program somewhere that calls 
passwd and expects to change it for root when it shouldn't. Maybe the 
logs have some security info that shows a password change occuring for 
this user?
