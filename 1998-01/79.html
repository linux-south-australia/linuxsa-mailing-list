The operation not permitted means you are running this as a user.

pppd under RH5 is (correctly) not setuid.

The prefered method is to use netcfg to connect, very easy.
From there you can become root and run netcfg and Enable the ppp interface
or alternatively, make it user startable and run usernet.  

Both these are X based programs, but you should be able to track down
what is called and work out a command line equivalent.

If you are going to use usernet check out the redhat errata for RH5 
on http://www.redhat.com.

Cheers.

Matthew 'Verrry Happy with RH5 ppp' Tippett

PS apparently RH are commiting to (more lightweight) console tools
so that X isn't a requirement.
