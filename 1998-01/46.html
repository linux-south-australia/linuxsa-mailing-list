Re: Laptop install

Paul,
	The fact that the eth0 interface is working suggests that the network card
is working OK.
I would suggest that you make sure you have the IP set up correctly for
this machine. ie. is it in the same sub-net as the host machine??  The eth0
interface suggests a mask of 255.255.255.0.  Is that the same as the host
machine??  If so then make sure the host machine is also 10.0.2.x and that
it's subnet mask is also 255.255.255.0.   If that is OK then PING is the
way to go.
If PING returns 100% packet loss then I'd check for bad cables etc.   I
assume it's 10BaseT (Twisted Pair cable) as 10Base2 is very rare for PCMCIA
cards?  If so check that the cables are wired correctly and that the hub is
working fine.  Don't use no 1 port on the hub if you can avoid it as it's
often used to link 2 hubs together and can sometimes be a problem.  If you
aren't using a hub then make sure you have a crossover cable between the
two machines, not a normal UTP patch lead.
Only 2 pairs are used..1&2 and 3 & 6, so a crossover cable is 1 -> 3 and 2
-> 6.
If you ARE using 10Base2 (co-axial cable) then check that the cable is ok
and that you have 50 ohm terminators on the T pieces attached to each
network card.  If there are more than 2 machines in the network then only
have the terminators at each end of the chain of course.
Hope this twigs the brain for you!
regards...Don
