Re: Auto mailing interface IP address.

On Wed, 28 Jan 1998, Don Collins wrote:

> I wish to be able to remotely administer the machine (using ssh) but
> because it is a dial up machine I can never know what the IP is.
> 
> I have tried adding the following to /etc/ppp/ip-up.local, but I only ever
> get a blank page mailed to me.
> 
> 	ifconfig | mail -s "Interface configurations" doncol@mail.mdt.net.au
> If I run that manually it works fine...but I want it to run automatically
> every time the interface comes up.

I don't know where the file might be located under RedHat (I'm used to
Debian) but look for the 'ip-up' file on your system. On my box it's
/etc/ppp/ip-up

It a script run when PPP is established and it already has access to
the local and remote IP address, so you can add lines similar to:

echo -e "Customer IP is $4\nRemote IP is $5" \
	| mail -s "Interface configurations" doncol@mail.mdt.net.au



Hope this helps,
