Re: Web / IMAP gateway

Dan Shearer wrote:
> 
> > Yes. IMP. Check out http://www.horde.org/ they have other stuff too.
> 
> This is fine until you need high performance. Then you need to use
> something like http://www.lysator.liu.se/~stewa/IMHO

I installed IMP, as I needed something pretty quickly.

It appears to be well-implemented with an excellent user interface.

It requires cookies and JavaScript.  This is a tad disappointing
as the browser features are used for nothing that you can't do with
standard HTML 3.2.

It is noticably slow, even when running on the same machine as the
IMAP server, so I'd be concerned about using it in a campus-wide
situation.  I'm assuming this is because of the continual opening
and closing of IMAP -- this has a large cost, especially in a
corporate environment where the IMAP process also needs to do
an LDAP query to authenticate the user.

> It also supports cyrus IMAP as well as generic IMAP daemons.

The code supplies differing INBOX names for the two servers.
This looks like an ancient workaround, as using the standard
INBOX name to Cryus works fine.
