AFAIK, Just about all browsers do this (or have an option to do this), as
"Mozilla" is more a legacy term for "image-enabled web browser". Back in the
old days when there was lynx and mozilla, if a browser identified itself as
mozilla it got treated a different way than just "any old hypertext
browser".

I use NetVampire to manager my downloads (better than getright), and it
gives the option of identifying itself as "Mozilla" to improve reliability
of connecting to web and ftp servers. For example, an ftp server might be
listening for an id of "mozilla" before it will allow anonymous
authentication. Some smart-arse with a console interface to ftp might be
restricted from any kind of anonymous login, whereas a "browser" is more
limited, and *perceived as* less of a risk.

I'm rambling. Maybe I'm making some of this up. But I /think/ I'm on the
right track. :)
