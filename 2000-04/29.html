[OT] Windows Telnet TIP

To those of you trying to use Windows Telnet to connect to a Linux box to do
some work here is a little tip:

Do not use the following syntax-

telnet mylinuxbox

It seems to attempt to be using ANSI terminal (you know the option that says
VT100/ANSI), which appears not to work.

Linux realises the attempt to use ANSI and sets the terminal to ANSI, of
course none of the cursor keys work properly (other keys may have the same
problem, got annoyed and didn't check), especially in VI(M). Newer versions
of ELM seem not to work either (the one with RH5.2 works, but the RH6.0
doesn't, it spits at a whole heap of things).

How to make it work:

type telnet by itself.

Hit Alt-C to connect and when the connection dialog shows up, type in the
host name. It will default to the telnet port and the terminal type will be
VT100. Everything that didn't work before, now works.

Hope this help someone out there.
