XWindows Tip

Another Tip, for something I found the other day..

There is probably another way around this that is much easier, or I missed
something...

If you install Gnome and KDE onto the same machine, you may come across a
problem when attempting to run X apps from an xterm.

Gnome, seems to set the environment variable DISPLAY to :0 which is fine and
works like a charm, whereas KDE uses hostname.somedomain.com:0 as the value
for DISPLAY. This is the problem. If in KDE, you run an xterm and type say
xlock, you will get a permissions error.

The quickest fix, go into Gnome (with the nice DISPLAY value) and type xhost
+hostname.somedomain.com (the + is optional). Go back into KDE and you can
now type xlock inside an xterm.

Did I miss something, somewhere ? Not sure, having looked at the man pages,
this is all I could work out.

Note XDM (read for gdm or kdm as well) is a biiiiig man page. xauth, xhost,
etc are much nicer to look at...

Hope this helps someone out there..
