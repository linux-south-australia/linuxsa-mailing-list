Font Server Not Serving!

Good Morning!!

I'm only a Linux Rookie and would really appreciate any help on the
following problem. The X font server does not appear to be accepting
requests on a server running RedHat 6.0. The font server is running, I have
stopped and restarted it. However, querying it with "fsinfo -server
servername:7100" returns the following error message:

_FSTransSocketINETConnect: Can't connect: errno = 111

This is the same message you actually get if you query the font server on
any port (ie a port that doesn't exist). I added "port 7100" into the config
file to ensure it was actually listening on this port.

When comparing to how the font server executes on a DEC box I noticed that
the socket IT creates in /tmp/.font-unix is fs7100 whereas the socket
created by linux is /tmp/.font-unix/fs-1.

Any help would be most appreciated.
