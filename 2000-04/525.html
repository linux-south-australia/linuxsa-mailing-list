ISDN can't achieve 128k (SOLVED)

Hi,

Recently I posted to the list with a problem relating to achieving a 128k
ISDN connection.  well, the problem has been solved...  When I checked
/var/log/messages I noticed that the HiSax driver required for ISDN wasn't
always loaded on the first attempt.  The ISDN4Linux FAQ
(http://www.isdn4linux.de/faq/) had further info on the error I was
receiving, and indicated it was due to a lack of unfragmented memory.  The
machine in question only had 16Mb at the time, and was running the default
Red Hat kernel, which I assume is probably a bit more memory hungry.  The
FAQ suggests recompiling the kernel to reduce memory usage, but being a
relative Linux Newbie, I chose to stick some more ram in the machine
instead.  Upgrading the machine to 64Mb seems to have fixed the problem now.

Thought people would like to know :)

Jim.
