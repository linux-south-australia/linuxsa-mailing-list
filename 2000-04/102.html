APMD

Hi All,

feel like confusing yourself (potentially)?  I just did an install of 
Redhat 6.2 and was wondering why the stupid server was rebooting all the 
time for no apparent reason.  It appears that the reason is that the stupid 
installer (me) had the apmd running (default install even when custom 
server was selected) and it was causing the desktop machine to reboot.  I 
will know in a day or so if I am right but I suspect I am looking at the logs.

A tip for new Linux users.  Don't start the APMD at boot time unless you 
need to.
