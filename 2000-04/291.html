Well, Samba only uses the homes share if you define it, and the main thing
I would remove from below is the path statement.

>MWP
>
>---------------------------------------------
>Heya, im trying to get my window machines to have networkdrive access to
>there homedir.  Currently i have this in my smb.conf
> [homes]
>   path = /home/%m
>   comment = Home Directories
>   browseable = no
>   read only = no
>   create mode = 0750
>
>And that shows the share of the machine name, but it prompts password, and
>the password is incorrect. (in any and all cases)

As is hinted above, this is likely to be an encrypted password problem.
Either:

a. Use the registry patch to switch off encrypted passwords, or

b. Implement an smbpasswd file and all that it implies.

client, version of Samba, etc.
