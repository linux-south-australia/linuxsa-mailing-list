Sam,
	with policies, try using the poledit tool (contained in the Admin dir
on your Win95/8 CD) to create a config.pol file, and place it in the
netlogon share.  It should only need to have read permissions.  This is
essentially just a .REG file with some scrambling algorithm applied.

The other thing that could be causing the problem is during the logon
process (if profiles are enabled) the Win95/8 box will try to read its
network profile location (as defined in the user account) to determine
if it is more recent and then should download it.

I have no idea how Samba defines each user's profile directory and/or
home directory.  I would be interested to find out though :)
