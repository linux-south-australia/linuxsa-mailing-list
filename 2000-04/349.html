> Question 2:
> Why do some browsers try to read the file "favicon.ico" in
> the browsed directory and the / directory?

IE feature (introduced around 4? definitely 5).  In their bookmark
file they have tiny icons alongside the site's name.

> But when I try to read http://www.microsoft.com/favicon.ico
> I get:
> 
> "Sorry, there is no microsoft.com Web page matching your request."

There should be one for Microsoft, but I'm not sure where it's
located, and also it may be triggering to the fact that you're
using a non-IE browser (I've come across some pages there before
that certainly behave differently depending on what browser string
you supply).  From memory, IE searches from the basename of the
URL all the way back to the root for the favicon.ico file.  I think
this is detailed in BugTraq when it first came out.

> whereas if I try http://slashdot.org/favicon.ico, I get a rather
> amusing result.

That is cute - I hope it causes severe discomfort.  :-)  BTW,
http://www.altavista.com/favicon.ico will return a result, but
with a Mime type of text/plain.  IE (in its spack wisdom) will
probably see the ".ico" part of the file name and react to that,
whereas Netscape won't be retrieving the image in the first place.
