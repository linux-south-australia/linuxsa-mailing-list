Re: [OT] Windows Telnet TIP

On Mon, Apr 03, 2000 at 09:45:25AM +0930, Robert Wuttke wrote:

 > To those of you trying to use Windows Telnet to connect to a Linux box to do
 > some work here is a little tip:
 > Do not use the following syntax-
 > telnet mylinuxbox
 > It seems to attempt to be using ANSI terminal (you know the option that says
 > VT100/ANSI), which appears not to work.
 
Part of this problem is that Windows is "advertising" itself as an ANSI
terminal in the telnet option negotiation sequence which takes place at
the beginning of a connection, but "ANSI" is a baseline bare-featured
terminal type which doesn't support many of the things we take for granted.

Despite Windows' announcement of itself as an ANSI terminal, it isn't really
ANSI at all -- It's closer to a VT102 (a superset of ANSI).

You'll find that if you set your TERM environment variable to "vt102" 
once you've logged in to your Linux box (and thereby ignore the bogus 
crud Windows is sending at the start of the telnet session), things will
generally work better (including extended attributes and cursor keys).
