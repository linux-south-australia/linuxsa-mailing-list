Re: Telnet

Eric March wrote:
> 
> I have an old 486 that happily sits in the corner and runs my internet
> connection.  I can get email, browse the web etc.
> 
> If I telnet from a sco box to linux it try to connect to the correct IP
> address -- connected to nimrod.ebm.com.au  -- escape character is "^]"
> -- then after about 10 seconds says connection closed by foreign host?
> 
couple of possibilities:

- reverse DNS.
your linux box is trying to resolve the name of your SCO box by reverse
DNS, and this is breaking. If you linux box is set to deny connections
if reverse DNS is not available, then this is a problem


- identd failed
your linux box is trying to find out who is telneting into it from the
SCO box. ident is a mechanism whereby your linux box can see who on the
SCO box is telneting in. If the SCO box isn't set up to do this, and
your linux box is configured to deny connection that fail ident, than
this is a problem


check you linux logs for reasons /var/log/messages and /var/log/secure
(for redhat)
