Re: MD5 and shadow file

* Edward Sawma (e-sawma@nwu.edu) wrote:
> Does anyone have any experience in directly modifying the shadow file with 
> MD5 encryption?  Basically, to make a long story short, I'm trying to write 
> a Perl script to create accounts on my server, and I want to be able to set 
> an initial password for an account.  I've installed the MD5 and 
> Crypt::PasswdMD5 modules, and neither seem to work.  I have discovered that 
> a "$1$" is needed at the beginning of the encrypted string to identify it 
> as MD5.  I think my problem is with the "salt".  I don't really know much 
> about this "salt" concept, or what the "salt" should be in order for the 
> encrypted string to be properly recognized by linux.

The safest way to acheive this would probably be to have your perl script
call useradd(8) or adduser(8). This has the added advantage of making your
script portable to other systems which do not use the same password format
as yours.

As I understand it, writing directly to /etc/passwd or /etc/shadow unless you
know -exactly- what you're doing is a bad idea.
