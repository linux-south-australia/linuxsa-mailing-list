Re: openLDAP

Hi.

I spent some time on OpenLDAP recently, so I can offer the results of my 
investigations!

OpenLDAP seems to be quite stable. I have implemented a company address 
book on it with no stability issues. On a PII300 it is very fast for 
address queries.

The whole schema thing is mind boggling. If you are able to deal with it, 
or do as I did (copy someone elses and work from there) there should be 
minimal problems.

I wasn't able to use PHP on the server, so I used a java app in the end as 
a gui. Have a look at LDAP Browser, http://www.iit.edu/~gawojar/ldap or 
http://www.mcs.anl.gov/~qawor/ldap

Once you get over the schema design, the rest is pretty straightforward. I 
spent some time making sure that I used the tags that MS Outlook expects. 
Rather than rekey all the data, I exported the mail tables and used 
perl/cut/bash/etc to massage it into the relevant ldif format and ran that 
into ldap.

Regards,

Michael
