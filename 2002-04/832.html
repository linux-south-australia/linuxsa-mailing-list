Re: IPOP3D & Slow Downloads

>When I check my mail from my mail server it downloads attachments (and
>emails obviously) at about 1.5mb sec - however when I transfer files to
>and from my box (using samba) it transfers at about 7.5mb/sec - is there
>any reason why I'm not getting near 7.5mb/sec from my mail server?

Have you checked that the fault isn't with the mail client instead of the
server? It might be decoding attachments on-the-fly, which could add enough
latency to slow down the transfer.

Since you mention sendmail, chances are you're using mbox mail files, which
require a lot of disk i/o to manipulate. In addition to copying the relevant
message data to the POP3 daemon, it also has to find the message you want in
the file, and lock it against further access, after checking itself for an
existing lock.

I've had no speed issues with a Maildir-based POP3, quite the contrary. Our
mail server turned out to be so fast that it was causing a fault in some
types of ISDN devices. Windows users were getting a blue-screen only when
they downloaded their mail and the manufacturer in Europe couldn't duplicate
the fault.
