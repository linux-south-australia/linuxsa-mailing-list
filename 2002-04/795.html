Telstra ADSL and Sendmail email

Hi All

I have a friend who I set up a Linux gateway, as part of my testing of this
install I tried to send an email from his windows computer thought to
telstra's email server (mail-hub.bigpond.net.au) the message would not go
thought.

After playing around a long while, I found that it was because the internal
email header did not look as if  the message came from telstra client IP
address/host name. (before someone asks this is not a NAT problem)

If telnet to the host from the Linux box I get a connection which would
allow me to send a message, but if I telnet from any computer on the inside
of the network I get a connection but then the smtp server at telstra send
kicks me off.

I've been trying to get sendmail to report the ADSL address for all the
header but have not been able to convince it work.

Has any one got any idea on how to get this to work ?
