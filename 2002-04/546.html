Re: best Linux/Unix UML program?

Quoting David Lloyd <davidl@tellurian.com.au>:

> Dia doesn't use Gnome if I recall. It is a Gtk+ program. Regardless of 
> which, I have the latest development version running nicely on a RedHat 
> 7.1 box.

GNOME is a desktop, and is also set of libraries.  gtk+, gdk, bonobo, eel, 
libgnome  etc are some of the GNOME libraries.  Sure, you can use some of these 
libraries without the others, but it's worth considering that gtk+ is basically 
developed by the GNOME people.  i.e. it's not worth drawing a distinction 
between them.

Likewise KDE has a set of libraries.  Some KDE programs use only some of the 
libraries.  No difference.

I guess to clarify, you don't need to run GNOME's panel or other desktop app to 
run Dia.  You just need the libraries that Dia uses (glibc doesn't belong to 
either GNOME or KDE :-)  Likewise I can run KWord on my GNOME desktop as it 
doesn't need KDE's panel etc.  You can mix and match and use the best from both 
worlds.  

I'm sure KDE using people wouldn't want to have to give up using the GIMP, even 
though it's not based on Qt.  Likewise I'm sure GNOME using people wouldn't 
want to give up using... never mind :-)
