Re: openLDAP

Ben Williams wrote:
> does anyone have any experience with openLDAP? i'm after opinions on
> it's reliability, ease of use, ease of administration, or any general
> comments. is it the best solution? are there better
> {proprietary|open-source} solutions (that run on linux, of course)?

I had some contact with it about a year ago.  It was a very useful 
product, but the LDAP/X500 schema was atrocious; like many Unix 
technologies, it's not very easy to use and the underlying format is a 
little arcane.  I'd recommend interfacing with it using some GUI product 
or some web technology (you might be able to patch into it using PHP or 
similar) if you can.

> i'm thinking of using it for an intranet project - publishing contact
> details that are entered via a web interface to some db back-end to an
> LDAP directory.

That sounds like a perfect use of it.  My company has an LDAP/X500 
contacts directory that I query using my mail client; it's very useful.

I hope this helps.
