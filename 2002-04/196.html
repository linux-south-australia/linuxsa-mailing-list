Pam_auth

Hi all,

The box in question has been running RH7.1 and pam_auth to authenticate the
squid users without a problem for some time.
Due to a Hdd crash and new Hdd the box is now running RH7.2 and I can only
get the first user to authenticate OK after restarting Squid, all usernames
after the first can't get through. Running pam_auth from the command line
and entering user/passwds at random gives the same result. Restarting Squid
always gives an OK for the first user entered and ERR for every user there
after.
I have tried this on two other RH7.2 boxes and get the same result.

Has anyone had this happen or know how to fix it?

My /etc/pam.d/squid file look like this

#%PAM-1.0
auth              required        /lib/security/pam_stack.so
service=system-auth
auth              required        /lib/security/pam_nologin.so
account         required        /lib/security/pam_stack.so
service=system-auth
password      required        /lib/security/pam_stack.so service=system-auth
session          required        /lib/security/pam_stack.so
service=system-auth

Thanks
