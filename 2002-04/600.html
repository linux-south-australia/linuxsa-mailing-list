Re: mixing strings and variables in c

On Fri, 19 Apr 2002, David Lloyd wrote:

> 
> Jayson:
> 
> > Sorry this is so stupid, but I just cant figure out how to declare
> > variables in c properly.

[deletia]

> But seriously, as a start point:
> 
> int c;
> 
> ...makes an integer called "c".
> 
> char c;
> 
> ...make a character called "c".
> 
> float c;
> 
> ...makes a floating point number called c.
> 
> char *c;
> 
> ...makes a pointer called c. It's like a perl reference but way more 
> dangerous to use.
> 
> A string is just a null terminated array of pointers:

Ummm, no, it is just a null terminated array of characters. A null 
terminated array of pointers is a different object entirely. A very useful 
object, however.

> char c[127];
> 
> ...that is to say an array of charcters with 128 spaces (C starts 
> counting from 0).

Ummm, again no. The 127 says that there are 127 spaces there, which C 
numbers from 0 to 126. If you want 128 spaces (well, characters, actually) 
then you need to say:

char c[128];
