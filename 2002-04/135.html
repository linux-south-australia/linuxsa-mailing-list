Re: Finding out specs of remote FreeBSD machine

On Friday,  5 April 2002 at 18:52:53 +0930, Daryl Tester wrote:
> Greg 'groggy' Lehey wrote:
>
>> The creation timestamp (ls -cl) will give you more useful
>> information than the last modification timestamp.
>
> Doesn't that give you the time the inode was last modified,

No, that's the modification timestamp.

> or does FreeBSD do something different from other Unices?

No.

There are three timestamps:

1.  Modification timestamp.  Notes when the inode was last modified.

2.  Creation timestamp.  Notes when the inode was created.  This can
    be later than the modification timestamp, for example after a 'cp
    -p' or a restore from tape.

3.  Access timestamp.  Notes when the inode was last accessed.  In
    FreeBSD, updates to this timestamp can be suppressed with the
    noatime option on mount, since updating these timestamps can be a
    significant performance issue on a busy file system (think web
    server).
