On Tue, 2002-04-16 at 22:21, Michael Kratz wrote:
> First thing I would do is go into your SSH config file and make sure that
> the option to allow root to logon is set to NO. If it is then to get root
> from SSH you'd have to login as a normal user and "su" to root.
> 
> If you don't use SSH, stop its service, or uninstall it.
> As a matter of fact make sure you have all the services you dont need,
> stopped, and default deny all in hosts.deny, that will stop them even
> getting a prompt, only ip addresses that you then specify in hosts.allow
> will let them in. (This can also be done in XINETD depending on which distro
> you are using)
> 
> So far, with a policy of deny all for everything and only allow what I want
> from where I want, I have not had anyone break into any of my boxes so far,
> and I run several, permanently connected boxes. I also keep all my packages
> and programs up to date.
> 
> Hope this helps,
> 
> Michael
