Re: db2setup

On Wed, 2002-04-24 at 23:28, Daryl Pilkington wrote:
> Now, I mounted the CD-ROM & changed to the directory where the bash
> script db2setup is & typed:
> db2setup

When a command is typed without a path, your shell searches every
directory in $PATH for an executable file called "db2setup".  The error
message you have received indicates that there is no such file.

The 2 most likely causes are:
1. you do not have your current directory in your path
2. the script is not executable

Depending on which problem you're having, try the appropriate option:
1. specify a path with the script (such as "./db2setup")
2. run "sh db2setup"
   (you could usually run "chmod u+x" to make it directly executable,
    but this may not be possible because a CD is read-only)

I hope this helps.
