Redhat 7.2 display and mouse conflict

Hi All,

I have just completed a Redhat 7.2 Workstation installation. 

I have installed Linux on a Compaq Deskpro 233 MMX Pentium with 96MB RAM. Both the keyboard and the mouse (2 button) use a PS/2 connection on to the motherboard. The graphics card is a S3 ViRGE GX with 2MB on board. 

During installation Anaconda has correctly setup the screen and the mouse behaves correctly throughout. On reboot after installation I get a mouse pointer and next to it a bar code. If I move the mouse on the desktop all hell breaks loose and I get a snail trail behind the mouse. If I use a scroll bar the window being scrolled is corrupted such that the information in it can not be read. If I click in the corrupted window and use the keyboard to move up and down through the text a line at a time is cleared.

Following this I used the Option "sw_cursor" under the device entry for the S3 ViRGE which had the effect of changing the bar code to a solid box under the mouse pointer. With the same behaviour as above in corrupting the desktop and windows.

Any ideas, I am a newbie in this arena so any help would be appreciated.
