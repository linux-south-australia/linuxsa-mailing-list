Re: ipchains logging help

I have the same problem.

I just add an extra rule BEFORE the final reject all which rejects that 
specific kind of packet:

eg. to not log people scanning for Windows filesharing: 

ipchains -A input -p tcp -d 0/0 137 -j DENY
ipchains -A input -s 0/0 -d 0/0 -p tcp -l -y -j REJECT

I tend to use DENY rather than REJECT as DENY just drops the packet and 
doesn't send an ICMP message back.  This can make it harder to scan a 
network.
