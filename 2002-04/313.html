Re: Cron Tripwire report from /etc/cron.daily redhat 7.2

Hi There,
    Try 'man tripwire' and do a search for where it is
installed and look for the doc file. When I set it up it
told you how to do it there, but it has been so long since I
did it that I have forgotten.

    If your file is getting huge then maybe you are scanning
too many files. Is it necessary to scan all files, or just
the important ones? The idea is that if someone hacks you,
then usually they want root access, and will then change
some files that you would regularly scan and so you would
find them out. Maybe you just need to trim the scan list and
you will be O.K. If they do get in, then often it is through
a security loophole that hasn't been updated or patched and
usually a good SysAdmin will be scanning for these at
regular Security sites, and Tripwire won't pick up these
bugs anyhow. Sorry to basically say RTFM, but the docs were
fairly straight forward from memory and you should fix your
problem pretty quickly.
    Cheers!
        Andrew.
