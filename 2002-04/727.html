RE: SMTP and Filtering

Shane Warner wrote:

> Using the Sendmail, is there some way to filter what email is 
> delivered to the server.

Yes, there are several good applications that bolt directly
onto Milter, which is a Sendmail extension to allow third
party apps to listen in on the SMTP conversation.

> For example, I would like to have spam emails bounced so that the 
> sending server thinks the account(s) do not exist any longer and 
> (hopefully) stop sending emails.

That's unlikely to work - spammers tend to use open relay
servers, deliver email in bulk with bogus return addresses
and don't care about undeliverable email.
