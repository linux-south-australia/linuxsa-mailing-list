Decades ago I was working on a Tandem machine with a rather primitive
shell.  When it encountered an error, it would just come out with the
number, so you needed to learn that ENOENT was "FILE SYSTEM ERROR
011".  People didn't like that, so I hacked the shell binary to call a
function which loaded an error message text from a file and printed
that instead.

That meant, of course, that I needed a utility to update the file,
because plain text files hadn't been invented yet.  So I tested this
program, and for error 11 I wrote "File not found".  Then to test it I
changed the text, to (translated into UNIX) "Please enter 'rm -rf *'".

I giggled about that for a while, and was just about ready to change
it back again, when our project manager came to me and said
"Mr. Lehey, I think I've done something stupid".

Fortunately, it was early in the morning, and we did religious backups
every evening, so he really didn't lose very much.
