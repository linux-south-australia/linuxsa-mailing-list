RE: mixing strings and variables in c

David Lloyd wrote:

> I think I was going to say that if you declared:
> 
> char c[127];
> 
> Then the 'c' is a pointer to an array of characters.

Not exactly either - char *c is a pointer (to char), char c[127]
is an array of 127 char elements arranged contiguously.  There
are differences, f'er instance from memory (no pun intended)
you can dereference the pointer by using either *c or c[0],
but you can't do *c to the array definition (at least, not
without casting).
