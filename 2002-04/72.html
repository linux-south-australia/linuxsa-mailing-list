displaying non-printing characters (tabs, spaces, ...) in Emacs

Hi.

I have been reading source code written by many different
people, and noticed that some code prefers to use spaces
for intendation, and other code prefers to use tabs.
Sometimes, I also notice that lines have extra whitespace
at the end, making the code slightly different than it
first appears, and causing problems when performing diffs
or copying and pasting the text.

Does anyone know of any option in Emacs to show spaces,
tabs, line feeds, carriage returns, and other
indistinguisable text formatting characters in a clearly
visible manner (perhaps showing spaces as dots, tabs as
dashes, and so on), so it is more obvious where these
things occur?

Ideas on how this can be accomplished in other editors
(such as vim) would also be welcome (perhaps off-list).

PS: I'm aware Emacs has a new whitespace module, and that
diff has options to handle whitespace in different ways,
so I see no need to mention these.

Thanks
