Re: Compressed Filesystem

On Wed, 10 Apr 2002, Charles wrote:

> Anybody know of any good compressed filesystems? I found one called e2compr,
> but Its old, incomplete and only works for 2.2 kernels.

There prbbably aren't any.  Compressed filesystems are a superficially
attractive idea which turns into a can of worms when your try to write
one.

DEC wrote one for its UNIX.  Almost everyone ran up their USENET cache on
the compressed filesystem only to see really, really bad performance.

A much better idea is to use a compressing file I/O library, there are
many of these including one in zlib for ANSI C applications.  Porting an
application to use the compressing library is easy.
