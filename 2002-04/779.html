Re: Newbie Samba Guide

I have tried to send my rough guide to a few people but it was too big, so I
don't know if anyone will get the file.

So, I was able to upload it here:

http://www.morton2001.org/samba.pdf
(my boss is a cpa and a politician)


It is called the "Samba sub beta".

It has some Novell and Windows 2K stuff in there too.
I might separate them eventually.

There are a few things I have to add after looking over the guide again.

One basic thing was setting up the workgroup name in Windows. I forgot that.

I have to also think about weather one idea that I have is actually
feasable....

In my effort to take some of the complication out of the set up of Samba, I
end up walking the reader through the entire install and use of SWAT.
However, it avoids the issue of utilizing passwords and user names for
authentication but leaves the option open for the reader to add these
features as he becomes more familiar with Samba. In the mean time I give the
reader a firewall as an interim measure.

Then I thought about IPv6.

My firewall is crude because it uses the manual specification of local ip
addresses.

I thought to myself, only if the ip addresses were not so predictable, the
firewall would act as a retinal scan does as a password.

Then I thought of IPv6 which uses the network card number to create a unique
ip number.

Could I use local IPv6 numbers in the ipchains firewall?

Am I being too optimistic or would this be bad security?

I also need to edit the firewall stuff heavily.

Anyway, I hope it is helpful. I know I am trying to learn by teaching...

James Leone
