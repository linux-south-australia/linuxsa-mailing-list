Re: Innovation

On Friday, 14 March 2003 at 20:14:45 +1030, Richard Russell wrote:
> <sip>
>
>> That said, it's surprising that so little real innovation has
>> come out of the OSS world. The main examples I can think of are:
>>
>> -	TeX: perhaps the first serious piece of open source software,
>> 	which showed the way for others.
>> -	X Window System: given away as source on tapes from the
>> beginning,
>> 	and the first generally available network-transparent windowing
>> 	system as far as I know.
>> -	Perl
>>
>> But these go back into ancient history.
>
> While we're getting into ancient history here, what about Unix itself?
> Wasn't it (and indeed, almost all early software (before the mid 70s)
> effectively open, although not licensed as such? I thought that it (or
> at least ye olde BSD versions) was usually distributed freely to whoever
> wanted it, and the comunity was effectively open, even if not legally
> so.

Unfortunately, it wasn't quite that good.  BSD UNIX never became free:
before BSD became free, it had to shed the "UNIX" name.

In the 70s, UNIX was initially not available At All outside AT&T.
Finally they licensed it to universities only.  BSD code (until the
early 90s, that meant the additional code needed to convert AT&T UNIX
into BSD UNIX) was available "for free", which meant $1000 for a tape.


>> Were these OSS innovations? (I really don't know.)
>>
>> -	e-mail, SMTP
>> -	SNMP
>> -	IRC
>> -	HTTP/HTML
>> -	PHP
>> -	Python
>
> I think (at least for the protocols listed above) that they were Open
> Systems rather than open source. Protocols like SMTP can't be Open
> Source, as they aren't source -- they are protocols. 

Sure, but pretty much the first implementation was sendmail, which has
always been free.

> Basically, every internet standard is open, as I understand. Anyone
> can implement them however they want, open or closed.

Most of the original implementations of the Internet Protocols was
done in BSD, and was thus open source subject to the issues I mention
above.
