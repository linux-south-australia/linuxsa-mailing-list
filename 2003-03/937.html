pam_unix logging

Hey All,

We have successfully got authentication going on our systems here using
LDAP now, but pam_unix is logging errors about unknown users when someone
tries to log in.

Now, the way I understand pam is working is it tries pam_unix (/etc/passwd
etc) and when that fails it goes onto the next module, pam_ldap, which is
successful.

Does anyone know if it is possible to get pam to complain about unknown
users after it has tried every module rather than each module doing it?
I've tried googling but every hit I got back was just snippets from syslog
showing the message but not actually talking about this particular
'problem'.

I would prefer _not_ to change it so it tries LDAP first if I can help it
as it is better (IMO) that system specific details are returned first and
then if that fails the more 'generic' details are returned...
