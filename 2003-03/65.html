Well then, IE is broken (or would that be Windows in general?).

A trailing full stop on a domain name causes the resolver to not use the
search list and only look up that host. Eg, a query on one of our internal
hosts:

-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-

[dross@devel dross]$ cat /etc/resolv.conf
search syc.net.au
nameserver 192.168.14.4
nameserver 172.16.0.1

[dross@devel dross]$ host www
www.syc.net.au is an alias for pri.syc.net.au.
pri.syc.net.au has address 172.16.0.1

[dross@devel dross]$ host www.
Host www not found: 3(NXDOMAIN)
