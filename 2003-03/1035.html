Printer inadequacy

I have a HP LaserJet 5L attached to a Linux (debian woody) server
running CUPS and SaMBa. I print from a Win XP machine. Some documents
seem to cause erros -- not all the page gets printed at once. The page
just doesn't come out, and there is an orange error light on on the
printer. I press the button on the printer, and it prints the page with
some bits missing. Sometimes it's table borders, sometimes cells.
Sometimes bits are just blurry or have other problems -- often text will
be fuzzy (like old newspaper photos), and shading will be half-done.
Sometimes large letters get printed with white rectangles inside the
black sections. Then, it prints another page with the missing bits on
it. This is annoying (duh!). Same thing happens if I print directly from
the lappy to the printer via the parallel port.

Is there any known solution to this? Can I change some setting in CUPS
or in the printer itself? Would I need to get more RAM for the printer?
Am I forced to resign myself to printer inadequacy? Is there a self-help
group for this? :)
