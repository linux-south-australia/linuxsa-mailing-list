> If my perception is right, then there's no need for sendmail to 
> hang around any more. It should go the museum of 1980s memorabilia.

I can't agree.  I want a "modern" sendmail configuration
   - compulsory TLS & password authentication of local users.
   - local user account information in LDAP.
   - compulsory TLS & public key authentication of remote servers
     in our organisation.
   - blacklist subscription.
   - spam assassin.
   - virus filtering.
   - stripping dodgy attachments.
   - server-side mail filtering with rules set by web front end.
   - delivery to IMAP server.

sendmail does the MTA component out of the box, just by fiddling with
sendmail.mc.  I've looked at Qmail and postfix and both require
source code fiddling to meet my reasonable requirements.

If you have a more traditional list of requirements then Postfix
is the better choice.  For large deployments you must configure
Linux to run 1000s of processes.

> The task - to modify the sendmail.cf file to achieve a specified objective.

Look at sendmail.mc
