Re: Troubles with mount and vfat

> Hi all
> Can someone please give some advice?  Am running a
> redhat8.0 box as a server, with a 40GB second hard
> drive with fat32 format containing files that are
> shared across a W2K network.
> We are able to mount the drive and can see it on the
> network, but are unable to set the permissions to
> allow users to write to the disk.
> Any clues as to how we can set permissions so that we
> can achieve write access?

VFAT does not support any kind of permissions at all, so the permissions 
are inherited from the mount point itself. Have a look at 'man mount', in
particular the options for uid and gid under the 'Mount options for fat'
heading. You might also want to use the samba 'force user' and 'force
group' options to match the uid and gid given in the mount command.

Eg, mount the drives as 'guest' and tell samba to 'force user' for that
share to 'guest'.
