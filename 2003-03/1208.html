RE: Redhat and Highpoint Raid controller

Yep, I"ve had this issue before with the new promise controller ATA-133.
I'm no expert but I think you have to find drivers for the controller and
recompile the kernel to use them.  

or the other option that i went with is get rid of the raid array, and use
software raid in linux.  perhaps test it before relying on it.  ie, once
it's done, unplug one drive and see if you can still boot off one drive,
plug in a blank drive and see if you can get it to remirror.  i've got a
redhat 8.0 machine with raid 5.  you can see the status of the arrays in
\proc\md5stat i think it is...

Hi again friendly list people :-)

I have an Epox motherboard with an onboard Highpoint 372 raid controller
which is giving me grief. 

When using two hard disk drives and creating a mirror set (raid 1),
Redhat 7.3 and 8, insists on seeing the drives separately. Windows only
sees one physcial drive which is how it should work, but for whatever
