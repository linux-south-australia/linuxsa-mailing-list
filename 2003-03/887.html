Re: OpenOffice (was: Innovation)

> Take an example: would you give a complete newbie Emacs to write texts
> with?  I suspect not.  But why not?

Because the learning curve is too great.  Doing something
simple should be simple -- text processing under Emacs is
not simple.

As a contrast you might take FrameMaker -- writing a
letter is easy.  But when the user has grown and
feels the need for markup rather than WYSIWYG, well
it turns out that FrameMaker has been doing markup
under the hood all along.

My problem with StarOffice is that it doesn't aim
high enough.  On the plus side, I notice that it's
not precluded from doing so (eg: file format allows
semantic markup) and that many of the people involved
want to make StarOffice a deeper tool.
