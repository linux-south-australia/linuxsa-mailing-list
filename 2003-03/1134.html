Re: PHP include_path

Quoting John Edwards <isplist@adam.com.au>:
> Richard Russell wrote:
> >----
> >;include_path = ".:/php/includes"
> >----
> <snip>
> 
> >I also discovered that if I looked through the libphp4.so file, I found
> >this string (".:/usr/share/pear") in it. Great. So it's compiled in.
> >
> This is not unlike the PATH variable in a shell, although it's function 
> is more like a library location directive in a compiler. Just add in a 
> list of directories you want PHP to look in for include files where the 
> path is not explicitly defined.
> 
> ie:
> include_path = ".:/usr/share/pear:/path/to/smarty/libs"

Eeek!  You probably don't want "." in there.  Indeed it's good practice if you 
keep most of your PHP code outside of your Apache's DocumentRoot for security 
reasons.  You don't want crackers to be able to upload a trojaned copy of some 
library function your code calls.

There's a reasonable paper out there called "A Study in Scarlet" which 
discusses PHP vunerabilities.  It's dated, but still good advice.  Go google, 
go google!
