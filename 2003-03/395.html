To clarify, the EDITOR environment variable will be passed onto
subsequent fork'd environments only if it's been marked for export
(which, surprisingly, is what the export command does - and they
say Unix is picky about its friends).  Therefore, if EDITOR hasn't
been marked for export, the "crontab -e" command won't see it, and
therefore will use its default editor (which, luckily, is vi  :-).

So, as David does, when in doubt, export!
