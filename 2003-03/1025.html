Home made SAN ?

Hi.

I've struck a problem with an outage on our tape backup system that looks like it
will drag on for a few days while the various issues get sorted, and I'm putting a
box together to act as an emergency backup repository, and perhaps take over the
filesharing from an ageing samba box.

Our main backup runs to 145GB uncompressed, and I'm thinking that a couple of
meaty IDE drives on a current P4 system with intel 845 series chipset would fill
the bill. 

I run a (smaller) backup to disk from the current backup server (backupexec W2k)
to some spare disk space on another W2k box, and I get 21 gig in 31 minutes with
software compression of 2.4 to 1 on the backupexec end, the backups are about 8.5
gig on the remote box. That would make the main backup about 60 gigs compressed
(similar data). Network is switched 100mbit.

Note that I'm not considering this for a permanent/routine backup solution, just
as a fall-back emergency store. We have the luxury of being able to get a
complete database backup in each night, and these tapes are stored off-site.

Any thoughts or suggestions please before I go and do this? I'll use Debian woody,
and probably go with an 80 and 200 gig IDE setup for starters.

Thanks,

Michael
