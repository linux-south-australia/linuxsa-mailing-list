Which is faster on modern CPU?

I've got kernel code that optionally collects variables.
The obvious code is:

   if (collecting_counters) {
       counter1++;
       counter2++;
   }

Of course, less instructions are issued if we simply
increment the counter regardless.  If the user asked
for the counter not to be collected then we don't
report the never-the-less-maintained variable to the
user.

   counter1++;
   counter2++;

But I'm not famililar with the CPU cache effects of
incrementing a variable (I'm told a i686 writes all
the way back to main memory), and thus which code has
less effect on an Intel CPU.

Comments?
