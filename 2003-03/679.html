Re: remote build/install of kernel

On Thu, Mar 13, 2003 at 09:11:03AM +1030, David Creed wrote:
> Just wondering how much of a problem it would be to NFS mount 
> /usr/src/linux from the remote machine, would that be a way to do it?
> or would the network be a bottleneck?
> 


David,

That sounds like it shoudl do it -- except that NFS is on my list of
deprecated technologies, like telnet and ftp.
As one Mark Newton once pointed out, if someone can break into one
machine, then NFS gets them some access to other machines.

It sounds from the comments that there is not standard technique for
cross-installing the kernel.
So I guess I'll use a data-stamp file and then run "find" to make a list
of files changed relative to the data-stamp, and then "scp" these
across at an appropriate point in the installation - either
just before the "make modules_install" or just after.
I prophesy that in 20 years' time, cross-installation of kernels will
be fully standardized. Mark ye!
