Samba vs. windows XP - blank password

hi all.

i've got a windows XP box and a redhat 8.0 box at home. samba is set up on
the RH box and running nicely - i can browse the shares on the RH box from
the XP box, the authentication seems to work going that way (i.e. viewing
the RH box from the XP box), etc. i've got accounts (both normal and via
`smbadduser`) on the RH box that match the usernames on the XP box.

now, to the issue at hand. on the XP box i've got a few user accounts, all
of which (apart from the administrator account) have no password set.
adding the usernames with `smbadduser` and merely hitting enter when
prompted for a password worked - the XP users can browse the RH box fine.
when i try a `smbclient -L xpbox` (logged in as a normal user) and just
hit enter at the password promt (for, hopefully, a blank password), i get
an error of 'Error returning browse list: NT_STATUS_ACCESS_DENIED'. if i
do `smbclient -L xpbox -U administrator` and enter the right (non-blank)
password, it works as expected.

i've tried googling, with no joy. anyone got any idea how to authenticate
with XP and blank passwords?

i should probably mention that there's not a domain server or anything in
effect; it's just a simple 2-machine workgroup.
