> > I have a GA - 586ID DUAL PENTIUM PCI - EISA BUS MAINBOARD on my linux
> > computer running Redhat 7.3. The board has six 72-pin memory slots and
> > my idea was to put in 6 x 16 = 96meg of RAM. My nephew installed the
> > linux and successfully got the OS to use both pentium processors. The
> > manual says the board supports up to 512 MB of DRAM memory, so it
> > doesn't have a problem with using more than 64 MB but the OS does. We
> > tried putting in a command we got off the internet to tell Linux to use
> > all the memory to no avail. We also tried swapping the first two 16's
> > for 32's and repeated the command but no change. Has anyone any idea on
> > what to do to make it work? Regards
> 
> Redhat 7.3 definitely supports more than 64M of memory, I have at least
> one 7.3 box with 1Gb in it. Depending on the bios on your motherboard you
> might need to tell LILO about it however.
> 
> When you boot 7.3 you get a graphical bootmenu. Hit CTRL+X to drop to a
> text 'Boot:' prompt and try:
> 
> Boot: linux mem=96M
> 
> If that is successful in making the kernel see the extra memory then you
> will need to modify the lilo.conf file to add an append line for your
> kernel image so that you don't have to do that every time. Read the lilo
> man page for more details.
