Re: Redhat 6.2 error

Michael,

> [yavno@beaker yavno]$ rpm -qa
> /usr/lib/rpm/rpmq: error in loading shared libraries:
> libdb-3.1.so: cannot open shared object file: No such file or
> directory

It can't find the right version of the Berkely Database. I'd suggest
going to:

 * http://www.rpmfind.net/

..and typing in that file name. And then installing it.

You're likely to fall over a heap of dependency problems attempting to
install it though.


DSL

(alternatively do something like:

# mount /mnt/cdrom
# find /mnt/cdrom | grep dbm

..and install anything that looks like it might be a version 3.1 of
Berkeley's DBM. If you're not sure, you can do:

# rpm -qi xyz.rpm

..where xyz.rpm is the result of the above find command.

If you're using KDE it might have the file find utility installed. Look
in your "K Menus")
