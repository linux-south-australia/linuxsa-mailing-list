> 1.  It ignores directory structures, just like Microsoft.  It doesn't
>     seem to understand the concept of a cwd at all.  If I create a new
>     document, it will always try to save it in my home directory.

is there really a concept of cwd in the linux gui?  This is an innocent 
question...not trying to push a point here.  What I mean is that I could be 
working on a word doc and typing an email at the same time.  if i went to add 
an attachment to the email, should that start at the directory the word doc 
was in?  if i select an attachment from a different dir, should openoffice go 
to this new dir when you select 'save as' ?  My StarOffice 6 (as you've said, 
and i agree, openoffice should be no different) always points to the last 
directory used _within StarOffice_.  For example, when i click 'open' it goes 
to the last place i opened a doc from previously.  When I click 'save as' it 
will go to the directory i opened the _current_ doc from.  That is exactly 
how I want it.  Not sure if that is what you meant though :-)
