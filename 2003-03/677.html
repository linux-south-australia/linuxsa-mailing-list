Re: remote build/install of kernel

Alan Kennington wrote:
> 
> Unfortunately not.

D'ohh!

> The linux kernel source "Makefile" file has a couple of things that
> can be changed to install things to different paths.
> But it's nothing as convenient as the "autoconf" style of
> thing like "./configure --prefix=/usr" or whatever.

You could try using something like fakeroot, which (IIRC) allows you to 
also specify a different root directory - thus when you finish you can 
tar up the contents of that dir structure and ship it across to other 
machines.

> But David's idea of just doing an ssh of a tarball or something
> just before the "make modules_install" sounds like it might be
> the bext I can do.

Another alternative would be to export /usr/src/linux via NFS and mount 
it on the target machines, and install. This is how I maintain my FBSD 
boxes.

> The thing that worries me in particular is that the host machine's
> properties could be used to some extent to determine what gets
> compiled or installed.

I would assume that unless you explictly specify arch-based 
optimisations when building, then it should be fine. This will probably 
hold true as long as the arches of the target machines are similar 
enough and you specified a generic-enough CPU for all the target 
machines in your kernel config. You also want to make sure sure that the 
userland on the target machines is up to scratch as well.

YMMV however; this is not something I have much experience with on Linux.

Still, I have long pined for something like FBSD's /etc/make.conf on Linux.
