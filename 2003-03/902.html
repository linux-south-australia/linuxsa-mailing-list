A customer:

> This is my current understanding of the Lindows.com tracking and 
> profiling system:
>
> When you sign up with Lindows.com you are issued a membership number 
> such as 103942794424199 (I of course changed some of the numbers).
>
> This membership number is tied to your email address, real address, 
> credit card info, etc. There is no way to use Click-N-Run without 
> giving Lindows.com all of this information.
>
> Each computer that you install LOS on is assigned a unique machine 
> number that is stored on your hard drive and associated with your 
> membership number.
>
> When you startup LindowsOS, Click-N-Run starts automatically, phones 
> home, and checks your 'machine number' (and presumably your IP 
> address) to make sure your 'membership number' does not have too many 
> 'machine numbers' running or listed. If there are too many your CNR 
> login is shut down and you have to call support.
>
> A list of all software that you have ever installed from Click-N-Run 
> is kept by Lindows.com and organized by 
> membershipNumber/machineNumber/softwareApp. Lindows.com promises that 
> they will keep this list for the rest of your lifetime. So far they 
> have been unwilling to promise that they will never sell or share this 
> list with their 'business partners'. So maybe they will sell your 
> profile some day.
>
> Software you purchase using Click-N-Buy will include embedded code 
> that will phone home to Lindows.com every time you start the software. 
> Lindows.com will check your membershipNumber/machineNumber/softwareApp 
> database and decide if you should be allowed to run the software.
>
> Lindows.com has developed a feature of Click-N-Run that lets them 
> automatically uninstall software from your computer without your 
> approval or knowledge. Because of the uproar this caused they have 
> said that they don't plan on doing this.
>
> The AOL/Netscape web browser is integrated into LindowsOS and is 
> impossibly difficult for the average user to remove.
>
> One of the marketing slogans Lindows.com has used is "no Microsoft 
> Activation codes". With this system they have more control over our 
> computers and our personal information than Microsoft has ever had.
>
> Looks to me like Lindows.com is doing things Microsoft can only dream 
> of doing. If Microsoft set up a system like this they would be 
> crucified by their customers and the press.
>
> This is certainly not what I thought I signed up for when I became an 
> Insider last June.

The President responds!

> imho this is worst than Windoze big brother...

> We have to have a copy protection option available to get certain 
> commercial programs in the Warehouse. A good example is Sims, which 
> we're working on with EA and Transgaming. Sims uses a "key CD," but 
> because we want the warehouse to be a 100% digital distribution 
> system, we give software vendors an option if they feel they must have 
> some sort of copy protection. If you like digital distribution, then 
> our approach is required since it doesn't use annoying USB keys, 
> unlock CDs, authorization codes, etc. Our approach is very simple, as 
> it works with the existing warehouse membership login process. It's a 
> tiny bit of code that simply checks to see if the person has purchased 
> the program. The only data it returns is a simple "yes" or "no," and 
> it only returns that info to the program, not the vendor. So, this is 
> hardly "Big Brother."
>
> Like it or not, if you want to use the software from certain 
> commercial vendors (particularly games), they'll require this. If we 
> didn't offer it, then LindowsOS users wouldn't get access to these 
> programs. If you don't like programs that use it, simply don't 
> Click-N-Run those programs, but those who do want them should have 
> that option. We don't want LindowsOS users to be limited in their 
> software choices.
>
> ...if they make it to easy they won't get any publishers with their

>
> Correct. It's hard enough convincing commercial software vendors to

>

>
> Will the products still work if we are off line or does it have to
