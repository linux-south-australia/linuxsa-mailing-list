Re: remote build/install of kernel

Alan Kennington wrote:
> Has anyone figured out what things you have to worry about
> when you compile a kernel on one machine and then install it
> to another machine?

We do this all the time.  The summary is the kernel build
process can create fine RPM or DEB packages.  If you've
got a lot of remote machines then use the standard RPM
or DEB updating tools (eg: autoupdate).

 > I like standards!

Such as treating the kernel software like all the
