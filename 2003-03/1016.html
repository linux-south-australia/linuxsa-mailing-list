Re: Samba and Unix password sync not working

On Fri, 2003-03-21 at 07:51, Andrew Galdes wrote:
> G'day all,
> 
> I have configured samba to sync the linux password when the samba password is changed. It isn't working though. The samba password changes ok but nothing happens to the linux password. This is my smb.conf relating to passwords:
> 
> unix password sync = yes
> passwd program = /usr/bin/passwd %u
> security = user
> 
> I have tried the "passwd chat" command but that didn't work either. I have left that as default. The logs don't show anything, no errors. Also, the samba doco says that if there are troubles changing the linux password, the samba password won't be changed. (Acording to O'Rielly.)

You'll need passwd chat. What did you set the passwd chat option to?
Have you tried any of the standard troubleshooting techniques, like
setting passwd program to "touch /tmp/passwd_%u" to see if it actually
executes?

How about turning up the debug level a bit (I think a debug level of 10
will contain the actual passwords typed as well as a heap of other
info). Start with level 3.
