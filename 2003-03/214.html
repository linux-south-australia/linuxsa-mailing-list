Re: sec:unclassified NTP Question

Alex,

> On a device that has two Ethernet interfaces, is it possible to bind
> one interface to act as an NTP Client and to bind the other interface
> to be an NTP stratum 1 server ?

I would assume that one could do this on a routing table level. That is
to say, if the IP of the stratum 1 server were something like 1.2.3.4
one could add:

route add -host 1.2.3.4 netmask 255.255.255.255 eth0

..if this makes any sense to you. I've had a preliminary look at the
NTP client documentation and not turned up anything but this was only
from a casual read and a few "greps".
