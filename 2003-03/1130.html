PEAR is the processing engine that PHP uses (or perhaps it is _A_
processing engine). In any event, if the include_path is commented out in
the php.ini file then it is using the default that is compiled in. I am
not sure whether it adds to the path or overwrites the path if you define
it in the php.ini. You could make it something like

include_path = ".:/usr/share/pear:/opt/smarty";

or whatever. The current path is the directory that your PHP script is
running from. So if I make a php file /home/darryl/public_html/test.php
then the current directory is /home/darryl/public_html.
