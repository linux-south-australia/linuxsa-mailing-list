RE: RH8.0 - POP3/outlook express problem.

Most POP servers lock the mail file while a POP session is active (this is
in fact required by the RFC) to ensure that no new mail is delivered while
the POP session is active, and also that the same user can't have two POP
session active at the same time.  Normally this means that if the users
connection fails (eg. dialup drops out), or they kill their client (as you
user did) the lock is left on the server for a period of time.
Unfortunately Outlook Express handles the locked mailbox situation rather
poorly, and simply prompts for a password repeatedly, but the real problem
is the server lock.  Once the original connection times out, the server will
release the lock and you can check email again.  It can take up to 15
minutes for the lock to be released.
