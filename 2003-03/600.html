Yeah, earlier versions of Moz (and hence NS6/7 unfortunately) don't seem 
match the ":hover" pseudo-class on all elements, so the CSS rule which 
causes the UL to be displayed ("#links:hover > ul { display: block; }") 
never gets matched and the UL never becomes visible. This is painful and 
requires a workaround.

That's also what is happening on IE5/Mac. Luckily IE5/6/Win ignores CSS2 
child selectors ">" and so doesn't hide the UL in the first place. 
Safari and latter versions of Moz handle ":hover" well, and apparently 
the next Netscape release will be based on Moz 1.4, so it should work in 
Navigator after that.

> (and PS. the js one, it's not immediately obvious you're supposed
> to click on the icon since it's not a hyperlink and the mouse
> cursor doesn't change etc)

Ah yes, some more ":hover" CSS effects on that H1 element would fix 
that. You wouldn't make it a link just to get the effect, that would be 
akin to spending a fortune on a top of the line graphics card for a 
headless server - there's no point, it's just plain wrong.
