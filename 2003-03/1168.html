RE: Samba vs. windows XP - blank password

Ben,

I'd use smbmount rather than smbclient.  If all you are wanting to do is
share files, then it's an easier way to go.

Check the man page for smbmount, but in a nutshell:
> smbmount //host/share /mount/point -o
"workgroup=WORKGROUP,username=joeblo,password="

The only gotcha is that you might need to setuid root on /usr/bin/smbmnt
if you are intending to use smbmount as a non-root user.  Similarly for
smbumount, although I can't remember which file needs to be setuid root
for that one...  The error will tell you if you try to use it.

hi all.

i've got a windows XP box and a redhat 8.0 box at home. samba is set up
on the RH box and running nicely - i can browse the shares on the RH box
from the XP box, the authentication seems to work going that way (i.e.
viewing the RH box from the XP box), etc. i've got accounts (both normal
and via
`smbadduser`) on the RH box that match the usernames on the XP box.

now, to the issue at hand. on the XP box i've got a few user accounts,
all of which (apart from the administrator account) have no password
set. adding the usernames with `smbadduser` and merely hitting enter
when prompted for a password worked - the XP users can browse the RH box
fine. when i try a `smbclient -L xpbox` (logged in as a normal user) and
just hit enter at the password promt (for, hopefully, a blank password),
i get an error of 'Error returning browse list:
NT_STATUS_ACCESS_DENIED'. if i do `smbclient -L xpbox -U administrator`
and enter the right (non-blank) password, it works as expected.

i've tried googling, with no joy. anyone got any idea how to
authenticate with XP and blank passwords?

i should probably mention that there's not a domain server or anything
in effect; it's just a simple 2-machine workgroup.
