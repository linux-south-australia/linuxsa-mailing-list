Re: XHost +

I don't think its too bad (please correct me if if i've overlooked a
posability). Your allowing any local user to make a connection to your X
server, so different processes under different usernames can display
stuff I suppose. If you still check that all connections are coming from
your server then theres nothing remote people can do. All the local
users can do is really run an x program, which is probably only a minor
annoyance. Attackers I guess would need to find a way to get a shell,
then use the more lax access controls together with some other bug like
one of the xterm title bar buffer over flows or something. So, as usual,
you can never be 100% sure your safe, but I wouldn't be too worried.

Ant
