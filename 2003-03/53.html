> Q; When recompiling the Redhat 7.3 kernel 2.4.18-3 what dir structure
> do I use.

It is customary to unpack kernels into the /usr/src/ directory.

> Currently I have been downloading the tar of 2.4.18-3 and tar -xvf in
> /root. This creates a dir called Linux off of /root and copies the
> file structure there.

Fair enough.

> Should I tar -xvf the tar ball to a different dir structure?

It is a matter of choice, but :

 * http://www.pathname.com/fhs/2.2/fhs-4.12.html
 * man hier

...both suggest that it should go into /usr/src somewhere.

I suggest doing the following:

# cd /usr/src
# tar xzf kernel-2.4.18.tar.gz
# ln -s linux-2.4.18 linux
# cd linux
# [make the kernel therre]

That is to say, unpack the kernel into /usr/src and leave it at its
default name. Then create a symbolic link to it called "linux". You may
need to move aside a directory called "linux" or delete a link called
"linux" depending on what system you are running.
