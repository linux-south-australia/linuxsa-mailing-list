Hi Dan,

First let me say welcome back.

To business:

Instead of using the old wvdial.conf backup, try re-running the
wvidalconf script as root to probe for your modem again & write a new
wvdial.conf.
-> 'wvdialconf /etc/wvdial.conf'

You should see some good diagnostics as it prints the scan results to
screen. Worthy to note that if you use a serial mouse it may already be
tying up /dev/ttyS0, 'ls -l /dev/mouse' to make sure, or the wvdialconf
script should pick this up.

Afterwards, edit wvdial.conf to suit & run wvdial
Hope this helps, it may or may not work - but it's the first thing I'd
try.
