Yeah, but you have to accept that there's a lot of crappy web sites out 
there that you can't see properly if you choose to view them like that. 
However my point is that this the the fault of the web site; it isn't 
your fault.

Have a look at: 
<http://web.vee.net/src/markup/css-menu-demo/css-only.html>;. This 
contans a dynamic menu which is implemented purely in CSS, no 
Javascript. It contains images and dynamic effects, but is still 
perfectly usable and legible in text-based browsers, and in versions of 
Netscape back to 0.96. Because it is implemented in CSS, and the markup 
is well-written, it is not only accessable but also degrades gracefully 
in older browsers. This example is not perfect, but it illustrates what 
it is possible to do with CSS.

Note that it's not a 100% copy of Luke's Flash (the top image isn't 
clicky), but it *is* usable in every web browser, regardless of 
Javascript, CSS and HTML capabilities.

Give me a moment longer and I'll come up with a Javascript version that 
implements the clicky functionality. I bet I can do it in < 20 lines of 
code.
