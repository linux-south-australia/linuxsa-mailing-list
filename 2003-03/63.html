Re: recompiling the Redhat 7.3 kernel 2.4.18-3 what dir structure do I

On Monday, Mar 3, 2003, at 09:49 Australia/Adelaide, Kim Hawtin wrote:
>
> After browsing through this I read through all the /var entries.
>
> Now, I've read various configuration guides as to how to split
> these file systems across disks/partitions, and I've wondered
> about whether or no to split /var off onto its own disk.
> Especially for a mail server or log server ...
>
> So, the question is, what is the experience of the list on this?
>
>

Depends a lot on what disk subsystem your using, how heavily utilised 
the server will be etc...

For a substantially used mail or log server, /var on a separate disk 
makes a lot of sense. Even more so, if you're stuck with IDE disks, in 
which case a separate channel of the disk containing /var would be a 
good idea too.

This is even more important on a general utility server where other 
filesystems may be accessed at the same time as the mail spool. Since a 
disk read head can't be at two places at the same time, you *will* 
suffer performance degradation if /var and another heavily used 
filesystem share a physical disk.

However, if you're not using it much, leaving it on the same disk as 
root is fine, but I'd still use a separate partition (since /var has a 
propensity to fill up, you don't want it on / ).
