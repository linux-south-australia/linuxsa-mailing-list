On Tuesday, March 11, 2003, at 04:16 PM, Mark Newton wrote:

> But if you're going to use Flash for something as mundane and essential
> as a navigation menu (especially if it's the only navigation menu you
> plan to provide), then think seriously about what you're doing any why
> you're doing it.  Perhaps, just perhaps, it's the completely wrong tool
> for the job.

Completely agree with this post;  I see so many sites developed with 
flash navigational systems that could have been achieved with 
javascript easily, seems that the web developer got lazy (making flash 
menu's is incredibly easy).  The same authors, strangely enough, don't 
seem to know a lot about optimizing flash files for size, and you'll 
end up with a 200k flash file where there should have been 1k of 
javascript.

There's a standard for javascript now, too (w3c), and almost all modern 
browsers comply reasonably well to it (ie, opera, gecko), so there's no 
excuse not to use it.

Flash can be very, very effective - I've seen some mindblowing sites.  
Just seems people try to substitute it in where its not terribly 
appropriate.
