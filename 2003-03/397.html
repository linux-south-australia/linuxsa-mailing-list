XHost +

Greetings Peoples,
Have a SUN Server running Oracle 9iAS witch has a Java service component. 
According to our DBA guys, there is a bug in SUN/Oracles code that means 
that the service doesn't run properly with properly unless the command xhost 
+ is run. I can get into Oracle's metalink to check this, but it fixed the 
problem on the server I was running this on when I ran "xhost + mincepie" 
(mincepie being the server name). 

What I am wondering is at what risk does this place the server. The network  
is private and firewalled, so there is no access from the big bad Internet 
(tm). As I understand the XHost command, you must have access to the server 
to take advantage of this. 

My questions would be:
What security risk does this place to the server from users, if any (and if 
it does, how might they take advantage of this so I can block it) ?
What security risk does this place the server at from non users (ie people 
who might be scanning the network or something) ?
Is there a way I can start the service without running Xhost ? 

Much thanks in advance.
