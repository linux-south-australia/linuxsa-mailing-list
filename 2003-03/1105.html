Re: Best OS for Dual Xeon

> If I'm going to do some benchmarking, I may as well make it reproducable
> and do it on common OS's. Today's -STABLE version of FreeBSD may not be a
> very good frame of reference for future tests. I also don't really have
> the time to tweak and tune, so it will be a case of install, recompile
> the kernel with SMP, and test out the default mysql port!

With an SMP aware kernel but HT unaware you should get some radically
different results with HT on from one test to another.  This depends on the
benchmark though.

For example, if your benchmark runs only 2 processes and these get
scheduled on the same physical CPU but different virtual CPUs, then it'll
(probably, depending on how independent the processes are) run much slower
than if they are scheduled on different physical CPUs.  Where they get
scheduled will be random with a non HT aware scheduler and therefore you'll
get randomly varying results from one run to another.
