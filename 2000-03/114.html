Re: Lilo

Brian Marr wrote:
> 
> Three Drives
> 
>         hda     Win95
>         hdb     RH 6.0
>         Hdc     Suse 6.3
> 
> When run from RH, lilo writes to the MBR.
> Both RH and Win95 will boot ok.
> 
> If I want to add Suse to the Lilo config, does lilo have to be run from
> Suse ?
No.

> 
> Will it overwrite the current config or just add to it ?
Not the config, just lilo in the MBR. Given that you have lilo running
and configured nice on the RH partition, I'd personally do the SuSE
config from there also.

> 
> The below Lilo configuration is booting Suse with the 2.2.5-15 kernel !
> :(
> 
> boot=/dev/hda
> map=/boot/map
> install=/boot/boot.b
> prompt
> timeout=60
> #Boot RedHat
> image=/boot/vmlinuz-2.2.12smp
>         label=RedHat
>         root=/dev/hdb1
> 
> #Boot Suse
> image=/boot/vmlinuz
>         label=Suse
>         root=/dev/hdc2  #hdc1 is /boot
> 
> 

try copying the SuSE kernel (/boot/vmlinuz) to the RH partition. As I
understand the paths and things are relative to the current root file
system (/), so if RH is running, lilo will look on /dev/hdb1, in /boot
for vmlinuz, whereas is SuSE was booted, it will look in the same place
on /dev/hdc2.

So, mount /dev/hdc2 to wherever. Change the /boot/vmlinuz line to
/wherever/boot/vmlinuz and re-run lilo.

Hope it helps,
