Re: Web / IMAP gateway

> On Wed, 29 Mar 2000, Dan Shearer wrote:
> > Actually what I meant was if you have very large folders and you hop
> > around them frequently then IMHO works faster.
> 
> Isn't this substantially an issue of imap servers?  If your imap server is
> optimised for large folders then hopping around will be quick.  If it's
> not -- and lest there be confusion UW's is not -- it will be very slow.

Not completely, especially if you are using a webmail client that likes to
fetch the contents of each message as you cursor down a mail index. If it
is a long index and you cursor down fast then you quickly start finding
some limitations outside the imap server to do with setting up and tearing
down that many imap connections.

Incidentally it is easy to come to an unfair judgement of the UW server on
performance grounds if you only use it with bsd format folders. UW
supports several much more efficient filetypes. (Nevertheless I far prefer
Cyrus to UW).
