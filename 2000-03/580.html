Don,
try: sed 's///' <name_of_file> > <new_name_of_file>

apologies if I misinterpreted the Ctrl-L character. In vi, 
the carriage-return character is invisible, and I have been wondering
how to get rid of that with sed (don't know yet!) but if the line-feed is 
visible, this should do it. When entering the Ctrl-L character, use the 
key combination [Ctrl]-V [Ctrl]-L to create the character.

hope this helps,
