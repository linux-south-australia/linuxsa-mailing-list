As far as I know, the only file permission thing you can set
on a per-directory basis is group ownership, by setting the
guid bit in the parent directory to make sure that the
group of all newly created files in the directory are
the same as the directory rather than the group of the user.

Everything else is done by combining the process mask
together with the per-file-context mask in the open()
command (or something like that).

Maybe there's some obscure shell which reads a further
mask from some hidden file in the directory.
You could write such a shell!!
