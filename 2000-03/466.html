I don't know about through IP masquerading, there must be some wag of
setenv-ing the display, but not sure, probably the same way you'd tlnet
back to your box.

> Also, the Uni machines use Solaris, I think. If I can get the uni machines
> to communicate with mine, will it work?

This is the beauty of the X-Windows system. All the applications are run
remotly, and only the display data is transfered to your local system. No
CPU load, but a fair ammount of network traffic. Your local machine need
only be capable of X-display. This can either be with Xfree86 (or
similar) under linux, or an emulator from windoze.

Be aware that this can be very slow over a modem, when I did this with
Netscape, it took 4 mins to start, and core dumped after 5. When I last
observed this, Martin Sandiford <ms@mcdev.com.au> observed there is a
piece of software called lbxproxy which can help here, but I believe there
are specific X-Server requirements.
