Re: graphics continued

Jake Hawkes wrote:

> font paths are used to tell the Xserver where to look for fonts.
> they can be filesystem paths or they can be unix sockets.
> in this case, unix/:-1 is supposed to be a socket, but its quite
> obviously wrong. You cant (as far as I know) have a socket with number
> -1.

I think the X server uses -1 to differentiate between a socket
in the Unix domain and one in the Internet domain.  The unix domain
sockets are sometimes referred to as file sockets (they're sockets
that live in the file system namespace, as opposed to the Internet
namespace), and have 's' in the type bit when you do an 'ls -l' on
them, eg -

[dt@nipnyep dt]$ ls -l /tmp/.font-unix/fs-1 
srwxrwxrwx   1 xfs      xfs             0 Mar  7 08:08 /tmp/.font-unix/fs-1

> Quisk fix, comment the fontpat unix:/-1 line out of you XF86Config file
> which is in /etc/X11

This may be falling over because xfs isn't running/installed, which
may be the real problem.
