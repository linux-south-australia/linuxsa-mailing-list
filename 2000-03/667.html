Simple Routing with PPP0

I am having a little difficulty wrapping my head around this one; I'm sure
there are members of the list who have done this before:

I have two workstations, IP addresses 172.16.1.2 and 172.16.1.3 (one is
Linux, one is W98). Both have a gateway of 172.16.1.1.

My server (172.16.1.1) is connected to these two by ethernet. I also have
PPP loaded in "demand" mode. Any internet traffic (ping, ftp, etc) started
on the server, destined for a different subnet, brings up PPP and resolves
it there (default gw)

The next obvious step is to have traffic from the workstations routed
through the server, to automatically dial PPP. This involves the server
acting as a router or gateway, routing traffic between the ethernet card and
PPP0.

Without getting too worried about security, what is the easiest way to set
up a static route between the two interfaces? I checked a number of HOWTOs
and resources, but they concentrate on firewalls (a good idea, but this week
I just want to get connected).

The server is RedHat 6.1, most major packages installed, default kernel.
Interfaces are:
ETH0: 172.16.1.1, 255.255.255.0
PPP0: noipdefault (get one from ISP), defaultroute, demand

Any hints would be appreciated. I'll deal with more advanced topics like
IPMASQ, Squid, etc later.
