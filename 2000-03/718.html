Indeed, as you commented, I had in mind to put the
web browser and httpd on the same machine
(or at least behind the same firewall), and then
run some sort of OpenSSH tunneling over the 
evil, hazardous Internet.

============================================================
Option 2:

|-----------------|             |------------------------|
|                 |             |                        |
|db  <->  postgres|  <-- Z -->  | httpd  <->  web browser|
|                 |             |                        |
|-----------------|             |------------------------|
                      openSSH

============================================================
The other option was:

Option 1:

|-----------------------------|             |------------|
|                             |             |            |
|db  <->  postgres  <->  httpd|  <-- Z -->  | web browser|
|                             |             |            |
|-----------------------------|             |------------|
                                  ssl/http

===========================================================

I sort of think that the following might be the situation:

			Option 2	Option 1
---------------------------------------------------
set-up difficulty	moderate	messy

convenience of use	good		not so good

traffic performance	good		not so good
---------------------------------------------------

However, one's persepective often changes when
things get going.
I'd be interested in any experiences.
