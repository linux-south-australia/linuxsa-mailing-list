Did you try searching for 'offline' in the squid config file?

When I did I found...

  #  TAG: offline_mode
  #       Enable this option and Squid will never try to validate cached
  #       objects.                                                                

So that looks like objects are always served from cache until they
expire. If they are not cached, nothing is returned. Have a look at
www.squid-cache.org for more info, but I would be surprised if there
wasn't more control than this. If not, have two different squid.conf's
and a script to swap them and restart squid when the link goes up or
down. ip-up.local would be a good place.
