Re: eSquire mailing list manager problems

Darryl Ross wrote:
> 
> > 554 "|/home/esquire/bin/relay testing"... Service unavailable
> 
> The permissions on /home/esquire/bin/relay are:
> 
> > -rwsr-xr-x   1 root     root        12890 Mar 19 23:16 relay
> 
> Do I have to do anything special to sendmail to get it to be able
> to use the relay program?

  cd /etc/smrsh
  ln -s ../home/esquire/bin/relay relay

And refer to the program as /etc/relay in the sendmail
configuration files.

I'm surprised that `relay' isn't a wrapper program that
gets out of being root and into some less powerful user
ID as quickly as possible.

When you've finished testing, make sure `relay' ends up
in a directory owned and writable only by root.
