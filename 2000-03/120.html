> I want to use a couple of perl modules which don't appear to be with the
> std RH perl RPM. So I tries to install some modules using CPAN.
> According to the destructions, I need to do a 'perl -MCPAN -eshell'.
> This gets me to a prompt, where types me: install Net::SMTP or install
> Bundle::libnet (cause perl seems to think the latter is a good idea). It
> tries to use lynx as I haven't installed the libnet module. I have my
> proxy details set up (and can use lynx for accessing other stuff). But
> it takes ages and ages and ages then times out.

did you set your proxy details in the CPAN config file?
generally it's kept in
/usr/lib/perl5/5.00503/CPAN/Config.pm

lynx may have proxies set, but that doesn't mean that CPAN does.
