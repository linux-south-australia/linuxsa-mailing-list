Re: kernel log

Subba Rao wrote:

> Each time my system dials to my ISP, it creates a log file on /.
> 
> home/                      kernel-log.10.36.6.186   kernel-log.10.37.51.48
> kernel-log.10.34.209.13   kernel-log.10.36.6.24    kernel-log.10.37.51.60
> kernel-log.10.34.209.158  kernel-log.10.36.6.241   kernel-log.10.37.51.74
> kernel-log.10.34.209.18   kernel-log.10.36.6.25    lib/

These look like IP addresses, and syslogd has no native facility for
creating file suffixes like these on log files.  Your /etc/syslog.conf
isn't getting creatively mangled by your dialup program, is it?  Does
/etc/syslog.conf's modified time change when you dial in?  What does
fuser say about these files?
