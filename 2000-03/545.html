Re: Login time restriction

/etc/security/time.conf might be a good place to start looking.
Failling that, "At" is your friend.

if [ $USER = "userA" ]; then
	echo "disconnect $USER" | at now + 9 hours
fi

(in a very simplified example)
(you'd want a similar logout script that removed the at entry.)

Or, you could use cron. Run a job every few minutes, that checks how
long users have been logged in for, and compares it to a table of
maximum times. If it's exceeded, boot the user.
You could do this with bash too, but I might be moving more towards PERL
for such a task, since it'd be easier.
