RE: Using fdisk in a RH6.1 install

> Hi Folks,
> 
> I'm trying to install a RH6.1 system with lots of partitions (15)
> and Disk Druid keeps "helping" me by moving things.  In the end
> it throws away two of the partitions it's previously accepted.
> These are listed as "Unallocated Requested Partitions" with an
> error code of "Allocation Succeeded" or something similar.
> 
> As I'd have preferred to specify the cylinder numbers directly,
> I thought I'd just switch to fdisk, but I'm not clear of how to
> do this in 6.1.  It's not possible to run it the Alt-F2 screen
> because it doesn't like /dev/hda at that point.

I believe you should be able to switch to the ALT-F2 screen, and
type `mknod /dev/hda`.  Then `fdisk /dev/hda` should work properly.

I prefer fdisk over Disk Druid any day..  but I like getting my
hands dirty.

HTH!
