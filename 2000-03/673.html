Re: Linux DBMS

An e-DBMS that I could recommend is Cache from InterSystems in Boston,
www.intersys.com

You can download a free trial of version 3.1 (Build 386).

The only problem is that it is object oriented and some of the tools are not
yet written for Linux and therefore require a win9x or nt box to do the
initial development.

Once the development is done, you can talk directly to the server, via
tcp/ip on port 1972 (active X does this on windows). You can go through SQL
via ODBC. There is also a built in Web development module called WebLink
Developer which is very easy to use and create systems with.

We are using it were I work for e-systems. Check out www.webpos.com.au to
see an older version of Cache and Weblink developer in actions.

Rob.
