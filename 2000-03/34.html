> Two things immediately leap to mind.

> 1)  If you're using the native OSF linker, it might be stripping the
>     comments section before linking (see 2).
> 2)  COFF, like ELF, has seperate sections for the various segments
>     (code, data, BSS, and others).  There may be a comments section
>     that the #ident string is being placed into.  I can't remember
>     if there are any utilities that come with OSF that will list
>     what sections are contained within a COFF file.  A workaround
>     is to use:
> 
>         static char ident[] = "#IDENT string goes here."
> 
> I'd examine the intermediate object file to see if the string is
> being placed in there; that would then isolate the problem to the
> compiler or the linker.
