I know you won't like this, but in the old days,
we used to write software to do this sort of thing.
About 5 years ago, I wrote a general tool to divide
a file into n equal parts, where n is any
integer or floating point number.

Here's the help page:

==================================================

---------------------------------------------------------------------
 /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/
 /-/ /-/ /-/ div /-/ /-/ a file division utility /-/ /-/ /-/ /-/ /-/
 /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /-/ /a/ /k/
---------------------------------------------------------------------
Linked on emu: Fri Dec 31 00:13:39 CST 1999 (UTC: 1999 Dec 30 Thu 13:43:39)

Arguments [defaults] for file division program "div":
  -f  <first_suffix> starting suffix for output files. [1]
  -h                 show this help page.
 [-i] <in_file>      input file name.
  -n  <n_parts>      number of files to divide the input file into. [2]
  -o  <base_name>    the base name for output files. [in_file]
  -t  <level>        set the trace level. [0] 

==================================================

If you ahve a file zzz, you type

div zzz

and you get zzz.1 and zzz.2.

The (open and free) source is at:

    http://www.topology.org/src/aktools/div.c

If you type "div -n 2.5 zzz", you get 3 files,
whose lengths are in the ratios 1 to 1 to 0.5.
I used to use this in the olden days for sending
files over links with a BER of 10^-1.

Unfortunately, you need my C++ library too, at:

    http://www.topology.org/src/aksl/README.html

Hmmm, I guess compiling C++ software is a bit
beyond the pale.

But I thought I'd mention it anyway.
