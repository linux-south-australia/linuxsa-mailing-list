Re: monitor / graphics

It looks like gdm or xdm are dying because of an "invalid" X config.
Type linux 3 at the LILO Boot: prompt (where linux is the lilo image
label - default = linux). This will boot you to a fully functional text
interface, where you can run startx. This will try to run X without the
GUI login thing and die with error messages if there are any problems
with the X config. If X starts fine, it's the XDM / GDM config and
someone else will need to help. It will most likely be something to do
with the X config (wrong mouse or Windows Plug 'n' Play-erizeing the
serial ports, causing them to be reversed ).

Hope it helps,
