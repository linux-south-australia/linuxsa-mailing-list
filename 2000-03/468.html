Re: /dev/fd0 again

Hi- a few hints re floppies

-IF you have a dos formatted floppy then typing mount /dev/fd0
**SHOULD** mount the floppy automatically as umsdos. ie/ no need to
include the filesystem type!
-IF you have an ext2 formatted floppy then typing mount /dev/fd0
**SHOULD** mount the floppy automatically as ext2. ie/ no need to enter
fs type
-IF you fdformat /dev/fd0h1440 and DON'T ALSO do "mke2fs /dev/fd0" you
will get the error described below! :)
-IF you try & mount a dosfloppy using the fstab line described hereunder
you will also get the same error!

>ie  (root) mount -t ext2 /dev/fd0 /mnt/floppy
> 
> It then comes up with - wrong fs type, bad option, bad superblock on
> /dev/fd0 or too many mounted file systems.
> Here are my fstab entries- /dev/fd0     /mnt/floppy    ext2 >user,noexec,nodev,nosuid,rw,noauto,sync,unhide  0  0
