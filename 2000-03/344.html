Adam Hawes <adam@overfiend.ace.net.au> wrote:

>I want to be able to break up the mail into one mailbox for each list I
>am subscribed  to (or by topic, with all linux going to the linux
>mailbox, all winblows going to another... etc).
>
>Is it possible to do this with fetchmail/procmail running as a normal
>user (me), and not the superuser.

I setup fetchmail/procmail/mailstat by first reading the information
at this URL
   http://www.faqs.org/faqs/mail/filtering-faq/

I've just tested that the following setup works when run as a normal
user.

BTW this works on my Redhat 6.0 machine.  It may not work without a
.forward file if procmail isn't the default delivery agent.

Fetchmail
=========
create a .fetchmailrc  file in your home directory like this
poll myisps.mail.server
  protocol pop3
  username  usernameAtISP
  password  passwordAtISP
  is  username here          #name of the user on your machine

Execute this by entering "fetchmail" or "fetchmail -d 300 -t 60"

Procmail
=========
create  a .procmailrc file in you users home directory like this.
It creates a rule for filtering LinuxSA mail into a file called
appropriately LinuxSA

BTW: the $HOME/mail  directory is where Pine looks for mail. the dir
you use may differ if you don't use Pine.

VERBOSE=off
MAILDIR=$HOME/mail      
LOGFILE=$HOME/procmaillog
:0
* ^Resent-Sender: linuxsa-request
LinuxSA                        


Mailstat
========
check your procmail log file with
  mailstat ~/procmaillog
check your previous procmail log file with
  mailstat -o ~/procmaillog


Sorry if this isn't what you're looking for.
