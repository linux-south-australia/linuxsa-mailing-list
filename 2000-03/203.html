Re: SysV Init versus BSD-isms

> While most of the versions of Linux I have used use SysV Init, I am lead to
> believe that some versions use the BSD approach of rc.local and etc.
> 
> Can anyone tell me which versions use the BSD approach, and where/how you
> would start Samba in such a version of Linux?

Long throw back here...

I did a talk on this back in 1997...

	http://www.linuxsa.org.au/meetings/1997-07/init/

But the upshot is that in a BSD style init system, there is a file which
basically contains all the work that needs to be done for a particular 
run level.

So in effect all you do is modify rc.3 (or similar) and add the line to
start smbd/nmbd in the appropriate fashion.
