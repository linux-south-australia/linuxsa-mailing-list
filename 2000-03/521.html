You will lose your data if you change the partitions. Each operating
system views the partition table differently. I have noticed that SCO
Unix reads it backwards, Solaris likes to use a different disk geometry
to the rest of the world, Linux will give them to you in the order they
were created and Windows FDISK won't allow you to create more than one
without them being on extended partitions. Windows also reads the
partitions in the order they appear in the partition table. I think
Windows moves the partition table entries around if another operating
system makes a partition that appears in the wrong order (get the
interface nice and THEN concentrate on the functionality). In short, I
wouldn't bother. ;-)
