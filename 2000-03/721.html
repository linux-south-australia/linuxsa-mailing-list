best public access for private database?

To any database/php3/encryption experts:

It seems to me that if I have a private database
(some personal and company accounts, for instance)
which I want to access from several physical sites,
then there are a couple of ways to use my
PHP3/PostgreSQL interface private:

----------------------------------------------
1. Encrypt the HTTP over the net using SSL or
OpenSSH or something.

    Postgres <-> httpd <- Z -> web browser.

This seems to require me to work out how to
set up the SSL module for Apache.

----------------------------------------------
2. Encrypt the link between a local
copy of all the PHP3 scripts and the PostgreSQL
back-end.

    Postgres <- Z -> httpd <-> web browser.

This seems to require me to work out how to
feed port 5432 over an OpenSSH link.

[Z denotes the encrypted link over the net.]

----------------------------------------------
Does anyone have any idea of what the "standard"
way of doing this would be?
There must surely be other people who have private
data which they want to access over the net
with the convenience of PHP3 scripts.
There are questions here about the relative
difficulty, the relative functionality, and the
relative speed performance.

Regards,
Alan Kennington.

------------------------------------------
PS. As an exercise, I put some of my database on the net
at http://topology.org/php3/b1.html as an exercise
in PHP3/PostgreSQL. A lot of work, but a great pleasure
when it finally works.

PPS. Geoffrey said in the last linuxSA talk not a
thing about his _relative_ experiences with MySQL
and PostgreSQL. People say that PostgreSQL is more
functional but slower - but _how_ much slower?
