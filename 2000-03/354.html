Re: Splitting a variable length file

Richard Sharpe wrote:

>> Does anyone have any suggestions as to the best way to accomplish this
>> file splitting?

> firstpart=`wc -l somefile`
> firstpart = `expr $firstpart - 43`
> head -$firstpart somefile

My only problem with this is that "wc -l" is returning the file name
as well as the line count, and when I desk checked the results it
caused expr to blow up.  Whether it's a GNUism or POSIXism (as I don't
recall coming across the problem before), I couldn't find anything that
would let wc just emit the line count, which seems kind of broken for
a Unix utility.  That's why I piped the output of wc into awk and
arithmetically attacked the result there.
