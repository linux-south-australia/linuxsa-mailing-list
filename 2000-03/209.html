OpenLDAP is quite easy to setup (I can do it ;-)). It's also quite small
and doesn't seem to eat resources much. The thing about OpenLDAP is all
of the extras that have been written for it. Like the Authentication
modules for Apache, PAM and squid. Samba can make use of LDAP as another
database to store accounts in, it can be set up to be an E-Mail address
book for Various new mail clients. You can even have a different
database backend (or write your own). The ldbm database is fine for most
things, but you may find that using a different database is better for
different things. I like the flexibility.
