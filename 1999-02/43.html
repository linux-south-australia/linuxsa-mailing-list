Re: how to move kde without breaking it!

On Wed, 3 Feb 1999, romana wrote:

|hi..we have a hdd with / nearly full....(78%) capacity, which is a pain
|when installing staroffice..ie, we cant. we allocated 106 mb to it,
|after reading the red hat install book,
|but now, after installing kde (which we love), it is nearly full, so how
|do i move the kde default dirs,(/opt/kde), to /usr/opt/kde/, without
|having to reinstall?

Space problems like this can be partially solved with careful use
symlinks. Just move /opt somewhere there is a lot of space. Then
create a symlink at /opt to the new location, and kde will still
think it's installed in /opt. Any new install under /opt will
take up space where you moved it to.

For example:
mv /opt /usr/opt
ln -s /usr/opt /opt
