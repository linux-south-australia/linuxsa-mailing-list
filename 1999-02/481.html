Re: patching kernels(?)

On Mon, 22 Feb 1999, Robert Mibus wrote:

> >Being a good net-citizen (and it's faster) I have down loaded 
> >the kernel patch from 2.2.0 to 2.2.1, only there are no instructions 
> >on how to make the 'patch'.
> >
> >(man patch talks about a -pnum, but with no explaination of what this
> >is)
> >
> >Q1: where do I put the patch file?
> >Q2: what command do I use to make the patch? 
> A1: i guess in /usr/src
> A2: cd /usr/src; patch < kernel-2.2.1.patch
> [or whatever the patch is called]
> 
> if not, try /usr/src/linux instead of /usr/src.
> 

Put the file in /usr/src/linux and type: scripts/patch-kernel

This will run a kernel script to patch the kernel and also clean up any
rejects. (it runs patch -p1 < patchfile)
The great thing about this script is that it works with compressed patches
as well.

Then copy your config file (.config) somewhere safe and run: make mrproper
Copy back your back config file and run: make oldconfig
This updates the config file for any new changes and prompts you if
anything has changed. All done!

> >Also, parallel ports seem to be disabled by default in the 
> >2.2.0 and 2.2.1 kernel configurations,  and it seems that very little
> >can be compiled into the kernel before i get complaints about it 
> >being too large (at the make zImage stage).
> 
> do a 'make bzImage' instead.

You should use modules more!
Although bzImage will make the kernel still, it uses more memory for data
allocation. Modules are much better solution.

Simply answer Y to all module questions under Loadable module support.
Then anything that you don't use continually such as CDROM drive can be M
instead of Y and automatically loaded when required.

To complete the module setup. Add 

*/5 * * * * root /sbin/rmmod -a

in /etc/crontab

Which automatically removes unused modules after 5 minutes.
