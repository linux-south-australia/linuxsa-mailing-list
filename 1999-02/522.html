The file might be corrupted -- If it's important, boot single user, zero
/var/run/utmp, then reboot multiuser.

Corruption can happen if some applications write utmp/wtmp records
which are a different size to other applications.  This can happen
if you've compiled and installed app A, then upgraded your include
files (which is where struct utmp is defined), then compiled and installed
app B which also uses utmp structures.  When app B writes a utmp
record app A won't necessarily be able to understand (all of) it, and
vice versa.

This used to be quite common when I ran MINIX and upgraded all the
time :-)  It also happens when include file maintainers say, "Hey!
Let's make this 64-bit clean!" and change all the short ints into
normal (long) ints.  It may not be what's happened here, but it's 
worth investigating.
