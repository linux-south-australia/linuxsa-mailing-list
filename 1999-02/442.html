Redhat 5.2:Sound problem

Thanks, all, for your help with my undermentioned >sound problem. It is
now up and working.

> I have one problem ,though, and that is to get my XPlayCD 2.1 up and
> working as  "user" in Redhat. It works OK at root. Is this a permission
> problem?
> If it is I have a problem knowing which file permission to change. I've
> tried changing permissions for /etc/rc.d/init.d/sound and
> /var/lock/subsys/sound as these were the only two sound files the "Find"
> utility came up with but  without success. I've also tried to su to root
> but I don't know the console command for starting XPlay if in fact that
> is a valid approach.?

Try chmod 666 /dev/cdrom (as root). If you don't want to do this for
some reason, you could do a chmod a+s /usr/X11/bin/XPlaycd (or wherever
it's located: try "whereis Xplaycd"), making the effective user for
everyone running Xplaycd root, though this is probably worse security
wise!

Dave Lawrence
