Trouble with modules

Hi Guys

I am having a lot of trouble with getting module support working in Redhat
5.1.

To demonstrate the problem I removed all scsi support from my kernel. scsi
support is by default compiled as a module. All this is done with a plain,
install everything, new install of redhat 5.1

I run the following commands

make menuconfig
(remove scsi support entirely)
make dep clean modules modules_install zlmage
depmod -a (depmod returns no errors)
edit lilo.conf to use new zImage
reboot

There is a line when the computer reboots that says 

Finding module dependencies .. can't open /lib/modules/preffered/modules.dep
kerneld starts Ok next line

and when I log in and type depmod -a I get lots of 

/lib/modules/2.0.34/scsi/allscsi.o_files unresolved symbols.

My questions are: 

is there a decent reference I can read to understand how modules work and
how to configure them?

do not work.

I have read the modules install how to but it is pretty slim. I have read
the man pages what I really need is a trouble shooting guide of some
description.

Any help would be appreciated.
