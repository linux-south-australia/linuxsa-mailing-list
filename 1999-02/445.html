Re: Sendmail question

On Sat, 20 Feb 1999, remnant wrote:

> Hey all 
> Heres a tricky one (well, with my knowledge of sendmail it is  :P).
> If I have a POP account setup on my sendmail server, lets say foo@bar.com, 
> and I create an alias called foo that redirects its mail to
> johnc@slashdot.org, whats going to happen?

An alias is another name hence all mail should be redirected so nothing
should end up in the pop account.

> Will the email get sent both to the alias address and the POP address?  
> Or, to achieve the same result, is it possible to create two aliases
> under the same name that send to different addresses?

I'm not sure using aliases but you could use a simple procmail rule.
ie: 

File .procmail of users home dir

:0 c
! johnc@slashdot.org

Here the c stands for carbon copy and the ! means remail
This way it a cc would get sent to the slashdot address but the message
would also remain accessable via pop.

Hope that helps
