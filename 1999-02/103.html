On Fri, 5 Feb 1999, Glen Cunningham wrote:

> Help please,
> As a newbie (in this area) i would like to big-note myself at our next
> local LUG meeting by demonstrating a ppp connection to an ISP using xisp
> (nobody here has done this yet!). This is what i have so far --
> 1. RH5.2
> 2. Modem is set to /dev/modem, a link to /dev/cua1, that's where it is.
> 3. Compiled and installed xisp -- it seems to work OK.
> 4. Set up the Options in xisp.  This is where i am going wrong (i think).
>      After 3 days of dealing with my ISPs helpline (both 'phone and email)
> has eventualy got roughly this answer "We don't know, but if you find out
> then let us know.)  So, cobbled together what i could extract from a
> working 'doze connection , but clearly something important is missing/wrong
> because when i click 'Connect' in xisp, the correct number is dialed, but
> after initial handshaking, the connection is broken (before anything useful
> happens).

There is a few things you must do to get it to connect.

1> set the permissions on pppd correctly.
It sounds as though you have set all the permissions up correctly as it
dials.

2> Check if your isp requries PAP authorisation. If it does then select
pap in the isp setup screen. Try again

3> If they don't then try this. Under options -> dialing & Logon  enter
the following in the two automatic login boxes:

ogin:				%U
word:				%P

The %U & %P mean send the username & password that you entered in the
account information screen.

This fixed my problem at senet. As I too would connect then get cut off.
