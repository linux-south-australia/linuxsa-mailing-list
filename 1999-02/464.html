Routing in a PPP network

Hi,

I am about to set up a hub-and-spoke network using PPP, where the hub site
will call out to all the nodes on the rim.

I need to set up the routing appropriately, so things work.

My network layout looks like this:

  172.30.1.0/24 is the hub
  172.30.2.0/24 is one network on the rim
  172.30.3.0/24 is another network on the rim, and so on

  Each node on the rim will have an IP address on its PPP link that is
  172.30.254.n where n is the network number of the network on the rim

At the rim nodes, routing is fine, as PPP can set up a default route.

At the hub, however, there will be lots of PPP connections out to the rim
networks, so a default route will not be suitable. What I need is to be
able to insert lots routing entries, one for each rim network.

I tried adding stuff to /etc/sysconfig/static-routes for each ppp link, but
this firstly does not work, and I could never ensure that a particular rim
node comes up on a particular pppn device.

I could add something to /etc/rc.d/init.d that would install all the
routing table entries along the lines of:

    route add -net 172.30.n.0 netmask 255.255.255.255 gw 172.30.254.n

Can anyone suggest a good technique for this?
