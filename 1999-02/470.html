>hi,

tighter
>as you go. try these settings...
>[global]
>workgroup = workgroup
>security = user
>allow hosts = your ip address here

Generally, in a closed network, allow hosts is not needed, and adds clutter.

>[your share directory name here]
>create mode = 0665
>path =  /your/share/name
>writable = yes
>valid users = your login name on the linux box
>write list = your login name on the linux box
>public = yes

This of course deletes the guest account, and Samba uses the guest account
when opening IPC$. IPC$ is used in browsing servers, AFAIK.

It would be interesting to see a log level 5 log from smbd when you try to
browse the server from your Win98 box.  It should do a NetShareEnum, but
before that it should do a TconX to IPC$.

It would seem that authentication is failing. Does your guest account
exist? What username is being sent?

What version of Samba are you using?
