Re: Turning off unnecessary services

> I can shut down all the services I'm not providing by following the
> instructions on /etc/inetd.conf , but how to NOT start the services that
> aren't controlled by the inetd super server, like httpd?

In /etc/rc.d/rc.* there are links to files in /etc/rc.d/init.d

the * is 1, 2, 3, 4, or 5 from memory. If you use Redhat, you
can use the program 'tksysv' as root to remove the services that you
do not require from starting. I recommend pruning out all that you
are not going to use (not just networking stuff either) to create
a Linux system that uses the minimum amount of memory.

I cannot tell you about GUI tools for other distributions for this.
The only way I can suggest it removing the links in /etc/rc.d/rc.*
to the services that you do not want to start. There are start and
stop links there, so remove them both.

Maybe somebody could suggest a more elegant way of doing this without
Redhat's tksysv tool. I think that Linuxconf may do it, but I have
never looked into it.

hope this helps...
