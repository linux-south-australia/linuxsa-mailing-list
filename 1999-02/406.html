Make sure you have the latest versions.  There are some libraries from a 
full win95/98/nt install that *ARE* needed.  It will have a GPF if it doesn't
have them.  I found that a version of the NT MSVCRT.DLL, comctl32.dll,  shell32.dll
are needed for it to run.

When it is trying to connect - use 'netstat -at' to get a list of connections 
from you machine.

There should be a line that looks like 

tcp        0      0 myhost:3499            203.17.185.180:2202     ESTABLISHED 

The important bit is that there should be a connection to 2202 somewhere with
the state being ESTABLISHED.  If you are getting a SYN_SENT state you are not 
getting through your firewall/proxyy.

It does work, and you can transfer funds too!  BTW you can use 
wine -managed netbank.exe to let wine be friendly with you window manager.
