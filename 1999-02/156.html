Re: XF86Config question

Benjamin Close wrote:

 > 	Does anyone know of a way to let a user have an alternative
 > XF86Config file without giving them root privilages?
 > One user wants a virtual desktop of 1024x768 and the other 800x600.

Solution:  tell the one who wants 800x600 that you're exceeding his
expectations :-)

I don't think it would be a good idea to give users the ability to 
frob their own XF86Config files simply because there's so much scope
to damage hardware (at worst) or lock up the machine (at best).
The X server also runs setuid root, so you might be opening yourself
up to buffer overflow bugs through stuff fed into the config file by
users.  [ unsubstantiated conjecture ]

 > Both
 > users log in via the console but one knows what they are doing, the other
 > has no idea hence I don't want to give them root permission (indicated
 > as one way to have seperate XF86Config files in the man page).

Ok, here's another:  You can run more than one X server at the same
time, right?  (they come up on separate virtual consoles).  Run one
at 1024x768 and the other at 800x600 and show your users how to use
CTRL-ALT-whatever to switch between them.  Use the -xf86config command
line switch on the X server to start it, and use xdm to provide login
windows (you need to do this because -xf86config only works if the
server is started as root).  See XFree86(1) for more details on
command line options.
