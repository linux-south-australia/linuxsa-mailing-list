That would work well.

In addition, the system you have handles large files by default. Try the
command "ulimit -a" and look for an entry "filesize". Check your kernel -
if you built it, did you enable large file support?

In general you have to do something special to make a Linux based on the
2.4 kernel not support large files via the 64-bit interfaces. Most, maybe
all distributions have worked out the correct kernel and glibc and
supporting libraries to supply.

There is a persistent myth I keep coming across that ext2 (not ext3!) has
a 2Gb filesize limit. This is incorrect. The limit is 16Gb, which I would
think is quite large enough for a filesystem that requires an fsck to
recover from inconsistencies. More capable filesystems such as XFS and JFS
have extremely high limits.
