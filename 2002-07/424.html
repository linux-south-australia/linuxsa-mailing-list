Re: Controlling Samba with xinetd

On Tue, 16 Jul 2002, Dan Shearer wrote:

> On 16 Jul 2002, Anthony Symons wrote:
> 
> > I thought each client got its own smbd process on the server anyway?
> 
> It does. This is not an inefficient approach (on any Unix that does shared
> copy-on-write) although a lot of people seem to think it is.

As a case in point, a guy in Alberta did a test recently of a system he 
was going to take to a customer. Using cifs-load-gen, he simulated 350 
clients starting up and copying 200 MB to the server.

He said that CPU didn't get much above 50%, and about 700MB of memory was 
used, but that only 100MB of that was taken up by the 350 smbd's unshared 
memory. Copy on write was working real well in this case.
