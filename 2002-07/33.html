>Further to Glen's reply :-)
>
>Watch out for the log level or debug level parameter. It can 
>reduce your 
>throughput drastically. Set it to 0 for best performance. Set 
>it higher 
>only when you are debugging problems.

Err, yes, I can vouch for that.  Had some minor issues with one
workstation after upgrading to 2.2.5 last week, so I turned up
the debug level to 10 for a bit to try & work out what was going
on, worked out the problem ok, but guess who forgot to set debugging 
back to 0............... :P

Didn't realise until someone from the finance dept. rang up wondering
why a report which usually takes a few minutes was taking 25 minutes 
and counting.  The server generated 250mb of samba logfiles in just 
under 3 hours, and each smbd process was killing the CPU and using up
ridiculous amounts of memory........sigh.......lesson learned.........
