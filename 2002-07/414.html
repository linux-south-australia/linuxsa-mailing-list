>I'm not sure what happens if you add a later service pack that may
>install past the limit though (BSoD I would imagine :)), so maybe
>separate boot/data partitions are the better way to go. 

Separate boot & data partitions is ALWAYS the better way to go with NT.

Our standard NT server setup was/is always a partition <=4GB for install
and system files only (ie: C:), then after install create 2 more partitions,
one (D:) for Applications and one (E:) for Data/Shared space.

This kind of setup will save much heartache if something does go AWOL on
your server such as a service pack screwing up your system or such like.
Of course having a separate drive/raid setup for your applications/data 
space is a good idea also.
