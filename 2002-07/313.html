Variables from URL not being passed to PHP

This is really bizarre. Perhaps it's because it's 01:15AM and I'm
still coding, but for some reason, all of the PHP that /I/ write
doesn't seem to want to pass variables from the URL.

For example, if I punch http://keetoo.alfred.cx/test.php?var=asd, for
some unexpliciable reason, the following script won't pick up the
variable $var:

<?php
printf("The contents of the variable is %s", $var);
?>

However, phpMyAdmin, installed in a directory alongside test.php makes
heavy use of variables contained in the URL, yet it seems to potter
along quite nicely.

What the bloody hell is going on? It's never happened until the last
installation of PHP. What I don't get is the inconsistency -- one
script has the problem, but the other doesn't.

I've tried stuffing around with file permissions and file ownership
(not that it should have any difference, mind) to no avail.

If anyone has got a pointer that could help, I'd be most greatful. As
I said before, it could be all to do with tiredness, but I've pulled
it apart as logically as I can, and I'm still having no joy.
