Re: postfix auto bcc

On Wed, 10 Jul 2002, Richard Russell wrote:

> mail I sent from one client while using another client. I've cosidered
> trying to make each client save its own sentmail in a common IMAP
> folder,
   : 
> anyone already do this? -- I'm thinking that I want
> an auto-BCC or something... any ideas?

For IMAP clients, the term is Fcc, for File Carbon Copy.

If you want to do this on the server side, it is a mail filtering function
rather than a mail transport function, ie procmail and friends.

> but I have a feeling that it may be easier to get postfix to
> do this for me... 

I think you'll find it harder. The thing about doing it server side is it
gives you more control and consistency. But since Fcc comes with every
IMAP client you're likely to use, and since there are all sorts of corner
cases to consider about email flows why not just fill in the Fcc field
when setting up a mail client at the same time as you set up the mail
server field?
