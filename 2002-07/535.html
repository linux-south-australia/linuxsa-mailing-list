> an initrd is an initial RAM disk.  My understanding is that a standard
> kernel can only be a certain size (about 1 MB?), but it may be necessary
> or desirable to have other drivers available at boot time (before the
> root filesystem becomes mounted).  To do this, a RAM disk image can be
> created that contains the necessary kernel modules (such as SCSI disk
> drivers which may be too big to fit in the monolithic base kernel), then
> your bootloader is instructed to load this RAM disk at boot.
> 
> Your existing initrd probably contains kernel modules from an older
> kernel, and it may be that there is an incompatibility between the old
> modules and the new kernel (maybe different symbols), or that your new
> kernel is not configured to accept modules from different kernel
> versions.
> 
> To create a new initrd for your new kernel, read the mkinitrd(8) and
> initrd(4) manual pages.  The Red Hat customization guide[1] might also
> be of use.
> 
> I hope this helps
