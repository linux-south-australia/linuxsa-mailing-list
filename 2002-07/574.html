Linux Security - Know Your Enemy - The Honeynet - How/Who Roots ALinux Box.

I found a good source on intrusion monitoring, and I thought I would 
pass it along.

> http://www.honeynet.org/book/

 
I read the book last night, and I came away with the impression that the 
following happens when an attack occurs..

1. The blackhat scans your system for a "vulnerability"
2. ' ' gains root by exploiting the "vulnerability" with an exploit script
3. ' ' telnets to the server and uses accounts created by the exploit script
4. ' ' checks to see who is on
5. ' ' edits the log files
6. ' ' replaces system binaries with trojans
7. ' ' downloads rootkit and installs it

Seems with all of the "vulnerabilities" out there, and how often scans 
occur, it is a wonder that most Linux boxen aren't "rooted."

I highly recommend a close look at this site, and their whitepapers.

However, a few things surprised me, and one thing isn't clear to me, but 
I would like some answers if people know them...

Surprised me:

1. The ease and speed in which the Red Hat Linux 5.1/6 boxen were "rooted."

2. The wide variety of operating systems the blackhats claimed to "root."

What is unclear to me:

1. I thought that if you have local IP numbers set up i.e. 192.168.x.x 
or 10.1.X.X behind a router with a masquerraded local IP number, that 
this kind of thing couldn't happen. Am I incorrect?

2. If the use of local IP numbers have little or no effect on intrusion 
deterance, are there any other methodologies that can keep an intruder 
from reaching the vulnerable computer?

Any thoughts?
