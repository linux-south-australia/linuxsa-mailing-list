Re: linux uptime

On Wed, Jul 17, 2002 at 10:46:08AM +0930, Glen Turner wrote:

 > Red Hat Linux has had now many kernel upgrades (each requiring a
 > five minute reboot) this year?
 
Following the philosophy that it's only necessary to install
patches which fix problems which you are actually experiencing
helps a -lot- here.  For the overwhelming majority of users,
that'd reduce the number of kernel upgrades required over the
last year to zero.

[ keeping in mind that it's basically never necessary to reboot
  to effect an upgrade of user-land utilities.  Anyone who bounces
  their box to fix the latest Apache or sshd vulnerability really
  needs to understand that they'd get higher system availability
  figures by applying a bit more thought to the problem... ]

 > Note that running redundant servers usually moves the problem to
 > another platform, which isn't a great help.  DNS is an exception
 > here --  the redundancy is designed into the protocol and is
 > implemented by the client.

If your safety-of-life system is so important, redundancy will
be designed into that too.
