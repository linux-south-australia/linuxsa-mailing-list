> Some programs arnt compiled with > 2gb support.

Hint for those compiling programs, by default, your program will have 32
bit file offsets and 32 bit system calls. Even if your system is fully
LFS-compliant, with all the correct libraries. 

However if you define _LARGEFILE64_SOURCE before you do any #includes, you
get 64-bit everywhere plus a set of new syscalls. I am pretty sure that
this behaviour is specified by Posix. This is where some distros come
unstuck, so it is a question of distro QA not whether the
kernel/filesystem/glibc can cope with large files.
