Re: grub and initrd

On Fri, 19 Jul 2002, Michael WARDLE wrote:

> > If someone can help explain why the old initrd doesnt work and where I
> > should start looking in order to create a suitable one, I'd be
> > grateful.
> 
> an initrd is an initial RAM disk.  My understanding is that a standard
> kernel can only be a certain size (about 1 MB?), but it may be necessary
> or desirable to have other drivers available at boot time (before the
> root filesystem becomes mounted).  To do this, a RAM disk image can be
> created that contains the necessary kernel modules (such as SCSI disk
> drivers which may be too big to fit in the monolithic base kernel), then
> your bootloader is instructed to load this RAM disk at boot.

It is not really an issue of kernel size, since with bzImages, they can be 
quite large. It is an issue of modular kernels that need to boot of 
anything that is not supported by the default kernel that most distro's 
ship. Using an initial ram disk approach, distro vendors, and you too, can 
avoid rebuilding the kernel to support SCSI and other disk types.

The problems, of course, arise at the point where you try to boot the 
system from hard disk for the first time.
