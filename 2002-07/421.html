Re: Controlling Samba with xinetd

On Tue, 16 Jul 2002, Freeman, Peter (ERHS) wrote:

> Was doing some research last night on high availablilty - 
> led me to a thought which I have seen around before but 
> decided it was not necessary.
> 
> What are your thoughts on using inetd to control the Samba daemon?  
> This (ought to) fix problems with an unstable process running as the 
> user and crashing.  Usually a restart of Samba is required to get it 
> going again

If you are seeing instability and Samba crashing, that is a bug, and we 
would appreciate seeing reports on it.

There is a fairly high startup cost for the first daemon that should not 
be inflicted on all users, so starting smbd out of [x]inetd is not often 
used.

If the per-client process crashes, the client will reconnect, so this 
should not be an issue, modulo any bugs you are seeing.
