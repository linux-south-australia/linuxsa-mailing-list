Samba update times

I'm seeing some samba weirdness and wondered if
anybody had thoughts on it. I am developing for an
embedded platform at the moment & have the following
setup:
- Source tree on win2k machine
- gcc cross compiler on linux machine with the source
tree mounted as smbfs
- rsh from win2k box into the linux box to run gcc

The problem is that when I edit the files under win2k,
it can take considerable time before the changes are
seen on the linux machine. Sometimes, I need to
alternately re-save the file (on win2k box) & then cat
(on linux box) before the changes are seen. Also, if I
gcc the file under linux without ensuring the file has
been updated, the file is replaced with the version
that linux was originally seeing & I loose my changes.
Does that make any sense to anybody? Anybody have any
clues? I'm talking text files of a hew 10's of kb
taking upto a minute or two to be synchronised accross
the share. I thought changes would be pretty speedy.

cheers,
