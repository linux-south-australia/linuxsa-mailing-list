It's trying to compile against a different kernel than the one that you;
look inside quickcam.sh itself and try to work out which kernel it is
actually attempting to build against.

> mod_quickcam.o: unresolved symbol video_proc_entry
> mod_quickcam.o: unresolved symbol kmap_pagetable
> mod_quickcam.o: unresolved symbol video_register_device
> mod_quickcam.o: unresolved symbol video_unregister_device

When you write a driver (or program), it can call functions or
procedures (or code) that is not part of that actual program. These
functions could be part of a standard library or kernel system calls for
example. The driver module is stating here that it is attempting to use
a function or some type of symbol that it needs to "link" (link -->
basically to work out where the function or symbol is actually defined
and use it) but it can't find it.

It's almost certainly got something to do with the compilation error
above.

Try:

depmod -a
ldconfig

(but I don't think the latter will help)

man depmod

bring it to a LinuxSA meeting.
