Re: debian/woody MySQL root password problem

OK, tried that - to no avail. I think this is a debian specific thing. It
still refuses to grant permissions even when the grant tables are turned
off.

Two different methods (one more debian than the other, but both
approximate the method described at the URL below):

sunya:~# mysqld --skip-grant-tables &
[1] 3445
sunya:~# mysqld: ready for connections
mysqladmin -u root password #######
mysqladmin: unable to change password; error: 'You must have privileges to
update tables in the mysql database to be able to change passwords for
others'
sunya:~# mysqladmin -u mysql password staghorn
mysqladmin: unable to change password; error: 'You must have privileges to
update tables in the mysql database to be able to change passwords for
others'


sunya:~# kill `cat /var/run/mysqld/mysqld.pid`
sunya:~# mysqld --skip-grant-tables &
[1] 3526
sunya:~# mysqld: ready for connections
mysqladmin -u root password #######
mysqladmin: unable to change password; error: 'You must have privileges to
update tables in the mysql database to be able to change passwords for
others'
