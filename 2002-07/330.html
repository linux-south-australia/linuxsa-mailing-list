> Hello, my service apache 1.3.12-2 gives error messages 
> following when I restart http service.
 
> # /etc/rc.d/init.d/httpd restart (enter) 
> shutting down HTTP                              FAILED 
> starting HTTP                                   OK 
> # /etc/rc.d/init.d/httpd status (enter) 
> httpd dead subsys locked.
> when i test my web page the browser give me this message.
> "Internet explorer can not open Internet site 
> http://www.togo-imet.com connection with the server was 
> initialized. "

Firstly, upgrade Apache, anything prior to 1.3.26 is vulnerable and a
security risk.

Secondly, before doing a restart, try doing:

ps -ef | grep http

to see if you have any apache processes running. If they're not
running the restart script will report FAILED on the shutdown because
it can't shutdown a process that isn't there.

If it is running, but restart still reports FAILED, try doing a "stop"
instead of a "restart" then check to see if it's still running.

If that didn't stop apache, it might be worth killing all the httpd
processes and deleting /var/run/apache.pid (might also be httpd.pid)
then try to start apache.

If it doesn't start properly, try:

/usr/local/apache/bin/apachectl configtest

(I'm assuming you've upgraded to apache 1.3.26 by this stage)

[BTW, the 2.0.3x branch seems to be quite stable and works well on
multi-processor systems.]
