On 23 Jul 2002, Michael Davies wrote:

> On Tue, 2002-07-23 at 15:55, Glen Turner wrote:
> > 
> > BTW, OpenOffice is slow because it is running over Wind/U,
> > a API that lets programs run above Window or UNIX without
> > rewriting.  This is a neat trick, but it makes OpenOffice
> > an amazing time and resoruce hog.  It's only the lack of
> > a viable alternative which makes OpenOffice at all
> > attractive.
> 
> Are you sure about Wind/U?  That isn't Open Source last time I checked -

You're quite right, it's not Wind/U at all. The abstract interfaces are
not provided by any one toolkit as far as I know, although I mostly took a
look at lower-level stuff (and backed away hastily :-) See
http://gsl.openoffice.org/ for the Graphics System Layer,
http://dba.openoffice.org/ for the database layer, and of course
http://udk.openoffice.org/ for the Universal Network Object dev kit.

Incidentally I found the URL for the bridges stuff I was grumbling about
earlier, see http://udk.openoffice.org/common/man/bridge.html and then
look at the implementation for your particular compiler. I welcome any
alternative interpretations to mine so far, which is "this is specific to
the particular sub-version and optimisation level of your C++
compiler". Or am I just feeling grim tonight?
