Linux uptimes are slowly getting better, but are nowhere near good
enough.

Let's say you need to run a safety of life service.  The typical figure
for that is 5 minutes of "scheduled outage with no service impact" per
annum.

Red Hat Linux has had now many kernel upgrades (each requiring a
five minute reboot) this year?

Note that running redundant servers usually moves the problem to
another platform, which isn't a great help.  DNS is an exception
here --  the redundancy is designed into the protocol and is
implemented by the client.
