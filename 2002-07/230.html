Re: [OT] MacOS for PC

On Mon, Jul 08, 2002 at 05:19:11AM +1000, Andrew Pullin wrote:

>     What do you mean by "make it work"? What I meant was If
> a USER wanted to send mail, then run the mail client, if
> they wanted to surf, run a Browser, if they wanted to
> balance the cheque book, run a spreadsheet or if they wanted
> to write a letter, run a word processor. These are everyday
> USER things, and I would have to say that if the machine was
> set up ready for a USER, then M$, Mac or a suitable Linux
> Distro would be more or less identical. All these systems
> are GUI based or menu driven and any user can pick it up in
> next to no time. KDE and Gnome were designed to be Windows
> and Mac like to make it easy to change over. Gnome used to
> have a Windows mask that even changed the toolbar to a
> "Start" button (it may still but I have used KDE for a while
> now).

"Make it work" refers to a scenario where a user purchases a machine
and decides that they want to do something with it (i.e., send mail,
surf, balance cheque books etc.). If you've just purchased a machine,
you've got to get it from the box and to a working state on your desk
that will allow you to do all that aforementioned fun stuff.

If the machine has been setup beforehand, a Linux or Windows box would
be roughly the same in terms of ease-of-use. Linux, perhaps, may have
a very slightly steeper learning curve, but I'd imagine most people
could get around it.

A Mac, on the other hand, fresh from a restore, will step the user
through everything. It introuduces them to the system and gives them
an opportunity to understand what they're doing.

If you've given the machine to them before that step, I'd argue that
they're [Macs] easier to navigate anyhow.

>     I think that you may have been thinking of something
> like Installing software, which is an ADMIN task, and in the
> particular scenario we were talking about, the parents
> (USERS) wouldn't be doing this, the son (ADMIN) would. IF
> this were the case, then I think that many people in this
> position would likely shy away from these type of tasks
> which ever OS they used. As far as fixing Linux, the very
> nature of Linux is that non root cannot break something
> important, they can break their own account, but not the
> system. In this case, whoever is ADMIN can easily fix it
> without a full reinstall by deleting and recreating the
> account at worst.

Yeah, that's true, but users need to be able to fix certain problems
without someone holding their hand. The fact that the machine will be
administered by a person who has a portion of a clue does not mean
that you don't need to worry about keeping administration easy.

I understand "the very nature of Linux". I understand that. But, we're
not just talking about Linux. I think Apple came into the equation
somewhere.

In the longer term, the Mac is a more sensible option. If "Dad" comes
home with a new printer, scanner or mouse (as Fathers tend to do),
it'll just work when you plug it into a Mac (or a Windows box, for
that matter). I'm affraid I can't say the same for Linux in a similar
situation.

>     In the case of a major break, the ADMIN person would do
> the reinstall anyhow, so what difference does it make to
> which OS you use in the first place. I think the original
> point was that the users of this system were unlikely to
> play and break anything, but even if they did, the admin
> probably wouldn't trust them to just stick a CD in and press
> a few buttons. So I still think that for this application,
> Linux is probably the best solution. The only thing I would
> add is that in this case M$ is probably the worst solution
> in that a USER has complete admin like access (unless you
> are using NT or such like) and all it takes is one of those
> "delete this important file because it is a virus" emails,
> and you could be in real trouble with a complete novice with
> admin like access. Well that's my 2c worth.

In the case of a major break, you don't want to have to burden your
administrator with hours of unnecessary stuffing around just because
you can. Stick a CD in and press a button. That's how it should be.

Linux may be a reasonable solution, I don't doubt that. However, when
you consider the merits of the other solutions, I think the Mac comes
out in front.
