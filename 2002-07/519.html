RE: WinXP Problem Connecting to Samba Servers?

>Here's a setting that might fix problems connecting to samba 
>servers from
>XP.  Control Panel, Administrative Tools, Local Security Policy.  Local
>Policies, Security Options and under there is a 'Microsoft 
>Network Client:
>Send unencrypted password to third-party SMB servers...

This setting is also in Win2000, however it's better to leave passwords
encrypted if possible, for obvious reasons if you've ever played with
packet sniffers......

The most common hassle with Samba and WinXP clients is the digital 
signing which XP has turned on by default and can be fixed with
a simple registry edit (the patch released with Samba sources only 
fixed one entry in the registry, there are a couple different places
to change it)

Unfortunately sending unencrypted passwords won't fix this problem.
