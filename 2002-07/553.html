traceroute v tracert

Hello all,
 
I was bored and decided to trace the route to yahoo from my work computer
which is running windows95. It took 45 seconds. 
 
I did the same test from my linux computer here which got its ip address
from the same dhcp server as the windows computer and it is in the same
subnet, it took 14 seconds. 
 
Note: 
-both computers are sitting next to each other.
-both counted the same number of hops
-i did the test twice
-i was bored!  :)
 
Thinking that the windows computer was slow becuase it was having trouble
trying to resolve the names, i "ping www.yahoo.com" and there was no
noticable delay in resolving the name. 
 
Why such a big difference?
