Re: Calculating Netmask Values

On Sun, Jul 07, 2002 at 02:40:17AM +1000, Andrew Pullin wrote:
> Hi All,
>     I have a rather annoying problem with a dual boot
> desktop at present. It works fine with the Redmond Product,
> but when I boot Linux (Suse 7.2 Pro) I discovered I couldn't
> surf with Netscape. On further investigation (I initially
> thought it was a DNS problem), I came across the following
> error msg in the boot log:
> 
> Setting up network device eth0
> doneSetting up IPv6 network interface (no address)done
> Setting up routing (using /etc/route.conf)
> Error while excuting:
>     /sbin/route add -net 203.XX.YYY.193 netmask 255.255.0.0
> gw 203.XX.YYY.193 dev eth0
>     route: netmask doesn't match route address

I would suggest two things:

1) network addresses are all 0's in the host part, so your address
should be 203.XX.0.0 if the netmask is correct. Alternately, if you
meant "-net 203.XX.YYY.192", then your netmask probably should have
been "255.255.255.192" or 255.255.255.224 or ...240 or ...248 or
...252, depending on the size of your subnet (that's /26, /27, /28,
/29 and /30 respectively -- meaning the number of bits in the network
address ... )

2) Your gateway appears to be on the network you are routing. The
gateway needs to be accessible -- on your current network or some
other accessible route. It is the "gateway" (forget the jargon, think
garden gate) to the network you are specifying. What you are saying
above is: "to get to this area, go via this gate", and if the gate is
only accessible from in the area, it won't work :)

2) 203.anything is a Class C network, so your netmask has to be at
least 255.255.255.0 ...
http://www.webopedia.com/DidYouKnow/Internet/2002/IPaddressing.html

Briefly, network classes, while being superseded by the CIDR concept,
are still important, and I have a feeling that any CIDR networks
(anything with other than 255 or 0 in the netmask) need to actually
subnet an old style network class -- you can't aggregate 16 class C's
into one /20 network AFAIK...

HTH
