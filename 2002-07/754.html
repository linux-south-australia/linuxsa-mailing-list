Reading "Smart Tags" from PHP

As much as it disgusts me, I'm doing some development at the moment
that requires me to consider the possibility that the user will be
using Microsoft Word as an authoring tool.

Yuck.

Microsoft, in its infinite wisdom, has decided that, no longer should
it put document information in the meta tags, but in this weird,
sort-of XML stuff in the <head> tag.

For example, the stuff that Word puts in the <head> tags of a document
that it has saved as HTML:

<xml>
 <o:DocumentProperties>
  <o:Subject>Code Management</o:Subject>
  <o:Author>Andrew Reid</o:Author>
  <o:Keywords>CVS Code Management Linux UNIX C C++ FreeBSD</o:Keywords>
  <o:LastAuthor>Andrew Reid</o:LastAuthor>
  <o:Revision>3</o:Revision>
  <o:TotalTime>25</o:TotalTime>
  <o:Created>2002-07-23T00:15:00Z</o:Created>
  <o:LastSaved>2002-07-23T00:16:00Z</o:LastSaved>
  <o:Pages>1</o:Pages>
  <o:Words>782</o:Words>
  <o:Characters>4461</o:Characters>
  <o:Category>Software Development</o:Category>
  <o:Lines>37</o:Lines>
  <o:Paragraphs>10</o:Paragraphs>
  <o:CharactersWithSpaces>5233</o:CharactersWithSpaces>
  <o:Version>10.2625</o:Version>
 </o:DocumentProperties>
</xml>

The fact that Word generated this is really neither here nor there.
I've just got to work out how I read these little gems, preferably
from PHP.

Now, I know there are some of you out there that'd probably work with
this stuff more often than I, and I daresay one of you might know how
the heck I suck this data out of the file.

If anyone is feeling particularly charitable and feels compelled to
donate some of their wisdom in this particular circumstance, I'd be
ever so greatful.

TIA,
