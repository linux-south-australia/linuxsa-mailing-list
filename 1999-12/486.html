Re: Network cards

Am I the only person here who thinks that spending $20 on a brand new
PCI network card is well worth it? More reliable, less headaches,
simpler installation, configuration, etc, etc.

Admitedly I paid a bit extra to get some 3com 10/100 cards, but i've got
budget 10baseT cards in most of my other boxes, and I'm pretty sure the
cards were sub $20 each. It all works fine, boot from windows to linux n
back again on those machines and neither has problems with PCI devices
changing IRQs. (surprisingly)

Toby

> This is probably an ugly hack:
> 
> Consider configuring each card from within your startup scripts.  Linux
> ethernet card configuration tools, to modify the settings in the EEPROM,
> are available from
>  http://cesdis.gsfc.nasa.gov/linux/diag/diagnostic.html
> There are tools for reconfiguring ne2000 and 3c5x9 cards.
> 
> If the reconfiguration requires a reboot, the card will have the wrong
> settings for the first boot.  Perhaps reboot if the network card module
> does not load, after reconfiguring the card and creating a flag file (to
> prevent reboot loops if it does not load for some other reason).
