Re: Proxy ARP prob

is there any particular reason you want to proxy arp?

i would imagine that a "regular" ppp link would not need one. However,
if you are using a ppp link to connect to networks both having
read-world IP's then, OK.

which one are you?

I have seen a box setup so that the ppp device had an ethernet address,
the command looks like this:
/sbin/arp --set 10.1.1.200 08:00:2B:94:DC:A5 pub

where 10.1.1.200 is the IP address of the PPP adapter

this command I belive is saying that "I know the MAC address of the IP
10.1.1.200".


not that this will probably solve your problem, but it may help.

I'm interested in any solution you may have.
