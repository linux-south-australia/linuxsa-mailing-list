Re: X Display

On Wed, Dec 08, 1999 at 09:19:14PM -0500, Subba Rao wrote:

 > I use XDM on my system. When I login and try to "su - localuser" in
 > a xterm, I get the following message:
 > /dev/ttypc: Operation not permitted

That's normal:  It's because /dev/ttypc is owned by root (because root
is logged in on it) and "localuser"'s .cshrc, .bashrc, .profile or
whatever is trying to do a "mesg y" or "mesg n".  The "mesg" command
does its stuff by chmod'ing its controlling terminal; you can only
chmod files you own, but since root owns the terminal (/dev/ttypc),
localuser can't chmod it.

The short answer is, "Ignore that message".
 
 > After I set and export the DISPLAY value to :0.0 and I try to call an
 > X application, I get,
 > Xlib: connection to ":0.0" refused by server
 > Xlib: Client is not authorized to connect to Server
 > Error: Can't open display: :0.0

That's because the magic secret in root's .Xauthority file can't
be seen by the localuser.

See the Xsecurity and xauth manpages ("man Xsecurity" and "man xauth")
to see how to fix this.

Note that although you can completely disable X security ("xhost +"),
this is definitely not recommended if you value the integrity of your
system.
