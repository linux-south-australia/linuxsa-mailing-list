Since you mention it's an Isa modem I'll assume it's not a winmodem (A
modem which makes the cpu do the work and the drivers specifications are
so secret that there is no drivers available for linux). You shouldn't
need to worry about an IO. 3e8 is normally com 3 under dos/windows. Which
would be /dev/ttyS2 under Linux.

Have you tried setting the IRQ using: setserial 

> 2. I have a cmi8330 onboard sound system which I am having problems

> for this when I was trying to set up the modem.
> I chose the following options

Since you have a 2.2 kernel have a look in:
/usr/src/linux/Documentation/sound/CMI8330 

It has an isapnp setup for you and instructions on how to get the sound
working.

> 4. While meandering through files trying to find more relevant

> proc/pci file as Appendix D below.
> Actually a few of the settings I expected in the following files i.e.
> printer on IRQ 7 dont appear to be present .

The 2.2 kernels by default don't use an irq for the printer port. They use
what is known as a polling mode. This is a bit slower printing but it
reduces a lot of overhead for the rest of the system. You can turn on the
irq but adding a line like: options parport_pc io=0x378 irq=7
to /etc/conf.modules
