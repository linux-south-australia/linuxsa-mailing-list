> Cool, I thought I was the only one who'd ever seen this problem. I had a
> problem with a legend computer with exactly this problem. It seems to be
> in the BIOS. Solution, replace the BIOS, replace the motherboard, or if
> you must boot on november 13th, press DELETE, reset the date to November
> 14th, boot then reset back once it is booted. Was this a Legend brand PC ?

While we don't know what machine this board originally resided in I
would almost be prepared to bet that it was the same board. 

Further tests reveal that replacing the BIOS chip stops the behaviour.
Obviously the tune resides in the BIOS chip. Place the BIOS chip in
another identical board and it functions normally. i.e. no tune. If
someone can read this chip and would like to have a go at decoding it I
sure it can be arranged.

The BIOS is an AMI BIOS 1993, version AB.
