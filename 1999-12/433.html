Re: Startup folder

is "startup" boot or login?

at boot time, a program called init runs stuff in /etc/rc.d/rc<n>.d/
where <n> is the runlevel -- usually either 3 (multiuser, no X) or 5
(multiuser with X) as specified in your /etc/inittab.

The files in the /etc/rc.d/rc<n>.d/ directory are symbolic links to
scripts elsewhere. These scripts control various services that run on
your system. They all start with S or K and a number, followed by some
text. The ones starting with S are started (using the command
"<scriptname> start") and the ones starting with K are stopped (using
the command "<scriptname> stop"). You can do this manually, if you
wanted...


On login, verious things get executed, in an order I'm not sure of. I
believe that it depends on your choice of shell and window manager.

Generally, there's scripts with names like ~/.profile, ~/.bash_profile,
~/.login, ~/.cshrc, /etc/profile, /etc/bashrc that get executed in some
order that is important, but I can never remember it.

Also, various window manager ones get executed (~/.twmrc, ~/.fvwmrc. etc
etc). Depending on the particular window manager, these could be scripts
to be executed or merely configuration files that are read and
interpreted by the wm.
