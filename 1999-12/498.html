Re: PPP over null modem between 2 machines (dos and linux)

On Wed, 22 Dec 1999, Ant wrote:

> I recently purchased a cheap laptop in order to put linux onto it. 
> Problem is it's only got a 520mb hdd and no cd.  I do have a xircom 10mb
> PCMCIA card that works under dos, but I decided I'd learn more by doing
> an installation over a null modem cable using pppd and then smb shares
> to a windows machine with the cd. 
>  I have been playing around with this, and read the serial how to, and
> also the modem how to, but they dont go into much detail about how it
> works with NO modem.  What I need too know is what, for my purposes, is
> a good getty to use, and also what the best way for the linux machine to
> sense an incoming connection is - since there is no connect string (do
> you set it to wait for an enter, or is there another way?).  

A null modem cable is VERY slow. In my experiance it seems to transfer 1MB
in about 30minutes. Try using a null printer cable. (same as a null modem
cable just uses the printer port and can transfer 1MB in about 3 minutes)

The PLIP howto will help you work out how to set one of these cables up.
