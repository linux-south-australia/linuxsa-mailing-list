Re: Redhat boot process....

MWP wrote:

> From what I can gather, "init" does most of the work late during booting...
> so can I somehow make init run my script rather than displaying the logon
> screen??

Well, init is the first "bona-fide" process that is executed by a Unix
kernel when it boots; you could either modify /etc/inittab (used by init)
to execute the processes you want (this is what runs your run level scripts,
and would be trivial to set up), or rewrite init yourself.  Note that in
the "old days", init used to be part of the kernel itself, but (danger:
wild speculation ahead) _I think_ that it is now a seperate statically-
linked program (end of wild speculation).
