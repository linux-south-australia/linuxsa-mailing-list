Re: To Linux gurus! 2nd Time.

If your kernel ends up too big, all you need to do is instead of running
'make zImage' run 'make bzImage' and copy the file called bzImage as your
new kernel to the /boot directory. This should work as it has before for
me when I got a kernel too big error...
