Email Service Monitoring...

Hi,

We have a situation in our office where our Exchange (<shudder>) server
sometimes decides not to send outbound email, but inbound mail works fine...
What I'd like to do is, set up a monitoring system which can alert me if the
mail isn't getting out...  Basically what I plan to do is use a script on
out NT Server to send an email to a specific account on my Linux box every
hour.

When the message arrives on the Linux box, I can use a sendmail alias to
deliver the message to a shell script (effectively allowing me to run a
shell script on receipt of an email message), and I can record the current
system time into a file using something like this...

echo `date` > /tmp/lastrectime

So, if our exchange server is sending mail, we'll have the time that it last
sent us a "ping" message in a file in /tmp/lastrectime ...  What I'd then
like to do, is have a cron job run hourly and check the value in
/tmp/lastrectime.  If its less than 2 hours old, then everything is cool.
If its more than 2 hours old, then we'll need to send an email to my SMS
service telling me all about it.  

That's the plan.... and I can do all of it no worries, except comparing
current time and the time in /tmp/lastrectime to see if its more than 2
hours old...  

Anyone got any ideas on how I can do the time comparison part?    Is there
an easier way to do what I'm proposing?

Jim.
