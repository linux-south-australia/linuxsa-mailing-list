Re: X :1 -query 'pluto'

The GNOME before october-gnome is broken. RedHat 6.0 (vanilla) had this
bug, so download october-gnome. The basics (as I understand it) is that
gnome decides that the X server belongs to someone or something else and
refuses authentication to the user on the console that you are talking
about. If you go back to console you'll probably see a MIT-MAGIC-COOKIE
error or failure.

Before downloading october-gnome (from a reputable mirror site such as
ftp://ftp.mirror.aarnet.edu.au), there are a number of steps you can
take to attempt to avoid this problem:

1) NEVER log in as root to X..always start and Xterm and su
2) ALWAYS start gnome/X from the same place
3) NEVER save a session; this seems to totally confuse the older version
of gnome

When the grey screen comes up simply kill the X server forcibly
(ctrl+alt+backspace) and start it again. Incidentally, remind everyone
that xhosts doesn't work without an x server running and that xauth is
about as understandable as hebrew is to someone who can't read it;
besides that startx scripts set up authentication properly - it's the
actual compiled code which goes west....
