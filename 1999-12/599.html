Dial on demand is still an option, but the demand is "user initiated". The
"diald" daemon program allows you to use a "special" fifo file to control
the dialling, so you can start and stop the link with commands like:

echo up > /etc/diald.fifo
echo down > /etc/diald.fifo

respectively. On my system diald is configured never to start the link up
except through fifo commands. Although I do not use a cgi program to
initiate it (just via command line), I see no reason why this should not be
possible and relatively easy to do.

> What would be ideal is a cgi-type program that runs on the
> Linux web server and can start the connection when the user
> clicks a link to it from a HTML page.
>
> I have tried a couple of programs with almost no success.
>
> Either the Linux "server" program crashed or would not bring
> up the link correctly, or the Windows client would not
> talk to the Linux box to initiate the connection.
>
> Can somebody recommend a GOOD, and relatively easy to configure
> program that will allow this.

Diald is "good", but "easy" depends on your tcp and *nix experience. I would
say it is of medium difficulty to get going properly, and will require the
usual several hours fiddling to get it working as you would expect.
