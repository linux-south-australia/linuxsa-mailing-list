Re: second ethernet card not communicating

the fact that ifconfig is not reporting errors means that the kernel is
recognising the card fine.

a quick way to test is ifconfig eth1, and if you get something like
this:
eth1      Link encap:Ethernet  HWaddr 00:00:B4:82:23:0D
          inet addr:10.1.1.1  Bcast:10.1.1.255  Mask:255.255.255.0
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:7080085 errors:0 dropped:0 overruns:0 frame:0
          TX packets:3068164 errors:0 dropped:0 overruns:0 carrier:0
          collisions:2409 txqueuelen:100
          Interrupt:10 Base address:0x300

then your ok.

what we really want to know is (insert guitar here) are ya gonna go my
way.

no, really we need to no if eth1 is configured. you say you can ping it,
is that 127.0.0.1? 'cos thats the loop back.

this is starting to sound more like a routing problem.....
