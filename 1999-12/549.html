>Simply, one server one login name is the way Windows networking
>works. Solutions, mount share from another box, or give user one
>permissions to read / read write user 2's directory.

Hmmm, I am not so sure of this :-)

NT has Logon As in the Map Network Drive dialog box, so you can access
network drives as different users.

I think that the problem is a fundamental misunderstanding about Samba.
When you have a [homes] section, you do not need, nor do you want, sections
for individual users.

When mapping a drive (accessing a share), Samba first looks for a section
in the smb.conf file with the name you have specified.  If that is not
found, it looks for a homes section and checks to see if the share name you
requested matches a username, and if that fails, it looks for a printers
section.
