RE: CHMOD

Quote from the doc -

==
One uses the `+' or '-' action symbols to add and/or subtract file
permissions. Here is how one does it:

   1.Specify a category of users with the abbreviations for the categories
(u, g or o). A special category 'a' also exists which
     means all users. Nb. One can add these together like ug, which means
the user and the group
   2.Specify an appropriate action symbol ('+' or '-')
   3.Specify a category of file permission (r, w, or x). Nb. As with the
specification for users, one can add these together
     like rw, which means read and write permissions

Here are two examples of how to use chmod on a file called `test'; for our
purposes we will assume that `test' has absolutely no
permissions at the start:

   1.chmod ug+rx test This gives read and execute permissions to user and
group
   2.chmod a-x test This removes execute permissions from all users
==

David


Has one missed the '=' operator?  One believe it's possible to set exact,
explicit permissions with it.  Perhaps one might consider adding it to one's
page?
