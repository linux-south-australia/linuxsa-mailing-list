RE: smclient

>I have a laptop running mandrake 8.2. The following command 
>will browse a
>computer for shares:
>
>$smbclient -L computer_name
>
>...however, after entering this command to browns shares on a 
>server (which
>is part of a windows domain, windows2k i am fairly sure) i am 
>asked for a
>password (no user name is requested) and skipping this (entering no
>password) will cause samba client to NOT see any shares on that server.

It is trying to connect using your current username and when you hit
enter at the password prompt it then tries to connect as guest as far
as I know.  (Sounds like guest access is disabled on the windows server,
good move, someone actually thought about security on their server :)

Basically you'll need to specify a valid user on the windows server 
in your smbclient command, "smbclient -L computername -U username

Of course if you want to become part of the domain rather than relying on
user level access with smbclient then you'll need a machine account on the 
PDC and a valid username and password, then join the domain 
with "smbpasswd -j domain-name".

The howtos & man pages for samba a really quite good and cover any caveats 
that you're likely to come across so keep reading
