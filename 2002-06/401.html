Stumped: Partition woes

Greetings...

Has anyone come across the situation where the standard Partition check at
boot up gives the warning "Invalid Partition Table" ?  It looks like this

<snip boot info>
Partition Check:
hda: hda1  hda2 <hda5 hda6 hda7 hda8>
hdb: "Invalid Partition Table"
<snip boot info>

I added a 80Gb drive to my machine last week, cfdisk'd it into one large
primary partition, ran mke2fs over it, added it to fstab and the first time
it booted I got the error message above when I checked dmesg.   The drive
for all intents and purposes works, mounts fine, checks out with e2fsck so
the fs is fine just the partition table is screwy....  but what and how do
I check ?   The only thing I can think of is I did not set the primary
partition on this drive bootable ?   But this shouldn't matter as it is a
second drive and the system boots of another drive.   Here is my current
file system

/dev/hda 40G (bootable)
/dev/hdb 80G

hda1  /     defaults
hda5  /usr
hda6  /tmp
hda7  /log
hda8  /home

hdb   /home/samba

I am currently using a 2.4.16 kernel (waiting for 2.4.19) with a Slackware
8.0 system.   While the system appears to be working the error message has
me wondering if something is possibly amiss ?

Comments ?
