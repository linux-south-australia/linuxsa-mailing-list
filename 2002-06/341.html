I think you answered exactly what he asked for, but perhaps not what he
wanted :)

The data will come back in to the source IP used to make the request. If
it's just the linux box itself making the requests (ie: a proxy server),
you may be in luck, as it should use the IP address of the interface it
makes the connection on.

If you have a routed subnet, it's going to be difficult.

If you have NAT/masq, there may be some homework to do to get it working
how you want it to with 2 seperate links.
