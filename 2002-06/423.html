Re: Secure POP

Martin Stacey wrote:
> How can I get this running on my Postfix mail server?

Hi Martin,

The IETF mail architecture differs from what you may expect --
messages are send and read via completely different mechanisms.

Postfix is used to send messages.  It can't be used to retrieve
messages from you mail storage.

POP servers are used to access your mail storage (IMAP is another
protocol used for this).  Popular POP servers, such as the UW POPd
server, support an SSL-secured version of POP.  For example, Red
Hat Linux ships with UW POPd already configured for SSL, all you
need to do is create the key certificates, as described in the
RH documentation.

There is also an old security feature called "POP before SMTP"
where you need to connect to a POP server and supply a valid
password before your IP address is allowed to send SMTP e-mail.

Don't bother with this anymore.  Configure postfix for TLS and
user authentication instead.
