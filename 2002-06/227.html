Re: Innovation

On Wed, 5 Jun 2002, michaeld@senet.com.au wrote:
> Quoting Martin Sandiford <ms@mcdev.com.au>:
>> 
>> Maybe 99.99999% (OK, slight exaggeration here) of all TCP/IP stacks
>> are based on the BSD implementation.  This is why they all have the
>> same bugs :) Microsoft didn't start shipping an OS with a native
>> TCP/IP stack until circa 1993 IIRC.
> 
> Umm, isn't the Linux TCP/IP stack a clean room implementation, last
> rewritten by Alan Cox while he was at Swansea University?

Sure.  Maybe we differ in what we mean by "based on".

I don't know for sure how many TCP/IP stacks have used BSD source
directly in their implementation, but many have certainly (and this
includes the Linux stack) been modeled after BSD behaviour.  One can see
this by navigating into /usr/src/linux/net/ipv4 and doing a grep for
"BSD".

External evidence (say the long ping bug, and various defragmentation
bugs) would also suggest a number of internal similarities in packet
handling between a number of systems.
