Indeed, there isn't.

I've developed a Net Café management system in perl/php you throw on top of
apache; theres ipchains, iptables, ipf, and ipfw modules for it.  (In other
words, it should run on most versions of Linux and BSD).

I'm at a café in sydney at the moment actually currently working on it.
There's two modes - per machine ip accounting, which is basically "click"
machine on, routing enabled, and usage is times, "click" machine off, and
amount is calculated.  The other mode links with samba, and cards for logons
can be randomly generated via a separate interface; users then logon to the
NT/2K/XP workstations, and there's a local client which times usage which is
capable of logoff's.  It does plenty more too, but I wont go into details.

Still a lot of bugs; working these out as we speak.

Not sure whether I'm going to sourceforge this or not yet;  the way I'm
considering on going is to have a "Lite" and a "Pro" version, the pro you
pay for, the Lite released under a BSD-type license.

Email me back if you're interested.
