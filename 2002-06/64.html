Well, if you're still getting the "not well-formed [snip] on line 24" 
error, it menas you've effectively got an XML syntax error. Specifying a 
DTD or using XML namespaces won't help because you're not even getting 
to the stage where you may be able to validate the document.

A well-formed XML document is something like syntactically correct code; 
you've got all the braces and semicolons in the right places, etc. A 
valid XML document is a like semantically correct code; you've got 
function calls spelt correctly, etc, etc.

However, after having downloaded the XML and XSLT, run them both through 
xmllint and Xalan to get some useful generated output, I'd say both your 
XML and XSLT are fine. And the odds of Sablotron being buggy is unlikely.

So I'd say that your problem lies elsewhere. Taking a guess, are you 
feeding Sablotron the wrong file? Given that the message says the error 
is in "/usr/local/www/gihsnet/shp/index.php", and that the URL for the 
generated XML is ".../shp/xml/?username=andrew" (note the "xml/" between 
the "shp/" and end of the URL) it sounds like you are trying to get 
Sablotron parsing your PHP code, not your generated XML.
