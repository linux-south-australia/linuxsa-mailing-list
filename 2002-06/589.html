Re: Backing up a Hard Drive

On Sat, Jun 22, 2002 at 08:19:45AM +0930, michael wrote:
> Hi.
> 
> Something I have never found the answer for:
> 
> If I dd a 5 gig drive onto a 20 gig drive, the 20 gig will 'lose' the
> extra space, right? Is there any way of recovering it?

Kinda...

if you dd the whole drive (hda), then the new drive will have the
exact same partitions as the old one, which will, of course, add up to
no more than 5G... If you had a spare primary partion on the old
drive, then you may be able to simply run fdisk (or sfdisk -- it tends
to be a little better than fdisk, though a little uglier), and define
that partition to use the rest of the disk... however, I'm not 100%
sure about that. As someone else has said, parted may be able to do
more for you...

As far as filesystem sizes go, dd will simply recreate the exact same
filesystem... so you'll need to use some sort of resizer to resize the
fs, if that's what you want -- or you can play musical partitions, and
use a file-level tool (cp -a, tar, etc) to transfer the files on a
small partition to a larger one...

I wonder if there's a filesystem equivalent to the ISO network layer
model... so we could talk about layer 1 being physical, layer 2 being
bits, layer 3 being blocks, layer 4 being inodes, layer 5 being the
filesystem interface, layer 6 being compression and encryption, and
layer 7 being file formats... hmm... In that (probably poory thought
through) model, dd works at either layer 2 or layer 3 (not sure),
dump works at layer 4, tar and cp work at layer 5, gzip etc at layer
6, and so on... hmm...
