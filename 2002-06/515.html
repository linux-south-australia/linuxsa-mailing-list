RE: spam, application/octet-stream

Daniel Mundy wrote:

> I have been getting spam in my inbox with a Content-Type of
> "application/octet-stream",

This is probably an executable - most attachments that are binary
data or executable can validly use this mime type.  Bear in mind
that it's the default MIME Type of a lot of attachments (such as
Excel spreadsheets, etc.).

> My first instinct was to filter out any mail with content type
> of application/octet-stream, but I couldn't end up getting this
> to work..

Yeah, don't do that.  It's quite legitimate to mention the phrase:
Content-type: application/octet-stream in an email message and
it not be considered the mime-type of a multipart (except in poorly
written mail products).  You actually need to parse the email to
determine whether the above was "legal" or not.  If you're intent
on doing this with procmail, it'd probably pay to run an ancillary
script that uses the metamail tools to determine accurately the
mime types of all the components of the email.

> I was wondering if anyone else had thoughts on this matter?

There was a thread about this on LinuxSA half a year ago or so -
it'd pay to read that regarding the above parsing problem.
