Re: Innovation

Toby Corkindale wrote:

> In the case of reiserfs3 (the
> one we get in the kernel), its also using balanced b-trees to store and
> locate inodes,

As used in OS/2's HPFS, and possibly BeFS (memory's a little flaky
on that last one).  I don't think it was new even in OS/2's time.
(In fact, HPFS looks suspiciously like NTFS).

> and small files don't have seperate metadata inodes.

As used in OS/9 68K V2.x file system (circa late 80's).  NTFS does
something similar to this - for small amounts of data, the data is
kept with the inode equivalent.  When Microsoft introduced file
system quotas (how innovative!), it didn't take this data into
account, which apparently led to a DoS condition by creating a
large number of small files that never impacted on your quota.

Everything new is old again - or something like that.
