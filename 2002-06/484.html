Re: Interesting DNS "trick"

On Wed, 19 Jun 2002, andrew.speer@isolutions.com.au wrote:
> Whilst researching split zones (or views) in bind 9, I came across an
> interesting "trick" (as in: any command with paramaters sufficiently
> advanced is indinguishable from magic) in this command:
> 
> nslookup -q=txt -class=CHAOS version.bind. 0.0.0.0 <-- or any name
> server IP/name here

You can put whatever you like in this response via the "version"
configuration option in the "options" section of named.conf.

I don't believe there is an option to do anything with the hostname, but
it's difficult for me to imagine how this could be a security issue for
a properly configured machine.  If the machine is reachable from the
Internet and I have the IP address, I could look up the in-addr.arpa
entry for it and presumably get the same thing.  If not, then maybe the
machines configuration needs some attention?
