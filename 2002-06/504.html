Re: Procmail destination folder for Spam Assassin

On Wed, 19 Jun 2002, James Leone wrote:

> Well thanks to all your help (especially Simon Hackett) I have spam
> assassin working on my local workstation.
>
> I use fetchmail and the  /home/james/.forward file to open procmail.
>
> As I understand it, procmail is configured through
> /home/james/.procmailrc, and that the variable MAILDIR=$HOME/mail should
> create or append my inbox file (which I would assume would be named
> "inbox") in the /home/james/mail directory.
>
> However, all it does for me is put my mail in the /var/spool/mail folder
> and the file is called "james" instead of "inbox"
>
> I am using SuSE Linux 8 Professional, running amavis, sendmail,
> fetchmail and procmail.

procmail may not even be called.

Put:

# maximum verbosity
VERBOSE=on

# logfile name
LOGFILE=$HOME/procmail.log

at the top of the .procmailrc file and send a few more emails in. If
~/procmail.log doesn't spring into existence, procmail probably isn't in
the picture :-)

Check the procmail man page, look under the heading 'NOTES'.
