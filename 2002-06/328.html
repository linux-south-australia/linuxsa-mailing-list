replacing bash (/bin/sh) with a real sh

Hi

Red Hat systems link /bin/sh to bash, so when I write a shell script
that should be interpreted by a simple Bourne shell (by beginning the
script with #!/bin/sh), the script is really interpreted by the
Bourne again shell, which has more features than the original Bourne
shell, so I don't know if my scripts will be compatible with systems
that don't use bash as /bin/sh (most other Unix systems I've seen).

(bash is supposed to have a sh or POSIX compatibility mode, but
whatever I get with #!/bin/sh isn't quite the same as sh.)

Has anybody attempted replacing /bin/sh with a true sh (or even a
somewhat lighter or POSIX standard shell such as ash or ksh) on a
Red Hat system?  Did you encounter any problems?
