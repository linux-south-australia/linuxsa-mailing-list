Yes,
    However,
        HardDrake sets it up automatically during mandrake install
        HardDrake can set this up for you post install

So either way you don't need to know much about the creation of scsi 
emulation.
Basically if you have an existing drive the new drive will be available as

/mnt/cdrom2  which is link to  /dev/scd0

HardDrake setup fixes the devfs system as required and appends lilo with 
the following line

append quiet devfs=mount hde=ide-scsi
(alter hde to suit your system)

Anyway this is just reference as you shouldn't need to do any of the 
alterations yourself.

dcd
Note: most burning programs prefer to address your atapi cdrom if youv'e 
left one in the system as the /dev/hd? address
many programs now don't address the atapi cdrom at all, unless you set 
them up as scsi and that you will have to do manually.
Gcombust, and Gnome Toaster both allow you to address the Atapi cdrom, 
and work in either KDE or Gnome on Mandrake 8.2
